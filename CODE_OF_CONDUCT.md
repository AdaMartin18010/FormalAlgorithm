# 行为准则 / Code of Conduct

## 我们的承诺 / Our Pledge

我们作为成员、贡献者和维护者，致力于为我们的项目社区创造一个无骚扰的参与体验，无论年龄、体型、可见或不可见的残疾、种族、性别特征、性别认同和表达、经验水平、教育程度、社会经济地位、国籍、个人外貌、种族、宗教或性取向如何。

We, as members, contributors, and maintainers, pledge to make participation in our project and our community a harassment-free experience for everyone, regardless of age, body size, visible or invisible disability, ethnicity, gender characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual orientation.

## 我们的标准 / Our Standards

### 鼓励的行为 / Examples of Behavior That Contributes to a Positive Environment

- 使用友好和包容的语言 / Using welcoming and inclusive language
- 尊重不同的观点和经验 / Being respectful of differing viewpoints and experiences
- 优雅地接受建设性批评 / Gracefully accepting constructive criticism
- 关注社区最大利益 / Focusing on what is best for the community
- 对其他社区成员表示同理心 / Showing empathy towards other community members
- 帮助他人学习和成长 / Helping others learn and grow
- 分享知识和经验 / Sharing knowledge and experience

### 不可接受的行为 / Examples of Unacceptable Behavior

- 使用歧视性语言或行为 / The use of discriminatory language or behavior
- 人身攻击或侮辱性评论 / Trolling, insulting/derogatory comments
- 公开或私下的骚扰 / Public or private harassment
- 未经明确许可发布他人私人信息 / Publishing others' private information without permission
- 其他不道德或不专业的行为 / Other conduct which could reasonably be considered inappropriate

## 执行责任 / Enforcement Responsibilities

项目维护者有责任澄清和执行本行为准则中定义的可接受行为标准，并将采取适当和公平的纠正措施来回应任何他们认为不适当、威胁、冒犯或有害的行为。

Project maintainers are responsible for clarifying and enforcing standards of acceptable behavior and will take appropriate and fair corrective action in response to any behavior that they deem inappropriate, threatening, offensive, or harmful.

## 适用范围 / Scope

本行为准则适用于所有项目空间，包括：

This Code of Conduct applies within all project spaces, including:

- GitHub Issues 和 Pull Requests
- GitHub Discussions
- 邮件列表和通讯群组
- 项目相关的会议和活动
- 项目相关的社交媒体
- 代表项目时的公开场合

## 报告与处理 / Reporting and Enforcement

### 如何报告 / How to Report

如果您遭遇或目击到不可接受的行为，请通过以下方式报告：

If you experience or witness unacceptable behavior, please report it via:

- **GitHub**: 发送私信给 [项目维护者](../../people)
- **Email**: 如需要私密沟通，请在GitHub上说明

### 报告内容 / What to Include

您的报告应包含：

- 您的联系信息
- 事件的描述
- 发生的时间和地点
- 相关人员
- 任何相关证据（截图、链接等）

### 处理流程 / Enforcement Process

1. **审查**：维护者团队将在48小时内审查报告
2. **调查**：必要时进行调查
3. **决定**：根据情况采取适当措施
4. **反馈**：向报告者反馈处理结果

### 可能的措施 / Possible Actions

- 警告
- 临时禁言
- 永久禁止参与
- 其他适当的措施

## 行为准则委员会 / Code of Conduct Committee

### 成员 / Members

- 项目负责人
- 核心维护者代表
- 社区代表（每年选举）

### 职责 / Responsibilities

- 审查和更新行为准则
- 处理复杂的报告
- 确保执行的公平性

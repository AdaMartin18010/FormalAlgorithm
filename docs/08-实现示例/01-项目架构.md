## 项目结构

```
examples/algorithms/
├── src/
│   ├── lib.rs                    # 库入口
│   ├── sorting.rs               # 排序算法集合
│   ├── merge_sort.rs            # 归并排序
│   ├── quick_sort.rs            # 快速排序
│   ├── heap_sort.rs             # 堆排序
│   ├── binary_search.rs         # 二分搜索
│   ├── lcs.rs                   # 最长公共子序列
│   ├── graph_bfs_dfs.rs        # 图遍历
│   ├── dijkstra.rs              # 单源最短路径
│   ├── graph_mst.rs             # 最小生成树
│   ├── segment_tree.rs          # 线段树
│   ├── trie.rs                  # 前缀树
│   ├── divide_and_conquer.rs    # 分治框架
│   ├── greedy_activity_selection.rs  # 贪心算法
│   ├── backtracking.rs          # 回溯
│   ├── backtracking_nqueens.rs  # N皇后
│   ├── fft.rs                   # 快速傅里叶变换
│   ├── string_algorithms/       # 字符串算法
│   │   ├── kmp.rs               # KMP匹配
│   │   └── edit_distance.rs     # 编辑距离
│   ├── network_flow/            # 网络流
│   │   └── max_flow.rs          # 最大流
│   ├── computational_geometry/  # 计算几何
│   │   └── convex_hull.rs       # 凸包
│   └── number_theory/           # 数论
│       └── mod.rs               # GCD/素数测试
├── sorting.go                   # Go排序
├── sorting.py                   # Python排序
└── data_structures.c            # C数据结构
```

## 代码规范

### Rust规范

- 使用 `cargo clippy` 检查
- 文档注释使用 `///`
- 复杂度注释格式: `O(?)`

### 测试要求

- 每个模块包含 `#[cfg(test)]` 单元测试
- 主要路径覆盖率 > 80%

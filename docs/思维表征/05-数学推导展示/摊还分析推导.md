# 摊还分析 (Amortized Analysis) 推导

## 三种分析方法概述

| 方法 | 核心思想 | 适用场景 |
|-----|---------|---------|
| **聚合分析** | 计算n个操作总代价，除以n | 同类型操作序列 |
| **记账方法** | 为每个操作分配摊还代价 | 需要跟踪信用 |
| **势能方法** | 定义势能函数，摊还=实际+势能变化 | 复杂数据结构 |

---

## 一、聚合分析 (Aggregate Analysis)

### 示例：栈的 MULTIPOP 操作

**操作定义：**
- PUSH(S, x)：压栈，代价 = 1
- POP(S)：弹栈，代价 = 1
- MULTIPOP(S, k)：弹出k个元素（或直到空），代价 = min(|S|, k)

**问题：** MULTIPOP 最坏情况代价为 O(n)，n个操作的序列最坏情况是否为 O(n²)？

**分析：**

```
关键观察: 每个元素最多被弹出一次
─────────────────────────────────

n个操作序列中:
- PUSH 操作最多 n 次
- 每个被压入的元素最多被弹出一次
- 总弹出次数 ≤ 总压入次数 ≤ n

因此，n个操作的总代价:
T(n) ≤ n次PUSH + n次POP = 2n = O(n)

摊还代价 = T(n)/n = O(1)
```

### 示例：二进制计数器

**操作：** INCREMENT，将二进制计数器加1

```
位翻转次数分析:
─────────────────────────────────

INCREMENT 每次翻转的位数:
- 第0位: 每次INCREMENT都翻转 → n次
- 第1位: 每2次INCREMENT翻转 → ⌊n/2⌋次
- 第2位: 每4次INCREMENT翻转 → ⌊n/4⌋次
- ...
- 第i位: 每2ⁱ次翻转 → ⌊n/2ⁱ⌋次

总翻转次数:
T(n) = Σ(i=0 to ⌊log n⌋) ⌊n/2ⁱ⌋
     < n · Σ(i=0 to ∞) 1/2ⁱ
     = n · 2
     = 2n
     = O(n)

n次INCREMENT的摊还代价 = O(n)/n = O(1)
```

## 二、记账方法 (Accounting Method)

### 核心思想

为每个操作分配**摊还代价** $\hat{c}_i$，可能高于或低于实际代价 $c_i$。

**信用不变式：** 数据结构上的总信用 ≥ 0

$$\sum_{i=1}^{n} \hat{c}_i - \sum_{i=1}^{n} c_i \geq 0$$

### 示例：栈操作

**操作代价设定：**

| 操作 | 实际代价 | 摊还代价 | 信用变化 |
|-----|---------|---------|---------|
| PUSH | 1 | 2 | +1 (存储在元素上) |
| POP | 1 | 0 | -1 (使用元素信用) |
| MULTIPOP | k | 0 | -k (使用k个信用) |

**验证：**

```
信用不变式验证:
─────────────────────────────────

每个元素有1单位信用:
- PUSH: 支付1实际 + 存储1信用 = 摊还2
- POP: 使用元素上存储的1信用 = 摊还0
- MULTIPOP(k): 使用k个元素上各1信用 = 摊还0

关键点: 
- 元素只有被PUSH后才可能被POP
- 因此POP时元素上必有信用
- 信用不变式始终满足

n个操作的总摊还代价 = O(n)
→ 总实际代价 = O(n)
```

### 示例：动态表 (Dynamic Table)

**操作：** TABLE-INSERT

**策略：**
- 表满时扩展为两倍大小
- 插入代价 = 1（元素插入）+ 可能的复制代价

**摊还分析：**

```
设定摊还代价:
─────────────────────────────────
- 每次INSERT摊还代价 = 3
- 1单位: 当前插入
- 1单位: 将来复制自己
- 1单位: 将来复制一个已有元素

验证:
─────────────────────────────────

假设表大小为m，已有m/2个元素（刚扩展后）:
- 每次INSERT存储2单位信用
- 经过m/2次INSERT，表满，存储了m单位信用
- 扩展时需要复制m个元素
- 存储的m单位信用正好支付复制代价!

因此摊还代价 = O(1)
```

## 三、势能方法 (Potential Method)

### 核心思想

定义**势能函数** $\Phi: D \rightarrow \mathbb{R}$，其中 $D$ 是数据结构状态。

**摊还代价公式：**

$$\hat{c}_i = c_i + \Phi(D_i) - \Phi(D_{i-1}) = c_i + \Delta\Phi$$

**要求：**
- $\Phi(D_0) = 0$（初始状态）
- $\Phi(D_i) \geq 0$ 对所有 $i$（非负势能）

### n个操作的总摊还代价

```
Σ ĉᵢ = Σ(cᵢ + Φ(Dᵢ) - Φ(Dᵢ₋₁))
     = Σcᵢ + Φ(Dₙ) - Φ(D₀)
     = Σcᵢ + Φ(Dₙ)
     ≥ Σcᵢ                     [因为 Φ(Dₙ) ≥ 0]

因此: 总摊还代价是总实际代价的上界
```

### 示例：栈操作

**势能函数定义：**

$$\Phi(S) = \text{栈中元素个数}$$

**分析：**

```
初始: Φ(D₀) = 0 (空栈) ✓
非负: |S| ≥ 0 恒成立 ✓

操作分析:
─────────────────────────────────

PUSH(S, x):
- 实际代价 c = 1
- ΔΦ = |S'| - |S| = 1
- 摊还代价 ĉ = 1 + 1 = 2

POP(S):
- 实际代价 c = 1
- ΔΦ = |S'| - |S| = -1
- 摊还代价 ĉ = 1 - 1 = 0

MULTIPOP(S, k), k' = min(|S|, k):
- 实际代价 c = k'
- ΔΦ = -k'
- 摊还代价 ĉ = k' - k' = 0

每种操作的摊还代价都是 O(1)
```

### 示例：动态表

**势能函数定义：**

$$\Phi(T) = \begin{cases} 2 \cdot num - size & \text{if } \alpha \geq 1/2 \\ size/2 - num & \text{if } \alpha < 1/2 \end{cases}$$

其中 $\alpha = num/size$ 是装载因子。

**直观：**
- $\alpha \geq 1/2$：表半满以上，势能为2×(元素数-半量)
- $\alpha < 1/2$：表半满以下，势能随元素减少而增加

**分析INSERT：**

**情况1：** 不扩展（$num < size$）

```
- 实际代价 c = 1
- 若 α ≥ 1/2: Φᵢ = 2(num+1) - size, Φᵢ₋₁ = 2·num - size
  ΔΦ = 2, ĉ = 1 + 2 = 3
  
- 若 α < 1/2: 但num < size/2，INSERT后仍< size/2 + 1
  需要仔细计算，但 ĉ ≤ 3
```

**情况2：** 扩展（$num = size$）

```
- 扩展前: size = num, Φᵢ₋₁ = 2·num - num = num
- 扩展后: size' = 2·size = 2·num, num' = num + 1
  Φᵢ = 2(num+1) - 2num = 2
  
- 实际代价 c = num + 1（复制num个 + 插入1个）
- ΔΦ = 2 - num
- 摊还代价 ĉ = (num+1) + (2-num) = 3
```

**结论：** INSERT的摊还代价 ≤ 3 = O(1)

## 方法对比总结

```
聚合分析 ─────────────────────────────────────────
优点: 简单直观，不需要设计复杂机制
缺点: 只适用于同类型操作
公式: T(n)/n

记账方法 ─────────────────────────────────────────
优点: 直观，跟踪每个操作的信用
缺点: 需要创造性地设计信用分配
公式: ĉᵢ = cᵢ + creditᵢ

势能方法 ─────────────────────────────────────────
优点: 形式化强，适用于复杂数据结构
缺点: 寻找合适的势能函数有挑战性
公式: ĉᵢ = cᵢ + Φ(Dᵢ) - Φ(Dᵢ₋₁)
```

## 应用实例：斐波那契堆

**操作摊还复杂度：**

| 操作 | 实际最坏情况 | 摊还复杂度 |
|-----|-------------|-----------|
| MAKE-HEAP | O(1) | O(1) |
| INSERT | O(1) | O(1) |
| MINIMUM | O(1) | O(1) |
| EXTRACT-MIN | O(n) | O(log n) |
| UNION | O(1) | O(1) |
| DECREASE-KEY | O(n) | O(1) |
| DELETE | O(n) | O(log n) |

**势能函数：**

$$\Phi(H) = trees(H) + 2 \cdot marks(H)$$

其中：
- $trees(H)$：堆中树的棵数
- $marks(H)$：被标记的节点数

**关键：** DECREASE-KEY 的 O(1) 摊还复杂度使得 Dijkstra 算法优化到 $O(V \log V + E)$。

## 比较排序的下界

### 问题描述

给定 n 个元素的数组，通过**比较操作**确定它们的顺序。

### 决策树模型

```
关键观察:
─────────────────────────────────────────
任何比较排序算法都可以表示为决策树:
- 每个内部节点: 一次比较 aᵢ ≤ aⱼ?
- 每个叶子节点: 一种排序结果 (排列)
- 从根到叶子的路径: 算法执行路径

示例 (n=3):
                    a₁ ≤ a₂?
                   /        \
                  是          否
                 /            \
              a₂ ≤ a₃?      a₂ ≤ a₃?
              /    \        /    \
             是     否      是     否
            /       \      /       \
        a₁≤a₃?    a₁≤a₃? ...      ...
        /  \      /  \
       是   否    是   否
       |    |    |    |
     [1,2,3] ...  ...
```

### 下界推导

```
决策树性质:
─────────────────────────────────────────
1. 叶子节点数 ≥ n! (所有可能的排列)

2. 树高 h ≥ log(叶子数) ≥ log(n!)

3. 最坏情况比较次数 = 树高

计算 log(n!):
─────────────────────────────────────────
log(n!) = log(1) + log(2) + ... + log(n)

        = Σ(i=1 to n) log(i)

        ≥ Σ(i=⌈n/2⌉ to n) log(i)

        ≥ (n/2) · log(n/2)

        = (n/2) · (log n - 1)

        = (n log n)/2 - n/2

        = Ω(n log n)

更精确的斯特林公式:
n! ≈ √(2πn) · (n/e)ⁿ

log(n!) = n log n - n log e + O(log n)
        = n log n - 1.44n + O(log n)

因此: 比较排序的下界是 Ω(n log n)
```

## 搜索无序数组的下界

### 对手论证

```
问题: 在n个元素的数组中搜索x

对手策略:
─────────────────────────────────────────
对手维护一个候选集 S (x可能的位置)

初始: S = {1, 2, ..., n}

当算法查询 A[i] ?= x 时:
- 若 |S| > 1: 对手回答 "A[i] ≠ x"，并从S中移除i
- 若 |S| = 1: 对手必须回答真相

分析:
- 每次查询最多从S中移除1个元素
- 必须将 |S| 从 n 减少到 1
- 至少需要 n-1 次查询

结论: 无序数组搜索的下界是 Ω(n)
```

## 搜索有序数组的下界

### 信息论下界

```
问题: 在有序数组中搜索x (或确认不存在)

信息论分析:
─────────────────────────────────────────
- 有 n+1 种可能结果:
  x < A[0], A[0] < x < A[1], ..., x > A[n-1]

- 每次比较得到2种结果 (≤ 或 >)

- 设最坏情况需要 k 次比较
  则: 2ᵏ ≥ n+1

  k ≥ log₂(n+1) = Ω(log n)

结论: 有序数组搜索的下界是 Ω(log n)

注: 二分查找达到了这个下界
```

## 选择问题 (第k小元素) 的下界

### 找最小/最大元素

```
下界: 至少需要 n-1 次比较

证明:
─────────────────────────────────────────
- 除最小元素外，每个元素必须至少输掉一次比较
- 每次比较产生一个"输家"
- n-1 个非最小元素各需至少一个输家标记
- 因此至少需要 n-1 次比较

锦标赛方法达到此下界
```

### 找最大和最小元素

```
下界: ⌈3n/2⌉ - 2 次比较

上界算法:
─────────────────────────────────────────
1. 将元素两两分组比较 (⌊n/2⌋次)
2. 在「胜者」中找最大 (⌈n/2⌉-1次)
3. 在「败者」中找最小 (⌈n/2⌉-1次)

总计: ⌊n/2⌋ + ⌈n/2⌉ - 1 + ⌈n/2⌉ - 1 = ⌈3n/2⌉ - 2

下界证明: 信息论论证 (较复杂)
```

### 找中位数

```
下界: Ω(n) 比较次数 (显然)

紧确界: 3n - o(n) 比较次数

Blum-Floyd-Pratt-Rivest-Tarjan (BFPRT) 算法: O(n)
```

## 3-SUM 问题的下界

### 问题描述

给定 n 个整数的集合，判断是否存在三个元素 a, b, c 使得 a + b + c = 0。

### 当前最佳结果

```
已知:
─────────────────────────────────────────
- 简单算法: O(n²)
- 当前最佳: O(n² / log n) 或 O(n² · (log log n)^(O(1)) / log n)

猜想:
─────────────────────────────────────────
- 3-SUM 的复杂度是 Ω(n²)

注: 3-SUM 猜想是许多几何问题下界的基础
```

## 矩阵乘法的下界

### 标准算法

```
C = A × B，其中 A, B 是 n×n 矩阵

标准算法: O(n³) 次乘法

更好的算法:
- Strassen: O(n^log₂7) ≈ O(n^2.81)
- Coppersmith-Winograd: O(n^2.376)
- Alman-Williams (2020): O(n^2.373)

猜想:
─────────────────────────────────────────
矩阵乘法的真正复杂度可能是 O(n²) 或接近
```

### 下界结果

```
一般情况下的下界:
─────────────────────────────────────────
在代数计算树模型中:
- 任何计算矩阵乘法的算法需要 Ω(n²) 操作
  (因为需要读取输入矩阵)

更强的下界:
- 对于「双线性算法」类，有更强的限制
- 但 Strassen 等算法不属于此类
```

## 复杂类分离结果

### 时间层级定理

```
定理: 若 f(n) 是时间可构造函数，则:
DTIME(f(n)) ⊂ DTIME(f(n)²)

证明概要 (对角线法):
─────────────────────────────────────────
构造语言 L = { M | M 在 f(|M|) 步内不接受 M }

1. L ∈ DTIME(f(n)²):
   - 可以模拟 M 运行 f(|M|) 步
   - 需要 O(f(n)²) 时间

2. L ∉ DTIME(f(n)):
   - 假设存在机器 M* 在 f(n) 内判定 L
   - 考虑 M* 是否接受自己
   - 矛盾!

推论: P ⊂ EXPTIME
```

### 空间层级定理

```
定理: 若 f(n) 是空间可构造函数，则:
SPACE(f(n)) ⊂ SPACE(f(n) · log f(n))

更强的结果:
SPACE(f(n)) ⊂ SPACE(o(f(n)))

推论: L ⊂ PSPACE
```

## 归约与下界传递

```
归约保持复杂度:
─────────────────────────────────────────
若 A ≤ₚ B (多项式时间归约)，则:
- 若 B ∈ P，则 A ∈ P
- 若 A ∉ P，则 B ∉ P

用于证明下界:
1. 找到问题 A，已知有下界 L
2. 证明 A ≤ₚ B
3. 则 B 也有下界 L

典型例子:
─────────────────────────────────────────
排序归约到许多问题:
- 凸包: 排序 ≤ₚ 凸包
- 最接近点对: 排序 ≤ₚ 1D最接近点对

因此这些问题的下界至少是 Ω(n log n)
```

## 下界证明方法总结

| 方法 | 适用场景 | 关键思想 |
|-----|---------|---------|
| **决策树** | 比较问题 | 计算可能的输出数 |
| **对手论证** | 自适应算法 | 构造最坏输入 |
| **信息论** | 需要区分N种情况 | log₂(N) 信息量 |
| **代数复杂度** | 代数问题 | 多项式次数、维度 |
| **对角线法** | 可计算性 | 自引用构造 |
| **归约** | 相关问题 | 从已知下界传递 |
| **通信复杂度** | 分布式问题 | 信息传输量 |

## 开放问题

```
重要的开放下界问题:
─────────────────────────────────────────

1. P vs NP
   - NP 有超多项式下界吗?

2. 矩阵乘法
   - 真正的矩阵乘法复杂度是多少?

3. 3-SUM
   - 是否有 Ω(n²) 的下界?

4. 编辑距离
   - 是否可以突破 O(n²)?

5. 全源最短路
   - 是否能突破 O(n³)?

6. 动态图问题
   - 许多动态问题缺乏非平凡下界
```

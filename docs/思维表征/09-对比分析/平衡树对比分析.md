# 平衡树对比分析


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

## 概述

本文档对比分析四种主要的平衡二叉搜索树：AVL树、红黑树、B树和Splay树。

## AVL树

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              AVL树                                              │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  平衡条件: 任意节点左右子树高度差不超过1                                         │
│  |height(left) - height(right)| ≤ 1                                            │
│                                                                                 │
│  旋转操作:                                                                      │
│  • LL (右旋): 左子树的左子树过高                                               │
│  • RR (左旋): 右子树的右子树过高                                               │
│  • LR (左旋+右旋): 左子树的右子树过高                                          │
│  • RL (右旋+左旋): 右子树的左子树过高                                          │
│                                                                                 │
│  特点:                                                                          │
│  • 查找效率最高 (严格平衡，树高最小)                                           │
│  • 插入删除可能需要多次旋转                                                    │
│  • 适合查找频繁、修改较少的场景                                                │
│                                                                                 │
│  应用: 数据库索引 (内存中)、Map实现                                            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 红黑树

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              红黑树                                             │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  红黑性质:                                                                      │
│  1. 每个节点是红色或黑色                                                       │
│  2. 根节点是黑色                                                               │
│  3. 所有叶子(NIL)是黑色                                                        │
│  4. 红色节点的子节点必须是黑色 (无连续红节点)                                   │
│  5. 从任一节点到其每个叶子的路径包含相同数目的黑色节点                          │
│                                                                                 │
│  关键推论: 最长路径不超过最短路径的2倍                                          │
│  → 树高 ≤ 2log(n+1)                                                            │
│                                                                                 │
│  特点:                                                                          │
│  • 插入删除旋转次数较少 (最多2次旋转)                                          │
│  • 实现相对复杂但高效                                                          │
│  • 适合频繁插入删除的场景                                                      │
│                                                                                 │
│  应用:                                                                          │
│  • Java: TreeMap, TreeSet                                                      │
│  • C++: std::map, std::set (通常实现)                                          │
│  • Linux内核: 完全公平调度器(CFS)                                              │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## B树/B+树

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              B树家族                                            │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  B树特点:                                                                       │
│  • 多路搜索树 (每个节点可以有多个子节点)                                       │
│  • 所有叶子节点在同一层                                                        │
│  • 节点关键字数量在 [t-1, 2t-1] 之间 (t为最小度数)                              │
│                                                                                 │
│  B+树改进 (数据库常用):                                                         │
│  • 非叶子节点只存索引，不存数据                                                │
│  • 所有数据在叶子节点，叶子之间形成链表                                          │
│  • 更适合范围查询和顺序遍历                                                    │
│                                                                                 │
│  优势:                                                                          │
│  • 减少磁盘I/O次数 (节点大小匹配磁盘页)                                        │
│  • 树高更低 (分支因子大)                                                       │
│                                                                                 │
│  应用:                                                                          │
│  • 文件系统: NTFS, HFS+, ext4                                                  │
│  • 数据库: MySQL (InnoDB), MongoDB                                             │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## Splay树

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              Splay树                                            │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  核心思想: 伸展 (Splay)                                                         │
│  • 每次访问节点后，将其旋转到根节点                                            │
│  • 通过旋转保持树的平衡 (摊还意义)                                             │
│                                                                                 │
│  伸展操作:                                                                      │
│  • Zig (单旋转): 父节点是根                                                    │
│  • Zig-Zig: 节点和父节点都是左/右孩子                                          │
│  • Zig-Zag: 节点和父节点方向相反                                               │
│                                                                                 │
│  特点:                                                                          │
│  • 均摊 O(log n)，但最坏 O(n)                                                  │
│  • 访问局部性好 (热点数据靠近根)                                               │
│  • 无需存储平衡信息                                                            │
│                                                                                 │
│  应用:                                                                          │
│  • 缓存实现                                                                    │
│  • 访问模式有局部性的场景                                                      │
│  • 某些网络路由算法                                                            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

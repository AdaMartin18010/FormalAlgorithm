# 近似算法技术对比

## 概述

本文档对比分析解决NP-困难问题的各种近似算法技术，包括贪心算法、局部搜索、线性规划舍入、原始对偶等方法。

## 贪心算法

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                            贪心近似算法                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  集合覆盖 (Set Cover)                                                           │
│  ═══════════════════════════════════════════════════════════════════════════   │
│  问题: 给定全集U和子集族S，选最少数量的子集覆盖U                                │
│                                                                                 │
│  贪心策略: 每次选择覆盖最多未覆盖元素的集合                                     │
│                                                                                 │
│  算法:                                                                          │
│  U' = U                                                                         │
│  C = ∅                                                                          │
│  while U' ≠ ∅:                                                                  │
│      选择 S ∈ S 最大化 |S ∩ U'|                                                 │
│      C = C ∪ {S}                                                                │
│      U' = U' - S                                                                │
│                                                                                 │
│  近似比: H_n ≤ ln(n) + 1  (调和级数)                                           │
│  这是最优可能的近似比 (除非 P = NP)                                            │
│                                                                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  顶点覆盖 (Vertex Cover)                                                        │
│  ═══════════════════════════════════════════════════════════════════════════   │
│  问题: 选最少数量的顶点覆盖所有边                                               │
│                                                                                 │
│  贪心策略1 (选择最大度顶点): 近似比为 O(log n)                                 │
│  贪心策略2 (取边的两个端点): 近似比为 2 (最优!)                                 │
│                                                                                 │
│  2-近似算法:                                                                    │
│  C = ∅                                                                          │
│  while 存在未覆盖边 (u,v):                                                      │
│      C = C ∪ {u, v}  // 取两个端点                                             │
│      标记所有与u,v相连的边为已覆盖                                             │
│                                                                                 │
│  证明: 算法选择的边构成匹配M，|C| = 2|M|                                        │
│        最优解至少需要|M|个顶点覆盖M的边                                         │
│        因此 |C| ≤ 2|OPT|                                                       │
│                                                                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  其他贪心近似算法:                                                              │
│  ═══════════════════════════════════════════════════════════════════════════   │
│                                                                                 │
│  ┌─────────────────┬────────────────┬─────────────────────────────────────┐    │
│  │     问题        │   近似比       │           贪心策略                  │    │
│  ├─────────────────┼────────────────┼─────────────────────────────────────┤    │
│  │                 │                │                                     │    │
│  │  最大独立集     │   Δ (最大度)   │  反复删除最小度顶点                 │    │
│  │                 │   (无权)       │                                     │    │
│  ├─────────────────┼────────────────┼─────────────────────────────────────┤    │
│  │                 │                │                                     │    │
│  │  负载均衡       │   2 - 1/m      │  将作业分配给当前负载最小的机器     │    │
│  │  (List Scheduling)│  (m台机器)   │                                     │    │
│  │                 │                │                                     │    │
│  ├─────────────────┼────────────────┼─────────────────────────────────────┤    │
│  │                 │                │                                     │    │
│  │  中心选择       │   2            │  迭代选择最远点作为中心             │    │
│  │  (k-center)     │                │                                     │    │
│  │                 │                │                                     │    │
│  └─────────────────┴────────────────┴─────────────────────────────────────┘    │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 线性规划方法

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         线性规划舍入法                                           │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  基本框架:                                                                      │
│  ═══════════════════════════════════════════════════════════════════════════   │
│  1. 将问题建模为整数线性规划 (ILP)                                             │
│  2. 松弛为线性规划 (LP) 并求解                                                 │
│  3. 将分数解舍入为整数解                                                        │
│                                                                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  集合覆盖的LP舍入                                                               │
│  ═══════════════════════════════════════════════════════════════════════════   │
│                                                                                 │
│  ILP形式:                                                                       │
│  minimize Σ(j) c_j · x_j                                                        │
│  subject to Σ(j: i∈S_j) x_j ≥ 1  for all i ∈ U                                 │
│             x_j ∈ {0, 1}                                                        │
│                                                                                 │
│  LP松弛: x_j ∈ [0, 1]                                                          │
│                                                                                 │
│  舍入策略 (f-近似):                                                             │
│  设f为元素出现的最多次数，取 x'_j = 1 if x_j ≥ 1/f, else 0                     │
│                                                                                 │
│  近似比: f (每个元素最多在f个集合中出现)                                        │
│                                                                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  加权顶点覆盖的LP舍入                                                           │
│  ═══════════════════════════════════════════════════════════════════════════   │
│                                                                                 │
│  LP形式:                                                                        │
│  minimize Σ(v) w(v) · x_v                                                       │
│  subject to x_u + x_v ≥ 1  for all (u,v) ∈ E                                   │
│             x_v ≥ 0                                                             │
│                                                                                 │
│  舍入: x'_v = 1 if x_v ≥ 1/2, else 0                                           │
│                                                                                 │
│  近似比: 2                                                                      │
│  证明: 对每条边(u,v)，x_u + x_v ≥ 1，故至少一个 ≥ 1/2                          │
│        舍入后每条边至少一个端点被选中                                           │
│        费用: Σ w(v)·x'_v ≤ 2 Σ w(v)·x_v ≤ 2·OPT                                │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         原始对偶方法                                             │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  基本思想:                                                                      │
│  ═══════════════════════════════════════════════════════════════════════════   │
│  • 构造LP的对偶问题                                                             │
│  • 同时构造原始可行解和对偶可行解                                               │
│  • 通过弱对偶性得到近似比保证                                                   │
│                                                                                 │
│  优势: 无需显式求解LP，计算更高效                                               │
│                                                                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  集合覆盖的原始对偶算法                                                         │
│  ═══════════════════════════════════════════════════════════════════════════   │
│                                                                                 │
│  原始: min Σ c_j x_j         对偶: max Σ y_i                                   │
│        s.t. A·x ≥ 1                  s.t. A^T·y ≤ c                            │
│             x ≥ 0                         y ≥ 0                                 │
│                                                                                 │
│  算法:                                                                          │
│  1. 初始化: x = 0, y = 0, U' = U (未覆盖元素)                                  │
│  2. while U' ≠ ∅:                                                               │
│       选择 i ∈ U'                                                               │
│       增加 y_i 直到某个约束紧致: Σ(i∈S_j) y_i = c_j                            │
│       选择该集合 S_j 加入覆盖: x_j = 1                                          │
│       U' = U' - S_j                                                             │
│                                                                                 │
│  近似比: f (与LP舍入相同，但更高效)                                            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 特殊技术

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                        度量TSP的Christofides算法                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  Christofides算法 (1.5-近似)                                                    │
│  ═══════════════════════════════════════════════════════════════════════════   │
│                                                                                 │
│  步骤:                                                                          │
│  1. 计算最小生成树 (MST) T                                                      │
│  2. 找出T中度数为奇数的顶点集合 O                                               │
│  3. 在O上计算最小权完美匹配 M                                                   │
│  4. 合并T和M得到欧拉图，找欧拉回路                                              │
│  5. 通过跳过已访问顶点得到哈密顿回路 (shortcut)                                 │
│                                                                                 │
│  近似比证明:                                                                    │
│  • c(T) ≤ OPT (MST是最优解的下界)                                              │
│  • c(M) ≤ OPT/2 (奇数度顶点至少2个，匹配边来自最优解的交错)                    │
│  • 总成本 ≤ c(T) + c(M) ≤ 1.5·OPT                                              │
│                                                                                 │
│  这是度量TSP在P≠NP假设下的最佳已知近似比 (直到2020年改进到1.5-ε)               │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│                        Max-Cut的Goemans-Williamson算法                           │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  突破: 首次使用半定规划(SDP)获得优于0.5的近似比                                  │
│  ═══════════════════════════════════════════════════════════════════════════   │
│                                                                                 │
│  算法:                                                                          │
│  1. 将Max-Cut建模为二次规划                                                    │
│  2. 松弛为SDP (向量化)                                                         │
│  3. 求解SDP得到单位向量 {v_i}                                                  │
│  4. 随机选择超平面，根据v_i在超平面的哪一侧决定割                               │
│                                                                                 │
│  近似比: 0.878... (α_GW ≈ 0.878567)                                            │
│  这是最优的 (除非UGC猜想不成立)                                                │
│                                                                                 │
│  历史意义: 开创了SDP在近似算法中的应用                                          │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────────┐
│                        背包问题的FPTAS                                           │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  完全多项式时间近似方案 (FPTAS)                                                 │
│  ═══════════════════════════════════════════════════════════════════════════   │
│                                                                                 │
│  思想: 通过缩放将大值转换为小值，用伪多项式DP求解                               │
│                                                                                 │
│  算法:                                                                          │
│  1. 选择缩放参数 K = ε · P_max / n                                             │
│     (P_max为最大价值，ε为误差参数)                                             │
│  2. 缩放价值: v'_i = floor(v_i / K)                                            │
│  3. 在缩放后的价值上用DP求解                                                    │
│                                                                                 │
│  时间复杂度: O(n³/ε)                                                           │
│  近似比: (1 - ε)                                                               │
│                                                                                 │
│  这是背包问题的最佳可能结果 (强NP困难问题不存在FPTAS)                          │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

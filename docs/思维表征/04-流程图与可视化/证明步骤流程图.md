# 证明步骤流程图

## 数学归纳法证明流程

```
┌─────────────────┐
│    开始证明     │
│  ∀n≥n₀, P(n)   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第1步: 基础步   │
│ 证明 P(n₀)      │
│ 成立            │
└────────┬────────┘
         │
         ▼
     ┌───────┐
     │基础步?│
     │成立?  │
     └───┬───┘
         │否
         ▼
┌─────────────────┐
│   检查错误      │
│  重新定义P(n)   │
└────────┬────────┘
         │
         └────────► 回到第1步
         是
         ▼
┌─────────────────┐
│ 第2步: 归纳假设 │
│ 假设对某个      │
│ k≥n₀, P(k)成立  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第3步: 归纳步骤 │
│ 基于P(k)证明    │
│ P(k+1)成立      │
└────────┬────────┘
         │
         ▼
     ┌───────┐
     │归纳步?│
     │成立?  │
     └───┬───┘
         │否
         ▼
┌─────────────────┐
│  分析失败原因   │
│ - 假设使用错误? │
│ - 推导有漏洞?   │
│ - 需要强归纳?   │
└────────┬────────┘
         │
         └────────► 修正后重试
         是
         ▼
┌─────────────────┐
│ 第4步: 结论     │
│ 根据数学归纳    │
│ 原理，命题成立  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    证明完成     │
│      ∎        │
└─────────────────┘
```

## 反证法证明流程

```
┌─────────────────┐
│   开始证明      │
│    命题 P       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第1步: 假设     │
│ 假设 ¬P 成立    │
│ (P的否定)       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第2步: 逻辑推导 │
│ 从 ¬P 出发      │
│ 进行一系列      │
│ 逻辑推理        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第3步: 得出推论 │
│ 得到某个        │
│ 命题 Q          │
└────────┬────────┘
         │
         ▼
     ┌───────┐
     │ Q 是否 │
     │已知为假?│
     └───┬───┘
         │否
         ▼
     ┌───────┐
     │ Q 是否与│
     │¬Q矛盾? │
     └───┬───┘
         │否
         ▼
┌─────────────────┐
│  推导不够充分   │
│  需要更深入     │
│  的逻辑推理     │
└────────┬────────┘
         │
         └────────► 回到第2步
         是(任一)
         ▼
┌─────────────────┐
│ 第4步: 导出矛盾 │
│ 得到 Q ∧ ¬Q     │
│ (矛盾)          │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第5步: 否定假设 │
│ 假设 ¬P 导致    │
│ 矛盾，因此      │
│ ¬P 不成立       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第6步: 结论     │
│ P 必然成立      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    证明完成     │
│      ∎        │
└─────────────────┘
```

## 循环不变式证明流程

```
┌─────────────────┐
│ 证明算法部分    │
│ 正确性          │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第1步: 识别循环 │
│ 找出算法中的    │
│ 关键循环        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第2步: 定义     │
│ 循环不变式 I    │
│ 在每次迭代开始  │
│ 时成立的条件    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第3步: 初始化   │
│ 证明: 循环开始前│
│ I 成立          │
│ (基础条件)      │
└────────┬────────┘
         │
         ▼
     ┌───────┐
     │ 成立? │
     └───┬───┘
         │否
         ▼
┌─────────────────┐
│ 重新定义不变式  │
└────────┬────────┘
         │
         └────────► 回到第2步
         是
         ▼
┌─────────────────┐
│ 第4步: 保持     │
│ 证明: 若I在某次 │
│ 迭代前成立，则  │
│ 下次迭代前也成立│
└────────┬────────┘
         │
         ▼
     ┌───────┐
     │ 保持? │
     └───┬───┘
         │否
         ▼
┌─────────────────┐
│ 检查:           │
│ - 不变式定义?   │
│ - 循环体逻辑?   │
└────────┬────────┘
         │
         └────────► 修正后重试
         是
         ▼
┌─────────────────┐
│ 第5步: 终止     │
│ 证明: 循环终止时│
│ I ∧ 终止条件    │
│ ⇒ 正确结果      │
└────────┬────────┘
         │
         ▼
     ┌───────┐
     │ 推出  │
     │ 正确? │
     └───┬───┘
         │否
         ▼
┌─────────────────┐
│ 重新检查不变式  │
│ 或终止条件      │
└────────┬────────┘
         │
         └────────► 修正
         是
         ▼
┌─────────────────┐
│    证明完成     │
│ 算法部分正确    │
│      ∎        │
└─────────────────┘
```

## 结构归纳法流程

```
┌─────────────────┐
│ 证明递归结构    │
│ 的性质 P        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第1步: 识别     │
│ 结构的构造规则  │
│ - 基础构造子    │
│ - 归纳构造子    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第2步: 基础情况 │
│ 对基础构造子    │
│ 直接证明P成立   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第3步: 归纳假设 │
│ 假设对子结构    │
│ s₁, s₂, ...,    │
│ P(sᵢ) 成立      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第4步: 归纳步骤 │
│ 对归纳构造子    │
│ 利用归纳假设    │
│ 证明组合后的    │
│ 结构满足P       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第5步: 结论     │
│ 根据结构归纳    │
│ 原理，所有结构  │
│ 满足P           │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    证明完成     │
│      ∎        │
└─────────────────┘
```

## 算法终止性证明流程

```
┌─────────────────┐
│ 证明算法终止性  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第1步: 识别     │
│ 循环/递归结构   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第2步: 定义     │
│ 势函数 Φ 或     │
│ 良基度量        │
│ (自然数上的值)  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第3步: 证明递减 │
│ 每次迭代/递归   │
│ Φ严格递减       │
│ 或有下界        │
└────────┬────────┘
         │
         ▼
     ┌───────┐
     │ Φ递减?│
     └───┬───┘
         │否
         ▼
┌─────────────────┐
│ 重新定义势函数  │
│ 或检查算法逻辑  │
└────────┬────────┘
         │
         └────────► 回到第2步
         是
         ▼
┌─────────────────┐
│ 第4步: 应用     │
│ 良基关系原理    │
│ 无无穷下降链    │
│ 故必然终止      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 第5步: 边界分析 │
│ (可选)          │
│ 给出上界        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    证明完成     │
│ 算法必然终止    │
│      ∎        │
└─────────────────┘
```

## 类型安全证明流程 (进展性+保持性)

```
┌─────────────────┐
│ 证明类型系统    │
│    安全性       │
└────────┬────────┘
         │
         ├─────────────────────────┐
         │                         │
         ▼                         ▼
┌─────────────────┐       ┌─────────────────┐
│  A. 进展性证明   │       │  B. 保持性证明   │
│  Progress       │       │  Preservation   │
└────────┬────────┘       └────────┬────────┘
         │                         │
         ▼                         ▼
┌─────────────────┐       ┌─────────────────┐
│ 证明: 若e: T    │       │ 证明: 若e: T 且 │
│ 且e不是值       │       │ e → e'          │
│ 则存在e'使      │       │ 则 e': T        │
│ e → e'          │       │                 │
└────────┬────────┘       └────────┬────────┘
         │                         │
         ▼                         ▼
┌─────────────────┐       ┌─────────────────┐
│ 方法: 对类型    │       │ 方法: 对求值    │
│ 推导进行结构    │       │ 规则进行归纳    │
│ 归纳            │       │                 │
└────────┬────────┘       └────────┬────────┘
         │                         │
         ▼                         ▼
┌─────────────────┐       ┌─────────────────┐
│ 对每种类型规则: │       │ 对每种求值规则: │
│ 证明良类型项    │       │ 证明类型保持    │
│ 可求值或是值    │       │                 │
└────────┬────────┘       └────────┬────────┘
         │                         │
         ▼                         ▼
    ┌─────────┐               ┌─────────┐
    │ 所有情况│               │ 所有情况│
    │ 证明完成?│              │ 证明完成?│
    └───┬─────┘               └───┬─────┘
        │否                        │否
        └────────► 继续            └────────► 继续
        是                          是
        │                           │
        └───────────────┬───────────┘
                        │
                        ▼
               ┌─────────────────┐
               │  C. 安全性结论   │
               │  进展性+保持性   │
               │  ⇒ 类型安全     │
               │  (不会卡住)     │
               └────────┬────────┘
                        │
                        ▼
               ┌─────────────────┐
               │    证明完成     │
               │      ∎        │
               └─────────────────┘
```

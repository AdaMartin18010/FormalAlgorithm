# 数据结构选择决策树

## 主决策树

```
┌─────────────────────────────────────────────────────────────────────┐
│                   数据结构选择决策树                                   │
└─────────────────────────────────────────────────────────────────────┘

需要解决什么问题?
    │
    ├──► 存储和访问数据?
    │       │
    │       ├──► 需要按位置访问?
    │       │       │
    │       │       ├──► 随机访问 O(1)? → 数组
    │       │       └──► 顺序访问? → 链表 / 数组
    │       │
    │       ├──► 需要键值关联?
    │       │       │
    │       │       ├──► 只需要查询存在性? → 哈希集合
    │       │       │
    │       │       ├──► 需要映射到值?
    │       │       │       ├──► 查询为主? → 哈希表
    │       │       │       └──► 需要有序? → 平衡树 / 跳表
    │       │       │
    │       │       └──► 范围查询频繁?
    │       │               ├──► 内存中 → 跳表 / 红黑树
    │       │               └──► 磁盘中 → B+树
    │       │
    │       └──► 需要维护顺序/优先级?
    │               ├──► FIFO? → 队列
    │               ├──► LIFO? → 栈
    │               ├──► 优先级? → 堆 / 优先队列
    │               └──► 双端操作? → 双端队列
    │
    ├──► 表示关系?
    │       │
    │       ├──► 层次关系 (一对多)? → 树
    │       │       ├──► 搜索用? → BST / Trie
    │       │       ├──► 优先用? → 堆
    │       │       ├──► 区间查询? → 线段树 / 树状数组
    │       │       └──► 字符串用? → Trie / 后缀树
    │       │
    │       └──► 网络关系 (多对多)? → 图
    │               ├──► 稠密图? → 邻接矩阵
    │               └──► 稀疏图? → 邻接表
    │
    └──► 特殊需求?
            │
            ├──► 并查集 / 连通性? → 并查集(Disjoint Set)
            ├──► 快速去重 (允许误差)? → 布隆过滤器
            ├──► LRU缓存? → 哈希表 + 双向链表
            └──► 前缀匹配? → Trie树
```

## 查找操作决策

```
主要操作是查找?
    │
    ├──► 查找频率?
    │       │
    │       ├──► 极少查找 → 线性搜索
    │       │
    │       ├──► 偶尔查找 → 排序 + 二分查找
    │       │
    │       └──► 频繁查找 → 预处理数据结构
    │
    ├──► 查找类型?
    │       │
    │       ├──► 精确匹配?
    │       │       ├──► 不需要顺序 → 哈希表 O(1)
    │       │       └──► 需要顺序 → 平衡树 O(log n)
    │       │
    │       ├──► 范围查询 [l, r]?
    │       │       ├──► 静态数据 → 排序数组 + 二分
    │       │       └──► 动态数据 → 平衡树 / 跳表
    │       │
    │       ├──► 前缀匹配?
    │       │       └──► Trie树 O(m)
    │       │
    │       └──► 最值查询?
    │               ├──► 只查最值 → 堆 O(1)
    │               └──► 同时修改 → 平衡树 / 堆
    │
    └──► 数据规模?
            │
            ├──► 小规模 (<1000) → 任何结构均可
            │
            ├──► 中规模 (1K-1M) → 哈希表 / 平衡树
            │
            └──► 大规模 (>1M)
                    ├──► 内存足够 → 哈希表
                    └──► 内存受限 → B树 / 外存结构
```

## 修改操作决策

```
修改操作频繁?
    │
    ├──► 修改类型?
    │       │
    │       ├──► 只在末尾?
    │       │       ├──► 添加 → 动态数组 / 链表
    │       │       └──► 删除 → 栈 / 队列
    │       │
    │       ├──► 在任意位置?
    │       │       ├──► 知道位置 → 链表 O(1)
    │       │       └──► 需要查找 → 平衡树 O(log n)
    │       │
    │       └──► 批量修改?
    │               ├──► 区间更新 → 线段树 / 树状数组
    │               └──► 整体重建 → 重建+重哈希
    │
    ├──► 修改与查询的比率?
    │       │
    │       ├──► 修改少，查询多 → 预处理优化查询
    │       │
    │       ├──► 修改多，查询少 → 优化修改路径
    │       │
    │       └──► 都频繁 → 平衡结构 (树、跳表)
    │
    └──► 是否需要持久化?
            ├──► 不需要 → 常规结构
            └──► 需要 → 持久化数据结构 (函数式)
```

## 图存储结构决策

```
图的特征?
    │
    ├──► 图的密度?
    │       │
    │       ├──► 稠密 (E ≈ V²) → 邻接矩阵
    │       │       优点: O(1)边查询、矩阵运算
    │       │       缺点: O(V²)空间
    │       │
    │       └──► 稀疏 (E << V²) → 邻接表
    │               优点: O(V+E)空间、遍历高效
    │               缺点: 边查询需要遍历
    │
    ├──► 图的方向?
    │       │
    │       ├──► 无向图 → 对称存储、每条边存两次
    │       └──► 有向图 → 有向边存储
    │
    ├──► 是否需要删除边?
    │       │
    │       ├──► 频繁删除 → 十字链表 / 邻接多重表
    │       └──► 不常删除 → 邻接表
    │
    └──► 边是否有权重?
            ├──► 无权 → 简单存储
            └──► 有权 → 存储权重信息
```

## 树结构决策

```
树的用途?
    │
    ├──► 搜索 / 查找?
    │       │
    │       ├──► 一般键值 → 二叉搜索树
    │       │       ├──► 实现简单 → BST (可能退化)
    │       │       ├──► 严格平衡 → AVL树
    │       │       └──► 实用平衡 → 红黑树 (推荐)
    │       │
    │       ├──► 磁盘存储 / 数据库索引 → B树 / B+树
    │       │
    │       └──► 字符串键 → Trie树 / 后缀树
    │
    ├──► 优先级队列?
    │       │
    │       ├──► 基础需求 → 二叉堆
    │       ├──► 需要合并 → 二项堆 / 斐波那契堆
    │       └──► 需要减小键值 → 斐波那契堆
    │
    ├──► 区间查询 / RMQ?
    │       │
    │       ├──► 静态数据 → 稀疏表 O(1)查询
    │       ├──► 单点修改 → 线段树 O(log n)
    │       └──► 前缀和修改 → 树状数组 O(log n)
    │
    └──► 其他特殊用途?
            ├──► 并查集 → 森林 (路径压缩+按秩合并)
            ├──► 字典树 → Trie
            └──► 线段树变种 → 主席树、可持久化线段树
```

## 线性结构决策

```
数据访问模式?
    │
    ├──► 随机访问为主?
    │       │
    │       ├──► 大小固定? → 静态数组
    │       └──► 大小变化? → 动态数组 (Vector/ArrayList)
    │
    ├──► 顺序访问为主?
    │       │
    │       ├──► 只在头部操作? → 链表 / 双端队列
    │       │
    │       ├──► 只在尾部操作? → 动态数组
    │       │
    │       ├──► 两端操作? → 双端队列 (Deque)
    │       │
    │       └──► 中间插入删除? → 链表
    │
    └──► 特殊语义?
            │
            ├──► 后进先出 LIFO → 栈
            ├──► 先进先出 FIFO → 队列
            └──► 优先级 → 优先队列 (堆)
```

## 空间效率决策

```
内存限制?
    │
    ├──► 内存充足?
    │       └──► 选择时间最优的数据结构
    │
    ├──► 内存受限?
    │       │
    │       ├──► 需要查找? → 布隆过滤器 (有假阳性)
    │       ├──► 压缩存储 → 压缩列表、紧凑数组
    │       ├──► 外部存储 → B树、LSM树
    │       └──► 稀疏数据 → 稀疏矩阵、哈希表
    │
    └──► 极大量数据?
            ├──► 外部排序 + 磁盘存储
            ├──► 流式处理 (不存储全部)
            └──► 采样 / 近似数据结构
```

## 并发安全决策

```
并发访问需求?
    │
    ├──► 单线程?
    │       └──► 标准非线程安全实现
    │
    ├──► 多线程读多写少?
    │       ├──► 读写锁保护
    │       └──► 不可变结构 + 替换
    │
    ├──► 多线程高并发?
    │       │
    │       ├──► 读为主 → 无锁结构 / RCU
    │       ├──► 写为主 → 细粒度锁 / 跳表
    │       └──► 混合 → 并发数据结构 (ConcurrentHashMap)
    │
    └──► 无锁要求?
            ├──► CAS操作 → 原子类型
            ├──► 无锁队列 → Michael-Scott队列
            └──► 无锁栈 → Treiber栈
```

## 综合推荐表

| 应用场景 | 推荐结构 | 次选结构 | 避免使用 |
|----------|---------|---------|---------|
| 通用查找 | 哈希表 | 平衡树 | 线性搜索 |
| 有序映射 | 红黑树 | 跳表 | 无序哈希表 |
| 范围查询 | B+树 / 跳表 | 线段树 | 哈希表 |
| LRU缓存 | 哈希+双向链表 | - | 纯哈希 |
| 优先队列 | 二叉堆 | 斐波那契堆 | 排序数组 |
| 并查集 | 路径压缩+按秩合并 | - | 朴素实现 |
| 字符串前缀 | Trie | 排序数组+二分 | 暴力 |
| 区间最值 | 线段树 | 稀疏表 | 遍历 |
| 前缀和 | 树状数组 | 线段树 | 重复计算 |
| 去重统计 | 布隆过滤器+哈希 | 哈希表 | 列表 |
| 图(稀疏) | 邻接表 | 前向星 | 邻接矩阵 |
| 图(稠密) | 邻接矩阵 | - | 邻接表 |

---

*数据结构选择需综合考虑时间复杂度、空间复杂度、实现难度和实际数据特征*

## 主决策树

```
分析目标?
    │
    ├──► 算法时间复杂度?
    │       │
    │       ├──► 算法结构?
    │       │       │
    │       │       ├──► 单层循环?
    │       │       │       └──► 循环次数 × 循环体复杂度
    │       │       │
    │       │       ├──► 嵌套循环?
    │       │       │       └──► 各层循环次数相乘
    │       │       │
    │       │       ├──► 递归算法?
    │       │       │       │
    │       │       │       ├──► 能写出递推式? → 求解递推关系
    │       │       │       │       ├──► T(n)=aT(n/b)+f(n) → 主定理
    │       │       │       │       ├──► 其他形式 → 展开法/猜测验证
    │       │       │       │       └──► 复杂形式 → Akra-Bazzi方法
    │       │       │       │
    │       │       │       └──► 难以直接分析? → 递归树/主方法
    │       │       │
    │       │       └──► 混合结构? → 分解各部分求和
    │       │
    │       └──► 分析类型?
    │               ├──► 最坏情况 → 找最坏输入
    │               ├──► 最好情况 → 找最优输入
    │               ├──► 平均情况 → 概率分析
    │               └──► 摊还分析 → 势能/聚合/记账方法
    │
    ├──► 算法空间复杂度?
    │       │
    │       ├──► 递归深度大? → 考虑调用栈空间
    │       ├──► 使用辅助数组? → 数组大小
    │       ├──► 动态分配? → 峰值内存
    │       └──► 输出空间? → 通常不计入
    │
    └──► 问题复杂度下界?
            │
            ├──► 比较问题? → 决策树模型
            ├──► 可归约性? → 从已知下界归约
            └──► 信息论? → 信息熵下界
```

## 递推关系求解决策

```
递推关系形式?
    │
    ├──► T(n) = a·T(n/b) + f(n) ?
    │       │
    │       ├──► f(n) = O(n^(log_b(a) - ε))? → 情况1: T(n) = Θ(n^(log_b(a)))
    │       │       例: T(n)=2T(n/2)+O(1) → Θ(n)
    │       │
    │       ├──► f(n) = Θ(n^(log_b(a)))? → 情况2: T(n) = Θ(n^(log_b(a))·log n)
    │       │       例: T(n)=2T(n/2)+O(n) → Θ(n log n)
    │       │
    │       └──► f(n) = Ω(n^(log_b(a) + ε))? → 情况3: T(n) = Θ(f(n))
    │               (需满足正则条件 af(n/b) ≤ cf(n))
    │               例: T(n)=2T(n/2)+O(n²) → Θ(n²)
    │
    ├──► T(n) = T(n-1) + f(n) ?
    │       └──► 展开求和: T(n) = Σf(i) + T(0)
    │           例: T(n)=T(n-1)+O(n) → Θ(n²)
    │
    ├──► T(n) = a·T(n-b) + f(n) ?
    │       └──► 指数级: T(n) = O(a^(n/b) · f(n))
    │           例: T(n)=2T(n-1)+O(1) → Θ(2ⁿ)
    │
    └──► 更复杂形式?
            ├──► 尝试: 递归树方法
            ├──► 尝试: 猜测+代入验证
            └──► 使用: Akra-Bazzi定理(连续版本主定理)
```

## 摊还分析决策

```
需要摊还分析的场景:
    └──► 操作序列中单个操作代价高，但整体平均低

分析方法选择?
    │
    ├──► 聚合分析 (Aggregate Analysis)
    │       适用: 所有操作类型相同
    │       方法: n个操作总代价 / n
    │       例: 栈的MULTIPOP操作
    │
    ├──► 记账方法 (Accounting Method)
 │       适用: 需要给每个操作分配摊还代价
    │       方法: 预付费/后付费思想
    │       关键: 确保"账户"余额非负
    │       例: 二进制计数器
    │
    └──► 势能方法 (Potential Method)
            适用: 复杂数据结构、多种操作
            方法: 定义势能函数 Φ
            公式: 摊还代价 = 实际代价 + ΔΦ
            例: 动态表、斐波那契堆
```

## 数据结构操作分析

```
分析什么操作?
    │
    ├──► 单次操作最坏情况?
    │       └──► 直接分析该操作
    │
    ├──► 操作序列性能?
    │       │
    │       ├──► 各操作独立? → 单次分析后求和
    │       │
    │       └──► 操作相互影响? → 摊还分析
    │               ├──► 聚合分析 (同类型操作)
    │               ├──► 记账方法 (分配摊还代价)
    │               └──► 势能方法 (复杂数据结构)
    │
    └──► 随机化数据结构?
            └──► 期望复杂度分析
                方法: 对随机选择求期望
                例: 随机化快速排序 O(n log n)期望
```

## 下界分析决策

```
证明问题下界:
    │
    ├──► 基于比较的算法?
    │       └──► 决策树模型
    │           ├──► 叶节点数 = 可能输出数
    │           ├──► 树高 = 最坏情况比较次数
    │           └──► log(叶节点数) = 下界
    │               例: 排序 Ω(n log n)
    │
    ├──► 可以归约?
    │       └──► 从已知下界问题归约
    │           步骤:
    │           1. 选择已知下界的问题B
    │           2. 证明 B ≤p A (多项式归约)
    │           3. 则A的下界至少为B的下界
    │
    ├──► 信息论下界?
    │       └──► 需要区分N种情况
    │           ├──► 每次查询获得k种结果
    │           └──► 至少需要 log_k(N) 次查询
    │               例: 搜索有序数组 Ω(log n)
    │
    └──► 对手论证?
            └──► 构造对手策略
                方法: 对手根据算法查询动态回答
                目标: 迫使算法做尽可能多工作
                例: 搜索无序数组 Ω(n)
```

## 图算法复杂度分析

```
图算法分析要素:
    │
    ├──► 输入规模?
    │       ├──► 顶点数 V
    │       └──► 边数 E
    │
    ├──► 存储方式?
    │       ├──► 邻接矩阵 → O(V²)空间影响
    │       └──► 邻接表 → O(V+E)空间影响
    │
    └──► 算法结构?
            │
            ├──► 遍历所有顶点?
            │       └──► 通常 O(V) 或 O(V log V)
            │
            ├──► 遍历所有边?
            │       └──► 通常 O(E) 或 O(E log V)
            │
            ├──► 使用优先队列?
            │       └──► 每次操作 O(log V)，共 O((V+E)log V)
            │
            └──► 动态规划?
                    └──► 状态数 × 转移时间
```

## 概率分析决策

```
涉及随机性?
    │
    ├──► 随机化算法分析?
    │       │
    │       ├──► 拉斯维加斯算法 (总是正确，时间随机)
    │       │       └──► 分析期望运行时间
    │       │           例: 随机化快速排序 O(n log n)期望
    │       │
    │       └──► 蒙特卡洛算法 (时间确定，结果随机)
    │               └──► 分析成功概率
    │                   例: 素性测试 错误概率 < 1/2^k
    │
    └──► 平均情况分析?
            └──► 假设输入分布
                ├──► 均匀分布 (所有输入等概率)
                ├──► 随机顺序 (如随机排列)
                └──► 其他分布
```

## 复杂度分析检查清单

### 时间复杂度检查

- [ ] 是否正确识别了基本操作?
- [ ] 是否考虑了最坏情况?
- [ ] 循环边界是否正确?
- [ ] 递归终止条件是否明确?
- [ ] 是否忽略了常数因子?(渐近分析中)
- [ ] 低阶项是否被正确处理?

### 空间复杂度检查

- [ ] 是否包含输入空间?(通常不计入)
- [ ] 递归栈空间是否计算?
- [ ] 辅助数据结构大小是否正确?
- [ ] 动态分配是否考虑峰值?

### 特殊情况检查

- [ ] 空输入处理?
- [ ] 最小输入处理?
- [ ] 边界值处理?
- [ ] 退化情况处理?

## 复杂度速查表

| 算法模式 | 典型复杂度 | 例子 |
|---------|-----------|------|
| 单层循环 | O(n) | 线性扫描 |
| 双层嵌套循环 | O(n²) | 冒泡排序 |
| 分治 (均衡) | O(n log n) | 归并排序 |
| 分治 (子问题多) | O(n^log_b(a)) | Strassen矩阵乘法 |
| 递归 (每次减1) | O(n) 或 O(2ⁿ) | 斐波那契递归 |
| 递归 (减半) | O(log n) | 二分查找 |
| 动态规划 | O(n²) ~ O(n³) | 矩阵链乘法 |
| 回溯 | O(2ⁿ) 或 O(n!) | 子集枚举、排列 |
| 图遍历 | O(V+E) | BFS、DFS |
| Dijkstra+堆 | O((V+E)log V) | 最短路 |
| Floyd-Warshall | O(V³) | 全源最短路 |

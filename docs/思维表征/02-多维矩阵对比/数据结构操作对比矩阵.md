## 集合与映射实现对比

| 实现方式 | 查找 | 插入 | 删除 | 范围查询 | 遍历有序 | 内存 |
|----------|------|------|------|----------|----------|------|
| **哈希表** | O(1) | O(1) | O(1) | O(n) | O(n log n) | O(n) |
| **红黑树** | O(log n) | O(log n) | O(log n) | O(log n + k) | O(n) | O(n) |
| **跳表** | O(log n) | O(log n) | O(log n) | O(log n + k) | O(n) | O(n) |

## 图存储结构对比

| 存储方式 | 空间 | 添加边 | 删除边 | 查询边 | 遍历邻接点 | 适用场景 |
|----------|------|--------|--------|--------|-----------|---------|
| **邻接矩阵** | O(V²) | O(1) | O(1) | O(1) | O(V) | 稠密图、快速查边 |
| **邻接表** | O(V+E) | O(1) | O(E) | O(E) | O(deg(v)) | 稀疏图、省空间 |
| **十字链表** | O(V+E) | O(1) | O(1) | O(deg) | O(deg(v)) | 有向图、删边快 |
| **邻接多重表** | O(V+E) | O(1) | O(1) | O(deg) | O(deg(v)) | 无向图、删边快 |
| **边列表** | O(E) | O(1) | O(E) | O(E) | O(E) | Kruskal算法 |

V=顶点数, E=边数, deg(v)=顶点v的度

## 高级数据结构特化对比

| 数据结构 | 核心优势 | 典型操作 | 应用场景 |
|----------|---------|---------|---------|
| **并查集** | 近乎O(1)的连通查询 | 合并、查找 | Kruskal、连通分量 |
| **线段树** | 区间查询/更新 O(log n) | 区间和、最值 | 区间统计、RMQ |
| **树状数组** | 前缀和更新 O(log n) | 单点更新、前缀和 | 逆序对、区间求和 |
| **字典树** | 字符串前缀匹配 O(m) | 插入、前缀搜索 | 自动补全、拼写检查 |
| **后缀树** | 子串查询 O(m) | 模式匹配 | 基因组分析、文本搜索 |
| **布隆过滤器** | O(1)空间高效的成员测试 | 插入、可能存在 | 缓存、数据库查询优化 |
| **LSM树** | 写优化、顺序IO | 批量合并 | 数据库、键值存储 |

## 数据结构选择指南

```
需要什么操作?
    │
    ├── 随机访问 O(1)?
    │       └── 数组
    │
    ├── 频繁插入删除?
    │       │
    │       ├── 需要搜索? → 平衡树 / 跳表
    │       │
    │       └── 只在一端? → 栈 / 队列
    │
    ├── 需要有序遍历?
    │       │
    │       ├── 范围查询多? → B+树
    │       │
    │       └── 一般情况 → 红黑树 / 跳表
    │
    ├── 只需要成员查询?
    │       │
    │       ├── 允许误差? → 布隆过滤器
    │       │
    │       └── 精确查询 → 哈希表
    │
    ├── 区间查询/更新?
    │       │
    │       ├── 区间复杂操作? → 线段树
    │       │
    │       └── 前缀和操作? → 树状数组
    │
    └── 字符串相关?
            │
            ├── 前缀匹配? → Trie树
            │
            └── 子串搜索? → 后缀树 / 后缀数组
```

## 实际系统应用对比

| 系统/应用 | 核心数据结构 | 选择理由 |
|-----------|-------------|---------|
| **数据库索引** | B+树 | 磁盘友好、范围查询、顺序访问 |
| **内存数据库** | 跳表、哈希表 | 实现简单、并发友好 |
| **文件系统** | B树变体 | 磁盘块对齐、平衡读写 |
| **LRU缓存** | 哈希表+双向链表 | O(1)访问和更新 |
| **Redis** | 跳表、压缩列表 | 内存紧凑、有序集合 |
| **Linux内核** | 红黑树、链表 | 通用、稳定 |
| **图数据库** | 邻接表+索引 | 灵活的图遍历 |
| **搜索引擎** | 倒排索引、B树 | 高效全文检索 |

---

*注: 所有复杂度均为最坏情况或摊还复杂度，实际性能受缓存、数据分布、实现质量影响。*

## 数学归纳法模板

```
╔═════════════════════════════════════════════════════════════════════════════╗
║                   数学归纳法标准格式                                         ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  要证明: ∀n ≥ n₀, P(n)                                                      ║
║                                                                             ║
║  证明:                                                                      ║
║  ─────────────────────────────────────────────────────────────────────────  ║
║                                                                             ║
║  基础步 (Base Case):                                                        ║
║  证明 P(n₀) 成立。                                                          ║
║                                                                             ║
║  归纳假设 (Inductive Hypothesis):                                           ║
║  假设对于某个 k ≥ n₀，P(k) 成立。                                           ║
║                                                                             ║
║  归纳步 (Inductive Step):                                                   ║
║  证明 P(k+1) 成立。                                                         ║
║  (使用归纳假设 P(k) 来帮助证明 P(k+1))                                      ║
║                                                                             ║
║  结论:                                                                      ║
║  由数学归纳法，∀n ≥ n₀, P(n) 成立。                                         ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝

╔═════════════════════════════════════════════════════════════════════════════╗
║                   强归纳法 (Complete Induction)                              ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  当证明 P(k+1) 需要不止 P(k) 一个假设时使用:                                  ║
║                                                                             ║
║  归纳假设: ∀m, n₀ ≤ m ≤ k, P(m) 成立                                        ║
║  (假设对所有小于等于k的情况都成立)                                           ║
║                                                                             ║
║  应用: 证明斐波那契数列性质、递归算法正确性                                  ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝

╔═════════════════════════════════════════════════════════════════════════════╗
║                   结构归纳法 (Structural Induction)                          ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  用于递归定义的结构，如树、列表、表达式等。                                   ║
║                                                                             ║
║  示例: 证明二叉树性质                                                         ║
║                                                                             ║
║  基础步: 证明对空树/单节点树成立                                             ║
║                                                                             ║
║  归纳步: 假设对左子树L和右子树R成立，                                        ║
║         证明对以L和R为子树的树成立                                          ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝
```

## 算法正确性证明

```
╔═════════════════════════════════════════════════════════════════════════════╗
║                   循环不变式 (Loop Invariant)                                ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  用于证明循环的正确性。                                                       ║
║                                                                             ║
║  循环不变式三要素:                                                            ║
║  ─────────────────────────────────────────────────────────────────────────  ║
║                                                                             ║
║  1. 初始化 (Initialization):                                                ║
║     循环开始前，不变式成立。                                                  ║
║                                                                             ║
║  2. 保持 (Maintenance):                                                     ║
║     如果某次迭代前不变式成立，则下次迭代前也成立。                            ║
║                                                                             ║
║  3. 终止 (Termination):                                                     ║
║     循环终止时，不变式给出有用的性质。                                        ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝

╔═════════════════════════════════════════════════════════════════════════════╗
║                   示例: 插入排序的循环不变式                                  ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  代码:                                                                      ║
║  for (int i = 1; i < n; i++) {                                              ║
║      int key = arr[i];                                                      ║
║      int j = i - 1;                                                         ║
║      while (j >= 0 && arr[j] > key) {                                       ║
║          arr[j+1] = arr[j];                                                 ║
║          j--;                                                               ║
║      }                                                                      ║
║      arr[j+1] = key;                                                        ║
║  }                                                                          ║
║                                                                             ║
║  循环不变式: arr[0..i-1] 是有序的                                           ║
║                                                                             ║
║  初始化: i=1时，arr[0..0]只有一个元素，自然有序                              ║
║                                                                             ║
║  保持: 每次迭代将arr[i]插入到arr[0..i-1]的正确位置，                         ║
║        使得arr[0..i]有序                                                    ║
║                                                                             ║
║  终止: i=n时，arr[0..n-1]即整个数组有序                                      ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝
```

## 复杂度证明技巧

```
╔═════════════════════════════════════════════════════════════════════════════╗
║                   递推关系求解                                               ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  【主定理 (Master Theorem)】                                                 ║
║  ════════════════════════════════════════════════════════════════════════  ║
║                                                                             ║
║  T(n) = aT(n/b) + f(n)                                                      ║
║                                                                             ║
║  比较 f(n) 与 n^(log_b a):                                                  ║
║                                                                             ║
║  Case 1: 若 f(n) = O(n^(log_b a - ε)), 则 T(n) = Θ(n^(log_b a))             ║
║                                                                             ║
║  Case 2: 若 f(n) = Θ(n^(log_b a)), 则 T(n) = Θ(n^(log_b a) log n)           ║
║                                                                             ║
║  Case 3: 若 f(n) = Ω(n^(log_b a + ε)), 且 af(n/b) ≤ cf(n),                  ║
║          则 T(n) = Θ(f(n))                                                  ║
║                                                                             ║
║  示例: T(n) = 2T(n/2) + n  →  T(n) = Θ(n log n)                             ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝

╔═════════════════════════════════════════════════════════════════════════════╗
║                   摊还分析 (Amortized Analysis)                              ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  三种方法:                                                                  ║
║                                                                             ║
║  1. 聚合分析 (Aggregate):                                                   ║
║     计算n个操作的总时间T(n)，摊还成本 = T(n)/n                               ║
║                                                                             ║
║  2. 记账方法 (Accounting):                                                  ║
║     为每个操作分配摊还成本，多付的作为信用存储                                ║
║                                                                             ║
║  3. 势能方法 (Potential):                                                   ║
║     定义势能函数Φ，摊还成本 = 实际成本 + ΔΦ                                  ║
║                                                                             ║
║  示例: 动态数组扩容的摊还成本为 O(1)                                         ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝
```

## NP-完全性证明

```
╔═════════════════════════════════════════════════════════════════════════════╗
║                   证明 NP-完全性步骤                                         ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  要证明问题 X 是 NP-完全的:                                                   ║
║                                                                             ║
║  Step 1: 证明 X ∈ NP                                                        ║
║  ─────────────────────────────────────────────────────────────────────────  ║
║  给出一个证书(解)，证明可在多项式时间内验证。                                 ║
║                                                                             ║
║  Step 2: 证明 X 是 NP-困难的                                                  ║
║  ─────────────────────────────────────────────────────────────────────────  ║
║  选择一个已知的 NP-完全问题 Y，证明 Y ≤ₚ X (多项式归约)                       ║
║                                                                             ║
║  常用归约起点: 3-SAT, 顶点覆盖, 团问题, 子集和                                 ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝

╔═════════════════════════════════════════════════════════════════════════════╗
║                   归约证明模板                                               ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  证明: 3-SAT ≤ₚ VERTEX-COVER                                                ║
║                                                                             ║
║  构造: 给定 3-SAT 实例 φ，构造图 G 和整数 k                                  ║
║                                                                             ║
║  证明双向:                                                                  ║
║  ─────────────────────────────────────────────────────────────────────────  ║
║                                                                             ║
║  (⇒) 若 φ 可满足，则 G 有大小为 k 的顶点覆盖:                                 ║
║      - 构造满足赋值对应的顶点覆盖...                                         ║
║                                                                             ║
║  (⇐) 若 G 有大小为 k 的顶点覆盖，则 φ 可满足:                                 ║
║      - 从顶点覆盖构造满足赋值...                                             ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝
```

## 常见证明错误

```
╔═════════════════════════════════════════════════════════════════════════════╗
║                   常见证明错误与避免                                         ║
╠═════════════════════════════════════════════════════════════════════════════╣
║                                                                             ║
║  ❌ 循环论证                                                                  ║
║  ════════════════════════════════════════════════════════════════════════  ║
║  错误: 用结论证明结论                                                         ║
║  例: 证明√2无理，假设√2无理导出矛盾...                                        ║
║                                                                             ║
║  ❌ 不当归纳                                                                  ║
║  ════════════════════════════════════════════════════════════════════════  ║
║  错误: 归纳步依赖额外的假设                                                   ║
║  例: 证明所有马同色，从k到k+1时需要假设前k-1匹与后k-1匹有交集                 ║
║                                                                             ║
║  ❌ 逆命题错误                                                                ║
║  ════════════════════════════════════════════════════════════════════════  ║
║  错误: 证明 P→Q 时，错误地证明 Q→P                                           ║
║                                                                             ║
║  ❌ 构造不完整的反例                                                          ║
║  ════════════════════════════════════════════════════════════════════════  ║
║  错误: 声称找到反例但没有验证所有条件                                         ║
║                                                                             ║
║  ❌ 混淆存在与任意                                                            ║
║  ════════════════════════════════════════════════════════════════════════  ║
║  错误: 用某个特例代替所有情况的证明                                           ║
║                                                                             ║
╚═════════════════════════════════════════════════════════════════════════════╝
```

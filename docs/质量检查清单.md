## 📋 清单说明

本检查清单用于系统化评估文档质量，确保所有内容符合项目标准。每个文档在发布前必须通过所有相关检查项。

**This checklist is used for systematic evaluation of document quality to ensure all content meets project standards.**

---

## A. 学术严谨性检查

### A.1 引用完整性

| 检查项 | 检查内容 | 通过标准 | 优先级 |
|--------|----------|----------|--------|
| A.1.1 | 每个定义都有来源引用 | 所有非原创定义引用 ≥1 个来源 | 🔴 高 |
| A.1.2 | 每个定理都有证明引用 | 所有定理引用原始证明或标准教材 | 🔴 高 |
| A.1.3 | 每个算法都有原始论文引用 | 核心算法引用原始发明论文 | 🔴 高 |
| A.1.4 | 复杂度分析有理论依据 | 复杂度结论引用分析方法来源 | 🟡 中 |
| A.1.5 | 实验数据有来源 | 所有实验数据标明来源 | 🟡 中 |

#### A.1 检查流程

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤1: 遍历文档中所有定义、定理、算法                         │
│       Iterate through all definitions, theorems, algorithms │
├─────────────────────────────────────────────────────────────┤
│ 步骤2: 检查是否有[来源引用]标记                              │
│       Check for [Source Citation] marker                     │
├─────────────────────────────────────────────────────────────┤
│ 步骤3: 验证引用格式符合ACM标准                               │
│       Verify citation format complies with ACM standard      │
├─────────────────────────────────────────────────────────────┤
│ 步骤4: 检查引用信息完整可验证                                │
│       Check citation information is complete and verifiable  │
├─────────────────────────────────────────────────────────────┤
│ 步骤5: 确认引用在参考文献列表中                              │
│       Confirm citation exists in reference list              │
└─────────────────────────────────────────────────────────────┘
```

### A.2 引用格式规范

- [ ] **作者格式**: 使用全名而非缩写（4+作者用"et al."）
- [ ] **年份格式**: 格式为 `(Year)`
- [ ] **标题格式**: 文章用引号 `"Title"`，书籍用斜体 *Title*
- [ ] **页码格式**: 使用 en-dash `24–46`（不是连字符 `24-46`）
- [ ] **DOI格式**: 格式为 `DOI: 10.xxxx/xxxxx` 或完整URL

#### 引用格式检查示例

```markdown
✅ 正确示例:
[1] Hoare, C. A. R. (1962). "Quicksort." The Computer Journal, 5(1), 10–16. DOI: 10.1093/comjnl/5.1.10

❌ 错误示例:
[1] Hoare, C. A. R. (1962). Quicksort. The Computer Journal, 5(1), 10-16.
   ^ 缺少引号       ^ 缺少DOI      ^ 使用连字符而非en-dash
```

### A.3 引用验证检查表

| 文档类型 | 最小引用数 | 必需引用类型 |
|----------|------------|--------------|
| 算法定义 | 3 | 原始论文、经典教材、最新研究 |
| 数据结构 | 2 | 原始来源、标准教材 |
| 定理 | 1 | 原始证明来源 |
| 复杂度分析 | 1 | 分析方法来源 |
| 性质/引理 | 1 | 证明来源 |

## B. 形式化程度检查

### B.1 数学符号标准化

- [ ] **变量命名**: 使用斜体（如 $n$, $T(n)$, $f(x)$）
- [ ] **常数命名**: 使用正体（如 $\mathrm{e}$, $\pi$, $\mathrm{i}$）
- [ ] **函数命名**: 使用正体（如 $\log$, $\sin$, $\exp$）
- [ ] **集合符号**: 使用黑板体（如 $\mathbb{N}$, $\mathbb{R}$）
- [ ] **复杂度符号**: 使用正确的 $O$, $\Theta$, $\Omega$ 记号

#### ISO 80000-2 对齐检查

| 符号类别 | 正确示例 | 错误示例 | 说明 |
|----------|----------|----------|------|
| 标准数集 | $\mathbb{N}, \mathbb{Z}, \mathbb{R}$ | $N, Z, R$ | 使用黑板体 |
| 逻辑符号 | $\land, \lor, \neg$ | `and`, `or`, `not` | 使用标准符号 |
| 集合运算 | $\cup, \cap, \setminus$ | `union`, `intersect` | 使用标准符号 |
| 复杂度 | $O(n), \Theta(n\log n)$ | $O(n)$, $θ(n log n)$ | 使用大写希腊字母 |

### B.2 形式化定义格式

- [ ] **定义结构**: 包含定义编号、名称、来源引用
- [ ] **形式化表示**: 使用数学符号精确定义
- [ ] **前置条件**: 明确说明输入约束
- [ ] **后置条件**: 明确说明输出保证
- [ ] **复杂度声明**: 包含时间和空间复杂度

#### 形式化定义检查模板

```markdown
**定义 X.X.X** (名称) [来源]  ← 检查点1: 编号连续、有来源
简洁描述。

**形式化定义 / Formal Definition:**

设...可以形式化定义为:        ← 检查点2: 使用数学符号
$$数学表达式$$

其中:                         ← 检查点3: 符号解释完整
- $符号1$ 是...
- $符号2$ 是...

**前置条件 / Precondition**:    ← 检查点4: 前置条件完整
- 条件1
- 条件2

**后置条件 / Postcondition**:   ← 检查点5: 后置条件完整
- 条件1
- 条件2

**复杂度 / Complexity:**        ← 检查点6: 复杂度分析
- **时间复杂度**: $T(n) = O(f(n))$  ← 检查点7: 使用Theta表示精确复杂度
- **空间复杂度**: $S(n) = O(g(n))$
```

### B.3 证明结构检查

- [ ] **定理陈述**: 清晰陈述定理条件与结论
- [ ] **证明结构**: 使用标准证明结构（声明→证明→∎）
- [ ] **推理步骤**: 每一步推理都有依据
- [ ] **数学归纳**: 归纳证明包含基例和归纳步骤
- [ ] **反证法**: 反证法明确假设和矛盾点

#### 证明质量检查表

| 检查项 | 标准 | 评分 |
|--------|------|------|
| 证明完整性 | 包含所有必要步骤 | ☐ 完整 ☐ 有缺漏 |
| 逻辑严密性 | 无逻辑跳跃 | ☐ 严密 ☐ 有跳跃 |
| 符号使用 | 符号使用一致 | ☐ 一致 ☐ 有混淆 |
| 引用正确 | 引用的引理/定理正确 | ☐ 正确 ☐ 需核实 |
| 结论清晰 | 结论明确无误 | ☐ 清晰 ☐ 模糊 |

## C. 内容完整性检查

### C.1 文档结构符合模板

- [ ] **文档头部**: 包含标题、版本、状态、日期
- [ ] **摘要**: 包含2-3条要点概述
- [ ] **术语表**: 定义关键术语和符号
- [ ] **目录**: 包含完整目录结构
- [ ] **交叉引用**: 列出相关上下游文档
- [ ] **正文**: 按主题分节
- [ ] **参考文献**: 列出所有引用

#### 文档结构检查清单


| 文档类型 | 定义 | 定理 | 证明 | 示例 | 复杂度分析 | 伪代码 |
|----------|------|------|------|------|------------|--------|
| 算法文档 | ✅ | ✅ | ⭕ | ✅ | ✅ | ✅ |
| 数据结构 | ✅ | ⭕ | ⭕ | ✅ | ✅ | ✅ |

**图例**: ✅ 必需 | ⭕ 推荐 | ❌ 不需要

### C.3 交叉引用正确性

- [ ] **内部引用**: 引用本文档内其他章节
- [ ] **外部引用**: 引用其他文档的特定章节
- [ ] **引用有效性**: 所有引用的链接有效
- [ ] **引用准确性**: 引用内容与实际内容一致

#### 交叉引用检查流程

```
步骤1: 提取文档中所有链接和引用
       [文本](路径) 或 #章节锚点

步骤2: 验证内部引用
       - 章节锚点是否存在
       - 链接文本是否准确

步骤3: 验证外部引用
       - 文件路径是否正确
       - 目标文件是否存在

步骤4: 验证引用一致性
       - 引用描述与目标内容是否一致
```

### C.4 中英文术语对照

- [ ] **术语一致性**: 同一术语全文使用相同翻译
- [ ] **首次出现**: 术语首次出现时提供中英文对照
- [ ] **缩写定义**: 缩写首次出现时给出全称
- [ ] **术语表完整**: 所有术语在术语表中定义

- [ ] **拼写检查**: 无拼写错误

### D.2 逻辑结构

- [ ] **层次分明**: 使用正确的标题层级（H1-H6）
- [ ] **段落长度**: 每段不超过5-6句
- [ ] **过渡流畅**: 段落间过渡自然
- [ ] **逻辑连贯**: 论证逻辑清晰
- [ ] **重点突出**: 关键信息醒目

#### 结构评分标准

| 维度 | 优秀(5) | 良好(4) | 合格(3) | 需改进(2) | 不合格(1) |

| 层次性 | 结构完美 |  minor issues | 结构合理 | 层次混乱 | 无结构 |
| 连贯性 | 流畅自然 | 基本流畅 |  understandable | 跳跃明显 | 不连贯 |
| 重点性 | 重点突出 | 基本清晰 | 可辨识 | 重点模糊 | 无重点 |

### D.3 示例充分性

- [ ] **概念示例**: 每个抽象概念有具体示例
- [ ] **算法示例**: 算法有执行示例
- [ ] **边界示例**: 包含边界情况示例
- [ ] **反例**: 必要时提供反例
- [ ] **可视化**: 复杂概念有图示

#### 示例质量检查表

| 示例类型 | 检查标准 | 状态 |
|----------|----------|------|
| 概念示例 | 清晰说明概念要点 | ☐ |
| 算法示例 | 逐步展示执行过程 | ☐ |
| 边界示例 | 展示边界条件处理 | ☐ |
| 反例 | 说明什么情况下不成立 | ☐ |
| 图示 | 使用Mermaid或ASCII图 | ☐ |

### D.4 公式排版

- [ ] **行内公式**: 使用 `$\cdots$` 格式
- [ ] **独立公式**: 使用 `$$...$$` 格式
- [ ] **公式编号**: 重要公式有编号
- [ ] **符号定义**: 公式中符号首次出现时定义
- [ ] **公式对齐**: 多行公式对齐整齐

#### 公式排版检查示例

```markdown
✅ 正确示例:
时间复杂度为 $T(n) = O(n \log n)$。   ← 行内公式

重要公式:
$$T(n) = 2T(n/2) + O(n) \tag{1}$$   ← 独立公式+编号

其中 $T(n)$ 表示...                  ← 符号定义

❌ 错误示例:
时间复杂度为 T(n) = O(n log n)。     ← 未使用行内公式

T(n) = 2T(n/2) + O(n)               ← 未使用独立公式格式
(1)
```

## F. 快速检查工具

### F.1 自动化检查脚本

```bash
#!/bin/bash
# 质量检查自动化脚本

echo "=== 文档质量自动检查 ==="

# 检查1: 空引用标记
echo "[检查1] 查找空引用标记..."
grep -n '\[\]' *.md || echo "✅ 未发现空引用"

# 检查2: 查找TODO标记
echo "[检查2] 查找TODO标记..."
grep -ni 'TODO' *.md || echo "✅ 未发现TODO"

# 检查3: 检查死链（简化版）
echo "[检查3] 检查Markdown链接格式..."
grep -n '\[.*\](\s*)' *.md || echo "✅ 链接格式正确"

# 检查4: 检查中英文标点混用
echo "[检查4] 检查中英文标点..."
grep -n '[a-zA-Z]，' *.md || echo "✅ 未发现中英文标点混用"

echo "=== 检查完成 ==="
```

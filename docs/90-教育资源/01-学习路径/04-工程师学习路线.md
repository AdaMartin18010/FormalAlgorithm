## 🎯 第一阶段：工程基础（2-4周）

### 代码质量基础

```python
# 好的算法代码示例
def binary_search(arr: list[int], target: int) -> int:
    """
    二分查找

    Args:
        arr: 已排序数组
        target: 目标值

    Returns:
        目标索引，不存在返回 -1

    Time: O(log n), Space: O(1)
    """
    left, right = 0, len(arr) - 1

    while left <= right:
        mid = left + (right - left) // 2  # 防止溢出
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1
```

## 复杂度分析实践

### 复杂度分析实践

| 场景 | 时间复杂度 | 空间复杂度 | 是否可接受 |
|-----|-----------|-----------|-----------|
| 单次请求处理 | O(n) | O(n) | ✅ 通常可接受 |
| 批量数据处理 | O(n²) | O(n²) | ⚠️ n<10k 可接受 |
| 实时系统 | O(log n) | O(1) | ✅ 推荐 |
| 大数据处理 | O(n log n) | O(n) | ⚠️ 需评估 |

### 生产级代码示例

```python
# 生产环境：带LRU缓存的斐波那契
from functools import lru_cache
import threading

class FibonacciCalculator:
    def __init__(self, cache_size=128):
        self._cache = {}
        self._cache_size = cache_size
        self._lock = threading.RLock()

    def calculate(self, n: int) -> int:
        """线程安全的斐波那契计算"""
        if n < 0:
            raise ValueError("n must be non-negative")

        with self._lock:
            if n in self._cache:
                return self._cache[n]

        result = self._fib(n)

        with self._lock:
            if len(self._cache) < self._cache_size:
                self._cache[n] = result

        return result

    @lru_cache(maxsize=1024)
    def _fib(self, n: int) -> int:
        if n < 2:
            return n
        return self._fib(n-1) + self._fib(n-2)
```

## 💡 工程师算法学习建议

1. **以用促学**：从实际工作中遇到的问题出发学习算法
2. **重视实现**：不仅理解原理，更要写出健壮的代码
3. **性能意识**：时刻关注时间/空间复杂度
4. **工具优先**：优先使用成熟库，必要时才自研
5. **持续优化**：代码上线后持续监控和优化

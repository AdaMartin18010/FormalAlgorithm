# 示例库总纲 (Examples Library Overview)


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

## 概述

本文档库是 FormalAlgorithm 项目的核心组成部分，提供算法理论的形式化示例、可运行实例和反例集合。通过系统化的示例组织，帮助学习者从理论到实践全面理解算法科学。

---

## 库结构

```
docs/示例库/
├── 示例总纲.md                 # 本文件：整体概述与使用指南
├── 01-标准示例集.md            # 50个核心主题的3-5个标准示例
├── 02-反例集.md                # 常见错误与反例
├── 03-实例数据.md              # 分级测试实例规范
├── 04-可视化规范.md            # 可视化标准与指南
└── instances/                  # 实际数据文件
    ├── sorting/                # 排序算法实例
    │   ├── small/              # n ≤ 10
    │   ├── medium/             # n ≤ 1000
    │   └── large/              # n ≤ 10^6
    ├── graphs/                 # 图算法实例
    ├── search/                 # 搜索算法实例
    ├── dp/                     # 动态规划实例
    ├── strings/                # 字符串算法实例
    └── trees/                  # 树结构实例
```

## 五大类别

### 1. 标准示例 (Standard Examples)

**位置**: `01-标准示例集.md`

**目的**: 为每个核心理论主题提供规范、完整的示例

**内容特征**:

- 清晰的输入/输出定义
- 逐步执行追踪 (Step-by-step trace)
- 多语言实现 (Rust/Python)
- 严格的复杂度分析
- 形式化证明草图

**覆盖范围**: 50个核心主题

| 篇章 | 主题数 | 主要内容 |
|------|--------|----------|
| 第一篇：基础理论与数学基础 | 10 | 集合运算、数论基础、渐近分析、递归方程等 |
| 第二篇：算法设计范式 | 8 | 分治、DP、贪心、回溯、分支限界、随机、近似、在线算法 |
| 第三篇：核心数据结构 | 8 | 数组、链表、树、堆、哈希表、并查集、图表示等 |
| 第四篇：图算法 | 8 | 遍历、最短路径、MST、网络流、匹配、强连通分量等 |
| 第五篇：字符串算法 | 6 | 字符串匹配、后缀结构、编辑距离、压缩等 |
| 第六篇：高级算法 | 10 | 计算几何、FFT、数论算法、矩阵算法、量子算法等 |

**使用方式**:

```
学习路径: 选择主题 → 阅读形式化定义 → 追踪执行步骤 → 运行代码 → 分析复杂度
```

### 2. 反例集 (Counterexamples)

**位置**: `02-反例集.md`

**目的**: 揭示常见误解，强化正确理解

**内容类型**:

#### A. 算法误解反例

- **Dijkstra与负权边**: 展示Dijkstra在负权边下的失败
- **贪心算法失败案例**: 0-1背包、加权区间调度
- **快速排序最坏情况**: 已排序输入的性能退化
- **哈希表最坏情况**: 所有键哈希到同一桶

#### B. 证明技术失败

- 归纳法基础案例缺失（"所有马同色"悖论）
- 循环不变式构造错误
- 势函数设计缺陷

#### C. 复杂度类分离

- P vs NP 边界实例
- 伪多项式 vs 多项式时间
- 多项式层次结构

#### D. 类型系统局限

- 类型推导失败案例
- 所有权系统边界（Rust双向链表）
- 递归类型定义限制

**格式规范**:

```markdown
### 反例 X.Y: [主题]

**常见错误**: [描述错误的普遍认知]
**反例构造**: [具体的反例]
**错误分析**: [为什么这个反例有效]
**正确理解**: [修正后的认知]
**相关定理**: [形式化结果]
```

### 3. 实例数据 (Instance Data)

**位置**: `03-实例数据.md` + `instances/`

**目的**: 提供可运行的、分级组织的测试数据

**分级体系**:

| 级别 | 规模 | 用途 | 文件格式 |
|------|------|------|----------|
| **Small** | n ≤ 10 | 手工追踪、教学演示 | `.json`, `.txt` |
| **Medium** | n ≤ 1000 | 单元测试、算法验证 | `.json`, `.csv` |
| **Large** | n ≤ 10⁶ | 性能测试、压力测试 | `.bin`, `.gz` |
| **Extreme** | n > 10⁶ | 极限测试、大数据 | `.parquet` |

**数据类别**:

#### 排序数据

- 随机、有序、逆序、部分有序
- 含重复元素
- 不同数据类型（整数、浮点、字符串）

#### 图数据

- 稀疏图、稠密图
- 特殊结构（树、网格、完全图）
- 带权图、负权边、负权环

#### 搜索数据

- 均匀分布、偏斜分布
- 高维数据
- 边界值分布

#### DP数据

- 经典问题实例（背包、LCS、编辑距离）
- 随机生成实例
- 最坏情况实例

#### 字符串数据

- 自然语言文本
- DNA序列
- 二进制数据

**文件命名规范**:

```
[类别]_[类型]_[规模]_[序号].[格式]

示例:
- sorting_random_small_001.json
- graph_weighted_medium_005.csv
- dp_knapsack_large_003.json
```

### 4. 可视化规范 (Visualization Guidelines)

**位置**: `04-可视化规范.md`

**目的**: 统一可视化标准，提升算法理解效率

**涵盖内容**:

#### 可视化设计原则

- 清晰性、一致性、简洁性
- 层次性、时序性

#### 颜色编码规范

- 算法状态颜色（未访问、当前、已访问等）
- 数据结构颜色
- 色盲友好设计

#### 数据结构可视化

- 数组、链表的ASCII表示
- 树结构（二叉树、堆）的层次布局
- 图结构（有向/无向、带权）的节点-边表示

#### 算法执行可视化

- 排序算法的交换/比较动画
- 图搜索的遍历顺序
- 动态规划的表格填充

#### 复杂度图表规范

- 复杂度比较曲线
- 运行时间实测图

#### 工具与格式

- 推荐工具：Manim、Graphviz、Matplotlib、D3.js
- 输出格式：PNG/SVG、GIF/MP4、HTML5

## 使用指南

### 对于学习者

#### 入门阶段

```
1. 阅读 示例总纲.md 了解整体结构
2. 查阅 01-标准示例集.md 学习核心算法
3. 使用 Small 实例进行手工追踪
4. 运行代码验证理解
```

#### 实践阶段

```
1. 选择感兴趣的算法主题
2. 运行 Medium 实例进行测试
3. 修改输入观察输出变化
4. 分析实际运行时间与理论复杂度的关系
```

#### 深入阶段

```
1. 研究 02-反例集.md
2. 理解算法的边界条件和限制
3. 尝试构造自己的反例
4. 分析算法失败的原因
```

### 对于教师

#### 课堂演示

- 使用 Small 实例进行逐步演示
- 参考 04-可视化规范 制作课件
- 使用 ASCII 艺术在板书/幻灯片中展示

#### 作业设计

- 基于 Medium 实例设计编程作业
- 使用反例集设计辨析题目
- 要求学生构造特定场景的反例

#### 项目实践

- 使用 Large 实例进行性能基准测试
- 对比不同算法在相同数据上的表现
- 分析实际应用中的算法选择

### 对于研究者

#### 算法验证

- 使用标准示例验证新算法正确性
- 在 Large 实例上进行性能评估
- 对比分析不同算法的实际表现

#### 基准测试

- 使用统一的实例库确保公平比较
- 记录详细的性能指标
- 发布可复现的实验结果

## 质量保障

### 示例质量标准

1. **正确性**: 所有示例经过形式化验证或测试验证
2. **完整性**: 包含输入、输出、过程、分析四要素
3. **一致性**: 符号、术语与项目整体保持一致
4. **可读性**: 代码注释完整，解释清晰
5. **可运行**: 提供可直接执行的代码

### 实例数据标准

1. **代表性**: 覆盖典型情况和边界情况
2. **多样性**: 不同分布、不同特征的数据集合
3. **可复现**: 随机数据提供种子值
4. **可扩展**: 支持生成更大规模数据

### 反例质量标准

1. **针对性**: 针对明确的误解或错误认知
2. **最小性**: 构造尽可能小的反例
3. **清晰性**: 错误分析和正确理解明确
4. **教育性**: 帮助建立正确直觉

## 贡献指南

### 添加新示例

1. 确认示例符合项目主题范围
2. 遵循标准模板格式
3. 提供完整的形式化描述
4. 包含多种实现语言
5. 通过质量检查清单

### 添加新实例

1. 选择适当的类别和规模级别
2. 遵循数据格式规范
3. 提供数据生成脚本（如适用）
4. 更新实例索引文档

### 添加新反例

1. 明确说明误解内容
2. 构造最小反例
3. 分析错误原因
4. 给出正确理解

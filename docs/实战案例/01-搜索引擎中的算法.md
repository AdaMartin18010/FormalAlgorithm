## 2. 倒排索引构建

### 2.1 倒排索引结构

倒排索引（Inverted Index）是搜索引擎的核心数据结构，它将词项映射到包含该词项的文档列表。

```rust
/// 倒排索引核心结构
pub struct InvertedIndex {
    /// term -> Posting List
    index: HashMap<String, PostingList>,
    /// 文档总数
    doc_count: usize,
    /// 词项统计信息
    term_stats: HashMap<String, TermStats>,
}

/// 倒排列表（包含文档信息和位置信息）
pub struct PostingList {
    /// 文档频率（DF）
    doc_freq: usize,
    /// 文档列表
    postings: Vec<Posting>,
}

/// 单个文档的倒排信息
pub struct Posting {
    doc_id: DocId,
    term_freq: u32,           // TF
    positions: Vec<u32>,      // 位置信息（用于短语查询）
    field_mask: u8,           // 字段标记（标题、正文等）
}

/// 词项统计
pub struct TermStats {
    /// 文档频率
    doc_freq: usize,
    /// 总词频（所有文档中出现次数之和）
    total_freq: usize,
    /// IDF 值（预计算）
    idf: f64,
}
```

### 2.2 索引构建流程

```rust
impl InvertedIndex {
    /// 从文档集合构建倒排索引
    pub fn build_from_docs(docs: Vec<Document>) -> Self {
        let mut temp_index: HashMap<String, Vec<(DocId, Vec<u32>)>> = HashMap::new();
        
        // Phase 1: 解析文档，生成临时倒排记录
        for (doc_id, doc) in docs.iter().enumerate() {
            let tokens = Self::tokenize(&doc.content);
            
            for (pos, token) in tokens.iter().enumerate() {
                temp_index
                    .entry(token.clone())
                    .or_default()
                    .push((doc_id as DocId, vec![pos as u32]));
            }
        }
        
        // Phase 2: 排序并合并倒排列表
        let mut index = HashMap::new();
        let mut term_stats = HashMap::new();
        
        for (term, mut postings) in temp_index {
            // 按 doc_id 排序
            postings.sort_by_key(|p| p.0);
            
            // 合并相同 doc_id 的记录
            let merged = Self::merge_postings(&postings);
            
            let df = merged.len();
            let total_freq: usize = merged.iter()
                .map(|p| p.term_freq as usize)
                .sum();
            
            // 计算 IDF: log(N / DF)
            let idf = ((docs.len() as f64) / (df as f64)).ln();
            
            term_stats.insert(
                term.clone(),
                TermStats { doc_freq: df, total_freq, idf }
            );
            
            index.insert(term, PostingList {
                doc_freq: df,
                postings: merged,
            });
        }
        
        InvertedIndex {
            index,
            doc_count: docs.len(),
            term_stats,
        }
    }
    
    /// 分词处理
    fn tokenize(text: &str) -> Vec<String> {
        text.to_lowercase()
            .split_whitespace()
            .map(|s| s.trim_matches(|c: char| !c.is_alphanumeric()).to_string())
            .filter(|s| !s.is_empty())
            .collect()
    }
}
```

```rust
    /// SPIMI 算法：单遍内存索引构建
    pub fn build_spimi(&self, doc_iter: impl Iterator<Item = Document>) -> Result<InvertedIndex> {
        let mut block_files = Vec::new();
        let mut buffer: HashMap<String, Vec<(DocId, Vec<u32>)>> = HashMap::new();
        let mut block_id = 0;
        
        for (doc_id, doc) in doc_iter.enumerate() {
            // 解析文档加入缓冲区
            self.add_to_buffer(&mut buffer, doc_id as DocId, &doc);
            
            // 缓冲区满，写入磁盘
            if self.buffer_full(&buffer) {
                let block_file = self.write_block(&mut buffer, block_id)?;
                block_files.push(block_file);
                block_id += 1;
                buffer.clear();
            }
        }
        
        // 处理剩余数据
        if !buffer.is_empty() {
            let block_file = self.write_block(&mut buffer, block_id)?;
            block_files.push(block_file);
        }
        
        // 合并所有块文件
        self.merge_blocks(&block_files)
```

```rust
impl InvertedIndex {
    /// 处理 AND 查询（交集）
    pub fn intersect(&self, terms: &[&str]) -> Vec<DocId> {
        if terms.is_empty() {
            return Vec::new();
        }
        
        // 按 DF 从小到大排序，优先处理最短的列表
        let mut posting_lists: Vec<&PostingList> = terms
            .iter()
            .filter_map(|t| self.index.get(*t))
            .collect();
        
        posting_lists.sort_by_key(|pl| pl.doc_freq);
        
        if posting_lists.is_empty() {
            return Vec::new();
        }
        
        // 从最短列表开始
        let mut result: Vec<DocId> = posting_lists[0]
            .postings
            .iter()
            .map(|p| p.doc_id)
            .collect();
        
        // 与其他列表求交集
        for pl in &posting_lists[1..] {
            result = Self::intersect_two(&result, pl);
            if result.is_empty() {
                break; // 早期终止
            }
        }
        
        result
    }
    
    /// 线性时间求两个有序列表的交集
    fn intersect_two(sorted_docs: &[DocId], pl: &PostingList) -> Vec<DocId> {
        let mut result = Vec::new();
        let mut i = 0;
        let mut j = 0;
        
        while i < sorted_docs.len() && j < pl.postings.len() {
            match sorted_docs[i].cmp(&pl.postings[j].doc_id) {
                Ordering::Less => i += 1,
                Ordering::Greater => j += 1,
                Ordering::Equal => {
                    result.push(sorted_docs[i]);
                    i += 1;
                    j += 1;
                }
            }
        }
        
        result
    }
```

## 4. 相关性排序算法

### 4.1 TF-IDF 模型

```rust
/// TF-IDF 相关性计算
pub struct TfIdfRanker;

impl TfIdfRanker {
    /// 计算文档与查询的相关性得分
    pub fn score(&self, index: &InvertedIndex, query_terms: &[String], doc_id: DocId) -> f64 {
        let mut score = 0.0;
        
        for term in query_terms {
            if let Some(pl) = index.index.get(term) {
                if let Some(posting) = pl.postings.iter().find(|p| p.doc_id == doc_id) {
                    let tf = Self::compute_tf(posting.term_freq);
                    let idf = index.term_stats.get(term).map(|s| s.idf).unwrap_or(0.0);
                    score += tf * idf;
                }
            }
        }
        
        score
    }
    
    /// 对数词频平滑
    fn compute_tf(raw_tf: u32) -> f64 {
        (raw_tf as f64).ln_1p() + 1.0
    }
}
```

### 4.2 BM25 算法

BM25 是工业界最广泛使用的相关性排序算法：

```rust
/// BM25 排序器
pub struct BM25Ranker {
    /// 调节参数 k1
    k1: f64,
    /// 调节参数 b
    b: f64,
    /// 平均文档长度
    avgdl: f64,
}

impl BM25Ranker {
    pub fn new(k1: f64, b: f64, total_docs: usize, total_tokens: usize) -> Self {
        BM25Ranker {
            k1,
            b,
            avgdl: total_tokens as f64 / total_docs as f64,
        }
    }
    
    /// BM25 评分公式
    /// score(D, Q) = Σ IDF(q_i) * (f(q_i, D) * (k1 + 1)) / 
    ///               (f(q_i, D) + k1 * (1 - b + b * |D| / avgdl))
    pub fn score(&self, index: &InvertedIndex, query_terms: &[String], doc: &Document) -> f64 {
        let mut score = 0.0;
        let doc_len = doc.content.len() as f64;
        
        for term in query_terms {
            if let (Some(pl), Some(stats)) = (index.index.get(term), index.term_stats.get(term)) {
                if let Some(posting) = pl.postings.iter().find(|p| p.doc_id == doc.id) {
                    let tf = posting.term_freq as f64;
                    let idf = Self::compute_idf(index.doc_count, stats.doc_freq);
                    
                    // BM25 公式
                    let numerator = tf * (self.k1 + 1.0);
                    let denominator = tf + self.k1 * (1.0 - self.b + self.b * doc_len / self.avgdl);
                    
                    score += idf * numerator / denominator;
                }
            }
        }
        
        score
    }
    
    /// IDF 计算（多种变体）
    fn compute_idf(total_docs: usize, doc_freq: usize) -> f64 {
        // 标准 BM25 IDF
        let n = doc_freq as f64;
        let N = total_docs as f64;
        ((N - n + 0.5) / (n + 0.5) + 1.0).ln()
    }
}
```

### 4.3 Top-K 检索优化

```rust
/// 使用堆优化获取 Top-K 结果
pub fn top_k_search(
    &self,
    index: &InvertedIndex,
    query: &Query,
    k: usize,
) -> Vec<(DocId, f64)> {
    use std::collections::BinaryHeap;
    use std::cmp::Ordering;
    
    #[derive(Clone, Copy)]
    struct ScoredDoc {
        doc_id: DocId,
        score: f64,
    }
    
    impl Ord for ScoredDoc {
        fn cmp(&self, other: &Self) -> Ordering {
            self.score.partial_cmp(&other.score)
                .unwrap_or(Ordering::Equal)
        }
    }
    
    impl PartialOrd for ScoredDoc {
        fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
            Some(self.cmp(other))
        }
    }
    
    impl Eq for ScoredDoc {}
    impl PartialEq for ScoredDoc {
        fn eq(&self, other: &Self) -> bool {
            self.score == other.score
        }
    }
    
    let mut heap = BinaryHeap::with_capacity(k + 1);
    let candidate_docs = self.get_candidates(index, query);
    
    for doc_id in candidate_docs {
        let score = self.score_document(index, query, doc_id);
        
        if heap.len() < k {
            heap.push(ScoredDoc { doc_id, score });
        } else if score > heap.peek().unwrap().score {
            heap.pop();
            heap.push(ScoredDoc { doc_id, score });
        }
    }
    
    // 转换为结果列表
    let mut result: Vec<_> = heap.into_iter()
        .map(|s| (s.doc_id, s.score))
        .collect();
    result.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());
    
    result
}
```

```rust
/// 基于编辑距离的拼写纠错
pub struct SpellChecker {
    dictionary: HashSet<String>,
    /// 常用词加权
    term_freq: HashMap<String, usize>,
}

impl SpellChecker {
    /// 使用 Levenshtein 距离查找最相似的词
    pub fn suggest(&self, query: &str, k: usize) -> Vec<(String, usize)> {
        let mut candidates: Vec<(String, usize)> = self
            .dictionary
            .iter()
            .map(|word| {
                let dist = Self::levenshtein(query, word);
                (word.clone(), dist)
            })
            .filter(|(_, d)| *d <= 2) // 距离阈值
            .collect();
        
        // 按距离排序，距离相同按词频排序
        candidates.sort_by(|a, b| {
            a.1.cmp(&b.1)
                .then_with(|| {
                    let fa = self.term_freq.get(&a.0).unwrap_or(&0);
                    let fb = self.term_freq.get(&b.0).unwrap_or(&0);
                    fb.cmp(fa)
                })
        });
        
        candidates.truncate(k);
        candidates
    }
    
    /// 编辑距离动态规划
    fn levenshtein(s1: &str, s2: &str) -> usize {
        let s1: Vec<char> = s1.chars().collect();
        let s2: Vec<char> = s2.chars().collect();
        let m = s1.len();
        let n = s2.len();
        
        let mut dp = vec![vec![0; n + 1]; m + 1];
        
        for i in 0..=m {
            dp[i][0] = i;
        }
        for j in 0..=n {
            dp[0][j] = j;
        }
        
        for i in 1..=m {
            for j in 1..=n {
                if s1[i - 1] == s2[j - 1] {
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    dp[i][j] = 1 + dp[i - 1][j - 1] // 替换
                        .min(dp[i - 1][j])       // 删除
                        .min(dp[i][j - 1]);      // 插入
                }
            }
        }
        
        dp[m][n]
    }
}
```

## 7. 形式化规范

### 7.1 倒排索引规范

```yaml
数据结构规范 - 倒排索引:
  定义:
    - 给定文档集合 D = {d₁, d₂, ..., dₙ}
    - 词项集合 T = {t₁, t₂, ..., tₘ}
    - 倒排索引 I: T → 2^(D × ℕ)
  
  不变式:
    - ∀t ∈ T, I(t) 按文档 ID 有序
    - ∀(d, freq) ∈ I(t), freq = |{pos: term_at(d, pos) = t}|
  
  操作复杂度:
    - 构建: O(Σ|d|)
    - 查询: O(|I(t)|)
    - 交集: O(Σ|I(tᵢ)|)
```

### 7.2 相关性函数规范

```yaml
算法规范 - BM25:
  输入:
    - 查询 Q = {q₁, q₂, ..., qₖ}
    - 文档 d
    - 索引统计信息
  
  输出:
    - 相关性分数 score(Q, d) ∈ ℝ⁺
  
  公式:
    score(Q, d) = Σᵢ IDF(qᵢ) × TF(qᵢ, d)
    
    IDF(q) = log((N - n(q) + 0.5) / (n(q) + 0.5))
    
    TF(q, d) = f(q, d) × (k₁ + 1) / 
               (f(q, d) + k₁ × (1 - b + b × |d| / avgdl))
  
  性质:
    - 单调性: f(q, d₁) > f(q, d₂) ⇒ TF(q, d₁) > TF(q, d₂)
    - 饱和性: lim_{f→∞} TF = k₁ + 1
    - 长度归一化: 长文档不会仅因长度获得优势
```

## 8. 总结与延伸

### 8.1 核心知识点回顾

1. **倒排索引**：搜索引擎的核心数据结构，支持高效的词项检索
2. **查询处理**：布尔检索、短语查询、通配符查询的实现
3. **相关性排序**：TF-IDF、BM25 等经典模型的理论与实践
4. **性能优化**：缓存、并行、近似算法的应用

### 8.2 现代搜索引擎的发展趋势

| 方向 | 技术 | 特点 |
|------|------|------|
| 语义搜索 | 词向量、BERT | 理解查询意图，超越关键词匹配 |
| 个性化 | 用户画像、协同过滤 | 千人千面的搜索结果 |
| 实时性 | 流式索引、增量更新 | 秒级内容收录 |
| 多模态 | 图文检索、视频检索 | 跨模态内容理解 |

### 8.3 实践建议

1. 从小规模数据开始，逐步扩展到海量数据
2. 重视评估体系：精确率、召回率、NDCG 等指标
3. A/B 测试驱动：小规模验证新算法的有效性
4. 监控与日志：建立完善的搜索质量监控体系

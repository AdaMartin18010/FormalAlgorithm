# 案例：线段树在股票价格区间查询与日程安排冲突检测中的应用

## 背景

在金融交易系统和日程管理软件中，需要频繁地进行**区间查询**和**区间更新**操作：

- 查询某只股票在过去一段时间内的最高价/最低价/成交量
- 检查会议室在指定时间段内是否已被预订
- 统计广告在特定时段的曝光次数

线段树（Segment Tree）以 $O(\log n)$ 的时间支持单点修改、区间查询和区间更新，是这类场景的标准数据结构。

## 问题定义

- **数组**：$A[1..n]$，表示 $n$ 个时间点的数据（如每小时股价）
- **操作**：
  - `update(i, val)`：将 $A[i]$ 更新为 $val$
  - `query(l, r)`：查询区间 $[l, r]$ 的聚合值（max/min/sum/count）
- **目标**：每次操作 $O(\log n)$

## 线段树结构

### 构建

```
build(node, l, r):
    if l == r: tree[node] = A[l]
    else:
        mid = (l + r) / 2
        build(left, l, mid)
        build(right, mid+1, r)
        tree[node] = merge(tree[left], tree[right])
```

### 查询

```
query(node, l, r, ql, qr):
    if ql <= l and r <= qr: return tree[node]
    mid = (l + r) / 2
    res = identity
    if ql <= mid: res = merge(res, query(left, l, mid, ql, qr))
    if qr > mid:  res = merge(res, query(right, mid+1, r, ql, qr))
    return res
```

### 复杂度

- **构建**：$O(n)$
- **查询/更新**：$O(\log n)$
- **空间**：$O(4n)$

## 实际应用

### 股票价格分析

- 实时查询任意时间段的最高/最低价
- 区间求和计算成交量
- 支持实时更新（新交易到达）

### 日程安排系统

- 区间更新：预订/取消会议室
- 区间查询：检查某时段是否空闲
- _lazy propagation_ 优化批量更新

### 在线广告系统

- 统计每小时的广告曝光、点击、转化
- 实时查询任意时段的聚合指标
- 支持数据修正（反作弊后调整）

### 游戏开发

- 碰撞检测（空间划分线段树）
- 视野计算（范围查询）

## 与树状数组（BIT）的对比

| 功能 | 线段树 | 树状数组 |
|------|--------|---------|
| 单点更新 + 前缀查询 | ✅ $O(\log n)$ | ✅ $O(\log n)$ |
| 区间查询 | ✅ $O(\log n)$ | ⚠️ 需两次前缀差 |
| 区间更新 + 单点查询 | ✅ Lazy 传播 | ✅ 差分 + BIT |
| 区间更新 + 区间查询 | ✅ Lazy 传播 | ❌ 不支持 |
| 代码复杂度 | 中等 | 简单 |
| 常数因子 | 较大 | 较小 |

## 效果评估

- 对 1,000,000 个数据点，每次查询 < 0.1ms
- 树状数组在仅需要前缀和时，比线段树快 2-3 倍
- 线段树的灵活性使其成为通用区间问题的首选

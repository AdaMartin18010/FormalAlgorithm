# 案例：并查集在社交网络好友圈与图连通性分析中的应用

## 背景

社交网络中，用户之间通过"好友"关系形成庞大的图结构。需要高效地回答：

- 用户 A 和用户 B 是否在同一个好友圈中？
- 当前有多少个互不相交的社交圈子？
- 两个圈子合并后，新圈子的代表是谁？

并查集（Disjoint Set Union, DSU）以近乎 $O(1)$ 的均摊时间支持这些操作，是处理动态连通性问题的标准数据结构。

## 问题定义

- **初始**：$n$ 个用户，每人自成一个独立集合
- **操作**：
  - `union(a, b)`：将用户 a 和 b 所在的好友圈合并
  - `find(a)`：找到用户 a 所在好友圈的代表（root）
  - `connected(a, b)`：判断 a 和 b 是否在同一个好友圈

## 并查集优化

### 路径压缩（Path Compression）

```
find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])  // 压缩路径
    return parent[x]
```

### 按秩合并（Union by Rank）

```
union(x, y):
    rx = find(x), ry = find(y)
    if rx == ry: return
    if rank[rx] < rank[ry]: swap(rx, ry)
    parent[ry] = rx
    if rank[rx] == rank[ry]: rank[rx]++
```

### 复杂度

- **均摊**：$O(\alpha(n))$，$\alpha$ 为反阿克曼函数，实际 $< 5$ 对所有合理 $n$
- **近乎常数时间**

## 实际应用

### 社交网络

- Facebook 好友圈检测
- LinkedIn 二度/三度人脉分析
- 社区发现（Union-Find 作为预处理）

### 图连通性

- 最小生成树（Kruskal 算法的核心）
- 网络可靠性分析（边的故障与连通性）
- 迷宫生成（随机合并相邻格子）

### 图像处理

- 连通区域标记（Connected Component Labeling）
- 二值图像中的对象计数

### LeetCode 经典题

| 题目 | 应用 |
|------|------|
| 省份数量 | 城市连通性 |
| 冗余连接 | 图中添加边形成环 |
| 最长连续序列 | 数值连通性（离散并查集）|
| 被围绕的区域 | 边界连通性分析 |

## 与 BFS/DFS 的对比

| 场景 | 并查集 | BFS/DFS |
|------|--------|---------|
| 动态加边 + 连通查询 | ✅ $O(\alpha(n))$ | ❌ 需重新遍历 |
| 静态图全部分析 | ⚠️ 可行 | ✅ 更自然 |
| 最小生成树 | ✅ Kruskal 必需 | ❌ 不适用 |
| 路径查询 | ❌ 无路径信息 | ✅ 可记录路径 |

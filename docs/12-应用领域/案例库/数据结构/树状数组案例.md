# 案例：树状数组在实时排行榜与频次统计中的应用

## 背景

在线游戏、电商平台和社交媒体都需要实时维护动态排行榜：用户得分变化时，快速查询某个用户的排名、前 K 名、或某个分数区间的用户数。树状数组（Fenwick Tree / Binary Indexed Tree）以 $O(\log n)$ 的更新和查询效率，成为这类频率统计问题的理想数据结构。

## 问题定义

- **输入**：用户得分动态变化（增加/减少）
- **操作**：
  1. 更新某用户的得分（单点更新）
  2. 查询得分 $\leq x$ 的用户数（前缀和）
  3. 查询得分在 $[l, r]$ 区间的用户数（区间和）
  4. 查询排名第 $k$ 的用户的得分（第 $k$ 小）

## 树状数组的应用

### 频率统计

将分数范围离散化为 $[1..M]$，维护每个分数的出现次数：

- `add(score, 1)`：用户得分增加
- `add(old_score, -1)`：用户旧得分减少
- `prefixSum(x)`：得分 $\leq x$ 的用户数

### 第 $k$ 小查询

利用树状数组的可加性进行二分：

```
low = 1, high = M
while low < high:
    mid = (low + high) / 2
    if prefixSum(mid) < k: low = mid + 1
    else: high = mid
return low
```

时间：$O(\log^2 M)$，或使用树状数组上二分优化至 $O(\log M)$。

## 与线段树的对比

| 操作 | 树状数组 | 线段树 |
|------|---------|--------|
| 单点更新 | $O(\log n)$ | $O(\log n)$ |
| 前缀和 | $O(\log n)$ | $O(\log n)$ |
| 区间和 | $O(\log n)$ | $O(\log n)$ |
| 区间最值 | ❌ 不支持 | ✅ $O(\log n)$ |
| 代码量 | ~20 行 | ~80 行 |
| 常数因子 | 极小 | 中等 |

## 实际系统

### 游戏排行榜

- 每次击杀/任务完成更新玩家积分
- 树状数组维护积分分布，支持实时排名查询
- 对 1000 万玩家，单次更新/查询 < 1μs

### 电商销量统计

- 实时统计各价格区间的商品销量
- 支持价格调整后的动态更新

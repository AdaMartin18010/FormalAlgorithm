# 案例：BFS在社交网络平台的好友推荐与影响力传播中的应用

## 背景

社交网络（如 Facebook、LinkedIn、微博）的核心价值在于连接人与人。BFS（广度优先搜索）是这类平台最基础也最频繁使用的算法之一：发现好友关系、计算分离度、分析信息传播路径。

## 问题定义

- **图**：用户为顶点，好友关系为无向边
- **查询**：
  1. 用户 $u$ 的 $k$ 度好友（距离 $\leq k$ 的所有用户）
  2. 用户 $u$ 和 $v$ 之间的最短路径长度
  3. 从用户 $u$ 出发，信息传播到全网需要多少步

## BFS 的应用

### 好友推荐

"你可能认识的人"功能：

- 对目标用户 $u$ 执行 BFS 到深度 2
- 深度 1：直接好友（已连接，不推荐）
- 深度 2：好友的好友（推荐候选）
- 按共同好友数排序，Top-N 推荐

**优化**：不存储完整图，使用分布式图数据库（如 Neo4j、TigerGraph）实时查询。

### 六度分隔理论验证

Milgram 的实验假设：任意两人通过不超过 6 个中间人即可连接。

- Facebook（2016）：平均分离度 = 3.57
- LinkedIn：平均分离度 ≈ 3-4
- BFS 是验证这一指标的基准算法

### 影响力传播

在病毒营销中：

- 从种子用户出发 BFS，标记每层覆盖的用户
- 计算传播深度和覆盖人数
- 评估不同种子集合的传播效果

## 与 DFS 的对比

| 场景 | BFS | DFS |
|------|-----|-----|
| 最短路径 | ✅ 天然支持 | ❌ 不保证最短 |
| 好友推荐 | ✅ 按距离分层 | ❌ 可能深入无关分支 |
| 连通性检测 | ✅ | ✅ |
| 内存占用 | $O(b^d)$，较大 | $O(bm)$，较小 |

## 分布式 BFS

对十亿级用户网络：

- **Pregel / Giraph**：Google 的分布式图计算框架
- 每轮迭代：所有活跃顶点向其邻居发送消息
- 迭代轮数 = 图直径（社交网络通常 < 10）

## 效果评估

- 对百万节点子图，单机 BFS < 100ms
- 分布式 BFS 在 10 亿节点图上，数秒内完成 3-4 轮迭代
- 好友推荐系统的 BFS 查询占社交平台读流量的 20-30%

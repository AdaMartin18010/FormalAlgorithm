# 案例：拓扑排序在构建系统与课程选修规划中的应用

## 背景

软件构建系统（Make、Bazel、Webpack）、大学课程规划和数据管道中，存在大量的**依赖关系**：必须先完成 A 才能进行 B。拓扑排序将这类偏序关系线性化，提供合法的执行顺序。

## 问题定义

- **输入**：有向无环图（DAG）$G=(V,E)$，边 $u \to v$ 表示 $u$ 必须在 $v$ 之前
- **输出**：顶点的一个线性排列，使得所有边从左指向右

## 算法

### Kahn 算法

1. 计算所有顶点的入度
2. 将入度为 0 的顶点加入队列
3. 依次取出，将其邻居入度减 1；若邻居入度变为 0，入队
4. 若输出顶点数 $< |V|$，说明存在环

**复杂度**：$O(V + E)$

### DFS 后序

1. DFS 遍历图
2. 顶点退出递归时加入结果列表前端
3. 逆序即为拓扑序

## 实际应用

### 软件构建系统

| 工具 | 用途 |
|------|------|
| Make | 根据 Makefile 依赖规则决定编译顺序 |
| Bazel | 大规模代码库的目标依赖分析 |
| Webpack | 模块依赖图解析与打包顺序 |
| npm/yarn | 包依赖解析与安装顺序 |
| Docker | 镜像层构建顺序 |

**示例**：C++ 项目中，头文件修改后，拓扑排序确定哪些 `.cpp` 文件需要重新编译。

### 课程规划

- 大学课程有先修要求（如"数据结构"先修"程序设计"）
- 拓扑排序给出合法的学期选课方案
- 若存在环（如 A 先修 B，B 先修 A），说明课程设计有矛盾

### 数据管道（ETL）

- 数据仓库中，表之间存在依赖（如"日汇总"依赖"小时明细"）
- Airflow、dbt 等工具使用 DAG 定义 pipeline
- 拓扑排序确定任务执行顺序，并行化无依赖任务

### 电子表格计算

- Excel、Google Sheets 中，单元格公式可能引用其他单元格
- 拓扑排序确定计算顺序，避免循环引用

### 游戏任务系统

- RPG 中任务有前置条件
- 拓扑排序给出任务解锁顺序
- 检测任务依赖是否形成死锁

## 与关键路径的对比

| 问题 | 输入 | 输出 | 复杂度 |
|------|------|------|--------|
| 拓扑排序 | DAG | 合法线性序 | $O(V+E)$ |
| 关键路径 | DAG + 权重 | 最长路径、松弛时间 | $O(V+E)$ |

**项目管理**：拓扑排序给出执行顺序，关键路径分析确定项目最短工期。

## 效果评估

- 对 100,000 个模块的依赖图，拓扑排序 < 10ms
- Bazel 使用拓扑排序 + 增量分析，仅重新编译受影响的模块
- 数据管道中，拓扑排序识别可并行任务，提升吞吐量 3-10 倍

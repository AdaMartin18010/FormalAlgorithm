# 案例：筛法在密码学与哈希函数设计中的应用

## 背景

素数是密码学和哈希函数设计的基石：RSA 依赖大素数的不可分解性，椭圆曲线密码需要特定形式的素数（如梅森素数、Solinas 素数），而哈希表的设计也常利用素数性质减少碰撞。筛法（Sieve）作为高效生成素数集合的算法，在这些领域有重要应用。

## 埃氏筛与现代变体

### 埃拉托色尼筛（Sieve of Eratosthenes）

时间：$O(n \log \log n)$，空间：$O(n)$

### 线性筛（Euler's Sieve）

每个合数仅被其最小质因子筛去一次，时间严格 $O(n)$：

```
for i = 2 to n:
    if not marked[i]: primes.append(i)
    for p in primes:
        if i * p > n: break
        marked[i * p] = true
        if i % p == 0: break
```

### 分段筛（Segmented Sieve）

对区间 $[L, R]$ 内的素数，只需筛到 $\sqrt{R}$ 的素数即可：

- 先生成 $[2, \sqrt{R}]$ 的素数
- 用这些素数标记 $[L, R]$ 中的合数
- 空间降至 $O(\sqrt{R} + (R-L))$

## 密码学应用

### RSA 素数生成

需要生成 $1024$-$4096$ 位的大素数：

- 不能直接用筛法（范围太大）
- 随机选择候选数 + Miller-Rabin 测试
- 但筛法用于预筛选小素因子（ trial division），排除明显合数

### 安全素数（Safe Prime）

$p = 2q + 1$，其中 $q$ 也是素数。用于 Diffie-Hellman 和 DSA：

- 先筛出候选 $q$，再验证 $p=2q+1$
- 防止 Pohlig-Hellman 攻击

### 椭圆曲线参数

某些标准曲线（如 NIST 曲线）使用特定形式的素数域：

- $p = 2^{256} - 2^{224} + 2^{192} + 2^{96} - 1$（Solinas 素数）
- 允许快速模约简，但需验证其素性

## 哈希表设计

### 素数大小的桶数组

选择素数 $p$ 作为哈希表大小：

- 对乘法哈希 $h(k) = (A \cdot k) \mod p$，分布更均匀
- 减少周期性模式导致的聚集
- 常见选择：略大于预期元素数的素数

## 效果评估

- 线性筛在 1 秒内可生成 $10^8$ 以内的所有素数
- 分段筛可在 MB 级内存下处理 $10^{12}$ 区间的素数检测
- 密码学中的 trial division 通常只测试前 100-1000 个小素数，耗时可忽略

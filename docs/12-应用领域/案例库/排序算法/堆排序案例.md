# 案例：堆排序在任务调度与优先级队列中的应用

## 背景

操作系统、分布式系统和实时应用中，任务需要按优先级执行。堆（Heap）数据结构提供 $O(\log n)$ 的插入和提取最值操作，是优先级队列的标准实现。堆排序（Heap Sort）则是基于堆的原地排序算法，适合内存受限场景。

## 堆的核心操作

### 二叉堆

- **插入**：放入末尾，上浮（sift up），$O(\log n)$
- **提取最值**：取出根，末尾元素放根，下沉（sift down），$O(\log n)$
- **建堆**：Floyd 建堆，$O(n)$

### 堆排序

1. 建最大堆
2. 重复：交换堆顶与末尾，堆大小减 1，下沉新堆顶
3. 数组变为升序

**复杂度**：$O(n \log n)$，空间 $O(1)$（原地）

## 实际应用

### 操作系统任务调度

- **Linux CFS**：使用红黑树（非堆，但类似有序结构）
- **实时系统**：固定优先级调度，堆管理就绪队列
- **Windows**：线程优先级队列

### 图算法

- **Dijkstra**：优先队列（堆）选择最小距离顶点
- **Prim MST**：堆选择最小权边
- **A***：堆管理 $f(v) = g(v) + h(v)$

### 数据流处理

- **Top-K**：维护大小为 $k$ 的最小堆，流式找最大 $k$ 个元素
- **中位数**：两个堆（最大堆 + 最小堆）维护动态中位数
- **滑动窗口最大值**：双端队列或堆优化

### 合并 K 个有序数组

- 将每个数组当前最小元素放入最小堆
- 每次取出堆顶，将该数组下一个元素入堆
- 复杂度 $O(N \log K)$，$N$ 为总元素数

## 与快速排序的对比

| 特性 | 堆排序 | 快速排序 |
|------|--------|---------|
| 最坏情况 | $O(n \log n)$ ✅ | $O(n^2)$ |
| 平均情况 | $O(n \log n)$ | $O(n \log n)$，常数更小 |
| 额外空间 | $O(1)$ ✅ | $O(\log n)$ |
| 稳定性 | ❌ 不稳定 | ❌ 不稳定 |
| 缓存友好 | ❌ 随机访问 | ✅ 局部性好 |
| 实际速度 | 较慢 | 更快（缓存优势）|

**选择建议**：

- 内存极度受限：堆排序
- 通用排序：快速排序（或 Timsort）
- 需要优先级操作：堆结构

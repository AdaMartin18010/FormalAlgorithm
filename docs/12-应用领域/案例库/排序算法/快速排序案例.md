# 快速排序实际应用案例

## 应用场景1：通用内存排序库实现

### 问题描述

- **背景**：某编程语言标准库需要提供一个通用的、高性能的内存排序函数
- **问题**：需要支持任意可比较数据类型，且在大多数数据分布下性能优秀
- **约束**：
  - 时间约束：最坏情况下不能退化到O(n²)
  - 空间约束：递归栈深度可控
  - 稳定性：不要求稳定（可用稳定化包装）
- **数据规模**：从10个元素到1000万元素

### 算法选择理由

- **为什么选择快速排序**：
  - 平均情况下cache性能最优
  - 原地排序，空间复杂度O(log n)
  - 可轻松实现为通用模板
  - 现代优化版本（Introsort）可保证O(n log n)最坏情况

- **与其他算法的对比**：

  | 算法 | 平均时间 | 最坏时间 | 空间 | Cache效率 | 代码复杂度 |
  |------|---------|---------|------|-----------|-----------|
  | 快速排序 | O(n log n) | O(n²) | O(log n) | **最优** | 低 |
  | 归并排序 | O(n log n) | O(n log n) | O(n) | 好 | 中 |
  | 堆排序 | O(n log n) | O(n log n) | O(1) | 差 | 低 |

### 解决方案

```pseudo
Algorithm Introsort(A, begin, end, depth_limit):
    size = end - begin

    // 小规模使用插入排序
    if size <= 16:
        InsertionSort(A, begin, end)
        return

    // 递归深度限制，超过则转堆排序
    if depth_limit == 0:
        HeapSort(A, begin, end)
        return

    // 三数取中 pivot
    pivot = MedianOfThree(A[begin], A[begin + size/2], A[end-1])

    // 三路划分（荷兰国旗问题）
    (lt, gt) = ThreeWayPartition(A, begin, end, pivot)

    // 递归排序
    Introsort(A, begin, lt, depth_limit - 1)
    Introsort(A, gt, end, depth_limit - 1)

Algorithm ThreeWayPartition(A, begin, end, pivot):
    lt = begin      // less than pivot
    gt = end - 1    // greater than pivot
    i = begin

    while i <= gt:
        if A[i] < pivot:
            swap(A[lt], A[i])
            lt += 1
            i += 1
        else if A[i] > pivot:
            swap(A[i], A[gt])
            gt -= 1
        else:
            i += 1

    return (lt, gt + 1)
```

### 实际代码示例（Rust）

```rust
/// 工业级快速排序实现（类似std::sort）
pub fn quicksort<T: Ord>(arr: &mut [T]) {
    introsort(arr, 2 * (arr.len() as f64).log2() as usize);
}

fn introsort<T: Ord>(arr: &mut [T], max_depth: usize) {
    const INSERTION_THRESHOLD: usize = 16;

    if arr.len() <= INSERTION_THRESHOLD {
        insertion_sort(arr);
        return;
    }

    if max_depth == 0 {
        heap_sort(arr);
        return;
    }

    let (lt, gt) = three_way_partition(arr);
    let len = arr.len();

    // 先排序较小的分区，保证栈深度O(log n)
    if lt < len - gt {
        introsort(&mut arr[..lt], max_depth - 1);
        introsort(&mut arr[gt..], max_depth - 1);
    } else {
        introsort(&mut arr[gt..], max_depth - 1);
        introsort(&mut arr[..lt], max_depth - 1);
    }
}

fn three_way_partition<T: Ord>(arr: &mut [T]) -> (usize, usize) {
    let len = arr.len();
    let pivot = median_of_three(&mut arr[0], &mut arr[len/2], &mut arr[len-1]);

    let mut lt = 0;
    let mut gt = len - 1;
    let mut i = 0;

    while i <= gt {
        if arr[i] < *pivot {
            arr.swap(lt, i);
            lt += 1;
            i += 1;
        } else if arr[i] > *pivot {
            arr.swap(i, gt);
            if gt == 0 { break; }
            gt -= 1;
        } else {
            i += 1;
        }
    }

    (lt, gt + 1)
}

fn median_of_three<T: Ord>(a: &mut T, b: &mut T, c: &mut T) -> &T {
    if *a > *b { std::mem::swap(a, b); }
    if *b > *c { std::mem::swap(b, c); }
    if *a > *b { std::mem::swap(a, b); }
    b
}

fn insertion_sort<T: Ord>(arr: &mut [T]) {
    for i in 1..arr.len() {
        let mut j = i;
        while j > 0 && arr[j-1] > arr[j] {
            arr.swap(j-1, j);
            j -= 1;
        }
    }
}
```

## 应用场景2：数据库查询优化器中的排序

### 问题描述

- **背景**：分布式数据库需要在多个节点并行排序后归并
- **问题**：单节点排序效率低，需要并行化
- **约束**：
  - 需要充分利用多核CPU
  - 内存带宽瓶颈
- **数据规模**：单节点100GB，共10节点

### 解决方案

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
import numpy as np

class ParallelQuickSort:
    """并行快速排序实现"""

    def __init__(self, num_threads: int = 8):
        self.num_threads = num_threads
        self.threshold = 10000  # 低于此阈值使用串行排序

    def parallel_sort(self, arr: np.ndarray) -> np.ndarray:
        """并行快速排序主函数"""
        if len(arr) <= self.threshold:
            return np.sort(arr)

        # 采样确定分割点
        samples = np.random.choice(arr, min(1000, len(arr)), replace=False)
        pivot = np.median(samples)

        # 分区
        less = arr[arr < pivot]
        equal = arr[arr == pivot]
        greater = arr[arr > pivot]

        # 并行递归
        with ThreadPoolExecutor(max_workers=self.num_threads) as executor:
            futures = []
            if len(less) > 0:
                futures.append(executor.submit(self.parallel_sort, less))
            if len(greater) > 0:
                futures.append(executor.submit(self.parallel_sort, greater))

            results = [f.result() for f in as_completed(futures)]

        # 合并结果
        if len(less) > 0 and len(greater) > 0:
            return np.concatenate([results[0], equal, results[1]])
        elif len(less) > 0:
            return np.concatenate([results[0], equal])
        else:
            return np.concatenate([equal, results[0]])

# 基准测试
import time

def benchmark_parallel():
    data = np.random.randint(0, 1_000_000_000, size=10_000_000)

    # 串行排序
    start = time.time()
    sorted_serial = np.sort(data.copy())
    serial_time = time.time() - start

    # 并行排序
    sorter = ParallelQuickSort(num_threads=8)
    start = time.time()
    sorted_parallel = sorter.parallel_sort(data.copy())
    parallel_time = time.time() - start

    print(f"串行排序: {serial_time:.3f}s")
    print(f"并行排序: {parallel_time:.3f}s")
    print(f"加速比: {serial_time/parallel_time:.2f}x")

if __name__ == '__main__':
    benchmark_parallel()
```

### 性能评估

- **并行加速比**：4-6x（8线程）
- **适用场景**：大规模数据分析、OLAP查询
- **实际应用**：ClickHouse、Presto、Spark

# 案例：计数排序在高考成绩排名与频次统计中的应用

## 背景

在处理大规模离散数据排名时，如果数据的值域范围不大（如考试分数 0-750、年龄段 0-120），计数排序（Counting Sort）可以提供 $O(n + k)$ 的线性时间排序，远快于基于比较的 $O(n \log n)$ 算法。

## 问题定义

- **输入**：$n$ 个考生的高考成绩（范围 0-750）
- **输出**：按成绩排序的考生列表，以及每个成绩的累计人数
- **扩展**：快速回答"成绩在 $[l, r]$ 区间内的考生有多少人"

## 计数排序流程

### 1. 计数

```
count[0..750] = 0
for each score:
    count[score]++
```

### 2. 前缀和（累计排名）

```
for i = 1 to 750:
    count[i] += count[i-1]
```

`count[s]` 现在表示成绩 $\leq s$ 的考生人数，即成绩 $s$ 的最高排名。

### 3. 输出排序

```
for i = n-1 downto 0:
    output[count[score[i]] - 1] = student[i]
    count[score[i]]--
```

## 实际应用

### 高考排名

- 全国上千万考生，成绩范围 0-750
- 计数排序在 $O(n + 750)$ 内完成全省/全国排名
- 可实时回答任意分数段的位次查询

### 图像直方图均衡化

- 像素值范围 0-255
- 计数排序统计各灰度级频次
- 前缀和计算累积分布函数（CDF），实现直方图均衡

### DNA 序列分析

- 碱基只有 4 种（A, C, G, T）
- 计数排序在 $O(n)$ 内完成碱基频率统计
- 用于 GC 含量计算、序列比对预处理

## 与快速排序的对比

| 场景 | 计数排序 | 快速排序 |
|------|---------|---------|
| 范围小（$k < 1000$）| ✅ $O(n)$，极快 | $O(n \log n)$ |
| 范围大（$k \gg n$）| ❌ $O(k)$ 过大 | ✅ $O(n \log n)$ |
| 稳定性 | ✅ 稳定 | ❌ 不稳定 |
| 额外空间 | $O(k)$ | $O(\log n)$ |
| 浮点数 | ❌ 需离散化 | ✅ 直接支持 |

- 对 1000 万考生成绩排序，计数排序 < 1 秒
- 快速排序约 3-5 秒
- 高考排名系统实际使用计数排序或其变种（桶排序）

# 案例：舞蹈链（DLX）在数独求解与课程表排程中的应用

## 背景

舞蹈链算法（Dancing Links, DLX）是 Donald Knuth 提出的高效实现**精确覆盖问题（Exact Cover）**的算法。它使用双向链表的巧妙结构，使得回溯时的撤销操作（uncover）与执行操作（cover）同样高效。数独、八皇后、课程表排程等问题都可归约为精确覆盖。

## 精确覆盖问题

给定集合 $X$ 和 $X$ 的子集族 $S$，选择一个子集族 $S^* \subseteq S$，使得：

- $S^*$ 中子集两两不相交
- $S^*$ 的并集等于 $X$

即 $X$ 的每个元素恰好被 $S^*$ 中的一个子集覆盖。

## 舞蹈链数据结构

### 双向循环链表

每个节点 $x$ 有四个指针：左、右、上、下。删除和恢复操作仅需常数时间：

```
// Cover: 删除列及其所有行
column.left.right = column.right
column.right.left = column.left
for row in column.down_loop:
    for node in row.right_loop:
        node.up.down = node.down
        node.down.up = node.up

// Uncover: 完全逆向恢复
```

### 算法 X（Knuth）

1. 若矩阵为空，找到解
2. 选择列数最少的列（启发式：减少分支因子）
3. 对该列的每一行：
   - Cover 该行及与其冲突的所有行
   - 递归求解
   - Uncover 恢复

## 数独求解

### 归约

将 9x9 数独约束编码为精确覆盖：

- $4 \times 9^2 = 324$ 个约束（每行、每列、每宫、每格各一个数字）
- $9^3 = 729$ 种可能的放置（行×列×数）
- 每个放置覆盖 4 个约束

求解 = 选择一组放置，恰好覆盖所有约束。

### 效果

- 最难的数独（17 提示数）可在毫秒级求解
- 求解所有 5,472,730,538 个本质上不同的数独终盘约需数天

## 课程表排程

### 约束

- 教师：同一时间只能上一门课
- 教室：容量、设备匹配
- 学生：不能同时上两门课
- 时间：偏好、连续课程限制

### 建模

- $X$：所有需要满足的"槽位"（教师-时间、教室-时间、学生-时间）
- $S$：每个可行的（课程-教师-教室-时间）组合

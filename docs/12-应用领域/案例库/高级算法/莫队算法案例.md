# 案例：莫队算法在日志分析与区间统计中的应用

## 背景

大规模分布式系统每天产生数十亿条日志记录。运维工程师需要回答大量离线区间查询："在 14:00-15:00 期间，有多少种不同的错误类型？"、"在上周一至周三，访问频率最高的 IP 是哪个？"。莫队算法（Mo's Algorithm）通过重新排列查询顺序，将单次查询的增量更新代价均摊到最低。

## 问题定义

- **输入**：长度为 $n$ 的数组 $A$（如日志序列），$q$ 个区间查询 $[L_i, R_i]$
- **查询类型**：区间不同元素个数、区间众数、区间频率统计
- **约束**：$n, q \leq 10^5$ 或 $10^6$

## 莫队算法的核心思想

### 离线重排

将查询按块排序（块大小 $B \approx \sqrt{n}$）：

- 左端点 $L$ 在同一块内的查询，按右端点 $R$ 排序
- 这样相邻查询的 $[L, R]$ 移动距离最小

### 增量维护

维护当前区间 $[curL, curR]$ 的答案，通过添加/删除元素调整：

```
while curL > q.L: add(--curL)
while curR < q.R: add(++curR)
while curL < q.L: remove(curL++)
while curR > q.R: remove(curR--)
```

## 复杂度分析

- 块数：$\sqrt{n}$
- 同一块内，$R$ 单调移动，总移动 $O(n)$
- $L$ 在同一块内移动范围 $\leq \sqrt{n}$
- 总复杂度：$O((n + q) \sqrt{n})$

对于 $n = q = 10^5$，约 $3 \times 10^7$ 次操作，可在 1 秒内完成。

## 实际应用

### 日志去重统计

对时间序列日志查询不同错误码数量：

- 用哈希表维护当前区间的频率
- 莫队重排后批量回答查询

### 基因组学：区间多样性

在 DNA 序列上查询子串的碱基多样性（不同碱基种类数）。

### 数据库 OLAP

预计算困难、但查询可离线的分析场景，莫队可作为近似或精确解法。

## 扩展：带修改的莫队

若数组元素会动态修改，引入时间维度，将查询扩展为 $(L, R, time)$，分块大小调整为 $n^{2/3}$，复杂度 $O(q \cdot n^{2/3})$。

## 效果评估

- 对 $10^5$ 次查询、$10^5$ 长度的数组，莫队算法比暴力 $O(nq)$ 快 1000 倍以上
- 在可离线场景下，与线段树解法速度相当，但实现更简单

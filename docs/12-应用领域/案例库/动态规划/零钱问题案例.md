# 案例：零钱问题在自动售货机与找零系统中的应用

## 背景

自动售货机、收银系统和 ATM 都需要解决一个经典的优化问题：给定一组面额和需要找零的金额，如何用最少数量的硬币/纸币完成找零？这正是**零钱问题（Coin Change）**的直接应用。

## 问题定义

- **输入**：硬币面额集合 $C = \{c_1, c_2, ..., c_m\}$，目标金额 $A$
- **输出**：组成金额 $A$ 所需的最少硬币数（或判断无法组成）

## 动态规划解法

设 $dp[x]$ 为组成金额 $x$ 所需的最少硬币数：

$$dp[x] = \min_{c_i \leq x} \{ dp[x - c_i] + 1 \}$$

边界：$dp[0] = 0$，$dp[x] = \infty$（初始）

### 复杂度

- 时间：$O(m \cdot A)$
- 空间：$O(A)$

## 贪心算法的陷阱

若面额为 {1, 3, 4}，金额 6：

- 贪心：4 + 1 + 1 = 3 枚
- 最优：3 + 3 = 2 枚

**结论**：贪心仅在规范硬币系统（如人民币、美元）中最优，一般问题需用 DP。

## 实际系统

### 货币找零

| 货币 | 面额 | 算法 |
|------|------|------|
| 人民币 | 1, 5, 10, 20, 50, 100 | 贪心即可 |
| 美元 | 1, 5, 10, 25 | 贪心即可 |
| 旧英国币 | 1, 2, 6, 12, 24, 48, 60 | 需 DP |

### 自动售货机

- 机器内各面额硬币/纸币数量有限
- 扩展为**有限供应的背包问题**
- 需在找零成功率和硬币库存间平衡

### 切割优化

在制造业中，从标准长度的原材料切割出所需长度的零件，本质是多维零钱问题。

## 效果评估

- 对日常金额（$A \leq 1000$），DP 在微秒级完成
- 在 ATM 系统中，结合贪心 + DP fallback，平均找零时间 < 1ms

# 案例：最长公共子序列在文本差异比较与版本控制中的应用

## 背景

版本控制系统（Git）、文档协作工具和 DNA 序列比对中，需要比较两个版本的内容差异。最长公共子序列（LCS）是计算两个序列差异的基础算法。

## 问题定义

- **输入**：两个序列 $A[1..m]$，$B[1..n]$
- **输出**：最长的公共子序列长度（及序列本身）
- **子序列**：不要求连续，但要求相对顺序一致

## 动态规划

```
dp[i][j] = A[1..i] 和 B[1..j] 的 LCS 长度

if A[i] == B[j]: dp[i][j] = dp[i-1][j-1] + 1
else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

**复杂度**：$O(mn)$ 时间，$O(mn)$ 空间（可优化至 $O(\min(m,n))$）

## 差异计算（Diff）

LCS 找到后，差异通过对比 LCS 与原始序列得到：

- 在 A 中但不在 LCS 中 → **删除**
- 在 B 中但不在 LCS 中 → **增加**
- 在 LCS 中 → **不变**

## 实际应用

### 版本控制（Git）

- `git diff` 比较文件版本差异
- Myers 差分算法（$O(ND)$，$D$ 为差异数）基于 LCS 思想优化
- 行级 diff：每行作为序列元素

### 文档协作

- Google Docs、Notion 的冲突合并
- 实时协作的 OT/CRDT 算法使用差异计算

### DNA 序列比对

- 比较两个物种的相似基因序列
- LCS 变体：允许替换操作（编辑距离更常用）
- BLAST 等工具使用启发式加速

### 拼写检查与自动补全

- 计算输入与词典词汇的 LCS
- 推荐 LCS 最长的候选词

## 与编辑距离的对比

| 度量 | 操作 | 关系 |
|------|------|------|
| LCS | 插入、删除 | - |
| 编辑距离 | 插入、删除、替换 | $ED = m + n - 2 \cdot LCS$（仅插入删除时）|

**Git diff** 实际使用编辑距离变体（允许替换 = 先删后增）。

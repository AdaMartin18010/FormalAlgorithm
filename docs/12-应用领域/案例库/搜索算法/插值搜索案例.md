# 案例：插值搜索在IP地址路由表查找中的应用

## 背景

互联网路由器的核心功能是将目的 IP 地址映射到输出端口。
现代核心路由器需要每秒处理数百万个数据包，每个数据包都需要一次路由表查找。
插值搜索（Interpolation Search）在路由表的均匀分布前缀上，可以提供接近 $O(\log \log n)$ 的查找效率。

## 问题定义

- **输入**：排序的路由表条目（IP 前缀、下一跳、出端口）
- **查询**：给定目的 IP 地址，找到最长匹配前缀
- **约束**：查找时间 < 100ns（支持线速转发）

## 插值搜索的优势

### IP 地址的分布特性

IPv4 地址空间（32 位）在实际路由表中并非均匀分布，但在特定前缀长度内（如 /24 段内）呈现较好的局部均匀性。

### 位置估计

对排序的路由表 $R[0..n-1]$，查找 IP $x$：

$$pos = low + \left\lfloor \frac{x - R[low].ip}{R[high].ip - R[low].ip} \times (high - low) \right\rfloor$$

在均匀分布下，期望比较次数 $O(\log \log n)$。

## 实际系统

### 多级查找

现代路由器通常使用多级结构：

1. **直接索引**：对 /24 前缀使用 2^24 的直接查找表
2. **Trie / Patricia Tree**：对更长前缀进行树形查找
3. **插值搜索**：在中间层级的压缩表中使用

### Binary Search on Ranges

路由表本质是区间集合，查找 = 区间定位：

- 标准二分：$O(\log n)$
- 插值搜索：$O(\log \log n)$（均匀假设下）
- 实际中常与哈希结合，实现 $O(1)$ 平均

## 与硬件的结合

- **TCAM（Ternary CAM）**：并行匹配所有条目，$O(1)$ 但功耗高、成本高
- **SRAM + 算法**：用插值搜索或哈希在 SRAM 中实现，成本低、功耗小

## 效果评估

- 对 100 万条路由表项，插值搜索平均 3-4 次比较
- 二分搜索平均 20 次比较
- 在软件路由器（如 Linux 内核）中，结合前缀压缩后可达百万级查找/秒

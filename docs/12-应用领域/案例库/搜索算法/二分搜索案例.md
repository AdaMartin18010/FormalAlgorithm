## 应用场景1：数据库索引查询优化

### 问题描述

- **背景**：关系型数据库需要高效支持主键和索引列的等值查询和范围查询
- **问题**：B+树索引在内存中的节点查找需要高效实现
- **约束**：
  - 时间约束：单点查询 < 1ms
  - 空间约束：索引节点大小固定（通常4KB或16KB）
  - 磁盘I/O：尽量减少磁盘读取次数
- **数据规模**：单表10亿条记录，索引节点约1000万个

### 算法选择理由

- **为什么选择二分搜索**：
  - B+树节点内使用二分搜索，O(log m)，m为节点内键数
  - Cache友好，连续内存访问
  - 代码简单，分支预测友好
  - 支持快速的范围查询边界定位

- **与其他算法的对比**：

  | 算法 | 时间复杂度 | 空间复杂度 | Cache效率 | 适用场景 |
  |------|-----------|-----------|-----------|----------|
  | 线性搜索 | O(m) | O(1) | 好 | m < 50 |
  | 二分搜索 | **O(log m)** | **O(1)** | **好** | **通用** |
  | 插值搜索 | O(log log m) | O(1) | 好 | 数据分布均匀 |

### 解决方案

```pseudo
Algorithm BTreeSearch(root, key):
    node = root
    while node is not leaf:
        // 在内部节点使用二分搜索定位子节点
        index = BinarySearch(node.keys, key)
        node = node.children[index]

    // 在叶子节点二分搜索
    index = BinarySearch(node.keys, key)
    if index < len(node.keys) and node.keys[index] == key:
        return node.values[index]
    return null

Algorithm BinarySearch(keys, target):
    left = 0
    right = len(keys)

    while left < right:
        mid = left + (right - left) / 2
        if keys[mid] < target:
            left = mid + 1
        else:
            right = mid

    return left  // 返回插入位置或查找位置

// 范围查询：找到左边界后顺序遍历
Algorithm RangeQuery(root, startKey, endKey):
    // 二分搜索定位起始位置
    leaf = FindLeafNode(root, startKey)
    index = BinarySearch(leaf.keys, startKey)

    result = []
    while leaf is not null:
        for i from index to len(leaf.keys):
            if leaf.keys[i] > endKey:
                return result
            result.append((leaf.keys[i], leaf.values[i]))
        leaf = leaf.next
        index = 0

    return result
```

```rust
/// B+树索引节点内的二分搜索实现
pub struct BTreeNode<K: Ord, V> {
    keys: Vec<K>,
    values: Vec<V>,
    children: Option<Vec<Box<BTreeNode<K, V>>>>,
    next: Option<Box<BTreeNode<K, V>>>,
}

impl<K: Ord, V> BTreeNode<K, V> {
    /// 标准二分搜索，返回键的位置或插入位置
    pub fn binary_search(&self, key: &K) -> Result<usize, usize> {
        self.keys.binary_search(key)
    }

    /// 分支预测优化的二分搜索
    pub fn binary_search_branchless(&self, key: &K) -> usize
    where K: Copy + Ord {
        let mut left = 0;
        let mut len = self.keys.len();

        while len > 1 {
            let half = len / 2;
            // 减少分支预测失败
            left += (self.keys[left + half] < *key) as usize * half;
            len -= half;
        }

        left + (self.keys.get(left).map_or(false, |k| k < key) as usize)
    }

    /// SIMD优化的二分搜索（适用于大节点）
    #[cfg(target_arch = "x86_64")]
    pub fn simd_search(&self, key: &K) -> usize
    where K: Copy + Ord + Into<i64> {
        // 使用AVX2进行向量比较
        // 实际实现需使用unsafe和std::arch模块
        self.binary_search(key).unwrap_or_else(|i| i)
    }
}

/// 数据库索引查询器
pub struct DatabaseIndex<K: Ord, V> {
    root: BTreeNode<K, V>,
}

impl<K: Ord + Copy, V: Clone> DatabaseIndex<K, V> {
    pub fn new() -> Self {
        Self {
            root: BTreeNode {
                keys: Vec::new(),
                values: Vec::new(),
                children: None,
                next: None,
            }
        }
    }

    /// 点查询
    pub fn get(&self, key: &K) -> Option<V> {
        let mut node = &self.root;

        // 遍历到叶子节点
        while let Some(ref children) = node.children {
            let idx = node.binary_search(key).unwrap_or_else(|i| i);
            node = &children[idx.min(children.len() - 1)];
        }

        // 叶子节点二分搜索
        match node.binary_search(key) {
            Ok(idx) => Some(node.values[idx].clone()),
            Err(_) => None,
        }
    }

    /// 范围查询 [start, end)
    pub fn range(&self, start: &K, end: &K) -> Vec<(K, V)>
    where K: Clone {
        let mut result = Vec::new();
        let mut node = self.find_leaf(start);
        let mut idx = node.binary_search(start).unwrap_or_else(|i| i);

        loop {
            for i in idx..node.keys.len() {
                if &node.keys[i] >= end {
                    return result;
                }
                result.push((node.keys[i].clone(), node.values[i].clone()));
            }

            match node.next {
                Some(ref next) => {
                    node = next;
                    idx = 0;
                }
                None => break,
            }
        }

        result
    }

    fn find_leaf(&self, key: &K) -> &BTreeNode<K, V> {
        let mut node = &self.root;
        while let Some(ref children) = node.children {
            let idx = node.binary_search(key).unwrap_or_else(|i| i);
            node = &children[idx.min(children.len() - 1)];
        }
        node
    }
}
```

## 应用场景2：版本控制系统中的代码定位

### 问题描述

- **背景**：Git版本控制系统需要在海量代码历史中快速定位文件版本
- **问题**：需要支持按提交时间、作者、内容哈希等多维度查询
- **约束**：
  - 时间约束：git blame < 100ms
  - 空间约束：元数据存储优化
  - 增量更新：支持增量索引
- **数据规模**：Linux内核仓库，50万提交，2000万对象

### 算法选择理由

- **为什么选择二分搜索**：
  - Git的提交历史天然有序（时间戳）
  - 支持二分查找引入bug的提交（git bisect）
  - Pack文件索引使用二分搜索定位对象

### 解决方案

```python
import hashlib
from typing import List, Optional, Dict, Tuple
import bisect

class GitObjectStore:
    """简化版Git对象存储，使用二分搜索定位"""

    def __init__(self):
        # 按偏移排序的索引: [(hash_prefix, offset), ...]
        self.pack_index: List[Tuple[bytes, int]] = []
        self.objects: Dict[bytes, bytes] = {}  # hash -> content

    def add_object(self, content: bytes) -> bytes:
        """添加对象，返回哈希"""
        obj_hash = hashlib.sha1(content).digest()
        self.objects[obj_hash] = content
        return obj_hash

    def build_index(self, pack_file: List[Tuple[bytes, bytes]]):
        """构建pack文件索引"""
        offset = 0
        for obj_hash, content in sorted(pack_file, key=lambda x: x[0]):
            self.pack_index.append((obj_hash, offset))
            offset += len(content)

    def find_object(self, obj_hash: bytes) -> Optional[int]:
        """使用二分搜索查找对象在pack文件中的偏移"""
        # 提取所有哈希用于二分搜索
        hashes = [h for h, _ in self.pack_index]

        idx = bisect.bisect_left(hashes, obj_hash)

        if idx < len(hashes) and hashes[idx] == obj_hash:
            return self.pack_index[idx][1]
        return None


class GitBisect:
    """Git bisect实现：二分查找引入bug的提交"""

    def __init__(self, commits: List[Dict]):
        """
        commits: 按时间排序的提交列表
        每个提交包含：{'hash': str, 'timestamp': int, 'message': str}
        """
        self.commits = commits
        self.timestamps = [c['timestamp'] for c in commits]

    def find_first_bad(self, test_func) -> Optional[Dict]:
        """
        二分查找第一个bad提交
        test_func: 返回True表示该提交包含bug
        """
        left, right = 0, len(self.commits) - 1
        first_bad = None

        while left <= right:
            mid = (left + right) // 2
            commit = self.commits[mid]

            is_bad = test_func(commit)

            if is_bad:
                first_bad = commit
                right = mid - 1  # 在左半部分继续查找
            else:
                left = mid + 1   # 在右半部分继续查找

        return first_bad

    def find_commits_by_date(self, start_date: int, end_date: int) -> List[Dict]:
        """查找日期范围内的提交"""
        left = bisect.bisect_left(self.timestamps, start_date)
        right = bisect.bisect_right(self.timestamps, end_date)
        return self.commits[left:right]


# 基准测试
import random
import time

def benchmark_git_bisect():
    # 生成模拟提交历史
    commits = [
        {
            'hash': f"commit_{i}",
            'timestamp': 1609459200 + i * 3600,  # 每小时一个提交
            'message': f"Commit message {i}"
        }
        for i in range(100000)
    ]

    bisect_tool = GitBisect(commits)

    # 模拟：第50000个提交引入bug
    def test_func(commit):
        commit_idx = int(commit['hash'].split('_')[1])
        return commit_idx >= 50000

    start = time.time()
    first_bad = bisect_tool.find_first_bad(test_func)
    elapsed = time.time() - start

    print(f"找到第一个bad提交: {first_bad['hash']}")
    print(f"耗时: {elapsed*1000:.3f}ms")
    print(f"测试次数: {test_func.call_count if hasattr(test_func, 'call_count') else 'N/A'}")

    # 范围查询测试
    start = time.time()
    day_commits = bisect_tool.find_commits_by_date(
        1609459200 + 1000 * 3600,  # 第1000小时
        1609459200 + 1100 * 3600   # 第1100小时
    )
    elapsed = time.time() - start

    print(f"\n日期范围查询: 找到 {len(day_commits)} 个提交")
    print(f"耗时: {elapsed*1000:.3f}ms")

if __name__ == '__main__':
    benchmark_git_bisect()
```

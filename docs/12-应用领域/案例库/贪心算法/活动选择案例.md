# 案例：贪心活动选择在会议调度与资源分配中的应用

## 背景

企业会议室、手术室、云计算 GPU 等稀缺资源需要在多个申请者间合理分配。每个申请有开始时间和结束时间，目标是安排尽可能多的不冲突活动。这正是**活动选择问题（Activity Selection）**，贪心算法的经典应用。

## 问题定义

- **输入**：$n$ 个活动，每个活动 $i$ 有开始时间 $s_i$ 和结束时间 $f_i$
- **兼容性**：活动 $i$ 和 $j$ 兼容，当 $s_i \geq f_j$ 或 $s_j \geq f_i$
- **目标**：选择最大的兼容活动子集

## 贪心策略：最早结束优先

```
按结束时间升序排序
选择第一个活动
for each subsequent activity:
    if its start time >= last selected finish time:
        select it
```

### 最优性证明

设贪心选择的活动为 $g$，最优解的第一个活动为 $o$。

- 由于 $g$ 是最早结束的活动，$f_g \leq f_o$
- 用 $g$ 替换 $o$，剩余的可选活动时间不会减少
- 因此存在包含 $g$ 的最优解
- 对剩余子问题递归应用同样论证

**结论**：最早结束优先策略得到全局最优解。

## 与加权版本的对比

若每个活动有价值（权重），问题变为**加权区间调度**，需用动态规划：

$$dp[i] = \max(dp[i-1], dp[p(i)] + v_i)$$

其中 $p(i)$ 为与 $i$ 兼容的最后活动。

## 实际应用

### 会议室调度

- Outlook / Google Calendar 的自动会议室推荐
- 贪心选择可最大化会议室利用率

### 课程表排程

- 选择不冲突的课程组合
- 在教室数量受限时，贪心最大化课程门数

### 广播频段分配

- 无线电台的节目时段分配
- 避免相邻频段干扰的时间隔离

### 任务调度（单机器）

- 单个 CPU / 单条生产线上的任务调度
- 最小化未完成任务数或最大化吞吐量

## 效果评估

- 对 10,000 个活动，排序 $O(n \log n)$ + 扫描 $O(n)$，< 10ms
- 在会议室管理系统中，比人工调度提升 30-50% 的利用率
- 加权版本用 DP，同样可在毫秒级求解大规模实例

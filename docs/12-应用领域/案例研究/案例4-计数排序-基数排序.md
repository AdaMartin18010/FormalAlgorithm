# 案例4: 计数排序与基数排序在整数/字符串排序中的应用


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

## 应用场景

在特定数据分布场景下，线性时间排序算法能显著超越比较排序的 $O(n \log n)$ 下界。计数排序和基数排序广泛应用于以下场景：

1. **数据库索引构建**：对整数主键、时间戳、哈希值进行排序
2. **字符串处理**：后缀数组构建、字典排序、基因组序列分析
3. **图像处理**：像素值排序、直方图均衡化
4. **机器学习**：特征分桶、决策树分裂点选择
5. **网络处理**：IP地址排序、端口扫描分析

特别是在大数据处理框架（如Apache Spark、Hadoop）中，基数排序常用于Shuffle阶段的排序优化，性能比通用排序提升3-10倍。

## 问题描述

### 问题定义

给定一个整数数组（或等价的可分解数据，如定长字符串），要求按非递减顺序排序。数据具有特定特征：整数范围有限或可按位分解。

### 输入

- 整数数组 $A[1..n]$，每个元素 $A[i] \in [0, k-1]$
- 对于基数排序：d位r进制数，即 $k = r^d$

### 输出

- 排序后的数组 $B[1..n]$

### 约束条件

- 对于计数排序：$k = O(n)$ 时可达到线性时间
- 对于基数排序：数字位数 $d$ 为常数或较小
- 稳定排序要求

## 算法选择理由

选择计数排序和基数排序的原因：

1. **线性时间复杂度**：当 $k = O(n)$ 时，计数排序达到 $O(n)$；基数排序达到 $O(d(n+r))$
2. **稳定性**：两者都是稳定排序，这对多关键字排序至关重要
3. **非比较排序**：不受 $\Omega(n \log n)$ 比较排序下界限制
4. **可并行化**：计数排序的计数阶段和基数排序的每趟都可高效并行
5. **缓存友好**：顺序访问模式，良好的空间局部性

与比较排序的对比：

- 当 $k \ll n$ 或数据可位分解时，线性排序远快于快速排序
- 需要额外空间（计数排序 $O(k)$，基数排序 $O(n)$）
- 不适合通用排序场景，但在特定领域不可替代

## 实现要点

### 1. 计数排序实现

```cpp
// 稳定计数排序
template<typename T>
void countingSort(const vector<T>& input, vector<T>& output, T max_val) {
    int n = input.size();
    int k = max_val + 1;

    // 1. 统计频率
    vector<int> count(k, 0);
    for (int i = 0; i < n; i++)
        count[input[i]]++;

    // 2. 计算前缀和（确定位置）
    for (int i = 1; i < k; i++)
        count[i] += count[i-1];

    // 3. 稳定放置元素（从后向前）
    output.resize(n);
    for (int i = n - 1; i >= 0; i--) {
        output[count[input[i]] - 1] = input[i];
        count[input[i]]--;
    }
}
```

**关键优化**：

- **范围检测**：先扫描找出min/max，减少count数组大小
- **原地优化**：当k较小时，可原地重排
- **并行计数**：多线程分段统计后合并

### 2. 基数排序（LSD）实现

```cpp
// LSD (Least Significant Digit) 基数排序
void radixSortLSD(vector<int>& arr, int bits_per_digit = 8) {
    int n = arr.size();
    int max_val = *max_element(arr.begin(), arr.end());
    int num_digits = 0;
    int r = 1 << bits_per_digit;  // 基数，如256

    // 计算需要的趟数
    while (max_val > 0) {
        max_val >>= bits_per_digit;
        num_digits++;
    }

    vector<int> output(n);

    for (int digit = 0; digit < num_digits; digit++) {
        // 当前位的掩码
        int shift = digit * bits_per_digit;

        // 计数排序当前位
        vector<int> count(r, 0);

        // 统计
        for (int i = 0; i < n; i++) {
            int digit_val = (arr[i] >> shift) & (r - 1);
            count[digit_val]++;
        }

        // 前缀和
        for (int i = 1; i < r; i++)
            count[i] += count[i-1];

        // 稳定放置（从后向前）
        for (int i = n - 1; i >= 0; i--) {
            int digit_val = (arr[i] >> shift) & (r - 1);
            output[count[digit_val] - 1] = arr[i];
            count[digit_val]--;
        }

        arr.swap(output);
    }
}
```

**优化策略**：

- **基数选择**：通常选择256（8位）或65536（16位），平衡趟数和每趟开销
- **Skewed Sort**：处理不均匀的位分布
- **MSD优化**：对字符串排序使用MSD+递归，早期终止

## 性能评估

### 时间复杂度

| 算法 | 时间复杂度 | 条件 |
|-----|-----------|-----|
| 计数排序 | $\Theta(n + k)$ | 范围k |
| 基数排序 | $\Theta(d(n + r))$ | d位r进制 |
| 后缀数组（倍增）| $O(n \log n)$ | 配合基数排序 |

### 空间复杂度

- **计数排序**：$O(n + k)$ — 输出数组 + 计数数组
- **基数排序**：$O(n + r)$ — 输出数组 + 计数数组

## 实际效果

### 应用案例

**Apache Spark排序**：

- 使用基数排序优化Shuffle阶段的排序
- 相比TimSort，大数据集性能提升5-10倍
- 在TPC-DS基准测试中整体查询性能提升15%

**基因组序列分析（BWA）**：

- 使用后缀数组进行DNA序列比对
- 基数排序构建后缀数组，处理数十亿碱基对
- 比传统方法快100倍以上

**Redis有序集合**：

- 使用基数排序处理批量插入后的重排序
- 在RDB持久化时排序key
- 内存数据库中的高效排序

**ClickHouse列式数据库**：

- 利用基数排序优化ORDER BY操作
- 对低基数列（如枚举值）特别高效
- 实时分析场景查询延迟降低40%

### 关键优势验证

1. **线性性能**：在数据范围有限时，确实达到接近线性的性能
2. **稳定性**：多关键字排序时保持相对顺序，正确性可验证
3. **并行效率**：并行基数排序在32核上达到接近线性的加速比
4. **内存带宽限制**：成为现代CPU上的主要瓶颈，优化方向转向减少内存访问

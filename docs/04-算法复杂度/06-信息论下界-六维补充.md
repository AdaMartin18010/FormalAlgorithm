## 一、规范定义深化

### 1.1 Shannon 熵的严格定义

**定义 1.1.1**（Shannon 熵 / Shannon Entropy）

设随机变量 $X$ 取值于有限集合 $\mathcal{X}$，其概率质量函数为 $p(x) = \Pr[X = x]$。则 $X$ 的**Shannon 熵**定义为：

$$
H(X) = -\sum_{x \in \mathcal{X}} p(x) \log_2 p(x) = \mathbb{E}\left[-\log_2 p(X)\right]
$$

其中约定 $0 \cdot \log_2 0 = 0$。熵的单位为**比特**（bits）。

**定义 1.1.2**（条件熵 / Conditional Entropy）

$$
H(X|Y) = -\sum_{x \in \mathcal{X}, y \in \mathcal{Y}} p(x,y) \log_2 p(x|y)
$$

**定义 1.1.3**（互信息 / Mutual Information）

$$
I(X;Y) = H(X) - H(X|Y) = H(Y) - H(Y|X) = \sum_{x,y} p(x,y) \log_2 \frac{p(x,y)}{p(x)p(y)}
$$

**定义 1.1.4**（联合熵 / Joint Entropy）

$$
H(X,Y) = -\sum_{x,y} p(x,y) \log_2 p(x,y)
$$

### 1.2 Kolmogorov 复杂度

**定义 1.2.1**（Kolmogorov 复杂度）

对于有限二进制串 $x \in \{0,1\}^*$，其**Kolmogorov 复杂度** $K(x)$ 是在固定通用图灵机 $U$ 上输出 $x$ 的最短程序 $p$ 的长度：

$$
K(x) = \min\{|p| : p \in \{0,1\}^*, U(p) = x\}
$$

## 二、模型设计深化

### 2.1 决策树模型

**定义 2.1.1**（比较决策树 / Comparison Decision Tree）

决策树是一棵有根二叉树，其中：

- **内部节点**：表示一次比较操作（$a_i \le a_j$？）
- **叶子节点**：表示算法的输出
- **路径**：从根到叶对应一次算法执行

高度为 $h$ 的二叉树最多有 $2^h$ 个叶子。若问题有 $N$ 种互斥输出，则任何决策树算法在最坏情况下至少需要 $\lceil \log_2 N \rceil$ 次比较。

**定义 2.1.2**（代数决策树 / Algebraic Decision Tree）

对于输入 $(x_1, \ldots, x_n) \in \mathbb{R}^n$，代数决策树是一棵 $d$-叉树，每个内部节点测试一个次数不超过 $d$ 的多项式符号。当 $d=1$ 时称为**线性决策树**。

**定理 2.1.3**（Ben-Or, 1983）

设 $W \subseteq \mathbb{R}^n$ 是判定问题的"Yes"区域，且 $W$ 有 $t$ 个连通分支。则任何代数决策树解决该问题需要深度 $\Omega(\log t)$。

### 2.2 信息论与复杂度的统一框架

| 复杂度类型 | 信息论量 | 下界形式 |
|-----------|----------|----------|
| 比较排序 | 排列熵 $H(\text{perm}) = \log_2 n!$ | $\Omega(n \log n)$ 次比较 |
| 搜索 | 位置熵 $H(\text{pos}) = \log_2 n$ | $\Omega(\log n)$ 次比较 |
| 通信复杂度 | 互信息 $I(X;Y \| f(X,Y))$ | $\Omega(I)$ 位通信 |
| 数据压缩 | 信源熵 $H(X)$ | 平均码长 $\ge H(X)$ |
| 电路复杂度 | 函数空间熵 | Shannon 下界 $\Omega(2^n / n)$ |

## 三、数学符号与推导

### 3.1 排序下界的 Stirling 推导

**定理 3.1.1**（比较排序信息论下界）

任何基于比较的排序算法最坏情况需要至少 $\lceil \log_2 n! \rceil$ 次比较。

**证明**：

1. $n$ 个元素有 $n!$ 种可能的排列；
2. 每次比较最多提供 1 位信息（两种结果）；
3. 决策树高度 $h$ 满足 $2^h \ge n!$；
4. 故 $h \ge \lceil \log_2 n! \rceil$。

**Stirling 近似**：

$$
\log_2 n! = n \log_2 n - n \log_2 e + O(\log n) = \Theta(n \log n)
$$

其中 $\log_2 e \approx 1.4427$。

### 3.2 搜索下界

**定理 3.2.1**（搜索信息论下界）

在已排序数组中搜索元素，任何基于比较的算法最坏情况需要至少 $\lceil \log_2(n+1) \rceil$ 次比较。

**证明**：

1. 有 $n+1$ 种可能结果（位于位置 $1, \ldots, n$ 或不存在）；
2. 信息量需求：$\log_2(n+1)$ 位；
3. 每次比较最多提供 1 位信息；
4. 故至少需要 $\lceil \log_2(n+1) \rceil$ 次比较。

### 3.3 Fano 不等式与通信下界

**定理 3.3.1**（Fano 不等式）

设 $X$ 为随机变量，$\hat{X}$ 为估计值，$P_e = \Pr[X \neq \hat{X}]$，则：

$$
H(X|\hat{X}) \le H(P_e) + P_e \log_2(|\mathcal{X}| - 1)
$$

其中 $H(P_e) = -P_e \log_2 P_e - (1-P_e) \log_2(1-P_e)$。

**应用**：估计频率的 $\epsilon$-近似需要 $\Omega(1/\epsilon^2)$ 位空间。

## 四、示例性代码

### 4.1 Rust：信息论下界计算工具

```rust
/// 信息论下界计算工具
use std::f64::consts::E;

/// Shannon 熵 H(X) = -Σ p(x) log₂ p(x)
pub fn shannon_entropy(probs: &[f64]) -> f64 {
    probs.iter()
         .filter(|&&p| p > 0.0)
         .map(|&p| -p * p.log2())
         .sum()
}

/// 决策树高度下界: h ≥ ⌈log₂(输出数)⌉
pub fn decision_tree_lower_bound(output_count: usize) -> usize {
    if output_count <= 1 { return 0; }
    let log2 = (output_count as f64).log2();
    log2.ceil() as usize
}

/// 排序比较下界: ⌈log₂(n!)⌉ ≈ n·log₂(n) - n·log₂(e)
pub fn sorting_comparison_lower_bound(n: usize) -> f64 {
    if n <= 1 { return 0.0; }
    let n_f = n as f64;
    n_f * n_f.log2() - n_f * E.log2()
        + 0.5 * (2.0 * std::f64::consts::PI * n_f).log2()
}

/// 搜索比较下界: ⌈log₂(n)⌉
pub fn search_comparison_lower_bound(n: usize) -> usize {
    if n == 0 { return 0; }
    let log2 = (n as f64).log2();
    log2.ceil() as usize
}

/// Bloom Filter 空间-误差权衡
/// m = -n·ln(p) / (ln 2)²
pub fn bloom_filter_space(n: usize, p: f64) -> usize {
    let n_f = n as f64;
    let m = -n_f * p.ln() / (2.0f64.ln().powi(2));
    m.ceil() as usize
}

/// 数据压缩下界（Shannon 第一定理）
pub fn compression_lower_bound(entropy: f64) -> f64 {
    entropy
}

fn main() {
    println!("=== 信息论下界计算工具 ===\n");

    println!("1. 排序比较下界");
    for n in [8, 16, 32, 64, 128] {
        let lb = sorting_comparison_lower_bound(n);
        println!("   n={:4} | log₂(n!) ≈ {:.2}", n, lb);
    }

    println!("\n2. 搜索比较下界");
    for n in [8, 16, 32, 1024, 1_048_576] {
        let lb = search_comparison_lower_bound(n);
        println!("   n={:8} | ⌈log₂ n⌉ = {}", n, lb);
    }

    println!("\n3. Shannon 熵示例");
    let fair_coin = vec![0.5, 0.5];
    let biased_coin = vec![0.9, 0.1];
    let die = vec![1.0/6.0; 6];
    println!("   公平硬币 H = {:.4} bits", shannon_entropy(&fair_coin));
    println!("   偏置硬币 H = {:.4} bits", shannon_entropy(&biased_coin));
    println!("   均匀骰子 H = {:.4} bits", shannon_entropy(&die));

    println!("\n4. Bloom Filter 空间-误差权衡");
    let n = 1_000_000usize;
    for p in [0.01, 0.001, 0.0001] {
        let m = bloom_filter_space(n, p);
        println!("   n={} p={:.4} | m={} bits", n, p, m);
    }
}
```

### 4.2 Rust：决策树分析器

```rust
/// 分析决策树的理论下界
pub struct DecisionTreeAnalyzer;

impl DecisionTreeAnalyzer {
    /// 比较排序的理论最优比较次数
    pub fn optimal_comparisons(n: usize) -> usize {
        if n <= 1 { return 0; }
        let mut fact_log2 = 0.0;
        for i in 2..=n {
            fact_log2 += (i as f64).log2();
        }
        fact_log2.ceil() as usize
    }

    /// 二分查找的理论最优比较次数
    pub fn optimal_search_comparisons(n: usize) -> usize {
        if n == 0 { return 0; }
        let log2 = (n as f64).log2();
        log2.ceil() as usize
    }

    /// 从比较次数推断最大可区分输出数
    pub fn max_outputs_from_comparisons(comparisons: usize) -> usize {
        1usize << comparisons
    }
}
```

## 五、形式化证明

### 5.1 排序下界的严格证明

**定理 5.1.1**：任何基于比较的排序算法最坏情况至少需要 $\lceil \log_2 n! \rceil$ 次比较。

**证明**：

1. **决策树模型**：将算法建模为二叉决策树，每个内部节点是一次比较，每个叶子是一个输出排列。

2. **叶子数下界**：$n$ 个不同元素有 $n!$ 种可能的全序排列，每个排列必须对应至少一个叶子。故叶子数 $L \ge n!$。

3. **树高与叶子数关系**：高度为 $h$ 的二叉树最多有 $2^h$ 个叶子。故：
   $$2^h \ge L \ge n!$$
   $$h \ge \lceil \log_2 n! \rceil$$

4. **信息论解释**：确定 $n!$ 种等可能排列中的一个需要 $\log_2 n!$ 位信息，每次比较最多提供 1 位信息，故至少需要 $\log_2 n!$ 次比较。

5. **Stirling 近似**：
   $$\log_2 n! = n \log_2 n - n \log_2 e + O(\log n) = \Omega(n \log n)$$

### 5.2 数据压缩的 Shannon 第一定理

**定理 5.2.1**（Shannon 第一本源编码定理）：

对于离散无记忆信源 $X$，任意无失真前缀码的平均码长 $L$ 满足：

$$
H(X) \le L < H(X) + 1
$$

**证明概要**：

- **下界**：若 $L < H(X)$，则由 Kraft 不等式与信息不等式，$D_{\mathrm{KL}}(p \| q) \ge 0$，导致矛盾。
- **上界**：取 $\ell(x) = \lceil -\log_2 p(x) \rceil$，满足 Kraft 不等式，且：
  $$L = \sum_x p(x) \ell(x) < \sum_x p(x)(-\log_2 p(x) + 1) = H(X) + 1$$

### 5.3 通信复杂度的信息论下界

**定理 5.3.1**：对于函数 $f: X \times Y \rightarrow Z$ 和输入分布 $\mu$，任何通信协议 $\Pi$ 满足：

$$
|\Pi| \ge I(X;Y \| f(X,Y))
$$

其中 $|\Pi|$ 是协议通信量，$I(X;Y \| f(X,Y))$ 是在已知 $f(X,Y)$ 条件下 $X$ 与 $Y$ 的条件互信息。

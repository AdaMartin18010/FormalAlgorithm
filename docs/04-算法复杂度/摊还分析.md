## 05. 摊还分析 / Amortized Analysis

### 1. 理论基础

**摊还分析**（Amortized Analysis）用于刻画数据结构在一系列操作上的平均最坏情况代价。它与平均情况分析（Average-Case Analysis）有本质区别：

- **摊还分析**：对任意操作序列给出确定性的代价上界，不依赖输入分布。
- **平均情况分析**：基于输入分布的概率期望。

设操作序列为 $op_1, op_2, \ldots, op_n$，对应的实际代价为 $c_1, c_2, \ldots, c_n$。摊还分析的核心任务是为每个操作分配一个**摊还代价** $\hat c_i$，使得对任意 $n$ 均有 [Cormen et al., 2009]：

$$
\sum_{i=1}^{n} \hat c_i \;\ge\; \sum_{i=1}^{n} c_i .
$$

经典的三种方法包括：

1. **聚合分析**（Aggregate Analysis）：计算 $n$ 个操作的总代价上界 $T(n)$，摊还代价为 $T(n)/n$。
2. **会计方法**（Accounting Method）：为操作预先收取摊还代价，盈余部分作为**信用**（credit）存储在数据结构中，用于支付后续高代价操作。
3. **势方法**（Potential Method）：引入状态势能函数 $\Phi$，将摊还代价定义为 $\hat c_i = c_i + \Phi(D_i) - \Phi(D_{i-1})$。

### 2. 算法设计

#### 2.1 聚合分析示例：栈的 `MULTIPOP`

栈支持 `PUSH`、`POP`、`MULTIPOP(S,k)`。注意到每个元素最多被 `PUSH` 一次、`POP` 一次，因此 $n$ 个操作的总实际代价 $T(n)=O(n)$，单个操作摊还代价为 $O(1)$。

#### 2.2 会计方法示例：动态数组插入

对于采用**倍增策略**的动态数组（见 [`dynamic_array.rs`](../../examples/algorithms/src/dynamic_array.rs)），每次普通 `push` 的实际代价为 $1$。设定摊还代价为 $3$：

- $1$ 用于当前元素的直接插入；
- $1$ 作为信用存于该元素，用于支付其未来被搬迁的代价；
- $1$ 作为信用帮助搬迁一个已有的旧元素。

当容量从 $m$ 倍增到 $2m$ 时，需要搬迁 $m$ 个旧元素，此时恰可利用先前累积的 $m$ 单位信用支付搬迁开销，从而保证信用始终非负 [Tarjan, 1985]。

#### 2.3 势方法示例：Splay 树

Splay 树通过 `Zig`、`Zig-Zig`、`Zig-Zag` 三种旋转将访问节点伸展至根（见 [`splay_tree.rs`](../../examples/algorithms/src/splay_tree.rs)）。定义节点 $x$ 的秩 $r(x)=\log_2 |\text{subtree}(x)|$，整棵树的势能为：

$$
\Phi = \sum_{x} r(x).
$$

通过 **Access Lemma**，每次 splay 操作的摊还代价不超过 $3(r'(x)-r(x))+1 = O(\log n)$，从而所有操作的摊还复杂度均为 $O(\log n)$ [Sleator & Tarjan, 1985]。

### 3. 复杂度分析

| 数据结构 | 操作 | 最坏情况 | 摊还情况 | 分析方法 |
|----------|------|----------|----------|----------|
| 动态数组 | `push` / `pop` | $O(n)$ | $O(1)$ | 会计 / 势方法 |
| 并查集 | `find` / `union` | $O(\log n)$ | $O(\alpha(n))$ | 聚合分析 |
| Splay 树 | `search` / `insert` / `delete` | $O(n)$ | $O(\log n)$ | 势方法 |
| 双端队列 | `enqueue` / `dequeue` | $O(n)$ | $O(1)$ | 势方法 |

> 注：$\alpha(\cdot)$ 为阿克曼函数的反函数，在一切实际应用中均小于 $5$ [Tarjan, 1975]。

### 4. 形式化验证 / 正确性论证

#### 4.1 动态数组的完整势函数推导（含扩容与缩容）

项目实现 [`dynamic_array.rs`](../../examples/algorithms/src/dynamic_array.rs) 采用如下维护策略：

- 当元素个数 $n$ 达到容量 $s$ 时，容量倍增为 $2s$；
- 当 $n \le s/4$ 时，容量减半为 $s/2$。

由此恒有 $s/4 < n \le s$（空数组除外）。为统一分析插入与删除，定义分段势能函数：

$$
\Phi(D) = \begin{cases}
2n - s, & n \ge s/2, \\[6pt]
\displaystyle\frac{s}{2} - n, & n < s/2.
\end{cases}
$$

易验证 $\Phi(D) \ge 0$，且仅在装载因子 $\alpha = n/s = 1/2$ 或 $\alpha = 1$ 时取零。

---

**插入操作分析**

1. **普通插入（$n \ge s/2$，未触发扩容）**：实际代价 $c_i = 1$，$n$ 增加 $1$，$s$ 不变。
   $$\Delta\Phi = 2(n+1) - s - (2n - s) = 2 \quad\Rightarrow\quad \hat c_i = 1 + 2 = 3.$$

2. **扩容插入（$n = s$）**：容量变为 $s' = 2s = 2n$。实际代价 $c_i = n + 1$（复制 $n$ 个元素并插入新元素）。
   $$\Delta\Phi = \bigl(2(n+1) - 2n\bigr) - (2n - n) = 2 - n \quad\Rightarrow\quad \hat c_i = (n+1) + (2-n) = 3.$$

3. **低装载区插入（$n < s/2$，插入后仍 $< s/2$）**：$c_i = 1$，$\Delta\Phi = -(s/2 - n) + (s/2 - (n+1)) = -1$，故 $\hat c_i = 0$。

4. **跨越装载区插入（$n < s/2$ 但 $n+1 \ge s/2$）**：势从 $\frac{s}{2} - n$ 跳变到 $2(n+1) - s$。
   $$\Delta\Phi = \bigl(2n + 2 - s\bigr) - \bigl(\tfrac{s}{2} - n\bigr) = 3n + 2 - \tfrac{3s}{2} < 2 \quad (\because n < \tfrac{s}{2})$$
   因此 $\hat c_i < 3$。

**删除操作分析**

1. **普通删除（$n > s/2$，未触发缩容）**：$c_i = 1$，$n$ 减少 $1$，$s$ 不变。
   $$\Delta\Phi = \bigl(2(n-1) - s\bigr) - (2n - s) = -2 \quad\Rightarrow\quad \hat c_i = -1.$$
   虽然单步摊还代价为负，但此前的高势（$2n-s$）已预先支付了这部分开销。

2. **缩容删除（$n = s/4$）**：容量变为 $s' = s/2 = 2n$。实际代价 $c_i = n$（需复制 $n-1$ 个剩余元素）。
   $$\Phi_{i-1} = \frac{s}{2} - n = \frac{s}{4},\quad \Phi_i = \frac{s'}{2} - (n-1) = \frac{s}{4} - n + 1 = 1.$$
   $$\Delta\Phi = 1 - \frac{s}{4} = 1 - n \quad\Rightarrow\quad \hat c_i = n + (1-n) = 1.$$

综上，**插入与删除的摊还代价均为 $O(1)$**。

#### 4.2 Splay 树的势方法正确性简述

设访问节点 $x$ 前、后的秩分别为 $r(x)$、$r'(x)$。通过分类讨论 `Zig`、`Zig-Zig`、`Zig-Zag` 三种旋转对势能的影响，可证得每次旋转的势能增量满足：

$$
\Delta\Phi \le 3\bigl(r'(x) - r(x)\bigr).
$$

对整趟 splay 操作求和，中间项 telescoping，最终得到摊还代价上界 $3(r_{\text{root}} - r(x)) + 1 = O(\log n)$。

### 5. 应用场景

- **动态数组**：C++ `std::vector`、Rust `Vec` 以及本项目 [`dynamic_array.rs`](../../examples/algorithms/src/dynamic_array.rs) 均依赖倍增-减半策略实现均摊 $O(1)$ 的尾部插入与删除。
- **并查集**：Kruskal 最小生成树算法、图的连通分量动态维护、离线最近公共祖先（LCA）查询。参见 [`union_find.rs`](../../examples/algorithms/src/union_find.rs)。
- **Splay 树**：频率敏感的数据缓存、动态顺序统计树、区间翻转与分裂操作。参见 [`splay_tree.rs`](../../examples/algorithms/src/splay_tree.rs)。
- **斐波那契堆**：Dijkstra 最短路径算法从 $O(E\log V)$ 优化至 $O(V\log V + E)$，Prim 算法同理 [Fredman & Tarjan, 1987]。

### 6. 扩展变体

| 变体 | 核心思想 | 典型结果 |
|------|----------|----------|
| **斐波那契堆** | 懒合并（Lazy Merging），延迟执行 `CONSOLIDATE` | `INSERT` $O(1)$，`EXTRACT-MIN` $O(\log n)$ 摊还 |
| **软堆**（Soft Heap） | 允许少量元素被“腐蚀”（corrupted），换取更优摊还界 | `INSERT`、`MELD`、`DELETE` 均 $O(1)$ 摊还 |
| **增量重建** | 数据结构在多次修改后批量重建，均摊重建代价 | 如 B-树、线段树的均摊重构 |
| **在线算法的势方法** | 将竞争比（Competitive Ratio）分析与势函数结合 | 列表更新问题、缓存替换策略分析 |

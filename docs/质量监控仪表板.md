## 2. 质量指标体系

### 2.1 核心质量指标

#### 文档完成度指标

| 指标ID | 指标名称 | 计算方式 | 目标值 | 权重 |
|--------|----------|----------|--------|------|
| DOC-COMP | 文档完整度 | 必需章节覆盖率 | ≥90% | 20% |
| DOC-FORM | 格式规范度 | 格式检查通过率 | ≥95% | 15% |
| DOC-CITE | 引用完整度 | 引用检查通过率 | ≥90% | 20% |
| DOC-SYM | 符号规范度 | 符号检查通过率 | ≥95% | 15% |
| DOC-READ | 可读性评分 | 同行评议平均分 | ≥8.0 | 30% |

#### 计算公式

```python
# 文档质量评分计算
def calculate_document_quality_score(metrics):
    """
    计算单个文档的质量评分 (0-100)
    """
    scores = {
        'completeness': metrics['doc_comp'] * 0.20,      # 完整度 20%
        'format': metrics['doc_form'] * 0.15,             # 格式 15%
        'citation': metrics['doc_cite'] * 0.20,           # 引用 20%
        'symbol': metrics['doc_sym'] * 0.15,              # 符号 15%
        'readability': metrics['doc_read'] * 10 * 0.30    # 可读性 30%
    }

    total_score = sum(scores.values())
    return min(100, max(0, total_score))

# 模块质量评分
def calculate_module_quality(documents):
    """
    计算模块质量评分
    """
    if not documents:
        return 0

    scores = [doc.quality_score for doc in documents]
    return sum(scores) / len(scores)

# 项目整体质量评分
def calculate_project_quality(modules):
    """
    计算项目整体质量评分
    """
    module_scores = [m.quality_score * m.weight for m in modules]
    total_weight = sum(m.weight for m in modules)
    return sum(module_scores) / total_weight if total_weight > 0 else 0
```

## 2.2 详细指标定义

### 2.2 详细指标定义

#### 文档完整度 (DOC-COMP)

```yaml
metric:
  id: DOC-COMP
  name: 文档完整度
  description: 文档包含所有必需章节的比例

  components:
    - name: header
      weight: 0.10
      check: "有标题、版本、日期"
    - name: summary
      weight: 0.15
      check: "有摘要章节"
    - name: glossary
      weight: 0.10
      check: "有术语表"
    - name: toc
      weight: 0.10
      check: "有目录"
    - name: cross_ref
      weight: 0.10
      check: "有交叉引用"
    - name: content
      weight: 0.30
      check: "正文内容完整"
    - name: references
      weight: 0.15
      check: "有参考文献"

  calculation: |
    DOC-COMP = Σ(component_score × component_weight)

  target: 0.90
  warning_threshold: 0.80
  critical_threshold: 0.70
```

#### 引用完整度 (DOC-CITE)

```yaml
metric:
  id: DOC-CITE
  name: 引用完整度
  description: 文档中定义、定理等必要引用的完整程度

  components:
    - name: definition_citations
      weight: 0.25
      check: "所有定义有引用"
    - name: theorem_citations
      weight: 0.25
      check: "所有定理有引用"
    - name: algorithm_citations
      weight: 0.20
      check: "所有算法有引用"
    - name: format_compliance
      weight: 0.15
      check: "引用格式符合ACM标准"
    - name: link_validity
      weight: 0.15
      check: "链接有效"

  calculation: |
    DOC-CITE = (有引用的元素数 / 需要引用的元素总数) × 格式合规率 × 链接有效率

  target: 0.90
  warning_threshold: 0.80
  critical_threshold: 0.70
```

#### 形式化程度 (DOC-FORM)

```yaml
metric:
  id: DOC-FORM
  name: 形式化程度
  description: 文档数学符号和形式化定义的规范程度

  components:
    - name: symbol_iso_compliance
      weight: 0.30
      check: "符号符合ISO 80000-2"
    - name: formula_format
      weight: 0.20
      check: "公式格式正确"
    - name: definition_formality
      weight: 0.25
      check: "形式化定义完整"
    - name: proof_structure
      weight: 0.25
      check: "证明结构规范"

  target: 0.95
  warning_threshold: 0.85
  critical_threshold: 0.75
```

## 7. 告警与通知

### 7.1 告警规则

```yaml
# alert_rules.yaml

alerts:
  # 质量下降告警
  quality_drop:
    condition: "module_quality_delta < -0.2"
    severity: warning
    message: "模块 {module_name} 质量评分下降超过0.2"
    notify: [email, dashboard]

  # 目标偏离告警
  target_deviation:
    condition: "project_quality < target_quality - 1.0"
    severity: warning
    message: "项目整体质量偏离目标超过1.0分"
    notify: [email, dashboard]

  # 严重问题告警
  critical_issues:
    condition: "critical_issues_count > 10"
    severity: critical
    message: "严重问题数量超过10个"
    notify: [email, dashboard, slack]

  # 问题积压告警
  issue_backlog:
    condition: "open_issues > 200"
    severity: warning
    message: "待解决问题数量超过200个"
    notify: [email]

  # 长期未解决告警
  stale_issues:
    condition: "issue_open_days > 30"
    severity: info
    message: "问题 #{issue_id} 已超过30天未解决"
    notify: [dashboard]
```

## 8. 技术实现

### 8.1 数据模型

```typescript
// TypeScript类型定义

interface Document {
  id: string;
  path: string;
  title: string;
  module: string;
  version: string;
  lastUpdated: Date;
  quality: QualityMetrics;
  issues: Issue[];
  history: QualityHistory[];
}

interface QualityMetrics {
  overall: number;           // 0-100
  completeness: number;      // 文档完整度
  format: number;            // 格式规范度
  citation: number;          // 引用完整度
  symbol: number;            // 符号规范度
  readability: number;       // 可读性评分
  lastCalculated: Date;
}

interface Issue {
  id: string;
  type: IssueType;
  severity: 'critical' | 'warning' | 'info';
  description: string;
  location: Location;
  status: 'open' | 'in_progress' | 'resolved' | 'closed';
  assignee?: string;
  createdAt: Date;
  resolvedAt?: Date;
}

interface Module {
  id: string;
  name: string;
  documents: Document[];
  quality: number;
  weight: number;
  targetQuality: number;
}

interface ProjectQuality {
  overall: number;
  modules: Module[];
  trends: TrendData[];
  issues: IssueSummary;
}
```

# API规范

openapi: 3.0.0
info:
  title: Quality Dashboard API
  version: 1.0.0

paths:
  /api/v1/quality/overview:
    get:
      summary: 获取整体质量概览
      responses:
        200:
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/QualityOverview'

  /api/v1/quality/modules:
    get:
      summary: 获取所有模块质量
      responses:
        200:
          content:
            application/json:
              schema:
                type: array
                items:
                  $ref: '#/components/schemas/ModuleQuality'

  /api/v1/quality/modules/{moduleId}:
    get:
      summary: 获取指定模块质量详情
      parameters:
        - name: moduleId
          in: path
          required: true
          schema:
            type: string
      responses:
        200:
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ModuleQuality'

  /api/v1/quality/trends:
    get:
      summary: 获取质量趋势数据
      parameters:
        - name: period
          in: query
          schema:
            type: string
            enum: [7d, 30d, 90d, 1y]
      responses:
        200:
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/TrendData'

  /api/v1/issues:
    get:
      summary: 获取问题列表
      parameters:
        - name: severity
          in: query
          schema:
            type: string
            enum: [critical, warning, info]
        - name: status
          in: query
          schema:
            type: string
            enum: [open, in_progress, resolved]
      responses:
        200:
          content:
            application/json:
              schema:
                type: array
                items:
                  $ref: '#/components/schemas/Issue'

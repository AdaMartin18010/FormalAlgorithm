## 1. 形式化定义 / Formal Definitions

### 1.1 数组的形式化定义

**定义 1.1** (数组 / Array) [Cormen 2022]
数组是一种线性数据结构，其元素在内存中**连续存储**，并通过基地址和索引偏移实现 $O(1)$ 随机访问。
**Definition 1.1** (Array)
An array is a linear data structure whose elements are stored **contiguously** in memory, enabling $O(1)$ random access via base address and index offset.

形式化地，长度为 $n$ 的数组 $A$ 可以表示为：

$$
A = \langle a_0, a_1, \ldots, a_{n-1} \rangle, \quad a_i \in \mathcal{T}
$$

其中 $\mathcal{T}$ 是元素类型域。数组支持的核心操作：

| 操作 | 签名 | 时间复杂度 | 说明 |
|-----|------|-----------|------|
| 访问 | $\text{Access}(A, i) \to a_i$ | $O(1)$ | 内存地址 = 基地址 + $i \times \text{sizeof}(\mathcal{T})$ |
| 更新 | $\text{Update}(A, i, v)$ | $O(1)$ | 直接写入对应内存位置 |
| 插入 | $\text{Insert}(A, i, v)$ | $O(n)$ | 需要移动 $[i, n-1]$ 的所有元素 |
| 删除 | $\text{Delete}(A, i)$ | $O(n)$ | 需要移动 $[i+1, n-1]$ 的所有元素 |
| 搜索 | $\text{Search}(A, v)$ | $O(n)$ | 线性扫描；若有序可用二分 $O(\log n)$ |

### 1.2 矩阵的形式化定义

**定义 1.2** (矩阵 / Matrix)
矩阵是数组在二维索引空间上的推广，可以视为**数组的数组**（行优先存储）或从索引对到值的映射。
**Definition 1.2** (Matrix)
A matrix is a generalization of an array to a two-dimensional index space, viewed as an **array of arrays** (row-major storage) or a mapping from index pairs to values.

形式化地，$m \times n$ 矩阵 $M$ 表示为：

$$
M = \begin{bmatrix}
m_{0,0} & m_{0,1} & \cdots & m_{0,n-1} \\
m_{1,0} & m_{1,1} & \cdots & m_{1,n-1} \\
\vdots & \vdots & \ddots & \vdots \\
m_{m-1,0} & m_{m-1,1} & \cdots & m_{m-1,n-1}
\end{bmatrix}
$$

在行优先存储中，元素 $M[i][j]$ 的内存地址为：

$$
\text{addr}(M[i][j]) = \text{base} + (i \times n + j) \times \text{sizeof}(\mathcal{T})
$$

### 1.3 原地算法的形式化约束

**定义 1.3** (原地算法 / In-Place Algorithm)
原地算法是指在计算过程中仅使用常数个额外变量（与输入规模无关）的算法。其空间复杂度满足：

$$
S(n) = O(1)
$$

对于数组/矩阵问题，原地算法通常要求直接修改输入数据结构，而非分配新的同等规模结构。

## 2. 核心思路与算法框架

### 2.1 数组操作的核心技巧

**技巧 1：双指针**
维护两个索引 `left` 和 `right`，根据条件相向或同向移动，常用于：

- 两数之和（相向双指针）
- 滑动窗口（同向双指针）
- 原地删除重复元素（快慢指针）

**技巧 2：前缀和**
预处理累积和数组 $\text{prefix}[i] = \sum_{k=0}^{i-1} a_k$，使得区间和在 $O(1)$ 内查询：

$$
\sum_{k=i}^{j} a_k = \text{prefix}[j+1] - \text{prefix}[i]
$$

**技巧 3：差分数组**
对于频繁的区间增减操作，使用差分数组 $d[i] = a[i] - a[i-1]$，将区间更新转化为单点更新。

### 2.2 矩阵操作的核心技巧

**技巧 1：按层处理（Layer-by-Layer）**
将矩阵视为由外到内的若干"层"（Layer），每层是一个环状结构。螺旋遍历、旋转图像等问题均可按层分解。

**技巧 2：转置 + 翻转**
矩阵顺时针旋转 90° 可以分解为：先转置（$M[i][j] \leftrightarrow M[j][i]$），再水平翻转每行。

**技巧 3：利用矩阵有序性**
若矩阵每行每列均有序（如杨氏矩阵），可以从右上角（或左下角）出发，利用 $O(1)$ 比较排除一行或一列，实现 $O(m+n)$ 搜索。

## 3. 经典题目详解

### 3.1 LeetCode 48 — Rotate Image

> **题目链接 / Problem Link**: [LeetCode 48. Rotate Image](https://leetcode.com/problems/rotate-image/)
> **难度 / Difficulty**: Medium

#### 形式化规约 / Formal Specification

**输入 / Input**: $n \times n$ 矩阵 $M$，元素为整数。
**输出 / Output**: 将 $M$ 原地顺时针旋转 90°。
**前置条件 / Precondition**:
$$
M \in \mathbb{Z}^{n \times n}, \quad n \geq 1
$$

**后置条件 / Postcondition**:
旋转后矩阵 $M'$ 满足：
$$
\forall i, j \in [0, n-1]: M'[i][j] = M[n-1-j][i]
$$

#### 核心思路 / Core Idea

**方法：转置 + 水平翻转**

矩阵顺时针旋转 90° 可以分解为两个更简单的操作：

1. **转置**：$M[i][j] \leftrightarrow M[j][i]$，将行变为列
2. **水平翻转**：每行反转，将列的上下顺序调整为旋转后的正确顺序

验证：对任意位置 $(i, j)$，转置后变为 $(j, i)$，再水平翻转（行内反转）变为 $(j, n-1-i)$。等等，这不对。让我重新推导。

正确的分解应该是：

1. **转置**：$M[i][j] \to M[j][i]$
2. **垂直翻转**（按列反转）：或等价地，**水平翻转每行**后再转置？

让我用公式严格推导目标：

- 原矩阵中位置 $(r, c)$ 的元素
- 顺时针旋转 90° 后应到达位置 $(c, n-1-r)$

方法 A：先转置再水平翻转每行

- 转置：$(r, c) \to (c, r)$
- 水平翻转每行：$(c, r) \to (c, n-1-r)$ ✅

方法 B：先垂直翻转每列再转置

- 垂直翻转：$(r, c) \to (n-1-r, c)$
- 转置：$(n-1-r, c) \to (c, n-1-r)$ ✅

两种方法均可。

#### 代码实现 / Code Implementations

- **Python**: [`examples/algorithms-python/src/leetcode/lc0048_rotate_image.py`](../../../examples/algorithms-python/src/leetcode/lc0048_rotate_image.py)
- **Rust**: [`examples/algorithms/src/leetcode/lc0048_rotate_image.rs`](../../../examples/algorithms/src/leetcode/lc0048_rotate_image.rs)

```python
# Python 参考实现
def rotate(matrix: list[list[int]]) -> None:
    n = len(matrix)
    # 转置
    for i in range(n):
        for j in range(i + 1, n):
            matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]
    # 水平翻转每行
    for i in range(n):
        matrix[i].reverse()
```

```rust
// Rust 参考实现
pub fn rotate(matrix: &mut Vec<Vec<i32>>) {
    let n = matrix.len();
    // 转置
    for i in 0..n {
        for j in (i + 1)..n {
            let tmp = matrix[i][j];
            matrix[i][j] = matrix[j][i];
            matrix[j][i] = tmp;
        }
    }
    // 水平翻转每行
    for i in 0..n {
        matrix[i].reverse();
    }
}
```

#### 复杂度分析 / Complexity Analysis

| 指标 / Metric | 值 / Value | 说明 / Note |
|--------------|-----------|------------|
| 时间复杂度 / Time | $O(n^2)$ | 访问矩阵每个元素一次 |
| 空间复杂度 / Space | $O(1)$ | 原地操作，仅使用临时变量交换 |

#### 正确性证明 / Correctness Proof

**定理 3.1.1** (LeetCode 48 正确性): 算法将 $n \times n$ 矩阵原地顺时针旋转 90°。

**证明 / Proof**:

**步骤 1 — 转置的正确性**:
转置操作将每个 $M[i][j]$（$i < j$）与 $M[j][i]$ 交换。转置后：

$$
M_{\text{transpose}}[i][j] = M[j][i]
$$

**步骤 2 — 水平翻转的正确性**:
对每行 $i$，将 $M_{\text{transpose}}[i][j]$ 与 $M_{\text{transpose}}[i][n-1-j]$ 交换。翻转后：

$$
M_{\text{final}}[i][j] = M_{\text{transpose}}[i][n-1-j] = M[n-1-j][i]
$$

**步骤 3 — 验证后置条件**:
我们需要证明 $M_{\text{final}}[i][j] = M_{\text{original}}[n-1-j][i]$。

由上式直接得到：

$$
M_{\text{final}}[i][j] = M[n-1-j][i]
$$

这正是顺时针旋转 90° 的定义。$\square$

### 3.2 LeetCode 54 — Spiral Matrix

> **题目链接 / Problem Link**: [LeetCode 54. Spiral Matrix](https://leetcode.com/problems/spiral-matrix/)
> **难度 / Difficulty**: Medium

#### 形式化规约 / Formal Specification

**输入 / Input**: $m \times n$ 矩阵 $M$。
**输出 / Output**: 按顺时针螺旋顺序返回所有元素。
**前置条件 / Precondition**:
$$
M \in \mathbb{Z}^{m \times n}, \quad m, n \geq 1
$$

**后置条件 / Postcondition**: 返回序列 $\langle e_1, e_2, \ldots, e_{mn} \rangle$，其中 $e_k$ 是按顺时针螺旋顺序访问的第 $k$ 个元素。

#### 核心思路 / Core Idea

**方法：按层模拟**

将矩阵视为由外到内的若干"层"。每一层由四条边组成：顶行、右列、底行、左列。维护四个边界变量：

- `top`: 当前层的最上行
- `bottom`: 当前层的最下行
- `left`: 当前层的最左列
- `right`: 当前层的最右列

每遍历完一条边，就收缩对应的边界。当边界交错时（`top > bottom` 或 `left > right`），遍历完成。

#### 代码实现 / Code Implementations

- **Python**: [`examples/algorithms-python/src/leetcode/lc0054_spiral_matrix.py`](../../../examples/algorithms-python/src/leetcode/lc0054_spiral_matrix.py)

```python
# Python 参考实现
def spiral_order(matrix: list[list[int]]) -> list[int]:
    if not matrix or not matrix[0]:
        return []

    result = []
    top, bottom = 0, len(matrix) - 1
    left, right = 0, len(matrix[0]) - 1

    while top <= bottom and left <= right:
        # 顶行：从左到右
        for col in range(left, right + 1):
            result.append(matrix[top][col])
        top += 1

        # 右列：从上到下
        for row in range(top, bottom + 1):
            result.append(matrix[row][right])
        right -= 1

        # 底行：从右到左（需检查是否仍有有效行）
        if top <= bottom:
            for col in range(right, left - 1, -1):
                result.append(matrix[bottom][col])
            bottom -= 1

        # 左列：从下到上（需检查是否仍有有效列）
        if left <= right:
            for row in range(bottom, top - 1, -1):
                result.append(matrix[row][left])
            left += 1

    return result
```

#### 复杂度分析 / Complexity Analysis

| 指标 / Metric | 值 / Value | 说明 / Note |
|--------------|-----------|------------|
| 时间复杂度 / Time | $O(mn)$ | 每个元素恰好访问一次 |
| 空间复杂度 / Space | $O(1)$ | 不计输出数组，仅使用边界变量 |

#### 正确性证明 / Correctness Proof

**定理 3.2.1** (LeetCode 54 正确性): 算法按顺时针螺旋顺序恰好访问矩阵中的每个元素一次。

**证明 / Proof**:

**循环不变式 / Loop Invariant**:
设 $S$ 为已收集到 `result` 中的元素集合，$U$ 为未访问的元素集合。每次外层循环开始时：

$$
Inv(top, bottom, left, right) \equiv
\begin{cases}
(1) & S \cup U = \text{所有矩阵元素}, \quad S \cap U = \emptyset \\
(2) & U \text{ 恰好是由边界 } [top, bottom] \times [left, right] \text{ 围成的子矩阵} \\
(3) & S \text{ 中的元素已按正确螺旋顺序收集}
\end{cases}
$$

**初始化**:
初始时 $top=0, bottom=m-1, left=0, right=n-1$，$U$ 为整个矩阵，$S = \emptyset$。不变式显然成立。

**保持**:
假设某次迭代开始时不变式成立。算法依次遍历：

1. 顶行 $(top, left)$ 到 $(top, right)$：这些元素属于 $U$ 的最上边界，按从左到右顺序加入 $S$，顺序正确。
2. 右列 $(top+1, right)$ 到 $(bottom, right)$：属于剩余 $U$ 的最右边界，从上到下顺序正确。
3. 底行（若 $top \leq bottom$）：从 $(bottom, right-1)$ 到 $(bottom, left)$，属于剩余 $U$ 的最下边界。
4. 左列（若 $left \leq right$）：从 $(bottom-1, left)$ 到 $(top, left)$，属于剩余 $U$ 的最左边界。

遍历完成后，边界向内收缩（$top \leftarrow top+1, right \leftarrow right-1, \ldots$），新的 $U$ 恰好是内层子矩阵。所有刚遍历的元素已从 $U$ 移入 $S$，顺序正确。

**终止**:
循环终止时 $top > bottom$ 或 $left > right$，即 $U = \emptyset$。由不变式 (1)，$S$ 包含所有元素；由不变式 (3)，顺序正确。$\square$

### 3.3 LeetCode 240 — Search a 2D Matrix II

> **题目链接 / Problem Link**: [LeetCode 240. Search a 2D Matrix II](https://leetcode.com/problems/search-a-2d-matrix-ii/)
> **难度 / Difficulty**: Medium

#### 形式化规约 / Formal Specification

**输入 / Input**: $m \times n$ 矩阵 $M$，以及目标值 $target$。
**输出 / Output**: 判断 $target$ 是否在 $M$ 中。
**前置条件 / Precondition**:
$$
\forall i \in [0, m-1]: M[i][0] \leq M[i][1] \leq \cdots \leq M[i][n-1]
$$
$$
\forall j \in [0, n-1]: M[0][j] \leq M[1][j] \leq \cdots \leq M[m-1][j]
$$

即矩阵每行从左到右非降序，每列从上到下非降序。

**后置条件 / Postcondition**:
$$
\text{result} = \begin{cases}
\text{true}, & \text{if } \exists i, j: M[i][j] = target \\
\text{false}, & \text{otherwise}
\end{cases}
$$

#### 核心思路 / Core Idea

**方法：右上角（或左下角）搜索法**

利用矩阵的双重有序性，从右上角 $(0, n-1)$ 出发：

- 若当前值 $= target$：找到，返回 `true`
- 若当前值 $> target$：当前列的所有元素都 $> target$（因为列向下递增），排除当前列，$col \leftarrow col - 1$
- 若当前值 $< target$：当前行的所有元素都 $< target$（因为行向右递增），排除当前行，$row \leftarrow row + 1$

每次比较可以排除一行或一列，最多 $m + n$ 次比较。

#### 代码实现 / Code Implementations

- **Python**: [`examples/algorithms-python/src/leetcode/lc0240_search_2d_matrix_ii.py`](../../../examples/algorithms-python/src/leetcode/lc0240_search_2d_matrix_ii.py)
- **Go**: [`examples/algorithms-go/leetcode/lc0240_search_2d_matrix_ii.go`](../../../examples/algorithms-go/leetcode/lc0240_search_2d_matrix_ii.go)

```python
# Python 参考实现
def search_matrix(matrix: list[list[int]], target: int) -> bool:
    if not matrix or not matrix[0]:
        return False

    m, n = len(matrix), len(matrix[0])
    row, col = 0, n - 1  # 从右上角开始

    while row < m and col >= 0:
        if matrix[row][col] == target:
            return True
        elif matrix[row][col] > target:
            col -= 1  # 排除当前列
        else:
            row += 1  # 排除当前行

    return False
```

```go
// Go 参考实现
func searchMatrix(matrix [][]int, target int) bool {
    if len(matrix) == 0 || len(matrix[0]) == 0 {
        return false
    }
    m, n := len(matrix), len(matrix[0])
    row, col := 0, n-1
    for row < m && col >= 0 {
        if matrix[row][col] == target {
            return true
        } else if matrix[row][col] > target {
            col--
        } else {
            row++
        }
    }
    return false
}
```

#### 复杂度分析 / Complexity Analysis

| 指标 / Metric | 值 / Value | 说明 / Note |
|--------------|-----------|------------|
| 时间复杂度 / Time | $O(m + n)$ | 每次比较排除一行或一列，最多 $m+n$ 步 |
| 空间复杂度 / Space | $O(1)$ | 仅使用两个索引变量 |

#### 正确性证明 / Correctness Proof

**定理 3.3.1** (LeetCode 240 正确性): 算法正确判断 $target$ 是否存在于双重有序矩阵中。

**证明 / Proof**:

**循环不变式 / Loop Invariant**:
设当前搜索位置为 $(row, col)$。不变式定义为：

$$
Inv(row, col) \equiv \text{若 } target \text{ 存在于矩阵中，则 } target \text{ 必在子矩阵 } [row, m-1] \times [0, col] \text{ 内}
$$

即：所有已排除的行（$[0, row-1]$）和列（$[col+1, n-1]$）中**不可能**包含 $target$。

**初始化**:
初始时 $row = 0, col = n-1$，子矩阵为整个矩阵 $[0, m-1] \times [0, n-1]$。不变式显然成立。

**保持**:
假设 $Inv(row, col)$ 成立，分三种情况：

- **情况 A**: $M[row][col] = target$。算法返回 `true`，正确。
- **情况 B**: $M[row][col] > target$。由于列向下递增，$\forall i \geq row: M[i][col] \geq M[row][col] > target$。因此第 $col$ 列中从 $row$ 向下的所有元素均 $> target$。又因为行向右递增，第 $col$ 列右侧的元素更大。故整个第 $col$ 列（从位置 $row$ 开始）及右侧不可能含 $target$。排除第 $col$ 列（$col \leftarrow col - 1$）后，$Inv(row, col-1)$ 成立。
- **情况 C**: $M[row][col] < target$。由于行向右递增，$\forall j \leq col: M[row][j] \leq M[row][col] < target$。因此第 $row$ 行中从 $0$ 到 $col$ 的所有元素均 $< target$。排除第 $row$ 行（$row \leftarrow row + 1$）后，$Inv(row+1, col)$ 成立。

**终止**:
循环终止时 $row = m$ 或 $col = -1$，即搜索空间为空。由不变式，若 $target$ 存在则必在空集中，矛盾。因此 $target$ 不存在，返回 `false` 正确。$\square$

## 5. 正确性证明框架

### 5.1 原地旋转 90° 的坐标变换证明

**定理 5.1** (坐标变换): 对 $n \times n$ 矩阵，操作"先转置再水平翻转每行"等价于顺时针旋转 90°。

**证明**:

设原矩阵中位置 $(r, c)$ 的元素值为 $v$。

**转置后**：位置变为 $(c, r)$，即元素 $v$ 现在位于 $M[c][r]$。

**水平翻转每行后**：行 $c$ 中的位置 $r$ 映射到位置 $n-1-r$，即元素 $v$ 现在位于 $M[c][n-1-r]$。

我们需要验证这是否等于旋转 90° 的目标位置。

顺时针旋转 90° 的几何变换为：

$$
(r, c) \xrightarrow{\text{rotate 90°}} (c, n-1-r)
$$

这与"转置 + 水平翻转"的结果完全一致。$\square$

### 5.2 螺旋遍历的覆盖性证明

**定理 5.2** (覆盖性): 按层模拟算法恰好访问矩阵中的每个元素一次。

**证明**:

对矩阵的层数 $k$ 进行归纳，其中 $k = \min(\lceil m/2 \rceil, \lceil n/2 \rceil)$ 是总层数。

**基例**: $k = 0$（空矩阵）或 $k = 1$（单元素矩阵）。算法直接访问该元素或返回空，覆盖性成立。

**归纳假设**: 假设对于层数 $< k$ 的矩阵，算法覆盖所有元素。

**归纳步骤**: 对于 $k$ 层矩阵，算法首先完整遍历最外层（四条边）。由于四条边的遍历范围互不重叠且合起来恰好是最外层的所有元素，最外层被完全覆盖。

然后边界向内收缩，剩余未遍历部分是一个 $(m-2) \times (n-2)$ 的子矩阵，其层数为 $k-1$。由归纳假设，内层子矩阵也被完全覆盖。

因此整个矩阵被恰好覆盖一次。$\square$

## 7. 常见错误与反模式

### 7.1 数组越界

**错误 / Mistake**: 在矩阵操作中混淆行列索引，或忘记检查空数组。

```python
# 错误：未检查空矩阵
for i in range(len(matrix)):
    for j in range(len(matrix[0])):  # ❌ matrix 为空时 matrix[0] 越界
        pass

# 正确
def search_matrix(matrix, target):
    if not matrix or not matrix[0]:
        return False
    # ...
```

### 7.2 螺旋遍历边界条件

**错误 / Mistake**: 在遍历底行和左列时，未检查边界是否已经交错，导致重复访问或越界。

```python
# 错误：遍历底行和左列时不检查边界
for col in range(right, left - 1, -1):
    result.append(matrix[bottom][col])  # ❌ 可能 top > bottom
bottom -= 1

# 正确
if top <= bottom:  # ✅ 先检查
    for col in range(right, left - 1, -1):
        result.append(matrix[bottom][col])
    bottom -= 1
```

### 7.3 原地修改时的信息丢失

**错误 / Mistake**: 在旋转图像时，未使用临时变量保存被覆盖的值，导致信息丢失。

```python
# 错误：直接覆盖
matrix[i][j] = matrix[j][i]  # ❌ matrix[i][j] 的原始值丢失
matrix[j][i] = matrix[i][j]  # 此时两个位置值相同

# 正确：使用临时变量或 Python 元组交换
matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]  # ✅
```

### 7.4 矩阵搜索方向错误

**错误 / Mistake**: 从左上角或右下角出发，无法利用有序性进行有效排除。

**分析**: 左上角是最小值，右下角是最大值。从左上角出发，若当前值 $< target$，下方和右方的值都可能 $> target$，无法确定排除方向。同理右下角。

**正确做法**: 必须从**右上角**或**左下角**出发，这两个位置具有"一个方向递增、另一个方向递减"的特性，才能确保每次比较可以排除一行或一列。

## 8. 自测问题

### 问题 1：矩阵旋转的替代方法

**Q**: 除了"转置 + 水平翻转"，还有没有其他方法实现顺时针旋转 90°？

**A**: 有。

**方法 B**：直接按层旋转。将矩阵由外到内分层，对每一层的每个元素，与顺时针方向的下三个位置进行四元交换：

$$
\text{temp} = M[i][j] \\
M[i][j] = M[n-1-j][i] \\
M[n-1-j][i] = M[n-1-i][n-1-j] \\
M[n-1-i][n-1-j] = M[j][n-1-i] \\
M[j][n-1-i] = \text{temp}
$$

这种方法不需要完整的转置操作，但逻辑更复杂。

### 问题 2：螺旋遍历的变种

**Q**: 如果要按逆时针螺旋顺序遍历矩阵，需要修改哪些部分？

**A**: 将四条边的遍历顺序调整为：左列（从上到下）→ 底行（从左到右）→ 右列（从下到上）→ 顶行（从右到左）。或者更简单：将整个矩阵先水平翻转，再用顺时针螺旋遍历。

### 问题 3：双重有序矩阵搜索的下界

**Q**: 在双重有序矩阵中搜索，$O(m+n)$ 是最优的吗？能否达到 $O(\log(mn))$？

**A**: 一般情况下的 $O(m+n)$ 是已知的最优比较算法之一。要达到 $O(\log(mn)) = O(\log m + \log n)$，需要更强的性质（如整个矩阵若展平为一维数组后全局有序，即"搜索二维矩阵 I"的条件），此时可以二分。在仅保证行列分别有序的条件下，信息论下界表明需要 $\Omega(m+n)$ 次比较。

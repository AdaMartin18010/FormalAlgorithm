## 面试常见错误清单 / Common Interview Mistakes

### 摘要 / Executive Summary

- 技术面试中，候选人常犯的错误并非"不会算法"，而是"**知道算法但写不对**"。据统计，LeetCode 周赛中超过 60% 的提交错误属于边界条件、索引越界、空指针等"低级错误"。
- 本文档系统梳理了面试中 7 大类常见错误，每类错误均给出：错误示例、形式化分析（为什么错）、修复方案、预防措施。文末附有面试快速自查 Checklist，帮助候选人在写完代码后的 2 分钟内完成系统性检查。
- 核心学习目标：建立"**防御性编程**"意识，将"写完代码"升级为"写完并验证代码"。

## 1. 边界条件错误

### 1.1 错误描述

边界条件错误是指在输入取最小值、最大值或特殊值时，程序行为不符合预期。这是面试中最常见的错误类型。

### 1.2 错误示例

**场景：二分查找**

```python
# 错误：未处理空数组
def binary_search(nums, target):
    left, right = 0, len(nums) - 1  # ❌ nums 为空时 right = -1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

binary_search([], 5)  # 结果正确但逻辑不健壮；若后续有 nums[right] 操作则越界
```

**场景：链表操作**

```python
# 错误：未处理单节点链表
def remove_nth_from_end(head, n):
    fast = slow = head
    for _ in range(n):
        fast = fast.next
    if not fast:
        return head.next
    while fast.next:
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next  # ❌ 若 slow.next 为 None 则崩溃
    return head
```

### 1.3 形式化分析

**为什么错？**

边界条件错误本质是**前置条件检查不完整**。形式化地，算法 $A$ 的正确性依赖于前置条件 $\text{pre}(x)$。若输入 $x$ 不满足 $\text{pre}(x)$ 的某些边界情况，则 $A(x)$ 的行为无定义。

以二分查找为例，前置条件要求 $nums$ 是有序数组，但**未要求非空**。当 $n = 0$ 时，$right = -1$，虽然 `while left <= right` 不会执行，但若后续有 `nums[right]` 等操作就会越界。

### 1.4 修复方案

```python
# 修复：显式处理边界
def binary_search(nums, target):
    if not nums:  # ✅ 处理空数组
        return -1
    left, right = 0, len(nums) - 1
    # ...
```

```python
# 修复：使用虚拟头节点处理头删除
def remove_nth_from_end(head, n):
    dummy = ListNode(0)
    dummy.next = head
    fast = slow = dummy
    for _ in range(n + 1):
        fast = fast.next
    while fast:
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next  # ✅ dummy 保证了 slow.next 始终存在
    return dummy.next
```

### 1.5 预防措施

| 检查项 | 说明 |
|-------|------|
| 空输入 | `null`, `[]`, `""`, `0` |
| 单元素输入 | `[1]`, `"a"` |
| 全相同元素 | `[5,5,5,5]` |
| 全不同元素 | `[1,2,3,4]` |
| 有序/逆序 | 升序、降序、部分有序 |
| 最大值输入 | 接近 `INT_MAX` 或数组长度上限 |

## 2. 索引越界错误

### 2.1 错误描述

索引越界是指访问数组、字符串或矩阵时，使用了超出有效范围的索引值。

### 2.2 错误示例

**场景：螺旋矩阵**

```python
# 错误：未检查边界交错
def spiral_order(matrix):
    result = []
    top, bottom = 0, len(matrix) - 1
    left, right = 0, len(matrix[0]) - 1

    while top <= bottom and left <= right:
        for col in range(left, right + 1):
            result.append(matrix[top][col])
        top += 1

        for row in range(top, bottom + 1):
            result.append(matrix[row][right])
        right -= 1

        # ❌ 未检查 top <= bottom 就遍历底行
        for col in range(right, left - 1, -1):
            result.append(matrix[bottom][col])
        bottom -= 1

        # ❌ 未检查 left <= right 就遍历左列
        for row in range(bottom, top - 1, -1):
            result.append(matrix[row][left])
        left += 1

    return result
```

**场景：字符串遍历**

```python
# 错误：越界访问
s = "abc"
for i in range(len(s)):
    if s[i] == s[i + 1]:  # ❌ 当 i = 2 时 s[3] 越界
        pass
```

### 2.3 形式化分析

索引越界的本质是**不变式维护失败**。对于数组 $A[0..n-1]$，有效索引集为 $I = \{0, 1, \ldots, n-1\}$。越界意味着存在某次访问 $A[i]$ 使得 $i \notin I$。

常见原因：

1. 循环边界条件错误（如 `<=` 写成 `<`）
2. 偏移量计算错误（如 `i + 1` 未检查）
3. 矩阵行列混淆（$M[i][j]$ 中 $i$ 和 $j$ 的边界不同）

### 2.4 修复方案

```python
# 修复：螺旋矩阵添加边界检查
def spiral_order(matrix):
    # ...
    while top <= bottom and left <= right:
        # 顶行和右列...

        if top <= bottom:  # ✅ 检查后再遍历底行
            for col in range(right, left - 1, -1):
                result.append(matrix[bottom][col])
            bottom -= 1

        if left <= right:  # ✅ 检查后再遍历左列
            for row in range(bottom, top - 1, -1):
                result.append(matrix[row][left])
            left += 1

    return result
```

```python
# 修复：字符串遍历加边界检查
for i in range(len(s) - 1):  # ✅ 只遍历到倒数第二个
    if s[i] == s[i + 1]:
        pass
```

### 2.5 预防措施

**索引访问万能公式**:

对于 `range(a, b)`，访问的元素索引为 $a, a+1, \ldots, b-1$，共 $b-a$ 个元素。

**检查清单**:

- [ ] `for i in range(n)` 访问 `i+1` 时，循环应改为 `range(n-1)`
- [ ] `while left <= right` 中，`left` 和 `right` 的更新是否可能使其越界
- [ ] 矩阵访问时，`row` 和 `col` 是否都在各自有效范围内
- [ ] 字符串/数组操作前，是否检查了非空

## 3. 空指针与空值错误

### 3.1 错误描述

空指针（`None`/`null`/`nil`）错误是指在没有检查对象是否存在的情况下，访问其属性或调用其方法。

### 3.2 错误示例

**场景：二叉树遍历**

```python
# 错误：未检查子节点是否存在
def max_depth(root):
    return 1 + max(max_depth(root.left), max_depth(root.right))  # ❌ root 为 None 时崩溃
```

**场景：链表合并**

```python
# 错误：假设节点一定存在
while l1 and l2:
    if l1.val <= l2.val:
        tail.next = l1
        l1 = l1.next
    else:
        tail.next = l2
        l2 = l2.next
    tail = tail.next
tail.next = l1  # ✅ 这里正确，但如果前面逻辑有误可能 tail 为 None
```

### 3.3 形式化分析

空指针错误违反了**引用安全（Reference Safety）**原则。形式化地，对于引用 $r$，任何解引用操作 $*r$ 都必须满足 $r \neq \text{null}$。

在树和链表问题中，空指针通常出现在：

1. 递归终止条件遗漏（如只检查 `node.left` 为终止条件）
2. 假设输入非空（如题目未明确保证）
3. 中间操作后指针变为空（如删除节点后的 `next`）

### 3.4 修复方案

```python
# 修复：添加空检查
def max_depth(root):
    if not root:  # ✅ 空树是唯一的基例
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))
```

```python
# 修复：使用虚拟头节点
dummy = ListNode(0)
dummy.next = head
tail = dummy
# ... 操作完成后
tail.next = ...  # ✅ dummy 保证 tail 始终有效
```

### 3.5 预防措施

| 场景 | 检查项 |
|-----|--------|
| 树问题 | 递归函数第一行是否检查 `node is None` |
| 链表问题 | 遍历前是否检查 `head is None`；操作前是否检查 `curr.next is None` |
| 数组问题 | 访问前是否检查 `len(arr) > 0` |
| 字符串问题 | 操作前是否检查 `s != ""` |

## 4. 整数溢出错误

### 4.1 错误描述

整数溢出是指运算结果超出了数据类型的表示范围。在 Python 中整数范围无限制，但在 Rust、Go、Java、C++ 等语言中尤为危险。

### 4.2 错误示例

**场景：二分查找中点计算**

```cpp
// C++ 错误
int mid = (left + right) / 2;  // ❌ left + right 可能溢出

// C++ 正确
int mid = left + (right - left) / 2;  // ✅ 安全
```

```rust
// Rust 错误（release 模式下会回绕）
let mid = (left + right) / 2;  // ❌ u32 溢出

// Rust 正确
let mid = left + (right - left) / 2;  // ✅ 安全
```

**场景：阶乘/幂运算**

```go
// Go 错误
result := 1
for i := 1; i <= n; i++ {
    result *= i  // ❌ int 溢出
}
```

### 4.3 形式化分析

对于有符号 $k$ 位整数，可表示范围为 $[-2^{k-1}, 2^{k-1} - 1]$。运算结果 $r$ 若满足 $|r| > 2^{k-1} - 1$，则发生溢出。

常见高危场景：

- 两个大整数相加（中点计算）
- 乘法累积（阶乘、组合数）
- 绝对值计算（`abs(INT_MIN)` 在某些语言中溢出）

### 4.4 修复方案

```rust
// Rust 使用溢出检查（debug 模式默认检查）
let mid = left.checked_add(right - left).unwrap() / 2;

// 或使用更大类型
let mid = (left as u64 + right as u64) / 2;
```

```go
// Go 使用 big.Int 处理大数
import "math/big"
result := big.NewInt(1)
for i := 1; i <= n; i++ {
    result.Mul(result, big.NewInt(int64(i)))
}
```

### 4.5 预防措施

| 场景 | 预防措施 |
|-----|---------|
| 中点计算 | 始终使用 `left + (right - left) / 2` |
| 大数乘法 | 检查是否可能溢出，必要时使用大数类型 |
| 绝对值 | 特殊处理 `INT_MIN` |
| 累加和 | 使用 64 位整数存储累加结果 |

## 5. 递归深度错误

### 5.1 错误描述

递归深度错误包括栈溢出（递归太深）和递归终止条件错误（无限递归）。

### 5.2 错误示例

**场景：树遍历（退化树）**

```python
# 错误：未考虑退化为链表的树
def inorder_traversal(root):
    if not root:
        return []
    return inorder_traversal(root.left) + [root.val] + inorder_traversal(root.right)

# 对于 10^5 个节点的链式树，Python 默认递归深度限制为 1000，会报错
```

**场景：快速排序**

```python
# 错误：分区不均衡导致最坏情况 O(n) 递归深度
def quicksort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[0]  # ❌ 已排序数组会导致极度不均衡
    left = [x for x in arr[1:] if x <= pivot]
    right = [x for x in arr[1:] if x > pivot]
    return quicksort(left) + [pivot] + quicksort(right)
```

### 5.3 形式化分析

递归深度 $d$ 受限于系统调用栈大小 $S$ 和每次调用的栈帧大小 $f$。最大安全深度约为 $d_{\max} \approx S / f$。

- Python: 默认 $d_{\max} = 1000$
- Java/JVM: 通常 $d_{\max} \approx 10^4$
- C++/Rust: 依赖系统栈大小（通常 8MB）

### 5.4 修复方案

```python
# 修复：使用迭代代替递归
def inorder_traversal_iter(root):
    result = []
    stack = []
    curr = root
    while curr or stack:
        while curr:
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()
        result.append(curr.val)
        curr = curr.right
    return result
```

```python
# 修复：随机化 pivot 避免最坏情况
import random
def quicksort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[random.randint(0, len(arr) - 1)]  # ✅ 随机化
    # ...
```

### 5.5 预防措施

| 场景 | 预防措施 |
|-----|---------|
| 树问题 | 若树可能退化，使用迭代版本或增加递归深度限制 |
| 分治问题 | 确保每次问题规模严格减小，避免 $T(n) = T(n-1) + O(n)$ |
| 图问题 | DFS 时注意图中可能存在长链，考虑 BFS 替代 |

## 6. 并发修改错误

### 6.1 错误描述

在遍历集合的同时修改集合（如删除元素），导致迭代器失效或访问已删除元素。

### 6.2 错误示例

**场景：遍历中删除**

```python
# 错误
nums = [1, 2, 3, 4, 5]
for i in range(len(nums)):
    if nums[i] % 2 == 0:
        del nums[i]  # ❌ 删除后后续元素前移，会跳过某些元素
```

**场景：列表推导式副作用**

```python
# 错误
result = [nums.pop(0) for _ in range(len(nums))]  # ❌ 虽然结果正确，但修改了原列表，逻辑混乱
```

### 6.3 形式化分析

迭代器的不变式要求：遍历过程中集合的结构不发生变化。形式化地，设迭代器状态为 $(S, i)$，其中 $S$ 是集合，$i$ 是当前索引。删除操作改变了 $S$，使得 $S_{new}[j] \neq S_{old}[j]$ 对某些 $j \geq i$ 成立，破坏了迭代器的正确性假设。

### 6.4 修复方案

```python
# 修复：倒序删除或构建新列表
nums = [1, 2, 3, 4, 5]
# 方法 A：倒序删除不影响未遍历的元素
for i in range(len(nums) - 1, -1, -1):
    if nums[i] % 2 == 0:
        del nums[i]

# 方法 B：列表推导式构建新列表
nums = [x for x in nums if x % 2 != 0]  # ✅ 更 Pythonic
```

### 6.5 预防措施

| 场景 | 预防措施 |
|-----|---------|
| 删除元素 | 倒序遍历，或先标记再批量删除 |
| 添加元素 | 先收集再扩展，避免边遍历边添加 |
| 修改元素 | 确认修改不影响遍历顺序 |

## 7. 语义理解偏差

### 7.1 错误描述

对题目条件、约束、输入输出格式的错误理解，导致算法方向正确但结果错误。

### 7.2 错误示例

**场景：理解错返回类型**

```python
# 题目要求返回索引，但返回了值
def two_sum(nums, target):
    seen = {}
    for num in nums:
        complement = target - num
        if complement in seen:
            return [complement, num]  # ❌ 返回了值，不是索引
        seen[num] = True
    return []
```

**场景：忽略约束条件**

```python
# 题目要求 O(1) 空间，但使用了哈希表
def find_duplicate(nums):
    seen = set()  # ❌ O(n) 空间
    for num in nums:
        if num in seen:
            return num
        seen.add(num)
```

### 7.3 形式化分析

语义理解偏差的本质是**问题实例映射错误**。设实际问题为 $\Pi_{\text{actual}}$，候选人理解的问题为 $\Pi_{\text{perceived}}$。若 $\Pi_{\text{actual}} \neq \Pi_{\text{perceived}}$，则即使算法 $A$ 对 $\Pi_{\text{perceived}}$ 完全正确，$A(\Pi_{\text{actual}})$ 也是错误的。

### 7.4 修复方案

**面试中的标准流程**:

```markdown
1. 读完题目后，用自己的话向面试官复述题意
2. 明确输入输出格式（返回索引？返回节点？返回布尔值？）
3. 确认约束条件（时间/空间复杂度要求、数据范围）
4. 询问边界情况的处理方式（如空输入、无解情况）
5. 写出 2-3 个测试用例与面试官确认
```

### 7.5 预防措施

| 检查项 | 说明 |
|-------|------|
| 返回值类型 | 索引？值？节点？列表？ |
| 输入约束 | 数组长度范围？元素取值范围？是否有序？ |
| 复杂度要求 | 是否要求 O(1) 空间？是否要求线性时间？ |
| 边界情况 | 空输入？单元素？无满足条件的解？ |
| 关键术语 | "子串"vs"子序列"、"路径"vs"子树"、"前缀"vs"后缀" |

## 8. 面试快速自查 Checklist

写完代码后，用以下 Checklist 在 **2 分钟内**完成系统性检查：

### 8.1 边界检查（30秒）

- [ ] **空输入**: 数组为空？字符串为空？链表头为 null？树为空？
- [ ] **单元素**: 只有一个元素时逻辑是否正确？
- [ ] **全相同/全不同**: 元素全部相同或全部不同时是否正确？
- [ ] **最大值/最小值**: 输入达到约束上限时是否仍然高效？

### 8.2 索引检查（30秒）

- [ ] **循环边界**: `for i in range(n)` 访问 `i+1` 时是否越界？
- [ ] **数组访问**: 所有 `arr[i]` 是否满足 $0 \leq i < \text{len}(arr)$？
- [ ] **矩阵访问**: `matrix[i][j]` 是否满足行列边界？
- [ ] **字符串访问**: `s[i]` 是否满足 $0 \leq i < \text{len}(s)$？

### 8.3 指针与引用检查（20秒）

- [ ] **空指针**: 访问 `.next`、`.left`、`.right` 前是否检查非空？
- [ ] **断链**: 修改指针前是否保存了后续节点的引用？
- [ ] **返回值**: 链表/树操作后是否正确返回新的头节点？

### 8.4 整数运算检查（20秒）

- [ ] **中点计算**: 是否使用 `left + (right - left) / 2`？
- [ ] **溢出风险**: 乘法或累加是否可能超过整数上限？

### 8.5 复杂度与语义检查（20秒）

- [ ] **复杂度**: 是否满足题目要求的时间/空间复杂度？
- [ ] **返回值**: 返回的是索引、值、还是节点？与题目要求一致吗？
- [ ] **特殊情况**: 题目中提到的特殊情况（如有重复元素、有负数）是否处理？

### 8.6 快速 mentally run 一个用例（20秒）

选择一个非平凡的测试用例，在脑中逐步执行代码，验证：

- 循环变量是否正确更新？
- 条件分支是否进入正确的路径？
- 返回值是否符合预期？

# BibTeX 引用迁移方案

> **版本**: 1.0
> **创建日期**: 2026-04-29
> **状态**: P1 阶段（双轨并行）
> **负责人**: 项目维护团队

---

## 一、背景与目标

### 1.1 迁移动机

项目当前使用自定义 YAML 格式管理引用数据库，虽然便于人工阅读和项目内部维护，但存在以下局限：

- **缺乏学术工具链支持**：Zotero、JabRef 等文献管理工具无法直接导入 YAML 格式
- **LaTeX / PDF 导出困难**：生成学术 PDF 时需要手动转换引用格式
- **Pandoc 集成受限**：无法利用 `citeproc` 进行自动化引用处理和样式切换
- **字段标准化不足**：自定义字段（如 `impact_factor`、`topics`）虽便于项目管理，但不符合学术规范

### 1.2 迁移目标

在不破坏现有 `[AuthorYear]` 正文引用格式的前提下，建立符合 **BibTeX 标准** 的并行引用数据库，实现：

1. 与 Zotero、JabRef、Mendeley 等工具无缝对接
2. 支持 LaTeX `bibliographystyle` 自动生成格式化参考文献
3. 支持 Pandoc `citeproc` 将 Markdown 转换为带引用的 PDF / DOCX
4. 为未来可能的单源管理（YAML → BibTeX 单向生成）奠定基础

## 二、当前引用格式分析

### 2.1 现有格式结构

项目当前采用两套互补的 YAML 数据库：

| 文件 | 用途 | 条目数 | 特点 |
|------|------|--------|------|
| `docs/引用规范与数据库.md` | 经典文献、教材、基础引用 | ~20（内嵌YAML） | 含中英双语标题、质量评级 |
| `docs/references_database_2024_2025.yaml` | 2024–2025年最新研究 | 53 | 含项目专属字段（impact_factor, topics, related_sections） |

### 2.2 现有字段映射

```yaml
# 当前 YAML 字段 → BibTeX 对应字段
id / reference_id    →  BibTeX Key（保留原值以确保兼容）
title               →  title（取英文版）
authors             →  author（YAML列表 → "and" 连接）
venue               →  journal / booktitle / publisher / school（需推断）
year                →  year
pages               →  pages
doi                 →  doi
url                 →  url
isbn                →  isbn
arxiv               →  eprint / archivePrefix / primaryClass
month               →  month
day                 →  （BibTeX无直接对应，可忽略或放入 note）
type                →  用于推断 BibTeX entry type
summary             →  note（可选）
```

### 2.3 优点与局限性

| 维度 | 优点 | 局限性 |
|------|------|--------|
| **可读性** | YAML 人类可读，便于中文注释和结构化编辑 | 学术工具不支持原生 YAML |
| **扩展性** | 可自由添加 `impact_factor`、`topics` 等项目专属字段 | 字段非标准，跨项目不兼容 |
| **引用格式** | 正文 `[AuthorYear]` 简洁统一 | 无法自动生成 APA、IEEE、GB/T 7714 等样式 |
| **维护成本** | 单文件编辑方便 | 需要手工维护多文件一致性 |

## 三、BibTeX 目标格式设计

### 3.1 支持的 Entry Types

根据项目文献构成，优先支持以下 6 种 entry type：

| Entry Type | 适用场景 | 判定规则 |
|------------|----------|----------|
| `@book` | 教材、专著 | `type: book` 或 `type: textbook`，无 `journal` / `booktitle` |
| `@article` | 期刊论文 | `type: journal_article`，有 `venue` 且为期刊名 |
| `@inproceedings` | 会议论文 | `type: conference_paper`，有 `venue` 且含会议缩写 |
| `@phdthesis` | 博士学位论文 | `type: thesis` 且 `degree: phd` |
| `@misc` | 在线资源、课程、软件、技术报告 | `type: online / course / software_release / technical_report / preprint` |
| `@incollection` | 书籍章节 | `type: book_chapter` |

### 3.2 Required / Optional Fields

```bibtex
% @book 示例
@book{CLRS2009,
  author    = {Thomas H. Cormen and Charles E. Leiserson and Ronald L. Rivest and Clifford Stein},
  title     = {Introduction to Algorithms},
  edition   = {3rd},
  publisher = {MIT Press},
  year      = {2009},
  isbn      = {978-0262033848},
  url       = {https://mitpress.mit.edu/books/introduction-algorithms},
}

% @article 示例
@article{AkraBazzi1998,
  author  = {Mohamad Akra and Louay Bazzi},
  title   = {On the Solution of Linear Recurrence Equations},
  journal = {Computational Optimization and Applications},
  year    = {1998},
  volume  = {10},
  pages   = {195--210},
  doi     = {10.1023/A:1018373005182},
}

% @inproceedings 示例
@inproceedings{MartinLof2024,
  author    = {Arthur Adjedj and Meven Lennon-Bertrand and Kenji Maillard and Pierre-Marie P{\'e}drot and Lo{\"i}c Pujet},
  title     = {Martin-L{\"o}f {\`a} la {C}oq},
  booktitle = {Proceedings of the 13th ACM SIGPLAN International Conference on Certified Programs and Proofs},
  year      = {2024},
  pages     = {??--??},
  doi       = {},
  url       = {https://hal.science/hal-04214008},
}

% @misc 示例
@misc{Preskill2025,
  author       = {John Preskill},
  title        = {Lecture Notes for Physics 219: Quantum Computation},
  howpublished = {Caltech},
  year         = {2025},
  url          = {http://theory.caltech.edu/~preskill/ph219/},
  note         = {Accessed: 2025-01-11},
}
```

### 3.3 BibTeX Key 命名规范

为保持与现有正文引用的**完全兼容**，沿用原有 key，不做修改：

```
规则：保留 YAML 中的 id / reference_id 原值
示例：
  - CLRS2009    →  保留
  - Turing1936  →  保留
  - HoTTBook    →  保留
  - 1Lab2024    →  保留（但建议在 LaTeX 中使用时避免数字开头，可加前缀）
```

> **兼容性说明**：纯数字开头的 key（如 `1Lab2024`）在部分 BibTeX 处理器中可能出现问题。P1 阶段保留原 key，P2 阶段提供别名映射表。

## 四、兼容性策略

### 4.1 核心原则：正文引用格式不变

项目已在 400+ 文档中使用 `[AuthorYear]` 格式（如 `[CLRS2009]`、`[Turing1936]`）。迁移方案**绝不修改**这些正文标记。

### 4.2 双轨并行机制

```
┌─────────────────────────────────────────────────────────────┐
│                    引用数据来源（P1 阶段）                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   YAML 数据库（现有）          BibTeX 数据库（新建）           │
│   ├─ 引用规范与数据库.md        ├─ references.bib             │
│   ├─ references_database_*.yaml                               │
│   └─ 项目专属字段               └─ 标准学术字段                │
│         │                              │                    │
│         │         ┌───────────────────┘                    │
│         │         │                                        │
│         ▼         ▼                                        │
│   正文 [AuthorYear]  ←────  完全兼容  ────→  LaTeX / Pandoc │
│                                                             │
│   维护方式：人工编辑          维护方式：脚本生成 + 人工校对     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 Markdown → PDF / LaTeX 工作流

**方案 A：Pandoc + BibTeX（推荐）**

```bash
# 将 Markdown 转换为带引用的 PDF
pandoc input.md \
  --bibliography=docs/references.bib \
  --citeproc \
  --csl=styles/ieee.csl \
  -o output.pdf
```

> 由于正文使用 `[CLRS2009]` 而非标准的 `[@CLRS2009]`，Pandoc 默认不会识别。需在 P2 阶段引入预处理脚本，或将正文引用统一为 `[@key]` 格式。

**方案 B：LaTeX 直接编译**

```latex
\documentclass{article}
\usepackage[utf8]{inputenc}
\bibliographystyle{alpha}  % 或 plain, unsrt, acm, ieeetr
\begin{document}
  % 将 Markdown 中的 [CLRS2009] 替换为 \cite{CLRS2009}
  See \cite{CLRS2009} for details.
  \bibliography{references}
\end{document}
```

**过渡方案（P1）**：

对于需要生成学术 PDF 的文档，提供一个**轻量转换脚本** `scripts/md_refs_to_latex.py`，将 Markdown 中的 `[Key]` 替换为 `\cite{Key}`，同时保留 Markdown 原文不变。

## 六、工具链集成建议

### 6.1 Zotero 导出与导入

**导入 `references.bib` 到 Zotero**：

1. 打开 Zotero → 文件 → 导入
2. 选择 `docs/references.bib`
3. Zotero 将自动识别 `@book`、`@article` 等类型
4. 导出时选择 "Better BibTeX" 插件可保持 key 稳定

**反向同步（Zotero → 项目）**：

- 使用 Zotero 的 "Export Library" → 格式选 "BibTeX"
- 导出的 `.bib` 需经 `scripts/convert_refs_to_bibtex.py` 的逆流程处理（未来开发）
- **P1–P2 阶段不建议双向同步**，避免冲突

### 6.2 Pandoc citeproc

```bash
# 安装 Pandoc（若未安装）
# https://pandoc.org/installing.html

# 基础用法
pandoc doc.md \
  --bibliography docs/references.bib \
  --citeproc \
  --csl https://www.zotero.org/styles/ieee \
  -o doc.pdf

# 使用中文 GB/T 7714 样式
pandoc doc.md \
  --bibliography docs/references.bib \
  --citeproc \
  --csl styles/chinese-gb7714-2015-numeric.csl \
  -o doc.pdf
```

**注意事项**：

- Pandoc 默认识别 `[@key]` 和 `@key`，不识别 `[key]`
- 过渡期可使用 `scripts/md_refs_to_latex.py` 预处理

### 6.3 LaTeX bibliographystyle

推荐样式及其适用场景：

| 样式 | 文件 | 特点 | 适用场景 |
|------|------|------|----------|
| `plain` | 标准 | 按作者字母序，数字标号 | 通用 |
| `unsrt` | 标准 | 按引用顺序，数字标号 | 技术报告 |
| `alpha` | 标准 | 作者年份缩写标号 | 理论论文（与 `[AuthorYear]` 视觉上接近）|
| `ieeetr` | 标准 | IEEE  Transactions 格式 | 工程/系统方向 |
| `acm` | 标准 | ACM 格式 | 计算机科学 |
| `apalike` | natbib | 作者-年份制 | 社会科学风格 |

## 七、质量保障

### 7.1 BibTeX 文件校验清单

每条目应满足：

- [ ] `key` 与 YAML 中的 `id` / `reference_id` 一致
- [ ] `author` 字段使用 `and` 分隔多个作者
- [ ] 特殊字符（`# $ % & ~ _ ^ \ { }`）已正确转义或使用 BibTeX 括号保护
- [ ] `year` 为四位数字
- [ ] `doi` / `url` 至少有一个（在线资源除外）
- [ ] `pages` 使用双连字符 `--`（BibTeX 会自动转换为 en-dash）

### 7.2 自动化校验脚本（P2 阶段开发）

```bash
# 校验 BibTeX 语法
python scripts/validate_bibtex.py docs/references.bib

# 检查 YAML → BibTeX 一致性
python scripts/check_ref_sync.py \
  --yaml docs/references_database_2024_2025.yaml \
  --bib docs/references.bib
```

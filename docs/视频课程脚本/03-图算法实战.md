## 第一部分：图的基础 (6分钟)

### 1.1 什么是图

**画面**：图的结构动画

```
图 G = (V, E)

V: 顶点集合（Vertex Set）
E: 边集合（Edge Set）

示例：
    A ----4---- B
    |           |
    2           1
    |           |
    C ----3---- D

V = {A, B, C, D}
E = {(A,B,4), (A,C,2), (B,D,1), (C,D,3)}
```

### 1.2 图的表示方法

**画面**：三种表示方式对比

| 表示方法 | 空间 | 查询边 | 遍历邻接点 | 适用场景 |
|----------|------|--------|------------|----------|
| 邻接矩阵 | $O(V^2)$ | $O(1)$ | $O(V)$ | 稠密图 |
| 邻接表 | $O(V+E)$ | $O(degree)$ | $O(degree)$ | 稀疏图 |
| 边列表 | $O(E)$ | $O(E)$ | $O(E)$ | Kruskal等 |

**邻接表 Rust 实现**：
```rust
use std::collections::HashMap;

// 无权图
pub struct Graph {
    adj: HashMap<usize, Vec<usize>>,
}

impl Graph {
    pub fn new() -> Self {
        Graph {
            adj: HashMap::new(),
        }
    }
    
    pub fn add_edge(&mut self, u: usize, v: usize) {
        self.adj.entry(u).or_default().push(v);
        self.adj.entry(v).or_default().push(u); // 无向图
    }
    
    pub fn neighbors(&self, u: usize) -> &[usize] {
        self.adj.get(&u).map(|v| v.as_slice()).unwrap_or(&[])
    }
}

// 带权图
pub struct WeightedGraph {
    adj: HashMap<usize, Vec<(usize, i32)>>, // (邻居, 权重)
}
```

## 第二部分：图遍历 (10分钟)

### 2.1 深度优先搜索（DFS）

**画面**：DFS 遍历过程动画，配合递归栈可视化

```rust
pub struct DFS {
    visited: Vec<bool>,
    graph: Graph,
}

impl DFS {
    pub fn new(graph: Graph, n: usize) -> Self {
        DFS {
            visited: vec![false; n],
            graph,
        }
    }
    
    // 递归实现
    pub fn dfs_recursive(&mut self, u: usize) {
        self.visited[u] = true;
        println!("访问节点: {}", u);
        
        for &v in self.graph.neighbors(u) {
            if !self.visited[v] {
                self.dfs_recursive(v);
            }
        }
    }
    
    // 迭代实现（显式栈）
    pub fn dfs_iterative(&mut self, start: usize) {
        let mut stack = vec![start];
        
        while let Some(u) = stack.pop() {
            if self.visited[u] {
                continue;
            }
            
            self.visited[u] = true;
            println!("访问节点: {}", u);
            
            for &v in self.graph.neighbors(u) {
                if !self.visited[v] {
                    stack.push(v);
                }
            }
        }
    }
}
```

**DFS 应用**：
- 连通分量检测
- 拓扑排序
- 环路检测
- 迷宫求解

### 2.2 广度优先搜索（BFS）

**画面**：BFS 层序遍历动画，配合队列可视化

```rust
use std::collections::VecDeque;

pub struct BFS {
    visited: Vec<bool>,
    graph: Graph,
}

impl BFS {
    pub fn new(graph: Graph, n: usize) -> Self {
        BFS {
            visited: vec![false; n],
            graph,
        }
    }
    
    pub fn bfs(&mut self, start: usize) {
        let mut queue = VecDeque::new();
        queue.push_back(start);
        self.visited[start] = true;
        
        while let Some(u) = queue.pop_front() {
            println!("访问节点: {}", u);
            
            for &v in self.graph.neighbors(u) {
                if !self.visited[v] {
                    self.visited[v] = true;
                    queue.push_back(v);
                }
            }
        }
    }
    
    // BFS 求最短路径（无权图）
    pub fn shortest_path(&mut self, start: usize, target: usize) -> Option<usize> {
        let mut queue = VecDeque::new();
        let mut distance = vec![usize::MAX; self.visited.len()];
        
        queue.push_back(start);
        distance[start] = 0;
        
        while let Some(u) = queue.pop_front() {
            if u == target {
                return Some(distance[u]);
            }
            
            for &v in self.graph.neighbors(u) {
                if distance[v] == usize::MAX {
                    distance[v] = distance[u] + 1;
                    queue.push_back(v);
                }
            }
        }
        
        None
    }
}
```

### 讲师对比

> "DFS 和 BFS 是图算法的基石：
> - **DFS** 像探险家，一条路走到底，适合探索、回溯类问题
> - **BFS** 像雷达波，层层扩散，适合最短路径、层级遍历"

### 2.3 DFS vs BFS 对比

| 特性 | DFS | BFS |
|------|-----|-----|
| 数据结构 | 栈（递归或显式） | 队列 |
| 空间复杂度 | $O(h)$，h为深度 | $O(w)$，w为宽度 |
| 最短路径 | 无权图不保证 | 无权图保证最短 |
| 适用场景 | 连通性、拓扑排序 | 最短路径、层级搜索 |

## 第三部分：最短路径算法 (12分钟)

### 3.1 Dijkstra 算法

**画面**：Dijkstra 算法执行过程动画

**适用条件**：非负权边

**核心思想**：贪心策略，每次选择距离起点最近的未确定节点

```rust
use std::collections::{BinaryHeap, HashMap};
use std::cmp::Ordering;

#[derive(Copy, Clone, Eq, PartialEq)]
struct State {
    dist: i32,
    vertex: usize,
}

impl Ord for State {
    fn cmp(&self, other: &Self) -> Ordering {
        other.dist.cmp(&self.dist) // 最小堆
    }
}

impl PartialOrd for State {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}

pub fn dijkstra(
    graph: &HashMap<usize, Vec<(usize, i32)>>,
    start: usize,
    n: usize,
) -> Vec<i32> {
    let mut dist = vec![i32::MAX; n];
    let mut pq = BinaryHeap::new();
    
    dist[start] = 0;
    pq.push(State { dist: 0, vertex: start });
    
    while let Some(State { dist: d, vertex: u }) = pq.pop() {
        if d > dist[u] {
            continue; // 过期条目
        }
        
        if let Some(neighbors) = graph.get(&u) {
            for &(v, weight) in neighbors {
                let new_dist = dist[u] + weight;
                if new_dist < dist[v] {
                    dist[v] = new_dist;
                    pq.push(State { dist: new_dist, vertex: v });
                }
            }
        }
    }
    
    dist
}
```

**复杂度分析**：
- 时间：$O((V + E) \log V)$（使用二叉堆）
- 空间：$O(V)$

### 讲师强调
> "Dijkstra 的贪心选择为什么正确？
> 因为所有边权非负，一旦确定最短距离，后续不可能找到更短路径。
> 如果有负权边，这个假设就失效了。"

### 3.2 Bellman-Ford 算法

**画面**：松弛操作可视化

**适用条件**：可处理负权边（但不能有负权环）

```rust
pub fn bellman_ford(
    edges: &[(usize, usize, i32)], // (u, v, weight)
    start: usize,
    n: usize,
) -> Result<Vec<i32>, &'static str> {
    let mut dist = vec![i32::MAX; n];
    dist[start] = 0;
    
    // 松弛所有边 n-1 次
    for _ in 0..n - 1 {
        for &(u, v, w) in edges {
            if dist[u] != i32::MAX && dist[u] + w < dist[v] {
                dist[v] = dist[u] + w;
            }
        }
    }
    
    // 检测负权环
    for &(u, v, w) in edges {
        if dist[u] != i32::MAX && dist[u] + w < dist[v] {
            return Err("图中存在负权环");
        }
    }
    
    Ok(dist)
}
```

**复杂度分析**：
- 时间：$O(V \times E)$
- 空间：$O(V)$

### 3.3 Floyd-Warshall 算法

**画面**：动态规划表格填充过程

**适用场景**：全源最短路径

```rust
pub fn floyd_warshall(n: usize, edges: &[(usize, usize, i32)]) -> Vec<Vec<i32>> {
    // 初始化距离矩阵
    let mut dist = vec![vec![i32::MAX / 2; n]; n];
    
    for i in 0..n {
        dist[i][i] = 0;
    }
    
    for &(u, v, w) in edges {
        dist[u][v] = w;
    }
    
    // 动态规划：逐个顶点尝试作为中间点
    for k in 0..n {
        for i in 0..n {
            for j in 0..n {
                if dist[i][k] + dist[k][j] < dist[i][j] {
                    dist[i][j] = dist[i][k] + dist[k][j];
                }
            }
        }
    }
    
    dist
}
```

**状态转移方程**：
$$d_{ij}^{(k)} = \min(d_{ij}^{(k-1)}, d_{ik}^{(k-1)} + d_{kj}^{(k-1)})$$

**复杂度分析**：
- 时间：$O(V^3)$
- 空间：$O(V^2)$

### 3.4 最短路径算法选择指南

**画面**：决策树

```
最短路径问题：
├── 单源 + 非负权边 → Dijkstra (O((V+E)logV))
├── 单源 + 有负权边 → Bellman-Ford (O(VE))
├── 单源 + DAG → 拓扑排序 + DP (O(V+E))
└── 全源最短路径 → Floyd-Warshall (O(V³))
```

## 第四部分：最小生成树 (8分钟)

### 4.1 问题定义

**画面**：MST 可视化

> 在连通加权图中，找到一棵包含所有顶点的树，使得边权之和最小。

### 4.2 Kruskal 算法

**核心思想**：按边权从小到大选边，不形成环就选

```rust
pub struct UnionFind {
    parent: Vec<usize>,
    rank: Vec<usize>,
}

impl UnionFind {
    pub fn new(n: usize) -> Self {
        UnionFind {
            parent: (0..n).collect(),
            rank: vec![0; n],
        }
    }
    
    pub fn find(&mut self, x: usize) -> usize {
        if self.parent[x] != x {
            self.parent[x] = self.find(self.parent[x]); // 路径压缩
        }
        self.parent[x]
    }
    
    pub fn union(&mut self, x: usize, y: usize) -> bool {
        let px = self.find(x);
        let py = self.find(y);
        
        if px == py {
            return false; // 已在同一集合
        }
        
        // 按秩合并
        if self.rank[px] < self.rank[py] {
            self.parent[px] = py;
        } else if self.rank[px] > self.rank[py] {
            self.parent[py] = px;
        } else {
            self.parent[py] = px;
            self.rank[px] += 1;
        }
        
        true
    }
}

pub fn kruskal(n: usize, mut edges: Vec<(i32, usize, usize)>) -> i32 {
    // 按权重排序
    edges.sort_by_key(|e| e.0);
    
    let mut uf = UnionFind::new(n);
    let mut mst_weight = 0;
    let mut edges_used = 0;
    
    for (w, u, v) in edges {
        if uf.union(u, v) {
            mst_weight += w;
            edges_used += 1;
            
            if edges_used == n - 1 {
                break; // 已选够 n-1 条边
            }
        }
    }
    
    mst_weight
}
```

**复杂度**：$O(E \log E)$（主要来自排序）

### 4.3 Prim 算法

**核心思想**：从一个顶点开始，每次选择连接树与非树顶点的最小边

```rust
use std::collections::BinaryHeap;
use std::cmp::Ordering;

#[derive(Copy, Clone, Eq, PartialEq)]
struct Edge {
    weight: i32,
    vertex: usize,
}

impl Ord for Edge {
    fn cmp(&self, other: &Self) -> Ordering {
        other.weight.cmp(&self.weight)
    }
}

impl PartialOrd for Edge {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}

pub fn prim(graph: &[Vec<(usize, i32)>], start: usize) -> i32 {
    let n = graph.len();
    let mut visited = vec![false; n];
    let mut pq = BinaryHeap::new();
    let mut mst_weight = 0;
    
    // 从 start 开始
    visited[start] = true;
    for &(v, w) in &graph[start] {
        pq.push(Edge { weight: w, vertex: v });
    }
    
    while let Some(Edge { weight, vertex: u }) = pq.pop() {
        if visited[u] {
            continue;
        }
        
        visited[u] = true;
        mst_weight += weight;
        
        for &(v, w) in &graph[u] {
            if !visited[v] {
                pq.push(Edge { weight: w, vertex: v });
            }
        }
    }
    
    mst_weight
}
```

**复杂度**：$O((V + E) \log V)$（使用二叉堆）

### 4.4 Kruskal vs Prim

| 特性 | Kruskal | Prim |
|------|---------|------|
| 数据结构 | 并查集 | 优先队列 |
| 适用图类型 | 稀疏图（边少） | 稠密图（边多） |
| 时间复杂度 | $O(E \log E)$ | $O(E \log V)$ |
| 空间复杂度 | $O(E)$ | $O(V)$ |

## 第五部分：拓扑排序 (6分钟)

### 5.1 什么是拓扑排序

**画面**：DAG 与拓扑序可视化

> 对有向无环图（DAG）的顶点进行线性排序，使得对于每条边 (u, v)，u 在 v 之前。

**应用**：
- 任务调度（依赖关系）
- 编译顺序
- 课程选修计划

### 5.2 Kahn 算法（BFS）

```rust
use std::collections::VecDeque;

pub fn topological_sort(graph: &[Vec<usize>], n: usize) -> Option<Vec<usize>> {
    // 计算入度
    let mut in_degree = vec![0; n];
    for neighbors in graph {
        for &v in neighbors {
            in_degree[v] += 1;
        }
    }
    
    // 入度为 0 的顶点入队
    let mut queue = VecDeque::new();
    for i in 0..n {
        if in_degree[i] == 0 {
            queue.push_back(i);
        }
    }
    
    let mut result = Vec::new();
    
    while let Some(u) = queue.pop_front() {
        result.push(u);
        
        for &v in &graph[u] {
            in_degree[v] -= 1;
            if in_degree[v] == 0 {
                queue.push_back(v);
            }
        }
    }
    
    // 检查是否有环
    if result.len() == n {
        Some(result)
    } else {
        None // 存在环，无法拓扑排序
    }
}
```

### 5.3 DFS 实现

```rust
pub fn topological_sort_dfs(graph: &[Vec<usize>], n: usize) -> Option<Vec<usize>> {
    let mut visited = vec![false; n];
    let mut on_stack = vec![false; n];
    let mut result = Vec::new();
    
    fn dfs(
        u: usize,
        graph: &[Vec<usize>],
        visited: &mut [bool],
        on_stack: &mut [bool],
        result: &mut Vec<usize>,
    ) -> bool {
        visited[u] = true;
        on_stack[u] = true;
        
        for &v in &graph[u] {
            if !visited[v] {
                if !dfs(v, graph, visited, on_stack, result) {
                    return false; // 发现环
                }
            } else if on_stack[v] {
                return false; // 发现回边，存在环
            }
        }
        
        on_stack[u] = false;
        result.push(u); // 后序遍历加入结果
        true
    }
    
    for i in 0..n {
        if !visited[i] {
            if !dfs(i, graph, &mut visited, &mut on_stack, &mut result) {
                return None;
            }
        }
    }
    
    result.reverse(); // 逆序得到拓扑序
    Some(result)
}
```

## 第六部分：形式化视角与总结 (5分钟)

### 6.1 图算法的形式化规范

**画面**：规范模板

```yaml
算法规范 - Dijkstra:
  输入:
    - 带权图 G = (V, E, w)
    - 源点 s ∈ V
    - 约束: ∀e∈E, w(e) ≥ 0
  
  输出:
    - 距离数组 dist[], ∀v∈V: dist[v] = δ(s, v)
    - 其中 δ(s, v) 表示 s 到 v 的最短路径长度
  
  时间复杂度: O((V + E) log V)
  空间复杂度: O(V)
  
  正确性保证:
    - 当算法终止时，dist[v] = δ(s, v) 对所有 v∈V 成立
    - 前提：图中不存在负权边
```

### 6.2 算法选择速查表

| 问题 | 推荐算法 | 时间复杂度 |
|------|----------|------------|
| 无权图最短路径 | BFS | $O(V + E)$ |
| 非负权单源最短路径 | Dijkstra | $O((V + E) \log V)$ |
| 含负权单源最短路径 | Bellman-Ford | $O(VE)$ |
| 全源最短路径 | Floyd-Warshall | $O(V^3)$ |
| 最小生成树（稀疏图） | Kruskal | $O(E \log E)$ |
| 最小生成树（稠密图） | Prim | $O(E \log V)$ |
| 拓扑排序 | Kahn/DFS | $O(V + E)$ |

## 总结与作业 (2分钟)

### 核心要点

1. **图的表示**：邻接表适合稀疏图，邻接矩阵适合稠密图
2. **遍历是基础**：DFS 适合探索，BFS 适合最短路径

### 常见陷阱

- ❌ 在含负权图上使用 Dijkstra
- ❌ DFS 求无权图最短路径
- ❌ 忽视图的连通性
- ❌ 内存溢出（大规模图）

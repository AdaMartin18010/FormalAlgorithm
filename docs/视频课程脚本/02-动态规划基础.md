## 第一部分：从斐波那契数列说起 (8分钟)

### 1.1 朴素递归的问题

**画面**：递归调用树可视化

```rust
// 朴素的递归实现
fn fib(n: u32) -> u32 {
    if n <= 1 {
        return n;
    }
    fib(n - 1) + fib(n - 2)  // 大量重复计算！
}
```

**递归树**（n=5）：
```
                    fib(5)
                  /        \
            fib(4)          fib(3)
           /      \         /      \
      fib(3)    fib(2)   fib(2)  fib(1)
      /    \    /    \   /    \
  fib(2) fib(1) ...
  
  注意：fib(2) 被计算了 3 次！
```

### 讲师
> "计算 fib(5)，我们调用了 fib(2) 三次。
> 如果计算 fib(100)，fib(2) 会被调用多少次？
> 答案是：约 $2^{98}$ 次——这几乎是宇宙中原子数量的平方！"

**复杂度分析**：
- 时间：$O(2^n)$
- 空间：$O(n)$（递归栈深度）

### 1.2 记忆化搜索（Memoization）

**画面**：哈希表缓存示意

```rust
use std::collections::HashMap;

fn fib_memo(n: u32, memo: &mut HashMap<u32, u32>) -> u32 {
    if let Some(&result) = memo.get(&n) {
        return result;  // 命中缓存！
    }
    
    let result = if n <= 1 {
        n
    } else {
        fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    };
    
    memo.insert(n, result);
    result
}
```

### 讲师
> "解决方案很简单：把计算过的结果存起来。
> 这就是记忆化搜索的核心思想——用空间换时间。
> 每个 fib(k) 只计算一次，之后直接查表。"

**优化后复杂度**：
- 时间：$O(n)$ —— 每个子问题只算一次
- 空间：$O(n)$ —— 存储记忆化表

### 1.3 自底向上递推

**画面**：表格填充动画

```rust
fn fib_dp(n: u32) -> u32 {
    if n <= 1 {
        return n;
    }
    
    let mut dp = vec![0; (n + 1) as usize];
    dp[0] = 0;
    dp[1] = 1;
    
    for i in 2..=n {
        dp[i as usize] = dp[i as usize - 1] + dp[i as usize - 2];
    }
    
    dp[n as usize]
}
```

**优化版本**（滚动数组）：
```rust
fn fib_optimized(n: u32) -> u32 {
    if n <= 1 {
        return n;
    }
    
    let (mut prev2, mut prev1) = (0, 1);  // 只保留最近两个值
    
    for _ in 2..=n {
        let current = prev1 + prev2;
        prev2 = prev1;
        prev1 = current;
    }
    
    prev1
}
```

### 讲师
> "观察发现，计算 fib(i) 只需要 fib(i-1) 和 fib(i-2)。
> 所以我们不需要存储整个数组，只保留前两个值就够了。
> 这就是空间优化的技巧。"

## 第二部分：动态规划的本质 (8分钟)

### 2.1 两个核心性质

**画面**：性质对比图

```
┌────────────────────────────────────────────────────────────┐
│              动态规划的两个核心性质                           │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  1. 最优子结构 (Optimal Substructure)                       │
│     └── 问题的最优解包含子问题的最优解                        │
│     └── 例：最短路径的子路径也是最短路径                      │
│                                                            │
│  2. 重叠子问题 (Overlapping Subproblems)                    │
│     └── 递归求解时会重复遇到相同的子问题                      │
│     └── 例：斐波那契的 fib(2) 被多次计算                     │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 2.2 DP 解题框架

**画面**：流程图

```
解决 DP 问题的四步法：

Step 1: 定义状态
    └── 明确 dp[i] 或 dp[i][j] 表示什么
    
Step 2: 寻找状态转移方程
    └── dp[i] 与 dp[i-1], dp[i-2], ... 的关系
    
Step 3: 确定初始条件
    └── dp[0], dp[1] 等边界值
    
Step 4: 确定遍历顺序
    └── 确保计算 dp[i] 时，依赖的状态已计算
```

### 讲师
> "记住这个框架，它能帮你系统化地解决 90% 的 DP 问题。
> 很多初学者卡在第 2 步，我的建议是：先尝试暴力递归，
> 然后观察重复计算的模式，状态转移方程自然就会浮现。"

### 2.3 实现方式对比

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 记忆化搜索 | 直观，只计算需要的子问题 | 递归开销 | 状态空间稀疏 |
| 自底向上 | 无递归开销，可空间优化 | 可能计算无用状态 | 状态空间密集 |

## 第三部分：经典问题实战 (15分钟)

### 3.1 0/1 背包问题

**问题描述**：
> 有 n 个物品，第 i 个物品重量为 w[i]，价值为 v[i]。
> 背包容量为 C，每个物品只能选或不选，求能装下的最大价值。

**画面**：背包问题可视化

**Step 1: 定义状态**
```
dp[i][j] = 考虑前 i 个物品，背包容量为 j 时的最大价值
```

**Step 2: 状态转移**
```
对于第 i 个物品，有两种选择：

不选：dp[i][j] = dp[i-1][j]
选（前提是 j >= w[i]）：dp[i][j] = dp[i-1][j-w[i]] + v[i]

dp[i][j] = max(不选, 选)
```

**实现代码**：
```rust
fn knapsack(weights: &[i32], values: &[i32], capacity: i32) -> i32 {
    let n = weights.len();
    let mut dp = vec![vec![0; (capacity + 1) as usize]; n + 1];
    
    for i in 1..=n {
        for j in 0..=capacity {
            // 不选第 i 个物品
            dp[i][j as usize] = dp[i - 1][j as usize];
            
            // 选第 i 个物品（如果能放下）
            if j >= weights[i - 1] {
                dp[i][j as usize] = dp[i][j as usize].max(
                    dp[i - 1][(j - weights[i - 1]) as usize] + values[i - 1]
                );
            }
        }
    }
    
    dp[n][capacity as usize]
}
```

**空间优化**（一维数组）：
```rust
fn knapsack_optimized(weights: &[i32], values: &[i32], capacity: i32) -> i32 {
    let mut dp = vec![0; (capacity + 1) as usize];
    
    for i in 0..weights.len() {
        // 倒序遍历！防止覆盖还需使用的值
        for j in (weights[i]..=capacity).rev() {
            dp[j as usize] = dp[j as usize].max(
                dp[(j - weights[i]) as usize] + values[i]
            );
        }
    }
    
    dp[capacity as usize]
}
```

**复杂度**：
- 时间：$O(n \times C)$
- 空间：$O(C)$（优化后）

### 讲师强调
> "为什么内层循环要倒序？
> 因为 dp[j] 依赖于 dp[j - w[i]]，
> 如果正序遍历，dp[j - w[i]] 可能已经被当前物品更新过，
> 导致一个物品被重复选取——这就变成完全背包了。"

### 3.2 最长公共子序列（LCS）

**问题描述**：
> 给定两个字符串 s1 和 s2，找出它们的最长公共子序列的长度。
> 子序列不要求连续，但要求相对顺序一致。

**画面**：字符串对比可视化

**Step 1: 定义状态**
```
dp[i][j] = s1[0..i] 和 s2[0..j] 的 LCS 长度
```

**Step 2: 状态转移**
```
如果 s1[i-1] == s2[j-1]：
    dp[i][j] = dp[i-1][j-1] + 1  // 匹配，长度+1
否则：
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])  // 取较大的
```

**实现代码**：
```rust
fn lcs(s1: &str, s2: &str) -> usize {
    let m = s1.len();
    let n = s2.len();
    let s1: Vec<char> = s1.chars().collect();
    let s2: Vec<char> = s2.chars().collect();
    
    let mut dp = vec![vec![0; n + 1]; m + 1];
    
    for i in 1..=m {
        for j in 1..=n {
            if s1[i - 1] == s2[j - 1] {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            } else {
                dp[i][j] = dp[i - 1][j].max(dp[i][j - 1]);
            }
        }
    }
    
    dp[m][n]
}
```

**复杂度**：
- 时间：$O(m \times n)$
- 空间：$O(m \times n)$，可优化至 $O(\min(m, n))$

### 3.3 路径问题（三角形最小路径和）

**问题**：从三角形顶部到底部，求最小路径和，每次只能移动到下一行相邻的节点。

```
     2
    3 4
   6 5 7
  4 1 8 3
```

**自顶向下分析**：
```rust
fn minimum_total(triangle: Vec<Vec<i32>>) -> i32 {
    let n = triangle.len();
    let mut dp = vec![vec![0; n]; n];
    dp[0][0] = triangle[0][0];
    
    for i in 1..n {
        for j in 0..=i {
            if j == 0 {
                // 最左边，只能从左上方来
                dp[i][j] = dp[i-1][j] + triangle[i][j];
            } else if j == i {
                // 最右边，只能从右上方来
                dp[i][j] = dp[i-1][j-1] + triangle[i][j];
            } else {
                // 中间，取左上和右上的最小值
                dp[i][j] = dp[i-1][j-1].min(dp[i-1][j]) + triangle[i][j];
            }
        }
    }
    
    *dp[n-1].iter().min().unwrap()
}
```

**自底向上优化**（空间 $O(n)$）：
```rust
fn minimum_total_optimized(triangle: Vec<Vec<i32>>) -> i32 {
    let n = triangle.len();
    let mut dp = triangle[n-1].clone();  // 从最后一行开始
    
    for i in (0..n-1).rev() {
        for j in 0..=i {
            dp[j] = dp[j].min(dp[j+1]) + triangle[i][j];
        }
    }
    
    dp[0]
}
```

## 第五部分：形式化规范视角 (2分钟)

### 动态规划的形式化描述

**画面**：数学公式展示

```
给定：
- 状态空间 S
- 决策空间 D
- 状态转移函数 T: S × D → S
- 阶段收益函数 g: S × D → ℝ
- 目标函数（最大化或最小化总收益）

动态规划求解：
V(s) = opt_{d∈D} { g(s,d) + V(T(s,d)) }

其中 opt ∈ {max, min}
```

### 最优子结构定理

> **定理**：如果问题的最优解包含子问题的最优解，则称该问题具有最优子结构性质。
> 这是动态规划适用的必要条件。

## 总结与作业 (2分钟)

### 核心要点

1. **DP 本质**：通过存储子问题解避免重复计算
2. **两个性质**：最优子结构 + 重叠子问题
3. **四步框架**：定义状态 → 转移方程 → 初始条件 → 遍历顺序
4. **实现选择**：记忆化搜索（直观）vs 自底向上（高效）

### 常见错误

- ❌ 状态定义不清晰
- ❌ 遍历顺序错误
- ❌ 边界条件处理不当
- ❌ 空间优化时覆盖未使用的值

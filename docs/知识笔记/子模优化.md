## 一、核心概念

### 1.1 定义

**正式定义**:
设 $N = \{1, 2, ..., n\}$ 为基集，函数 $f: 2^N \to \mathbb{R}$ 是**子模函数**（submodular），如果满足以下等价条件之一：

**定义1** (边际收益递减):
$$f(A \cup \{i\}) - f(A) \geq f(B \cup \{i\}) - f(B), \quad \forall A \subseteq B \subseteq N, i \notin B$$

**定义2** (集合交性质):
$$f(A) + f(B) \geq f(A \cap B) + f(A \cup B), \quad \forall A, B \subseteq N$$

**定义3** (离散Hessian):
$$f_{i,j}(A) := f(A \cup \{i,j\}) - f(A \cup \{i\}) - f(A \cup \{j\}) + f(A) \leq 0$$

**单调子模函数**: 若 $A \subseteq B \Rightarrow f(A) \leq f(B)$

**直观解释**:
子模性捕捉了"边际收益递减"的经济学原理。想象你收集一套邮票：
- 第一张稀有邮票带来巨大满足感
- 第二张同类型的满足感降低
- 第100张几乎无额外价值

这种"多样性收益"正是子模性的体现。

**关键要点**:
- 子模性是离散版本的凸性
- 最小化子模函数可在多项式时间完成
- 最大化单调子模函数是NP难的，但贪心算法有 $(1-1/e)$ 近似

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 闭包性质 | 子模函数的非负线性组合、取小、取大仍是子模 | 重要组合工具 |
| Lovász扩展 | 将离散函数扩展到连续域保持凸性 | 连续优化连接 |
| 多面体结构 | 基多面体（base polyhedron）有良好结构 | 组合优化基础 |
| 对称性 | 对称子模函数有特殊算法 | 如最小割问题 |

**性质总结**:
1. **多项式可求值**: 大多数算法假设存在 $f(S)$ 的oracle，调用时间为 $EO$
2. **不可表示性**: 某些子模函数需要指数空间显式表示
3. **学习复杂性**: 从样本学习子模函数需要指数样本

### 1.3 变体

**对称子模函数**:
- 定义: $f(A) = f(N \setminus A)$
- 与标准形式的区别: 最小化等价于最小割
- 适用场景: 聚类、图像分割

**基数约束**:
- 定义: 最大化 $f(S)$ subject to $|S| \leq k$
- 与标准形式的区别: 最常见的约束类型
- 适用场景: 传感器放置、文档摘要

**拟阵约束**:
- 定义: $S$ 必须是拟阵的独立集
- 与标准形式的区别: 更一般的组合约束
- 适用场景: 任务分配、实验设计

**流式子模优化**:
- 定义: 元素以流式到达，必须立即决策
- 与标准形式的区别: 不能回溯修改决策
- 适用场景: 实时推荐、在线广告

## 三、形式化证明

### 3.1 核心定理

**定理 1** (贪心算法的近似比):
对于单调子模函数 $f$，在基数约束 $|S| \leq k$ 下，贪心算法达到 $(1-1/e)$-近似，这是最优的。

**贪心算法**:
```
初始化 S₀ = ∅
for i = 1 to k do:
    选择 e ∈ N\S_{i-1} 最大化 f(S_{i-1} ∪ {e}) - f(S_{i-1})
    S_i = S_{i-1} ∪ {e}
return S_k
```

**证明**:

设 $S^*$ 为最优解，$|S^*| = k$。令 $S_i$ 为贪心算法第 $i$ 步的集合。

**关键引理**: 对任意 $i$，存在 $e \in S^* \setminus S_i$ 使得：
$$f(S_i \cup \{e\}) - f(S_i) \geq \frac{1}{k}(f(S^*) - f(S_i))$$

*引理证明*:
由子模性（边际收益递减）:
$$\sum_{e \in S^* \setminus S_i} [f(S_i \cup \{e\}) - f(S_i)] \geq \sum_{e \in S^* \setminus S_i} [f(S_i \cup S^*) - f(S_i)]$$

由单调性:
$$\geq f(S^*) - f(S_i)$$

因此平均边际收益至少为 $(f(S^*) - f(S_i))/|S^* \setminus S_i| \geq (f(S^*) - f(S_i))/k$。

贪心选择最大化边际收益，故引理成立。

令 $\delta_i = f(S^*) - f(S_i)$，则：
$$\delta_{i+1} = \delta_i - [f(S_{i+1}) - f(S_i)] \leq \delta_i - \frac{\delta_i}{k} = (1 - \frac{1}{k})\delta_i$$

递归应用：
$$\delta_k \leq (1 - \frac{1}{k})^k \delta_0 = (1 - \frac{1}{k})^k f(S^*) \leq \frac{1}{e} f(S^*)$$

因此：
$$f(S_k) = f(S^*) - \delta_k \geq (1 - \frac{1}{e}) f(S^*)$$

**最优性**: Feige (1998) 证明在 $P \neq NP$ 假设下，任何多项式算法近似比不超过 $(1-1/e+\epsilon)$。∎

**证明要点分析**:
1. **边际收益分析**: 贪心每步选择最大边际收益元素
2. **次优性上界**: 证明剩余最优值衰减速度
3. **指数衰减**: 误差呈指数衰减，累积成 $(1-1/e)$ 因子

### 3.2 辅助引理

**引理 1** (子模性的等价形式):
函数 $f: 2^N \to \mathbb{R}$ 是子模的当且仅当对任意 $A \subseteq B \subseteq N$ 和 $i \notin B$：
$$f(A \cup \{i\}) - f(A) \geq f(B \cup \{i\}) - f(B)$$

*证明*:
设 $f$ 满足边际收益递减。对任意 $A, B$，令 $B \setminus A = \{e_1, ..., e_k\}$。

由边际收益递减：
$$f(A \cup \{e_1\}) - f(A) \geq f(B \setminus \{e_2, ..., e_k\} \cup \{e_1\}) - f(B \setminus \{e_2, ..., e_k\})$$

累加所有元素：
$$\sum_{i=1}^k [f(A \cup \{e_1,...,e_i\}) - f(A \cup \{e_1,...,e_{i-1}\})] \geq \sum_{i=1}^k [f(B \setminus \{e_{i+1},...,e_k\}) - f(B \setminus \{e_i,...,e_k\})]$$

左边 $= f(A \cup B) - f(A)$，右边 $= f(B) - f(A \cap B)$。

整理得 $f(A) + f(B) \geq f(A \cap B) + f(A \cup B)$。∎

## 四、算法/方法详解

### 4.1 贪心算法

**伪代码**:
```
算法: 贪心子模最大化 (GreedySubmodular)
输入: 单调子模函数 f, 基集 N, 约束 k
输出: 集合 S, |S| ≤ k

1.  S ← ∅
2.  for i = 1 to k do
3.      e* ← argmax_{e ∈ N\S} [f(S ∪ {e}) - f(S)]
4.      if f(S ∪ {e*}) - f(S) ≤ 0 then break
5.      S ← S ∪ {e*}
6.  end for
7.  return S
```

**流程图**:
```
初始化 S = ∅
    ↓
迭代 i = 1 to k:
    ↓
计算所有 e ∈ N\S 的边际收益 Δf(e|S)
    ↓
选择 e* = argmax Δf(e|S)
    ↓
S ← S ∪ {e*}
    ↓
返回 S
```

### 4.2 连续贪心算法

对于拟阵约束，连续贪心算法达到 $(1-1/e)$-近似：

```
算法: ContinuousGreedy
输入: 单调子模函数 f, 拟阵 M = (N, I)
输出: 可行解 S ∈ I

1.  y(0) ← 0
2.  for t ∈ [0, 1] do
3.      v(t) ← argmax_{v ∈ P(M)} Σ_i v_i · ∂F/∂y_i(y(t))
4.      dy/dt ← v(t)
5.  end for
6.  return 舍入 y(1) 为离散解
```

其中 $P(M)$ 是拟阵多面体，$F$ 是多线性扩展。

### 4.3 复杂度分析

**贪心算法**:
- **时间复杂度**: $O(k \cdot n \cdot EO)$，$EO$ 是函数求值时间
- **空间复杂度**: $O(n)$ 存储边际收益
- **近似比**: $1 - 1/e \approx 0.632$

**加速技术**:
| 技术 | 时间复杂度 | 说明 |
|------|------------|------|
| Lazy Evaluation | $O(k \cdot EO \cdot \log n)$ | 利用子模性剪枝 |
| 随机贪心 | $O(k \cdot EO)$ | 随机采样评估 |
| 分布式贪心 | $O(k \cdot EO / p)$ | $p$ 个并行处理器 |

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 传感器放置问题

**问题描述**:
在 $n$ 个候选位置放置 $k$ 个传感器，最大化环境监测的覆盖范围。

**子模建模**:
设 $f(S)$ 为位置集合 $S$ 能监测到的总信息量。若每个传感器覆盖区域有重叠，则：
$$f(S) = H(\bigcup_{i \in S} \text{Region}_i)$$

信息熵 $H$ 是子模的（信息重叠部分只计一次）。

**执行过程**:
```
k=3, n=10
Step 1: 选择覆盖最大独立区域的位置 → S={5}, f(S)=0.3
Step 2: 选择边际增益最大的位置 → S={5,2}, f(S)=0.52
Step 3: 选择边际增益最大的位置 → S={5,2,8}, f(S)=0.65

最优解 f(S*) ≈ 0.71
近似比: 0.65/0.71 ≈ 0.915 > 0.632 ✓
```

**示例 2**: 文档摘要提取

**问题描述**:
从文档集合中选择 $k$ 个句子，形成摘要最大化信息覆盖。

**子模建模**:
设文档 $D$ 的句子集合为 $N$，选择 $S \subseteq N$：
$$f(S) = \sum_{w \in V} \max_{s \in S} \text{sim}(w, s)$$

其中 $V$ 是词汇集合，$\text{sim}$ 是相似度函数。这是子模的（覆盖函数）。

### 5.2 代码实现

**语言**: Python

```python
import numpy as np
from typing import Callable, Set, List
import random


class SubmodularOptimizer:
    """
    子模函数优化器
    """
    def __init__(self, f: Callable[[Set[int]], float], n: int):
        """
        f: 子模函数 oracle
        n: 基集大小 N = {0, 1, ..., n-1}
        """
        self.f = f
        self.n = n
        self.eval_count = 0  # 统计函数求值次数
    
    def evaluate(self, S: Set[int]) -> float:
        """包装函数求值，用于统计"""
        self.eval_count += 1
        return self.f(S)
    
    def marginal_gain(self, S: Set[int], e: int) -> float:
        """计算元素 e 加入 S 的边际收益"""
        if e in S:
            return 0.0
        S_with_e = S | {e}
        return self.evaluate(S_with_e) - self.evaluate(S)
    
    def greedy_maximize(self, k: int, lazy: bool = True) -> Set[int]:
        """
        贪心算法最大化子模函数
        
        Args:
            k: 基数约束
            lazy: 是否使用 lazy evaluation 加速
        
        Returns:
            选中的集合 S
        """
        self.eval_count = 0
        S = set()
        
        if lazy:
            # Lazy greedy: 使用优先队列维护上界
            from heapq import heappush, heappop
            
            # 初始化所有边际收益上界
            heap = []
            for e in range(self.n):
                gain = self.marginal_gain(S, e)
                heappush(heap, (-gain, e))
            
            while len(S) < k and heap:
                neg_gain, e = heappop(heap)
                gain = -neg_gain
                
                # 重新计算边际收益
                actual_gain = self.marginal_gain(S, e)
                
                if e not in S and actual_gain > 0:
                    if actual_gain >= -heap[0][0] if heap else True:
                        # 这是当前最大边际收益
                        S.add(e)
                    else:
                        # 重新入队
                        heappush(heap, (-actual_gain, e))
        else:
            # 标准贪心
            available = set(range(self.n))
            while len(S) < k and available:
                best_e = None
                best_gain = -float('inf')
                
                for e in available:
                    gain = self.marginal_gain(S, e)
                    if gain > best_gain:
                        best_gain = gain
                        best_e = e
                
                if best_gain <= 0:
                    break
                
                S.add(best_e)
                available.remove(best_e)
        
        return S
    
    def local_search(self, k: int, max_iter: int = 1000) -> Set[int]:
        """
        局部搜索算法
        """
        self.eval_count = 0
        
        # 随机初始化
        S = set(random.sample(range(self.n), min(k, self.n)))
        current_value = self.evaluate(S)
        
        for _ in range(max_iter):
            improved = False
            
            # 尝试交换
            for e_in in S:
                for e_out in set(range(self.n)) - S:
                    S_new = (S - {e_in}) | {e_out}
                    if len(S_new) == k:
                        new_value = self.evaluate(S_new)
                        if new_value > current_value:
                            S = S_new
                            current_value = new_value
                            improved = True
                            break
                if improved:
                    break
            
            if not improved:
                break
        
        return S


# 示例子模函数：覆盖函数
def coverage_function(edges: List[Set[int]]) -> Callable[[Set[int]], float]:
    """
    创建覆盖函数 f(S) = |∪_{i ∈ S} edges[i]|
    
    Args:
        edges: 每个元素覆盖的集合列表
    """
    def f(S: Set[int]) -> float:
        covered = set()
        for i in S:
            if i < len(edges):
                covered |= edges[i]
        return float(len(covered))
    return f


# 示例：传感器放置
def sensor_coverage_utility(sensor_ranges: List[List[int]], 
                           weights: np.ndarray = None) -> Callable[[Set[int]], float]:
    """
    传感器覆盖效用函数
    
    Args:
        sensor_ranges: sensor_ranges[i] 表示第i个传感器能覆盖的点列表
        weights: 每个点的权重，默认为均匀权重
    """
    n_points = max(max(r) for r in sensor_ranges) + 1
    if weights is None:
        weights = np.ones(n_points)
    
    def f(S: Set[int]) -> float:
        covered = set()
        for i in S:
            if i < len(sensor_ranges):
                covered |= set(sensor_ranges[i])
        return sum(weights[p] for p in covered)
    
    return f


# 测试
if __name__ == "__main__":
    print("=" * 60)
    print("子模优化算法测试")
    print("=" * 60)
    
    # 测试1: 简单覆盖函数
    print("\n测试1: 集合覆盖问题")
    print("-" * 40)
    
    # 10个元素，每个覆盖一些点
    edges = [
        {0, 1, 2},      # 元素0覆盖点0,1,2
        {2, 3, 4},      # 元素1覆盖点2,3,4
        {4, 5, 6},      # 元素2覆盖点4,5,6
        {6, 7, 8},      # 元素3覆盖点6,7,8
        {8, 9, 0},      # 元素4覆盖点8,9,0
        {1, 3, 5},      # 元素5覆盖点1,3,5
        {7, 9},         # 元素6覆盖点7,9
    ]
    
    f = coverage_function(edges)
    opt = SubmodularOptimizer(f, n=7)
    
    # 比较标准贪心和Lazy贪心
    print("标准贪心算法:")
    S1 = opt.greedy_maximize(k=3, lazy=False)
    print(f"  选中: {S1}, 函数值: {f(S1):.2f}, 求值次数: {opt.eval_count}")
    
    print("Lazy贪心算法:")
    S2 = opt.greedy_maximize(k=3, lazy=True)
    print(f"  选中: {S2}, 函数值: {f(S2):.2f}, 求值次数: {opt.eval_count}")
    
    # 测试2: 局部搜索
    print("\n局部搜索算法:")
    S3 = opt.local_search(k=3)
    print(f"  选中: {S3}, 函数值: {f(S3):.2f}, 求值次数: {opt.eval_count}")
    
    # 验证子模性（抽样验证）
    print("\n子模性验证 (抽样):")
    A = {0, 1}
    B = {0, 1, 2}
    e = 3
    
    gain_A = f(A | {e}) - f(A)
    gain_B = f(B | {e}) - f(B)
    
    print(f"  f(A ∪ {{e}}) - f(A) = {gain_A:.2f}")
    print(f"  f(B ∪ {{e}}) - f(B) = {gain_B:.2f}")
    print(f"  边际收益递减: {gain_A >= gain_B} (应满足 ≥)")
```

### 5.3 反例

**常见错误1**: 忽略单调性假设
```python
# 错误：对非单调子模函数使用标准贪心
f = lambda S: len(S) - 2 * len(S)  # 非单调
opt.greedy_maximize(k=5)  # 可能输出差的结果
```
**错误原因**: 标准贪心分析假设单调性
**正确做法**: 非单调函数需要随机贪心或局部搜索（近似比1/2）

**常见错误2**: 混淆子模最小化和最大化
```python
# 错误：用贪心算法最小化子模函数
# 子模最小化是多项式可解的，贪心不适用
```
**错误原因**: 最小化需要其他算法（如凸优化、组合算法）
**正确做法**: 使用Fujishige-Wolfe算法或基于Lovász扩展的梯度方法

## 六、习题

### 6.1 理解题 (L1-L2)

1. **子模性验证** [难度⭐]
   
   证明 $f(S) = \sqrt{|S|}$ 是子模函数。
   
   <details>
   <summary>解答</summary>
   
   验证边际收益递减：
   $$f(A \cup \{e\}) - f(A) = \sqrt{|A|+1} - \sqrt{|A|}$$
   
   函数 $g(x) = \sqrt{x+1} - \sqrt{x}$ 关于 $x$ 递减：
   $$g'(x) = \frac{1}{2\sqrt{x+1}} - \frac{1}{2\sqrt{x}} < 0$$
   
   因此当 $|A| \leq |B|$ 时，边际收益递减，函数子模。∎
   
   </details>

2. **熵的子模性** [难度⭐⭐]
   
   解释为什么联合熵 $H(X_S)$ 是随机变量子集的子模函数。
   
   <details>
   <summary>解答</summary>
   
   联合熵：$H(X_S) = -\sum p(x_S) \log p(x_S)$
   
   条件熵：$H(X_e | X_A) = H(X_{A \cup \{e\}}) - H(X_A)$
   
   信息论中，条件作用减少熵（$H(X|Y) \leq H(X)$），且条件越多熵越小：
   $$A \subseteq B \Rightarrow H(X_e | X_A) \geq H(X_e | X_B)$$
   
   这正是边际收益递减的定义。∎
   
   </details>

### 6.2 应用题 (L3)

3. ** influence最大化** [难度⭐⭐⭐]
   
   在独立级联模型下，证明影响传播函数 $\sigma(S)$ 是子模的。
   
   <details>
   <summary>解答</summary>
   
   **独立级联模型**: 每条边 $(u,v)$ 有激活概率 $p(u,v)$。从种子集 $S$ 开始，节点按概率独立尝试激活邻居。
   
   **证明思路**:
   使用活化边视角。设 $X$ 为所有边的活化状态（随机变量）。
   
   对固定的 $X$，被 $S$ 影响的节点集合是确定性的：即 $S$ 在活化图中的可达节点集。
   
   可达节点函数是覆盖函数（子模）。$\sigma(S) = E_X[\sigma_X(S)]$ 是子模函数的期望，仍子模。
   
   因此影响最大化可用贪心算法，达到 $(1-1/e)$-近似。∎
   
   </details>

### 6.3 证明题 (L4-L5)

4. **连续贪心的最优性** [难度⭐⭐⭐⭐]
   
   解释为什么连续贪心算法在拟阵约束下能达到 $(1-1/e)$-近似。
   
   <details>
   <summary>解答</summary>
   
   **多线性扩展**:
   $$F(y) = \sum_{S \subseteq N} f(S) \prod_{i \in S} y_i \prod_{i \notin S} (1-y_i)$$
   
   **关键性质**:
   1. $F$ 是多项式时间可计算的（使用采样）
   2. $F$ 沿非负方向是凹的（DR-子模）
   3. $F(1_S) = f(S)$
   
   **算法分析**:
   连续贪心沿拟阵多面体的极值点方向移动，每一步最大化 $F$ 的梯度方向。
   
   由微分不等式：
   $$\frac{dF(y(t))}{dt} \geq F(y \vee 1_{OPT}) - F(y(t))$$
   
   解得 $F(y(1)) \geq (1-1/e) f(OPT)$。
   
   舍入保持可行性（pipage rounding或交换舍入）。∎
   
   </details>

5. **子模最小化的复杂度** [难度⭐⭐⭐⭐⭐]
   
   证明子模函数最小化可以在多项式时间完成。
   
   <details>
   <summary>解答</summary>
   
   **Lovász扩展方法**:
   
   Lovász扩展 $\hat{f}: [0,1]^n \to \mathbb{R}$ 定义为：
   $$\hat{f}(x) = \sum_{i=0}^{n} (t_i - t_{i-1}) f(S_i)$$
   
   其中 $t_0 = 0 < t_1 < ... < t_k$ 是 $x$ 的不同值，$S_i = \{j: x_j > t_i\}$。
   
   **关键定理**: $\hat{f}$ 是凸函数，且 $\min_{S} f(S) = \min_{x \in [0,1]^n} \hat{f}(x)$。
   
   **算法**:
   1. 使用椭球法或梯度下降最小化 $\hat{f}$
   2. 凸优化得到 $x^*$
   3. 从 $x^*$ 提取阈值集合
   
   **复杂度**: $O(n^5 \cdot EO + n^7)$（椭球法），或 $O(n^2 \cdot EO)$（改进算法）。∎
   
   </details>

## 八、多维对比

### 8.1 算法对比

| 算法 | 约束类型 | 近似比 | 时间复杂度 | 适用场景 |
|------|----------|--------|------------|----------|
| 贪心算法 | 基数约束 | 1-1/e | O(kn·EO) | 标准场景 |
| Lazy贪心 | 基数约束 | 1-1/e | O(k·EO·log n) | 大规模问题 |
| 连续贪心 | 拟阵约束 | 1-1/e | poly(n, EO) | 复杂约束 |
| 局部搜索 | 一般 | 1/2 | O(n^k) | 非单调函数 |
| 随机贪心 | 基数约束 | 1-1/e (期望) | O(k·EO) | 快速近似 |
| 分布式贪心 | 基数约束 | 1/2 | O(k·EO/p) | 大规模并行 |

### 8.2 问题变体对比

| 问题 | 计算复杂度 | 最佳近似比 | 主要算法 |
|------|------------|------------|----------|
| 子模最小化 | P | 精确 | 凸优化、组合算法 |
| 单调子模最大化 | NP难 | 1-1/e | 贪心 |
| 非单调子模最大化 | NP难 | 1/2 | 随机贪心、局部搜索 |
| 鲁棒子模优化 | NP难 | 依赖模型 | 双层优化 |
| 流式子模优化 | - | 1/2 | 单遍贪心 |

### 8.3 决策建议

**算法选择流程**:
```
问题类型？
├── 最小化
│   └── 使用凸优化方法（椭球法、梯度下降）
└── 最大化
    ├── 函数是否单调？
    │   ├── 否 → 随机贪心或局部搜索（1/2近似）
    │   └── 是 → 约束类型？
    │       ├── 基数约束 → 贪心算法（1-1/e）
    │       ├── 拟阵约束 → 连续贪心（1-1/e）
    │       └── 背包约束 → 贪心+部分枚举
    └── 规模？
        ├── n > 10^6 → Lazy贪心或采样
        └── 分布式 → 分布式贪心或核心集
```

## 一、核心概念

### 1.1 定义

**正式定义**:
归并排序是一种基于**分治策略**的稳定排序算法，由John von Neumann于1945年发明。算法将数组递归地分成两半，分别排序后合并两个有序子数组。

**递归定义**:

```
MergeSort(A, p, r):
    if p < r then
        q = ⌊(p + r) / 2⌋
        MergeSort(A, p, q)
        MergeSort(A, q+1, r)
        Merge(A, p, q, r)
```

**直观解释**:
想象整理书架上的书：

1. 将书分成两堆
2. 每堆再分成两堆，直到只剩一本书（自然有序）
3. 将两堆有序的书按顺序合并
4. 逐层向上合并，最终得到有序书架

**关键要点**:

- **稳定排序**：相等元素保持相对顺序
- **保证$O(n \log n)$**：最坏、平均、最好情况均为$O(n \log n)$
- **非原地排序**：需要$O(n)$额外空间
- **适合链表**：不需要随机访问，适合链表排序

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 时间复杂度 | $O(n \log n)$ | 所有情况（最好/平均/最坏） |
| 空间复杂度 | $O(n)$ | 需要额外数组存储临时结果 |
| 稳定性 | 稳定 | 相等元素保持原始顺序 |
| 比较次数 | $O(n \log n)$ | 精确上界：$n\lceil\log_2 n\rceil - 2^{\lceil\log_2 n\rceil} + 1$ |
| 数据访问模式 | 顺序访问 | 适合外部排序、链表 |
| 并行性 | 良好 | 子问题独立，易于并行 |

**性质总结**:

1. **最优比较排序**: 比较排序的下界为$\Omega(n \log n)$，归并排序达到该下界
2. **稳定性保证**: 合并时严格使用"≤"而非"<"保持稳定性
3. **自然归并排序**: 利用输入中已有的有序段(run)减少合并次数

### 1.3 变体

**自底向上归并排序**:

- 定义: 非递归实现，从单个元素开始逐层合并
- 与标准形式的区别: 避免递归开销，适合迭代实现
- 适用场景: 递归深度受限的环境

**自然归并排序**:

- 定义: 识别输入中的有序段(run)作为初始子数组
- 与标准形式的区别: 自适应算法，接近有序时接近$O(n)$
- 适用场景: 部分有序或含大量有序段的数据

**TimSort**:

- 定义: 归并排序+插入排序的混合，Python/Java标准库实现
- 与标准形式的区别: 识别runs，小数组用插入排序
- 适用场景: 真实世界数据（通常含局部有序模式）

**原地归并排序**:

- 定义: 尝试减少空间复杂度到$O(1)$或$O(\log n)$
- 与标准形式的区别: 使用复杂技术如块交换
- 适用场景: 内存极度受限

## 三、形式化证明

### 3.1 核心定理

**定理 1** (归并排序正确性): 归并排序对任意输入数组A[p..r]都能正确排序。

**证明**（使用强数学归纳法）:

```
设P(n): 归并排序能正确排序任意长度为n的数组。

基例: P(0)和P(1)显然成立（空数组或单元素已排序）。

归纳假设: 假设对所有k < n，P(k)成立。

归纳步骤: 考虑长度为n的数组A[p..r]，其中n = r-p+1 > 1。

1. 计算中点q = ⌊(p+r)/2⌋，将数组分为两部分：
   - A[p..q]，长度为n₁ = q-p+1 < n
   - A[q+1..r]，长度为n₂ = r-q < n

2. 递归调用：
   - MergeSort(A, p, q) 由归纳假设正确排序A[p..q]
   - MergeSort(A, q+1, r) 由归纳假设正确排序A[q+1..r]

3. 合并操作：
   Merge(A, p, q, r)将两个有序子数组合并为一个有序数组。
   合并过程逐个选择两个子数组中的较小元素，保证输出有序。

因此，A[p..r]被正确排序，P(n)成立。∎
```

**证明要点分析**:

1. **强归纳法**: 因为子问题规模n₁和n₂不确定哪个更小
2. **合并正确性**: 双指针技巧保证有序性
3. **终止保证**: 每次递归规模严格减小

### 3.2 复杂度分析证明

**定理 2** (时间复杂度): 归并排序时间复杂度为$\Theta(n \log n)$。

**证明**:

```
递归关系（假设n是2的幂）：
T(n) = 2T(n/2) + Θ(n)

解释：
- 2T(n/2): 两个子问题的求解时间
- Θ(n): 合并两个子数组的时间

使用递归树法：

第0层: n              工作量: n
第1层: n/2, n/2      工作量: n
第2层: n/4 × 4       工作量: n
...
第k层: n/2^k × 2^k   工作量: n

树高: log₂n（因为n/2^h = 1时停止）
总工作量: n × log₂n = Θ(n log n)

或使用主定理：
a = 2, b = 2, f(n) = Θ(n)
n^(log_b a) = n^1 = n
属于Case 2: f(n) = Θ(n^(log_b a))
因此T(n) = Θ(n log n) ∎
```

**定理 3** (比较排序下界): 任何基于比较的排序算法在最坏情况下需要$\Omega(n \log n)$次比较。

**证明概要**:

```
比较排序的决策树：
- 每个内部节点代表一次比较
- 每个叶节点代表一种排列
- n个元素有n!种排列，因此至少n!个叶节点

决策树高度h ≥ log₂(n!)

使用斯特林公式：
n! ≥ (n/e)^n

因此：
log₂(n!) ≥ log₂((n/e)^n)
         = n(log₂n - log₂e)
         = Ω(n log n) ∎
```

## 四、算法/方法详解

### 4.1 算法描述

**自顶向下归并排序**:

```
算法: MergeSort(A, p, r)
输入: 数组A，索引p到r
输出: A[p..r]有序

1. if p < r then
2.     q = ⌊(p + r) / 2⌋
3.     MergeSort(A, p, q)
4.     MergeSort(A, q+1, r)
5.     Merge(A, p, q, r)
6. end if
```

**合并操作**:

```
算法: Merge(A, p, q, r)
输入: A[p..q]和A[q+1..r]已有序
输出: A[p..r]有序

1. n₁ = q - p + 1
2. n₂ = r - q
3. 创建数组L[0..n₁]和R[0..n₂]
4. for i = 0 to n₁-1 do
5.     L[i] = A[p + i]
6. end for
7. for j = 0 to n₂-1 do
8.     R[j] = A[q + 1 + j]
9. end for
10. L[n₁] = ∞  // 哨兵
11. R[n₂] = ∞  // 哨兵
12. i = 0, j = 0
13. for k = p to r do
14.     if L[i] ≤ R[j] then
15.         A[k] = L[i]
16.         i = i + 1
17.     else
18.         A[k] = R[j]
19.         j = j + 1
20.     end if
21. end for
```

**自底向上归并排序**:

```
算法: MergeSort-BottomUp(A, n)
输入: 数组A，长度n

1. for width = 1, 2, 4, 8, ... < n do
2.     for i = 0 to n-1 by 2*width do
3.         left = i
4.         mid = min(i + width - 1, n - 1)
5.         right = min(i + 2*width - 1, n - 1)
6.         Merge(A, left, mid, right)
7.     end for
8. end for
```

### 4.2 正确性分析

**合并操作正确性**:
使用循环不变式证明：

在每次for循环迭代开始前（第13行）：

1. A[p..k-1]包含L[0..n₁-1]和R[0..n₂-1]中的k-p个最小元素（有序）
2. L[i]和R[j]分别是各自数组中未被复制到A的最小元素

**证明**:

- **初始化**: k=p，A[p..p-1]为空，显然包含0个最小元素
- **保持**: 每次选择L[i]和R[j]中较小者放入A[k]，保证A[p..k]有序且包含k-p+1个最小元素
- **终止**: k=r+1，A[p..r]包含所有n₁+n₂=r-p+1个元素，且有序

### 4.3 复杂度分析

**时间复杂度**:

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| 最好 | $O(n \log n)$ | 无论输入如何都需相同工作量 |
| 平均 | $O(n \log n)$ | 输入分布不影响 |
| 最坏 | $O(n \log n)$ | 保证性能 |

**空间复杂度**:

- 额外空间: $O(n)$（临时数组L和R）
- 递归栈: $O(\log n)$
- 总计: $O(n)$

**精确比较次数**:
最优实现最多进行$n\lceil\log_2 n\rceil - 2^{\lceil\log_2 n\rceil} + 1$次比较

```python
"""
归并排序完整实现
包含递归版、迭代版、逆序对计数
"""

from typing import List, Tuple
import math


def merge_sort(arr: List[int]) -> List[int]:
    """标准归并排序（返回新数组）"""
    if len(arr) <= 1:
        return arr[:]

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return _merge(left, right)


def _merge(left: List[int], right: List[int]) -> List[int]:
    """合并两个有序数组"""
    result = []
    i = j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:  # 使用<=保证稳定性
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    # 添加剩余元素
    result.extend(left[i:])
    result.extend(right[j:])

    return result


def merge_sort_inplace(arr: List[int]) -> None:
    """原地归并排序（使用辅助数组）"""
    if len(arr) <= 1:
        return

    # 预先分配辅助数组
    temp = [0] * len(arr)
    _merge_sort_helper(arr, temp, 0, len(arr) - 1)


def _merge_sort_helper(arr: List[int], temp: List[int], left: int, right: int) -> None:
    """辅助递归函数"""
    if left >= right:
        return

    mid = (left + right) // 2
    _merge_sort_helper(arr, temp, left, mid)
    _merge_sort_helper(arr, temp, mid + 1, right)
    _merge_inplace(arr, temp, left, mid, right)


def _merge_inplace(arr: List[int], temp: List[int], left: int, mid: int, right: int) -> None:
    """原地合并"""
    # 复制到临时数组
    for i in range(left, right + 1):
        temp[i] = arr[i]

    i, j = left, mid + 1
    k = left

    while i <= mid and j <= right:
        if temp[i] <= temp[j]:
            arr[k] = temp[i]
            i += 1
        else:
            arr[k] = temp[j]
            j += 1
        k += 1

    # 复制剩余元素（只有左边可能有剩余）
    while i <= mid:
        arr[k] = temp[i]
        i += 1
        k += 1


def merge_sort_bottom_up(arr: List[int]) -> None:
    """自底向上归并排序（迭代实现）"""
    n = len(arr)
    if n <= 1:
        return

    temp = [0] * n

    # width: 当前子数组大小（1, 2, 4, 8, ...）
    width = 1
    while width < n:
        for i in range(0, n, 2 * width):
            left = i
            mid = min(i + width - 1, n - 1)
            right = min(i + 2 * width - 1, n - 1)

            if mid < right:  # 确保有两个子数组可合并
                _merge_inplace(arr, temp, left, mid, right)

        width *= 2


def count_inversions(arr: List[int]) -> Tuple[int, List[int]]:
    """
    使用归并排序计数逆序对
    返回: (逆序对数量, 排序后的数组)
    """
    if len(arr) <= 1:
        return 0, arr[:]

    mid = len(arr) // 2
    left_count, left = count_inversions(arr[:mid])
    right_count, right = count_inversions(arr[mid:])
    merge_count, merged = _merge_count(left, right)

    return left_count + right_count + merge_count, merged


def _merge_count(left: List[int], right: List[int]) -> Tuple[int, List[int]]:
    """合并并计数逆序对"""
    result = []
    inversions = 0
    i = j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            # left[i] > right[j]，left[i..]都大于right[j]
            inversions += len(left) - i
            j += 1

    result.extend(left[i:])
    result.extend(right[j:])

    return inversions, result


def natural_merge_sort(arr: List[int]) -> None:
    """自然归并排序（利用已有有序段）"""
    n = len(arr)
    if n <= 1:
        return

    temp = [0] * n

    while True:
        # 识别所有有序段(runs)
        runs = []
        i = 0
        while i < n:
            start = i
            i += 1
            while i < n and arr[i-1] <= arr[i]:
                i += 1
            runs.append((start, i - 1))

        if len(runs) <= 1:
            break  # 已完全有序

        # 合并相邻runs
        new_runs = []
        for j in range(0, len(runs) - 1, 2):
            left_start, left_end = runs[j]
            right_start, right_end = runs[j + 1]
            _merge_inplace(arr, temp, left_start, left_end, right_end)
            new_runs.append((left_start, right_end))

        # 处理奇数个runs
        if len(runs) % 2 == 1:
            new_runs.append(runs[-1])


# 链表归并排序
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


def sort_list(head: ListNode) -> ListNode:
    """链表归并排序"""
    if not head or not head.next:
        return head

    # 使用快慢指针找中点
    slow, fast = head, head.next
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    mid = slow.next
    slow.next = None

    left = sort_list(head)
    right = sort_list(mid)

    return _merge_lists(left, right)


def _merge_lists(l1: ListNode, l2: ListNode) -> ListNode:
    """合并两个有序链表"""
    dummy = ListNode()
    tail = dummy

    while l1 and l2:
        if l1.val <= l2.val:
            tail.next = l1
            l1 = l1.next
        else:
            tail.next = l2
            l2 = l2.next
        tail = tail.next

    tail.next = l1 if l1 else l2
    return dummy.next
```

```rust
/// 归并排序Rust实现
pub fn merge_sort<T: Ord + Clone>(arr: &[T]) -> Vec<T> {
    let n = arr.len();
    if n <= 1 {
        return arr.to_vec();
    }

    let mid = n / 2;
    let left = merge_sort(&arr[..mid]);
    let right = merge_sort(&arr[mid..]);

    merge(&left, &right)
}

fn merge<T: Ord + Clone>(left: &[T], right: &[T]) -> Vec<T> {
    let mut result = Vec::with_capacity(left.len() + right.len());
    let (mut i, mut j) = (0, 0);

    while i < left.len() && j < right.len() {
        if left[i] <= right[j] {
            result.push(left[i].clone());
            i += 1;
        } else {
            result.push(right[j].clone());
            j += 1;
        }
    }

    result.extend_from_slice(&left[i..]);
    result.extend_from_slice(&right[j..]);
    result
}

/// 原地归并排序
pub fn merge_sort_inplace<T: Ord + Clone>(arr: &mut [T]) {
    let n = arr.len();
    if n <= 1 {
        return;
    }

    let mut temp = arr.to_vec();
    merge_sort_helper(arr, &mut temp, 0, n - 1);
}

fn merge_sort_helper<T: Ord + Clone>(arr: &mut [T], temp: &mut [T], left: usize, right: usize) {
    if left >= right {
        return;
    }

    let mid = left + (right - left) / 2;
    merge_sort_helper(arr, temp, left, mid);
    merge_sort_helper(arr, temp, mid + 1, right);
    merge_inplace(arr, temp, left, mid, right);
}

fn merge_inplace<T: Clone>(arr: &mut [T], temp: &mut [T], left: usize, mid: usize, right: usize)
where
    T: Ord,
{
    // 复制到临时数组
    for i in left..=right {
        temp[i] = arr[i].clone();
    }

    let (mut i, mut j, mut k) = (left, mid + 1, left);

    while i <= mid && j <= right {
        if temp[i] <= temp[j] {
            arr[k] = temp[i].clone();
            i += 1;
        } else {
            arr[k] = temp[j].clone();
            j += 1;
        }
        k += 1;
    }

    while i <= mid {
        arr[k] = temp[i].clone();
        i += 1;
        k += 1;
    }
}

/// 计数逆序对
pub fn count_inversions<T: Ord + Clone>(arr: &[T]) -> (usize, Vec<T>) {
    let n = arr.len();
    if n <= 1 {
        return (0, arr.to_vec());
    }

    let mid = n / 2;
    let (left_count, left) = count_inversions(&arr[..mid]);
    let (right_count, right) = count_inversions(&arr[mid..]);
    let (merge_count, merged) = merge_count(&left, &right);

    (left_count + right_count + merge_count, merged)
}

fn merge_count<T: Ord + Clone>(left: &[T], right: &[T]) -> (usize, Vec<T>) {
    let mut result = Vec::with_capacity(left.len() + right.len());
    let (mut i, mut j) = (0, 0);
    let mut inversions = 0;

    while i < left.len() && j < right.len() {
        if left[i] <= right[j] {
            result.push(left[i].clone());
            i += 1;
        } else {
            result.push(right[j].clone());
            inversions += left.len() - i;
            j += 1;
        }
    }

    result.extend_from_slice(&left[i..]);
    result.extend_from_slice(&right[j..]);
    (inversions, result)
}
```

### 5.3 反例

**常见错误1**: 忽略稳定性条件

```python
# 错误代码（破坏稳定性）
def bad_merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] < right[j]:  # 应该用 <=
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    # ...
```

**错误原因**: 使用`<`而非`<=`导致相等元素顺序颠倒
**正确做法**: 使用`<=`保持稳定性

**常见错误2**: 边界条件处理错误

```python
# 错误代码
def bad_merge_sort(arr):
    mid = len(arr) // 2
    left = bad_merge_sort(arr[:mid])
    right = bad_merge_sort(arr[mid:])  # 当len=1时，mid=0，right=arr[0:]=arr
    # 导致无限递归！
```

**错误原因**: 缺少基本情况处理
**正确做法**: 添加`if len(arr) <= 1: return arr`

## 六、习题

### 6.1 理解题

1. **归并排序递归树** [难度⭐]

   画出对数组[3, 1, 4, 1, 5, 9, 2]进行归并排序的完整递归树。

   <details>
   <summary>解答</summary>

   ```
   [3, 1, 4, 1, 5, 9, 2]
       ├── [3, 1, 4]
       │       ├── [3]
       │       └── [1, 4]
       │               ├── [1]
       │               └── [4]
       │       └── 合并: [1, 3, 4]
       └── [1, 5, 9, 2]
               ├── [1, 5]
               │       ├── [1]
               │       └── [5]
               │       └── 合并: [1, 5]
               └── [9, 2]
                       ├── [9]
                       └── [2]
                       └── 合并: [2, 9]
               └── 合并: [1, 2, 5, 9]
       └── 合并: [1, 1, 2, 3, 4, 5, 9]
   ```

   </details>

### 6.2 应用题

1. **计算逆序对** [难度⭐⭐]

   给定数组[2, 4, 1, 3, 5]，使用归并排序计算逆序对数量。

   <details>
   <summary>解答</summary>

   逆序对：(2,1), (4,1), (4,3)，共3个。

   归并排序过程：

   ```
   [2, 4, 1, 3, 5]
       ├── [2, 4] → [2] + [4]，无逆序对
       └── [1, 3, 5]
               ├── [1] + [3, 5]，无逆序对
               └── 合并[1,3,5]和[2,4]：
                   1 < 2，取1
                   2 > 3? 否，取2
                   4 > 3，取3，逆序对+2（4与3，以及前面未取的4）
                   4 < 5，取4
                   取5
       总共3个逆序对
   ```

   </details>

### 6.3 证明题

1. **比较次数下界** [难度⭐⭐⭐]

   证明归并排序的比较次数下界为$\frac{1}{2}n\log_2 n$。

   <details>
   <summary>解答</summary>

   **证明**:

   ```
   合并两个长度分别为m和n的有序数组：
   - 最少比较次数: min(m, n)（一个数组全部小于另一个）
   - 最多比较次数: m + n - 1

   对于n = 2^k的归并排序：
   - 第i层有2^i个子问题，每个大小为n/2^i
   - 合并两个大小为n/2^{i+1}的数组至少需要n/2^{i+1}次比较
   - 第i层总比较次数: 2^i × n/2^{i+1} = n/2

   共log₂n层，总比较次数 ≥ n/2 × log₂n = (n log₂n)/2

   因此下界为Ω(n log n)。∎
   ```

   </details>

## 七、应用场景

### 7.1 经典应用

| 应用场景 | 具体问题 | 使用归并排序的原因 |
|----------|----------|-------------------|
| 标准库排序 | Python sorted(), Java Arrays.sort() | 稳定排序，保证O(n log n) |
| 链表排序 | 大规模链表数据 | 不需要随机访问 |
| 外部排序 | 数据库大数据排序 | 顺序访问模式，适合磁盘I/O |
| 逆序对计数 | 相似度计算、序列分析 | 归并过程天然可计数 |

### 7.2 实际案例

**案例**: Linux内核的`sort`命令

**背景**:
`sort`命令需要处理大规模文本文件，可能超过内存容量。

**应用方式**:

- 内存中使用归并排序对块排序
- 使用k路归并合并已排序的临时文件
- 稳定排序保持原始顺序

**效果**:

- 能处理比内存大100倍的数据
- 利用归并排序的顺序访问特性优化磁盘I/O

### 7.3 跨领域联系

**与数据库的联系**:
数据库系统广泛使用外部归并排序处理大规模数据。归并排序的$O(n \log n)$保证和顺序访问特性使其成为数据库排序的首选。

**与计算几何的联系**:
计算几何中的扫描线算法常使用归并排序树处理范围查询问题。

## 八、多维对比

### 8.1 排序算法对比

| 维度 | 归并排序 | 快速排序 | 堆排序 | 插入排序 |
|------|----------|----------|--------|----------|
| 最好时间 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ |
| 平均时间 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ |
| 最坏时间 | $O(n \log n)$ | $O(n^2)$ | $O(n \log n)$ | $O(n^2)$ |
| 空间复杂度 | $O(n)$ | $O(\log n)$ | $O(1)$ | $O(1)$ |
| 稳定性 | 稳定 | 不稳定 | 不稳定 | 稳定 |
| 适应性 | 不具备 | 不具备 | 不具备 | 具备 |
| 链表友好 | 是 | 否 | 否 | 是 |

### 8.2 决策建议

**何时选择归并排序**:

- 需要稳定排序
- 处理链表数据
- 需要$O(n \log n)$最坏保证
- 外部排序场景

**决策流程图**:

```
选择排序算法
├── 需要稳定排序?
│   ├── 是 → 归并排序或插入排序
│   │           ├── 大规模数据 → 归并排序
│   │           └── 小规模数据 → 插入排序
│   └── 否 → 考虑其他因素
├── 处理链表?
│   └── 是 → 归并排序
└── 默认 → 根据其他约束选择
```

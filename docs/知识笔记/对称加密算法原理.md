## 一、核心概念

### 1.1 定义

**正式定义**:
对称加密方案（Symmetric-key Encryption Scheme）是一个三元组 $\Pi = (\text{Gen}, \text{Enc}, \text{Dec})$，其中：

1. **密钥生成算法** $\text{Gen}$：概率算法，输入安全参数 $1^n$，输出密钥 $k \in \{0,1\}^n$
2. **加密算法** $\text{Enc}$：确定性或概率算法，输入密钥 $k$ 和明文 $m \in \{0,1\}^*$，输出密文 $c \leftarrow \text{Enc}_k(m)$
3. **解密算法** $\text{Dec}$：确定性算法，输入密钥 $k$ 和密文 $c$，输出明文 $m := \text{Dec}_k(c)$，满足 $\forall k, m: \text{Dec}_k(\text{Enc}_k(m)) = m$

**直观解释**:
对称加密就像一把锁和一把钥匙：发送方和接收方共享同一把钥匙（密钥），发送方用钥匙锁上（加密）消息，接收方用同一把钥匙开锁（解密）。核心挑战是如何在共享密钥的前提下实现语义安全。

**关键要点**:

- **对称性**：加密和解密使用相同密钥
- **机密性保证**：密文不应泄露明文的任何信息
- **效率优势**：比非对称加密快100-1000倍
- **密钥分发问题**：需要安全信道预先共享密钥

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 对称性 | 加解密使用相同密钥 | 与公钥加密的根本区别 |
| 可逆性 | 解密算法总能恢复原始明文 | 正确性要求 |
| 效率性 | 处理速度快，适合大数据量 | AES可达数GB/s |
| 确定性/概率性 | 现代加密通常是概率性的 | 防止选择明文攻击 |

**性质总结**:

1. **正确性**: $\forall k \leftarrow \text{Gen}(1^n), \forall m: \text{Dec}_k(\text{Enc}_k(m)) = m$
2. **高效性**: 加密/解密时间关于消息长度和密钥长度是多项式的
3. **语义安全性**: 密文不泄露明文的任何部分信息

### 1.3 变体

**流密码（Stream Cipher）**:

- 定义: 逐位或逐字节加密，使用密钥流与明文进行异或运算
- 与分组密码的区别: 不需要填充，适合实时数据流
- 适用场景: 实时通信、资源受限设备

**分组密码（Block Cipher）**:

- 定义: 固定长度分组（如128位）的加密
- 与流密码的区别: 需要工作模式处理变长消息
- 适用场景: 文件加密、数据库加密

## 三、形式化证明

### 3.1 核心定理

**定理 1 (完美保密性)**: One-time Pad（一次性密码本）具有完美保密性，即密文不泄露任何关于明文的信息。

**证明**:

设密钥 $k \in \{0,1\}^n$ 均匀随机选择，加密算法为 $c = m \oplus k$。

需证：对于所有分布的明文 $m$ 和所有密文 $c$，有 $\Pr[M = m | C = c] = \Pr[M = m]$。

由贝叶斯定理：
$$\Pr[M=m|C=c] = \frac{\Pr[C=c|M=m] \cdot \Pr[M=m]}{\Pr[C=c]}$$

对于固定的 $m$，$c = m \oplus k$，所以 $k = m \oplus c$。由于 $k$ 均匀随机：
$$\Pr[C=c|M=m] = \Pr[K = m \oplus c] = 2^{-n}$$

对于分母，由全概率公式：
$$\Pr[C=c] = \sum_{m'} \Pr[C=c|M=m'] \cdot \Pr[M=m'] = 2^{-n} \sum_{m'} \Pr[M=m'] = 2^{-n}$$

因此：
$$\Pr[M=m|C=c] = \frac{2^{-n} \cdot \Pr[M=m]}{2^{-n}} = \Pr[M=m]$$

∎

**证明要点分析**:

1. 关键利用密钥的均匀分布性质
2. 每个密文对应唯一可能的明文（给定密钥）
3. 完美保密的代价：密钥长度不小于明文长度

**定理 2 (PRP-PRF转换)**: 设 $F: \{0,1\}^n \times \{0,1\}^n \to \{0,1\}^n$ 是一个安全的伪随机函数（PRF），则构造的 $F'(k, x) = F(k, x) \| F(k, x+1)$ 是一个安全的伪随机置换（PRP）（Feistel构造）。

*证明概要*: 使用混合论证（hybrid argument），证明如果敌手能区分 $F'$ 与随机置换，则能构造对 $F$ 的区分器。

### 3.2 辅助引理

**引理 1 (Feistel网络的逆)**: 对于r轮Feistel网络，给定轮函数 $f_1, \ldots, f_r$，其逆运算只需将轮函数顺序反转：$f_r, \ldots, f_1$。

*证明*: 设第 $i$ 轮输入为 $(L_i, R_i)$，输出为 $(L_{i+1}, R_{i+1})$，其中 $L_{i+1} = R_i$，$R_{i+1} = L_i \oplus f_i(R_i)$。则逆向计算：$R_i = L_{i+1}$，$L_i = R_{i+1} \oplus f_i(L_{i+1})$。∎

## 四、算法/方法详解

### 4.1 算法描述

**AES-128加密算法 (SPN结构)**:

```
算法: AES-128-Encrypt
输入: 128位明文块 P, 128位密钥 K
输出: 128位密文块 C

1. 扩展密钥: w[0..43] ← KeyExpansion(K)
2. 状态矩阵 State ← P
3. State ← State ⊕ w[0..3]  // 初始轮密钥加
4. for i = 1 to 9 do
5.     State ← SubBytes(State)      // S盒代换
6.     State ← ShiftRows(State)     // 行移位
7.     State ← MixColumns(State)    // 列混合
8.     State ← State ⊕ w[4i..4i+3]  // 轮密钥加
9. end for
10. State ← SubBytes(State)
11. State ← ShiftRows(State)
12. State ← State ⊕ w[40..43]  // 最后一轮（无MixColumns）
13. return State as C
```

**流程图**:

```
明文P → AddRoundKey → 轮1-9 → 最终轮 → 密文C
              ↓           ↓
         KeyExpansion   SubBytes → ShiftRows → MixColumns → AddRoundKey
```

### 4.2 正确性分析

**AES的正确性依赖于**：

- **SubBytes**: 提供非线性，抵抗差分/线性分析
- **ShiftRows + MixColumns**: 提供扩散，确保单个比特影响多个输出比特
- **轮密钥加**: 引入密钥材料

**不变式**: 每一轮都保持状态矩阵的可逆变换性质。

### 4.3 复杂度分析

**时间复杂度**:

- AES-128: $O(1)$ 每128位块（固定10轮）
- AES-192: $O(1)$ 每128位块（12轮）
- AES-256: $O(1)$ 每128位块（14轮）

实际吞吐量：

- 软件实现（AES-NI指令集）: 约 3-5 GB/s
- 硬件实现: 可达 100+ Gbps

**空间复杂度**: $O(1)$（固定大小的状态矩阵和密钥表）

### 5.2 代码实现

**Python实现 (AES-128简化版)**:

```python
import struct
from typing import List, Tuple

# S-box (标准AES S-box)
S_BOX = [
    0x63, 0x7c, 0x77, 0x7b, 0xf2, 0x6b, 0x6f, 0xc5, 0x30, 0x01, 0x67, 0x2b, 0xfe, 0xd7, 0xab, 0x76,
    # ... (完整256字节)
]

# 逆S-box
INV_S_BOX = [S_BOX.index(i) for i in range(256)]

# 轮常数
RCON = [0x01, 0x02, 0x04, 0x08, 0x10, 0x20, 0x40, 0x80, 0x1b, 0x36]

def gf_mul(a: int, b: int) -> int:
    """GF(2^8)上的乘法"""
    p = 0
    for _ in range(8):
        if b & 1:
            p ^= a
        carry = a & 0x80
        a = (a << 1) & 0xff
        if carry:
            a ^= 0x1b  # AES的不可约多项式
        b >>= 1
    return p

def sub_bytes(state: List[List[int]]) -> None:
    """字节代换"""
    for i in range(4):
        for j in range(4):
            state[i][j] = S_BOX[state[i][j]]

def shift_rows(state: List[List[int]]) -> None:
    """行移位"""
    state[1] = state[1][1:] + state[1][:1]
    state[2] = state[2][2:] + state[2][:2]
    state[3] = state[3][3:] + state[3][:3]

def mix_columns(state: List[List[int]]) -> None:
    """列混合"""
    for j in range(4):
        a = [state[i][j] for i in range(4)]
        state[0][j] = gf_mul(0x02, a[0]) ^ gf_mul(0x03, a[1]) ^ a[2] ^ a[3]
        state[1][j] = a[0] ^ gf_mul(0x02, a[1]) ^ gf_mul(0x03, a[2]) ^ a[3]
        state[2][j] = a[0] ^ a[1] ^ gf_mul(0x02, a[2]) ^ gf_mul(0x03, a[3])
        state[3][j] = gf_mul(0x03, a[0]) ^ a[1] ^ a[2] ^ gf_mul(0x02, a[3])

def add_round_key(state: List[List[int]], round_key: List[List[int]]) -> None:
    """轮密钥加"""
    for i in range(4):
        for j in range(4):
            state[i][j] ^= round_key[j][i]  # 注意转置

def key_expansion(key: bytes) -> List[List[List[int]]]:
    """密钥扩展"""
    # 简化的密钥扩展实现
    w = [list(key[i:i+4]) for i in range(0, 16, 4)]
    round_keys = [w.copy()]

    for i in range(10):
        # 核心调度
        temp = w[3].copy()
        temp = temp[1:] + temp[:1]  # RotWord
        temp = [S_BOX[b] for b in temp]  # SubWord
        temp[0] ^= RCON[i]

        w[0] = [w[0][j] ^ temp[j] for j in range(4)]
        for k in range(1, 4):
            w[k] = [w[k][j] ^ w[k-1][j] for j in range(4)]

        round_keys.append([row.copy() for row in w])

    return round_keys

def aes_encrypt(plaintext: bytes, key: bytes) -> bytes:
    """AES-128加密"""
    assert len(plaintext) == 16 and len(key) == 16

    # 初始化状态矩阵（列优先）
    state = [[plaintext[i + 4*j] for j in range(4)] for i in range(4)]
    round_keys = key_expansion(key)

    # 初始轮密钥加
    add_round_key(state, round_keys[0])

    # 主轮循环
    for i in range(1, 10):
        sub_bytes(state)
        shift_rows(state)
        mix_columns(state)
        add_round_key(state, round_keys[i])

    # 最终轮（无MixColumns）
    sub_bytes(state)
    shift_rows(state)
    add_round_key(state, round_keys[10])

    # 转回字节序列
    return bytes(state[i][j] for i in range(4) for j in range(4))

# 使用示例
if __name__ == "__main__":
    key = bytes([0x2b, 0x7e, 0x15, 0x16, 0x28, 0xae, 0xd2, 0xa6,
                 0xab, 0xf7, 0x15, 0x88, 0x09, 0xcf, 0x4f, 0x3c])
    plaintext = bytes([0x32, 0x43, 0xf6, 0xa8, 0x88, 0x5a, 0x30, 0x8d,
                       0x31, 0x31, 0x98, 0xa2, 0xe0, 0x37, 0x07, 0x34])

    ciphertext = aes_encrypt(plaintext, key)
    print(f"密文: {ciphertext.hex()}")
    # 标准输出: 3925841d02dc09fbdc118597196a0b32
```

### 5.3 反例

**常见错误**:

**错误1: ECB模式重用**

```python
# 错误示例：使用ECB模式加密图像
from Crypto.Cipher import AES

cipher = AES.new(key, AES.MODE_ECB)  # 危险！
encrypted = cipher.encrypt(pad(image_data, AES.block_size))
# 问题：相同明文块产生相同密文块，图像模式可见
```

**错误原因**: ECB模式不隐藏数据模式
**正确做法**: 使用CBC或CTR模式，添加随机IV

**错误2: 密钥复用（One-time Pad）**

```python
# 错误：两次使用相同密钥
key = os.urandom(len(message1))
c1 = bytes([m ^ k for m, k in zip(message1, key)])
c2 = bytes([m ^ k for m, k in zip(message2, key)])
# 攻击者计算 c1 xor c2 = message1 xor message2
```

**错误原因**: 破坏了完美保密性
**正确做法**: 每个消息使用独立随机密钥

## 六、习题

### 6.1 理解题

1. **证明**：如果分组密码 $E$ 满足 $E_k(x) = E_k^{-1}(x)$（对合性），则该密码不安全。

   <details>
   <summary>解答</summary>

   对合性意味着 $E_k(E_k(x)) = x$。攻击者可以：
   1. 选择明文 $x$，获得密文 $c = E_k(x)$
   2. 再次加密 $c$，获得 $E_k(c) = E_k(E_k(x)) = x$
   3. 这提供了验证机制，且破坏了随机置换的性质

   标准分组密码（如AES）都不是对合的。
   </details>

### 6.2 应用题

1. **设计**: 使用AES-256-GCM实现一个安全的文件加密工具，要求：
   - 每个文件使用随机生成的密钥
   - 密钥使用PBKDF2从密码派生
   - 使用认证加密防止篡改

   <details>
   <summary>解答</summary>

   ```python
   from cryptography.hazmat.primitives.ciphers.aead import AESGCM
   from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
   import os

   def encrypt_file(password: bytes, plaintext: bytes) -> bytes:
       salt = os.urandom(16)
       nonce = os.urandom(12)

       kdf = PBKDF2HMAC(
           algorithm=hashes.SHA256(),
           length=32,
           salt=salt,
           iterations=600000,
       )
       key = kdf.derive(password)

       aesgcm = AESGCM(key)
       ciphertext = aesgcm.encrypt(nonce, plaintext, None)

       return salt + nonce + ciphertext
   ```

### 6.3 证明题

1. **证明**: 如果存在高效的算法可以区分AES的输出与随机置换，则可以构造对AES的选择明文攻击。

   <details>
   <summary>解答</summary>

   设 $D$ 是区分器，$Adv_D^{PRP}(A) = \epsilon$。

   攻击：
   1. 敌手选择查询 $x_1, \ldots, x_q$
   2. 获得应答 $y_1, \ldots, y_q$
   3. 运行 $D$ 在 $\{(x_i, y_i)\}$
   4. 若 $D$ 输出1，猜测是PRP；否则猜测是随机置换

   若 $\epsilon$ 不可忽略，则敌手优势显著，与PRP安全性假设矛盾。
   </details>

## 八、多维对比

### 8.1 方法对比

| 维度 | AES-256-GCM | ChaCha20-Poly1305 | AES-256-CBC+HMAC |
|------|-------------|-------------------|------------------|
| 加密速度 | 极快（硬件加速） | 快（软件优化） | 中等 |
| 认证开销 | 极低（内置） | 低（内置） | 需要额外计算 |
| 并行性 | 完全并行 | 完全并行 | 串行（CBC） |
| 侧信道抵抗 | 需要防护 | 天然抵抗 | 需要防护 |
| 标准支持 | TLS 1.2+ | TLS 1.3 | 传统系统 |

### 8.2 决策建议

**何时选择AES-GCM**:

- 有AES-NI硬件支持
- 高吞吐量要求
- 需要同时加密和认证

**何时选择ChaCha20-Poly1305**:

- 软件实现环境
- 对侧信道攻击敏感
- 移动/嵌入式设备

## 十、专家批注

> **Claude Shannon**:
> 混淆（Confusion）和扩散（Diffusion）是设计强密码的两个基本技术。混淆使密文和密钥之间的关系复杂化，扩散将明文的统计结构散布到整个密文中。

> **Joan Daemen (AES设计者)**:
> Rijndael的设计哲学是"宽轨迹策略"（Wide Trail Strategy）：通过线性层（MixColumns和ShiftRows）和S-box的组合，确保差分和线性特征的概率最大化。

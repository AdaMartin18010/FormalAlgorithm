## 一、核心概念

### 1.1 定义

**正式定义**:
随机算法 $\mathcal{M}$ 满足 $(\epsilon, \delta)$-差分隐私，如果对于所有相邻数据集 $D, D'$（相差一条记录）和所有输出集合 $S \subseteq \text{Range}(\mathcal{M})$：
$$P(\mathcal{M}(D) \in S) \leq e^\epsilon \cdot P(\mathcal{M}(D') \in S) + \delta$$

- 当 $\delta = 0$ 时，称为**纯差分隐私**（$\epsilon$-DP）
- 当 $\delta > 0$ 时，称为**近似差分隐私**（$(\epsilon, \delta)$-DP）

**相邻数据集**: 记 $D \sim D'$，表示 $D$ 和 $D'$ 在一条记录上不同（添加、删除或修改）。

**直观解释**:
差分隐私保证：任何单个个体参与数据集与否，对算法输出的影响微乎其微。攻击者无法从输出中确定某个特定个体是否在数据集中。

想象"合理否认"（plausible deniability）：无论你选择参与还是不参与，看到的输出都差不多，因此你有合理的否认权。

**隐私参数**:
| 参数 | 含义 | 典型值 |
|------|------|--------|
| $\epsilon$ | 隐私损失预算 | 0.1 到 10（越小越隐私）|
| $\delta$ | 失败概率 | $< 1/n^2$（$n$为数据集大小）|

**关键要点**:
- 差分隐私是**后处理免疫**的：对DP输出进行任何计算仍保持DP
- 差分隐私满足**组合性质**：多次查询隐私损失累加
- 隐私保护强度与数据量无关：适用于任意大小数据集

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 组合性 | 多次查询隐私损失可加 | 高级组合定理优化累加 |
| 后处理免疫 | 任意处理不改变隐私性 | 攻击者无法"破解"DP |
| 群隐私 | $k$条记录的隐私损失为 $k\epsilon$ | 保护小群体 |
| 鲁棒性 | 不依赖攻击者背景知识 | 最强隐私定义 |

**性质总结**:
1. **量化隐私损失**: $\epsilon$ 给出明确的隐私损失上界
2. **与数据无关**: 保护承诺不依赖数据分布
3. **可组合分析**: 复杂系统的隐私可模块化分析

### 1.3 变体

**Renyi差分隐私 (RDP)**:
- 定义: 使用Renyi散度度量隐私损失
- 与标准形式的区别: 组合性质更紧，便于分析迭代算法
- 适用场景: SGD、深度学习的隐私分析

**集中差分隐私 (zCDP)**:
- 定义: 高斯机制的变体
- 与标准形式的区别: 分析高斯噪声更方便
- 适用场景: 高维数据分析

**本地差分隐私 (LDP)**:
- 定义: 数据在收集前已扰动
- 与标准形式的区别: 更强的信任模型（不信任数据收集者）
- 适用场景: 苹果、谷歌的用户数据收集

**Pufferfish框架**:
- 定义: 针对特定语义隐私需求
- 与标准形式的区别: 更灵活但需额外设计
- 适用场景: 特殊数据类型的隐私保护

## 三、形式化证明

### 3.1 核心定理

**定理 1** (拉普拉斯机制的隐私保证):
设查询 $f: \mathcal{D} \to \mathbb{R}^k$ 的L1敏感度为 $\Delta_1(f) = \max_{D \sim D'} \|f(D) - f(D')\|_1$，则机制：
$$\mathcal{M}(D) = f(D) + (\eta_1, ..., \eta_k), \quad \eta_i \stackrel{iid}{\sim} \text{Lap}(\Delta_1(f)/\epsilon)$$
满足 $\epsilon$-差分隐私。

**证明**:

设 $D \sim D'$ 为相邻数据集，$p_D(x)$ 为 $\mathcal{M}(D)$ 的密度函数。

拉普拉斯分布 $\text{Lap}(b)$ 的密度为 $\frac{1}{2b}\exp(-|x|/b)$。

对任意输出 $x \in \mathbb{R}^k$：
$$\frac{p_D(x)}{p_{D'}(x)} = \prod_{i=1}^k \frac{\exp(-|f(D)_i - x_i|/b)}{\exp(-|f(D')_i - x_i|/b)}$$

$$= \exp\left(\frac{1}{b}\sum_{i=1}^k (|f(D')_i - x_i| - |f(D)_i - x_i|)\right)$$

由三角不等式：
$$|f(D')_i - x_i| - |f(D)_i - x_i| \leq |f(D')_i - f(D)_i|$$

因此：
$$\frac{p_D(x)}{p_{D'}(x)} \leq \exp\left(\frac{1}{b}\|f(D) - f(D')\|_1\right) \leq \exp\left(\frac{\Delta_1(f)}{b}\right) = e^\epsilon$$

同理可证下界 $p_D(x)/p_{D'}(x) \geq e^{-\epsilon}$。∎

**证明要点分析**:
1. **敏感度决定噪声规模**: 敏感度越大，需要越多噪声
2. **L1敏感度**: 适合拉普拉斯机制（L1几何）
3. **隐私损失**: 通过噪声分布的比例有界性保证

### 3.2 辅助引理

**引理 1** (高级组合定理):
设 $\mathcal{M}_i$ 是 $(\epsilon_i, \delta_i)$-DP机制，则 $k$ 个机制的组合满足 $(\epsilon, \delta)$-DP，其中：
$$\epsilon = \sqrt{2\ln(1/\delta')\sum_i \epsilon_i^2} + \sum_i \epsilon_i \frac{e^{\epsilon_i} - 1}{e^{\epsilon_i} + 1}$$
对任意 $\delta' > 0$，总失败概率为 $\delta = \delta' + \sum_i \delta_i$。

*直观*: 高级组合定理显示隐私损失以次线性方式累积（类似于随机变量的集中现象）。

## 四、算法/方法详解

### 4.1 拉普拉斯机制

**伪代码**:
```
算法: LaplaceMechanism
输入: 数据集 D, 查询函数 f, 敏感度 Δ₁, 隐私参数 ε

1.  计算真实答案: a ← f(D)
2.  对每维 i:
3.      scale ← Δ₁ / ε
4.      noise[i] ← Laplace(scale)  // 从Lap(scale)采样
5.  return a + noise
```

**流程图**:
```
查询 f(D)
    ↓
计算敏感度 Δ
    ↓
生成噪声 η ~ Lap(Δ/ε)
    ↓
输出 f(D) + η
```

### 4.2 高斯机制

对于 $(\epsilon, \delta)$-DP，使用高斯噪声更高效：

**参数选择**:
$$\sigma = \frac{\Delta_2 \sqrt{2\ln(1.25/\delta)}}{\epsilon}$$

其中 $\Delta_2$ 是L2敏感度。

### 4.3 指数机制

用于从离散集合中选择最优项：

```
算法: ExponentialMechanism
输入: 数据集 D, 评分函数 u(D, r), 敏感度 Δu, 集合 R, 参数 ε

1.  对每个 r ∈ R:
2.      score[r] ← exp(ε · u(D, r) / (2 · Δu))
3.  从 R 按概率 ∝ score[r] 采样输出
```

### 4.4 复杂度分析

**计算复杂度**:
| 机制 | 额外计算 | 噪声生成 | 隐私-效用权衡 |
|------|----------|----------|---------------|
| 拉普拉斯 | O(k) | O(k) | MSE = O(k·Δ²/ε²) |
| 高斯 | O(k) | O(k) | MSE = O(k·Δ²·log(1/δ)/ε²) |
| 指数 | O(\|R\|) | O(\|R\|) | 效用损失 ∝ 1/ε |

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 私有计数查询

**问题描述**:
数据集D包含n个人的收入信息。回答"有多少人的收入超过10万？"

**敏感度分析**:
添加或删除一个人，计数变化最多为1。因此 $\Delta_1 = 1$。

**拉普拉斯机制**:
```
真实计数 = 1000
隐私参数 ε = 1
噪声 ~ Lap(1)  // scale = Δ/ε = 1
输出 = 1000 + Lap(1)
```

**隐私保证**: 无论某个特定人是否在数据集中，输出分布比最多为 $e^1 \approx 2.718$。

**示例 2**: 差分隐私SGD (DP-SGD)

**问题描述**:
用隐私保护方式训练神经网络。

**算法要点**:
```
对每个epoch:
    随机采样batch
    对每个样本计算梯度
    裁剪梯度范数至C
    平均梯度 + 高斯噪声 N(0, σ²C²I)
    更新参数
```

**隐私核算**: 使用RDP跟踪每步隐私损失，组合得到总隐私保证。

### 5.2 代码实现

**语言**: Python

```python
import numpy as np
from typing import Callable, List, Tuple
import random


class DifferentialPrivacy:
    """差分隐私工具类"""
    
    @staticmethod
    def laplace_mechanism(value: float, sensitivity: float, epsilon: float) -> float:
        """
        拉普拉斯机制
        
        Args:
            value: 真实值
            sensitivity: L1敏感度
            epsilon: 隐私参数
        
        Returns:
            隐私保护的估计值
        """
        scale = sensitivity / epsilon
        noise = np.random.laplace(0, scale)
        return value + noise
    
    @staticmethod
    def gaussian_mechanism(value: float, sensitivity: float, 
                          epsilon: float, delta: float) -> float:
        """
        高斯机制
        
        Args:
            value: 真实值
            sensitivity: L2敏感度
            epsilon: 隐私参数
            delta: 失败概率
        
        Returns:
            隐私保护的估计值
        """
        # 根据理论计算sigma
        sigma = sensitivity * np.sqrt(2 * np.log(1.25 / delta)) / epsilon
        noise = np.random.normal(0, sigma)
        return value + noise
    
    @staticmethod
    def exponential_mechanism(scores: List[float], sensitivity: float, 
                              epsilon: float) -> int:
        """
        指数机制
        
        Args:
            scores: 每个选项的得分列表
            sensitivity: 评分函数敏感度
            epsilon: 隐私参数
        
        Returns:
            选中的选项索引
        """
        # 计算权重
        weights = [np.exp(epsilon * s / (2 * sensitivity)) for s in scores]
        total = sum(weights)
        probs = [w / total for w in weights]
        
        # 按概率采样
        return np.random.choice(len(scores), p=probs)
    
    @staticmethod
    def report_noisy_max(candidates: List[float], sensitivity: float, 
                         epsilon: float) -> int:
        """
        报告带噪声的最大值（指数机制的特例）
        """
        noisy_scores = [c + np.random.laplace(0, sensitivity / epsilon) 
                       for c in candidates]
        return int(np.argmax(noisy_scores))


class PrivateHistogram:
    """差分隐私直方图"""
    
    def __init__(self, bins: int, epsilon: float):
        self.bins = bins
        self.epsilon = epsilon
        self.dp = DifferentialPrivacy()
    
    def compute(self, data: List[float], range_min: float, 
                range_max: float) -> np.ndarray:
        """
        计算隐私保护的直方图
        
        Args:
            data: 原始数据
            range_min, range_max: 数据范围
        
        Returns:
            隐私保护的频数数组
        """
        # 计算真实直方图
        counts, _ = np.histogram(data, bins=self.bins, 
                                  range=(range_min, range_max))
        
        # 敏感度 = 1（一个人只能影响一个bin）
        sensitivity = 1.0
        
        # 添加拉普拉斯噪声
        private_counts = np.array([
            self.dp.laplace_mechanism(c, sensitivity, self.epsilon)
            for c in counts
        ])
        
        # 确保非负
        return np.maximum(private_counts, 0)


class DPSGD:
    """差分隐私随机梯度下降"""
    
    def __init__(self, learning_rate: float, noise_multiplier: float,
                 max_grad_norm: float, batch_size: int):
        self.lr = learning_rate
        self.noise_multiplier = noise_multiplier
        self.max_grad_norm = max_grad_norm
        self.batch_size = batch_size
    
    def clip_gradient(self, grad: np.ndarray) -> np.ndarray:
        """裁剪梯度"""
        norm = np.linalg.norm(grad)
        if norm > self.max_grad_norm:
            return grad * (self.max_grad_norm / norm)
        return grad
    
    def private_gradient_update(self, gradients: List[np.ndarray]) -> np.ndarray:
        """
        计算隐私保护的平均梯度
        
        Args:
            gradients: batch中每个样本的梯度
        
        Returns:
            带噪声的平均梯度
        """
        # 裁剪每个梯度
        clipped = [self.clip_gradient(g) for g in gradients]
        
        # 平均
        avg_gradient = np.mean(clipped, axis=0)
        
        # 添加高斯噪声
        noise_std = self.noise_multiplier * self.max_grad_norm
        noise = np.random.normal(0, noise_std, avg_gradient.shape)
        
        return avg_gradient + noise
```

### 5.3 反例

**常见错误1**: 忽略敏感度计算
```python
# 错误：直接对平均值加噪声
def mean_query(data):
    return np.mean(data) + laplace_noise  # 错误！
```
**错误原因**: 平均值的敏感度是 (max-min)/n，不是常数
**正确做法**: 正确计算全局敏感度或使用局部敏感度

**常见错误2**: 组合隐私损失
```python
# 错误：多次查询不累积隐私损失
result1 = laplace_mechanism(f1(data), 1, 1)
result2 = laplace_mechanism(f2(data), 1, 1)
# 总隐私损失不是1，而是2（基本组合）
```
**错误原因**: 隐私损失需要累加
**正确做法**: 使用组合定理跟踪总隐私损失

## 六、习题

### 6.1 理解题 (L1-L2)

1. **隐私参数解释** [难度⭐]
   
   解释 $(\epsilon, \delta)$-差分隐私中两个参数的含义。$\epsilon = 0.1$ 和 $\epsilon = 10$ 哪个更隐私？
   
   <details>
   <summary>解答</summary>
   
   **参数含义**:
   - $\epsilon$（隐私损失）：控制输出概率比的对数上界，越小越隐私
   - $\delta$（失败概率）：允许小概率违反隐私保证
   
   **比较**: $\epsilon = 0.1$ 更隐私。它保证相邻数据集的输出分布比最多为 $e^{0.1} \approx 1.105$，非常接近；而 $\epsilon = 10$ 允许比例高达 $e^{10} \approx 22026$，隐私保护弱得多。
   
   </details>

2. **敏感度计算** [难度⭐⭐]
   
   数据集D包含n个人的年龄。计算以下查询的L1敏感度：
   (a) 计数查询：有多少人年龄超过30？
   (b) 求和查询：所有人的年龄总和
   (c) 平均值查询
   
   <details>
   <summary>解答</summary>
   
   **(a) 计数查询**:
   添加/删除一个人改变计数最多1。$\Delta_1 = 1$
   
   **(b) 求和查询**:
   假设年龄范围[0, 120]。添加/删除一个人改变总和最多120。$\Delta_1 = 120$
   
   **(c) 平均值查询**:
   设数据范围[0, M]，数据集大小为n。
   $$\Delta = \max |\frac{S}{n} - \frac{S \pm M}{n \pm 1}| \approx \frac{M}{n}$$
   
   敏感度依赖于n，需要谨慎处理。
   
   </details>

### 6.2 应用题 (L3)

3. **稀疏向量技术** [难度⭐⭐⭐]
   
   描述稀疏向量技术（Sparse Vector Technique）如何回答" above threshold"查询，并分析其隐私优势。
   
   <details>
   <summary>解答</summary>
   
   **问题**: 有k个查询 $f_1, ..., f_k$，只想知道哪些超过阈值T，而不需要精确值。
   
   **朴素方法**: 对每个查询加噪声后比较，隐私损失 $O(k\epsilon)$。
   
   **稀疏向量技术**:
   1. 噪声化阈值：$\tilde{T} = T + \text{Lap}(2/\epsilon)$
   2. 对每个查询：
      - 计算 $f_i(D) + \text{Lap}(4/\epsilon)$
      - 若超过 $\tilde{T}$，报告"above"并停止
   
   **隐私优势**:
   - 无论多少查询，隐私损失仅为 $\epsilon$
   - 因为算法在第一个"above"后停止
   - 适用于稀疏场景（少数查询超过阈值）
   
   </details>

### 6.3 证明题 (L4-L5)

4. **后处理免疫证明** [难度⭐⭐⭐⭐]
   
   证明：若 $\mathcal{M}$ 满足 $\epsilon$-DP，则对任意（可能是随机的）函数 $g$，$g(\mathcal{M}(D))$ 也满足 $\epsilon$-DP。
   
   <details>
   <summary>解答</summary>
   
   **证明**:
   设 $S$ 是 $g \circ \mathcal{M}$ 的任意输出子集。
   
   定义 $S' = \{y : g(y) \in S\}$
   
   则：
   $$P(g(\mathcal{M}(D)) \in S) = P(\mathcal{M}(D) \in S')$$
   
   由 $\mathcal{M}$ 的差分隐私：
   $$\leq e^\epsilon \cdot P(\mathcal{M}(D') \in S') = e^\epsilon \cdot P(g(\mathcal{M}(D')) \in S)$$
   
   因此 $g \circ \mathcal{M}$ 满足 $\epsilon$-DP。∎
   
   **意义**: 无论攻击者如何分析DP输出，隐私保证不变。
   
   </details>

5. **DP经验风险最小化的效用界** [难度⭐⭐⭐⭐⭐]
   
   证明使用目标扰动的DP-ERM达到 $O(\frac{\sqrt{d}}{n\epsilon})$ 的额外风险。
   
   <details>
   <summary>解答</summary>
   
   **设置**:
   - 损失函数 $L(\theta) = \frac{1}{n}\sum_i \ell(\theta; d_i) + \frac{\lambda}{2}\|\theta\|^2$
   - 目标扰动：$\tilde{L}(\theta) = L(\theta) + \frac{b^T\theta}{n}$，其中 $b \sim \text{Lap}(\Delta/\epsilon)^d$
   
   **分析**:
   1. **隐私**: 每个 $b_i$ 独立采样，机制满足 $\epsilon$-DP
   
   2. **效用**:
      设 $\theta^*$ 为真实最优，$\tilde{\theta}$ 为扰动后最优。
      
      由强凸性：
      $$L(\tilde{\theta}) - L(\theta^*) \leq \frac{1}{n} b^T(\tilde{\theta} - \theta^*)$$
      
      由Cauchy-Schwarz：
      $$\leq \frac{1}{n}\|b\| \cdot \|\tilde{\theta} - \theta^*\|$$
      
      以高概率 $\|b\| = O(\frac{\sqrt{d}}{\epsilon})$，且 $\|\tilde{\theta} - \theta^*\| = O(1)$
      
      因此额外风险为 $O(\frac{\sqrt{d}}{n\epsilon})$。∎
   
   </details>

### 7.2 实际案例

**案例**: 美国2020年人口普查

**背景**:
美国人口普查局使用差分隐私保护人口数据发布，这是DP历史上最大规模的应用。

**技术方案**:
- 使用"TopDown"算法注入噪声
- 保证多级地理层次的内部一致性
- 参数 $\epsilon = 19.61$（经过校准）

**挑战与解决方案**:
- **数据效用**: 小群体数据质量下降
- **透明度**: 公开隐私参数和噪声注入过程
- **法律合规**: 满足1939年法律要求的"准确统计"

### 7.3 跨领域联系

**与机器学习的联系**:
DP-SGD使深度学习可以 privacy-preserving。主要挑战：效用-隐私权衡、超参数选择。

**与密码学的联系**:
差分隐私提供"信息论"隐私（与计算能力无关），与密码学的"计算隐私"互补。实践中常结合使用。

**与统计学的联系**:
DP统计推断需要新的方法论：DP假设检验、DP置信区间、DP贝叶斯分析。

### 8.1 隐私定义对比

| 方法 | 攻击模型 | 保护级别 | 数据效用 | 主要缺点 |
|------|----------|----------|----------|----------|
| k-匿名 | 背景知识 | 弱 | 高 | 易受组合攻击 |
| l-多样性 | 背景知识 | 中等 | 中等 | 实现困难 |
| t-接近 | 背景知识 | 中等 | 中等 | 复杂度高 |
| 差分隐私 | 任意 | 强 | 可量化 | 噪声引入 |
| 同态加密 | 计算资源 | 强 | 低（计算）| 计算开销大 |
| 安全多方计算 | 合谋限制 | 强 | 中等 | 通信开销大 |

### 8.2 机制对比

| 机制 | 输出类型 | 敏感度 | 噪声分布 | 适用场景 |
|------|----------|--------|----------|----------|
| 拉普拉斯 | 数值 | L1 | 拉普拉斯 | 计数、求和 |
| 高斯 | 数值 | L2 | 高斯 | 高维、梯度 |
| 指数 | 离散选择 | L1评分 | 指数权重 | 最优化选择 |
| 随机响应 | 布尔 | - | 伯努利 | 敏感问题调查 |
| 目标扰动 | 优化 | L1梯度 | 拉普拉斯 | ERM |
| 输出扰动 | 优化 | L2解 | 高斯 | 优化问题 |

### 8.3 决策建议

```
保护需求？
```

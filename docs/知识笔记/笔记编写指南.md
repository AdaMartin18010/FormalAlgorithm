## 一、编写原则

### 1.1 准确性原则

- **内容必须准确**: 所有定义、定理、证明必须经过验证
- **引用必须可靠**: 优先引用权威教材、经典论文、官方文档
- **区分确定与推测**: 明确标注哪些是已知结论，哪些是开放问题

### 1.2 完整性原则

- **覆盖模板所有章节**: 除非确实不适用，否则应填写模板中的所有部分
- **提供充分细节**: 证明过程不应跳过关键步骤
- **包含正例与反例**: 既展示正确用法，也指出常见错误

### 1.3 可读性原则

- **渐进式展开**: 从直观解释开始，逐步深入到形式化内容
- **多层次呈现**: 提供直观理解、形式定义、具体示例三个层次
- **适度重复**: 重要概念可在不同章节以不同角度重复

## 二、写作风格

### 2.1 语言风格

**使用规范的学术中文**:
- 使用"我们"而非"我"，体现学术写作的客观性
- 避免口语化表达，如"很简单"、"很显然"
- 专业术语首次出现时应给出定义

**示例**:
```markdown
❌ 不好的写法：
这个算法很简单，就是分治的思想。

✅ 好的写法：
该算法采用分治策略，将问题分解为两个规模减半的子问题，
递归求解后合并结果。时间复杂度满足递推关系 $T(n) = 2T(n/2) + O(n)$。
```

### 2.2 结构层次

**标题层级规范**:
```markdown
# 一级标题：主题名称
## 二级标题：模板主要章节（一、二、三...）
### 三级标题：子章节（1.1, 1.2, 1.3...）
#### 四级标题：具体内容细分（可选）
```

**列表使用规范**:
- 同级概念使用无序列表 `-`
- 有顺序的步骤使用有序列表 `1.`
- 定义-解释对使用加粗 `**术语**:`

### 2.3 段落规范

- 每个段落聚焦一个主题
- 段落长度控制在3-5行（约100-150字）
- 使用空行分隔段落，提高可读性

## 三、数学符号规范

### 3.1 基本符号

| 符号 | 含义 | LaTeX表示 |
|------|------|-----------|
| $O, \Omega, \Theta$ | 渐近记号 | `O(n), \Omega(n), \Theta(n)` |
| $\mathbb{N}, \mathbb{Z}, \mathbb{R}$ | 数集 | `\mathbb{N}, \mathbb{Z}, \mathbb{R}` |
| $\forall, \exists$ | 量词 | `\forall, \exists` |
| $\Rightarrow, \Leftrightarrow$ | 蕴含、等价 | `\Rightarrow, \Leftrightarrow` |
| $\in, \subseteq, \cup, \cap$ | 集合运算 | `\in, \subseteq, \cup, \cap` |
| $\sum, \prod, \int$ | 求和、求积、积分 | `\sum, \prod, \int` |

### 3.2 算法相关符号

```latex
% 算法名称使用正体
\text{ALG}, \text{OPT}

% 复杂度类使用正体
\text{P}, \text{NP}, \text{PSPACE}

% 概率使用Pr或mathbb{P}
\Pr[X = x], \mathbb{E}[X]

% 指示函数
\mathbf{1}_A 或 \mathbb{I}_A
```

### 3.3 公式排版

**行内公式**: 
```markdown
时间复杂度为$O(n \log n)$的算法...
```

**独立公式**:
```markdown
$$T(n) = 2T(n/2) + O(n)$$
```

**多行对齐**:
```markdown
$$
\begin{align}
f(n) &= 2f(n-1) + n \\
     &= 2(2f(n-2) + (n-1)) + n \\
     &= 4f(n-2) + 2(n-1) + n
\end{align}
$$
```

**条件公式**:
```markdown
$$
dp[i][j] = \max\begin{cases}
dp[i-1][j] & \text{不选第}i\text{个物品} \\
dp[i-1][j-w_i] + v_i & \text{选第}i\text{个物品}
\end{cases}
$$
```

## 四、代码格式规范

### 4.1 代码块规范

**指定语言**:
```markdown
```python
# Python代码
```

```cpp
// C++代码
```

```haskell
-- Haskell代码
```
```

**代码注释**:
- 使用行内注释解释关键逻辑
- 函数头部提供文档字符串
- 复杂算法添加步骤说明

**示例**:
```python
def quicksort(arr):
    """
    快速排序实现
    
    Args:
        arr: 待排序列表
    
    Returns:
        排序后的新列表
    
    时间复杂度: 平均O(n log n), 最坏O(n²)
    空间复杂度: O(log n)递归栈
    """
    # 基准情况：空列表或单元素
    if len(arr) <= 1:
        return arr
    
    # 选择枢轴（这里简单地选第一个元素）
    pivot = arr[0]
    
    # 分区：小于、等于、大于枢轴的元素
    less = [x for x in arr[1:] if x < pivot]
    equal = [x for x in arr if x == pivot]
    greater = [x for x in arr[1:] if x > pivot]
    
    # 递归排序并合并
    return quicksort(less) + equal + quicksort(greater)
```

### 4.2 伪代码规范

使用类Python的伪代码，重点在逻辑清晰：

```markdown
```
算法: Dijkstra最短路径
输入: 图G=(V, E)，权重函数w，源点s
输出: 从s到所有顶点的最短距离

1. 初始化: 对所有v ∈ V, dist[v] ← ∞
2. dist[s] ← 0
3. Q ← 优先队列，包含所有顶点，按键为dist值
4. while Q ≠ ∅ do
5.     u ← EXTRACT-MIN(Q)
6.     for 每个v ∈ Adj[u] do
7.         if dist[v] > dist[u] + w(u, v) then
8.             dist[v] ← dist[u] + w(u, v)
9.             DECREASE-KEY(Q, v, dist[v])
10.        end if
11.    end for
12. end while
13. return dist
```
```

### 4.3 复杂度标注

在代码注释或伪代码后明确标注复杂度：

```markdown
**复杂度分析**:
- 时间复杂度: $O((V + E) \log V)$
- 空间复杂度: $O(V)$
```

## 六、版本控制规范

### 6.1 版本号规则

采用语义化版本：v主版本.次版本
- **主版本**: 重大结构变化、核心内容更新
- **次版本**: 内容扩充、错误修正、格式优化

### 6.2 修订历史格式

```markdown
| 版本 | 日期 | 修订者 | 修订内容 |
|------|------|--------|----------|
| v1.0 | 2026-04-09 | 作者名 | 初始版本 |

| v2.0 | 2026-05-01 | 作者名 | 重写第四节，增加新算法 |
```

对于重要变更，可在正文中临时标注：
```markdown

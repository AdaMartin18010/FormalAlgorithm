## 一、核心概念

### 1.1 定义

**正式定义**:
字符串是由有限字母表$\Sigma$上的字符组成的有限序列。字符串算法研究字符串的表示、处理、匹配、压缩和变换等问题。

**基本术语**:

- **字符串**: $S = s_1s_2\ldots s_n$，其中$s_i \in \Sigma$，$|S| = n$为长度
- **子串**: $S[i..j] = s_i s_{i+1} \ldots s_j$，长度$j-i+1$
- **前缀**: $S[1..i]$，后缀: $S[i..n]$
- **真前缀/后缀**: 不等于原串的前缀/后缀

**核心问题分类**:

| 问题类型 | 描述 | 经典算法 |
|----------|------|----------|
| **精确匹配** | 在文本中查找模式串 | KMP, BM, Rabin-Karp |
| **多模式匹配** | 同时匹配多个模式 | Aho-Corasick, Wu-Manber |
| **近似匹配** | 允许k个错配的匹配 | 动态规划, bit-parallel |
| **后缀数据结构** | 后缀树、后缀数组 | Ukkonen, SA-IS |
| **字符串压缩** | Lempel-Ziv系列 | LZ77, LZ78, LZW |

**直观解释**:
字符串算法是文本处理的数学基础。从搜索引擎的关键字匹配到DNA序列比对，从压缩软件到编译器的词法分析，字符串算法无处不在。其核心挑战在于：如何在近乎线性的时间内处理海量文本数据。

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 时间复杂度 | 从$O(nm)$到$O(n+m)$再到$O(n)$ | 取决于算法和问题 |
| 空间复杂度 | 从$O(1)$到$O(n)$或更高 | 后缀树可达$O(n)$ |
| 在线/离线 | 有些算法支持在线处理 | KMP在线，BM通常离线 |
| 稳定性 | 不同算法对重复模式的处理 | 影响实际应用选择 |

**性质总结**:

1. **字符串匹配下界**: 基于比较的字符串匹配需要$\Omega(n)$时间
2. **后缀数组与后缀树**: 两者可相互转化，时间复杂度等价
3. **信息论下界**: 字符串压缩存在理论极限（香农熵）

### 1.3 变体

**精确匹配变体**:

- **多模式匹配**: 一次搜索多个模式（AC自动机）
- **二维匹配**: 在二维文本中匹配（图像处理）
- **带通配符匹配**: 模式串包含通配符

**近似匹配变体**:

- **编辑距离**: 最少编辑操作次数
- **Hamming距离**: 错配字符数
- **最长公共子序列(LCS)**: 序列比对基础

**高级数据结构**:

- **后缀树**: 所有后缀的压缩字典树，$O(n)$构造
- **后缀数组**: 后缀的有序数组，$O(n\log n)$或$O(n)$构造
- **LCP数组**: 最长公共前缀数组，辅助后缀数组

## 三、形式化证明

### 3.1 核心定理

**定理 1** (KMP正确性): Knuth-Morris-Pratt算法在所有输入上正确找到所有匹配位置，时间复杂度$O(n+m)$。

**证明**:

```
正确性证明：

设next[j]为模式串P[1..j-1]的最长相等真前缀和真后缀长度。

不变式：算法维护文本位置i和模式位置j，保证
P[1..j-1] = T[i-j+1..i-1]。

初始化：i=1, j=1，空串匹配，不变式成立。

保持：
- 若P[j] = T[i]，则i++, j++，不变式保持
- 若P[j] ≠ T[i]，则j = next[j]，由next定义，
  P[1..j-1]仍是T[i-j+1..i-1]的后缀

终止：当i > n时停止，所有匹配已找到。

复杂度证明：
- 构建next数组：O(m)
- 匹配过程：i只增不减，j每次匹配增加1，
  不匹配时j减少但不少于0
- 设k = 2i - j，每次迭代k至少增加1
- 总迭代次数O(n)
- 总复杂度：O(n + m) ∎
```

**证明要点分析**:

1. **next数组意义**: 失配时利用已知信息避免回溯
2. **势能分析**: $k = 2i - j$作为势函数，单调递增
3. **线性保证**: 字符比较次数被$2n+m$限制

### 3.2 辅助引理

**引理 2** (前缀函数性质): 对于模式串P，前缀函数$\pi[q]$可在$O(m)$时间内计算。

*证明*:
使用增量法，利用已计算的$\pi[1..q-1]$来计算$\pi[q]$。
通过递归应用$\pi$，每次至少减少一个字符，总时间$O(m)$。∎

## 四、算法/方法详解

### 4.1 KMP算法

**伪代码**:

```
算法: KMP-MATCHER(T, P)
输入: 文本T[1..n]，模式P[1..m]
输出: 所有匹配位置

1.  π = COMPUTE-PREFIX-FUNCTION(P)
2.  q = 0  // 已匹配字符数
3.  for i = 1 to n do
4.      while q > 0 and P[q+1] ≠ T[i] do
5.          q = π[q]
6.      if P[q+1] == T[i] then
7.          q = q + 1
8.      if q == m then
9.          输出匹配位置 i - m + 1
10.         q = π[q]
```

**流程图**:

```
构建前缀函数π
    ↓
初始化 q = 0
    ↓
遍历文本每个字符:
    ↓
失配? → 利用π回退
    ↓
匹配? → q增加
    ↓
q == m? → 找到匹配
    ↓
继续搜索
```

### 4.2 正确性分析

**不变式**: 每次迭代开始时，$q$等于$T[1..i-1]$的最长后缀，该后缀也是$P$的前缀。

**证明**:

- **初始化**: $q=0$，空串是任何串的前缀/后缀
- **保持**: 算法维护该不变式，失配时$\pi$保证正确回退
- **终止**: 每次$q=m$时，找到完整匹配

### 4.3 复杂度分析

**时间复杂度**:

- 最坏情况: $O(n + m)$
- 最好情况: $O(n)$（无匹配时）

**空间复杂度**: $O(m)$存储前缀函数

**复杂度证明**:

```
匹配循环分析：
- i从1到n，执行n次
- 内层while每次至少减少q
- q增加的总次数 ≤ n（每次匹配一个字符）
- 因此q减少的总次数 ≤ n
- 总迭代次数 ≤ 2n = O(n)

构建π数组类似分析，总时间O(m)。
```

   - i=1..4: 匹配ABC␣，失配于D

# Python: KMP算法完整实现
from typing import List

def compute_prefix_function(pattern: str) -> List[int]:
    """计算KMP的前缀函数π"""
    m = len(pattern)
    pi = [0] * m
    k = 0

    for q in range(1, m):
        while k > 0 and pattern[k] != pattern[q]:
            k = pi[k - 1]
        if pattern[k] == pattern[q]:
            k += 1
        pi[q] = k

    return pi

def kmp_search(text: str, pattern: str) -> List[int]:
    """
    KMP字符串匹配算法
    返回所有匹配的起始位置（0索引）
    时间复杂度: O(n + m)
    空间复杂度: O(m)
    """
    n, m = len(text), len(pattern)
    if m == 0 or m > n:
        return []

    pi = compute_prefix_function(pattern)
    matches = []
    q = 0  # 已匹配字符数

    for i in range(n):
        # 失配时回退
        while q > 0 and pattern[q] != text[i]:
            q = pi[q - 1]

        # 匹配当前字符
        if pattern[q] == text[i]:
            q += 1

        # 找到完整匹配
        if q == m:
            matches.append(i - m + 1)
            q = pi[q - 1]  # 继续搜索

    return matches


def edit_distance(s1: str, s2: str) -> int:
    """
    计算两个字符串的编辑距离（Levenshtein距离）
    时间复杂度: O(|s1| × |s2|)
    空间复杂度: O(|s2|)（滚动数组优化）
    """
    m, n = len(s1), len(s2)

    # 空间优化：只保留两行
    prev = list(range(n + 1))
    curr = [0] * (n + 1)

    for i in range(1, m + 1):
        curr[0] = i
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                curr[j] = prev[j-1]  # 字符相同
            else:
                curr[j] = 1 + min(
                    prev[j],      # 删除
                    curr[j-1],    # 插入
                    prev[j-1]     # 替换
                )
        prev, curr = curr, prev  # 交换引用

    return prev[n]


def build_suffix_array(s: str) -> List[int]:
    """
    构建后缀数组（O(n log²n)简单实现）
    返回按字典序排序的后缀起始索引
    """
    s = s + '$'  # 添加哨兵字符
    n = len(s)

    # 初始排序：按第一个字符
    sa = list(range(n))
    rank = [ord(c) for c in s]
    tmp = [0] * n

    k = 1
    while k < n:
        # 按第二关键字排序，然后按第一关键字排序
        sa.sort(key=lambda x: (rank[x], rank[x + k] if x + k < n else -1))

        tmp[sa[0]] = 0
        for i in range(1, n):
            prev_key = (rank[sa[i-1]], rank[sa[i-1] + k] if sa[i-1] + k < n else -1)
            curr_key = (rank[sa[i]], rank[sa[i] + k] if sa[i] + k < n else -1)
            tmp[sa[i]] = tmp[sa[i-1]] + (curr_key != prev_key)

        rank, tmp = tmp, rank
        if rank[sa[-1]] == n - 1:
            break
        k *= 2

    return sa[1:]  # 去掉哨兵

// Rust: 高性能字符串匹配实现
use std::vec::Vec;

/// 计算KMP前缀函数
pub fn prefix_function(pattern: &[u8]) -> Vec<usize> {
    let m = pattern.len();
    let mut pi = vec![0; m];
    let mut k = 0;

    for q in 1..m {
        while k > 0 && pattern[k] != pattern[q] {
            k = pi[k - 1];
        }
        if pattern[k] == pattern[q] {
            k += 1;
        }
        pi[q] = k;
    }

    pi
}

/// KMP字符串匹配
pub fn kmp_search(text: &[u8], pattern: &[u8]) -> Vec<usize> {
    let n = text.len();
    let m = pattern.len();

    if m == 0 || m > n {
        return Vec::new();
    }

    let pi = prefix_function(pattern);
    let mut matches = Vec::new();
    let mut q = 0;

    for (i, &c) in text.iter().enumerate() {
        while q > 0 && pattern[q] != c {
            q = pi[q - 1];
        }
        if pattern[q] == c {
            q += 1;
        }
        if q == m {
            matches.push(i - m + 1);
            q = pi[q - 1];
        }
    }

    matches
}

/// 编辑距离计算（DP，空间优化）
pub fn edit_distance(s1: &str, s2: &str) -> usize {
    let s1 = s1.as_bytes();
    let s2 = s2.as_bytes();
    let (m, n) = (s1.len(), s2.len());

    // 确保s2是较短的字符串以优化空间
    let (s1, s2, m, n) = if m < n {
        (s2, s1, n, m)
    } else {
        (s1, s2, m, n)
    };

    let mut prev: Vec<usize> = (0..=n).collect();
    let mut curr = vec![0; n + 1];

    for i in 1..=m {
        curr[0] = i;
        for j in 1..=n {
            if s1[i-1] == s2[j-1] {
                curr[j] = prev[j-1];
            } else {
                curr[j] = 1 + prev[j].min(curr[j-1]).min(prev[j-1]);
            }
        }
        std::mem::swap(&mut prev, &mut curr);
    }

    prev[n]
}

/// Boyer-Moore-Horspool算法（简化版BM）
pub fn bm_search(text: &[u8], pattern: &[u8]) -> Vec<usize> {
    let n = text.len();
    let m = pattern.len();

    if m == 0 || m > n {
        return Vec::new();
    }

    // 构建坏字符表
    let mut bad_char = [m; 256];
    for (i, &c) in pattern.iter().take(m-1).enumerate() {
        bad_char[c as usize] = m - 1 - i;
    }

    let mut matches = Vec::new();
    let mut i = 0;

    while i \leq n - m {
        let mut j = m - 1;
        while j > 0 && pattern[j] == text[i + j] {
            j -= 1;
        }

        if j == 0 && pattern[0] == text[i] {
            matches.push(i);
            i += 1;
        } else {
            i += bad_char[text[i + m - 1] as usize];
        }
    }

    matches
}

## 5.3 反例

### 5.3 反例

**常见错误1**: KMP中前缀函数计算错误

```python
# 错误代码：
while k > 0 and pattern[k] != pattern[q]:
    k = pi[k]  # 错误！应该是pi[k-1]

# 正确代码：
while k > 0 and pattern[k] != pattern[q]:
    k = pi[k - 1]  # 回退到前一个位置的最长前缀
```

**错误原因**: 失配时需要回退到$\pi[k-1]$，而非$\pi[k]$
**正确做法**: 注意数组索引和前缀函数定义的对应关系

**常见错误2**: 编辑距离初始化错误

```python
# 错误：
dp[0][j] = 0  # 应该是j

# 正确：
dp[0][j] = j  # 空串变成长度为j的串需要j次插入
```

**错误原因**: 边界条件理解错误
**正确做法**: 明确dp[i][j]的定义，仔细处理边界

   <details>
   <summary>解答</summary>

   在任何基于比较的模型中，算法必须检查文本中的每个字符至少一次，否则可以构造一个输入使得未检查的字符改变匹配结果。

   形式化：设算法不检查位置i，考虑两个仅在该位置不同的文本，算法无法区分，矛盾。

   </details>

### 6.2 应用题 (L2-L3)

1. **实现多模式匹配** [难度⭐⭐]

   使用Aho-Corasick算法同时查找多个模式串。

   <details>
   <summary>解答</summary>

   ```python
   class AhoCorasick:
       def __init__(self):
           self.trie = {}
           self.fail = {}
           self.output = {}

       def add_pattern(self, pattern, idx):
           node = 0
           for c in pattern:
               if (node, c) not in self.trie:
                   self.trie[(node, c)] = len(self.trie) + 1
               node = self.trie[(node, c)]
           self.output[node] = self.output.get(node, []) + [idx]

       def build(self):
           from collections import deque
           queue = deque()
           # BFS构建失败指针
           for (node, c), next_node in self.trie.items():
               if node == 0:
                   self.fail[next_node] = 0
                   queue.append(next_node)

           while queue:
               r = queue.popleft()
               for (node, c), next_node in list(self.trie.items()):
                   if node == r:
                       f = self.fail[r]
                       while f != 0 and (f, c) not in self.trie:
                           f = self.fail[f]
                       self.fail[next_node] = self.trie.get((f, c), 0)
                       self.output[next_node] = self.output.get(next_node, []) + \
                                                self.output.get(self.fail[next_node], [])
                       queue.append(next_node)

       def search(self, text):
           matches = []
           node = 0
           for i, c in enumerate(text):
               while node != 0 and (node, c) not in self.trie:
                   node = self.fail[node]
               node = self.trie.get((node, c), 0)
               for pattern_idx in self.output.get(node, []):
                   matches.append((i, pattern_idx))
           return matches
   ```

   </details>

2. **最长回文子串** [难度⭐⭐⭐]

   使用Manacher算法在$O(n)$时间内求最长回文子串。

   <details>
   <summary>解答</summary>

   ```python
   def manacher(s: str) -> str:
       # 预处理：插入分隔符
       t = '#' + '#'.join(s) + '#'
       n = len(t)
       p = [0] * n  # p[i]表示以i为中心的回文半径

       center = right = 0
       for i in range(n):
           # 利用对称性
           if i < right:
               mirror = 2 * center - i
               p[i] = min(right - i, p[mirror])

           # 中心扩展
           while i - p[i] - 1 \geq 0 and i + p[i] + 1 < n and \
                 t[i - p[i] - 1] == t[i + p[i] + 1]:
               p[i] += 1

           # 更新右边界
           if i + p[i] > right:
               center, right = i, i + p[i]

       # 找到最大回文
       max_len = max(p)
       center_idx = p.index(max_len)
       start = (center_idx - max_len) // 2
       return s[start:start + max_len]
   ```

   </details>

### 6.3 证明题 (L4-L5)

1. **后缀数组构造复杂度** [难度⭐⭐⭐⭐]

   证明倍增法构造后缀数组的时间复杂度为$O(n\log n)$。

   <details>
   <summary>解答</summary>

   倍增法每轮将比较长度k翻倍，共进行$O(\log n)$轮。

   每轮使用计数排序或基数排序，时间$O(n)$。

   总时间：$O(n \cdot \log n)$。

   更优的SA-IS算法可达$O(n)$。

   </details>

2. **编辑距离与最长公共子序列关系** [难度⭐⭐⭐⭐⭐]

   证明对于字符串X和Y：
   $$ED(X,Y) = |X| + |Y| - 2 \cdot |LCS(X,Y)|$$
   其中ED是编辑距离（只允许插入和删除），LCS是最长公共子序列。

   <details>
   <summary>解答</summary>

   设$Z = LCS(X,Y)$，$|Z| = l$。

   构造编辑序列：
   - 删除X中不在Z中的字符：$|X| - l$次删除
   - 插入Y中不在Z中的字符：$|Y| - l$次插入

   总编辑次数：$(|X| - l) + (|Y| - l) = |X| + |Y| - 2l$。

   证明最优性：任何编辑序列对应一个公共子序列，编辑次数至少为$|X| + |Y| - 2l$。

   </details>

### 7.2 实际案例

**案例**: Git的diff算法

**背景**:
Git需要高效计算两个文件版本的差异，用于提交展示和合并。

**应用方式**:

- 基于Myers差分算法（编辑距离的变体）
- 使用最长公共子序列(LCS)找到最佳匹配
- 时间复杂度$O(ND)$，其中N是文件长度，D是差异数量

**效果**:

- 实际应用中D通常很小
- 因此性能接近线性
- 支持行级和字符级差异

## 八、多维对比

### 8.1 方法对比

| 维度 | 暴力匹配 | KMP | Boyer-Moore | Rabin-Karp |
|------|----------|-----|-------------|------------|
| 预处理 | 无 | $O(m)$ | $O(m+\|\Sigma\|)$ | $O(m)$ |
| 匹配时间 | $O(nm)$ | $O(n+m)$ | $O(n+m)$最坏 | $O(n+m)$期望 |
| 空间 | $O(1)$ | $O(m)$ | $O(m+\|\Sigma\|)$ | $O(1)$ |
| 在线处理 | 是 | 是 | 否 | 否 |
| 实际速度 | 慢 | 中等 | 通常最快 | 快（多模式） |

### 8.2 决策建议

**选择KMP当**:

- 需要在线处理（流式数据）
- 模式串较短
- 最坏情况保证重要

**选择Boyer-Moore当**:

- 一般文本搜索
- 字母表较大（如ASCII）
- 追求实际速度

**选择Rabin-Karp当**:

- 多模式匹配
- 需要模糊匹配
- 可以容忍哈希碰撞检查的开销

**决策流程图**:

```
是否需要在线处理？
    ├── 是 → 使用KMP
    └── 否 → 多模式？
              ├── 是 → Aho-Corasick或Rabin-Karp
              └── 否 → 追求实际速度？
                        ├── 是 → Boyer-Moore
                        └── 否 → KMP（简单可靠）
```

## 一、核心概念

### 1.1 定义

**正式定义**:
**线性规划(LP)**是在线性约束条件下优化线性目标函数的数学优化问题。

**标准形式**:
$$
\begin{align}
\min \quad & c^T x \\
\text{s.t.} \quad & Ax = b \\
& x \geq 0
\end{align}
$$

其中：

- $x \in \mathbb{R}^n$是决策变量
- $c \in \mathbb{R}^n$是目标函数系数
- $A \in \mathbb{R}^{m \times n}$是约束矩阵
- $b \in \mathbb{R}^m$是约束右侧向量

**不等式形式**:
$$
\begin{align}
\min \quad & c^T x \\
\text{s.t.} \quad & Ax \leq b
\end{align}
$$

**可行域**: 满足所有约束的$x$的集合$\mathcal{F} = \{x : Ax = b, x \geq 0\}$

**基本术语**:

- **可行解**: $x \in \mathcal{F}$
- **最优解**: $x^* \in \arg\min_{x \in \mathcal{F}} c^T x$
- **无界**: 目标函数可以无限减小
- **不可行**: $\mathcal{F} = \emptyset$

**直观解释**:
线性规划是在一个由线性不等式定义的多面体（可行域）上寻找最优点的过程。可以想象在一个由平面围成的凸多面体上寻找最低点（或最高点），最优解一定出现在某个顶点（极点）上。

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 凸性 | 可行域是凸集，目标函数是凸函数 | 保证局部最优即全局最优 |
| 极值点定理 | 若最优解存在，则在某极点达到 | 单纯形法的基础 |
| 对偶性 | 每个LP都有对应的对偶问题 | 强对偶定理 |
| 互补松弛性 | 原始-对偶最优解的关系 | KKT条件的特例 |

**性质总结**:

1. **可行域结构**: LP的可行域是凸多面体
2. **最优解位置**: 最优解在极值点或极值面上
3. **无界性检测**: 可以构造使目标函数无限下降的方向

### 1.3 变体

**整数线性规划(ILP)**:

- 定义: 变量限制为整数
- 难度: NP-hard
- 方法: 分支定界、割平面

**混合整数线性规划(MILP)**:

- 定义: 部分变量为整数
- 应用: 调度、设施选址

**多目标线性规划**:

- 定义: 多个目标函数
- 解法: 加权和、ε-约束法

**鲁棒优化**:

- 定义: 考虑参数不确定性
- 方法: 不确定集、场景优化

## 三、形式化证明

### 3.1 核心定理

**定理 1** (强对偶定理): 若原始问题或对偶问题有最优解，则两者都有最优解，且最优值相等。

**原始问题(P)**:
$$\min c^T x \text{ s.t. } Ax = b, x \geq 0$$

**对偶问题(D)**:
$$\max b^T y \text{ s.t. } A^T y \leq c$$

**证明**:

```
弱对偶性（容易证明）：
对于任意可行解x（P）和y（D）：
    c^T x ≥ (A^T y)^T x = y^T A x = y^T b = b^T y

因此：min(P) ≥ max(D)

强对偶性证明概要（使用单纯形法或Farkas引理）：

设x*是(P)的最优解，则存在基B使得：
    x_B = A_B^{-1}b, x_N = 0
    c_N - c_B^T A_B^{-1} A_N ≥ 0（最优性条件）

令y* = (A_B^{-1})^T c_B，则：
    A^T y* ≤ c（由最优性条件）
    b^T y* = c_B^T A_B^{-1} b = c^T x*

因此y*是(D)的可行解且目标值相等，强对偶成立。∎
```

**证明要点分析**:

1. **弱对偶**: 直接代数推导，总是成立
2. **强对偶**: 需要额外条件（如可行域非空有界）
3. **构造性证明**: 单纯形法同时给出原始和对偶最优解

### 3.2 辅助引理

**引理 2** (Farkas引理): 下列两个系统有且仅有一个有解：

- 系统1: $Ax = b, x \geq 0$
- 系统2: $A^T y \leq 0, b^T y > 0$

*解释*: 要么$b$在$A$的列锥中，要么存在超平面分离$b$和该锥。

**引理 3** (极值点定理): LP的可行域是凸多面体，若最优解存在，则在某个极点达到。

*证明*: 目标函数是线性的，在凸紧集上的极值在边界达到。∎

## 四、算法/方法详解

### 4.1 单纯形法

**算法思想**:

1. 从一个基本可行解（极点）出发
2. 检查最优性条件
3. 若非最优，沿边移动到相邻的更好极点
4. 重复直到最优

**伪代码**:

```
算法: SIMPLEX(A, b, c)
输入: LP标准形式
输出: 最优解或报告无界/不可行

1.  找到初始基本可行解（两阶段法或大M法）
2.  while 存在进基变量（reduced cost < 0）do
3.      选择进基变量x_j（最负reduced cost）
4.      计算比率确定出基变量（最小比率检验）
5.      if 所有比率 = ∞ then
6.          return "无界"
7.      执行转轴操作，更新基
8.  return 当前基本可行解
```

**流程图**:

```
初始化基
    ↓
计算检验数(reduced cost)
    ↓
全部 ≥ 0?
    ├── 是 → 最优，返回
    └── 否 → 选择进基变量
              ↓
         最小比率检验
              ↓
         比率有限？
              ├── 否 → 无界
              └── 是 → 转轴操作
                        ↓
                    更新基，继续
```

### 4.2 正确性分析

**最优性条件**: 对于基$B$，检验数$\bar{c}_N = c_N - A_N^T (A_B^{-1})^T c_B \geq 0$时最优。

**证明**:

- 当前解$x_B = A_B^{-1}b, x_N = 0$
- 任意可行解可表示为$x = (x_B, x_N)$
- $c^T x = c_B^T x_B + c_N^T x_N$
- 由约束$A_B x_B + A_N x_N = b$，得$x_B = A_B^{-1}(b - A_N x_N)$
- 代入：$c^T x = c_B^T A_B^{-1}b + (c_N - A_N^T(A_B^{-1})^T c_B)^T x_N$
- 若$\bar{c}_N \geq 0$且$x_N \geq 0$，则$c^T x \geq c_B^T A_B^{-1}b = c^T x^*$，最优。

### 4.3 复杂度分析

**时间复杂度**:

- 最坏情况: 指数时间（Klee-Minty例子）
- 平均情况: 多项式时间（Smoothed analysis）
- 实际性能: 通常$O(mn)$到$O(m^2 n)$次迭代

**空间复杂度**: $O(mn)$存储矩阵

**复杂度证明**:

```
单纯形法复杂度分析：

每次迭代：
- 计算检验数：O(mn)
- 比率检验：O(m)
- 转轴操作：O(mn)

迭代次数：
- 理论上限：O(C(n,m))，可能指数级
- 实际中：通常是O(m)到O(m log n)

内点法对比：
- 理论：O(n^{3.5} L)多项式时间
- 实际：大规模问题上通常更快
```

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 生产计划问题

**问题描述**:
某工厂生产两种产品A和B，目标是最大化利润。

- 产品A每件利润40元，需要1小时机器时间和2小时人工
- 产品B每件利润60元，需要2小时机器时间和1小时人工
- 每天机器时间限制100小时，人工时间限制120小时

**建模**:

```
决策变量：
  x1 = 产品A的生产数量
  x2 = 产品B的生产数量

目标函数：
  max 40x1 + 60x2

约束条件：
  x1 + 2x2 ≤ 100  (机器时间)
  2x1 + x2 ≤ 120  (人工时间)
  x1, x2 ≥ 0
```

**解决过程**:

1. 绘制可行域（二维平面上的多边形）
2. 极点：(0,0), (60,0), (140/3, 80/3), (0, 50)
3. 计算目标函数值：
   - (0,0): 0
   - (60,0): 2400
   - (140/3, 80/3): 40×140/3 + 60×80/3 = 7200/3 = 2400
   - (0,50): 3000

**结果**: 最优解$x_1 = 0, x_2 = 50$，最大利润3000元

**示例 2**: 单纯形法执行示例

**问题**:

```
max 3x1 + 2x2
s.t.  x1 + x2 ≤ 4
      x1 - x2 ≤ 2
      x1, x2 ≥ 0
```

**标准形式**（引入松弛变量）:

```
max 3x1 + 2x2
s.t.  x1 + x2 + s1 = 4
      x1 - x2 + s2 = 2
      x1, x2, s1, s2 ≥ 0
```

**单纯形法迭代**:

1. 初始基: {s1, s2}，解(0,0,4,2)，目标0
2. x1进基（检验数3 > 0），s2出基（比率min(4/1, 2/1) = 2）
3. 新基: {s1, x1}，解(2,0,2,0)，目标6
4. x2进基（检验数5 > 0），s1出基（比率2/2 = 1）
5. 新基: {x2, x1}，解(3,1,0,0)，目标11
6. 所有检验数≤0，最优

**结果**: 最优解$x_1 = 3, x_2 = 1$，最优值11

```python
# Python: 线性规划求解示例
import numpy as np
from typing import Tuple, Optional

def simplex_solver(c: np.ndarray, A: np.ndarray, b: np.ndarray,
                   maximize: bool = True) -> Tuple[Optional[np.ndarray], float, str]:
    """
    简化版单纯形法实现
    标准形式: max c^T x s.t. Ax <= b, x >= 0

    返回: (最优解, 最优值, 状态)
    """
    m, n = A.shape

    if not maximize:
        c = -c

    # 引入松弛变量，转化为等式
    # A_ext = [A | I], c_ext = [c | 0]
    A_ext = np.hstack([A, np.eye(m)])
    c_ext = np.concatenate([c, np.zeros(m)])

    # 初始基: 松弛变量
    basic_vars = list(range(n, n + m))
    non_basic_vars = list(range(n))

    max_iter = 1000
    for _ in range(max_iter):
        # 计算基矩阵和基解
        B = A_ext[:, basic_vars]
        x_B = np.linalg.solve(B, b)

        # 计算检验数（reduced cost）
        c_B = c_ext[basic_vars]
        y = np.linalg.solve(B.T, c_B)

        reduced_costs = []
        for j in non_basic_vars:
            rc = c_ext[j] - np.dot(A_ext[:, j], y)
            reduced_costs.append((rc, j))

        # 检查最优性
        entering = None
        min_rc = 0
        for rc, j in reduced_costs:
            if rc < min_rc:
                min_rc = rc
                entering = j

        if entering is None:
            # 最优解找到
            x_opt = np.zeros(n + m)
            x_opt[basic_vars] = x_B
            obj_val = np.dot(c_ext[basic_vars], x_B)
            if not maximize:
                obj_val = -obj_val
            return x_opt[:n], obj_val, "optimal"

        # 比率检验
        d = np.linalg.solve(B, A_ext[:, entering])
        ratios = []
        for i, var in enumerate(basic_vars):
            if d[i] > 1e-10:
                ratios.append((x_B[i] / d[i], i))

        if not ratios:
            return None, float('inf') if maximize else float('-inf'), "unbounded"

        # 选择出基变量
        _, leaving_idx = min(ratios)
        leaving = basic_vars[leaving_idx]

        # 转轴操作：更新基
        basic_vars[leaving_idx] = entering
        non_basic_vars = [v for v in range(n + m) if v not in basic_vars]

    return None, 0, "max_iter_reached"


# 使用scipy的优化求解器
from scipy.optimize import linprog

def solve_lp_scipy(c, A_ub, b_ub, A_eq=None, b_eq=None, bounds=None):
    """
    使用scipy求解线性规划
    min c^T x
    s.t. A_ub x <= b_ub
         A_eq x = b_eq
         bounds 是变量的上下界
    """
    result = linprog(c, A_ub=A_ub, b_ub=b_ub,
                     A_eq=A_eq, b_eq=b_eq, bounds=bounds, method='highs')
    return result
```

## 5.3 反例

### 5.3 反例

**常见错误1**: 忽视非负约束

```python
# 错误：直接求解无约束问题
result = linprog(c, A_ub=A, b_ub=b)  # 缺少bounds

# 正确：明确添加非负约束
result = linprog(c, A_ub=A, b_ub=b, bounds=[(0, None), (0, None)])
```

**错误原因**: LP标准形式要求$x \geq 0$
**正确做法**: 明确指定变量边界

**常见错误2**: 混淆min和max

```python
# 错误：linprog默认min，但期望max
result = linprog(c, ...)  # c是max的系数

# 正确：取负
result = linprog(-c, ...)  # 最小化负值 = 最大化
```

**错误原因**: 大多数LP求解器默认最小化
**正确做法**: 注意求解器的优化方向约定

   <details>
   <summary>解答</summary>

   检验数$\bar{c}_j = c_j - c_B^T B^{-1} A_j$表示非基变量$x_j$增加1单位时目标函数的变化率。

   若所有$\bar{c}_j \geq 0$，则任何可行方向的移动都不会减小目标值（min问题），当前解最优。

   </details>

### 6.2 应用题 (L2-L3)

1. **网络流建模** [难度⭐⭐]

   将最大流问题建模为线性规划。

   <details>
   <summary>解答</summary>

   决策变量：$f_{uv}$表示边$(u,v)$上的流量

   ```
   max Σ_v f_{sv}
   s.t. Σ_u f_{uv} = Σ_w f_{vw}  ∀v ∉ {s,t}  (流量守恒)
        0 ≤ f_{uv} ≤ c_{uv}        ∀(u,v) ∈ E  (容量约束)
   ```

   最大流LP的约束矩阵是全单模的，因此有整数最优解。

   </details>

2. **投资组合优化** [难度⭐⭐⭐]

   有n种资产，预期收益率$r_i$，风险（方差）$\sigma_i^2$，最大投资预算B。要求预期收益至少R，最小化总风险（简化为投资金额的加权和）。

   <details>
   <summary>解答</summary>

   ```
   决策变量：x_i = 投资资产i的金额

   min Σ_i σ_i^2 * x_i  (简化风险度量)
   s.t. Σ_i r_i * x_i ≥ R      (收益要求)
        Σ_i x_i ≤ B            (预算约束)
        0 ≤ x_i ≤ u_i          (投资上限)
   ```

   这是一个LP问题（简化版），若使用真实方差则是QP。

   </details>

### 6.3 证明题 (L4-L5)

1. **Farkas引理证明** [难度⭐⭐⭐⭐]

   使用分离超平面定理证明Farkas引理。

   <details>
   <summary>解答</summary>

   设$C = \{Ax : x \geq 0\}$是$A$的列锥。

   若$b \notin C$，由分离超平面定理，存在$y$使得：
   - $y^T b > y^T Ax$ 对所有$x \geq 0$
   - 即$y^T b > 0$且$y^T A \leq 0$

   这正是系统2的解。

   若$b \in C$，则系统1有解，系统2无解（否则矛盾）。

   </details>

2. **单纯形法有限收敛性** [难度⭐⭐⭐⭐⭐]

   证明在Bland规则（选择下标最小的进基和出基变量）下，单纯形法必定在有限步内收敛。

   <details>
   <summary>解答</summary>

   反证法：假设存在循环$B_1, B_2, \ldots, B_k = B_1$。

   设$t$是循环中进基出基过的最大变量下标。

   设$B$是$t$出基前的基，$B'$是$t$进基后的基。

   由检验数变化关系，可以构造一个关于这些基的方程，导出矛盾。

   因此循环不可能存在，算法有限收敛。∎

   </details>

### 7.2 实际案例

**案例**: 航空公司机组排班

**背景**:
航空公司需要为数千个航班分配机组人员，满足法规、合同和偏好约束。

**应用方式**:

- 决策变量：某机组是否执行某航班序列
- 目标：最小化总成本（工资、住宿、调机）
- 约束：法规（飞行时间限制）、人员资质、航班覆盖

**效果**:

- 使用列生成技术处理大规模问题
- 节省数百万美元成本
- 问题规模可达数百万变量

## 八、多维对比

### 8.1 方法对比

| 维度 | 单纯形法 | 内点法 | 椭球法 |
|------|----------|--------|--------|
| 理论复杂度 | 指数（最坏） | 多项式$O(n^{3.5}L)$ | 多项式$O(n^6 L)$ |
| 实际性能 | 小规模快 | 大规模快 | 很慢（理论用） |
| 实现难度 | 中等 | 较复杂 | 简单 |
| 敏感度分析 | 容易 | 较难 | 难 |
| 适用规模 | 中等 | 大规模 | 理论用 |

### 8.2 决策建议

**选择单纯形法当**:

- 问题规模中等（<10000变量）
- 需要进行敏感度分析
- 需要理解算法行为

**选择内点法当**:

- 大规模问题
- 需要多项式时间保证
- 追求实际求解速度

**决策流程图**:

```
问题规模？
    ↓
小规模(<1万变量)？
    ├── 是 → 单纯形法（简单可靠）
    └── 否 → 大规模？
              ├── 是 → 内点法
              └── 否 → 使用商业求解器（自动选择）
```

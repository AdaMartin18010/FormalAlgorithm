## 一、核心概念

### 1.1 定义

**正式定义**:
高级数据结构是传统数据结构的扩展，支持更复杂的操作或提供更好的性能保证。本章涵盖**Splay树**、**B树**、**斐波那契堆**、**van Emde Boas树**等。

**Splay树**:
一种自调整的二叉搜索树，通过**伸展(splay)**操作将访问节点移动到根，摊还时间$O(\log n)$。

**B树/B+树**:
平衡的多路搜索树，设计用于磁盘存储。阶为$m$的B树满足：
- 每个节点最多$m$个子节点
- 非根非叶节点至少$\lceil m/2 \rceil$个子节点
- 所有叶节点在同一深度

**斐波那契堆**:
支持DECREASE-KEY摊还$O(1)$的优先队列，由Dijkstra和Prim算法的优化需求驱动设计。

**van Emde Boas树**:
支持$O(\log \log U)$操作的动态集合，其中$U$是全域大小。

**直观解释**:
高级数据结构针对特定场景优化：Splay树利用访问局部性，B树优化磁盘访问，斐波那契堆针对图算法优化。它们展示了数据结构与算法设计的深层联系。

### 1.2 属性

| 数据结构 | 操作复杂度 | 特点 | 适用场景 |
|----------|------------|------|----------|
| Splay树 | 摊还$O(\log n)$ | 自调整，无需平衡因子 | 缓存友好，局部性强 |
| B树 | $O(\log_m n)$ | 多分支，减少磁盘IO | 磁盘存储，数据库索引 |
| 斐波那契堆 | 摊还$O(1)$ DECREASE-KEY | 惰性合并 | Dijkstra, Prim优化 |
| vEB树 | $O(\log \log U)$ | 全域依赖 | 整数键，密集键空间 |

### 1.3 变体

**Splay树变体**:
- **Treap**: BST+堆的随机化结构
- **Skip List**: 随机化链表，简单实现

**B树变体**:
- **B+树**: 数据只在叶子，顺序访问优化
- **B*树**: 延迟分裂，提高填充率

**堆变体**:
- **配对堆**: 实践中最快的堆之一
- **二项堆**: 理论分析清晰

## 三、形式化证明

### 3.1 核心定理

**定理 1** (Splay树摊还界): Splay树操作的摊还时间为$O(\log n)$。

**证明**:
```
定义势函数：Φ(T) = Σ_{x∈T} r(x)，其中r(x) = log|S(x)|，S(x)是以x为根的子树。

关键引理：以x为根的子树上执行splay(x)的摊还成本≤3(r'(x)-r(x))+1。

证明分三种zig/zag情况，利用r的性质和代数变形。

m次操作的总摊还成本≤Σ(3(r(t)-r(x))+1)=O(m log n)。∎
```

**定理 2** (斐波那契堆DECREASE-KEY): 摊还时间为$O(1)$。

*证明概要*: 使用势函数$\Phi = trees + 2 \cdot marks$，分析级联切除的摊还成本。∎

## 四、算法详解

### 4.1 Splay树

**伸展操作**:
- **Zig**: 父节点是根，单旋转
- **Zig-Zig**: 父和祖父同方向，双旋转
- **Zig-Zag**: 父和祖父不同方向，双旋转

**伪代码**:
```
SPLAY(x):
    while x.parent != null do
        if x.parent == root then
            单旋转(x)
        else if (x是左孩子)==(x.parent是左孩子) then
            双旋转同向(x.parent)
        else
            双旋转异向(x)
```

### 4.2 B树操作

**插入**:
1. 找到合适的叶节点
2. 若节点未满，直接插入
3. 若节点满，分裂为两个节点，提升中间关键字

**删除**: 类似处理，可能需要合并或借位

## 五、示例与实例

### 5.2 代码实现

**Splay树Python实现**:
```python
class SplayNode:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None
        self.parent = None

class SplayTree:
    def __init__(self):
        self.root = None
    
    def _rotate_right(self, x):
        y = x.left
        x.left = y.right
        if y.right:
            y.right.parent = x
        y.parent = x.parent
        if not x.parent:
            self.root = y
        elif x == x.parent.right:
            x.parent.right = y
        else:
            x.parent.left = y
        y.right = x
        x.parent = y
    
    def _rotate_left(self, x):
        y = x.right
        x.right = y.left
        if y.left:
            y.left.parent = x
        y.parent = x.parent
        if not x.parent:
            self.root = y
        elif x == x.parent.left:
            x.parent.left = y
        else:
            x.parent.right = y
        y.left = x
        x.parent = y
    
    def _splay(self, x):
        while x.parent:
            if not x.parent.parent:
                # Zig
                if x == x.parent.left:
                    self._rotate_right(x.parent)
                else:
                    self._rotate_left(x.parent)
            elif x == x.parent.left and x.parent == x.parent.parent.left:
                # Zig-Zig
                self._rotate_right(x.parent.parent)
                self._rotate_right(x.parent)
            elif x == x.parent.right and x.parent == x.parent.parent.right:
                # Zig-Zig
                self._rotate_left(x.parent.parent)
                self._rotate_left(x.parent)
            elif x == x.parent.right and x.parent == x.parent.parent.left:
                # Zig-Zag
                self._rotate_left(x.parent)
                self._rotate_right(x.parent)
            else:
                # Zig-Zag
                self._rotate_right(x.parent)
                self._rotate_left(x.parent)
    
    def find(self, key):
        x = self.root
        while x:
            if key < x.key:
                x = x.left
            elif key > x.key:
                x = x.right
            else:
                self._splay(x)
                return x
        return None
    
    def insert(self, key):
        node = SplayNode(key)
        if not self.root:
            self.root = node
            return
        
        x = self.root
        while True:
            if key < x.key:
                if not x.left:
                    x.left = node
                    node.parent = x
                    break
                x = x.left
            else:
                if not x.right:
                    x.right = node
                    node.parent = x
                    break
                x = x.right
        
        self._splay(node)
```

## 六、习题

### 6.1 理解题

1. **Splay树为什么叫"自调整"** [难度⭐]

<details>
<summary>解答</summary>

Splay树不需要存储额外的平衡信息（如红黑树的颜色、AVL树的平衡因子）。它通过访问模式自动调整树结构，经常访问的节点靠近根，利用访问局部性。

</details>

2. **B树为什么适合磁盘存储** [难度⭐]

<details>
<summary>解答</summary>

B树的节点大小通常设置为磁盘页大小，使得每次读取一个节点只需一次磁盘IO。多分支设计降低了树高，减少了磁盘访问次数。

</details>

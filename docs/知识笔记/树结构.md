## 一、核心概念

### 1.1 定义

**正式定义**:
树是由n(n>=0)个节点组成的有限集合，满足以下条件之一：
1. n=0，称为空树
2. n>0，有且仅有一个特定的根节点，其余节点可分为m个互不相交的子集，每个子集又是一棵树（子树）

**二叉树定义**:
二叉树是每个节点最多有两个子节点的树结构，分别称为左子树和右子树。

**形式化描述**:
```
二叉树 T 要么是空树 nil，要么是包含以下部分的三元组:
  T = (root, left, right)
其中:
  root: 根节点存储的数据
  left: 左子树（也是二叉树）
  right: 右子树（也是二叉树）
```

**直观解释**:
树结构就像家族谱系：
- 根节点 = 祖先
- 子节点 = 后代
- 叶节点 = 无后代的成员
- 深度 = 辈分代数
- 高度 = 家族历史长度

**关键要点**:
- 树是非线性结构，适合表示层次关系
- 二叉树是最常用的树形变体
- 平衡树保证操作的对数时间复杂度
- 树遍历策略（前序、中序、后序、层序）各有应用场景

### 1.2 属性

| 属性 | 描述 | 公式/备注 |
|------|------|-----------|
| 深度 | 节点到根的路径长度 | 根深度为0 |
| 高度 | 节点到叶子的最长路径 | 叶子高度为0 |
| 度数 | 节点的子节点数 | 二叉树节点度数∈{0,1,2} |
| 层次 | 深度+1 | 根在第1层 |
| 满二叉树 | 每层节点数达到最大 | 第k层最多2^(k-1)个节点 |
| 完全二叉树 | 除最后一层外都是满的 | 可用数组高效存储 |

**性质总结**:
1. **节点数与高度**: n个节点的二叉树高度至少为⌊log₂n⌋，最多为n-1
2. **叶节点数**: 满二叉树的叶节点数 = 内部节点数 + 1
3. **完全二叉树**: 可以用数组存储，节点i的左右子节点分别在2i和2i+1

### 1.3 变体

**二叉搜索树(BST)**:
- 定义: 左子树所有节点 < 根 < 右子树所有节点
- 与标准形式的区别: 有序性保证搜索效率
- 适用场景: 动态维护有序数据集

**平衡二叉树(AVL/红黑树)**:
- 定义: 左右子树高度差不超过1（AVL）或保持特定平衡条件（红黑树）
- 与标准形式的区别: 通过旋转保持平衡，保证O(log n)操作
- 适用场景: 需要稳定性能的数据库索引、集合实现

**B/B+树**:
- 定义: 多路平衡搜索树，每个节点有多个关键字和子树
- 与标准形式的区别: 专为磁盘存储设计，减少I/O次数
- 适用场景: 文件系统、数据库索引

## 三、形式化证明

### 3.1 核心定理

**定理 1** (二叉树节点数与高度): 具有n个节点的二叉树高度至少为⌊log₂n⌋。

**证明**:
```
设高度为h的二叉树最多有M(h)个节点。

第k层最多有2^k个节点，因此：
M(h) = 2^0 + 2^1 + ... + 2^h = 2^(h+1) - 1

若树有n个节点，则：
n ≤ M(h) = 2^(h+1) - 1
n + 1 ≤ 2^(h+1)
log₂(n+1) ≤ h+1
h ≥ log₂(n+1) - 1

由于h是整数，h ≥ ⌈log₂(n+1)⌉ - 1 = ⌊log₂n⌋。∎
```

**证明要点分析**:
1. **上界分析**: 先求高度h的满二叉树最多节点数
2. **不等式转换**: 从节点数限制反推高度下限
3. **取整处理**: 利用对数性质和整数约束得到紧致界

**直觉理解**:
要使树尽可能"矮"，必须让每个节点都有两个子节点（满二叉树）。即使如此，n个节点也需要至少log₂n层。

### 3.2 辅助引理

**引理 1** (BST中序遍历有序): 二叉搜索树的中序遍历结果是非递减序列。

*证明*:
```
对树高进行归纳。

基例: 空树或单节点树，显然有序。

归纳假设: 假设左右子树的中序遍历都有序。

归纳步骤: BST的中序遍历 = 左子树(所有<根) + [根] + 右子树(所有>根)
由归纳假设，左右子树内部有序；
由BST性质，左子树所有元素 < 根 < 右子树所有元素。
因此整体有序。∎
```

**引理 2** (AVL树高度): n个节点的AVL树高度为O(log n)。

*证明*:
```
设N(h)为高度h的AVL树最少节点数。

根节点的左右子树高度最多相差1，因此：
N(h) = N(h-1) + N(h-2) + 1

这与斐波那契数列类似，可以证明N(h) ≥ φ^h（φ≈1.618）。

因此：
n ≥ N(h) ≥ φ^h
h ≤ log_φ(n) = O(log n)。∎
```

## 四、算法/方法详解

### 4.1 算法描述

**BST搜索伪代码**:
```
算法: TREE-SEARCH(x, k)
输入: 树根x，搜索关键字k
输出: 关键字为k的节点，或NIL

1. if x == NIL or k == x.key then
2.     return x
3. end if
4. if k < x.key then
5.     return TREE-SEARCH(x.left, k)
6. else
7.     return TREE-SEARCH(x.right, k)
8. end if
```

**BST插入伪代码**:
```
算法: TREE-INSERT(T, z)
输入: 树T，待插入节点z

1. y = NIL
2. x = T.root
3. while x != NIL do
4.     y = x
5.     if z.key < x.key then
6.         x = x.left
7.     else
8.         x = x.right
9.     end if
10. end while
11. z.parent = y
12. if y == NIL then
13.     T.root = z
14. else if z.key < y.key then
15.     y.left = z
16. else
17.     y.right = z
18. end if
```

### 4.2 正确性分析

**BST搜索正确性**:

**不变式**: 每次递归调用时，若k在树中，则必在以x为根的子树中。

**证明**:
- **初始化**: x为根，若k在树中当然在根的子树中
- **保持**: 
  - 若k < x.key，由BST性质，k若在树中必在左子树
  - 若k > x.key，由BST性质，k若在树中必在右子树
- **终止**: x为NIL时k不在树中，或找到k时返回正确节点

### 4.3 复杂度分析

| 操作 | 普通BST | AVL/红黑树 | 完全二叉树 |
|------|---------|------------|------------|
| 搜索 | O(h) = O(n)最坏 | O(log n) | O(log n) |
| 插入 | O(h) = O(n)最坏 | O(log n) | O(log n)* |
| 删除 | O(h) = O(n)最坏 | O(log n) | O(log n)* |
| 遍历 | O(n) | O(n) | O(n) |

*完全二叉树通常不动态插入删除

**示例 2**: AVL树旋转

**问题描述**:

    5
   /
  3
 /

```

```

### 5.2 代码实现

**语言**: Python

```python
class TreeNode:
    def __init__(self, key, val=None):
        self.key = key
        self.val = val
        self.left = None
        self.right = None
        self.parent = None
        # AVL额外属性
        self.height = 1

class BST:
    def __init__(self):
        self.root = None
    
    def search(self, key):
        return self._search(self.root, key)
    
    def _search(self, node, key):
        if node is None or key == node.key:
            return node
        if key < node.key:
            return self._search(node.left, key)
        return self._search(node.right, key)
    
    def insert(self, key, val=None):
        new_node = TreeNode(key, val)
        self._insert_node(new_node)
    
    def _insert_node(self, z):
        y = None
        x = self.root
        while x is not None:
            y = x
            if z.key < x.key:
                x = x.left
            else:
                x = x.right
        z.parent = y
        if y is None:
            self.root = z
        elif z.key < y.key:
            y.left = z
        else:
            y.right = z
    
    def inorder(self):
        result = []
        self._inorder(self.root, result)
        return result
    
    def _inorder(self, node, result):
        if node is not None:
            self._inorder(node.left, result)
            result.append(node.key)
            self._inorder(node.right, result)
    
    def find_min(self):
        x = self.root
        while x and x.left:
            x = x.left
        return x


# AVL树实现
class AVLTree(BST):
    def _height(self, node):
        return node.height if node else 0
    
    def _update_height(self, node):
        if node:
            node.height = 1 + max(self._height(node.left),
                                  self._height(node.right))
    
    def _balance_factor(self, node):
        return self._height(node.left) - self._height(node.right)
    
    def _rotate_right(self, y):
        x = y.left
        T2 = x.right
        
        x.right = y
        y.parent = x
        y.left = T2
        if T2:
            T2.parent = y
        
        self._update_height(y)
        self._update_height(x)
        return x
    
    def _rotate_left(self, x):
        y = x.right
        T2 = y.left
        
        y.left = x
        x.parent = y
        x.right = T2
        if T2:
            T2.parent = x
        
        self._update_height(x)
        self._update_height(y)
        return y
    
    def _rebalance(self, node):
        self._update_height(node)
        balance = self._balance_factor(node)
        
        # 左重
        if balance > 1:
            if self._balance_factor(node.left) < 0:
                node.left = self._rotate_left(node.left)
            return self._rotate_right(node)
        
        # 右重
        if balance < -1:
            if self._balance_factor(node.right) > 0:
                node.right = self._rotate_right(node.right)
            return self._rotate_left(node)
        
        return node
    
    def insert(self, key, val=None):
        self.root = self._insert(self.root, key, val)
    
    def _insert(self, node, key, val):
        if node is None:
            return TreeNode(key, val)
        
        if key < node.key:
            node.left = self._insert(node.left, key, val)
            node.left.parent = node
        else:
            node.right = self._insert(node.right, key, val)
            node.right.parent = node
        
        return self._rebalance(node)


# 测试
if __name__ == "__main__":
    # BST测试
    bst = BST()
    for key in [5, 3, 7, 1, 4, 6, 8]:
        bst.insert(key)
    print("BST中序遍历:", bst.inorder())
    print("搜索4:", bst.search(4).key if bst.search(4) else "Not found")
    
    # AVL测试
    avl = AVLTree()
    for key in [1, 2, 3, 4, 5, 6, 7]:
        avl.insert(key)
    print("AVL中序遍历:", avl.inorder())
```

**代码说明**:
- `BST`: 基础二叉搜索树实现
- `AVLTree`: 继承BST，添加高度管理和旋转操作
- `_rebalance`: 根据平衡因子决定旋转策略

### 5.3 反例

**常见错误1**: 未处理空树情况
```python
# 错误代码
def find_min(node):
    while node.left:  # 若node为None会报错
        node = node.left
    return node
```
**错误原因**: 未检查node是否为None
**正确做法**: 添加空指针检查

**常见错误2**: 错误的删除逻辑
```python
# 错误代码
def delete(node, key):
    if key < node.key:
        delete(node.left, key)
    # 未更新node.left引用！
```
**错误原因**: 递归删除后未更新父节点的子树引用
**正确做法**: `node.left = delete(node.left, key)`

## 六、习题

### 6.1 理解题 (L1)

1. **遍历顺序** [难度⭐]
   
   对以下树，写出前序、中序、后序遍历结果：
   ```
       1
      / \
     2   3
    / \
   4   5
   ```
   
   <details>
   <summary>解答</summary>
   
   - **前序**（根-左-右）: 1, 2, 4, 5, 3
   - **中序**（左-根-右）: 4, 2, 5, 1, 3
   - **后序**（左-右-根）: 4, 5, 2, 3, 1
   
   </details>

2. **BST性质** [难度⭐]
   
   若BST中某节点值为x，其左子树中的最大值是什么？
   
   <details>
   <summary>解答</summary>
   
   左子树中的最大值是x的前驱（predecessor）。
   若左子节点存在，则是左子树的最右节点；
   否则需要向上追溯到第一个小于x的祖先。
   
   </details>

### 6.2 应用题 (L2-L3)

1. **最近的公共祖先(LCA)** [难度⭐⭐]
   
   在BST中找出两个节点的最近公共祖先。
   
   <details>
   <summary>解答</summary>
   
   ```python
   def lca(root, p, q):
       while root:
           if p.key < root.key and q.key < root.key:
               root = root.left
           elif p.key > root.key and q.key > root.key:
               root = root.right
           else:
               return root
       return None
   ```
   
   利用BST性质：若p、q分别位于当前节点两侧（或其中之一等于当前节点），则当前节点即为LCA。
   
   </details>

2. **判断平衡二叉树** [难度⭐⭐]
   
   给定二叉树，判断它是否是高度平衡的（左右子树高度差不超过1）。
   
   <details>
   <summary>解答</summary>
   
   ```python
   def is_balanced(root):
       def check(node):
           if not node:
               return 0, True
           
           left_height, left_balanced = check(node.left)
           right_height, right_balanced = check(node.right)
           
           height = 1 + max(left_height, right_height)
           balanced = (left_balanced and right_balanced and 
                      abs(left_height - right_height) <= 1)
           
           return height, balanced
       
       return check(root)[1]
   ```
   
   自底向上计算高度并检查平衡性，每个节点只访问一次，O(n)。
   
   </details>

### 6.3 证明题 (L4-L5)

1. **BST删除正确性** [难度⭐⭐⭐]
   
   证明BST删除操作中，用后继节点替换被删节点后仍保持BST性质。
   
   <details>
   <summary>解答</summary>
   
   **证明**:
   
   设删除节点x，其有两个子节点，后继为y（右子树的最小节点）。
   
   需要证明替换后：
   1. 左子树所有节点 < y
   2. 右子树所有节点 > y
   
   由后继定义：
   - y是x右子树的最小节点，因此y > 左子树所有节点（因为它们 < x < y）
   - y < x右子树的其他所有节点（因为y是最小）
   
   因此替换后BST性质保持。∎
   
   </details>

### 7.2 实际案例

**案例**: Linux内核CFS调度器

**背景**:
Linux的完全公平调度器(CFS)需要高效地管理可运行进程。

**应用方式**:
使用红黑树维护进程的虚拟运行时间(vruntime)。
- 最左侧节点 = vruntime最小的进程（下一个运行）
- 插入/删除: O(log n)
- 查找最小: O(1)（缓存最左节点）

**效果**:
- O(log n)的调度决策时间
- 公平性保证（vruntime最小的先运行）

### 7.3 跨领域联系

**与编译原理的联系**:
抽象语法树(AST)是编译器的核心数据结构，用于语法分析和代码生成。

**与机器学习的联系**:
决策树、随机森林等算法直接基于树结构，用于分类和回归任务。

## 八、多维对比

### 8.1 方法对比

| 数据结构 | 搜索 | 插入 | 删除 | 有序遍历 | 适用场景 |
|----------|------|------|------|----------|----------|
| 无序数组 | O(n) | O(1)* | O(n) | O(n log n) | 不常搜索 |
| 有序数组 | O(log n) | O(n) | O(n) | O(n) | 静态数据 |
| 链表 | O(n) | O(1) | O(1) | O(n) | 频繁插入删除 |
| BST | O(h) | O(h) | O(h) | O(n) | 动态有序数据 |
| AVL/红黑树 | O(log n) | O(log n) | O(log n) | O(n) | 需要保证性能 |
| 哈希表 | O(1)平均 | O(1)平均 | O(1)平均 | O(n log n) | 无需有序性 |

*均摊或特定位置

### 8.2 决策建议

**何时选择BST**:
- 需要动态维护有序数据集
- 需要范围查询（区间查找）
- 内存有限，无法接受哈希表的额外开销

**何时选择平衡树**:
- 数据分布可能偏斜
- 需要保证最坏情况性能
- 实现复杂度可接受

**决策流程图**:
```
需要维护有序性？
├── 否 → 哈希表（O(1)平均操作）
└── 是 → 数据规模固定？
          ├── 是 → 排序数组（简单、缓存友好）
          └── 否 → 数据分布均匀？
                    ├── 是 → 普通BST（简单实现）
                    └── 否 → 平衡树（AVL/红黑树）
```

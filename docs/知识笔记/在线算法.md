## 一、核心概念

### 1.1 定义

**正式定义**:
**在线算法**是在输入逐个到达时必须立即做出不可撤销决策的算法。

**离线算法**: 可以等待所有输入到达后再做决策。

**竞争比(Competitive Ratio)**: 在线算法性能的核心度量。

对于最小化问题，算法$ALG$是$c$-竞争的，如果：

$$ALG(\sigma) \leq c \cdot OPT(\sigma) + \alpha$$

其中$\sigma$是任意输入序列，$OPT$是最优离线算法，$\alpha$是常数。

通常要求$\alpha = 0$（严格竞争）或$\alpha$与输入规模无关。

**直观解释**:
在线算法像是一场"无法回头的旅程"——你必须在不知道未来的情况下做决策。竞争分析回答：相对于"全知全能"的离线最优算法，我们最多会差多少倍？从缓存替换到 ski rental，在线算法无处不在。

### 1.2 属性

| 问题 | 最佳竞争比 | 下界 |
|------|------------|------|
| 滑雪租赁 | 2 | 2 |
| 分页(LRU) | k | k |
| 列表访问(MTF) | 2 | 2 |
| k服务器 | 2k-1 | k |
| 负载均衡 | 2 | 1.5+ |

### 1.3 分析技术

**势函数方法**: 定义势函数比较在线和离线状态
**对偶拟合**: 使用LP对偶分析
**随机化**: 随机选择可能优于确定性下界

## 三、形式化证明

### 3.1 核心定理

**定理 1** (LRU分页): LRU算法的竞争比为$k$。

**证明**:

```
将请求序列按k个不同页面划分为阶段。

每个阶段：
- OPT至少产生1次缺页（阶段首请求）
- LRU产生至多k次缺页（新阶段k个不同页面）

因此竞争比 <= k。

下界：存在序列使LRU产生k次缺页而OPT仅1次。∎
```

**定理 2** (滑雪租赁): 购买阈值$b$次租赁的最优竞争比为2。

*证明*: 设滑雪天数为$T$。若$T < b$，在线租赁花费$T$，最优也$T$。若$T \geq b$，在线花费$2b$，最优购买花费$b$。竞争比=2。∎

## 四、算法详解

### 4.1 滑雪租赁

**问题**: 不知道滑雪天数，每天可租($1)或买($b)。

**算法**: 租b-1次，第b次买。

**竞争比**: 2

### 4.2 分页算法

**LRU**: 替换最久未使用
**FIFO**: 替换最早进入
**LFU**: 替换使用最少

**竞争比**: LRU和FIFO都是$k$（缓存大小）。

## 五、示例与实例

### 5.2 代码实现

```python
# LRU缓存实现（带竞争分析统计）
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()
        self.misses = 0
        self.requests = 0

    def get(self, key):
        self.requests += 1
        if key not in self.cache:
            self.misses += 1
            return -1
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key, value):
        self.requests += 1
        if key in self.cache:
            self.cache.move_to_end(key)
        else:
            self.misses += 1
            if len(self.cache) >= self.capacity:
                self.cache.popitem(last=False)
        self.cache[key] = value

    def competitive_ratio(self, opt_misses):
        """计算竞争比"""
        if opt_misses == 0:
            return float('inf') if self.misses > 0 else 1
        return self.misses / opt_misses


# 滑雪租赁算法
class SkiRental:
    def __init__(self, buy_cost):
        self.b = buy_cost
        self.rental_days = 0
        self.bought = False

    def ski(self):
        """滑雪一天，返回当日花费"""
        if self.bought:
            return 0  # 已购买，无额外费用

        self.rental_days += 1
        if self.rental_days < self.b:
            return 1  # 继续租
        else:
            self.bought = True
            return self.b  # 第b天购买

    def total_cost(self, actual_days):
        """计算给定实际天数的总花费"""
        if actual_days < self.b:
            return actual_days  # 一直租
        else:
            return self.b - 1 + self.b  # 租b-1次，然后买


# 列表访问(Move-To-Front)
class MoveToFront:
    def __init__(self, items):
        self.list = list(items)
        self.access_cost = 0

    def access(self, item):
        """访问元素，移到前端"""
        idx = self.list.index(item)
        self.access_cost += idx + 1  # 访问成本=位置
        # 移到前端
        self.list.pop(idx)
        self.list.insert(0, item)
        return idx

    def get_cost(self):
        return self.access_cost


# 负载均衡（贪心在线算法）
class OnlineLoadBalancing:
    def __init__(self, m_machines):
        self.m = m_machines
        self.load = [0] * m_machines

    def assign_job(self, job_time):
        """在线分配作业到负载最轻的机器"""
        min_load_machine = min(range(self.m), key=lambda i: self.load[i])
        self.load[min_load_machine] += job_time
        return min_load_machine

    def makespan(self):
        """完成时间 = 最大负载"""
        return max(self.load)

    def competitive_ratio(self, opt_makespan):
        """竞争比上界 2 - 1/m"""
        return self.makespan() / opt_makespan if opt_makespan > 0 else 1
```

## 六、习题

### 6.1 理解题

1. **为什么LRU竞争比是k** [难度⭐]

<details>
<summary>解答</summary>

将请求序列按k个不同页面划分为阶段。每个阶段LRU最多k次缺页（k个不同页面），而OPT至少1次。竞争比<=k。

</details>

1. **ski rental下界2** [难度⭐⭐]

<details>
<summary>解答</summary>

设算法在第d天买。若实际T < d，在线花费d-1，最优T，比值(d-1)/T可以任意大。若实际T >= d，在线花费d-1+b，最优min(T,b)。当T=b时，在线d-1+b，最优b，令d=b得比值(2b-1)/b -> 2。

</details>

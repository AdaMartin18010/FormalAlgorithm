## 一、核心概念

### 1.1 定义

**正式定义**:
编译器的**前端**（Front End）将源代码转换为中间表示（IR），包含两个主要阶段：

1. **词法分析（Lexical Analysis）**: 将字符流转换为记号（Token）序列
   - 输入: 源程序字符串
   - 输出: $(token, attribute)$ 序列
   - 实现: 有限自动机（FA/DFA/NFA）

2. **语法分析（Syntax Analysis / Parsing）**: 将记号序列转换为抽象语法树（AST）
   - 输入: Token序列
   - 输出: 抽象语法树或解析树
   - 实现: 上下文无关文法（CFG），LL/LR分析器

**上下文无关文法（CFG）定义**:
CFG是四元组 $G = (V, \Sigma, R, S)$：

- $V$: 非终结符集合
- $\Sigma$: 终结符集合（与$V$不相交）
- $R$: 产生式规则集合 $A \to \alpha$，其中 $A \in V$，$\alpha \in (V \cup \Sigma)^*$
- $S \in V$: 开始符号

**直观解释**:
词法分析就像将一句话分成单词并标注词性："我/代词 吃饭/动词"；语法分析就像分析句子结构："主语(我) + 谓语(吃饭)"，构建树形结构表示语句的层次关系。

**关键要点**:

- **词法分析**: 正则语言，用DFA高效识别
- **语法分析**: 上下文无关语言，用栈结构处理嵌套
- **分离设计**: 词法分析简化文法设计，提高模块化
- **错误恢复**: 编译器友好的错误报告机制

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 词法层次 | 正则语言（Type-3） | 可由DFA识别 |
| 语法层次 | 上下文无关语言（Type-2） | 可由PDA识别 |
| 歧义性 | 文法可能有多个解析树 | 需要消除歧义或优先规则 |
| 递归性 | 支持嵌套结构 | 表达式、语句块等 |

**性质总结**:

1. **确定性**: DFA每次转移唯一确定
2. **封闭性**: 正则语言对并、交、补、连接、闭包封闭
3. **泵引理**: 用于证明语言非正则/非上下文无关
4. **最左/最右推导**: 对应不同的分析策略

### 1.3 变体

**LL分析（自上而下）**:

- 定义: Left-to-right, Leftmost derivation
- 与LR的区别: 预测性分析，需消除左递归
- 适用场景: 手写递归下降解析器、简单语言

**LR分析（自下而上）**:

- 定义: Left-to-right, Rightmost derivation in reverse
- 与LL的区别: 移进-归约，更强大，可处理更多文法
- 适用场景: 自动生成解析器（Yacc/Bison）

**GLR/GLL分析**:

- 定义: 通用分析器，处理所有CFG
- 与LL/LR的区别: 支持歧义文法，指数级复杂度
- 适用场景: 自然语言处理、复杂编程语言

## 三、形式化证明

### 3.1 核心定理

**定理 1 (NFA与DFA等价性)**: 对于每个NFA $N$，存在一个DFA $D$ 使得 $L(N) = L(D)$。

**证明**:

设 $N = (Q, \Sigma, \delta, q_0, F)$ 为NFA。

构造DFA $D = (Q', \Sigma, \delta', q_0', F')$：

- $Q' = 2^Q$（$Q$的幂集）
- $q_0' = \text{ε-closure}(q_0)$
- $\delta'(R, a) = \text{ε-closure}(\bigcup_{r \in R} \delta(r, a))$
- $F' = \{R \in Q' : R \cap F \neq \emptyset\}$

对输入字符串 $w = a_1 a_2 \ldots a_n$：

NFA接受 $w$ 当且仅当存在状态序列 $q_0, q_1, \ldots, q_n$ 使得 $q_n \in F$ 且 $q_{i+1} \in \delta(q_i, a_{i+1})$。

DFA的状态 $R_i = \delta'(q_0', a_1 \ldots a_i)$ 恰好是NFA在处理前缀后可达的所有状态。

因此 $w \in L(N) \iff R_n \cap F \neq \emptyset \iff R_n \in F' \iff w \in L(D)$。

∎

**证明要点分析**:

1. 子集构造法的核心思想
2. ε-闭包处理NFA的ε转移
3. 状态爆炸问题（DFA状态数指数级增长）

**定理 2 (LL(1)文法判定)**: 文法 $G$ 是LL(1)的当且仅当对于每个非终结符 $A$ 和任意两个不同产生式 $A \to \alpha \mid \beta$：

1. $\text{FIRST}(\alpha) \cap \text{FIRST}(\beta) = \emptyset$，且
2. 若 $\epsilon \in \text{FIRST}(\alpha)$，则 $\text{FIRST}(\beta) \cap \text{FOLLOW}(A) = \emptyset$

*证明*: 这两个条件确保分析器能唯一确定使用哪个产生式。∎

### 3.2 辅助引理

**引理 1 (泵引理 for 正则语言)**: 若 $L$ 是正则语言，则存在泵长度 $p$，使得对任意 $w \in L$ 且 $|w| \geq p$，可将 $w = xyz$ 满足：

1. $|xy| \leq p$
2. $|y| > 0$
3. $\forall i \geq 0: xy^i z \in L$

*证明*: DFA有有限状态，长字符串必有重复状态。∎

**引理 2 (消除左递归)**: 任何CFG可转换为等价的无左递归文法。

*构造*: 对于 $A \to A\alpha \mid \beta$，改写为：

- $A \to \beta A'$
- $A' \to \alpha A' \mid \epsilon$
∎

## 四、算法/方法详解

### 4.1 算法描述

**子集构造法（NFA → DFA）**:

```
算法: Subset-Construction
输入: NFA N = (Q, Σ, δ, q0, F)
输出: 等价DFA D = (Q', Σ, δ', q0', F')

1. q0' ← ε-closure({q0})
2. Q' ← {q0'}
3. Worklist ← {q0'}
4. while Worklist ≠ ∅ do
5.     T ← 从Worklist移除一个状态
6.     for each a ∈ Σ do
7.         U ← ε-closure(∪_{t∈T} δ(t, a))
8.         if U ∉ Q' then
9.             Q' ← Q' ∪ {U}
10.            Worklist ← Worklist ∪ {U}
11.        δ'(T, a) ← U
12.    end for
13. end while
14. F' ← {R ∈ Q' : R ∩ F ≠ ∅}
15. return D
```

**递归下降解析（LL）**:

```
算法: Recursive-Descent-Parse
输入: Token序列, 文法G

// 为每个非终结符A编写解析函数
function parse_A():
    if lookahead ∈ FIRST(α1) then
        // 使用产生式 A → α1
        for each symbol X in α1:
            if X is terminal then
                match(X)
            else
                parse_X()
    else if lookahead ∈ FIRST(α2) then
        // 使用产生式 A → α2
        ...
    else if ε ∈ FIRST(α1) and lookahead ∈ FOLLOW(A) then
        // 使用 ε 产生式
        return
    else
        error()
```

**LR(0)分析表构造**:

```
算法: LR0-Items-Construction
输入: 增广文法 G'
输出: LR(0)项集规范族 C

1. C ← {Closure({[S' → •S]})}
2. repeat
3.     for each I ∈ C do
4.         for each X ∈ (V ∪ Σ) do
5.             J ← Goto(I, X)
6.             if J ≠ ∅ and J ∉ C then
7.                 C ← C ∪ {J}
8. until C不再变化
9. return C

Closure(I):
    J ← I
    repeat
        for each [A → α•Bβ] ∈ J do
            for each B → γ do
                J ← J ∪ {[B → •γ]}
    until J不再变化
    return J

Goto(I, X):
    J ← ∅
    for each [A → α•Xβ] ∈ I do
        J ← J ∪ {[A → αX•β]}
    return Closure(J)
```

**流程图**:

```
源代码 → 词法分析 → 记号流 → 语法分析 → AST → 语义分析
              ↑              ↑
           正则表达式      上下文无关文法
              ↓              ↓
              NFA          LL/LR分析器
              ↓
            DFA
              ↓
          词法分析器
```

### 4.2 正确性分析

**子集构造的正确性**:

- 不变式: DFA状态对应NFA状态的ε-闭包
- 终止: 状态有限（幂集）
- 接受条件: 包含NFA接受状态的DFA状态为接受状态

**LL分析的正确性**:

- 最左推导对应递归下降的深度优先遍历
- FIRST/FOLLOW集确保预测的唯一性

**LR分析的正确性**:

- 项集编码了所有可能的推导状态
- 移进-归约决策基于栈状态和向前看符号

### 4.3 复杂度分析

**词法分析复杂度**:

- NFA到DFA: 最坏 $O(2^n \cdot m)$，$n$为NFA状态数，$m$为字母表大小
- DFA模拟: $O(|w|)$ 时间，$O(1)$ 每字符

**语法分析复杂度**:

- LL(1)分析: $O(|w|)$ 时间，$O(|V| + |\Sigma|)$ 空间
- LR分析: $O(|w|)$ 时间，$O(|G|)$ 空间（$|G|$为文法大小）
- 通用CFG分析: $O(|w|^3)$（CYK算法）

**示例 2: 表达式文法的LL(1)分析**

**文法**:

```
E  → T E'
E' → + T E' | ε
T  → F T'
T' → * F T' | ε
F  → ( E ) | id
```

**FIRST集**:

- FIRST(E) = FIRST(T) = FIRST(F) = {(, id}
- FIRST(E') = {+, ε}
- FIRST(T') = {*, ε}

**FOLLOW集**:

- FOLLOW(E) = FOLLOW(E') = {), $}
- FOLLOW(T) = FOLLOW(T') = {+, ), $}

**预测分析表**:

| 非终结符 | id | + | * | ( | ) | $ |
|----------|----|---|---|---|---|---|
| E | E→TE' | | | E→TE' | | |
| E' | | E'→+TE' | | | E'→ε | E'→ε |

### 5.2 代码实现

**Python词法分析器实现**:

```python
import re
from enum import Enum, auto
from typing import List, Tuple, Optional
from dataclasses import dataclass

class TokenType(Enum):
    NUMBER = auto()
    PLUS = auto()
    MINUS = auto()
    MULT = auto()
    DIV = auto()
    LPAREN = auto()
    RPAREN = auto()
    ID = auto()
    EOF = auto()

@dataclass
class Token:
    type: TokenType
    value: str
    line: int
    column: int

class Lexer:
    """基于正则表达式的词法分析器"""

    # 记号定义: (类型, 正则表达式)
    TOKEN_SPECS = [
        (TokenType.NUMBER, r'\d+(?:\.\d+)?'),
        (TokenType.PLUS, r'\+'),
        (TokenType.MINUS, r'-'),
        (TokenType.MULT, r'\*'),
        (TokenType.DIV, r'/'),
        (TokenType.LPAREN, r'\('),
        (TokenType.RPAREN, r'\)'),
        (TokenType.ID, r'[a-zA-Z_][a-zA-Z0-9_]*'),
        ('SKIP', r'[ \t\n]+'),
        ('MISMATCH', r'.'),
    ]

    def __init__(self, text: str):
        self.text = text
        self.pos = 0
        self.line = 1
        self.column = 1
        # 编译联合正则表达式
        self.token_regex = '|'.join(f'(?P<{name.name if isinstance(name, Enum) else name}>{pattern})'
                                    for name, pattern in self.TOKEN_SPECS)
        self.regex = re.compile(self.token_regex)

    def error(self, msg: str):
        raise SyntaxError(f'{msg} at line {self.line}, column {self.column}')

    def get_next_token(self) -> Token:
        """获取下一个记号"""
        if self.pos >= len(self.text):
            return Token(TokenType.EOF, '', self.line, self.column)

        match = self.regex.match(self.text, self.pos)
        if not match:
            self.error(f'Invalid character: {self.text[self.pos]!r}')

        token_type = match.lastgroup
        value = match.group()

        # 更新行列位置
        newline_count = value.count('\n')
        if newline_count > 0:
            self.line += newline_count
            self.column = len(value) - value.rfind('\n')
        else:
            self.column += len(value)

        self.pos = match.end()

        # 处理跳过字符
        if token_type == 'SKIP':
            return self.get_next_token()
        elif token_type == 'MISMATCH':
            self.error(f'Unexpected character: {value!r}')

        return Token(TokenType[token_type], value, self.line,
                    self.column - len(value))

    def tokenize(self) -> List[Token]:
        """分析所有记号"""
        tokens = []
        while True:
            token = self.get_next_token()
            tokens.append(token)
            if token.type == TokenType.EOF:
                break
        return tokens

# 递归下降语法分析器
class Parser:
    """LL(1)递归下降解析器"""

    def __init__(self, tokens: List[Token]):
        self.tokens = tokens
        self.pos = 0
        self.current = tokens[0]

    def error(self, msg: str):
        raise SyntaxError(f'{msg} at {self.current}')

    def eat(self, token_type: TokenType):
        """消费预期类型的记号"""
        if self.current.type == token_type:
            self.pos += 1
            self.current = self.tokens[self.pos]
        else:
            self.error(f'Expected {token_type}, got {self.current.type}')

    def parse(self):
        """开始符号 E"""
        return self.expr()

    def expr(self):
        """E → T ((PLUS | MINUS) T)*"""
        node = self.term()
        while self.current.type in (TokenType.PLUS, TokenType.MINUS):
            op = self.current.type
            self.eat(op)
            right = self.term()
            node = BinOp(node, op, right)
        return node

    def term(self):
        """T → F ((MULT | DIV) F)*"""
        node = self.factor()
        while self.current.type in (TokenType.MULT, TokenType.DIV):
            op = self.current.type
            self.eat(op)
            right = self.factor()
            node = BinOp(node, op, right)
        return node

    def factor(self):
        """F → NUMBER | ID | LPAREN expr RPAREN"""
        token = self.current
        if token.type == TokenType.NUMBER:
            self.eat(TokenType.NUMBER)
            return Num(token)
        elif token.type == TokenType.ID:
            self.eat(TokenType.ID)
            return Var(token)
        elif token.type == TokenType.LPAREN:
            self.eat(TokenType.LPAREN)
            node = self.expr()
            self.eat(TokenType.RPAREN)
            return node
        else:
            self.error(f'Unexpected token: {token}')

# AST节点
@dataclass
class Num:
    token: Token
    value: float = None
    def __post_init__(self):
        self.value = float(self.token.value)

@dataclass
class Var:
    token: Token
    name: str = None
    def __post_init__(self):
        self.name = self.token.value

@dataclass
class BinOp:
    left: object
    op: TokenType
    right: object

# 使用示例
if __name__ == "__main__":
    # 词法分析
    code = "(1 + 2) * 3 + x"

    tokens = lexer.tokenize()

    parser = Parser(tokens)
    ast = parser.parse()

```

**Rust实现 (简化版)**:

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum Token {
    Number(f64),
    Plus,
    Minus,
    Multiply,
    Divide,
    LParen,
    RParen,
    Identifier(String),
    EOF,
}

pub struct Lexer<'a> {
    input: &'a str,
    position: usize,
    current_char: Option<char>,
}

impl<'a> Lexer<'a> {
    pub fn new(input: &'a str) -> Self {
        let mut lexer = Lexer {
            input,
            position: 0,
            current_char: input.chars().next(),
        };
        lexer
    }

    fn advance(&mut self) {
        self.position += 1;
        self.current_char = self.input[self.position..].chars().next();
    }

    fn skip_whitespace(&mut self) {
        while let Some(ch) = self.current_char {
            if ch.is_whitespace() {
                self.advance();
            } else {
                break;
            }
        }
    }

    fn number(&mut self) -> f64 {
        let mut result = String::new();
        while let Some(ch) = self.current_char {
            if ch.is_digit(10) || ch == '.' {
                result.push(ch);
                self.advance();
            } else {
                break;
            }
        }
        result.parse().unwrap()
    }

    fn identifier(&mut self) -> String {
        let mut result = String::new();
        while let Some(ch) = self.current_char {
            if ch.is_alphanumeric() || ch == '_' {
                result.push(ch);
                self.advance();
            } else {
                break;
            }
        }
        result
    }

    pub fn next_token(&mut self) -> Token {
        self.skip_whitespace();

        match self.current_char {
            None => Token::EOF,
            Some(ch) => match ch {
                '+' => { self.advance(); Token::Plus }
                '-' => { self.advance(); Token::Minus }
                '*' => { self.advance(); Token::Multiply }
                '/' => { self.advance(); Token::Divide }
                '(' => { self.advance(); Token::LParen }
                ')' => { self.advance(); Token::RParen }
                ch if ch.is_digit(10) => Token::Number(self.number()),
                ch if ch.is_alphabetic() => Token::Identifier(self.identifier()),
                _ => panic!("Unknown character: {}", ch),
            }
        }
    }
}

fn main() {
    let mut lexer = Lexer::new("1 + 2 * (3 - 4)");
    loop {
        let token = lexer.next_token();
        println!("{:?}", token);
        if token == Token::EOF {
            break;
        }
    }
}
```

### 5.3 反例

**常见错误**:

**错误1: 正则表达式歧义**

```
IF      = if
ID      = [a-z]+

输入: "ifx"
```

**错误原因**: 可能匹配为IF+ID或完整ID
**正确做法**: 使用最长匹配（maximal munch）规则

**错误2: 左递归导致无限循环**

```python
def parse_expr():
    node = parse_expr()  # 直接左递归！
    if current == PLUS:
        eat(PLUS)
        node = BinOp(node, parse_term())
    return node
```

**错误原因**: 递归下降无法处理直接左递归
**正确做法**: 消除左递归或使用LR分析

## 六、习题

### 6.1 理解题

1. **解释**: 为什么词法分析和语法分析通常分离为两个阶段，而不是直接对字符流进行上下文无关分析？

   <details>
   <summary>解答</summary>

   分离的原因：
   1. **简化设计**: 正则语言更简单，DFA高效且易实现
   2. **模块化**: 可独立修改词法或语法规则
   3. **性能**: DFA比PDA更快（线性时间vs需要栈操作）
   4. **错误恢复**: 词法错误和语法错误可分别处理
   5. **可移植性**: 词法规则与具体语言无关的部分可复用

   直接CFG分析的问题：
   - 需要处理大量无关注释、空白符
   - 符号表查找效率低
   - 错误定位不准确
   </details>

### 6.2 应用题

1. **设计**: 为一个简单的SQL子集设计词法分析器和语法分析器，支持SELECT、FROM、WHERE子句。

   <details>
   <summary>解答</summary>

   ```python
   # 词法规则
   TOKEN_SPECS = [
       ('SELECT', r'(?i)SELECT'),
       ('FROM', r'(?i)FROM'),
       ('WHERE', r'(?i)WHERE'),
       ('AND', r'(?i)AND'),
       ('OR', r'(?i)OR'),
       ('ID', r'[a-zA-Z_][a-zA-Z0-9_]*'),
       ('NUMBER', r'\d+'),
       ('STRING', r"'[^']*'"),
       ('OPERATOR', r'[=<>!]+'),
       ('COMMA', r','),
       ('SKIP', r'[ \t\n]+'),
   ]

   # 语法规则 (EBNF)
   query      ::= SELECT column_list FROM table_name WHERE condition
   column_list::= column (COMMA column)* | STAR
   table_name ::= ID
   condition  ::= expr ((AND | OR) expr)*
   expr       ::= ID OPERATOR value
   value      ::= NUMBER | STRING | ID
   ```

   </details>

### 6.3 证明题

1. **证明**: 语言 $L = \{a^n b^n c^n : n \geq 0\}$ 不是上下文无关的。

   <details>
   <summary>解答</summary>

   使用CFG的泵引理。

   假设 $L$ 是上下文无关的，设泵长度为 $p$。

   选择 $s = a^p b^p c^p \in L$，$|s| = 3p \geq p$。

   由泵引理，$s = uvxyz$ 满足：
   1. $|vxy| \leq p$
   2. $|vy| > 0$
   3. $\forall i \geq 0: uv^i xy^i z \in L$

   由于 $|vxy| \leq p$，$vxy$ 不能同时包含 $a, b, c$。

   情况1: $vxy$ 只含 $a$ 或只含 $a,b$
   - 泵后 $a, b, c$ 数量不等，不在 $L$ 中

   情况2: $vxy$ 只含 $b, c$
   - 同理矛盾

   因此 $L$ 不是上下文无关的。
   </details>

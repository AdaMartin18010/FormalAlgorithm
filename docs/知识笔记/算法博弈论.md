## 一、核心概念

### 1.1 定义

**正式定义**:
算法博弈论是研究**策略性互动**中算法问题的学科，核心关注：
1. **计算复杂性**: 博弈均衡的计算难度
2. **算法设计**: 高效计算均衡的算法
3. **机制设计**: 设计激励相容的规则
4. **效率分析**: 均衡与社会最优的差距（Price of Anarchy）

**基本博弈模型**:
| 模型 | 描述 | 典型应用 |
|------|------|----------|
| **策略式博弈** | 同时行动，收益矩阵 | 拍卖、资源分配 |
| **扩展式博弈** | 序贯行动，博弈树 | 棋类、谈判 |
| **贝叶斯博弈** | 不完全信息 | 拍卖、信号传递 |
| **演化博弈** | 种群动态 | 生物学、学习 |

**纳什均衡 (Nash Equilibrium)**:
策略组合 $s^* = (s_1^*, ..., s_n^*)$ 是纳什均衡，如果对所有玩家 $i$:
$$u_i(s_i^*, s_{-i}^*) \geq u_i(s_i, s_{-i}^*), \quad \forall s_i \in S_i$$

即：给定他人策略，没有任何玩家能单方面偏离获益。

**直观解释**:
想象交通路线选择：
- 每个司机（玩家）选择路线（策略）以最小化通勤时间
- 纳什均衡：没有任何司机能通过单方面改道而更快
- 但均衡可能不是社会最优（可能存在囚徒困境）

**关键要点**:
- 纳什均衡普遍存在（混合策略），但计算可能困难
- 机制设计：设计规则使自私行为导向社会最优
- Price of Anarchy衡量市场失效程度

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 存在性 | 有限博弈必有混合策略纳什均衡 | Nash, 1950 |
| 唯一性 | 一般博弈可能有多个均衡 | 选择困难 |
| PPAD完全性 | 计算纳什均衡的计算复杂性 | Daskalakis et al., 2009 |
| Price of Anarchy | 均衡与社会最优的比值 | Koutsoupias & Papadimitriou |

**性质总结**:
1. **均衡选择**: 多个均衡时如何预测结果
2. **学习动态**: 玩家如何收敛到均衡
3. **近似均衡**: $\epsilon$-纳什均衡的计算效率

### 1.3 变体

**零和博弈**:
- 定义: $\sum_i u_i = 0$，一方所得即另一方所失
- 特点: 存在多项式时间算法（线性规划）
- 应用: 安全博弈、对抗训练

**势博弈 (Potential Games)**:
- 定义: 存在势函数 $\Phi$ 使 $u_i(s_i', s_{-i}) - u_i(s_i, s_{-i}) = \Phi(s_i', s_{-i}) - \Phi(s_i, s_{-i})$
- 特点: 纯策略纳什均衡存在，最佳响应动态收敛
- 应用: 网络路由、资源分配

**拥塞博弈 (Congestion Games)**:
- 定义: 玩家选择资源，成本随使用者数量增加
- 特点: 是势博弈，PoA有界
- 应用: 交通、网络路由

## 三、形式化证明

### 3.1 核心定理

**定理 1** (纳什均衡存在性):
每个有限策略式博弈至少存在一个混合策略纳什均衡。

**证明** (Kakutani不动点定理):

定义最佳响应对应 $BR_i(s_{-i}) = \arg\max_{s_i} u_i(s_i, s_{-i})$。

定义整体最佳响应对应 $BR(s) = BR_1(s_{-1}) \times ... \times BR_n(s_{-n})$。

 Nash均衡即 $BR$ 的不动点。

**验证Kakutani条件**:
1. **定义域**: 策略空间是紧凸集（单纯形）
2. **非空性**: $BR(s)$ 非空（效用函数连续，紧集上极值可达）
3. **凸性**: $BR(s)$ 凸（效用拟凹）
4. **上半连续性**: 效用函数连续保证

由Kakutani不动点定理，$BR$ 有不动点，即纳什均衡存在。∎

**证明要点分析**:
1. **不动点视角**: 均衡是不动点，证明存在性转化为证明对应性质
2. **凸性关键**: 混合策略的凸组合仍是混合策略
3. **非构造性**: 证明不给出计算算法

### 3.2 辅助引理

**引理 1** (零和博弈与线性规划):
零和博弈的纳什均衡可通过线性规划在多项式时间求解。

**引理 2** (势博弈收敛):
势博弈中，最佳响应动态（异步）必定收敛到纯策略纳什均衡。

## 四、算法/方法详解

### 4.1  Lemke-Howson算法

用于计算两人博弈的纳什均衡（通过路径跟踪）。

**伪代码**:
```
算法: LemkeHowson
输入: 两人博弈 (A, B)
输出: 纳什均衡

1.  构造标记的单纯形图
2.  选择初始几乎完全标记顶点
3.  沿满足互补条件的边移动
4.  遇到完全标记顶点时停止
5.  返回对应的策略组合
```

**复杂度分析**:
- **最坏情况**: 指数时间
- **平均情况**: 多项式时间（Smoothed Analysis）

### 4.2 无憾学习 (No-Regret Learning)

玩家通过历史反馈学习，长期无憾保证收敛到均衡。

**Multiplicative Weights Update**:
```
算法: MWU
输入: 动作集 [n], 学习率 η

初始化: w_i(1) = 1 for all i

for t = 1 to T:
    选择动作 i with probability ∝ w_i(t)
    观察损失 l_i(t)
    
    for each i:
        w_i(t+1) = w_i(t) · (1 - η)^{l_i(t)}
        
// 平均策略收敛到相关均衡
```

**复杂度分析**:
- 每轮时间: $O(n)$
- 收敛到 $\epsilon$-均衡: $O(\log n / \epsilon^2)$ 轮

### 4.3 VCG机制

**Vickrey-Clarke-Groves机制**: 激励相容的社会福利最大化机制。

```
机制: VCG
输入: 玩家报价 b_i(v)

1.  选择结果 x* = argmax_x Σ_i b_i(x)
2.  玩家i的支付: p_i = Σ_{j≠i} b_j(x*_{-i}) - Σ_{j≠i} b_j(x*)
    
// x*_{-i} 是不含i时的最优结果
```

**性质**:
- **激励相容**: 真实报价是占优策略
- **社会福利最大化**: 选择社会最优结果
- **个体理性**: 参与不降低效用（适当条件下）

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 囚徒困境

**收益矩阵**:
```
               玩家B
           合作     背叛
玩家A 合作  (3, 3)  (0, 5)
      背叛  (5, 0)  (1, 1)
```

**分析**:
- 纳什均衡: (背叛, 背叛)，收益 (1, 1)
- 社会最优: (合作, 合作)，收益 (3, 3)
- **Price of Anarchy**: 3/1 = 3

**示例 2**: 路由博弈 (Pigou例子)

**问题**: 从s到t有两条路径：
- 路径1: 延迟 = 1（固定）
- 路径2: 延迟 = x（依赖流量x）

**均衡**: 所有流量走路径2，延迟 = 1
**最优**: 分流，一半走每条路径，平均延迟 = 0.75
**PoA**: 1 / 0.75 = 4/3

### 5.2 代码实现

**语言**: Python

```python
import numpy as np
from typing import List, Tuple, Dict, Callable
from scipy.optimize import linprog
import itertools


class NormalFormGame:
    """标准式博弈"""
    
    def __init__(self, payoff_matrices: List[np.ndarray]):
        """
        Args:
            payoff_matrices: 每个玩家的收益矩阵
        """
        self.n = len(payoff_matrices)  # 玩家数
        self.u = payoff_matrices
        self.strategies = payoff_matrices[0].shape
    
    def expected_payoff(self, player: int, strategies: List[np.ndarray]) -> float:
        """计算期望收益"""
        result = self.u[player].copy()
        for i, s in enumerate(strategies):
            if i != player:
                # 对i的维度求和，加权s
                axis = i if i < player else i - 1
                result = np.tensordot(result, s, axes=([axis], [0]))
        return float(np.dot(result.flatten(), strategies[player]))
    
    def best_response(self, player: int, opponent_strategies: List[np.ndarray]) -> np.ndarray:
        """计算最佳响应"""
        n_strategies = self.strategies[player]
        payoffs = np.zeros(n_strategies)
        
        for s in range(n_strategies):
            strat = np.zeros(n_strategies)
            strat[s] = 1.0
            temp_strats = opponent_strategies.copy()
            temp_strats.insert(player, strat)
            payoffs[s] = self.expected_payoff(player, temp_strats)
        
        best = np.argmax(payoffs)
        br = np.zeros(n_strategies)
        br[best] = 1.0
        return br


class ZeroSumGameSolver:
    """零和博弈求解器（线性规划）"""
    
    @staticmethod
    def solve(payoff_matrix: np.ndarray) -> Tuple[np.ndarray, np.ndarray, float]:
        """
        求解零和博弈
        
        Returns:
            (row_strategy, col_strategy, game_value)
        """
        m, n = payoff_matrix.shape
        
        # 行玩家（最大化）的LP
        # max v
        # s.t. Σ_i p_i * A[i,j] >= v for all j
        #      Σ_i p_i = 1
        #      p_i >= 0
        
        # 转换为标准形式
        c = np.zeros(m + 1)
        c[-1] = -1  # 最大化v
        
        A_ub = np.hstack([-payoff_matrix.T, np.ones((n, 1))])
        b_ub = np.zeros(n)
        
        A_eq = np.zeros((1, m + 1))
        A_eq[0, :m] = 1
        b_eq = np.array([1])
        
        bounds = [(0, None)] * m + [(None, None)]
        
        result = linprog(c, A_ub=A_ub, b_ub=b_ub, 
                        A_eq=A_eq, b_eq=b_eq, bounds=bounds, 
                        method='highs')
        
        if result.success:
            row_strategy = result.x[:m]
            game_value = result.x[-1]
            
            # 列玩家（最小化）的策略
            c_col = np.zeros(n + 1)
            c_col[-1] = 1
            
            A_ub_col = np.hstack([payoff_matrix, -np.ones((m, 1))])
            b_ub_col = np.zeros(m)
            
            A_eq_col = np.zeros((1, n + 1))
            A_eq_col[0, :n] = 1
            b_eq_col = np.array([1])
            
            bounds_col = [(0, None)] * n + [(None, None)]
            
            result_col = linprog(c_col, A_ub=A_ub_col, b_ub=b_ub_col,
                                A_eq=A_eq_col, b_eq=b_eq_col, bounds=bounds_col,
                                method='highs')
            
            col_strategy = result_col.x[:n] if result_col.success else None
            
            return row_strategy, col_strategy, game_value
        
        return None, None, None


class VCGMechanism:
    """VCG机制实现"""
    
    def __init__(self, outcomes: List, allocation_cost: Callable):
        """
        Args:
            outcomes: 可能的结果集合
            allocation_cost: 结果的成本函数
        """
        self.outcomes = outcomes
        self.cost = allocation_cost
    
    def allocate(self, valuations: List[Dict]) -> Tuple[any, List[float]]:
        """
        执行VCG机制
        
        Args:
            valuations: 每个玩家对每个结果的估值 [{outcome: value}]
            
        Returns:
            (选择的结果, 各玩家的支付)
        """
        n = len(valuations)
        
        # 选择社会最优结果
        best_outcome = None
        best_welfare = float('-inf')
        
        for outcome in self.outcomes:
            welfare = sum(v.get(outcome, 0) for v in valuations) - self.cost(outcome)
            if welfare > best_welfare:
                best_welfare = welfare
                best_outcome = outcome
        
        # 计算支付
        payments = []
        for i in range(n):
            # 不含i时的最优结果
            best_without_i = None
            best_welfare_without_i = float('-inf')
            
            for outcome in self.outcomes:
                welfare = sum(v.get(outcome, 0) for j, v in enumerate(valuations) 
                            if j != i) - self.cost(outcome)
                if welfare > best_welfare_without_i:
                    best_welfare_without_i = welfare
                    best_without_i = outcome
            
            # VCG支付
            others_value_best_without_i = sum(
                v.get(best_without_i, 0) for j, v in enumerate(valuations) if j != i
            )
            others_value_best = sum(
                v.get(best_outcome, 0) for j, v in enumerate(valuations) if j != i
            )
            
            payment = others_value_best_without_i - others_value_best
            payments.append(payment)
        
        return best_outcome, payments


class CongestionGame:
    """拥塞博弈"""
    
    def __init__(self, resources: List[str], 
                 cost_functions: Dict[str, Callable[[int], float]]):
        """
        Args:
            resources: 资源列表
            cost_functions: 每个资源的成本函数 c_e(n)
        """
        self.resources = resources
        self.cost_func = cost_functions
    
    def compute_cost(self, resource: str, num_users: int) -> float:
        """计算资源在给定用户数下的成本"""
        return self.cost_func[resource](num_users)
    
    def potential_function(self, allocation: Dict[str, int]) -> float:
        """
        计算Rosenthal势函数
        
        Φ = Σ_e Σ_{k=1}^{n_e} c_e(k)
        """
        potential = 0.0
        for resource, count in allocation.items():
            for k in range(1, count + 1):
                potential += self.compute_cost(resource, k)
        return potential
    
    def find_pure_nash(self, player_strategies: Dict[int, List[str]], 
                       max_iter: int = 1000) -> Dict[int, List[str]]:
        """
        通过最佳响应动态寻找纯策略纳什均衡
        
        Args:
            player_strategies: 每个玩家可用的资源集合
            
        Returns:
            纳什均衡分配
        """
        # 随机初始化
        allocation = {r: 0 for r in self.resources}
        player_alloc = {}
        
        for player, available in player_strategies.items():
            choice = np.random.choice(available)
            player_alloc[player] = [choice]
            allocation[choice] += 1
        
        # 最佳响应动态
        for _ in range(max_iter):
            improved = False
            
            for player, available in player_strategies.items():
                current_cost = sum(
                    self.compute_cost(r, allocation[r]) 
                    for r in player_alloc[player]
                )
                
                best_cost = current_cost
                best_choice = player_alloc[player]
                
                for r in available:
                    # 模拟切换到r
                    temp_allocation = allocation.copy()
                    for old_r in player_alloc[player]:
                        temp_allocation[old_r] -= 1
                    temp_allocation[r] = temp_allocation.get(r, 0) + 1
                    
                    new_cost = self.compute_cost(r, temp_allocation[r])
                    
                    if new_cost < best_cost:
                        best_cost = new_cost
                        best_choice = [r]
                        improved = True
                
                # 更新分配
                for old_r in player_alloc[player]:
                    allocation[old_r] -= 1
                player_alloc[player] = best_choice
                for new_r in best_choice:
                    allocation[new_r] = allocation.get(new_r, 0) + 1
            
            if not improved:
                break
        
        return player_alloc

    for player, choice in equilibrium.items():
        count[choice[0]] += 1
    
    print(f"均衡状态: top={count['top']}, bottom={count['bottom']}")
    print(f"top路径成本: {cost_func['top'](count['top'])}")
    print(f"bottom路径成本: {cost_func['bottom'](count['bottom'])}")
    print(f"(纳什均衡时两条路径成本应相等)")
```

**代码说明**:
- `NormalFormGame`: 标准式博弈表示
- `ZeroSumGameSolver`: 线性规划求解零和博弈
- `VCGMechanism`: VCG机制实现
- `CongestionGame`: 拥塞博弈与势函数

## 六、习题

### 6.1 理解题 (L1-L2)

1. **纳什均衡验证** [难度⭐]
   
   给定两人博弈收益矩阵，验证给定策略是否为纳什均衡。
   
   ```
   A的行策略: [0.6, 0.4]
   B的列策略: [0.3, 0.7]
   
   收益矩阵A:
   [[3, 0],
    [1, 2]]
   
   收益矩阵B:
   [[2, 1],
    [0, 3]]
   ```
   
   <details>
   <summary>解答</summary>
   
   **验证步骤**:
   1. 计算A的期望收益
   2. 检查是否存在偏离使A收益增加
   3. 对B重复
   
   **计算**:
   - A的期望收益: 0.6*(0.3*3 + 0.7*0) + 0.4*(0.3*1 + 0.7*2) = 0.54 + 0.68 = 1.22
   - 若A偏离到纯策略[1,0]: 0.3*3 + 0.7*0 = 0.9 < 1.22 ✓
   - 若A偏离到纯策略[0,1]: 0.3*1 + 0.7*2 = 1.7 > 1.22 ✗
   
   不是纳什均衡！
   
   </details>

2. **Price of Anarchy计算** [难度⭐⭐]
   
   计算Pigou例子的PoA（线性延迟函数 $c(x) = x$）。
   
   <details>
   <summary>解答</summary>
   
   **设置**:
   - 路径1: 延迟 = 1
   - 路径2: 延迟 = x（x为流量）
   - 总流量 = 1
   
   **均衡**:
   所有流量走路径2，延迟 = 1
   
   **社会最优**:
   最小化总延迟: min x·x + (1-x)·1 = x² + 1 - x
   求导: 2x - 1 = 0, x = 0.5
   最优延迟 = 0.25 + 0.5 = 0.75
   
   **PoA** = 1 / 0.75 = 4/3
   
   </details>

### 6.2 应用题 (L3)

3. **第二价格拍卖分析** [难度⭐⭐⭐]
   
   证明第二价格密封拍卖（Vickrey拍卖）中，真实报价是占优策略。
   
   <details>
   <summary>解答</summary>
   
   **设置**:
   - 最高报价者获胜
   - 支付第二高报价
   - 玩家i的估值为 $v_i$
   
   **证明**:
   考虑玩家i，设其他玩家的最高报价为 $b_{-i}^{max}$。
   
   **情况1**: $v_i > b_{-i}^{max}$
   - 真实报价: 获胜，支付 $b_{-i}^{max}$，效用 = $v_i - b_{-i}^{max} > 0$
   - 报高价: 仍获胜，支付相同，效用相同
   - 报低价 (< $b_{-i}^{max}$): 不获胜，效用 = 0
   
   **情况2**: $v_i < b_{-i}^{max}$
   - 真实报价: 不获胜，效用 = 0
   - 报高价 (> $b_{-i}^{max}$): 获胜，支付 $b_{-i}^{max}$，效用 = $v_i - b_{-i}^{max} < 0$
   
   因此真实报价最优。∎
   
   </details>

### 6.3 证明题 (L4-L5)

4. **平滑博弈的PoA界** [难度⭐⭐⭐⭐]
   
   定义 $(\lambda, \mu)$-平滑博弈并证明PoA ≤ $\lambda / (1 - \mu)$。
   
   <details>
   <summary>解答</summary>
   
   **定义**: 成本最小化博弈是 $(\lambda, \mu)$-平滑的，如果对任意策略组合 $s, s^*$:
   $$\sum_i c_i(s_i^*, s_{-i}) \leq \lambda \cdot C(s^*) + \mu \cdot C(s)$$
   
   其中 $C$ 是社会总成本。
   
   **证明**:
   设 $s$ 是纳什均衡，$s^*$ 是社会最优。
   
   由纳什均衡定义: $c_i(s) \leq c_i(s_i^*, s_{-i})$ 对所有 $i$。
   
   求和:
   $$C(s) = \sum_i c_i(s) \leq \sum_i c_i(s_i^*, s_{-i})$$
   
   由平滑性:
   $$\leq \lambda \cdot C(s^*) + \mu \cdot C(s)$$
   
   整理:
   $$(1 - \mu) C(s) \leq \lambda C(s^*)$$
   
   $$\frac{C(s)}{C(s^*)} \leq \frac{\lambda}{1 - \mu}$$
   
   </details>

5. **PPAD完全性** [难度⭐⭐⭐⭐⭐]
   
   解释为什么计算纳什均衡是PPAD完全的，并讨论其意义。
   
   <details>
   <summary>解答</summary>
   
   **PPAD类**: "Polynomial Parity Arguments on Directed graphs"，包含存在性由有向图奇偶性论证保证的问题。
   
   **纳什均衡属于PPAD**:
   - Lemke-Howson算法是路径跟踪算法
   - 在有向图上从已知点开始跟踪，必到达均衡
   - 属于PPAD
   
   **PPAD难性** (Daskalakis-Goldberg-Papadimitriou, 2009):
   通过从END-OF-LINE问题归约证明。
   
   **意义**:
   1. **计算困难**: 不太可能有多项式时间算法
   2. **但不NP难**: 与SAT不同，不是NP完全的证据
   3. **近似可能**: 近似均衡可在多项式时间找到
   4. **复杂性理论**: 位于P和NP之间的新复杂性类
   
   </details>

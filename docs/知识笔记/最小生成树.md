## 一、核心概念

### 1.1 定义

**正式定义**:
给定连通无向图G=(V, E)和权重函数w: E → R，**最小生成树**是G的一棵生成树T，使得T中所有边的权重之和最小。

**生成树**: 包含图中所有顶点的无环连通子图。

**MST性质**:

- 对于n个顶点的连通图，MST有n-1条边
- MST可能不唯一（当存在相同权重的边时）
- 总权重最小的生成树

**直观解释**:
想象连接一组城市的道路网络：

- 每个城市是一个顶点
- 道路是边，建设成本是权重
- MST就是连接所有城市的最小总成本道路方案
- 必须保证所有城市连通，且没有环路（避免浪费）

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 存在性 | 连通图必有MST | 不连通图有最小生成森林 |
| 唯一性 | 边权互异时唯一 | 否则可能有多个MST |
| 边数 | \|V\|-1 | 生成树的定义 |
| 最优子结构 | MST的子树是子图的MST | 可用于分治算法 |
| 贪心选择 | 局部最优导致全局最优 | Cut Property和Cycle Property |

**Cut Property**:
对于图的任意割(S, V-S)，跨越割的权重最小的边必属于某棵MST。

**Cycle Property**:
对于图中任意环，权重最大的边不属于任何MST。

### 1.3 算法变体

**Prim算法**:

- 从一个顶点开始，逐步扩展树
- 每次选择连接树与非树顶点的最小权重边
- 时间复杂度: O(E logV)（使用堆）

**Kruskal算法**:

- 按权重排序所有边
- 每次选择不形成环的最小权重边
- 使用并查集检测环
- 时间复杂度: O(E logE) = O(E logV)

**Boruvka算法**:

- 并行算法，同时添加多条安全边
- 时间复杂度: O(E logV)
- 适用于分布式实现

## 三、形式化证明

### 3.1 Cut Property证明

**定理** (Cut Property): 设(S, V-S)是图G的一个割，e是跨越该割的权重最小的边，则e属于G的某棵MST。

**证明**:

```
设T是G的一棵MST。

情况1: 如果e ∈ T，结论成立。

情况2: 如果e ∉ T，则T ∪ {e}包含一个环。
这个环必包含另一条跨越割的边e'（因为环从S到V-S再回到S）。

令T' = T ∪ {e} - {e'}。
由于w(e) ≤ w(e')，有w(T') ≤ w(T)。
又T是MST，所以w(T') = w(T)，T'也是MST且包含e。
```

### 3.2 Prim算法正确性

**定理**: Prim算法输出一棵MST。

**证明**:

```
循环不变式: 算法维护的树T是某棵MST的子集。

初始化: T = ∅，显然是任何MST的子集。

保持: 每次迭代选择连接T与V-T的最小权重边(u,v)。
考虑割(V(T), V-V(T))，(u,v)是跨越该割的最小权重边。
由Cut Property，(u,v)属于某棵MST。
因此T ∪ {(u,v)}仍是某棵MST的子集。

终止: |T| = |V|-1，T是一棵生成树且是MST的子集，故T是MST。
```

## 四、算法详解

### 4.1 Prim算法

```
算法: Prim(G, w, r)
输入: 图G，权重w，根顶点r
输出: MST的边集

1.  for each u ∈ V do
2.      key[u] = ∞
3.      π[u] = NIL
4.  end for
5.  key[r] = 0
6.  Q = V  // 优先队列
7.  while Q ≠ ∅ do
8.      u = Extract-Min(Q)
9.      for each v ∈ Adj[u] do
10.         if v ∈ Q and w(u,v) < key[v] then
11.             π[v] = u
12.             key[v] = w(u,v)
13.         end if
14.     end for
15. end while
```

### 4.2 Kruskal算法

```
算法: Kruskal(G, w)
输入: 图G，权重w
输出: MST的边集

1.  A = ∅
2.  for each v ∈ V do
3.      Make-Set(v)
4.  end for
5.  按权重非递减排序E
6.  for each (u,v) ∈ E (按排序顺序) do
7.      if Find-Set(u) ≠ Find-Set(v) then
8.          A = A ∪ {(u,v)}
9.          Union(u,v)
10.     end if
11. end for
12. return A
```

## 五、示例与代码

### 5.1 标准示例

**示例图**:

```
    2       3
A ------> B ------> C
| 4       | 1       | 5
|         |         |
v         v         v
D <------ E <------ F
    6       2
```

**Prim执行过程（从A开始）**:

| 步骤 | 当前顶点 | 加入边 | 权重 | MST边集 |
|------|----------|--------|------|---------|
| 1 | A | - | - | ∅ |
| 2 | B | (A,B) | 2 | {(A,B)} |
| 3 | E | (B,E) | 1 | {(A,B),(B,E)} |
| 4 | C | (B,C) | 3 | + (B,C) |
| 5 | F | (E,F) | 2 | + (E,F) |
| 6 | D | (A,D) | 4 | + (A,D) |

总权重: 2+1+3+2+4 = 12

### 5.2 Python实现

```python
import heapq
from typing import Dict, List, Set, Tuple


class UnionFind:
    """并查集"""
    def __init__(self):
        self.parent = {}
        self.rank = {}

    def make_set(self, x):
        self.parent[x] = x
        self.rank[x] = 0

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py:
            return
        if self.rank[px] < self.rank[py]:
            px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]:
            self.rank[px] += 1


def prim(graph: Dict[str, Dict[str, int]], start: str) -> List[Tuple[str, str, int]]:
    """Prim算法 - O(E logV)"""
    mst = []
    visited = {start}
    edges = []

    # 初始化：添加start的所有邻接边
    for neighbor, weight in graph[start].items():
        heapq.heappush(edges, (weight, start, neighbor))

    while edges and len(visited) < len(graph):
        weight, u, v = heapq.heappop(edges)

        if v in visited:
            continue

        visited.add(v)
        mst.append((u, v, weight))

        # 添加v的所有邻接边
        for neighbor, w in graph[v].items():
            if neighbor not in visited:
                heapq.heappush(edges, (w, v, neighbor))

    return mst


def kruskal(edges: List[Tuple[str, str, int]], vertices: Set[str]) -> List[Tuple[str, str, int]]:
    """Kruskal算法 - O(E logE)"""
    # 按权重排序
    edges = sorted(edges, key=lambda x: x[2])

    uf = UnionFind()
    for v in vertices:
        uf.make_set(v)

    mst = []
    for u, v, weight in edges:
        if uf.find(u) != uf.find(v):
            mst.append((u, v, weight))
            uf.union(u, v)

        if len(mst) == len(vertices) - 1:
            break

    return mst


# 测试
if __name__ == "__main__":
    # 邻接表表示
    graph = {
        'A': {'B': 2, 'D': 4},
        'B': {'A': 2, 'C': 3, 'E': 1},
        'C': {'B': 3, 'F': 5},
        'D': {'A': 4, 'E': 6},
        'E': {'B': 1, 'D': 6, 'F': 2},
        'F': {'C': 5, 'E': 2}
    }

    print("Prim算法结果:")
    mst_prim = prim(graph, 'A')
    total = sum(w for _, _, w in mst_prim)
    for u, v, w in mst_prim:
        print(f"  {u} -- {v}: {w}")
    print(f"总权重: {total}")

    # 边列表表示
    edges = [
        ('A', 'B', 2), ('A', 'D', 4),
        ('B', 'C', 3), ('B', 'E', 1),
        ('C', 'F', 5),
        ('D', 'E', 6),
        ('E', 'F', 2)
    ]
    vertices = {'A', 'B', 'C', 'D', 'E', 'F'}

    print("\nKruskal算法结果:")
    mst_kruskal = kruskal(edges, vertices)
    total = sum(w for _, _, w in mst_kruskal)
    for u, v, w in mst_kruskal:
        print(f"  {u} -- {v}: {w}")
    print(f"总权重: {total}")
```

## 六、习题

### 6.1 理解题

1. **Cut Property应用** [难度⭐]

   给定图的割，如何确定安全边？

   <details>
   <summary>解答</summary>
   跨越割的所有边中权重最小的边是安全边，必属于某棵MST。这是Prim和Kruskal算法的理论基础。
   </details>

### 6.2 应用题

1. **次小生成树** [难度⭐⭐⭐]

   如何求次小生成树（权重第二小的生成树）？

   <details>
   <summary>解答</summary>

   算法步骤：
   1. 先求MST，记录总权重W
   2. 对MST中每条边e，删除e后求MST'，记录权重
   3. 所有MST'中权重最小且>W的就是次小生成树

   复杂度: O(VE)，可用LCA优化到O(V²)或O(E logV)

   </details>

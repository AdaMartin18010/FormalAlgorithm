## 一、核心概念

### 1.1 定义

**正式定义**:
**信息熵（Shannon Entropy）**: 离散随机变量 $X$ 的熵定义为：
$$H(X) = -\sum_{i} p(x_i) \log_2 p(x_i)$$

表示描述 $X$ 所需的平均比特数。

**条件熵**: $H(Y|X) = \sum_{x} p(x) H(Y|X=x)$

**互信息**: $I(X;Y) = H(X) - H(X|Y) = H(Y) - H(Y|X)$

**信道容量**: $C = \max_{p(x)} I(X;Y)$

**信源编码定理**: 无损压缩的极限是 $H(X)$ 比特/符号。

**信道编码定理**: 当速率 $R < C$ 时，存在编码使得错误概率任意小。

**纠错码**: 向消息添加冗余以检测和纠正传输错误。
- **汉明距离**: $d(x, y) = $ 不同比特数
- **最小距离**: $d_{min} = \min_{c_1 \neq c_2} d(c_1, c_2)$
- **纠错能力**: $t = \lfloor (d_{min} - 1) / 2 \rfloor$

**直观解释**:
信息熵就像"不确定性的大小"——一个事件的熵越大，了解它所需的信息就越多。想象猜一个单词：如果只有两种可能且等概率，只需要1比特（是/否）；如果有1024种等可能，需要10比特。纠错码则像"保险单"：多发送一些冗余信息，即使部分损坏也能恢复原意。

**关键要点**:
- **熵的界限**: $0 \leq H(X) \leq \log |\mathcal{X}|$
- **数据压缩极限**: 无损压缩无法突破熵
- **信道容量**: 可靠通信的理论上限
- **码率与纠错权衡**: 更多冗余 → 更强纠错 → 更低有效速率

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 非负性 | $H(X) \geq 0$ | 零当且仅当确定 |
| 链式法则 | $H(X,Y) = H(X) + H(Y|X)$ | 条件熵分解 |
| 数据处理不等式 | $I(X;Z) \leq I(X;Y)$ | 信息不增 |
| 渐近等分性 | 典型序列集中概率 | 信源编码基础 |

**性质总结**:
1. **熵的凹性**: $H(p)$ 是概率分布的凹函数
2. **Fano不等式**: 估计误差下界
3. **Kraft不等式**: 前缀码存在的充要条件
4. **Hamming界**: 纠错码的理论界限

### 1.3 变体

**Huffman编码**:
- 定义: 最优前缀码，高频符号短编码
- 与香农码的区别: 实际最优，非渐进最优
- 适用场景: 已知概率分布的无损压缩

**算术编码**:
- 定义: 将整个消息编码为区间内的数
- 与Huffman的区别: 可接近熵极限，无整数比特限制
- 适用场景: 高熵源、自适应压缩

**LDPC码 / Turbo码**:
- 定义: 接近香农限的纠错码
- 与传统码的区别: 迭代译码，接近理论极限
- 适用场景: 通信系统、深空通信

## 三、形式化证明

### 3.1 核心定理

**定理 1 (信源编码定理 - Shannon, 1948)**: 对于独立同分布信源 $X$，
- 存在速率 $R > H(X)$ 的编码，错误概率 $P_e \to 0$
- 任何速率 $R < H(X)$ 的编码，错误概率 $P_e \not\to 0$

**证明**:

**典型序列**: 长度为 $n$ 的序列 $x^n$ 是 $\epsilon$-典型的，如果：
$$\left| -\frac{1}{n} \log p(x^n) - H(X) \right| < \epsilon$$

**渐近等分性（AEP）**: 当 $n \to \infty$，
$$P(x^n \in A_\epsilon^{(n)}) \to 1$$
$$|A_\epsilon^{(n)}| \leq 2^{n(H(X)+\epsilon)}$$

**可达性**:
1. 只编码典型序列，约 $2^{nH}$ 个
2. 分配索引需要 $nH$ 比特
3. 非典型序列错误概率趋于0

**逆命题**:
若 $R < H$，由鸽巢原理，必有某些典型序列共享码字，错误概率不趋于0。

∎

**证明要点分析**:
1. AEP将概率集中在典型集
2. 典型集大小约 $2^{nH}$
3. 编码本质是典型集的标记

**定理 2 (信道编码定理)**: 对于离散无记忆信道，可靠通信的充要条件是速率 $R < C = \max_{p(x)} I(X;Y)$。

*证明概要*: 随机编码论证 + 联合典型译码。∎

### 3.2 辅助引理

**引理 1 (熵的凹性)**: $H(p)$ 是概率向量 $p$ 的凹函数。

*证明*: $H(p) = -\sum p_i \log p_i$，二阶导数为负。∎

**引理 2 (数据处理不等式)**: 若 $X \to Y \to Z$ 形成马尔可夫链，则 $I(X;Z) \leq I(X;Y)$。

*证明*: $I(X;Y,Z) = I(X;Z) + I(X;Y|Z) = I(X;Y) + I(X;Z|Y)$，由马尔可夫性 $I(X;Z|Y) = 0$。∎

## 四、算法/方法详解

### 4.1 算法描述

**Huffman编码构造**:
```
算法: Huffman-Code
输入: 符号集 S = {s1,...,sn}，概率 p = {p1,...,pn}
输出: 前缀码 C

1. 为每个符号创建叶节点，权重为概率
2. while 节点数 > 1 do
3.     选择两个权重最小的节点 x, y
4.     创建新节点 z，权重为 w(x) + w(y)
5.     z.left = x, z.right = y
6.     将z加入集合
7. end while
8. 
9. // 从根生成码字
10. DFS遍历树，左分支加0，右分支加1
11. return 叶节点的码字
```

**汉明码编码**:
```
算法: Hamming-Encode
输入: 4位数据 d = (d1,d2,d3,d4)
输出: 7位码字 c = (c1,...,c7)

// (7,4)汉明码
// 校验位位置: 1,2,4
// c3 = d1, c5 = d2, c6 = d3, c7 = d4

1. c3 = d1, c5 = d2, c6 = d3, c7 = d4
2. // 计算校验位
3. c1 = c3 ⊕ c5 ⊕ c7  // 覆盖位3,5,7
4. c2 = c3 ⊕ c6 ⊕ c7  // 覆盖位3,6,7
5. c4 = c5 ⊕ c6 ⊕ c7  // 覆盖位5,6,7
6. return (c1,c2,c3,c4,c5,c6,c7)

汉明码译码:
1. 计算校验子 s = (s1,s2,s4)
   s1 = c1 ⊕ c3 ⊕ c5 ⊕ c7
   s2 = c2 ⊕ c3 ⊕ c6 ⊕ c7
   s4 = c4 ⊕ c5 ⊕ c6 ⊕ c7
2. if s == 0 then
3.     无错误
4. else
5.     错误位置 = s（二进制转十进制）
6.     翻转该位
7. end if
```

**流程图**:
```
信源 → 熵计算 → 信源编码 → 信道编码 → 信道 → 译码 → 恢复
            ↓           ↓
        Huffman      汉明/LDPC
        算术编码     Turbo
```

### 4.2 正确性分析

**Huffman编码最优性**:
- 贪心选择最小概率符号合并
- 归纳证明产生最优前缀码
- 平均码长 $L \leq H(X) + 1$

**汉明码纠错能力**:
- 最小距离 $d_{min} = 3$
- 可纠正1位错误
- 可检测2位错误

### 4.3 复杂度分析

**Huffman编码**:
- 构造: $O(n \log n)$（使用堆）
- 编码: $O(m)$，$m$ 为消息长度

**汉明码**:
- 编码: $O(n)$
- 译码: $O(n)$
- 硬件实现: 线性反馈移位寄存器

## 五、示例与实例

### 5.1 标准示例

**示例 1: Huffman编码**

**符号概率**:
| 符号 | 概率 |
|------|------|
| a | 0.4 |
| b | 0.3 |
| c | 0.2 |
| d | 0.1 |

**构造过程**:
1. 合并 d(0.1) 和 c(0.2) → 节点(0.3)
2. 合并 b(0.3) 和 节点(0.3) → 节点(0.6)
3. 合并 a(0.4) 和 节点(0.6) → 根(1.0)

**编码结果**:
- a: 0
- b: 10
- c: 110
- d: 111

**平均码长**: $0.4 \times 1 + 0.3 \times 2 + 0.2 \times 3 + 0.1 \times 3 = 1.9$ 比特/符号

**熵**: $H = -0.4\log 0.4 - 0.3\log 0.3 - 0.2\log 0.2 - 0.1\log 0.1 \approx 1.846$

**示例 2: (7,4)汉明码**

**数据**: $d = (1, 0, 1, 1)$

**编码**:
- $c_3 = 1, c_5 = 0, c_6 = 1, c_7 = 1$
- $c_1 = 1 \oplus 0 \oplus 1 = 0$
- $c_2 = 1 \oplus 1 \oplus 1 = 1$
- $c_4 = 0 \oplus 1 \oplus 1 = 0$

**码字**: $(0, 1, 1, 0, 0, 1, 1)$

**传输错误**: 第5位翻转 → $(0, 1, 1, 0, 1, 1, 1)$

**译码**:
- $s_1 = 0 \oplus 1 \oplus 1 \oplus 1 = 1$
- $s_2 = 1 \oplus 1 \oplus 1 \oplus 1 = 0$
- $s_4 = 0 \oplus 1 \oplus 1 \oplus 1 = 1$
- 校验子 $(s_4, s_2, s_1) = (1, 0, 1)_2 = 5$ → 第5位错误

### 5.2 代码实现

**Python实现**:

```python
import heapq
from collections import defaultdict, Counter
import math

def huffman_code(symbols_probs):
    """
    构造Huffman编码
    
    symbols_probs: [(symbol, probability), ...]
    返回: {symbol: code}
    """
    # 优先队列: (概率, 计数器, 节点)
    heap = []
    counter = 0
    
    for symbol, prob in symbols_probs:
        heapq.heappush(heap, (prob, counter, symbol))
        counter += 1
    
    # 构建树
    while len(heap) > 1:
        p1, c1, left = heapq.heappop(heap)
        p2, c2, right = heapq.heappop(heap)
        
        node = (left, right)
        heapq.heappush(heap, (p1 + p2, counter, node))
        counter += 1
    
    # 生成编码
    codes = {}
    
    def generate_codes(node, code=""):
        if isinstance(node, str):
            codes[node] = code or "0"  # 单符号特殊情况
        else:
            generate_codes(node[0], code + "0")
            generate_codes(node[1], code + "1")
    
    if heap:
        _, _, root = heap[0]
        generate_codes(root)
    
    return codes

def calculate_entropy(probs):
    """计算熵"""
    entropy = 0
    for p in probs:
        if p > 0:
            entropy -= p * math.log2(p)
    return entropy

def calculate_average_length(symbols_probs, codes):
    """计算平均码长"""
    avg_len = 0
    for symbol, prob in symbols_probs:
        avg_len += prob * len(codes[symbol])
    return avg_len

# (7,4)汉明码
def hamming_encode_74(data):
    """编码4位数据为7位汉明码"""
    assert len(data) == 4
    d1, d2, d3, d4 = [int(b) for b in data]
    
    # 校验位
    c1 = d1 ^ d2 ^ d4
    c2 = d1 ^ d3 ^ d4
    c4 = d2 ^ d3 ^ d4
    
    # 码字: c1 c2 d1 c4 d2 d3 d4
    return [c1, c2, d1, c4, d2, d3, d4]

def hamming_decode_74(received):
    """译码并纠错"""
    assert len(received) == 7
    c = [int(b) for b in received]
    
    # 计算校验子
    s1 = c[0] ^ c[2] ^ c[4] ^ c[6]  # c1 ⊕ d1 ⊕ d2 ⊕ d4
    s2 = c[1] ^ c[2] ^ c[5] ^ c[6]  # c2 ⊕ d1 ⊕ d3 ⊕ d4
    s4 = c[3] ^ c[4] ^ c[5] ^ c[6]  # c4 ⊕ d2 ⊕ d3 ⊕ d4
    
    syndrome = s4 * 4 + s2 * 2 + s1
    
    if syndrome != 0:
        # 纠正错误
        c[syndrome - 1] ^= 1
    
    # 提取数据位
    data = [c[2], c[4], c[5], c[6]]
    return data, syndrome

# 使用示例
if __name__ == "__main__":
    # Huffman编码示例
    symbols = [('a', 0.4), ('b', 0.3), ('c', 0.2), ('d', 0.1)]
    codes = huffman_code(symbols)
    
    print("Huffman编码:")
    for symbol, prob in symbols:
        print(f"  {symbol}: {codes[symbol]} (概率 {prob})")
    
    entropy = calculate_entropy([p for _, p in symbols])
    avg_len = calculate_average_length(symbols, codes)
    print(f"\n熵: {entropy:.3f} bits")
    print(f"平均码长: {avg_len:.3f} bits")
    print(f"效率: {entropy/avg_len*100:.1f}%")
    
    # 汉明码示例
    print("\n(7,4)汉明码:")
    data = "1011"
    encoded = hamming_encode_74(data)
    print(f"数据: {data}")
    print(f"编码: {''.join(map(str, encoded))}")
    
    # 引入错误
    received = encoded.copy()
    received[4] ^= 1  # 第5位翻转
    print(f"接收(含错): {''.join(map(str, received))}")
    
    decoded, syndrome = hamming_decode_74(received)
    print(f"校验子: {syndrome} (错误位置: {syndrome if syndrome else '无'})")
    print(f"译码: {''.join(map(str, decoded))}")
```

## 5.3 反例
### 5.3 反例

**常见错误**:

**错误1: 忽视前缀码条件**
```python
# 错误：非前缀码
codes = {'a': '0', 'b': '01', 'c': '1'}
```

**错误原因**: 违反信道编码定理
**正确做法**: 速率 $R < C$

## 六、习题

### 6.1 理解题

1. **证明**: 对于二元信源，熵 $H(p) = -p\log p - (1-p)\log(1-p)$ 在 $p = 0.5$ 时取最大值1。

   <details>
   <summary>解答</summary>
   
   计算导数：
   $$\frac{dH}{dp} = -\log p + \log(1-p) = \log\frac{1-p}{p}$$
   
   令导数为0：
   $$\frac{1-p}{p} = 1 \Rightarrow p = 0.5$$
   
   二阶导数验证为负，故是最大值。
   
   $H(0.5) = -0.5\log 0.5 - 0.5\log 0.5 = 1$ bit。
   ∎
   </details>

### 6.2 应用题

1. **设计**: 设计一个LZ77压缩器的简化版本。

   <details>
   <summary>解答</summary>
   
   ```python
   def lz77_compress(data, window_size=1024):
       """
       LZ77压缩（简化版）
       
       输出: [(offset, length, next_char), ...]
       """
       result = []
       i = 0
       
       while i < len(data):
           # 搜索窗口内的最长匹配
           best_len = 0
           best_offset = 0
           
           search_start = max(0, i - window_size)
           
           for j in range(search_start, i):
               # 尝试从位置j匹配
               length = 0
               while (i + length < len(data) and 
                      data[j + length] == data[i + length] and
                      length < 255):  # 最大匹配长度
                   length += 1
               
               if length > best_len:
                   best_len = length
                   best_offset = i - j
           
           if best_len > 0:
               # 输出匹配
               next_char = data[i + best_len] if i + best_len < len(data) else ''
               result.append((best_offset, best_len, next_char))
               i += best_len + 1
           else:
               # 无匹配，输出字面量
               result.append((0, 0, data[i]))
               i += 1
       
       return result
   ```
   </details>

### 6.3 证明题

1. **证明**: Hamming界（球包装界）：对于 $q$ 元 $(n, M, d)$ 码，有
   $$M \leq \frac{q^n}{\sum_{i=0}^{t} \binom{n}{i}(q-1)^i}$$
   其中 $t = \lfloor (d-1)/2 \rfloor$。

   <details>
   <summary>解答</summary>
   
   **证明**:
   
   以每个码字为中心，半径为 $t$ 的球互不相交（否则距离 $\leq 2t < d$，矛盾）。
   
   每个球包含：
   $$V_q(n, t) = \sum_{i=0}^{t} \binom{n}{i}(q-1)^i$$
   个向量（选择 $i$ 个位置，每个位置 $q-1$ 种错误）。
   
   所有 $M$ 个球的并集包含于 $q^n$ 个可能向量：
   $$M \cdot V_q(n, t) \leq q^n$$
   
   因此：
   $$M \leq \frac{q^n}{\sum_{i=0}^{t} \binom{n}{i}(q-1)^i}$$
   ∎
   </details>

**背景**: 5G使用LDPC（数据信道）和Polar码（控制信道）

**应用方式**:
1. LDPC接近香农限，适合大数据块
2. Polar码低时延，适合控制信息

**效果**: 高频谱效率，低误码率

### 7.3 跨领域联系

**与机器学习的联系**:
变分推断、信息瓶颈理论使用信息论概念。

**与统计物理的联系**:
统计力学熵与信息熵数学形式相同，MaxEnt原理。

## 八、多维对比

### 8.1 方法对比

| 维度 | Huffman | 算术编码 | LZ77 | LDPC |
|------|---------|---------|------|------|
| 压缩比 | 好 | 更好 | 很好 | N/A |
| 速度 | 快 | 中等 | 快 | 快（硬件）|
| 复杂度 | 低 | 中 | 低 | 高 |
| 适应性 | 静态 | 自适应 | 自适应 | 固定码 |
| 用途 | 通用 | 高精度 | 通用 | 纠错 |

### 8.2 决策建议

**何时选择Huffman**:
- 已知概率分布
- 快速压缩/解压
- 资源受限环境

**何时选择LDPC**:
- 高可靠性通信
- 接近香农限
- 硬件实现可行

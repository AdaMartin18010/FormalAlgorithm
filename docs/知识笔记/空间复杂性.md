## 一、核心概念

### 1.1 定义

**正式定义**:

**空间复杂性类**: 设f: N → N，定义
$$\text{SPACE}(f(n)) = \{L : L \text{ 被使用} O(f(n)) \text{ 空间的 DTM 判定}\}$$
$$\text{NSPACE}(f(n)) = \{L : L \text{ 被使用} O(f(n)) \text{ 空间的 NTM 判定}\}$$

**PSPACE**: 多项式空间
$$\text{PSPACE} = \bigcup_{k \geq 1} \text{SPACE}(n^k)$$

**NPSPACE**: 非确定性多项式空间
$$\text{NPSPACE} = \bigcup_{k \geq 1} \text{NSPACE}(n^k)$$

**L** (LOGSPACE): 
$$\text{L} = \text{SPACE}(\log n)$$

**NL** (NLOGSPACE):
$$\text{NL} = \text{NSPACE}(\log n)$$

**关键要点**:
- Savitch定理: NSPACE(s(n)) ⊆ SPACE(s(n)²)，故PSPACE = NPSPACE
- 空间层次定理: 更多空间严格更强
- L ⊆ NL ⊆ P ⊆ NP ⊆ PSPACE ⊆ EXP
- 带可以被重用，空间比时间更"可复用"

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 时间-空间关系 | TIME(t) ⊆ SPACE(t) | 每步最多访问一个新单元 |
| 空间可复用 | 同一单元可多次读写 | 与时间的本质区别 |
| Savitch定理 | 非确定性到确定性的平方代价 | 对数空间下开放 |
|  Immerman-Szelepcsényi | NL = co-NL | 非确定对数空间对补封闭 |

### 1.3 变体

**离线图灵机**: 输入只读，不计入空间
**在线图灵机**: 输入可写，计入空间
**交替图灵机**: 引入全称/存在状态，与空间复杂性相关

## 三、形式化证明

### 3.1 Savitch定理

**定理**: 对s(n) ≥ log n，
$$\text{NSPACE}(s(n)) \subseteq \text{SPACE}(s(n)^2)$$

**证明概要**:

设NTM N使用s(n)空间。对配置图：
- 顶点数 ≤ 2^O(s(n))
- 可达性问题可用确定性搜索

关键思想：使用递归的"中间相遇"技术。

定义REACHABLE(c₁, c₂, k): 能否在k步内从c₁到c₂

递归式：
REACHABLE(c₁, c₂, k) = ∨_{c_m} [REACHABLE(c₁, c_m, k/2) ∧ REACHABLE(c_m, c₂, k/2)]

递归深度O(s(n))，每层需要O(s(n))空间存储配置，总空间O(s(n)²)。

**推论**: PSPACE = NPSPACE

### 3.2 Immerman-Szelepcsényi定理

**定理**: NL = co-NL

**证明概要**:

证明PATH（有向图可达性）的补在NL中。

关键：用非确定性"计数"可达顶点数。

设图有n个顶点，c_k为从s出发k步内可达的顶点数。

归纳证明：若已知c_k，可在NL中验证某顶点t不在k+1步可达集中。

通过非确定性猜测c_k个不同顶点并验证，实现补集的NL判定。

## 四、算法/方法详解

### 4.1 空间高效算法

**图可达性 (s-t PATH)**:
- NL算法：非确定性猜测路径，O(log n)空间存储当前顶点
- UL算法（无向图）：Reingold的确定性O(log n)空间算法

**2-SAT ∈ NL**:
- 将公式转化为有向图
- 检查矛盾对是否互相可达

### 4.2 复杂度分析

**空间与时间的关系**:
- TIME(t(n)) ⊆ SPACE(t(n))
- SPACE(s(n)) ⊆ TIME(2^O(s(n)))

**PSPACE算法的一般模式**:
- 深度优先搜索配置图
- 空间用于递归栈和当前配置

## 六、习题

### 6.1 理解题 (L1-L2)

1. **Savitch定理应用** [难度⭐⭐]
   
   解释为什么Savitch定理不意味着P = NP。
   
   <details>
   <summary>解答</summary>
   
   Savitch定理给出NSPACE(s(n)) ⊆ SPACE(s(n)²)。
   
   对于多项式空间，平方后仍是多项式，故PSPACE = NPSPACE。
   
   但对于多项式时间，平方后仍是多项式，不能得出P = NP，
   因为定理只适用于空间，不适用于时间。
   
   实际上，对时间而言，指数加速是不可能的（时间层次定理）。
   
   </details>

2. **空间复用** [难度⭐]
   
   证明：若L ∈ SPACE(s(n))，则L ∈ TIME(2^O(s(n)))。
   
   <details>
   <summary>解答</summary>
   
   使用s(n)空间的TM最多有2^O(s(n))个不同配置。
   若运行超过这么多步，必重复配置，进入无限循环。
   
   因此，等价地可在2^O(s(n))步内判定，通过模拟并检测循环。
   
   </details>

### 6.2 证明题 (L3-L4)

3. **PSPACE = NPSPACE** [难度⭐⭐⭐]
   
   用Savitch定理完整证明PSPACE = NPSPACE。
   
   <details>
   <summary>解答</summary>
   
   (⊇) 显然：确定性是非确定性的特例。
   
   (⊆) 设L ∈ NPSPACE，则存在k使L ∈ NSPACE(n^k)。
   由Savitch定理，L ∈ SPACE((n^k)²) = SPACE(n^{2k}) ⊆ PSPACE。
   
   故PSPACE = NPSPACE。
   
   </details>

4. **NL ⊆ P** [难度⭐⭐⭐]
   
   证明NL ⊆ P。
   
   <details>
   <summary>解答</summary>
   
   设L ∈ NL，则存在NTM N使用O(log n)空间判定L。
   
   配置数上界：2^O(log n) = n^O(1) = poly(n)
   
   模拟N的所有计算路径（使用BFS/DFS），时间多项式。
   
   故L ∈ P。
   
   </details>

## 一、核心概念

### 1.1 定义

**正式定义**:

计算几何是研究几何对象（点、线、多边形等）的算法设计与分析的学科，主要关注在离散和组合环境下的几何问题求解。

**基本几何元素的形式化定义**:

```
点 (Point): P = (x, y) ∈ ℝ² 或 (x, y, z) ∈ ℝ³

向量 (Vector): v⃗ = (vₓ, vᵧ) = P₂ - P₁ = (x₂-x₁, y₂-y₁)

直线 (Line): ax + by + c = 0 或参数形式 L(t) = P₀ + t·v⃗, t ∈ ℝ

线段 (Line Segment): S = {P₀ + t·v⃗ | t ∈ [0, 1]}

多边形 (Polygon): 由顶点序列 P₀, P₁, ..., Pₙ₋₁ 定义的有界平面区域
```

**直观解释**:

计算几何研究如何用计算机高效地处理几何问题。例如：判断两条道路是否相交、确定一个位置是否在某个区域内、找到一组城市中距离最近的两个城市等。这些问题在计算机图形学、机器人学、GIS（地理信息系统）等领域有广泛应用。

**关键要点**:

- **精度问题**: 浮点数运算可能引入误差，需要鲁棒的几何计算
- **退化情况**: 算法必须处理点共线、边重合等特殊情况
- **方向测试**: 大多数算法的基础是判断三个点的相对方向（顺时针/逆时针）
- **输出敏感性**: 某些算法的复杂度取决于输出大小而非仅输入规模

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 维度 | 2D平面或3D空间 | 大多数基础算法针对2D设计 |
| 退化度 | 输入是否包含共线、重合点 | 影响算法鲁棒性 |
| 凸性 | 区域是否为凸集 | 凸问题通常更容易解决 |
| 点集规模 | n = 点的数量 | 决定算法选择 |
| 输出规模 | h = 凸包顶点数等 | 某些算法是输出敏感的 |

**性质总结**:

1. **凸包性质**: 点集S的凸包CH(S)是包含S的最小凸集，任意两点连线都在CH(S)内
2. **DCEL性质**: 双向边连接表(Doubly Connected Edge List)保证平面图的对偶性可高效计算
3. **扫描线不变性**: 扫描线算法维护的扫描线状态只与当前扫描位置相交的几何对象有关

### 1.3 变体

**精确计算几何 (Exact Geometric Computing)**:

- 定义: 使用任意精度算术避免浮点误差
- 与标准形式的区别: 牺牲速度换取精度保证
- 适用场景: CAD/CAM、科学计算等对精度要求极高的领域

**随机化几何算法 (Randomized Geometric Algorithms)**:

- 定义: 使用随机采样来优化期望复杂度
- 与标准形式的区别: 最坏情况复杂度可能较差，但期望复杂度通常更优
- 适用场景: 高维空间、大规模点集处理

**增量式算法 (Incremental Algorithms)**:

- 定义: 逐点构建解，每次插入后更新结果
- 与标准形式的区别: 适合动态场景，支持在线插入
- 适用场景: 动态凸包、在线几何查询

## 三、形式化证明

### 3.1 核心定理

**定理 3.1 (凸包正确性)**: Andrew's单调链算法正确计算点集的凸包。

**证明**:

设输入点集为S，|S| = n。算法步骤如下：

1. 按x坐标（再按y坐标）排序所有点
2. 构建下凸包：从左到右扫描，维护栈，确保栈顶三点构成右转
3. 构建上凸包：从右到左扫描，同样维护栈
4. 合并上下凸包

**正确性论证**:

*引理3.1.1*: 下凸包构造过程中，栈中点始终保持凸性。

*证明*: 设栈中从底到顶为p₀, p₁, ..., pₖ。每次加入新点p时：

- 若cross(pₖ₋₁, pₖ, p) ≤ 0（右转或共线），弹出pₖ
- 重复直到cross > 0，然后将p入栈

由于每次弹出都是因为形成非左转（右转或共线），保证剩余点形成凸链。由归纳法，栈始终为凸链。

*引理3.1.2*: 最终下凸包包含S中所有"底部"边界点。

*证明*: 按x排序后，任何凸包边界上从左到右的点必然在下凸包中。算法保留所有可能属于凸包的点，移除的节点必然在凸包内部。

*定理证明*: 上下凸包合起来包含S的所有极值点。由于每个凸包都在极端侧，它们只在端点相交，合起来形成完整凸包。

**证明要点分析**:

1. **扫描顺序保证**: 按x排序确保我们不会遗漏任何可能的凸包顶点
2. **栈维护策略**: 栈的LIFO特性允许我们回溯检查局部凸性
3. **极值点覆盖**: 上下扫描保证覆盖所有方向上的极值点

**直觉理解**:
想象用橡皮筋套住所有钉子（点），凸包就是橡皮筋的形状。单调链算法就像从左右两侧分别"拉伸"橡皮筋，确保在每一步都保持凸性。

### 3.2 辅助引理

**引理 3.2 (叉积方向性)**: 对于三点A, B, C，叉积 cross(B-A, C-A) 的符号决定C相对于向量AB的方向。

*证明*:

设 A = (xₐ, yₐ), B = (xᵦ, yᵦ), C = (xᵧ, yᵧ)

向量 AB = (xᵦ-xₐ, yᵦ-yₐ) = (dx₁, dy₁)
向量 AC = (xᵧ-xₐ, yᵧ-yₐ) = (dx₂, dy₂)

叉积（z分量）: cross = dx₁·dy₂ - dy₁·dx₂

根据行列式的几何意义，这等于 |AB|·|AC|·sin(θ)，其中θ是从AB到AC的有向角。

- cross > 0: sin(θ) > 0，C在AB左侧（逆时针/左转）
- cross < 0: sin(θ) < 0，C在AB右侧（顺时针/右转）
- cross = 0: sin(θ) = 0，三点共线

**引理 3.3 (最近点对分界性质)**: 在一维情况下，最近点对的分治算法中，合并步骤只需检查跨越分界线的有限个点。

*证明*:

设δ为左右子问题的最小距离。对于分界线上的任意点，另一侧与之距离<δ的点必然位于一个δ×2δ的矩形区域内。由于同侧点距离≥δ，该区域内最多有6个点（鸽巢原理）。因此每个点只需检查常数个点。

## 四、算法/方法详解

### 4.1 方向判断算法

**伪代码**:

```
算法: Orientation
输入: 三点 p, q, r
输出: 方向 (0:共线, 1:顺时针, 2:逆时针)

1. val ← (q.y - p.y) × (r.x - q.x) - (q.x - p.x) × (r.y - q.y)
2. if val = 0 then return 0      ▷ 共线
3. if val > 0 then return 1      ▷ 顺时针（右转）
4. return 2                       ▷ 逆时针（左转）
```

**流程图**:

```
        ┌─────────────┐
        │  计算叉积值  │
        └──────┬──────┘
               │
         ┌─────┴─────┐
         ▼           ▼
    ┌────────┐  ┌──────────┐
    │ val=0? │  │ val>0?   │
    └───┬────┘  └────┬─────┘
        │            │
   ┌────┴────┐  ┌────┴─────┐
   ▼         ▼  ▼          ▼
┌──────┐ ┌──────┐ ┌──────────┐
│ 共线 │ │顺时针│ │  逆时针  │
│  0   │ │  1   │ │    2     │
└──────┘ └──────┘ └──────────┘
```

### 4.2 线段相交检测算法

**伪代码**:

```
算法: SegmentIntersection
输入: 线段 p1q1, p2q2
输出: 是否相交

1. o1 ← Orientation(p1, q1, p2)
2. o2 ← Orientation(p1, q1, q2)
3. o3 ← Orientation(p2, q2, p1)
4. o4 ← Orientation(p2, q2, q1)

5. if o1 ≠ o2 and o3 ≠ o4 then
6.     return TRUE                    ▷ 正常相交

7. ▷ 处理特殊情况（共线）
8. if o1 = 0 and OnSegment(p1, p2, q1) then return TRUE
9. if o2 = 0 and OnSegment(p1, q2, q1) then return TRUE
10. if o3 = 0 and OnSegment(p2, p1, q2) then return TRUE
11. if o4 = 0 and OnSegment(p2, q1, q2) then return TRUE

12. return FALSE
```

**正确性分析**:

**不变式**: 两条线段相交当且仅当每条线段跨越另一条线段所在的直线。

**证明**:

- **初始化**: 方向测试确定点相对于直线的位置
- **保持**: 一般情况（步骤6）检查严格跨越；特殊情况（步骤8-11）处理共线重叠
- **终止**: 所有可能相交情况都被覆盖

### 4.3 凸包算法对比

#### 4.3.1 Graham Scan

**伪代码**:

```
算法: GrahamScan
输入: 点集 P，|P| = n
输出: 凸包顶点列表

1. 找到y坐标最小的点 p₀（若有多个取最左）
2. 按相对于p₀的极角排序其余点
3. 初始化栈 S ← [p₀, p₁, p₂]
4. for i ← 3 to n-1 do
5.     while S顶部两点与pᵢ不构成左转 do
6.         S.pop()
7.     S.push(pᵢ)
8. return S
```

**复杂度分析**:

- 找最低点: O(n)
- 极角排序: O(n log n)
- 扫描过程: 每个点最多入栈一次、出栈一次 → O(n)
- **总时间**: O(n log n)
- **空间**: O(n)

#### 4.3.2 Andrew's Monotone Chain

**伪代码**:

```
算法: MonotoneChain
输入: 点集 P
输出: 凸包顶点列表

1. 按x坐标（再按y）排序点
2. if n ≤ 1 then return P

3. ▷ 构建下凸包
4. lower ← 空列表
5. for p in 排序后的P do
6.     while lower长度≥2 且 cross(lower[-2], lower[-1], p) ≤ 0 do
7.         lower.pop()
8.     lower.append(p)

9. ▷ 构建上凸包
10. upper ← 空列表
11. for p in reverse(排序后的P) do
12.     while upper长度≥2 且 cross(upper[-2], upper[-1], p) ≤ 0 do
13.         upper.pop()
14.     upper.append(p)

15. ▷ 合并（去掉重复端点）
16. return lower[0:-1] + upper[0:-1]
```

**复杂度分析**:

- 排序: O(n log n)
- 两次扫描: 各O(n)
- **总时间**: O(n log n)
- **空间**: O(n)
- **优势**: 无需计算三角函数，数值稳定性更好

#### 4.3.3 Jarvis March (Gift Wrapping)

**伪代码**:

```
算法: JarvisMarch
输入: 点集 P
输出: 凸包顶点列表

1. 找到最左下点 p₀
2. hull ← [p₀]
3. current ← p₀

4. repeat
5.     next ← P[0]
6.     for p in P do
7.         if p ≠ current and (next = current or
8.            Orientation(current, next, p) = 2) then
9.             next ← p
10.    if next = p₀ then break
11.    hull.append(next)
12.    current ← next
13. until next = p₀

14. return hull
```

**复杂度分析**:

- 外层循环h次（h为凸包顶点数）
- 内层循环n次
- **总时间**: O(nh)
- 最坏情况（所有点在凸包上）: O(n²)
- **适用**: h较小时的输出敏感场景

### 4.4 最近点对算法（分治）

**伪代码**:

```
算法: ClosestPair
输入: 点集 Px（按x排序）, Py（按y排序）
输出: 最近点对距离

1. if |P| ≤ 3 then
2.     return BruteForceClosest(P)

3. mid ← ⌊|P|/2⌋
4. Qx ← Px[0:mid], Rx ← Px[mid:]
5. 根据mid将Py分为Qy和Ry

6. δ₁ ← ClosestPair(Qx, Qy)
7. δ₂ ← ClosestPair(Rx, Ry)
8. δ ← min(δ₁, δ₂)

9. ▷ 构建中线条带
10. mid_x ← Px[mid].x
11. strip ← {p ∈ Py | |p.x - mid_x| < δ}

12. ▷ 检查条带内的点
13. for i ← 0 to |strip|-1 do
14.     for j ← i+1 to min(i+7, |strip|) do
15.         if dist(strip[i], strip[j]) < δ then
16.             δ ← dist(strip[i], strip[j])

17. return δ
```

**复杂度分析**:

- 递归式: T(n) = 2T(n/2) + O(n)
- 由主定理: **T(n) = O(n log n)**
- **空间**: O(n)
- **关键**: 条带内每个点只需检查其后最多7个点

### 4.5 扫描线算法基础

**基本概念**: 维护一条扫描线（通常为垂直线x = x₀）从左到右移动，只关心与扫描线相交的几何对象。

**事件点处理**:

```
算法: SweepLine
输入: 几何对象集合 S
输出: 根据问题定义的输出

1. 初始化事件队列 Q（通常是最小堆）
2. 初始化扫描线状态 T（通常是平衡二叉树）

3. while Q 非空 do
4.     event ← Q.extractMin()
5.     UpdateSweepLinePosition(event.x)
6.     HandleEvent(event, T, Q)
7.
8. return Output
```

**复杂度分析**:

- 事件数: O(n) 到 O(n²) 取决于问题
- 每次事件处理: O(log n)（使用平衡树）
- 总复杂度: O((n+k) log n)，k为输出规模

## 五、示例与实例

### 5.1 标准示例

**示例 5.1**: 凸包构造示例

**问题描述**:
给定点集 P = {(0,0), (1,1), (2,2), (2,0), (1,0), (1.5, 0.5), (0,2), (2,2)}

**解决过程**:

1. 按x排序: (0,0), (0,2), (1,0), (1,1), (1.5,0.5), (2,0), (2,2)
2. 下凸包构建:
   - 加入(0,0), (0,2) → cross检查
   - 加入(1,0): cross((0,0),(0,2),(1,0)) < 0, 弹出(0,2)
   - 继续构建得到下凸包: (0,0), (2,0), (2,2)
3. 上凸包构建（反向）:
   - 得到: (2,2), (0,2), (0,0)
4. 合并: (0,0), (2,0), (2,2), (0,2)

**结果**: 凸包为四边形 (0,0)-(2,0)-(2,2)-(0,2)

### 5.2 代码实现

#### Python实现

**基础几何原语**:

```python
from dataclasses import dataclass
from typing import List, Tuple, Optional
import math

@dataclass
class Point:
    x: float
    y: float

    def __lt__(self, other: 'Point') -> bool:
        return (self.x, self.y) < (other.x, other.y)

    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Point):
            return False
        return (self.x, self.y) == (other.x, other.y)

    def __hash__(self) -> int:
        return hash((self.x, self.y))

def cross(o: Point, a: Point, b: Point) -> float:
    """计算叉积 (OA × OB)"""
    return (a.x - o.x) * (b.y - o.y) - (a.y - o.y) * (b.x - o.x)

def dot(a: Point, b: Point) -> float:
    """点积"""
    return a.x * b.x + a.y * b.y

def dist_sq(a: Point, b: Point) -> float:
    """距离平方（避免开方）"""
    return (a.x - b.x)**2 + (a.y - b.y)**2

def orientation(p: Point, q: Point, r: Point) -> int:
    """
    判断三点方向
    返回: 0=共线, 1=顺时针, 2=逆时针
    """
    val = cross(p, q, r)
    if abs(val) < 1e-9:
        return 0
    return 1 if val > 0 else 2

def on_segment(p: Point, q: Point, r: Point) -> bool:
    """检查点q是否在线段pr上"""
    if orientation(p, q, r) != 0:
        return False
    return (min(p.x, r.x) <= q.x <= max(p.x, r.x) and
            min(p.y, r.y) <= q.y <= max(p.y, r.y))
```

**线段相交检测**:

```python
def segments_intersect(p1: Point, q1: Point, p2: Point, q2: Point) -> bool:
    """判断两条线段是否相交"""
    o1 = orientation(p1, q1, p2)
    o2 = orientation(p1, q1, q2)
    o3 = orientation(p2, q2, p1)
    o4 = orientation(p2, q2, q1)

    # 一般情况
    if o1 != o2 and o3 != o4:
        return True

    # 特殊情况（共线）
    if o1 == 0 and on_segment(p1, p2, q1): return True
    if o2 == 0 and on_segment(p1, q2, q1): return True
    if o3 == 0 and on_segment(p2, p1, q2): return True
    if o4 == 0 and on_segment(p2, q1, q2): return True

    return False
```

**凸包算法（Andrew's Monotone Chain）**:

```python
def convex_hull(points: List[Point]) -> List[Point]:
    """
    Andrew's Monotone Chain算法
    返回凸包顶点，按逆时针顺序
    """
    n = len(points)
    if n <= 1:
        return points[:]

    # 按x坐标排序（x相同按y）
    pts = sorted(set(points))

    # 构建下凸包
    lower = []
    for p in pts:
        while len(lower) >= 2 and cross(lower[-2], lower[-1], p) <= 0:
            lower.pop()
        lower.append(p)

    # 构建上凸包
    upper = []
    for p in reversed(pts):
        while len(upper) >= 2 and cross(upper[-2], upper[-1], p) <= 0:
            upper.pop()
        upper.append(p)

    # 合并（去掉重复端点）
    return lower[:-1] + upper[:-1]
```

**点与多边形关系（射线法）**:

```python
def point_in_polygon(point: Point, polygon: List[Point]) -> bool:
    """
    射线法判断点是否在多边形内
    从点向右发射水平射线，计算与边相交次数
    """
    n = len(polygon)
    if n < 3:
        return False

    x, y = point.x, point.y
    inside = False

    j = n - 1
    for i in range(n):
        xi, yi = polygon[i].x, polygon[i].y
        xj, yj = polygon[j].x, polygon[j].y

        # 检查边是否与射线相交
        if ((yi > y) != (yj > y)) and (x < (xj - xi) * (y - yi) / (yj - yi) + xi):
            inside = not inside
        j = i

    return inside

def point_in_polygon_winding(point: Point, polygon: List[Point]) -> bool:
    """
    环绕数法判断点是否在多边形内
    更鲁棒，适用于自相交多边形
    """
    n = len(polygon)
    winding = 0

    for i in range(n):
        j = (i + 1) % n
        pi, pj = polygon[i], polygon[j]

        if pi.y <= point.y:
            if pj.y > point.y and cross(point, pi, pj) > 0:
                winding += 1
        else:
            if pj.y <= point.y and cross(point, pi, pj) < 0:
                winding -= 1

    return winding != 0
```

```python
def polygon_area(polygon: List[Point]) -> float:
    """
    鞋带公式计算多边形有向面积
    正面积表示逆时针，负面积表示顺时针
    """
    n = len(polygon)
    if n < 3:
        return 0.0

    area = 0.0
    for i in range(n):
        j = (i + 1) % n
        area += polygon[i].x * polygon[j].y
        area -= polygon[j].x * polygon[i].y

    return area / 2.0

def polygon_area_absolute(polygon: List[Point]) -> float:
    """返回多边形面积的绝对值"""
    return abs(polygon_area(polygon))
```

**最近点对（分治）**:

```python
def closest_pair(points: List[Point]) -> Tuple[float, Point, Point]:
    """
    分治算法求最近点对
    返回: (最小距离平方, 点1, 点2)
    """
    n = len(points)
    if n <= 1:
        return (float('inf'), None, None)

    # 按x和y排序的点
    px = sorted(points, key=lambda p: p.x)
    py = sorted(points, key=lambda p: p.y)

    def closest_recursive(px: List[Point], py: List[Point]) -> Tuple[float, Point, Point]:
        n = len(px)

        if n <= 3:
            # 暴力求解
            best = float('inf')
            best_pair = (None, None)
            for i in range(n):
                for j in range(i + 1, n):
                    d = dist_sq(px[i], px[j])
                    if d < best:
                        best = d
                        best_pair = (px[i], px[j])
            return (best, best_pair[0], best_pair[1])

        # 分割
        mid = n // 2
        mid_point = px[mid]

        # 分割py
        pyl = [p for p in py if p.x <= mid_point.x]
        pyr = [p for p in py if p.x > mid_point.x]

        # 递归求解
        dl, pl1, pl2 = closest_recursive(px[:mid], pyl)
        dr, pr1, pr2 = closest_recursive(px[mid:], pyr)

        # 合并
        d = min(dl, dr)
        best_pair = (pl1, pl2) if dl <= dr else (pr1, pr2)

        # 构建条带
        strip = [p for p in py if abs(p.x - mid_point.x) < math.sqrt(d)]

        # 检查条带（只需检查后面最多7个点）
        for i in range(len(strip)):
            for j in range(i + 1, min(i + 8, len(strip))):
                if strip[j].y - strip[i].y >= math.sqrt(d):
                    break
                dst = dist_sq(strip[i], strip[j])
                if dst < d:
                    d = dst
                    best_pair = (strip[i], strip[j])

        return (d, best_pair[0], best_pair[1])

    return closest_recursive(px, py)
```

```rust
use std::cmp::Ordering;
use std::f64;

#[derive(Debug, Clone, Copy, PartialEq)]
pub struct Point {
    pub x: f64,
    pub y: f64,
}

impl Point {
    pub fn new(x: f64, y: f64) -> Self {
        Point { x, y }
    }

    pub fn dist_sq(&self, other: &Point) -> f64 {
        (self.x - other.x).powi(2) + (self.y - other.y).powi(2)
    }
}

impl Eq for Point {}

impl PartialOrd for Point {
    fn partial_cmp(&self, other: &Point) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}

impl Ord for Point {
    fn cmp(&self, other: &Point) -> Ordering {
        self.x
            .partial_cmp(&other.x)
            .unwrap()
            .then_with(|| self.y.partial_cmp(&other.y).unwrap())
    }
}

/// 叉积计算 (OA × OB)
pub fn cross(o: &Point, a: &Point, b: &Point) -> f64 {
    (a.x - o.x) * (b.y - o.y) - (a.y - o.y) * (b.x - o.x)
}

/// 方向判断: 0=共线, 1=顺时针, 2=逆时针
pub fn orientation(p: &Point, q: &Point, r: &Point) -> i32 {
    let val = cross(p, q, r);
    if val.abs() < 1e-9 {
        0
    } else if val > 0.0 {
        1
    } else {
        2
    }
}

/// 检查点q是否在线段pr上
pub fn on_segment(p: &Point, q: &Point, r: &Point) -> bool {
    if orientation(p, q, r) != 0 {
        return false;
    }
    q.x >= p.x.min(r.x) && q.x <= p.x.max(r.x) &&
    q.y >= p.y.min(r.y) && q.y <= p.y.max(r.y)
}

/// 线段相交检测
pub fn segments_intersect(p1: &Point, q1: &Point, p2: &Point, q2: &Point) -> bool {
    let o1 = orientation(p1, q1, p2);
    let o2 = orientation(p1, q1, q2);
    let o3 = orientation(p2, q2, p1);
    let o4 = orientation(p2, q2, q1);

    // 一般情况
    if o1 != o2 && o3 != o4 {
        return true;
    }

    // 特殊情况
    if o1 == 0 && on_segment(p1, p2, q1) { return true; }
    if o2 == 0 && on_segment(p1, q2, q1) { return true; }
    if o3 == 0 && on_segment(p2, p1, q2) { return true; }
    if o4 == 0 && on_segment(p2, q1, q2) { return true; }

    false
}

/// Andrew's Monotone Chain凸包算法
pub fn convex_hull(points: &[Point]) -> Vec<Point> {
    let n = points.len();
    if n <= 1 {
        return points.to_vec();
    }

    // 排序并去重
    let mut pts: Vec<Point> = points.to_vec();
    pts.sort();
    pts.dedup();

    if pts.len() <= 1 {
        return pts;
    }

    // 构建下凸包
    let mut lower = Vec::new();
    for &p in &pts {
        while lower.len() >= 2 &&
              cross(&lower[lower.len()-2], &lower[lower.len()-1], &p) <= 0.0 {
            lower.pop();
        }
        lower.push(p);
    }

    // 构建上凸包
    let mut upper = Vec::new();
    for &p in pts.iter().rev() {
        while upper.len() >= 2 &&
              cross(&upper[upper.len()-2], &upper[upper.len()-1], &p) <= 0.0 {
            upper.pop();
        }
        upper.push(p);
    }

    // 合并（去掉重复端点）
    lower.pop();
    upper.pop();
    lower.extend(upper);
    lower
}

/// 多边形面积（鞋带公式）
pub fn polygon_area(polygon: &[Point]) -> f64 {
    let n = polygon.len();
    if n < 3 {
        return 0.0;
    }

    let mut area = 0.0;
    for i in 0..n {
        let j = (i + 1) % n;
        area += polygon[i].x * polygon[j].y;
        area -= polygon[j].x * polygon[i].y;
    }
    area / 2.0
}

/// 射线法判断点是否在多边形内
pub fn point_in_polygon(point: &Point, polygon: &[Point]) -> bool {
    let n = polygon.len();
    if n < 3 {
        return false;
    }

    let (x, y) = (point.x, point.y);
    let mut inside = false;
    let mut j = n - 1;

    for i in 0..n {
        let (xi, yi) = (polygon[i].x, polygon[i].y);
        let (xj, yj) = (polygon[j].x, polygon[j].y);

        if ((yi > y) != (yj > y)) &&
           (x < (xj - xi) * (y - yi) / (yj - yi) + xi) {
            inside = !inside;
        }
        j = i;
    }

    inside
}

/// 暴力法求最近点对（用于小规模情况）
fn brute_force_closest(points: &[Point]) -> (f64, Option<Point>, Option<Point>) {
    let n = points.len();
    let mut best = f64::INFINITY;
    let mut best_pair = (None, None);

    for i in 0..n {
        for j in (i + 1)..n {
            let d = points[i].dist_sq(&points[j]);
            if d < best {
                best = d;
                best_pair = (Some(points[i]), Some(points[j]));
            }
        }
    }

    (best, best_pair.0, best_pair.1)
}

/// 最近点对（分治）
pub fn closest_pair(points: &[Point]) -> (f64, Option<Point>, Option<Point>) {
    let n = points.len();
    if n <= 1 {
        return (f64::INFINITY, None, None);
    }

    let mut px: Vec<Point> = points.to_vec();
    let mut py: Vec<Point> = points.to_vec();
    px.sort_by(|a, b| a.x.partial_cmp(&b.x).unwrap());
    py.sort_by(|a, b| a.y.partial_cmp(&b.y).unwrap());

    fn closest_recursive(px: &[Point], py: &[Point]) -> (f64, Option<Point>, Option<Point>) {
        let n = px.len();

        if n <= 3 {
            return brute_force_closest(px);
        }

        let mid = n / 2;
        let mid_point = px[mid];

        // 分割py
        let (pyl, pyr): (Vec<Point>, Vec<Point>) = py.iter()
            .partition(|&p| p.x <= mid_point.x);

        // 递归
        let (dl, pl1, pl2) = closest_recursive(&px[..mid], &pyl);
        let (dr, pr1, pr2) = closest_recursive(&px[mid..], &pyr);

        // 合并
        let (mut d, mut best_pair) = if dl <= dr {
            (dl, (pl1, pl2))
        } else {
            (dr, (pr1, pr2))
        };

        // 构建条带
        let strip: Vec<Point> = py.iter()
            .filter(|&p| (p.x - mid_point.x).abs() < d.sqrt())
            .cloned()
            .collect();

        // 检查条带
        for i in 0..strip.len() {
            for j in (i + 1)..strip.len().min(i + 8) {
                if strip[j].y - strip[i].y >= d.sqrt() {
                    break;
                }
                let dst = strip[i].dist_sq(&strip[j]);
                if dst < d {
                    d = dst;
                    best_pair = (Some(strip[i]), Some(strip[j]));
                }
            }
        }

        (d, best_pair.0, best_pair.1)
    }

    closest_recursive(&px, &py)
}
```

### 6.1 理解题

1. **为什么凸包算法的时间复杂度下界是Ω(n log n)？** [难度⭐⭐]

   *提示*: 思考排序与凸包的关系

   <details>
   <summary>解答</summary>

   可以通过归约证明：如果能用少于O(n log n)的时间计算凸包，那么可以用凸包来排序。给定n个数x₁, ..., xₙ，构造点集(xᵢ, xᵢ²)，这些点都在抛物线y=x²上。它们的凸包顶点按x坐标有序排列，因此提取凸包顶点即可得到排序结果。由于比较排序的下界是Ω(n log n)，凸包的下界也是Ω(n log n)。

   </details>

2. **解释为什么最近点对算法中每个点只需检查条带内后面的7个点？** [难度⭐⭐]

   <details>
   <summary>解答</summary>

   设δ为左右子问题的最小距离。条带是宽度为2δ的垂直区域。将该区域划分为δ/2 × δ/2的小方格，每个方格内最多有一个点（否则同侧距离<δ）。对于条带内任意点，与之距离<δ的点必然位于其上方2行×4列=8个方格内。但最左侧一列可能在同侧，所以最多检查7个点。

   </details>

### 6.2 应用题

1. **实现一个判断两个凸多边形是否相交的算法。** [难度⭐⭐⭐]

   <details>
   <summary>解答</summary>

   方法1：分离轴定理(SAT)。对于凸多边形，如果存在一条轴（垂直于任一边）使得两多边形在该轴上的投影不重叠，则不相交。检查所有可能的分离轴，若都重叠则相交。

   方法2：检查是否有边相交，或一个多边形包含另一个多边形的任一顶点。时间复杂度O(nm)。

   </details>

2. **给定平面上一组点，找到包含所有点的最小面积矩形。** [难度⭐⭐⭐⭐]

   *提示*: 最小矩形的一条边必然与凸包的一条边共线

   <details>
   <summary>解答</summary>

   算法步骤：
   1. 计算凸包O(n log n)
   2. 对于凸包的每条边作为底边：
      - 使用旋转卡壳法找到对踵点（最高点）
      - 找到最左、最右点
      - 计算矩形面积
   3. 返回最小面积

   时间复杂度：O(n)（旋转卡壳的均摊分析）

   </details>

### 6.3 证明题

1. **证明：对于平面n个点，凸包顶点数h的期望值是O(n^(1/3))（点在单位圆内均匀随机分布）。** [难度⭐⭐⭐⭐]

   <details>
   <summary>解答</summary>

   证明思路（Rényi-Sulanke, 1963）：

   对于单位圆内的均匀随机点，考虑点Pᵢ成为凸包顶点的概率。Pᵢ在凸包上当且仅当存在一条过Pᵢ的直线使得所有其他点都在其一侧。

   通过几何概率分析，点P在半径r处的成为凸包顶点的概率约为(1-r²)^(n-1)。

   积分计算期望值：
   E[h] = n × ∫₀¹ 2πr(1-r²)^(n-1) dr × (常数因子)

   通过变量替换u=r²，可得E[h] = O(n^(1/3))。

   </details>

### 7.2 实际案例

**案例**: 自动驾驶中的障碍物检测

**背景**:
自动驾驶车辆需要实时检测周围环境中的障碍物（车辆、行人、建筑物等），并计算安全行驶路径。

**应用方式**:

1. **点云处理**: 激光雷达产生的点云数据用凸包近似障碍物边界
2. **碰撞检测**: 使用分离轴定理检测车辆与障碍物的潜在碰撞
3. **路径规划**: 使用Voronoi图（基于计算几何）规划安全路径
4. **实时性**: 采用近似凸包算法在精度与速度间权衡

**效果**:

- 碰撞检测延迟从10ms降至2ms
- 路径规划准确率提升15%

### 7.3 跨领域联系

**与计算机图形学的联系**:
光栅化算法需要将几何图元（三角形）转换为像素，依赖于点在三角形内的判断。抗锯齿、纹理映射都需要精确的几何计算。

**与机器学习的联系**:

- **几何深度学习**: 在点云、网格数据上直接学习
- **支持向量机**: 最大间隔分类本质上是几何问题
- **降维**: PCA、t-SNE等涉及高维几何

## 八、多维对比

### 8.1 凸包算法对比

| 维度 | Graham Scan | Jarvis March | Andrew's Monotone Chain |
|------|-------------|--------------|------------------------|
| 时间复杂度 | O(n log n) | O(nh) | O(n log n) |
| 空间复杂度 | O(n) | O(1) | O(n) |
| 数值稳定性 | 一般（需三角函数） | 好 | 优（仅比较） |
| 实现难度 | 中等 | 简单 | 简单 |
| 适用场景 | 一般情况 | h较小 | 推荐首选 |
| 处理共线点 | 可配置 | 自然处理 | 可配置 |

### 8.2 点与多边形检测方法对比

| 维度 | 射线法 | 环绕数法 |
|------|--------|----------|
| 时间复杂度 | O(n) | O(n) |
| 自相交多边形 | 可能错误 | 正确 |
| 点在边界上 | 需特殊处理 | 需特殊处理 |
| 实现复杂度 | 简单 | 中等 |
| 鲁棒性 | 一般 | 好 |

### 8.3 决策建议

**何时选择Graham Scan**:

- 需要极角排序的其他信息时
- 教学演示目的

**何时选择Jarvis March**:

- 凸包顶点数h远小于n
- 内存受限（仅需O(1)额外空间）

**何时选择Andrew's Monotone Chain**:

- 大多数生产环境的首选
- 对数值稳定性有要求

**决策流程图**:

```
                    ┌─────────────┐

           ▼               ▼               ▼
    ┌────────────┐  ┌────────────┐  ┌────────────┐
    │ h << n?    │  │ 内存受限?  │  │ 一般情况   │
    └──────┬─────┘  └──────┬─────┘  └──────┬─────┘
           │               │               │
           ▼               ▼               ▼
    ┌────────────┐  ┌────────────┐  ┌────────────┐
    │Jarvis March│  │Jarvis March│  │ Andrew's   │
```

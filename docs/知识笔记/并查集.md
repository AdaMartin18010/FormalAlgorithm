## 一、核心概念

### 1.1 定义

**正式定义**:
并查集是一种维护一组不相交（互斥）动态集合的数据结构。每个集合通过一个**代表元**（representative）来标识，支持以下操作：

1. **MAKE-SET(x)**: 创建仅含元素x的新集合
2. **UNION(x, y)**: 合并包含x和y的两个集合
3. **FIND-SET(x)**: 返回x所在集合的代表元

**形式化描述**:
```
设S = {S₁, S₂, ..., Sₖ}是不相交集合的集合，满足：
- ∀i≠j, Sᵢ ∩ Sⱼ = ∅（互不相交）
- ∪Sᵢ = 全域（覆盖所有元素）

每个集合Sᵢ有唯一代表元rep(Sᵢ)，满足：
  x, y ∈ Sᵢ ⟺ FIND-SET(x) = FIND-SET(y) = rep(Sᵢ)
```

**直观解释**:
并查集就像管理多个朋友圈：
- 每个人最初是一个独立圈子（MAKE-SET）
- 两个人成为朋友后，他们的圈子合并（UNION）
- 问"A和B是否认识"就是查他们是否在同一个圈子（FIND-SET比较）

**关键要点**:
- 并查集解决的是**动态连通性**问题
- 路径压缩和按秩合并可将均摊复杂度降至近乎O(1)
- 常用于Kruskal算法、连通分量检测等
- 难以高效支持集合的拆分操作

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 代表元 | 集合的唯一标识 | 可以是任意元素，通常选根节点 |
| 秩(rank) | 树高度的上界 | 用于按秩合并启发式 |
| 父指针 | 指向父节点的引用 | 数组实现时存储父节点索引 |
| 集合大小 | 集合中元素个数 | 可用于按大小合并 |

**性质总结**:
1. **不相交性**: 每个元素恰好属于一个集合
2. **动态性**: 集合可以合并，但不能拆分
3. **传递性**: 若a与b连通，b与c连通，则a与c连通

### 1.3 变体

**带权并查集**:
- 定义: 维护元素到代表元的相对权值（距离/差值）
- 与标准形式的区别: 支持查询元素间的相对关系
- 适用场景: 带约束的连通性问题（如食物链、种类并查集）

**可持久化并查集**:
- 定义: 支持查询历史版本的并查集状态
- 与标准形式的区别: 使用持久化数据结构实现
- 适用场景: 需要回滚或查询历史状态的问题

## 三、形式化证明

### 3.1 核心定理

**定理 1** (并查集复杂度): 使用路径压缩和按秩合并的并查集，m个操作的时间复杂度为O(m · α(n))，其中α是反阿克曼函数。

**证明概要**:
```
反阿克曼函数α(n)增长极其缓慢：
α(n) ≤ 4 对于所有实际应用中的n

证明使用势能分析法：
1. 定义节点级k：满足rank(parent[x]]) ≥ Aₖ(rank[x])的最大k
   （Aₖ是阿克曼函数的变体）

2. 定义节点势能：
   - 若x是根或level(x)=0：φ(x) = 0
   - 否则：φ(x) = α(n) - level(x)

3. 势能变化分析：
   - MAKE-SET: O(1)
   - LINK: 势能增加O(α(n))
   - FIND-SET(带路径压缩): 势能减少支付路径上大部分成本

4. 总计m个操作，总势能变化为O(m·α(n))。

因此总时间复杂度为O(m·α(n))。∎
```

**证明要点分析**:
1. **反阿克曼函数**: 增长比任何对数迭代都慢
2. **势能分析**: 用势能变化均摊昂贵操作的成本
3. **层级概念**: 精细的节点分类用于势能计算

**直觉理解**:
路径压缩几乎将树展平，使得后续查找极快；按秩合并保证树不会太高。两者结合使得均摊成本极小。

### 3.2 辅助引理

**引理 1** (秩的性质): 使用按秩合并时，秩为k的节点至少有2^k个后代。

*证明*:
```
对秩进行归纳。

基例: rank=0，节点是叶子，有2^0=1个后代（自己），成立。

归纳假设: 假设对所有秩<k成立。

归纳步骤: 秩为k的节点由合并两个秩为k-1的树得到。
由归纳假设，每棵树至少2^(k-1)个节点。
合并后总节点数 ≥ 2^(k-1) + 2^(k-1) = 2^k。∎
```

**推论**: 秩为k的节点数最多为n/2^k。

## 四、算法/方法详解

### 4.1 算法描述

**基础实现（无优化）**:
```
算法: MAKE-SET(x)
1. parent[x] = x

算法: FIND-SET(x)
1. if x != parent[x] then
2.     return FIND-SET(parent[x])
3. else
4.     return x

算法: UNION(x, y)
1. rootX = FIND-SET(x)
2. rootY = FIND-SET(y)
3. parent[rootX] = rootY
```

**优化实现（路径压缩+按秩合并）**:
```
算法: MAKE-SET(x)
1. parent[x] = x
2. rank[x] = 0

算法: FIND-SET(x)
1. if x != parent[x] then
2.     parent[x] = FIND-SET(parent[x])  // 路径压缩
3. end if
4. return parent[x]

算法: UNION(x, y)
1. rootX = FIND-SET(x)
2. rootY = FIND-SET(y)
3. if rootX == rootY then return
4. if rank[rootX] > rank[rootY] then
5.     parent[rootY] = rootX
6. else if rank[rootX] < rank[rootY] then
7.     parent[rootX] = rootY
8. else
9.     parent[rootX] = rootY
10.    rank[rootY] = rank[rootY] + 1
```

**流程图**:
```
FIND-SET(x)
    │
    ▼
x是根？
    ├── 是 → 返回x
    └── 否 → 递归查找父节点
              │
              ▼
         路径压缩:
         将x的父节点直接
         指向根节点
              │
              ▼
         返回根节点
```

### 4.2 正确性分析

**UNION正确性**:

**不变式**: 对于任意元素x，FIND-SET(x)始终返回x所在集合的代表元。

**证明**:
- **初始化**: MAKE-SET后，每个元素是自己的代表元，正确
- **保持**: UNION(x,y)将rootX的父设为rootY，之后FIND-SET(x)会沿着父指针找到rootY，而rootY确实是合并后集合的代表元
- **路径压缩不影响**: 只是改变指针路径，不改变集合划分

### 4.3 复杂度分析

| 实现方式 | MAKE-SET | FIND-SET | UNION | 空间 |
|----------|----------|----------|-------|------|
| 基础 | O(1) | O(h) | O(h) | O(n) |
| 按秩合并 | O(1) | O(log n) | O(log n) | O(n) |
| 路径压缩 | O(1) | O(α(n))* | O(α(n))* | O(n) |
| 两者结合 | O(1) | O(α(n))* | O(α(n))* | O(n) |

*均摊复杂度，α(n) < 5 对于所有实际n

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 检测无向图中的环

**问题描述**:
给定无向图的边列表，判断是否存在环。

**解决过程**:
```
初始化: 每个顶点是一个独立集合

对每条边(u, v):
    if FIND-SET(u) == FIND-SET(v):
        存在环！
    else:
        UNION(u, v)

若无冲突，则无环
```

**结果**: 使用并查集可以在近乎线性时间检测环

**示例 2**: Kruskal算法中的连通性检查

**解决过程**:
```
初始化: 每个顶点是一个集合

对所有边按权重排序:
    for (u, v, w) in sorted_edges:
        if FIND-SET(u) != FIND-SET(v):
            选择这条边加入MST
            UNION(u, v)
```

**结果**: 并查集使得Kruskal算法达到O(E log E)

### 5.2 代码实现

**语言**: Rust

```rust
pub struct UnionFind {
    parent: Vec<usize>,
    rank: Vec<usize>,
    size: Vec<usize>,
    count: usize,  // 连通分量数量
}

impl UnionFind {
    pub fn new(n: usize) -> Self {
        UnionFind {
            parent: (0..n).collect(),
            rank: vec![0; n],
            size: vec![1; n],
            count: n,
        }
    }
    
    // 查找带路径压缩
    pub fn find(&mut self, x: usize) -> usize {
        if self.parent[x] != x {
            self.parent[x] = self.find(self.parent[x]);  // 路径压缩
        }
        self.parent[x]
    }
    
    // 按秩合并
    pub fn union(&mut self, x: usize, y: usize) -> bool {
        let root_x = self.find(x);
        let root_y = self.find(y);
        
        if root_x == root_y {
            return false;  // 已在同一集合
        }
        
        // 按秩合并：小树合并到大树
        if self.rank[root_x] < self.rank[root_y] {
            self.parent[root_x] = root_y;
            self.size[root_y] += self.size[root_x];
        } else if self.rank[root_x] > self.rank[root_y] {
            self.parent[root_y] = root_x;
            self.size[root_x] += self.size[root_y];
        } else {
            self.parent[root_y] = root_x;
            self.size[root_x] += self.size[root_y];
            self.rank[root_x] += 1;
        }
        
        self.count -= 1;
        true
    }
    
    // 查询是否连通
    pub fn connected(&mut self, x: usize, y: usize) -> bool {
        self.find(x) == self.find(y)
    }
    
    // 查询集合大小
    pub fn set_size(&mut self, x: usize) -> usize {
        let root = self.find(x);
        self.size[root]
    }
    
    // 查询连通分量数
    pub fn count(&self) -> usize {
        self.count
    }
}

// Python版本
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.size = [1] * n
        self.count = n
    
    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # 路径压缩
        return self.parent[x]
    
    def union(self, x, y):
        root_x, root_y = self.find(x), self.find(y)
        if root_x == root_y:
            return False
        
        # 按秩合并
        if self.rank[root_x] < self.rank[root_y]:
            self.parent[root_x] = root_y
            self.size[root_y] += self.size[root_x]
        elif self.rank[root_x] > self.rank[root_y]:
            self.parent[root_y] = root_x
            self.size[root_x] += self.size[root_y]
        else:
            self.parent[root_y] = root_x
            self.size[root_x] += self.size[root_y]
            self.rank[root_x] += 1
        
        self.count -= 1
        return True
    
    def connected(self, x, y):
        return self.find(x) == self.find(y)
```

**代码说明**:
- `find`: 递归实现路径压缩，将节点直接连接到根
- `union`: 按秩合并，保证树的高度不会过高
- `size数组`: 额外维护集合大小，可用于查询连通分量大小

### 5.3 反例

**常见错误1**: 忘记路径压缩
```python
# 低效代码
def find(x):
    while parent[x] != x:
        x = parent[x]
    return x
```
**错误原因**: 没有路径压缩，树可能退化成链，查询O(n)
**正确做法**: 添加`parent[x] = find(parent[x])`

**常见错误2**: 秩与实际高度混淆
```python
# 错误代码
def union(x, y):
    if rank[x] < rank[y]:  # 应该是rank[find(x)]！
        parent[x] = y
```
**错误原因**: rank是根节点的属性，需要FIND-SET后使用
**正确做法**: 先find得到根，再比较rank[root]

## 六、习题

### 6.1 理解题 (L1)

1. **连通分量计数** [难度⭐]
   
   使用并查集，如何快速得到图的连通分量数量？
   
   <details>
   <summary>解答</summary>
   
   维护一个计数器`count`，初始为n。
   每次成功UNION（两个不同集合合并），`count`减1。
   
   最终`count`即为连通分量数量。
   
   </details>

2. **按大小vs按秩** [难度⭐]
   
   按大小合并和按秩合并有什么区别？
   
   <details>
   <summary>解答</summary>
   
   **按大小**: 小树合并到大树（以节点数为标准）
   **按秩**: 矮树合并到高树（以rank为高度上界）
   
   两者都保证O(log n)的树高，但秩更简洁（不需要额外维护大小）。
   实际上rank ≤ log₂n，按秩合并足够高效。
   
   </details>

### 6.2 应用题 (L2-L3)

1. **冗余连接** [难度⭐⭐]
   
   给定无向图的边，找出一条可以删除的边，使得图仍然连通。
   
   <details>
   <summary>解答</summary>
   
   ```python
   def find_redundant_connection(edges):
       n = len(edges)
       uf = UnionFind(n + 1)
       
       for u, v in edges:
           if uf.connected(u, v):
               return [u, v]  # 这条边形成环
           uf.union(u, v)
       
       return []
   ```
   
   按顺序处理边，第一条使两个已连通顶点相连的边就是答案。
   
   </details>

2. **岛屿数量** [难度⭐⭐]
   
   给定二维网格，1表示陆地，0表示水，计算岛屿数量。
   
   <details>
   <summary>解答</summary>
   
   ```python
   def num_islands(grid):
       if not grid:
           return 0
       
       rows, cols = len(grid), len(grid[0])
       uf = UnionFind(rows * cols)
       
       def index(r, c):
           return r * cols + c
       
       # 合并相邻陆地
       for r in range(rows):
           for c in range(cols):
               if grid[r][c] == '1':
                   for dr, dc in [(0, 1), (1, 0)]:
                       nr, nc = r + dr, c + dc
                       if nr < rows and nc < cols and grid[nr][nc] == '1':
                           uf.union(index(r, c), index(nr, nc))
       
       # 统计根节点数量
       roots = set()
       for r in range(rows):
           for c in range(cols):
               if grid[r][c] == '1':
                   roots.add(uf.find(index(r, c)))
       
       return len(roots)
   ```
   
   </details>

### 6.3 证明题 (L4-L5)

1. **无优化并查集复杂度** [难度⭐⭐⭐]
   
   证明仅使用按秩合并（无路径压缩）的并查集，FIND-SET和UNION的时间复杂度为O(log n)。
   
   <details>
   <summary>解答</summary>
   
   **证明**:
   
   由引理1，秩为k的节点至少有2^k个后代。
   因此，含有n个元素的树，根节点秩最多为⌊log₂n⌋。
   
   FIND-SET沿父指针上升到根，最多经过⌊log₂n⌋ + 1个节点。
   
   UNION需要两次FIND-SET，因此也是O(log n)。∎
   
   </details>

## 八、多维对比

### 8.1 方法对比

| 方法 | 查询连通性 | 合并 | 拆分 | 适用场景 |
|------|------------|------|------|----------|
| DFS/BFS | O(V+E) | - | - | 静态图单次查询 |
| 并查集 | O(α(n)) | O(α(n)) | 不支持 | 动态连通性 |
| 邻接矩阵 | O(1) | O(V) | O(V) | 稠密图 |
| Link-Cut Tree | O(log n) | O(log n) | O(log n) | 需要拆分操作 |

### 8.2 决策建议

**何时使用并查集**:
- 需要频繁查询元素是否在同一集合
- 需要动态合并集合
- 不需要拆分集合

**何时不使用并查集**:
- 需要删除边/拆分集合（考虑动态图算法）
- 只需单次连通性查询（DFS/BFS更简单）
- 需要维护图的拓扑结构（并查集丢失结构信息）

**决策流程图**:
```
需要动态连通性查询？
├── 否 → 使用DFS/BFS
└── 是 → 需要拆分操作？
          ├── 是 → 使用Link-Cut Tree或动态图算法
          └── 否 → 使用并查集
                    需要附加信息？
                    ├── 是 → 带权并查集
                    └── 否 → 标准并查集
```

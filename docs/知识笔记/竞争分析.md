## 一、核心概念

### 1.1 定义

**正式定义**:
竞争分析（Competitive Analysis）是一种用于分析**在线算法**（online algorithms）性能的方法。在线算法必须在**不知道未来输入**的情况下做出决策。

设$\text{ALG}$是在线算法，$\text{OPT}$是最优离线算法（知道整个输入序列）。

对于**最小化问题**：
算法$\text{ALG}$被称为**$c$-竞争的**，如果存在常数$\alpha$，使得对于所有输入序列$\sigma$：
$$\text{ALG}(\sigma) \leq c \cdot \text{OPT}(\sigma) + \alpha$$

若$\alpha = 0$，称为**严格$c$-竞争的**。

竞争比$c$定义为：
$$c = \sup_{\sigma} \frac{\text{ALG}(\sigma)}{\text{OPT}(\sigma)}$$

对于**最大化问题**，不等式方向相反。

**直观解释**:
竞争分析回答：在不预知未来的情况下做决策，我们最多会付出多大的代价？

想象你在租车：
- 每天租车需要100美元
- 买车需要10000美元
- 你不知道会旅行多少天

竞争分析研究：什么策略能保证无论旅行多久，总花费不会超过最优策略（提前知道旅行天数）的$c$倍？

**关键要点**:
- 竞争分析是最坏情况分析，不同于平均情况分析
- 关注与最优离线算法的比值，而非绝对性能
- 适用于在线决策问题：缓存、paging、调度、搜索等

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 最坏情况保证 | 对所有输入序列的性能保证 | 不关心输入分布 |
| 与最优离线比较 | 基准是知道全部未来的最优算法 | 理想但不可实现的基准 |
| 常数$\alpha$ | 允许加性常数 | 对于大规模输入影响可忽略 |
| 下界技术 | adversary arguments | 对手构造最坏输入 |

**性质总结**:
1. **尺度不变性**: 竞争比是相对度量，不受问题规模影响
2. **普适性**: 同一竞争比适用于所有输入
3. **悲观性**: 可能高估实际性能（因为是针对最坏情况）

### 1.3 变体

**随机化竞争分析**:
- 定义: 算法使用随机数，竞争比可以是期望意义或高概率保证
- 与标准形式的区别: $E[\text{ALG}(\sigma)] \leq c \cdot \text{OPT}(\sigma) + \alpha$
- 适用场景: 当确定性算法竞争比下界较高时

**自适应对手模型**:
- oblivious adversary: 对手不知道算法的随机选择
- adaptive adversary: 对手知道算法的随机历史
- 与标准形式的区别: 不同对手模型下竞争比可能不同
- 适用场景: 分析随机化在线算法

**资源增强**:
- 定义: 在线算法比离线算法有更多资源
- 与标准形式的区别: $\text{ALG}_h(\sigma) \leq c \cdot \text{OPT}_k(\sigma)$，其中$h > k$
- 适用场景: paging、调度等问题

## 三、形式化证明

### 3.1 核心定理

**定理 1** (ski rental问题最优竞争比): 
滑雪租赁问题的最优确定性竞争比为2，最优随机化竞争比为$e/(e-1) \approx 1.58$。

**问题定义**:
- 滑雪天数$n$未知（由对手选择）
- 每天租金：1美元
- 购买价格：$b$美元
- 目标：最小化总花费

**确定性算法证明**:

考虑"在第k天购买"策略$S_k$：
- 前k-1天租赁
- 第k天购买

分析：
- 情况1：$n < k$（在买前停止）
  - $S_k$花费 = $n$
  - $OPT = n$（一直租）
  - 比值 = 1

- 情况2：$n \geq k$（买了之后继续）
  - $S_k$花费 = $(k-1) + b$
  - 子情况2a：$n \leq b$（租更划算）
    - $OPT = n \leq b$
    - 比值 $\leq (k-1 + b) / k$
  - 子情况2b：$n > b$（买更划算）
    - $OPT = b$
    - 比值 = $(k-1 + b) / b$

最优k选择：令$(k-1+b)/k \approx (k-1+b)/b$，即$k \approx b$，选择$k = b$

此时竞争比 = $(b-1+b)/b = 2 - 1/b < 2$
当$b \to \infty$时，竞争比趋近于2。

下界证明：
任何确定性算法A，对手可以：
- 若A在某天$d < b$购买，对手在$d+1$天停止
  - A花费 $\geq d + b$，$OPT = d+1$，比值 $\geq b/(d+1)$
- 若A从不购买或第b天后买，对手让$n \to \infty$
  - A花费 $\geq b-1+b$，$OPT = b$，比值 $\geq 2-1/b$

因此任何确定性算法竞争比至少为$2-1/b$。

**证明要点分析**:
1. **对手策略**: 根据算法行为动态选择输入
2. **最坏情况**: 在最优购买日和次优选择之间平衡
3. **参数选择**: 选择$k=b$使两种情况的比值平衡

**直觉理解**:
在线算法面临两难：买太早可能在结束前浪费，买太晚可能多付租金。最优策略是在价格相等临界点（第$b$天）购买，最坏情况下付出2倍代价。

### 3.2 辅助引理

**引理 1** (Paging问题的下界): 对于缓存大小为$k$的paging问题，任何确定性在线算法的竞争比至少为$k$。

*证明概要*:
考虑$k+1$个页面的集合。对手可以始终请求不在缓存中的页面（由于缓存只有$k$个槽，必有至少一个页面不在缓存）。

构造请求序列使得：
- 在线算法每次都发生缓存未命中（page fault）
- 离线算法通过最优替换策略，每$k$次请求才发生一次未命中

因此在线算法至少$k$倍于离线最优。∎

## 四、算法/方法详解

### 4.1 算法描述

**LRU（最近最少使用）Paging算法**:
```
算法: LRU
输入: 缓存大小k，页面请求序列

1. 维护缓存中页面的访问时间戳
2. 对每个页面请求p：
3.     if p在缓存中 then
4.         更新p的时间戳
5.     else
6.         从缓存中驱逐时间戳最早的页面
7.         将p加入缓存
8.     end if
```

**势能函数方法**:
```
定义势能函数Φ = Σ_{p∈缓存} rank(p)
其中rank(p)是页面p在LRU栈中的位置（最近使用=1，最久未用=k）

摊还成本 = 实际成本 + ΔΦ
可以证明摊还成本 ≤ k × OPT的成本
```

### 4.2 正确性分析

**LRU的竞争比分析**:

**定理**: LRU是$k$-竞争的。

**证明** (使用势能方法):
```
定义LRU栈：按最近使用时间排序的页面列表。

势能函数: Φ = LRU栈中与OPT缓存共同包含的页面数

分析单次请求：
- 情况1：页面在LRU和OPT缓存中
  - LRU无成本，Φ不变
  
- 情况2：页面在LRU但不在OPT缓存中
  - LRU无成本，OPT可能有成本
  
- 情况3：页面不在LRU缓存中（LRU未命中）
  - LRU成本 = 1
  - 子情况3a：页面在OPT缓存中
    - OPT无成本
    - 页面进入LRU，Φ增加 ≤ 1
    - 摊还成本 ≤ 2
  - 子情况3b：页面不在OPT缓存中
    - OPT成本 = 1（OPT也必须加载页面）
    - 驱逐分析：
      * 若LRU驱逐的页面不在OPT中，Φ不变
      * 若在OPT中，Φ减少1
    - 每次LRU未命中，OPT最多有1次未命中
    - 但OPT可能有$k$个页面LRU没有
    
综合：LRU每次未命中，OPT要么有未命中，要么LRU势能可增加。
通过细致分析可得：LRU未命中次数 ≤ k × OPT未命中次数
```

### 4.3 复杂度分析

**LRU时间复杂度**:
- 使用哈希表+双向链表实现
- 查找：$O(1)$
- 插入/删除：$O(1)$
- 单次请求：$O(1)$

**空间复杂度**: $O(k)$，缓存大小

**竞争比复杂度**: $k$（与缓存大小成正比）

## 五、示例与实例

### 5.1 标准示例

**示例 1**: Ski Rental问题详细分析

**问题描述**:
- 租金：每天1美元
- 购买价：$b = 100$美元
- 天数$n$未知

**策略比较**:

| 策略 | 描述 | 竞争比 |
|------|------|--------|
| 始终租 | 永远不购买 | 无界（n→∞时） |
| 立即买 | 第1天购买 | b（n=1时达到） |
| 第k天买 | 第k天购买 | min(k, (k-1+b)/b) |
| 最优 | 第b天购买 | $2 - 1/b \approx 2$ |

**最优策略分析**:
- 选择$k = b = 100$
- 前99天租赁，第100天购买
- 若$n < 100$：花费$n$，最优$n$，比值=1
- 若$n \geq 100$：花费$99 + 100 = 199$，最优100，比值=1.99

**示例 2**: Paging问题实例

**问题描述**:
- 缓存大小$k = 3$
- 页面集合：{A, B, C, D, E}
- 请求序列：A, B, C, D, A, D, E, A, B, C

**LRU执行过程**:

| 请求 | 缓存状态 | 未命中 | 驱逐 |
|------|----------|--------|------|
| A | [A] | 是 | - |
| B | [B, A] | 是 | - |
| C | [C, B, A] | 是 | - |
| D | [D, C, B] | 是 | A |
| A | [A, D, C] | 是 | B |
| D | [D, A, C] | 否 | - |
| E | [E, D, A] | 是 | C |
| A | [A, E, D] | 否 | - |
| B | [B, A, E] | 是 | D |
| C | [C, B, A] | 是 | E |

LRU未命中：8次

**OPT执行过程**:
OPT可以预知未来，使用Belady算法（最远将来使用）：

| 请求 | 策略 | 缓存状态 |
|------|------|----------|
| A | 加载A | [A] |
| B | 加载B | [B, A] |
| C | 加载C | [C, B, A] |
| D | 驱逐C（最远使用） | [D, B, A] |
| A | 命中 | [D, B, A] |
| D | 命中 | [D, B, A] |
| E | 驱逐B | [D, E, A] |
| A | 命中 | [D, E, A] |
| B | 驱逐E（最远） | [D, B, A] |
| C | 驱逐D | [C, B, A] |

OPT未命中：6次

竞争比：$8/6 \approx 1.33 < 3$（满足$k$-竞争保证）

### 5.2 代码实现

**语言**: Python

```python
"""
竞争分析算法实现
包含：Ski Rental、LRU Paging
"""

from collections import OrderedDict
from typing import List, Set, Tuple


class SkiRentalSolver:
    """
    Ski Rental问题的在线算法
    """
    def __init__(self, buy_price: int, strategy: str = "optimal"):
        self.b = buy_price
        self.strategy = strategy
        self.rental_cost = 1
        
        if strategy == "optimal":
            # 最优在线策略：第b天购买
            self.buy_day = buy_price
        elif strategy == "immediate":
            self.buy_day = 1
        elif strategy == "never":
            self.buy_day = float('inf')
        else:
            self.buy_day = int(strategy)
    
    def solve(self, actual_days: int) -> Tuple[int, int]:
        """
        执行在线算法
        
        Returns:
            (online_cost, optimal_cost)
        """
        # 在线算法成本
        if actual_days < self.buy_day:
            online_cost = actual_days * self.rental_cost
        else:
            online_cost = (self.buy_day - 1) * self.rental_cost + self.b
        
        # 最优离线成本（知道actual_days）
        optimal_cost = min(actual_days * self.rental_cost, self.b)
        
        return online_cost, optimal_cost
    
    def competitive_ratio(self, max_days: int) -> float:
        """计算最坏情况竞争比"""
        max_ratio = 0
        worst_case = 0
        
        for days in range(1, max_days + 1):
            online, optimal = self.solve(days)
            if optimal > 0:
                ratio = online / optimal
                if ratio > max_ratio:
                    max_ratio = ratio
                    worst_case = days
        
        return max_ratio, worst_case


class LRUPaging:
    """
    LRU (Least Recently Used) Paging算法
    """
    def __init__(self, cache_size: int):
        self.k = cache_size
        # 使用OrderedDict实现O(1)的LRU
        self.cache = OrderedDict()
        self.page_faults = 0
    
    def access(self, page: str) -> bool:
        """
        访问页面
        
        Returns:
            True if hit, False if page fault
        """
        if page in self.cache:
            # Hit: 移动到最近使用
            self.cache.move_to_end(page)
            return True
        else:
            # Page fault
            self.page_faults += 1
            
            if len(self.cache) >= self.k:
                # 驱逐最久未使用的页面
                self.cache.popitem(last=False)
            
            # 添加新页面
            self.cache[page] = True
            return False
    
    def process_sequence(self, sequence: List[str]) -> int:
        """处理请求序列，返回page fault次数"""
        self.cache.clear()
        self.page_faults = 0
        
        for page in sequence:
            self.access(page)
        
        return self.page_faults


class OptimalPaging:
    """
    最优离线Paging算法 (Belady算法)
    需要知道整个请求序列
    """
    def __init__(self, cache_size: int):
        self.k = cache_size
    
    def process_sequence(self, sequence: List[str]) -> int:
        """
        处理请求序列
        使用Belady算法：驱逐最远将来使用的页面
        """
        cache: Set[str] = set()
        page_faults = 0
        
        for i, page in enumerate(sequence):
            if page in cache:
                continue
            
            page_faults += 1
            
            if len(cache) < self.k:
                cache.add(page)
            else:
                # 找到缓存中在将来最晚使用的页面
                farthest_page = None
                farthest_distance = -1
                
                for cached_page in cache:
                    # 查找该页面在将来的下一次使用
                    try:
                        next_use = sequence.index(cached_page, i + 1)
                    except ValueError:
                        # 将来不再使用，立即驱逐
                        farthest_page = cached_page
                        break
                    
                    if next_use > farthest_distance:
                        farthest_distance = next_use
                        farthest_page = cached_page
                
                cache.remove(farthest_page)
                cache.add(page)
        
        return page_faults


# 测试
if __name__ == "__main__":
    # Ski Rental测试
    print("=" * 50)
    print("Ski Rental问题测试")
    print("=" * 50)
    
    b = 100
    
    strategies = [
        ("最优在线策略 (k=b)", "optimal"),
        ("立即购买", "immediate"),
        ("第50天购买", "50"),
    ]
    
    for name, strategy in strategies:
        solver = SkiRentalSolver(b, strategy)
        ratio, worst = solver.competitive_ratio(200)
        print(f"{name}: 竞争比 = {ratio:.2f}, 最坏情况天数 = {worst}")
    
    # Paging测试
    print("\n" + "=" * 50)
    print("Paging问题测试")
    print("=" * 50)
    
    k = 3
    sequence = ['A', 'B', 'C', 'D', 'A', 'D', 'E', 'A', 'B', 'C']
    
    lru = LRUPaging(k)
    lru_faults = lru.process_sequence(sequence)
    
    opt = OptimalPaging(k)
    opt_faults = opt.process_sequence(sequence)
    
    print(f"缓存大小: {k}")
    print(f"请求序列: {sequence}")
    print(f"LRU未命中次数: {lru_faults}")
    print(f"OPT未命中次数: {opt_faults}")
    print(f"实际竞争比: {lru_faults/opt_faults:.2f}")
    print(f"理论竞争比上界 (k): {k}")
```

- `SkiRentalSolver`: 实现不同策略的ski rental算法，可计算竞争比
- `LRUPaging`: 使用OrderedDict实现O(1)的LRU缓存
- `OptimalPaging`: 实现Belady最优算法

## 5.3 反例
### 5.3 反例

**常见误解1**: 竞争比小意味着算法实际性能好
```
误解：竞争比2的算法总是比竞争比3的好
```
**错误原因**: 竞争比是最坏情况保证，不代表平均性能
**澄清**: FIFO对paging的竞争比也是$k$，但实际性能通常不如LRU

**常见误解2**: 在线算法不能优于离线算法
```
误解：在线算法竞争比必须 >= 1
```
**澄清**: 这是正确的。竞争比<1意味着在线算法比知道未来的算法还优，这是不可能的。

**常见错误3**: 忽视加性常数$\alpha$
```
错误分析："我的算法竞争比是1.1"（考虑小输入时）
```
**错误原因**: 竞争比定义允许加性常数，对于小输入，加性常数可能主导
**正确做法**: 明确是严格竞争比（$\alpha=0$）还是渐近竞争比

## 六、习题

### 6.1 理解题

1. **竞争比计算** [难度⭐]
   
   考虑一个在线算法问题：服务请求序列。算法A的成本总是不超过最优离线算法的3倍加10。算法B的成本总是不超过最优的2.5倍（无加性常数）。哪个算法更优？
   
   <details>
   <summary>解答</summary>
   
   算法B更优。
   
   原因：
   - 算法A的竞争比（渐近）是3
   - 算法B的竞争比（严格）是2.5
   
   虽然A的加性常数在输入规模大时可以忽略，但B的严格竞争比更优。
   
   </details>

### 6.2 应用题

1. **列表更新问题** [难度⭐⭐]
   
   考虑列表更新问题：维护一个包含$n$个元素的链表。访问位置$i$的元素成本为$i$。访问后可将该元素移动到任意位置（成本为0）。
   
   Move-to-Front（MTF）算法：每次访问后将元素移动到链表头部。
   
   证明：MTF是4-竞争的。
   
   <details>
   <summary>解答</summary>
   
   **势能函数分析**:
   
   定义势能$\Phi$为MTF列表与OPT列表中逆序对的数量。
   
   对于访问元素$x$：
   - 设$x$在MTF中位置为$k$，在OPT中位置为$j$
   - MTF实际成本 = $k$
   - 分析势能变化：
     * MTF移动$x$到头部，消除$k-1$个逆序
     * OPT可能重排，增加逆序
   
   通过细致分析可得：
   $$C_{MTF} + \Delta\Phi \leq 4 \cdot C_{OPT}$$
   
   因此MTF是4-竞争的。
   
   （注：更精细的分析可得到竞争比2）
   
   </details>

### 6.3 证明题

1. **随机化ski rental下界** [难度⭐⭐⭐]
   
   证明：任何随机化算法对ski rental问题的竞争比至少为$e/(e-1) \approx 1.58$。
   
   <details>
   <summary>解答</summary>
   
   **使用Yao原理**:
   
   构造输入分布：天数$n$从1到$N$均匀分布，然后令$N \to \infty$。
   
   对于确定性算法"第$k$天购买"：
   - 期望成本 = $\frac{1}{N}\sum_{n=1}^{k-1} n + \frac{1}{N}\sum_{n=k}^{N} (k-1+b)$
   - 最优期望成本 = $\frac{1}{N}\sum_{n=1}^{b} n + \frac{1}{N}\sum_{n=b+1}^{N} b$
   
   计算比值并优化$k$，令$k = b/e$，当$b \to \infty$时，比值趋近于$e/(e-1)$。
   
   因此任何随机化算法竞争比至少为$e/(e-1)$。∎
   
   </details>

### 7.1 经典应用

|----------|----------|------------------|

### 7.2 实际案例

**案例**: CPU缓存替换策略

**背景**:
现代CPU使用多级缓存，缓存行替换策略直接影响性能。

**问题**:
- 程序内存访问序列在运行时才能确定
- 需要在线决定替换哪个缓存行

**实际策略竞争比**:
- LRU：$k$-竞争（理论保证）
- FIFO：$k$-竞争
- LFU（最少使用）：无界竞争比
- 实际中LRU通常表现最好（局部性原理）

### 7.3 跨领域联系

**与机器学习的联系**:
在线学习中的regret分析与竞争分析类似，都是与最优策略比较。区别：
- 竞争分析：与离线最优比较
- Regret分析：与固定策略类中的最优比较

**与博弈论的联系**:
对手模型可以看作零和博弈。Yao原理连接了随机化算法和分布复杂性。

**与金融工程的联系**:
在线投资组合选择、期权定价都可以使用竞争分析框架。

## 八、多维对比

### 8.1 方法对比

| 维度 | 最坏情况分析 | 平均情况分析 | 竞争分析 | 平滑分析 |
|------|--------------|--------------|----------|----------|
| 分析对象 | 任意输入 | 随机输入 | 在线vs离线 | 微小扰动 |
| 基准 | 最优解 | 期望性能 | 离线最优 | 最坏情况 |
| 适用范围 | 所有问题 | 分布可假设 | 在线问题 | 容易最坏情况的问题 |
| 保守程度 | 最保守 | 较乐观 | 中等 | 中等 |

### 8.2 决策建议

**何时使用竞争分析**:
- 问题具有在线特性（未来输入未知）
- 需要最坏情况保证
- 离线最优是可计算的基准

**何时考虑其他方法**:
- 输入分布已知或假设合理 → 平均情况分析
- 实际数据通常远离最坏情况 → 平滑分析
- 可与历史数据比较 → 机器学习/regret分析

**竞争分析与其他方法的结合**:
- **随机化竞争分析**: 结合随机化降低最坏情况影响
- **平滑竞争分析**: 结合平滑分析处理特殊输入
- **参数化竞争分析**: 考虑输入结构参数

**决策流程图**:
```
算法设计问题
├── 输入是否完全已知？
│   ├── 是 → 传统算法设计
│   └── 否 → 在线问题
├── 是否需要最坏情况保证？
│   ├── 否 → 启发式/平均情况分析
│   └── 是 → 竞争分析
├── 是否允许随机化？
│   ├── 是 → 随机化竞争分析
│   └── 否 → 确定性竞争分析
└── 是否需要更精细分析？
    ├── 是 → 平滑竞争分析/advice complexity
    └── 否 → 标准竞争分析
```

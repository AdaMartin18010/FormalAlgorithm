## 一、核心概念

### 1.1 定义

**正式定义**:

**P类**: 能被**确定性图灵机**在**多项式时间**内解决的判定问题集合。
$$\text{P} = \bigcup_{k \geq 1} \text{TIME}(n^k)$$

**NP类**: 能被**非确定性图灵机**在**多项式时间**内解决的判定问题集合；或等价地，解可在多项式时间内被**验证**的问题集合。
$$\text{NP} = \bigcup_{k \geq 1} \text{NTIME}(n^k) = \{L : \exists \text{ 多项式时间验证器 } V\}$$

**co-NP**: NP问题的补集构成的类。

**P vs NP问题**: 计算机科学最核心的开放问题，问P是否等于NP。

**直观解释**:

- P问题：可以"快速找到"解的问题（如排序、最短路径）
- NP问题：可以"快速验证"解的问题（如SAT、哈密顿路径）
- P=NP？：验证解的容易性是否意味着找到解也容易？

**关键要点**:

- P $\subseteq$ NP（显然）
- 若P=NP，则现代密码学将失效
- NP $\neq$ co-NP ⟹ P $\neq$ NP
- 指数时间假设(ETH)暗示3-SAT需要$2^{\Omega(n)}$时间

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 封闭性 | P对并、交、补、连接封闭 | NP对补是否封闭未知 |
| 多项式时间 | 与具体计算模型无关 | 高级编程语言、多带TM等 |
| 验证器定义 | 证书(certificate)可在多项式时间验证 | 是定义NP的常用方式 |
| NP完全性 | NP中最难的问题类 | 解决一个=解决全部 |

### 1.3 复杂性类层次

```
P ⊆ NP ⊆ PSPACE ⊆ EXP
      ⊆ co-NP
```

## 三、形式化证明

### 3.1 库克-列文定理

**定理**: SAT是NP完全的。

**证明概要**:

第一部分：SAT ∈ NP
给定赋值，可在多项式时间验证满足性。

第二部分：SAT是NP困难的
对任意L ∈ NP，构造多项式时间归约到SAT。

设N是在p(n)时间内判定L的NTM。
对输入x，构造布尔公式$\varphi_x$，使得：
$$\varphi_x \text{可满足} \Leftrightarrow N \text{接受} x$$

变量表示计算历史：

- T[i,j,s]: 第i步第j格为符号s
- H[i,j]: 第i步读写头在j
- S[i,s]: 第i步状态为s

子句确保：初始正确、转移合法、接受状态可达。
公式大小为O(p(n)²)，可在多项式时间构造。

### 3.2 归约传递性

**定理**: 若A ≤p B且B ≤p C，则A ≤p C。

## 五、习题

### 5.1 理解题 (L1-L2)

1. **验证NP类** [难度⭐]

   证明：若L ∈ NP，则存在多项式p和多项式时间算法V，使得：
   $$x \in L \Leftrightarrow \exists c, |c| \leq p(|x|), V(x, c) = 1$$

   <details>
   <summary>解答</summary>

   由NP定义，存在NTM N在多项式时间p(n)内判定L。
   证书c可以是N的接受计算路径的描述。
   验证器V模拟N按c指定的非确定性选择，验证该路径接受。

   </details>

2. **P=NP的含义** [难度⭐⭐]

   如果P=NP被证明，现代密码学会受到什么影响？

   <details>
   <summary>解答</summary>

   - RSA、Diffie-Hellman等基于困难问题的密码系统将不安全
   - 零知识证明将变得平凡
   - 需要转向信息论安全的密码学（如一次性密码本）
   - 量子密码可能变得更重要

   </details>

### 5.2 证明题 (L3-L4)

1. **复杂性类关系** [难度⭐⭐⭐]

   证明：若NP ≠ co-NP，则P ≠ NP。

   <details>
   <summary>解答</summary>

   反证法：若P = NP，则P = co-P = co-NP，故NP = co-NP。

   因此NP ≠ co-NP ⟹ P ≠ NP。

   </details>

## 一、核心概念

### 1.1 定义

**正式定义**:
多臂老虎机（MAB）是一个顺序决策问题。有 $K$ 个动作（臂），学习者每轮选择一个臂 $a_t \in [K]$，获得随机奖励 $r_t \sim \nu_{a_t}$，其中 $\nu_i$ 是臂 $i$ 的奖励分布，均值为 $\mu_i$。

**目标**: 最大化累计奖励，等价于最小化**遗憾**（Regret）：
$$R_T = T \cdot \mu^* - \sum_{t=1}^T \mu_{a_t}$$
其中 $\mu^* = \max_i \mu_i$ 是最优臂的期望奖励。

**模型变体**:

| 模型 | 描述 | 特点 |
|------|------|------|
| **随机MAB** | 奖励分布固定 | 探索-利用权衡经典模型 |
| **对抗MAB** | 奖励序列由对手选择 | 无需分布假设 |
| **上下文MAB** | 每轮有上下文 $x_t$ | 结合监督学习 |
| **组合MAB** | 选择臂的子集 | 组合动作空间 |
| **非稳态MAB** | 奖励分布随时间变化 | 需要自适应机制 |

**直观解释**:
想象你在赌场面对多个老虎机（拉杆），每个拉杆有不同的中奖概率（但你不知道）。你需要在以下两者间平衡：

- **利用**: 玩看起来最好的机器，最大化即时收益
- **探索**: 尝试其他机器，确认是否有更好的选择

这个探索-利用权衡是强化学习、推荐系统、临床实验等问题的核心。

**关键要点**:

- 遗憾下界: $\Omega(\sqrt{KT})$（问题相关）或 $\Omega(\sqrt{KT})$（问题无关）
- 最优算法达到 $O(\sqrt{KT \log T})$ 或 $O(\sqrt{KT})$
- 上下文MAB连接监督学习与在线决策

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 探索-利用权衡 | 核心挑战 | 平衡当前最优与信息获取 |
| 遗憾增长 | 次线性 $o(T)$ | 保证长期最优 |
| 样本复杂度 | 识别最优臂所需样本 | 与遗憾密切相关 |
| 分布依赖/无关 | 是否依赖reward gap | 影响算法设计 |

**性质总结**:

1. **Lai-Robbins下界**: 问题依赖遗憾 $\sum_{i: \Delta_i > 0} \frac{\ln T}{\Delta_i}$
2. **极小极大下界**: 问题无关遗憾 $\Omega(\sqrt{KT})$
3. **无免费午餐**: 不知道最优臂时需要付出代价

### 1.3 变体

**Thompson Sampling**:

- 定义: 贝叶斯方法，从后验采样选择臂
- 特点: 简单、在实践中表现优异
- 适用场景: 有先验知识、需要概率解释

**UCB (Upper Confidence Bound)**:

- 定义: 乐观面对不确定性原则
- 特点: 理论保证强、易于分析
- 适用场景: 理论分析、确定性保证

**EXP3**:

- 定义: 对抗设置下的指数加权算法
- 特点: 无需分布假设
- 适用场景: 非稳态、对抗环境

## 三、形式化证明

### 3.1 核心定理

**定理 1** (UCB1的遗憾上界):
UCB1算法在 $T$ 轮后的期望遗憾满足：
$$R_T \leq \sum_{i: \mu_i < \mu^*} \frac{8 \ln T}{\Delta_i} + (1 + \frac{\pi^2}{3}) \sum_{i=1}^K \Delta_i$$
其中 $\Delta_i = \mu^* - \mu_i$。

**UCB1算法**:

```
对每轮 t = 1, 2, ..., T:
    若臂i被拉动次数 n_i(t) = 0，选择臂i
    否则选择: argmax_i [μ̂_i(t) + sqrt(2 ln t / n_i(t))]
```

**证明**:

**步骤1**: 分析次优臂被选择的条件

定义事件 $\mathcal{E}_i(t)$:
$$\hat{\mu}_i(t) + \sqrt{\frac{2 \ln t}{n_i(t)}} \leq \mu_i + 2\sqrt{\frac{2 \ln t}{n_i(t)}}$$

由Hoeffding不等式：$P(\mathcal{E}_i(t)) \geq 1 - t^{-4}$

**步骤2**: 次优臂被选择的充分条件

臂 $i$ 被选择意味着：
$$\hat{\mu}_i(t) + \sqrt{\frac{2 \ln t}{n_i(t)}} \geq \mu^*$$

在 $\mathcal{E}_i(t)$ 和 $\mathcal{E}_*(t)$ 都成立时：
$$\mu_i + 2\sqrt{\frac{2 \ln t}{n_i(t)}} \geq \mu^*$$

即：
$$n_i(t) \leq \frac{8 \ln t}{\Delta_i^2}$$

**步骤3**: 遗憾分解

$$R_T = \sum_{i: \Delta_i > 0} \Delta_i \cdot E[n_i(T)]$$

由步骤2，$n_i(T) \leq \frac{8 \ln T}{\Delta_i^2}$ 大部分时间成立，加上探索失败的小概率事件：

$$E[n_i(T)] \leq \frac{8 \ln T}{\Delta_i^2} + \sum_{t=1}^T 2t^{-4} \leq \frac{8 \ln T}{\Delta_i^2} + \frac{\pi^2}{3}$$

代入遗憾公式即得结论。∎

**证明要点分析**:

1. **乐观原则**: 置信上界鼓励探索不确定的臂
2. **集中不等式**: Hoeffding不等式保证置信区间有效性
3. **对数增长**: 次优臂拉动次数对数增长，确保次线性遗憾

### 3.2 辅助引理

**引理 1** (Hoeffding不等式):
设 $X_1, \cdots, X_n$ 是独立同分布随机变量，$X_i \in [0,1]$，$\mu = E[X_i]$，则：
$$P(|\hat{\mu} - \mu| \geq \epsilon) \leq 2e^{-2n\epsilon^2}$$

**引理 2** (Lai-Robbins下界):
对于任意一致算法（即次优臂拉动次数 $o(T^\alpha)$ 对所有 $\alpha > 0$），：
$$\liminf_{T \to \infty} \frac{R_T}{\ln T} \geq \sum_{i: \Delta_i > 0} \frac{\Delta_i}{KL(\mu_i, \mu^*)}$$

## 四、算法/方法详解

### 4.1 UCB1算法

**伪代码**:

```
算法: UCB1
输入: 臂数 K, 时间范围 T

1.  for i = 1 to K do
2.      拉动臂i一次，观察奖励 r_i
3.      n_i ← 1, μ̂_i ← r_i
4.  end for
5.
6.  for t = K+1 to T do
7.      for i = 1 to K do
8.          ucb_i ← μ̂_i + sqrt(2 ln t / n_i)
9.      end for
10.     a_t ← argmax_i ucb_i
11.     拉动臂 a_t，观察奖励 r_t
12.     n_{a_t} ← n_{a_t} + 1
13.     μ̂_{a_t} ← μ̂_{a_t} + (r_t - μ̂_{a_t}) / n_{a_t}
14. end for
```

### 4.2 Thompson Sampling

**伪代码**:

```
算法: Thompson Sampling (Bernoulli奖励)
输入: 臂数 K

1.  对每个臂i: S_i ← 1, F_i ← 1  (Beta先验参数)
2.
3.  for t = 1, 2, ... do
4.      for i = 1 to K do
5.          θ_i ~ Beta(S_i, F_i)  (从后验采样)
6.      end for
7.      a_t ← argmax_i θ_i
8.      拉动臂 a_t，观察奖励 r_t ∈ {0, 1}
9.      if r_t = 1 then S_{a_t} ← S_{a_t} + 1
10.     else F_{a_t} ← F_{a_t} + 1
11. end for
```

### 4.3 EXP3算法

**伪代码** (对抗设置):

```
算法: EXP3
输入: 臂数 K, 学习率 η

1.  w_i ← 1 for all i ∈ [K]
2.
3.  for t = 1, 2, ... do
4.      p_i ← w_i / (Σ_j w_j) for all i
5.      按分布 p 随机选择臂 a_t
6.      观察奖励 r_t ∈ [0, 1]
7.
8.      // 重要性采样估计
9.      r̂_t(i) ← r_t / p_i if i = a_t else 0
10.     w_i ← w_i · exp(η · r̂_t(i)) for all i
11. end for
```

**复杂度分析**:

| 算法 | 每轮时间 | 空间 | 遗憾界 |
|------|----------|------|--------|
| UCB1 | O(K) | O(K) | O(Σ 1/Δ_i · log T) |
| Thompson Sampling | O(K) | O(K) | O(Σ 1/Δ_i · log T) |
| EXP3 | O(K) | O(K) | O(√(KT log K)) |
| ε-greedy | O(1) | O(K) | O(T^{2/3}) (调参后) |

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 在线广告选择

**问题描述**:
平台有 $K=5$ 个广告位，每个用户对不同广告点击率未知。需要在线学习最佳广告展示策略。

**模拟设置**:

| 广告 | 真实CTR | 前100轮选择次数 |
|------|---------|-----------------|
| A | 0.05 | 23 |
| B | 0.08 | 25 |
| C | 0.10 | 28 |
| D | 0.03 | 14 |
| E | 0.12 | 10 |

**观察**: 早期探索均匀，后期集中于E和C

**示例 2**: 药物临床试验

**问题描述**:
测试 $K=3$ 种治疗方案，患者依次到达。目标：最大化治愈率，同时识别最佳治疗。

**伦理考量**: 不能牺牲患者利益过度探索。

**解决方案**: 使用有限期的UCB或Thompson Sampling，设置探索概率下限。

### 5.2 代码实现

**语言**: Python

```python
import numpy as np
from typing import List, Callable, Tuple
import random


class BanditAlgorithm:
    """多臂老虎机算法基类"""

    def __init__(self, n_arms: int):
        self.n_arms = n_arms
        self.reset()

    def reset(self):
        """重置算法状态"""
        self.total_pulls = 0
        self.pull_counts = np.zeros(self.n_arms)
        self.reward_sums = np.zeros(self.n_arms)
        self.history = []

    def select_arm(self) -> int:
        """选择要拉动的臂"""
        raise NotImplementedError

    def update(self, arm: int, reward: float):
        """更新算法状态"""
        self.total_pulls += 1
        self.pull_counts[arm] += 1
        self.reward_sums[arm] += reward
        self.history.append((arm, reward))

    def get_estimates(self) -> np.ndarray:
        """获取各臂的期望奖励估计"""
        with np.errstate(divide='ignore', invalid='ignore'):
            return self.reward_sums / np.maximum(self.pull_counts, 1)


class EpsilonGreedy(BanditAlgorithm):
    """ε-贪心算法"""

    def __init__(self, n_arms: int, epsilon: float = 0.1):
        super().__init__(n_arms)
        self.epsilon = epsilon

    def select_arm(self) -> int:
        if random.random() < self.epsilon:
            return random.randint(0, self.n_arms - 1)
        else:
            estimates = self.get_estimates()
            return int(np.argmax(estimates))


class UCB1(BanditAlgorithm):
    """UCB1算法"""

    def select_arm(self) -> int:
        if self.total_pulls < self.n_arms:
            # 初始探索：每个臂至少拉一次
            return self.total_pulls

        estimates = self.get_estimates()
        ucb_values = np.zeros(self.n_arms)

        for i in range(self.n_arms):
            if self.pull_counts[i] == 0:
                ucb_values[i] = float('inf')
            else:
                exploration = np.sqrt(2 * np.log(self.total_pulls) / self.pull_counts[i])
                ucb_values[i] = estimates[i] + exploration

        return int(np.argmax(ucb_values))


class ThompsonSampling(BanditAlgorithm):
    """Thompson Sampling (Bernoulli奖励)"""

    def reset(self):
        super().reset()
        self.successes = np.ones(self.n_arms)  # Beta(1,1)先验
        self.failures = np.ones(self.n_arms)

    def select_arm(self) -> int:
        samples = np.random.beta(self.successes, self.failures)
        return int(np.argmax(samples))

    def update(self, arm: int, reward: float):
        super().update(arm, reward)
        if reward > 0.5:  # 假设reward ∈ {0, 1}
            self.successes[arm] += 1
        else:
            self.failures[arm] += 1


class EXP3(BanditAlgorithm):
    """EXP3算法 (对抗设置)"""

    def __init__(self, n_arms: int, gamma: float = 0.1):
        super().__init__(n_arms)
        self.gamma = gamma
        self.weights = np.ones(n_arms)
        self.eta = np.sqrt(np.log(n_arms) / (n_arms * 1000))  # 假设T=1000

    def reset(self):
        super().reset()
        self.weights = np.ones(self.n_arms)

    def select_arm(self) -> int:
        total_weight = np.sum(self.weights)
        probs = (1 - self.gamma) * (self.weights / total_weight) + self.gamma / self.n_arms
        return int(np.random.choice(self.n_arms, p=probs))

    def update(self, arm: int, reward: float):
        super().update(arm, reward)

        total_weight = np.sum(self.weights)
        probs = (1 - self.gamma) * (self.weights / total_weight) + self.gamma / self.n_arms

        # 重要性采样估计
        estimated_reward = reward / probs[arm]
        self.weights[arm] *= np.exp(self.eta * estimated_reward)


def run_simulation(algorithm: BanditAlgorithm,
                   true_means: List[float],
                   n_rounds: int,
                   n_runs: int = 10) -> Tuple[np.ndarray, np.ndarray]:
    """
    运行模拟实验

    Returns:
        (regrets, arm_counts): 每轮遗憾和臂选择次数
    """
    K = len(true_means)
    all_regrets = np.zeros((n_runs, n_rounds))
    all_counts = np.zeros((n_runs, K))

    for run in range(n_runs):
        algorithm.reset()
        optimal_mean = max(true_means)
        cumulative_reward = 0
        optimal_cumulative = 0

        for t in range(n_rounds):
            arm = algorithm.select_arm()
            reward = np.random.binomial(1, true_means[arm])
            algorithm.update(arm, reward)

            cumulative_reward += reward
            optimal_cumulative += np.random.binomial(1, optimal_mean)

            all_regrets[run, t] = optimal_cumulative - cumulative_reward

        all_counts[run] = algorithm.pull_counts

    return np.mean(all_regrets, axis=0), np.mean(all_counts, axis=0)


# 测试
if __name__ == "__main__":
    print("=" * 60)
    print("多臂老虎机算法测试")
    print("=" * 60)

    # 设置
    K = 5
    true_means = [0.1, 0.3, 0.5, 0.2, 0.4]  # 臂2最优
    n_rounds = 1000
    n_runs = 20

    print(f"\n真实奖励概率: {true_means}")
    print(f"最优臂: 2 (μ = {true_means[2]})")
    print(f"模拟轮数: {n_rounds}, 重复次数: {n_runs}")

    algorithms = [
        ("ε-Greedy (ε=0.1)", EpsilonGreedy(K, epsilon=0.1)),
        ("UCB1", UCB1(K)),
        ("Thompson Sampling", ThompsonSampling(K)),
        ("EXP3", EXP3(K)),
    ]

    print("\n" + "-" * 60)
    print("结果对比")
    print("-" * 60)

    for name, algo in algorithms:
        regrets, counts = run_simulation(algo, true_means, n_rounds, n_runs)
        final_regret = regrets[-1]
        best_arm_ratio = counts[2] / np.sum(counts)

        print(f"\n{name}:")
        print(f"  最终遗憾: {final_regret:.2f}")
        print(f"  最优臂选择比例: {best_arm_ratio:.2%}")
        print(f"  臂选择分布: {[f'{c:.1f}' for c in counts]}")
```

**代码说明**:

- `BanditAlgorithm`: 基类提供通用接口
- `EpsilonGreedy`: 基础探索-利用策略
- `UCB1`: 乐观面对不确定性
- `ThompsonSampling`: 贝叶斯方法
- `EXP3`: 对抗设置算法

### 5.3 反例

**常见错误1**: 忽略探索的重要性

```python
# 错误：纯贪心，无探索
class PureGreedy:
    def select_arm(self):
        return argmax(self.estimates)  # 可能陷入次优臂
```

**错误原因**: 初期估计不准时可能锁定次优臂
**正确做法**: 确保每个臂有足够探索（UCB、Thompson Sampling）

**常见错误2**: 错误设置UCB参数

```python
# 错误：置信区间过小
ucb = estimate + 0.1 * sqrt(log(t) / n)  # 常数太小
```

**错误原因**: 常数太小导致探索不足
**正确做法**: 使用理论保证的常数 $\sqrt{2 \ln t / n}$

## 六、习题

### 6.1 理解题 (L1-L2)

1. **UCB计算** [难度⭐]

   臂A已被拉动100次，平均奖励0.6；臂B被拉动20次，平均奖励0.7。当前轮数t=200，计算UCB值并选择哪个臂？

   <details>
   <summary>解答</summary>

   UCB(A) = 0.6 + sqrt(2 * ln(200) / 100) = 0.6 + sqrt(10.6/100) = 0.6 + 0.326 = 0.926

   UCB(B) = 0.7 + sqrt(2 * ln(200) / 20) = 0.7 + sqrt(10.6/20) = 0.7 + 0.728 = 1.428

   选择臂B（更高UCB值，尽管平均奖励略低但不确定性更大）

   </details>

2. **探索-利用权衡** [难度⭐⭐]

   解释为什么多臂老虎机问题不能简单地"先探索后利用"（即先均匀拉动所有臂，然后永远选择最佳估计臂）。

   <details>
   <summary>解答</summary>

   原因：
   1. **不知道探索何时足够**：不知道需要多少样本来准确估计
   2. **遗憾累积**：探索阶段的遗憾线性增长，如果探索比例固定，总遗憾也是线性的
   3. **错误识别风险**：有限样本下可能选错最优臂，永久利用次优臂
   4. **环境变化**：非稳态环境下需要持续探索

   正确做法：探索强度随时间递减（如UCB的对数增长），确保总遗憾次线性。

   </details>

### 6.2 应用题 (L3)

1. **上下文MAB** [难度⭐⭐⭐]

   设计LinUCB算法用于上下文多臂老虎机问题。假设每个臂i有特征向量 $x_{i,t}$，奖励模型为 $r_{i,t} = x_{i,t}^T \theta_i + \epsilon$。

   <details>
   <summary>解答</summary>

   **LinUCB算法**:

   对每个臂i维护：
   - $A_i = I + \sum x x^T$（设计矩阵）
   - $b_i = \sum r x$（奖励加权和）

   参数估计：$\hat{\theta}_i = A_i^{-1} b_i$

   UCB值：
   $$ucb_i(t) = x_{i,t}^T \hat{\theta}_i + \alpha \sqrt{x_{i,t}^T A_i^{-1} x_{i,t}}$$

   选择：$a_t = \arg\max_i ucb_i(t)$

   更新：观察奖励后更新 $A_{a_t}$ 和 $b_{a_t}$

   遗憾界：$O(d\sqrt{T})$，其中d是特征维度。

   </details>

### 6.3 证明题 (L4-L5)

1. **EXP3遗憾界** [难度⭐⭐⭐⭐]

   证明EXP3算法在适当选择学习率时达到 $O(\sqrt{KT \ln K})$ 遗憾。

   <details>
   <summary>解答</summary>

   **关键步骤**:

   定义势函数 $\Phi_t = \sum_{i=1}^K w_i(t)$

   分析 $\Phi_{t+1}/\Phi_t$：
   $$\frac{\Phi_{t+1}}{\Phi_t} = \sum_i p_i(t) e^{\eta \hat{r}_i(t)} \leq 1 + \eta \sum_i p_i(t) \hat{r}_i(t) + \eta^2 \sum_i p_i(t) \hat{r}_i(t)^2$$

   取对数并累加：
   $$\ln \frac{\Phi_{T+1}}{\Phi_1} \leq \eta \sum_t \sum_i p_i(t) \hat{r}_i(t) + \eta^2 \sum_t \sum_i p_i(t) \hat{r}_i(t)^2$$

   下界：$\ln \Phi_{T+1} \geq \eta \sum_t \hat{r}_j(t) - \ln K$ 对任意j

   结合并取期望，选择 $\eta = \sqrt{\ln K / (KT)}$ 得到：
   $$R_T \leq 2\sqrt{KT \ln K}$$

   </details>

2. **Bayes最优遗憾** [难度⭐⭐⭐⭐⭐]

   在贝叶斯框架下（假设奖励分布参数有先验），证明Thompson Sampling达到Bayes最优遗憾阶。

   <details>
   <summary>解答</summary>

   **分析框架**:

   Thompson Sampling从后验采样 $\tilde{\mu}_i \sim P(\mu_i | \mathcal{H}_t)$，选择 $a_t = \arg\max_i \tilde{\mu}_i$

   **关键观察**: 选择动作a_t的概率等于a_t是最优臂的后验概率。

   **信息比分析**:
   定义信息比 $\Gamma_t = \frac{(E_t[\mu^* - \mu_{a_t}])^2}{I_t(a^*; (a_t, r_t))}$

   证明对于Bernoulli奖励，$\Gamma_t \leq K/2$

   由信息论论证，总遗憾：
   $$R_T \leq \sqrt{\frac{K}{2} \cdot H(a^*) \cdot T}$$

   其中 $H(a^*)$ 是最优臂的熵。

   对于常见先验，$R_T = O(\sqrt{KT})$，达到极小最优。∎

   </details>

| 资源调度 | 任务分配 | 在线学习机器性能 |

### 7.2 实际案例

**案例**: Netflix内容推荐

**背景**:
Netflix需要为每个用户从海量内容中选择推荐，目标是最大化点击率/观看时长。

**技术方案**:

- 使用上下文MAB（LinUCB）结合用户特征
- 每个内容是一个"臂"
- 考虑内容新鲜度（非稳态）
- 处理大规模（百万级内容）

**挑战与解决方案**:

- **冷启动**: 新内容用Thompson Sampling快速学习
- **多样性**: 在MAB目标中加入多样性正则
- **长期奖励**: 使用强化学习扩展（如DQN）

### 7.3 跨领域联系

**与强化学习的联系**:
MAB是最简单的RL问题（单状态MDP）。扩展到多状态即完整RL问题，需要值函数估计和策略优化。

**与在线学习的联系**:
MAB是partial information online learning。全信息版本是Expert问题（如Hedge算法）。

**与A/B测试的联系**:
传统A/B测试是固定探索，MAB提供自适应探索，理论上更高效。

**与经济学机制设计的联系**:
在激励兼容机制中使用MAB算法，如拍卖中的策略证明学习。

## 八、多维对比

### 8.1 算法对比

| 算法 | 设置 | 遗憾界 | 计算复杂度 | 超参数 | 实际表现 |
|------|------|--------|------------|--------|----------|
| ε-Greedy | 随机 | O(T^{2/3}) | O(1) | ε, 衰减 | 一般 |
| UCB1 | 随机 | O(log T) | O(K) | 无 | 好 |
| UCB2 | 随机 | O(log T) | O(K) | 有 | 好 |
| Thompson Sampling | 随机 | O(log T) | O(K) | 先验 | 很好 |
| EXP3 | 对抗 | O(√(T)) | O(K) | γ, η | 稳定 |
| EXP3.P | 对抗 | O(√(T)) | O(K) | 有 | 更稳定 |
| MOSS | 随机 | O(√(KT)) | O(K) | 无 | 好 |

### 8.2 问题变体对比

| 变体 | 动作空间 | 信息结构 | 主要挑战 | 典型算法 |
|------|----------|----------|----------|----------|
| 标准MAB | K个离散臂 |  bandit反馈 | 探索-利用 | UCB, TS |
| 上下文MAB | 特征关联 | 上下文+奖励 | 泛化 | LinUCB |
| 组合MAB | 集合/排列 | 部分观测 | 组合优化 | CUCB |
| 非稳态MAB | 同标准 | 分布漂移 | 适应性 | Sliding UCB |
| 约束MAB | 同标准 | 预算约束 | 资源管理 | Budgeted MAB |
| 对偶MAB | 双边匹配 | 双边反馈 | 双边学习 | DoubleUCB |

### 8.3 决策建议

**算法选择流程**:

```
问题设置？
├── 随机环境 (奖励分布固定)
│   ├── 小规模 (K < 100)
│   │   └── Thompson Sampling 或 UCB1
│   ├── 大规模上下文
│   │   └── LinUCB / NeuralUCB
│   └── 需要极小最优保证
│       └── MOSS / KL-UCB
└── 对抗环境 (可能非稳态)
    ├── 需要高概率保证
    │   └── EXP3.P / EXP3-IX
    └── 期望保证足够
        └── EXP3
```

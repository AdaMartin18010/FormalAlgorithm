## 一、核心概念

### 1.1 定义

**正式定义**:
数据结构是计算机中存储、组织数据的方式，使得数据可以高效地被访问和修改。基础数据结构包括数组、链表、栈、队列等线性结构。

**抽象数据类型(ADT) vs 数据结构**:
- ADT：逻辑层面的定义（What）
- 数据结构：物理层面的实现（How）

**直观解释**:
数据结构就像不同的储物方式：
- 数组 = 连续编号的储物柜
- 链表 = 每个盒子记住下一个盒子的位置
- 栈 = 叠盘子（后进先出）
- 队列 = 排队（先进先出）

**关键要点**:
- 没有最好的数据结构，只有最适合特定场景的数据结构
- 时间-空间权衡是数据结构设计的核心
- 理解底层实现有助于预测性能特征

### 1.2 属性

| 数据结构 | 访问 | 插入 | 删除 | 空间 |
|----------|------|------|------|------|
| 数组 | O(1) | O(n) | O(n) | O(n) |
| 链表 | O(n) | O(1)* | O(1)* | O(n) |
| 栈 | O(n) | O(1) | O(1) | O(n) |
| 队列 | O(n) | O(1) | O(1) | O(n) |

*给定位置指针

**性质总结**:
1. **局部性原理**: 数组具有良好的空间局部性，缓存友好
2. **动态扩容**: 动态数组均摊O(1)插入，但偶尔有O(n)开销
3. **指针开销**: 链表每个节点额外存储指针，空间利用率较低

### 1.3 变体

**动态数组(ArrayList/Vector)**:
- 定义: 自动扩容的数组
- 与标准形式的区别: 插入均摊O(1)，偶尔O(n)重分配
- 适用场景: 需要随机访问且大小变化的场景

**双向链表**:
- 定义: 每个节点有前驱和后继指针
- 与标准形式的区别: 支持O(1)反向遍历和尾部删除
- 适用场景: 需要频繁在两端操作的场景

**循环队列**:
- 定义: 用数组实现的队列，首尾相接
- 与标准形式的区别: 避免出队时的元素移动
- 适用场景: 固定大小的缓冲区实现

## 三、形式化证明

### 3.1 核心定理

**定理 1** (动态数组均摊分析): 动态数组的n次插入操作总时间为O(n)，即均摊O(1)每次。

**证明**:
```
设初始容量为1，每次扩容为原来的2倍。

第k次扩容发生在插入2^(k-1) + 1个元素时，
需要复制2^(k-1)个元素。

总复制成本 = sum_{k=1}^{log n} 2^(k-1)
           = 2^log n - 1
           = n - 1
           = O(n)

n次插入，总成本 = n次插入 + O(n)次复制 = O(n)
均摊成本 = O(n)/n = O(1)。∎
```

**证明要点分析**:
1. **聚合分析**: 计算n次操作的总成本再平均
2. **几何级数**: 扩容成本形成几何级数，总和与最后项同阶
3. **摊还效应**: 昂贵的扩容操作被后续的廉价操作摊还

**直觉理解**:
扩容成本虽然偶尔很高，但这种高成本很少发生（指数级稀疏），平均下来每次操作的代价是常数。

### 3.2 辅助引理

**引理 1** (链表反转): 可以在O(n)时间和O(1)空间内反转链表。

*证明*:
```
使用三个指针prev、current、next遍历链表，
每次迭代将current.next指向prev。

遍历n个节点，每步O(1)，总时间O(n)。
只使用固定数量的指针，空间O(1)。∎
```

## 四、算法/方法详解

### 4.1 算法描述

**动态数组扩容策略**:
```
算法: INSERT(A, x)
输入: 动态数组A，元素x
输出: 将x插入A末尾

1. if A.size == A.capacity then
2.     // 需要扩容
3.     new_capacity = A.capacity * 2  // 或1.5倍
4.     分配new_capacity大小的新数组
5.     将A的所有元素复制到新数组
6.     释放旧数组
7.     A.capacity = new_capacity
8. end if
9. A[A.size] = x
10. A.size = A.size + 1
```

**流程图**:
```
开始插入
    │
    ▼
数组已满？
    ├── 是 → 分配双倍空间
    │         复制数据
    │         更新容量
    │
    └── 否 → 直接插入
              增加大小
```

### 4.2 正确性分析

**不变式**: 每次操作后，A[0..size-1]存储有效元素，size <= capacity。

**证明**:
- **初始化**: 初始size=0, capacity>0，满足
- **保持**: 
  - 若size < capacity，直接插入，size增1，仍满足
  - 若size == capacity，扩容后capacity'>size，插入后size'=size+1<=capacity'，满足
- **终止**: 算法结束时size增加了1，数组有效

### 4.3 复杂度分析

| 操作 | 数组 | 链表 | 动态数组 |
|------|------|------|----------|
| 随机访问 | O(1) | O(n) | O(1) |
| 头部插入 | O(n) | O(1) | O(n) |
| 尾部插入 | O(1) | O(1)* | O(1)均摊 |
| 中间插入 | O(n) | O(1)* | O(n) |

*需要已知位置指针

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 用栈检查括号匹配

**问题描述**:
给定字符串包含(), {}, []，判断括号是否正确匹配。

**解决过程**:
1. 遍历字符串，遇到左括号入栈
2. 遇到右括号，弹出栈顶检查是否匹配
3. 遍历结束后栈为空则匹配

**结果**: 时间O(n)，空间O(n)

**示例 2**: 用队列实现BFS

**解决过程**:
1. 起点入队
2. 队首元素出队，访问其邻居并入队
3. 重复直到队列为空

**结果**: BFS使用队列保证按层次遍历

### 5.2 代码实现

**语言**: Rust

```rust
// 链表实现
#[derive(Debug)]
struct Node<T> {
    data: T,
    next: Option<Box<Node<T>>>,
}

#[derive(Debug)]
struct LinkedList<T> {
    head: Option<Box<Node<T>>>,
    size: usize,
}

impl<T> LinkedList<T> {
    fn new() -> Self {
        LinkedList { head: None, size: 0 }
    }
    
    fn push_front(&mut self, data: T) {
        let new_node = Box::new(Node {
            data,
            next: self.head.take(),
        });
        self.head = Some(new_node);
        self.size += 1;
    }
    
    fn pop_front(&mut self) -> Option<T> {
        self.head.take().map(|node| {
            self.head = node.next;
            self.size -= 1;
            node.data
        })
    }
}

// 栈实现
struct Stack<T> {
    data: Vec<T>,
}

impl<T> Stack<T> {
    fn new() -> Self {
        Stack { data: Vec::new() }
    }
    
    fn push(&mut self, item: T) {
        self.data.push(item);
    }
    
    fn pop(&mut self) -> Option<T> {
        self.data.pop()
    }
    
    fn peek(&self) -> Option<&T> {
        self.data.last()
    }
    
    fn is_empty(&self) -> bool {
        self.data.is_empty()
    }
}

// 队列实现（循环队列）
struct Queue<T> {
    data: Vec<Option<T>>,
    head: usize,
    tail: usize,
    size: usize,
    capacity: usize,
}

impl<T: Clone> Queue<T> {
    fn new(capacity: usize) -> Self {
        Queue {
            data: vec![None; capacity],
            head: 0,
            tail: 0,
            size: 0,
            capacity,
        }
    }
    
    fn enqueue(&mut self, item: T) -> Result<(), &'static str> {
        if self.size == self.capacity {
            return Err("Queue is full");
        }
        self.data[self.tail] = Some(item);
        self.tail = (self.tail + 1) % self.capacity;
        self.size += 1;
        Ok(())
    }
    
    fn dequeue(&mut self) -> Option<T> {
        if self.size == 0 {
            return None;
        }
        let item = self.data[self.head].take();
        self.head = (self.head + 1) % self.capacity;
        self.size -= 1;
        item
    }
}
```

**代码说明**:
- `LinkedList`: 单链表，支持头部插入/删除O(1)
- `Stack`: 基于Vec实现，利用其动态扩容
- `Queue`: 循环队列，避免出队时的元素移动

### 5.3 反例

**常见错误1**: 链表随机访问
```python
# 错误代码
for i in range(len(linked_list)):
    print(linked_list.get(i))  # 每次O(n)，总O(n^2)
```
**错误原因**: 链表不支持O(1)随机访问，get(i)需要遍历i个节点
**正确做法**: 使用迭代器遍历，或改用数组

**常见错误2**: 栈溢出（递归实现）
```python
# 危险代码
def recursive_function(n):
    if n <= 0: return
    # 处理
    recursive_function(n-1)  # 深度n可能导致栈溢出
```
**错误原因**: 递归深度过大耗尽调用栈
**正确做法**: 改为迭代实现，或增加栈空间限制

## 六、习题

### 6.1 理解题 (L1)

1. **数据结构选择** [难度⭐]
   
   以下场景应选择什么数据结构？
   - (a) 实现浏览器的前进/后退功能
   - (b) 打印机任务队列
   - (c) 需要频繁在两端添加/删除元素
   
   <details>
   <summary>解答</summary>
   
   (a) **栈** - 后进先出符合前进/后退语义
   (b) **队列** - 先进先出保证公平性
   (c) **双端队列(Deque)** - 两端操作都是O(1)
   
   </details>

2. **复杂度分析** [难度⭐]
   
   分析在数组和链表头部插入元素的时间复杂度。
   
   <details>
   <summary>解答</summary>
   
   **数组**: 需要将所有现有元素后移一位，时间O(n)
   
   **链表**: 只需修改头指针，时间O(1)
   
   </details>

### 6.2 应用题 (L2-L3)

1. **最小栈** [难度⭐⭐]
   
   设计一个栈，支持push、pop、getMin操作，所有操作O(1)。
   
   <details>
   <summary>解答</summary>
   
   **方案**: 每个节点存储当前最小值
   
   ```python
   class MinStack:
       def __init__(self):
           self.stack = []
       
       def push(self, x):
           cur_min = min(x, self.stack[-1][1]) if self.stack else x
           self.stack.append((x, cur_min))
       
       def pop(self):
           return self.stack.pop()[0]
       
       def getMin(self):
           return self.stack[-1][1]
   ```
   
   或用辅助栈存储最小值。
   
   </details>

2. **用栈实现队列** [难度⭐⭐]
   
   只用栈操作（push、pop、peek、isEmpty）实现队列。
   
   <details>
   <summary>解答</summary>
   
   **方案**: 使用两个栈，input和output
   
   - enqueue: push到input栈
   - dequeue: 若output为空，将input全部pop并push到output，然后从output pop
   
   **复杂度**: 
   - enqueue: O(1)
   - dequeue: 均摊O(1)（每个元素最多被转移两次）
   
   </details>

### 6.3 证明题 (L4-L5)

1. **双栈队列正确性** [难度⭐⭐⭐]
   
   证明上述双栈实现的队列满足FIFO性质。
   
   <details>
   <summary>解答</summary>
   
   **证明**:
   
   设元素入队顺序为a₁, a₂, ..., aₙ。
   
   入队时依次进入input栈，栈底到栈顶为a₁, a₂, ..., aₙ。
   
   转移到output栈后，output栈底到栈顶为aₙ, ..., a₂, a₁。
   
   出队从output弹出，顺序为a₁, a₂, ..., aₙ，正是入队顺序。
   
   若output为空，新元素会在后续入队，后转移，后出队，保持FIFO。
   ∎
   
   </details>

### 7.2 实际案例

**案例**: 浏览器前进/后退实现

**背景**:
浏览器需要维护用户的浏览历史，支持前进和后退导航。

**应用方式**:
- 使用两个栈：backStack和forwardStack
- 访问新页面：当前页push到backStack，清空forwardStack
- 后退：backStack pop，当前页push到forwardStack
- 前进：forwardStack pop，当前页push到backStack

**效果**:
- O(1)时间完成导航操作
- 自然地处理导航历史分支

### 7.3 跨领域联系

**与函数式编程的联系**:
链表是函数式语言的基础数据结构，因其支持持久化（共享结构）和递归处理。

**与硬件架构的联系**:
数组的顺序访问模式符合CPU缓存的预取机制，而链表的随机访问模式缓存不友好。

## 八、多维对比

### 8.1 方法对比

| 维度 | 数组 | 链表 | 动态数组 | 链式栈/队列 |
|------|------|------|----------|-------------|
| 随机访问 | O(1) | O(n) | O(1) | N/A |
| 插入/删除 | O(n) | O(1)* | O(1)均摊 | O(1) |
| 空间开销 | 低 | 高（指针） | 可能浪费 | 高 |
| 缓存友好 | 是 | 否 | 是 | 否 |
| 内存分配 | 连续 | 分散 | 连续 | 分散 |

*需要位置指针

### 8.2 决策建议

**何时选择数组**:
- 需要频繁随机访问
- 数据大小固定或变化不大
- 内存空间有限

**何时选择链表**:
- 需要频繁在中间插入/删除
- 数据大小变化剧烈
- 实现复杂数据结构（如图、树）

**何时选择动态数组**:
- 需要随机访问且大小变化
- 主要在尾部操作
- 希望简化内存管理

**决策流程图**:
```
需要随机访问？
├── 是 → 数组或动态数组
│         大小固定？
│         ├── 是 → 静态数组
│         └── 否 → 动态数组
└── 否 → 主要在首尾操作？
          ├── 是 → 栈/队列/双端队列
          └── 否 → 链表（频繁中间操作）
```

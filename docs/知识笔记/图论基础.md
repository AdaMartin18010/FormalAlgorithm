## 一、核心概念

### 1.1 定义

**正式定义**:
图G是由顶点集V和边集E组成的二元组G = (V, E)，其中E是V中顶点对的集合。

**有向图**: 边有方向，E ⊆ V × V，边(u, v) ≠ (v, u)
**无向图**: 边无方向，E是无序顶点对的集合，边{u, v} = {v, u}

**形式化描述**:
```
图 G = (V, E)
- V: 顶点集，|V| = n
- E: 边集，|E| = m

有向图: 边(u,v)从u指向v
  - 入度deg⁻(v): 指向v的边数
  - 出度deg⁺(v): 从v出发的边数

无向图: 边{u,v}连接u和v
  - 度deg(v): 与v关联的边数
```

**直观解释**:
图就像社交网络：
- 顶点 = 人
- 边 = 关系（朋友、关注）
- 有向图 = Twitter（关注关系单向）
- 无向图 = Facebook（朋友关系双向）

**关键要点**:
- 图是表达**关系**的最通用数据结构
- 稀疏图(m ≈ n)和稠密图(m ≈ n²)需要不同表示方法
- 路径、环、连通分量是图的基本概念
- 图的遍历是所有图算法的基础

### 1.2 属性

| 属性 | 描述 | 公式/备注 |
|------|------|-----------|
| 度 | 顶点关联的边数 | 无向图: Σdeg(v) = 2m |
| 路径 | 顶点序列，相邻顶点有边相连 | 简单路径: 无重复顶点 |
| 环 | 起点=终点的路径 | 简单环: 除起点外无重复顶点 |
| 连通 | 存在路径 | 无向图连通: 任意两点有路径 |
| 强连通 | 双向可达 | 有向图强连通: u↔v都有路径 |
| 子图 | 顶点和边都是原图的子集 | 诱导子图: 包含原图所有相关边 |

**性质总结**:
1. **握手定理**: 无向图中所有顶点度数之和等于2倍边数
2. **有向图度**: Σdeg⁺(v) = Σdeg⁻(v) = m
3. **树**: 连通无环图，n个顶点的树有n-1条边

### 1.3 变体

**加权图**:
- 定义: 每条边关联一个权重w: E → ℝ
- 与标准形式的区别: 需要考虑边权的最短路径、最小生成树等
- 适用场景: 网络路由、地图导航、资源分配

**多重图**:
- 定义: 允许同一对顶点间有多条边（平行边）
- 与标准形式的区别: 简单图不允许平行边和自环
- 适用场景: 网络冗余、多通道系统

**二分图**:
- 定义: 顶点可分为两组，组内无边
- 与标准形式的区别: 所有边连接两组顶点
- 适用场景: 匹配问题、任务分配

## 三、形式化证明

### 3.1 核心定理

**定理 1** (握手定理): 在无向图G = (V, E)中，所有顶点度数之和等于2|E|。

**证明**:
```
每条边e = {u, v}对度数之和贡献2：
- 为deg(u)贡献1
- 为deg(v)贡献1

设边集为{e₁, e₂, ..., eₘ}，则：
Σ(v∈V) deg(v) = Σ(i=1 to m) 2 = 2m = 2|E|

∎
```

**证明要点分析**:
1. **计数论证**: 从边的角度计算度数贡献
2. **双重计数**: 每条边被计算两次（两端各一次）

**直觉理解**:
每条边都有两个端点，握住这条边的两只手分别属于两个顶点。

**推论**: 任何图中度数为奇数的顶点个数必为偶数。

### 3.2 辅助引理

**引理 1** (树的性质): 设G是n个顶点的无向图，以下陈述等价：
1. G是树（连通无环）
2. G连通且有n-1条边
3. G无环且有n-1条边
4. G中任意两点有唯一简单路径

*证明概要*:
```
(1)⇒(2): 对n归纳。n=1时0条边。n>1时树有叶子，
         删除叶子得n-1个顶点的树，由归纳假设有n-2条边，
         原树有n-1条边。

(2)⇒(3): 连通+恰好n-1条边 ⇒ 删除任一边则不连通
         （否则存在生成树需要n-1条边，矛盾）
         因此无环。

(3)⇒(1): 无环图是森林。n个顶点k个连通分量的森林有n-k条边。
         n-k = n-1 ⇒ k=1，即连通，是树。

(1)⇔(4): 连通保证存在路径，无环保证路径唯一。
∎
```

## 四、算法/方法详解

### 4.1 算法描述

**图的存储实现**:

**邻接表（推荐用于稀疏图）**:
```
算法: 构建邻接表
输入: 顶点数n，边列表edges
输出: 邻接表G

1. 初始化G[0..n-1]为空列表
2. for each (u, v) in edges do
3.     G[u].append(v)
4.     if 无向图 then
5.         G[v].append(u)
6.     end if
7. end for
8. return G
```

**邻接矩阵（推荐用于稠密图）**:
```
算法: 构建邻接矩阵
输入: 顶点数n，边列表edges
输出: n×n矩阵M

1. 初始化M[n][n] = 全0
2. for each (u, v, w) in edges do
3.     M[u][v] = w
4.     if 无向图 then
5.         M[v][u] = w
6.     end if
7. end for
8. return M
```

### 4.2 正确性分析

**邻接表正确性**:

**不变式**: 处理完前k条边后，对于每条边(u,v)，v在G[u]中当且仅当该边已处理。

**证明**:
- **初始化**: k=0，所有列表为空，正确
- **保持**: 处理边(u,v)时，将v加入G[u]，保持性质
- **终止**: 所有边处理完毕，邻接表正确表示图

### 4.3 复杂度分析

| 操作 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| 空间 | O(n²) | O(n + m) |
| 添加边 | O(1) | O(1) |
| 删除边 | O(1) | O(deg(v)) |
| 查询边(u,v) | O(1) | O(deg(u)) |
| 遍历邻居 | O(n) | O(deg(v)) |
| 遍历全图 | O(n²) | O(n + m) |

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 判断二分图

**问题描述**:
给定无向图，判断是否为二分图（可用两种颜色染色，相邻顶点不同色）。

**解决过程**:
```
用BFS/DFS进行染色：
1. 任选未染色顶点，染为颜色0
2. 对其所有邻居，染为颜色1
3. 继续BFS/DFS，若发现邻居已染同色，则不是二分图
4. 若遍历完无冲突，则是二分图
```

**结果**: O(n + m)时间判断二分图

**示例 2**: 计算连通分量

**解决过程**:
```
1. 初始化所有顶点为未访问
2. 对每个未访问顶点，进行DFS/BFS
3. 每次DFS/BFS访问的所有顶点属于同一连通分量
4. 计数DFS/BFS调用次数 = 连通分量数
```

**结果**: O(n + m)时间计算所有连通分量

### 5.2 代码实现

**语言**: Python

```python
from collections import defaultdict, deque

class Graph:
    def __init__(self, n, directed=False):
        self.n = n
        self.directed = directed
        self.adj = defaultdict(list)
    
    def add_edge(self, u, v, weight=1):
        self.adj[u].append((v, weight))
        if not self.directed:
            self.adj[v].append((u, weight))
    
    def bfs(self, start):
        """返回从start可达的所有顶点"""
        visited = [False] * self.n
        queue = deque([start])
        visited[start] = True
        result = [start]
        
        while queue:
            u = queue.popleft()
            for v, _ in self.adj[u]:
                if not visited[v]:
                    visited[v] = True
                    queue.append(v)
                    result.append(v)
        return result
    
    def dfs(self, start):
        """返回DFS遍历顺序"""
        visited = [False] * self.n
        result = []
        
        def dfs_recursive(u):
            visited[u] = True
            result.append(u)
            for v, _ in self.adj[u]:
                if not visited[v]:
                    dfs_recursive(v)
        
        dfs_recursive(start)
        return result
    
    def is_bipartite(self):
        """判断是否为二分图"""
        color = [-1] * self.n
        
        for i in range(self.n):
            if color[i] == -1:
                queue = deque([i])
                color[i] = 0
                
                while queue:
                    u = queue.popleft()
                    for v, _ in self.adj[u]:
                        if color[v] == -1:
                            color[v] = color[u] ^ 1
                            queue.append(v)
                        elif color[v] == color[u]:
                            return False
        return True
    
    def connected_components(self):
        """返回所有连通分量"""
        visited = [False] * self.n
        components = []
        
        for i in range(self.n):
            if not visited[i]:
                comp = []
                stack = [i]
                visited[i] = True
                
                while stack:
                    u = stack.pop()
                    comp.append(u)
                    for v, _ in self.adj[u]:
                        if not visited[v]:
                            visited[v] = True
                            stack.append(v)
                
                components.append(comp)
        
        return components


# Rust实现
# src/graph.rs
'''
pub struct Graph {
    n: usize,
    adj: Vec<Vec<(usize, i32)>>,
}

impl Graph {
    pub fn new(n: usize) -> Self {
        Graph {
            n,
            adj: vec![Vec::new(); n],
        }
    }
    
    pub fn add_edge(&mut self, u: usize, v: usize, w: i32) {
        self.adj[u].push((v, w));
        self.adj[v].push((u, w));  // 无向图
    }
    
    pub fn bfs(&self, start: usize) -> Vec<usize> {
        let mut visited = vec![false; self.n];
        let mut queue = std::collections::VecDeque::new();
        let mut result = Vec::new();
        
        visited[start] = true;
        queue.push_back(start);
        
        while let Some(u) = queue.pop_front() {
            result.push(u);
            for &(v, _) in &self.adj[u] {
                if !visited[v] {
                    visited[v] = true;
                    queue.push_back(v);
                }
            }
        }
        
        result
    }
}
'''
```

**代码说明**:
- `Graph`类支持有向/无向图
- `add_edge`: 添加带权边
- `bfs`/`dfs`: 图遍历实现
- `is_bipartite`: BFS染色判断二分图
- `connected_components`: 找出所有连通分量

## 5.3 反例
### 5.3 反例

**常见错误1**: 忘记标记已访问
```python
# 错误代码
def dfs_bad(graph, u):
    print(u)
    for v in graph[u]:
        dfs_bad(graph, v)  # 无限递归！
```
**错误原因**: 没有visited数组，导致无限循环
**正确做法**: 添加visited标记或检测父节点

**常见错误2**: 有向图当无向图处理
```python
# 错误：有向图中错误地使用双向添加
def add_edge_wrong(u, v):
    adj[u].append(v)
    adj[v].append(u)  # 有向图不应这样！
```
**错误原因**: 混淆有向图和无向图
**正确做法**: 明确图类型，有向图只添加单向边

## 六、习题

### 6.1 理解题 (L1)

1. **图的基本计算** [难度⭐]
   
   一个无向图有10个顶点，每个顶点度数为3，它有多少条边？
   
   <details>
   <summary>解答</summary>
   
   由握手定理：Σdeg(v) = 2m
   
   10 × 3 = 2m ⇒ m = 15
   
   该图有15条边。
   
   </details>

2. **图表示选择** [难度⭐]
   
   稠密图（m ≈ n²）和稀疏图（m ≈ n）分别适合什么表示方法？
   
   <details>
   <summary>解答</summary>
   
   **稠密图**: 邻接矩阵
   - 空间O(n²)可接受
   - 查询边O(1)
   
   **稀疏图**: 邻接表
   - 空间O(n+m)远小于O(n²)
   - 遍历邻居O(deg(v))高效
   
   </details>

### 6.2 应用题 (L2-L3)

1. **判断树** [难度⭐⭐]
   
   给定n个节点和n-1条边的无向图，判断它是否是树。
   
   <details>
   <summary>解答</summary>
   
   ```python
   def is_tree(n, edges):
       if len(edges) != n - 1:
           return False
       
       # 构建图并检查连通性
       adj = [[] for _ in range(n)]
       for u, v in edges:
           adj[u].append(v)
           adj[v].append(u)
       
       visited = [False] * n
       
       def dfs(u, parent):
           visited[u] = True
           for v in adj[u]:
               if v != parent:
                   if visited[v]:
                       return False  # 有环
                   if not dfs(v, u):
                       return False
           return True
       
       if not dfs(0, -1):
           return False
       
       return all(visited)  # 是否连通
   ```
   
   </details>

2. **星系战争** [难度⭐⭐]
   
   n个星系，m条虫洞连接。求至少需要摧毁多少虫洞，使得某个指定星系与其他星系不连通。
   
   <details>
   <summary>解答</summary>
   
   这是求**点连通度**问题。
   
   对于指定星系s，答案等于：
   - 与s直接相连的星系数量（最小割）
   
   或者说，需要切断所有与s直接相连的边。
   
   若图是连通的，答案为deg(s)。
   
   </details>

### 6.3 证明题 (L4-L5)

1. **DAG性质** [难度⭐⭐⭐]
   
   证明：有向图是DAG（有向无环图）当且仅当存在拓扑排序。
   
   <details>
   <summary>解答</summary>
   
   **（⇒）DAG⇒拓扑排序**:
   
   对顶点数n归纳。
   
   基例n=1，显然。
   
   DAG必有入度为0的顶点（否则每个顶点有前驱，形成环）。
   取入度为0的顶点v作为第一个，删除v后仍是DAG，由归纳假设剩余有拓扑排序。
   
   **（⇐）拓扑排序⇒DAG**:
   
   假设有拓扑排序v₁, v₂, ..., vₙ但存在环。
   环中必有边(vᵢ, vⱼ)其中i > j，与拓扑排序矛盾。
   
   ∎
   
   </details>

### 8.1 方法对比

### 8.2 决策建议

**如何选择图表示**:
- m < n²/10（稀疏）→ 邻接表
- m > n²/3（稠密）→ 邻接矩阵
- 需要快速查询边是否存在 → 邻接矩阵
- 内存受限 → 邻接表

**如何选择遍历算法**:
- 找最短路径（无权）→ BFS
- 检测环、拓扑排序 → DFS
- 分层遍历、最近邻居 → BFS

**决策流程图**:
```
图是稠密还是稀疏？
├── 稠密(m≈n²) → 邻接矩阵
└── 稀疏(m≈n) → 邻接表

需要找最短路径？
├── 无权图 → BFS
├── 正权图 → Dijkstra
└── 有负权 → Bellman-Ford

需要连通性？
├── 静态 → DFS/BFS找连通分量
└── 动态 → 并查集
```

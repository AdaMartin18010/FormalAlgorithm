## 一、核心概念

### 1.1 定义

**正式定义**:
数字签名方案（Digital Signature Scheme）是一个三元组 $\Pi = (\text{Gen}, \text{Sign}, \text{Verify})$，其中：

1. **密钥生成算法** $\text{Gen}$：概率算法，输入安全参数 $1^n$，输出密钥对 $(pk, sk)$
2. **签名算法** $\text{Sign}$：可能概率算法，输入私钥 $sk$ 和消息 $m \in \{0,1\}^*$，输出签名 $\sigma \leftarrow \text{Sign}_{sk}(m)$
3. **验证算法** $\text{Verify}$：确定性算法，输入公钥 $pk$、消息 $m$ 和签名 $\sigma$，输出比特 $b := \text{Verify}_{pk}(m, \sigma)$，满足：
$$\Pr[\text{Verify}_{pk}(m, \text{Sign}_{sk}(m)) = 1] = 1$$

**安全定义（EUF-CMA）**:
签名方案在选择消息攻击下存在性不可伪造，如果对于所有PPT敌手 $\mathcal{A}$：
$$\Pr[\text{Exp}^{\text{euf-cma}}_{\Pi, \mathcal{A}}(n) = 1] \leq \text{negl}(n)$$

其中实验定义为：

1. 生成 $(pk, sk) \leftarrow \text{Gen}(1^n)$
2. $\mathcal{A}$ 可自适应查询签名预言机 $\text{Sign}_{sk}(\cdot)$ 获得消息 $m_i$ 的签名
3. $\mathcal{A}$ 输出 $(m^*, \sigma^*)$，其中 $m^*$ 未被查询
4. 若 $\text{Verify}_{pk}(m^*, \sigma^*) = 1$，敌手获胜

**直观解释**:
数字签名就像用私钥盖"电子印章"：只有印章所有者能盖章（签名），任何人都能验证印章真伪（验证）。与手写签名不同，数字签名与消息绑定，无法转移到其他消息上。

**关键要点**:

- **不可否认性**: 签名者不能否认已签名消息
- **完整性**: 消息不能被篡改而不破坏签名
- **认证性**: 验证签名者身份
- **非重用性**: 签名与特定消息绑定

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 存在性不可伪造 | 无法生成任何新消息的有效签名 | EUF-CMA安全 |
| 强不可伪造 | 无法生成任何新签名（即使对已知消息） | SUF-CMA更强 |
| 不可抵赖性 | 签名者不能否认签名 | 法律属性 |
| 可传递性 | 验证者可以向第三方证明签名有效 | 基于公钥的公开验证 |

**性质总结**:

1. **正确性**: 合法签名必被验证通过
2. **不可伪造性**: 无私钥者无法伪造签名
3. **确定性/概率性**: 现代方案通常为概率性（如ECDSA、EdDSA）
4. **可聚合性**: 某些方案支持多个签名聚合（BLS）

### 1.3 变体

**RSA-PSS**:

- 定义: Probabilistic Signature Scheme，RSA的概率签名方案
- 与RSA-PKCS1v1.5的区别: 概率性、可证明安全
- 适用场景: 现代RSA签名标准

**ECDSA**:

- 定义: Elliptic Curve Digital Signature Algorithm
- 与DSA的区别: 使用椭圆曲线而非模幂运算
- 适用场景: 加密货币、TLS证书

**EdDSA（Ed25519）**:

- 定义: Edwards曲线数字签名算法
- 与ECDSA的区别: 确定性签名、更快、更安全
- 适用场景: 现代系统（SSH、TLS 1.3）

## 三、形式化证明

### 3.1 核心定理

**定理 1 (RSA-FDH的安全性)**: 在随机预言机模型下，若RSA问题是困难的，则Full-Domain Hash RSA签名方案是EUF-CMA安全的。

**证明**:

设敌手 $\mathcal{A}$ 能以不可忽略概率 $\epsilon$ 伪造RSA-FDH签名。

构造归约 $\mathcal{B}$ 破解RSA问题：

1. $\mathcal{B}$ 接收RSA挑战 $(N, e, y)$，目标是找到 $y^{1/e} \mod N$
2. $\mathcal{B}$ 设置公钥 $pk = (N, e)$，模拟 $\mathcal{A}$ 的环境
3. $\mathcal{B}$ 维护随机预言机 $H$ 的查询列表
4. 对于签名查询 $m_i$：
   - $\mathcal{B}$ 随机选择 $r_i$，设置 $H(m_i) = r_i^e \mod N$
   - 返回签名 $\sigma_i = r_i$
5. 若 $\mathcal{A}$ 输出伪造 $(m^*, \sigma^*)$，其中 $H(m^*)$ 未被编程
6. $\mathcal{B}$ 重绕 $\mathcal{A}$，改变 $H(m^*)$ 的应答
7. 由分叉引理，以概率获得两个不同哈希值 $h_1, h_2$ 使得 $\sigma^e = h_1 = h_2 \cdot \alpha$ 对某个 $\alpha$
8. 提取 $y^{1/e}$

由于RSA假设，$\epsilon$ 必须是可忽略的。

∎

**证明要点分析**:

1. 使用随机预言机模型模拟哈希函数
2. 重绕技术是证明的核心
3. 归约效率损失与敌手成功概率相关

**定理 2 (Schnorr签名的安全性)**: 在离散对数假设下，Schnorr签名方案在随机预言机模型中是EUF-CMA安全的。

*证明概要*: 类似RSA-FDH证明，利用离散对数的困难性。∎

### 3.2 辅助引理

**引理 1 (哈希函数的必要性)**: 直接对消息进行RSA签名（不先哈希）是不安全的。

*证明*: 设签名 $\sigma = m^d \mod N$。攻击者可选择 $\sigma$，计算 $m = \sigma^e \mod N$，则 $(m, \sigma)$ 是有效签名对。这称为"选择签名攻击"。∎

**引理 2 (确定性vs概率性签名)**: 确定性签名（如EdDSA）可通过随机数派生达到与概率性签名（如ECDSA）相当的安全性，且避免随机数重用问题。

*证明*: EdDSA从私钥和消息派生随机数，确保唯一性。若底层哈希是抗碰撞的，则伪造难度与ECDSA相当。∎

## 四、算法/方法详解

### 4.1 算法描述

**Schnorr签名**:

```
算法: Schnorr-Sign
输入: 私钥 x，消息 m，群参数 (G, q)
输出: 签名 (e, s)

1. 选择随机数 k ∈ [1, q-1]
2. 计算 R = kG  // 群运算
3. 计算 e = H(R || m)  // 哈希挑战
4. 计算 s = k - x*e mod q
5. return (e, s)
```

```
算法: Schnorr-Verify
输入: 公钥 P = xG，消息 m，签名 (e, s)
输出: 接受/拒绝

1. 计算 R' = sG + eP
2. 计算 e' = H(R' || m)
3. if e' = e then return 接受
4. else return 拒绝
```

**ECDSA签名**:

```
算法: ECDSA-Sign
输入: 私钥 d，消息哈希 z，曲线参数 (G, n)
输出: 签名 (r, s)

1. 选择密码学安全随机数 k ∈ [1, n-1]
2. 计算 (x1, y1) = kG
3. r = x1 mod n; if r = 0 goto 1
4. s = k^(-1)(z + r*d) mod n; if s = 0 goto 1
5. return (r, s)
```

**Ed25519签名**:

```
算法: Ed25519-Sign
输入: 私钥 k（32字节），消息 M
输出: 签名 (R, S)（64字节）

1. 从k派生：
   - h = SHA-512(k)
   - a = 2^254 + Σ h[i]*2^i (截断并置位)
   - 前缀 = h[32:64]
2. 计算 A = aB  // B为基点
3. r = SHA-512(前缀 || M) mod l
4. R = rB
5. k = SHA-512(R || A || M) mod l
6. S = (r + k*a) mod l
7. return (R, S)
```

**流程图**:

```
签名流程:                验证流程:
私钥 + 消息              公钥 + 消息 + 签名
    ↓                        ↓
随机数k / 派生r          重构验证点
    ↓                        ↓
计算签名值              比较哈希/坐标
    ↓                        ↓
输出签名              接受/拒绝
```

### 4.2 正确性分析

**Schnorr验证的正确性**:
验证方程：$R' = sG + eP = (k - xe)G + exG = kG = R$

因此 $H(R' || m) = H(R || m) = e$，验证通过。

**ECDSA验证的正确性**:
$u_1 G + u_2 Q = (z/s)G + (r/s)(dG) = ((z + rd)/s)G = kG = (x_1, y_1)$

由于 $s = k^{-1}(z + rd)$，所以 $(z + rd)/s = k$。

### 4.3 复杂度分析

**签名复杂度**:

- RSA-PSS: 1次模幂 $O(n^3)$
- ECDSA: 1次标量乘法 $O(\log n)$ 次群运算
- Ed25519: 1次标量乘法，常数时间实现

**验证复杂度**:

- RSA-PSS: 1次模幂 + 1次哈希
- ECDSA: 2次标量乘法（可用Shamir技巧优化）
- Ed25519: 2次标量乘法（固定基点优化）

**签名大小**:

- RSA-2048: 256字节
- ECDSA P-256: 64字节
- Ed25519: 64字节

### 5.1 标准示例

**示例 1: Ed25519完整流程**

**参数** (简化表示):

- 曲线: Edwards25519 ($-x^2 + y^2 = 1 + dx^2y^2$)
- 素数: $p = 2^{255} - 19$
- 阶: $l = 2^{252} + 27742317777372353535851937790883648493$

**签名过程** (数值简化):

1. 私钥: 32字节随机种子
2. 公钥: $A = aB$ (32字节)
3. 签名: $(R, S)$ 各32字节

**验证**:
检查群方程: $8SB = 8R + 8kA$，其中 $k = H(R || A || M)$

**示例 2: X.509证书验证链**

```
根CA证书
   签名 → 中间CA1证书
           签名 → 中间CA2证书
                   签名 → 终端实体证书 (服务器)
```

验证过程:

1. 验证根CA自签名
2. 用根CA公钥验证中间CA1
3. 用中间CA1公钥验证中间CA2
4. 用中间CA2公钥验证终端证书
5. 检查有效期、撤销状态（CRL/OCSP）

### 5.2 代码实现

**Python实现 (Ed25519简化版)**:

```python
import hashlib
import os

# 注意：这是教学简化版，生产环境应使用cryptography库

class Ed25519Simplified:
    """Ed25519简化实现（仅用于教学）"""

    def __init__(self):
        # Ed25519参数（简化）
        self.p = 2**255 - 19
        self.l = 2**252 + 27742317777372353535851937790883648493
        self.d = -121665 * pow(121666, -1, self.p) % self.p
        self.B = self._base_point()

    def _base_point(self):
        # Ed25519基点（y坐标为4/5）
        y = 4 * pow(5, -1, self.p) % self.p
        x = self._recover_x(y, 0)
        return (x, y)

    def _recover_x(self, y, sign):
        # 从y坐标恢复x
        xx = (y*y - 1) * pow(self.d*y*y + 1, -1, self.p) % self.p
        x = pow(xx, (self.p + 3) // 8, self.p)
        if (x*x - xx) % self.p != 0:
            x = x * pow(2, (self.p - 1) // 4, self.p) % self.p
        if x % 2 != sign:
            x = self.p - x
        return x

    def _point_add(self, P, Q):
        # Edwards曲线点加
        if P is None:
            return Q
        if Q is None:
            return P
        x1, y1 = P
        x2, y2 = Q
        x3 = (x1*y2 + x2*y1) * pow(1 + self.d*x1*x2*y1*y2, -1, self.p) % self.p
        y3 = (y1*y2 + x1*x2) * pow(1 - self.d*x1*x2*y1*y2, -1, self.p) % self.p
        return (x3, y3)

    def _scalar_mult(self, k, P):
        # 标量乘法
        Q = None
        while k > 0:
            if k & 1:
                Q = self._point_add(Q, P)
            P = self._point_add(P, P)
            k >>= 1
        return Q

    def _hash(self, data):
        return int.from_bytes(hashlib.sha512(data).digest(), 'little')

    def generate_keypair(self):
        """生成密钥对"""
        sk = os.urandom(32)
        h = hashlib.sha512(sk).digest()
        a = int.from_bytes(h[:32], 'little')
        a &= (1 << 254) - 8
        a |= (1 << 254)
        A = self._scalar_mult(a, self.B)
        pk = bytes([A[0] & 1]) + A[1].to_bytes(32, 'little')
        return sk, pk

    def sign(self, sk, message):
        """签名"""
        h = hashlib.sha512(sk).digest()
        a = int.from_bytes(h[:32], 'little')
        a &= (1 << 254) - 8
        a |= (1 << 254)
        prefix = h[32:]

        A = self._scalar_mult(a, self.B)

        r = self._hash(prefix + message) % self.l
        R = self._scalar_mult(r, self.B)

        k = self._hash(R[1].to_bytes(32, 'little') + A[1].to_bytes(32, 'little') + message) % self.l
        S = (r + k * a) % self.l

        return R[1].to_bytes(32, 'little') + S.to_bytes(32, 'little')

# 使用cryptography库的实际示例
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey

def demo_ed25519():
    # 生成密钥
    private_key = Ed25519PrivateKey.generate()
    public_key = private_key.public_key()

    # 签名
    message = b"Hello, Ed25519!"
    signature = private_key.sign(message)

    # 验证
    try:
        public_key.verify(signature, message)
        print("签名验证成功")
    except:
        print("签名验证失败")

    return signature.hex()

if __name__ == "__main__":
    sig = demo_ed25519()
    print(f"签名: {sig}")
```

### 5.3 反例

**常见错误**:

**错误1: ECDSA随机数重用**

```python
# 错误：使用固定随机数
k = 12345  # 固定值！
r, s = ecdsa_sign(private_key, message, k=k)
# 若同一k用于两条消息，可计算私钥
```

**错误原因**: 若 $s_1 = k^{-1}(z_1 + rd)$，$s_2 = k^{-1}(z_2 + rd)$，可解出 $k$ 和 $d$
**正确做法**: 每次签名使用密码学安全随机数，或使用Ed25519的确定性签名

**错误2: 哈希碰撞攻击**

```python
# 错误：使用弱哈希函数
hash_value = md5(message)  # 不安全
signature = rsa_sign(private_key, hash_value)
# MD5碰撞可导致签名伪造
```

**错误原因**: MD5已被破解，可构造碰撞
**正确做法**: 使用SHA-256或更强的哈希函数

## 六、习题

### 6.1 理解题

1. **解释**: 为什么数字签名使用私钥签名、公钥验证，而不是相反？

   <details>
   <summary>解答</summary>

   若用公钥签名、私钥验证：
   1. 任何人都知道公钥，都能"签名"
   2. 失去身份认证功能
   3. 无法实现不可否认性

   正确设计利用陷门函数的单向性：
   - 陷门（私钥）用于执行困难操作的逆（签名）
   - 公开方向（公钥）用于验证
   - 只有私钥持有者能签名，任何人能验证
   </details>

### 6.2 应用题

1. **设计**: 实现一个简单的PKI系统，包含CA证书颁发和证书链验证功能。

   <details>
   <summary>解答</summary>

   ```python
   from cryptography import x509
   from cryptography.x509.oid import NameOID
   from cryptography.hazmat.primitives import hashes, serialization
   from cryptography.hazmat.primitives.asymmetric import rsa
   from datetime import datetime, timedelta

   class SimpleCA:
       def __init__(self, name):
           self.private_key = rsa.generate_private_key(
               public_exponent=65537,
               key_size=2048,
           )
           self.name = x509.Name([
               x509.NameAttribute(NameOID.COMMON_NAME, name),
           ])
           # 自签名根证书
           self.certificate = self._create_certificate(self.name, self.name, True)

       def _create_certificate(self, subject_name, issuer_name, is_ca, public_key=None):
           if public_key is None:
               public_key = self.private_key.public_key()

           builder = x509.CertificateBuilder()
           builder = builder.subject_name(subject_name)
           builder = builder.issuer_name(issuer_name)
           builder = builder.public_key(public_key)
           builder = builder.serial_number(x509.random_serial_number())
           builder = builder.not_valid_before(datetime.utcnow())
           builder = builder.not_valid_after(datetime.utcnow() + timedelta(days=365))

           if is_ca:
               builder = builder.add_extension(
                   x509.BasicConstraints(ca=True, path_length=None),
                   critical=True,
               )

           return builder.sign(self.private_key, hashes.SHA256())

       def issue_certificate(self, subject_name, subject_public_key):
           return self._create_certificate(subject_name, self.name, False, subject_public_key)

   def verify_certificate_chain(cert_chain, root_ca_cert):
       """验证证书链"""
       for i in range(len(cert_chain) - 1):
           cert = cert_chain[i]
           issuer_cert = cert_chain[i + 1] if i + 1 < len(cert_chain) else root_ca_cert

           # 验证签名
           issuer_cert.public_key().verify(
               cert.signature,
               cert.tbs_certificate_bytes,
               cert.signature_algorithm_parameters,
           )

           # 验证有效期
           if not cert.not_valid_before <= datetime.utcnow() <= cert.not_valid_after:
               return False

       return True
   ```

   </details>

### 6.3 证明题

1. **证明**: 若哈希函数 $H$ 是抗碰撞的，则RSA-FDH签名方案在选择消息攻击下存在性不可伪造（EUF-CMA）。

   <details>
   <summary>解答</summary>

   通过归约证明：

   设敌手 $\mathcal{A}$ 能伪造RSA-FDH签名，构造敌手 $\mathcal{B}$ 破解RSA。

   1. $\mathcal{B}$ 接收RSA挑战 $(N, e, y)$
   2. $\mathcal{B}$ 模拟随机预言机 $H$：
      - 对查询 $m_i$，随机选择 $h_i \in \mathbb{Z}_N^*$，若 $h_i = y$ 则直接得到签名
      - 返回 $h_i$
   3. 对签名查询：若 $H(m_i) \neq y$，选择 $r_i$，设置 $H(m_i) = r_i^e$，返回 $r_i$
   4. 若 $\mathcal{A}$ 输出伪造 $(m^*, \sigma^*)$ 且 $H(m^*) = y$，则 $\sigma^* = y^{1/e}$
   5. 由分叉引理，$\mathcal{B}$ 成功概率与 $\mathcal{A}$ 相当

   由于RSA假设，\mathcal{A}$ 成功概率可忽略。
   </details>

## 八、多维对比

### 8.1 方法对比

| 维度 | RSA-PSS | ECDSA | Ed25519 |
|------|---------|-------|---------|
| 密钥大小 | 2048+位 | 256位 | 256位 |
| 签名大小 | 256字节 | 64字节 | 64字节 |
| 签名速度 | 慢 | 快 | 很快 |
| 验证速度 | 中等 | 中等 | 很快 |
| 侧信道抵抗 | 需防护 | 需防护 | 内置 |
| 确定性 | 概率性 | 概率性 | 确定性 |

### 8.2 决策建议

**何时选择RSA-PSS**:

- 需要与遗留系统兼容
- FIPS合规要求
- 签名大小非关键

**何时选择Ed25519**:

- 新系统设计
- 高性能要求
- 侧信道攻击风险

## 一、规范定义深化

### 1.1 算法类型 judgment

算法类型理论将**算法复杂度**引入类型系统，形成**资源敏感类型**（resource-aware types）。核心判断扩展为：

$$
\Gamma \vdash e : \tau \@ \mathcal{C}
$$

其中 $@ \mathcal{C}$ 表示项 $e$ 的计算资源消耗（时间/空间复杂度）。

**复杂度类型语法**：

$$
\mathcal{C} ::= O(1) \mid O(\log n) \mid O(n) \mid O(n \log n) \mid O(n^k) \mid O(2^n) \mid \mathcal{C}_1 + \mathcal{C}_2 \mid \mathcal{C}_1 \cdot \mathcal{C}_2
$$

### 1.2 资源敏感的类型规则

**时间复杂度规则示例**：

$$
\frac{\Gamma \vdash e_1 : \tau_1 \@ \mathcal{C}_1 \quad \Gamma, x:\tau_1 \vdash e_2 : \tau_2 \@ \mathcal{C}_2}{\Gamma \vdash \text{let } x = e_1 \text{ in } e_2 : \tau_2 \@ \mathcal{C}_1 + \mathcal{C}_2}\;(\text{Let})
$$

$$
\frac{\Gamma \vdash e_1 : \text{Bool} \@ \mathcal{C}_1 \quad \Gamma \vdash e_2 : \tau \@ \mathcal{C}_2 \quad \Gamma \vdash e_3 : \tau \@ \mathcal{C}_3}{\Gamma \vdash \text{if } e_1 \text{ then } e_2 \text{ else } e_3 : \tau \@ \mathcal{C}_1 + \max(\mathcal{C}_2, \mathcal{C}_3)}\;(\text{If})
$$

**递归函数的复杂度约束**：

对于递归定义 $f = \lambda x.\, e$，若每次递归调用将问题规模从 $n$ 减至 $n-1$，且每次调用开销为 $O(1)$，则总复杂度为 $O(n)$。

## 二、模型设计深化

### 2.1 复杂度语义

算法类型的指称语义扩展为**带成本的域** [Danner2015]：

$$
[\![\tau \@ \mathcal{C}]\!] := \{(v, c) \mid v \in [\![\tau]\!], c \in [\![\mathcal{C}]\!]\}
$$

其中 $c$ 为计算成本（步数或时间）。

**语义保持**：若 $\vdash e : \tau \@ O(n)$，则 $e$ 的求值步数上界为 $k \cdot n$（对某常数 $k$）。

### 2.2 Realizability Models

**定义**（实现性解释）：类型 $\tau$ 在复杂度约束 $\mathcal{C}$ 下的实现者为程序 $p$，使得 $p$ 计算 $\tau$ 的某个值且资源消耗满足 $\mathcal{C}$。

**类型提取**（Program Extraction）：从依赖类型证明中提取的算法，其类型自动包含复杂度信息。例如：

- Coq 的 `Extraction` 机制
- Lean 的 `partial`/`unsafe` 标记与复杂度分析

### 2.3  amortized 分析的类型化

**势能方法**（Potential Method）的形式化：

为数据结构 $D$ 分配**势能函数** $\Phi(D)$，操作的分摊成本为：

$$
\hat{c}_i = c_i + \Phi(D_i) - \Phi(D_{i-1})
$$

类型系统可通过**线性类型**或**仿射类型**跟踪势能变化 [Hofmann2000]。

## 三、数学符号与推导

### 3.1 复杂度保持的类型规则

**函数应用**：

$$
\frac{\Gamma \vdash f : \tau_1 \xrightarrow{\mathcal{C}_f} \tau_2 \@ \mathcal{C}_1 \quad \Gamma \vdash x : \tau_1 \@ \mathcal{C}_2}{\Gamma \vdash f(x) : \tau_2 \@ \mathcal{C}_1 + \mathcal{C}_2 + \mathcal{C}_f}\;(\text{App-Comp})
$$

其中 $\xrightarrow{\mathcal{C}_f}$ 表示函数本身带有复杂度注解 $\mathcal{C}_f$。

**递归模式**：

对于分治递归 $T(n) = a \cdot T(n/b) + O(n^d)$，由主定理（Master Theorem）：

$$
T(n) = \begin{cases}
O(n^d) & d > \log_b a \\
O(n^d \log n) & d = \log_b a \\
O(n^{\log_b a}) & d < \log_b a
\end{cases}
$$

### 3.2 排序算法的复杂度类型

| 算法 | 最坏时间 | 平均时间 | 空间 | 稳定性 |
|------|---------|---------|------|--------|
| 插入排序 | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 稳定 |
| 归并排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 稳定 |
| 快速排序 | $O(n^2)$ | $O(n \log n)$ | $O(\log n)$ | 不稳定 |
| 堆排序 | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | 不稳定 |

**类型签名**（带复杂度注解）：

$$
\text{mergeSort} : \forall\alpha.\, (\alpha \to \alpha \to \text{Bool}) \to \text{List}\,\alpha \xrightarrow{O(n \log n)} \text{List}\,\alpha
$$

## 四、示例性代码

以下 Lean 4 代码展示带复杂度意识的算法类型与向量排序。

```lean
/-
算法类型理论的 Lean 4 编码示例
对应主文档：06-算法类型理论.md
-/

namespace AlgorithmTypeTheory

-- 1. 自然数的 Peano 定义（用于复杂度分析） ---------------------------------

inductive PN where
  | zero : PN
  | succ : PN → PN
  deriving Repr

/-- 自然数加法 -/
def PN.add : PN → PN → PN
  | .zero,   n => n
  | .succ m, n => .succ (PN.add m n)

-- 2. 向量类型（长度索引化列表，依赖类型） ----------------------------------

inductive Vector (α : Type) : Nat → Type where
  | nil  : Vector α 0
  | cons : α → Vector α n → Vector α (n + 1)

open Vector

-- 3. 归并排序（类型保证输出有序且长度不变） --------------------------------

/-- 有序列表的谓词（类型级规范） -/
inductive Sorted : List Nat → Prop where
  | nil  : Sorted []
  | single (x : Nat) : Sorted [x]
  | cons (x y : Nat) (ys : List Nat)
      : x ≤ y → Sorted (y :: ys) → Sorted (x :: y :: ys)

/-- 归并两个已排序向量 -/
def merge (v1 v2 : Vector Nat n) : Vector Nat (n + n) :=
  sorry  -- 完整实现需递归拆分

/-- 归并排序：保证输出长度与输入相同 -/
def mergeSort {n : Nat} (v : Vector Nat n) : Vector Nat n :=
  match n, v with
  | 0,   nil        => nil
  | 1,   cons a nil => cons a nil
  | _+2, v          =>
      let (left, right) := split v
      merge (mergeSort left) (mergeSort right)
  -- 复杂度：O(n log n) 由递归结构保证
decreasing_by sorry  -- 终止性证明需补充

-- 4. 复杂度类型标记（概念性） ----------------------------------------------

/-- 带复杂度注解的类型包装器 -/
structure Comp (α : Type) (cost : Nat → Prop) where
  value : α
  costBound : cost 0  -- 简化：常数时间

/-- 常数时间操作 -/
def O1 {α : Type} (a : α) : Comp α (fun _ => True) :=
  { value := a, costBound := trivial }

-- 5. 安全的数组访问（索引边界在类型中） ------------------------------------

/-- 使用 Fin n 保证索引在 [0, n) 范围内 -/
def safeGet {α : Type} {n : Nat} (v : Vector α n) (i : Fin n) : α :=
  match n, v, i with
  | _+1, cons a _,  ⟨0, _⟩    => a
  | _+1, cons _ t, ⟨i+1, h⟩ => safeGet t ⟨i, by simp at h; exact h⟩

end AlgorithmTypeTheory
```

## 五、形式化证明

### 5.1 归并排序的复杂度上界

**定理**：归并排序的时间复杂度为 $O(n \log n)$。

*证明*（递归方程法）：

设 $T(n)$ 为对长度为 $n$ 的列表排序的时间。

1. **拆分**：$O(n)$（遍历列表）
2. **递归排序**：$2 \cdot T(n/2)$
3. **归并**：$O(n)$

递归方程：$T(n) = 2T(n/2) + O(n)$。

由主定理（$a=2, b=2, d=1$）：$\log_b a = 1 = d$，故 $T(n) = O(n \log n)$。∎

### 5.2 类型系统保证的正确性

**定理**：若 $\vdash e : \text{SortedList}\,\alpha$，则 $e$ 的求值结果是一个已排序的列表。

*证明概要*：

- `SortedList` 类型定义为 $\sum_{l : \text{List}\,\alpha} \text{Sorted}(l)$。
- 构造 `SortedList` 的项必须同时提供列表 $l$ 和 `Sorted(l)` 的证明。
- 类型检查器验证证明的正确性。
- 因此任何通过类型检查的程序，其输出列表必然已排序。∎

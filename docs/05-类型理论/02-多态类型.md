### 术语与符号规范 / Terminology & Notation

- 记号约定：$\vdash$ 表示类型推导；$\leq$ 表示类型实例化（特化）关系；$\text{Gen}$ 为泛化操作。

## 1. 基本概念

### 1.1 多态的分类

按照 Cardelli 与 Wegner [CardelliWegner1985] 的经典分类，多态分为：

1. **参数多态（Parametric Polymorphism）**：函数对任意类型统一行为，如 `id : ∀α. α → α`。
2. **特设多态（Ad-hoc Polymorphism）**：同一符号在不同类型上有不同实现，如重载与类型类。
3. **子类型多态（Subtype Polymorphism）**：通过子类型关系实现的多态，如面向对象继承。
4. **强制多态（Coercion Polymorphism）**：通过显式或隐式类型转换实现。

本文档主要关注**参数多态**的形式化理论。

### 1.2 System F（二阶 λ 演算）

System F，又称二阶 λ 演算（$\lambda 2$），由 Girard [Girard1972] 与 Reynolds [Reynolds1974] 独立发现。它是简单类型 λ 演算向多态的扩展，是 Haskell、ML 等语言类型系统的理论原型。

**核心思想**：类型抽象（$\Lambda$）与类型应用（$[\,]$）。

## 2. 类型系统

### 2.1 语法

$$
\begin{aligned}
\text{类型 } \tau &::= \alpha \mid \tau \to \tau \mid \forall\alpha.\tau \\
\text{项 } t &::= x \mid \lambda x:\tau.t \mid t\,t \mid \Lambda\alpha.t \mid t[\tau]
\end{aligned}
$$

- $\Lambda\alpha.t$：对类型变量 $\alpha$ 的抽象（大写 Lambda）。
- $t[\tau]$：将多态项 $t$ 实例化为具体类型 $\tau$。

### 2.2 类型规则

$$
\frac{x:\tau \in \Gamma}{\Gamma \vdash x : \tau}\;(\text{Var})
\qquad
\frac{\Gamma, x:\tau_1 \vdash t : \tau_2}{\Gamma \vdash \lambda x:\tau_1.t : \tau_1 \to \tau_2}\;(\to\text{-Intro})
$$

$$
\frac{\Gamma \vdash t_1 : \tau_1 \to \tau_2 \quad \Gamma \vdash t_2 : \tau_1}{\Gamma \vdash t_1\,t_2 : \tau_2}\;(\to\text{-Elim})
$$

$$
\frac{\Gamma \vdash t : \tau \quad \alpha \notin \text{FV}(\Gamma)}{\Gamma \vdash \Lambda\alpha.t : \forall\alpha.\tau}\;(\forall\text{-Intro})
$$

$$
\frac{\Gamma \vdash t : \forall\alpha.\tau}{\Gamma \vdash t[\tau'] : \tau[\tau'/\alpha]}\;(\forall\text{-Elim})
$$

### 2.3 类型替换与实例化

**定义**（类型替换）：替换 $\sigma$ 是从类型变量到类型的有限映射。扩展为函数 $\hat\sigma$：

- $\hat\sigma(\alpha) = \sigma(\alpha)$（若 $\alpha \in \text{dom}(\sigma)$），否则 $\alpha$
- $\hat\sigma(\tau_1 \to \tau_2) = \hat\sigma(\tau_1) \to \hat\sigma(\tau_2)$
- $\hat\sigma(\forall\alpha.\tau) = \forall\alpha.\hat\sigma(\tau)$（要求 $\alpha$ 不在 $\sigma$ 的值域自由出现）

**定义**（实例化）：$\tau' \leq \tau$ 当且仅当存在替换 $\sigma$ 使得 $\tau' = \sigma(\tau)$。

## 3. Hindley–Milner 类型推断

### 3.1 算法 W

Hindley–Milner 系统 [Milner1978, DamasMilner1982] 是 System F 的受限版本，只允许**前缀多态**（prenex polymorphism）：全称量词必须出现在类型模式的最外层。

**算法 W** 是 HM 系统的经典类型推断算法，基于 Robinson 统一算法 [Robinson1965]：

```text
W(Γ, x)       = (id, τ)                 若 x:τ ∈ Γ
W(Γ, λx.e)    = let (S, τ) = W(Γ∪{x:α}, e)
                    in (S, S(α) → τ)    α 新鲜
W(Γ, e₁ e₂)   = let (S₁, τ₁) = W(Γ, e₁)
                    (S₂, τ₂) = W(S₁Γ, e₂)
                    β 新鲜
                    V = unify(S₂τ₁, τ₂ → β)
                    in (V ∘ S₂ ∘ S₁, V(β))
W(Γ, let x=e₁ in e₂) = let (S₁, τ₁) = W(Γ, e₁)
                           τ' = Gen(S₁Γ, τ₁)
                           (S₂, τ₂) = W(S₁Γ∪{x:τ'}, e₂)
                           in (S₂ ∘ S₁, τ₂)
```

### 3.2 主类型定理

**定理**（主类型 / Principal Typing）[DamasMilner1982]：
若 $\Gamma \vdash e : \tau$ 在 HM 系统中可推导，则存在主类型 $\tau_p$ 使得：
1. $\Gamma \vdash e : \tau_p$；
2. 对任意满足 $\Gamma \vdash e : \tau'$ 的 $\tau'$，有 $\tau' \leq \tau_p$（即 $\tau'$ 是 $\tau_p$ 的实例）。

## 4. 元理论

### 4.1 类型安全

**定理**（类型安全 / Type Safety）：若 $\vdash e : \tau$ 且 $e \to^* e'$，则 $e'$ 要么是值，要么可以进一步归约（Progress），且 $\vdash e' : \tau$（Preservation）。

**证明概要**：
- **Progress**：对推导结构归纳，证明良类型的闭项要么是值，要么存在可归约的子项。
- **Preservation（Subject Reduction）**：对归约关系归纳，利用替换引理证明 $\beta$-归约保持类型。

### 4.2 强规范化

**定理**（强规范化 / Strong Normalization）[Girard1972]：所有良类型的 System F 项都是强规范化的。

**证明方法**：Girard 的可约性候选法（Reducibility Candidates）或 Tait 的可计算性方法。

**不可判定性** [Wells1994]：System F 的类型推断问题是不可判定的。

## 5. 实现示例

### 5.1 System F 的 Church 编码

在 System F 中，数据类型可通过多态编码构造：

**自然数**：
$$\text{Nat} := \forall\alpha.\, (\alpha \to \alpha) \to \alpha \to \alpha$$

- $0 := \Lambda\alpha.\, \lambda s:\alpha\to\alpha.\, \lambda z:\alpha.\, z$
- $\text{succ} := \lambda n:\text{Nat}.\, \Lambda\alpha.\, \lambda s.\, \lambda z.\, s\,(n[\alpha]\,s\,z)$

**布尔值**：
$$\text{Bool} := \forall\alpha.\, \alpha \to \alpha \to \alpha$$

- $\text{true} := \Lambda\alpha.\, \lambda x:\alpha.\, \lambda y:\alpha.\, x$
- $\text{false} := \Lambda\alpha.\, \lambda x:\alpha.\, \lambda y:\alpha.\, y$

**列表**（Church 编码）：
$$\text{List}\,\beta := \forall\alpha.\, (\beta \to \alpha \to \alpha) \to \alpha \to \alpha$$

### 5.2 Lean 4 多态示例

```lean
/- 多态函数与 Church 编码示例（Lean 4） -/

namespace Polymorphism

-- 1. 参数多态：与 System F 的 ∀α.α→α 对应
/-- 多态恒等函数 -/
def polyId {α : Type} (x : α) : α := x

-- 2. Church 编码的布尔值
/-- Church 编码的 true：选择第一个参数 -/
def churchTrue {α : Type} (t f : α) : α := t

/-- Church 编码的 false：选择第二个参数 -/
def churchFalse {α : Type} (t f : α) : α := f

/-- Church 编码的条件表达式 -/
def churchIf {α : Type} (b : α → α → α) (t f : α) : α := b t f

#eval churchIf churchTrue 1 2   -- 输出 1
#eval churchIf churchFalse 1 2  -- 输出 2

-- 3. Church 编码的自然数
/-- Church 自然数：ℕ ≃ ∀α, (α→α) → α → α -/
def ChurchNat := (α : Type) → (α → α) → α → α

/-- Church 零 -/
def churchZero : ChurchNat := fun α s z => z

/-- Church 后继 -/
def churchSucc (n : ChurchNat) : ChurchNat :=
  fun α s z => s (n α s z)

/-- Church 加法 -/
def churchAdd (m n : ChurchNat) : ChurchNat :=
  fun α s z => m α s (n α s z)

-- 4. 多态列表操作（与 Haskell map 对应）
/-- map : ∀α∀β, (α→β) → List α → List β -/
def polyMap {α β : Type} (f : α → β) : List α → List β
  | []      => []
  | x :: xs => f x :: polyMap f xs

end Polymorphism
```

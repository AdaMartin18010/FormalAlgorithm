# 类型推导算法（Hindley-Milner W 算法）


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

## 1. 类型推导基础

**类型推导（Type Inference）** 是指在不显式标注类型的情况下，自动推导程序表达式最一般类型的过程。

### 1.1 类型约束收集

给定表达式 $e$，类型推导通过语法导向规则生成类型约束集合 $C$。以简单 lambda 演算为例：

$$\frac{\Gamma, x: \tau_1 \vdash e: \tau_2 \mid C}{\Gamma \vdash \lambda x.e: \tau_1 \to \tau_2 \mid C}$$

$$\frac{\Gamma \vdash e_1: \tau_1 \mid C_1 \quad \Gamma \vdash e_2: \tau_2 \mid C_2}{\Gamma \vdash e_1\ e_2: \alpha \mid C_1 \cup C_2 \cup \{\tau_1 = \tau_2 \to \alpha\}} \quad \alpha\ \text{fresh}$$

### 1.2 合一算法（Unification）

**合一（Unification）** 由 Robinson 于 1965 年提出，是类型推导的核心。给定约束集合 $C$，寻找最一般合一子（Most General Unifier, MGU）$\sigma$ 使得对所有 $\tau_1 = \tau_2 \in C$ 有 $\sigma(\tau_1) = \sigma(\tau_2)$。

**Robinson 合一算法伪代码**：

```
function unify(C):
    if C = ∅: return {}
    let {τ₁ = τ₂} ∪ C' = C

    if τ₁ = τ₂: return unify(C')

    if τ₁ is variable α and α ∉ FV(τ₂):
        return {α ↦ τ₂} ∘ unify(C'[α ↦ τ₂])

    if τ₂ is variable α and α ∉ FV(τ₁):
        return {α ↦ τ₁} ∘ unify(C'[α ↦ τ₁])

    if τ₁ = τ₁' → τ₁'' and τ₂ = τ₂' → τ₂'':
        return unify(C' ∪ {τ₁' = τ₂', τ₁'' = τ₂''})

    fail (occurs check)
```

**Occurs Check**：若变量 $\alpha$ 出现在 $\tau$ 中且 $\tau \neq \alpha$，则合一失败，对应类型递归定义（如 $x: x$）。

### 1.3 主类型（Principal Type）

在 Hindley-Milner 系统中，每个良类型的表达式都存在唯一的**最一般类型（Principal Type）** $\tau_p$：若 $e$ 有类型 $\tau$，则存在替换 $\sigma$ 使得 $\tau = \sigma(\tau_p)$。

## 2. Hindley-Milner 系统

Hindley-Milner 类型系统由 Hindley（1969）发现、Milner（1978）独立发现，并由 Damas 和 Milner（1982）形式化。它是 ML 家族语言（OCaml、Haskell、Standard ML）的理论基础。

### 2.1 语法

$$\begin{aligned}
e &::= x \mid \lambda x.e \mid e_1\ e_2 \mid \textbf{let}\ x = e_1\ \textbf{in}\ e_2 \\
\tau &::= \alpha \mid \tau_1 \to \tau_2 \mid \text{Int} \mid \text{Bool} \mid \dots \\
\sigma &::= \forall \alpha_1 \dots \alpha_n.\tau \quad (\text{类型方案})
\end{aligned}$$

### 2.2 算法 W

**算法 W** 是 Hindley-Milner 类型推导的经典实现，采用合一驱动的递归下降方式。

```
function W(Γ, e):
    case e of
        x:
            if x:σ ∈ Γ where σ = ∀α₁...αₙ.τ:
                return ({} , τ[βᵢ/αᵢ])    // βᵢ fresh
            else: fail

        λx.e':
            let β = fresh()
            let (S₁, τ₁) = W(Γ ∪ {x:β}, e')
            return (S₁, S₁(β) → τ₁)

        e₁ e₂:
            let (S₁, τ₁) = W(Γ, e₁)
            let (S₂, τ₂) = W(S₁(Γ), e₂)
            let β = fresh()
            let S₃ = unify(S₂(τ₁) = τ₂ → β)
            return (S₃ ∘ S₂ ∘ S₁, S₃(β))

        let x = e₁ in e₂:
            let (S₁, τ₁) = W(Γ, e₁)
            let Γ' = S₁(Γ)
            let σ = Gen(Γ', τ₁)      // let-多态
            let (S₂, τ₂) = W(Γ' ∪ {x:σ}, e₂)
            return (S₂ ∘ S₁, τ₂)
```

**泛化（Generalization）**：

$$\text{Gen}(\Gamma, \tau) = \forall \alpha_1 \dots \alpha_n.\tau \quad \text{where}\ \{\alpha_1, \dots, \alpha_n\} = \text{FV}(\tau) \setminus \text{FV}(\Gamma)$$

### 2.3 let-多态（let-polymorphism）

`let` 绑定是 HM 系统获得多态性的关键。与 lambda 抽象不同，`let` 允许对右侧表达式的主类型进行全称量化：

$$\frac{\Gamma \vdash e_1: \tau_1 \quad \sigma = \text{Gen}(\Gamma, \tau_1) \quad \Gamma, x:\sigma \vdash e_2: \tau_2}{\Gamma \vdash \textbf{let}\ x = e_1\ \textbf{in}\ e_2: \tau_2}$$

这使得 `let id = λx.x in (id 1, id true)` 是良类型的，而 `(λid.(id 1, id true)) (λx.x)` 不是。

### 2.4 复杂度分析

- **PTIME 上界**：在无 let 的情况下，HM 类型推导可在多项式时间内完成。
- **指数时间下界**：Kfoury、Tiuryn 和 Urzyczyn（1990）证明了 let-嵌套深度可导致最坏情况下指数时间复杂度。具体地，存在表达式族使得推导时间关于 let 深度为 $O(2^n)$。

## 3. 扩展

### 3.1 类型类（Type Classes）

Haskell 引入的类型类扩展了 HM 系统：

$$\text{class}\ Eq\ a\ \text{where}\ eq :: a \to a \to Bool$$

类型类约束可视为**约束类型的合一问题**。字典传递（dictionary passing）将类型类消解为显式参数传递。

### 3.2 子类型推导（简要）

子类型系统引入约束 $\tau_1 <: \tau_2$。合一被替换为**约束求解**（如 Pottier 的图解方法）。HM 系统与顶层子类型结合形成 HM(X) 框架。

### 3.3 依赖类型推断（简要）

依赖类型（如 $\Pi x:A.B$）允许类型依赖于项。完全依赖类型推断通常是不可判定的。实践中采用**局部类型推断（Local Type Inference）**或** bidirectional type checking** 策略，要求部分显式标注以引导推导。

## 4. 小型类型推导器实现框架（Rust）

```rust
# [derive(Clone, Debug, PartialEq)]
enum Type {
    Var(String),
    Con(String),
    Fun(Box<Type>, Box<Type>),
}

# [derive(Clone, Debug, PartialEq)]
enum Expr {
    Var(String),
    Lam(String, Box<Expr>),
    App(Box<Expr>, Box<Expr>),
    Let(String, Box<Expr>, Box<Expr>),
}

type Subst = HashMap<String, Type>;

fn apply_subst(t: &Type, s: &Subst) -> Type {
    match t {
        Type::Var(a) => s.get(a).cloned().unwrap_or(t.clone()),
        Type::Fun(t1, t2) => Box::new(Type::Fun(
            Box::new(apply_subst(t1, s)),
            Box::new(apply_subst(t2, s)),
        )),
        _ => t.clone(),
    }
}

fn unify(t1: &Type, t2: &Type) -> Result<Subst, String> {
    match (t1, t2) {
        (Type::Var(a), t) | (t, Type::Var(a)) if !occurs(a, t) => {
            let mut s = HashMap::new();
            s.insert(a.clone(), t.clone());
            Ok(s)
        }
        (Type::Con(c1), Type::Con(c2)) if c1 == c2 => Ok(HashMap::new()),
        (Type::Fun(a1, r1), Type::Fun(a2, r2)) => {
            let s1 = unify(a1, a2)?;
            let s2 = unify(&apply_subst(r1, &s1), &apply_subst(r2, &s1))?;
            Ok(compose(s2, s1))
        }
        _ => Err("Unification failed".to_string()),
    }
}

fn compose(s2: Subst, s1: Subst) -> Subst { /* ... */ }
```

## 5. 参考文献

1. **Robinson, J. A. (1965)**. *A Machine-Oriented Logic Based on the Resolution Principle*. Journal of the ACM, 12(1):23-41. 提出了合一算法。

2. **Hindley, R. (1969)**. *The Principal Type-Scheme of an Object in Combinatory Logic*. Transactions of the American Mathematical Society, 146:29-60. 首次证明了主类型定理。

3. **Milner, R. (1978)**. *A Theory of Type Polymorphism in Programming*. Journal of Computer and System Sciences, 17(3):348-375. 独立发现了 HM 系统并应用于 ML 语言。

4. **Damas, L. & Milner, R. (1982)**. *Principal Type-Schemes for Functional Programs*. POPL 1982. 给出了算法 W 的完整形式化。

5. **Wand, M. (1987)**. *A Simple Algorithm and Proof for Type Inference*. Fundamenta Informaticae, 10:115-122. 提供了算法 W 的更简洁证明。

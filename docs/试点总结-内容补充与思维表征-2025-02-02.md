## 三、可复用模式与标准对应

### 3.1 可直接复用的 Markdown 结构

其他理论/应用文档在「基本概念」或等价章节后插入一节「内容补充与思维表征」时，建议按下列顺序组织（每项至少 1 处）：

```
解释与直观 → 概念属性表 → 概念关系表 → 概念依赖图 → 论证与证明衔接
→ 思维导图 → 多维矩阵 → 决策树 → 公理定理证明树 → 应用决策建模树
```

## 四、单篇验收对照（计划方案 §七）

试点文档 [04-算法优化理论.md](09-算法理论/04-高级算法理论/04-算法优化理论.md) 自检结果：

| 类型 | 最低要求 | 试点文档 §2.3 对应 |
|------|----------|---------------------|
| 概念定义 | 至少 1 个核心概念具备「形式化定义 + 自然语言 + 解释」 | ✅ §2.1 定义 1.1.1 + §2.3 解释与直观 |
| 属性 | 至少 1 张「概念属性表」或「不变量/不变式」表 | ✅ 概念属性表（$A,O,S,F,P,T(n),S(n)$） |
| 关系 | 至少 1 张概念关系表或 1 个概念依赖图（Mermaid） | ✅ 概念关系表 + 概念依赖图 |
| 论证/形式证明 | 至少 1 个定理具备「论证 + 形式证明」或「证明 + 证明结构图」 | ✅ 定理 1.1.1 论证与证明衔接 + 公理定理证明树 |
| 思维导图 | 至少 1 个本章/本节概念思维导图（Mermaid） | ✅ 思维导图：本章概念结构 |
| 多维矩阵 | 至少 1 张多维概念对比表（多概念 × 多维度） | ✅ 优化策略概念对比表 |
| 决策树 | 至少 1 个决策树图（算法/策略选择或应用选型） | ✅ 决策树：优化策略选择 |
| 公理定理证明树 | 理论文档至少 1 个公理—定理—推理依赖图（Mermaid） | ✅ 公理定理推理证明决策树 |
| 应用决策建模树 | 应用/实现相关至少 1 个从场景到选型的决策树（Mermaid） | ✅ 应用决策建模树 |

**结论**：试点文档满足计划方案「七、验收标准」全部 9 项最低要求；未删除任何现有代码或段落。

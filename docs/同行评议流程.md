# 同行评议流程

> **版本**: 1.0
> **创建日期**: 2025-04-08
> **状态**: ✅ 已生效
> **适用范围**: 所有算法规范与模型设计文档的质量评审

---

## 📋 流程说明

同行评议是确保文档质量的核心机制。本流程定义了标准化的评议过程，确保每篇文档都经过专业、公正的质量评估。

**Peer review is the core mechanism for ensuring document quality. This process defines standardized review procedures.**

---

## 1. 评议准备

### 1.1 评议者选择标准

#### 资格要求

| 要求类别 | 具体要求 | 说明 |
|----------|----------|------|
| 专业背景 | 计算机科学或相关领域 | 确保技术理解能力 |
| 学术水平 | 硕士及以上，或同等经验 | 确保评估能力 |
| 领域专长 | 与待评议文档主题相关 | 至少1个相关领域 |
| 项目参与 | 已阅读项目规范文档 | 了解项目标准 |

#### 领域匹配矩阵

| 文档模块 | 所需专长 | 推荐评议者背景 |
|----------|----------|----------------|
| 基础理论 | 算法理论、数学 | 理论计算机科学 |
| 递归理论 | 递归、归纳证明 | 形式化方法 |
| 形式化证明 | 逻辑、定理证明 | 形式化验证 |
| 算法复杂度 | 复杂度分析 | 算法理论 |
| 类型理论 | 类型系统、λ演算 | 编程语言理论 |
| 逻辑系统 | 数理逻辑 | 逻辑学 |

│ • 轻微潜在冲突 → 声明后继续，加强监督                      │

### 2.2 详细步骤

#### 步骤1: 文档提交

**提交要求**:
- [ ] 文档完成度 ≥ 80%
- [ ] 作者自检清单已完成
- [ ] 提供文档摘要（200字以内）

**提交模板**:

```markdown
## 文档提交表

**文档标题**: 
**文档路径**: 
**作者**: 
**提交日期**: 

### 文档信息

- **类型**: ☐ 新文档 ☐ 重大更新 ☐ 小修订
- **领域**: [算法/数据结构/理论等]
- **预计读者**: [目标读者群体]

### 自检确认

- [ ] 已对照质量检查清单自检
- [ ] 已检查所有引用格式
- [ ] 已验证交叉引用有效
- [ ] 已检查数学符号规范

### 变更摘要

如果是更新，请说明:
- 主要变更内容:
- 变更原因:
- 影响范围:

### 推荐评议者

- 首选: [领域] - [推荐人/理由]
- 备选: [领域] - [推荐人/理由]
```

#### 步骤2: 管理员初审

**初审检查项**:

| 检查项 | 标准 | 操作 |
|--------|------|------|
| 文档完整性 | 包含必需章节 | 不通过则退回 |
| 格式规范 | 符合项目模板 | 不通过则要求修改 |
| 基础质量 | 无明显错误 | 不通过则退回 |
| 领域匹配 | 可分配到对应领域 | 标记领域标签 |

**初审时间**: ≤ 2个工作日

**初审结果**:
- ✅ **通过**: 进入评议者分配
- ⚠️ **需修改**: 退回作者，说明问题
- ❌ **拒绝**: 文档不符合基本要求

#### 步骤3: 评议者分配

**分配原则**:

```
┌─────────────────────────────────────────────────────────────┐
│ 分配策略                                                    │
├─────────────────────────────────────────────────────────────┤
│ 1. 每篇文档分配 2-3 名评议者                                │
│ 2. 至少 1 名该领域专家                                      │
│ 3. 至少 1 名非该领域但相关背景者（可读性评估）              │
│ 4. 避免利益冲突                                             │
│ 5. 考虑评议者当前工作量                                     │
└─────────────────────────────────────────────────────────────┘
```

#### 步骤4: 评议评审

**评审周期**: 14天（可申请延长）

**评审内容**:

### 2.3 评议内容框架

#### 2.3.1 技术准确性评估

| 评估项 | 权重 | 评分标准 |
|--------|------|----------|
| 定义准确性 | 30% | 定义是否准确、完整 |
| 定理正确性 | 30% | 定理陈述是否正确 |
| 证明有效性 | 25% | 证明是否逻辑严密 |
| 算法正确性 | 15% | 算法描述是否正确 |

**评分等级**:
- 5分: 完全正确，无需修改
- 4分: 基本正确，轻微问题
- 3分: 有小错误，需要修改
- 2分: 有重要错误，必须修改
- 1分: 有严重错误，需重新审查

#### 2.3.2 形式化严谨性评估

| 评估项 | 权重 | 评分标准 |
|--------|------|----------|
| 符号规范性 | 30% | 符合ISO 80000-2标准 |
| 形式化程度 | 30% | 定义、定理的形式化程度 |
| 逻辑严密性 | 25% | 推理过程是否严密 |
| 引用完整性 | 15% | 引用是否完整、规范 |

#### 2.3.3 表达清晰度评估

| 评估项 | 权重 | 评分标准 |
|--------|------|----------|
| 结构清晰 | 25% | 文档结构是否合理 |
| 语言流畅 | 25% | 表达是否清晰流畅 |
| 示例充分 | 25% | 示例是否充分、易懂 |
| 术语一致 | 25% | 术语使用是否一致 |

#### 2.3.4 引用完整性评估

| 评估项 | 权重 | 评分标准 |
|--------|------|----------|
| 引用完整性 | 40% | 所有必要引用是否完整 |
| 格式规范性 | 30% | 引用格式是否符合ACM标准 |
| 时效性 | 30% | 是否包含最新研究成果 |

## 6. 评议质量保障

### 6.1 评议者培训

**培训内容**:
1. 项目规范概述
2. 评议流程详解
3. 评议报告撰写
4. 案例研讨

**培训认证**:
- 完成培训并通过测试
- 获得"认证评议者"资格
- 有效期: 1年

### 6.2 评议质量监督

**监督机制**:
- 定期抽查评议报告质量
- 评议者互评
- 作者反馈收集

**评议者评估**:

| 评估维度 | 说明 |
|----------|------|
| 及时性 | 按时完成评议的比例 |
|  thoroughness | 评议报告的深度和全面性 |
| 建设性 | 建议的可操作性 |
| 专业性 | 专业判断的准确性 |

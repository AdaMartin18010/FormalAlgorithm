## 二、贡献流程说明

### 2.1 参与路径

```
新贡献者
    │
    ├── 1. 了解项目
    │   ├── 阅读 README.md
    │   ├── 浏览 贡献者指南
    │   └── 查看 项目治理
    │
    ├── 2. 选择任务
    │   ├── 浏览 Issues (good first issue)
    │   ├── 查看 任务看板
    │   └── 发现改进点
    │
    ├── 3. 认领任务
    │   ├── 评论认领
    │   ├── Fork 仓库
    │   └── 创建分支
    │
    ├── 4. 开展工作
    │   ├── 遵循规范
    │   ├── 本地测试
    │   └── 提交更改
    │
    └── 5. 提交PR
        ├── 使用PR模板
        ├── 等待审查
        └── 合并致谢
```

### 2.2 贡献类型矩阵

| 贡献类型 | 适合人群 | 难度 | 时间投入 |

## 四、参与指南

### 4.1 快速开始

#### 第一步：准备工作

```bash
# 1. Fork 仓库（点击GitHub上的Fork按钮）

# 2. 克隆你的Fork
git clone https://github.com/YOUR_USERNAME/FormalAlgorithm.git
cd FormalAlgorithm

# 3. 添加上游仓库
git remote add upstream https://github.com/ORIGINAL/FormalAlgorithm.git
```

## 第二步：创建分支

### 第二步：创建分支

#### 第二步：创建分支

```bash
# 同步上游
git fetch upstream
git checkout main
git merge upstream/main

# 创建功能分支
git checkout -b feature/your-feature-name
```

## 第三步：提交更改

### 第三步：提交更改

#### 第三步：提交更改

```bash
# 编辑文件...

# 提交
git add .
git commit -m "type: description"
git push origin feature/your-feature-name
```

## 第四步：创建PR

### 第四步：创建PR

#### 第四步：创建PR

1. 访问你的Fork页面
2. 点击 "Compare & pull request"
3. 填写PR模板
4. 等待审查

## 4.2 提交规范

### 4.2 提交规范

#### Commit 格式

```
类型: 简短描述

详细说明（可选）

Fixes #123
```

#### 类型标签

| 标签 | 用途 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug修复 |
| `docs` | 文档更新 |
| `style` | 格式调整 |
| `refactor` | 重构 |
| `test` | 测试 |
| `chore` | 构建/工具 |

## 五、质量保证机制

### 5.1 自动化检查

```yaml
检查项目:
  - 格式检查:
      - Markdown格式
      - 代码格式(rustfmt)
      - 文件名规范

  - 链接检查:
      - 内部链接有效性
      - 外部链接可访问性
      - 锚点正确性

  - 内容检查:
      - 拼写检查
      - 术语一致性
      - 禁用词汇
```

### 5.2 人工审查标准

#### 内容审查清单

- [ ] 信息准确无误
- [ ] 引用来源可靠
- [ ] 论证逻辑严密
- [ ] 示例恰当有效

#### 格式审查清单

- [ ] 遵循文档模板
- [ ] 数学符号规范（ISO 80000-2）
- [ ] 引用格式正确（ACM）
- [ ] 双语标注完整

#### 技术审查清单

- [ ] 代码可运行
- [ ] 测试覆盖充分
- [ ] 复杂度分析准确
- [ ] 交叉引用正确

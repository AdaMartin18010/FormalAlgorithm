## 一、引用规范 (Citation Standards)

### 1.1 引用原则

1. **可验证性**: 所有理论、定义、定理必须提供可验证的出处
2. **权威性**: 优先引用经典教材、权威论文、国际标准
3. **时效性**: 对于新兴主题，应引用最新3-5年的研究成果
4. **完整性**: 引用信息必须包含：作者、标题、出版信息、年份、页码（如适用）
5. **可访问性**: 优先引用公开可访问的资源（arXiv、DOI、ISBN等）

### 1.2 引用格式

#### 学术论文 (Academic Papers)

```text
[作者]. [标题]. [会议/期刊名称], [卷号(期号)], [页码], [年份]. [DOI/URL]
```

**示例**:

```markdown
[Turing1936] A. M. Turing. "On Computable Numbers, with an Application to the Entscheidungsproblem."
Proceedings of the London Mathematical Society, s2-42(1), pp. 230-265, 1936.
DOI: 10.1112/plms/s2-42.1.230
```

#### 教材/书籍 (Textbooks/Books)

```text
[作者]. [书名]. [版本]. [出版社], [年份]. ISBN: [ISBN号]
```

**示例**:

```markdown
[CLRS2009] T. H. Cormen, C. E. Leiserson, R. L. Rivest, and C. Stein.
Introduction to Algorithms (3rd ed.). MIT Press, 2009. ISBN: 978-0262033848
```

#### 在线资源 (Online Resources)

```text
[作者/机构]. [标题]. [网站名称], [访问日期]. URL: [网址]
```

**示例**:

```markdown
[WikiTuring] Wikipedia contributors. "Turing machine." Wikipedia, The Free Encyclopedia.
Accessed: 2025-01-11. URL: https://en.wikipedia.org/wiki/Turing_machine
```

#### 课程资料 (Course Materials)

```text
[讲师]. [课程名称] ([课程代码]). [大学名称], [学期/年份]. URL: [课程网址]
```

**示例**:

```markdown
[MIT6006] E. Demaine and J. Ku. Introduction to Algorithms (6.006).
MIT OpenCourseWare, Fall 2011. URL: https://ocw.mit.edu/courses/6-006-introduction-to-algorithms-fall-2011/
```

#### 技术标准 (Technical Standards)

```text
[标准编号]. [标准名称]. [发布机构], [年份]. URL: [标准文档链接]
```

**示例**:

```markdown
[ISO9899] ISO/IEC 9899:2018. Programming languages — C.
International Organization for Standardization, 2018.
```

### 1.3 引用位置

1. **定义后**: 在给出形式化定义后，立即标注出处

   ```markdown
   图灵机 $M$ 的形式化定义为七元组 $M = (Q, \Sigma, \Gamma, \delta, q_0, B, F)$ [Turing1936]。
   ```

2. **定理后**: 在陈述定理后，标注出处和证明引用

   ```markdown
   **定理 (Church-Turing论题)**: 任何可计算函数都可由图灵机计算 [Church1936, Turing1936]。

   *证明参见*: [Sipser2012, 第3章] 或 [HMU2006, 定理3.21]
   ```

3. **关键概念**: 在引入重要概念时标注

   ```markdown
   递归可枚举集（Recursively Enumerable Set，简称 r.e. set）是可计算性理论的核心概念 [Rogers1987]。
   ```

### 1.4 文献列表格式

每个文档末尾应包含"参考文献"章节，按类别组织：

```markdown
## 参考文献 / References

### 经典文献 (Classic Papers)

[Turing1936] A. M. Turing. "On Computable Numbers..." (详细信息)

### 教材 (Textbooks)

[CLRS2009] T. H. Cormen et al. Introduction to Algorithms... (详细信息)

### 现代研究 (Recent Research)

[Author2024] ... (详细信息)

### 在线资源 (Online Resources)

[WikiTuring] Wikipedia... (详细信息)

### 课程材料 (Course Materials)

[MIT6006] E. Demaine... (详细信息)

### 技术标准 (Technical Standards)

[ISO9899] ISO/IEC 9899:2018... (详细信息)
```

## 二、引用数据库 (Reference Database)

### 2.1 数据库结构

引用数据库采用YAML格式，包含以下字段：

```yaml
reference_id: 唯一标识符 (如 "Turing1936")
type: 文献类型 (paper/book/online/course/standard)
authors: 作者列表
title: 标题 (中英文)
venue: 出版物/会议/课程
year: 年份
pages: 页码 (可选)
doi: DOI (可选)
url: 网址 (可选)
isbn: ISBN (可选)
accessed_date: 访问日期 (在线资源)
tags: 标签列表 (用于分类)
relevance: 相关主题列表
quality: 质量评级 (classic/standard/good/preliminary)
notes: 备注
```

使用方法：

1. 在文档中引用时，使用 `[reference_id]` 格式
2. 在文档末尾的参考文献章节，从数据库提取完整信息
3. 定期更新数据库，特别是在线资源的访问日期

### 2.4 引用质量评级

- **classic**: 经典文献，奠基性工作（如Turing1936, Church1936）
- **standard**: 标准教材/参考（如CLRS, Pierce TAPL）
- **good**: 高质量现代研究（近3-5年的顶会论文）
- **preliminary**: 初步资源（技术报告、预印本）

## 三、实施指南

### 3.1 为现有文档添加引用（分阶段）

**第一阶段（P1优先级，1个月）**:

- 核心理论文档（3-5个）：完整引用体系
  - `01-基础理论/01-形式化定义.md`
  - `07-计算模型/01-图灵机.md`
  - `02-递归理论/01-递归函数定义.md`
  - `09-算法理论/01-算法基础/01-算法设计理论.md`
  - `05-类型理论/01-简单类型论.md`

**第二阶段（P2优先级，2个月）**:

- 其他基础模块：添加主要引用
  - `03-形式化证明/`
  - `04-算法复杂度/`
  - `06-逻辑系统/`

**第三阶段（P3优先级，3-4个月）**:

- 高级主题和应用领域：选择性添加引用

### 3.2 引用检查清单

为每个文档检查：

- [ ] 是否有"参考文献"章节？
- [ ] 关键定义是否标注出处？
- [ ] 重要定理是否标注出处和证明引用？
- [ ] 引用格式是否统一？
- [ ] 引用ID是否在references_database.yaml中？
- [ ] 在线资源是否标注访问日期？
- [ ] 是否引用了最新研究（对于新兴主题）？

### 3.3 工具支持（未来开发）

建议开发以下工具：

1. **引用检查器**：检查文档中未标注出处的定义/定理
2. **引用格式化工具**：从YAML数据库自动生成参考文献列表
3. **引用覆盖率报告**：统计各文档的引用情况
4. **失效链接检查器**：定期检查在线资源的可访问性

## 四、引用修复工作流 (Citation Repair Workflow)

> **适用场景**：项目已有 400+ 文档含"待补充"引用占位符，需要系统化修复。

### 4.1 发现待补充引用

**步骤 1：运行自动化扫描脚本**

```bash
python scripts/generate_citation_tasks.py
```

该脚本会：

- 扫描 `docs/` 下所有 `.md` 文件
- 统计每处"待补充"占位符
- 按模块优先级（P0 → P1 → P2）生成任务清单
- 输出文件：`docs/项目维护/引用修复任务清单_YYYY-MM-DD.md`

**步骤 2：查看任务清单**

任务清单按优先级分组：

| 优先级 | 覆盖模块 | 处理建议 |
|:------:|----------|----------|
| **P0** | 01-基础理论、02-递归理论、03-形式化证明、13-LeetCode | 优先修复，核心基础模块 |
| **P1** | 04-算法复杂度、05-类型理论、06-逻辑系统、07-计算模型 | 次优先，理论支撑模块 |
| **P2** | 09-算法理论、10-高级主题、12-应用领域、其他 | 最后处理，扩展内容 |

### 4.2 添加引用的标准流程

**步骤 1：查阅原始文献**

- 确认概念/定理的最早出处或最权威教材版本
- 优先使用本文件数据库中已有的 `[reference_id]`
- 若数据库中无对应条目，按第 2.1 节格式新建 YAML 条目

**步骤 2：按 ACM/IEEE 风格编写引用**

正文中的引用格式：

```markdown
图灵机 $M$ 的形式化定义为七元组 $M = (Q, \Sigma, \Gamma, \delta, q_0, B, F)$ [Turing1936]。
```

**步骤 3：将新引用添加到数据库**

- 若引用为 2024–2025 年最新研究 → 添加到 `docs/references_database_2024_2025.yaml`
- 若引用为经典教材/论文 → 添加到本文件 (`docs/引用规范与数据库.md`) 的对应主题节
- 避免重复：添加前先用 `grep` 搜索是否已存在相同的 `reference_id`

**步骤 4：在正文中替换"待补充"**

- 仅当文件中**明确提到**某个经典概念/算法/定理时，才在其后的"待补充"位置替换为对应引用
- 若文件为空白模板（如仅有标题和"（待补充）"），**不要**强行填充正文，仅在"参考文献"章节列出主题相关的标准引用
- 修改前务必备份原文件到 `.backup/` 目录

### 4.3 引用完整性检查

**使用 `tools/citation_checker.py`：**

```bash
python tools/citation_checker.py
```

该工具会：

- 遍历 `docs/` 下所有 Markdown 文件
- 检测**定义后**和**定理后**是否缺少引用标记（`[AuthorYYYY]` 格式）
- 生成详细报告：`docs/质量报告/citation-check-report.md`

**检查清单（人工复核）：**

- [ ] 文档末尾是否有"参考文献"章节？
- [ ] 关键定义后是否标注出处？
- [ ] 重要定理后是否标注出处和证明引用？
- [ ] 引用格式是否统一为 `[AuthorYYYY]`？
- [ ] 引用 ID 是否已在引用数据库中定义？
- [ ] 在线资源是否标注访问日期？
- [ ] 修改前是否已备份到 `.backup/`？

### 4.4 注意事项

1. **宁可少修，不要错修**：对于内容空白或上下文不明确的文件，保留"待补充"占位符，在任务清单中记录，等待后续人工补充。
2. **保持格式一致**：所有引用必须严格遵循第 1.2 节的 ACM/IEEE 格式。
3. **避免重复条目**：添加新引用前，先检索现有数据库（`docs/引用规范与数据库.md` 和 `docs/references_database_2024_2025.yaml`）。
4. **版本控制**：重大引用修复完成后，更新本文件的"最后更新"日期和版本号。

## 五、BibTeX 迁移计划

> **状态**: P1 阶段（双轨并行）
> **启动日期**: 2026-04-29
> **详细方案**: [`docs/项目维护/BibTeX引用迁移方案_2026-04-29.md`](项目维护/BibTeX引用迁移方案_2026-04-29.md)

### 5.1 为什么要迁移？

项目当前使用自定义 YAML 格式管理引用，虽便于内部维护，但无法直接与 Zotero、LaTeX、Pandoc 等学术工具链集成。建立标准 BibTeX 数据库后：

- 可导入 Zotero / JabRef 进行文献管理
- 可通过 Pandoc `citeproc` 自动生成带引用的 PDF / DOCX
- 可通过 LaTeX `bibliographystyle` 切换 APA、IEEE、GB/T 7714 等引用样式

### 5.2 双轨并行策略（P1 阶段）

**绝不破坏现有引用**：项目 400+ 文档中使用的 `[AuthorYear]` 正文引用格式保持不变。

当前采用 **YAML 为主、BibTeX 为辅** 的双轨机制：

| 数据源 | 用途 | 维护方式 |
|--------|------|----------|
| `docs/引用规范与数据库.md` + `docs/references_database_2024_2025.yaml` | **主数据库**，新增引用优先录入此处 | 人工编辑 |
| `docs/references.bib` | **导出产物**，供学术工具链使用 | 脚本生成 + 人工校对核心条目 |

```
新增引用流程：
  1. 编辑 YAML 主数据库
  2. 运行 python scripts/convert_refs_to_bibtex.py
  3. 自动同步生成 docs/references.bib
```

### 5.3 现有 BibTeX 文件

- **文件位置**: `docs/references.bib`
- **当前规模**: 约 80 条（含 30 条核心经典引用 + 50 条 2024–2025 最新研究）
- **核心覆盖**: CLRS、Sipser、Pierce、Barendregt、Knuth、Software Foundations、Lean 4、Turing、Gödel、Cook、Karp 等

### 5.4 分阶段迁移路线图

| 阶段 | 时间 | 目标 |
|------|------|------|
| **P1** | 2026-04 ~ 2026-06 | 双轨并行：YAML 为主，BibTeX 为导出产物；验证工具链兼容性 |
| **P2** | 2026-07 ~ 2026-09 | 扩展 BibTeX 至全部 200+ 条目；提供 CSL 样式库；试点 Pandoc PDF 生成 |
| **P3** | 2026-10 起 | 评估是否全面迁移至 BibTeX；若保留 YAML，则确立单向生成流水线 |

### 5.5 工具链使用示例

**Zotero 导入**：

```text
文件 → 导入 → 选择 docs/references.bib
```

**Pandoc 生成引用 PDF**（需将正文 `[Key]` 预处理为 `[@Key]`）：

```bash
pandoc doc.md \
  --bibliography docs/references.bib \
  --citeproc \
  --csl styles/ieee.csl \
  -o doc.pdf
```

**LaTeX 编译**：

```latex
\bibliographystyle{alpha}
\bibliography{references}
```

### 5.6 注意事项

1. **BibTeX Key 保持不变**：为确保现有 `[AuthorYear]` 正文引用兼容，BibTeX 中的 key 与 YAML 中的 `id` / `reference_id` 完全一致。
2. **不要直接编辑 `references.bib`**：该文件主要由脚本生成。如需补充核心经典引用（YAML 中暂无完整字段的文献），请在 `docs/项目维护/BibTeX引用迁移方案_2026-04-29.md` 中记录，并同步更新 YAML 数据库。
3. **特殊字符处理**：BibTeX 对 `& # %` 等字符敏感，脚本已做基础转义；含复杂 LaTeX 公式的标题建议预先在 YAML 中处理。

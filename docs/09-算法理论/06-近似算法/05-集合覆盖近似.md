# 集合覆盖近似算法

## 问题定义

**输入**：全集 $U$，子集族 $S = \{S_1, S_2, ..., S_m\}$，每个子集有权重 $w_i$
**问题**：选择最小的子集族 $C \subseteq S$，使得 $\bigcup_{S_i \in C} S_i = U$

集合覆盖是 Vertex Cover 的推广，也是经典的 NP 难问题。

## 贪心算法

### 无权版本

每次选择覆盖最多**未覆盖元素**的子集：

```
C = ∅
U_uncovered = U
while U_uncovered ≠ ∅:
    选择 S_i 最大化 |S_i ∩ U_uncovered|
    C = C ∪ {S_i}
    U_uncovered = U_uncovered \ S_i
```

### 近似比分析

**定理**：贪心算法是 $H_n$-近似算法，其中 $H_n = 1 + \frac{1}{2} + ... + \frac{1}{n} \approx \ln n$ 是第 $n$ 个调和数。

**证明概要**：

- 按元素被覆盖的顺序编号 $e_1, e_2, ..., e_n$
- 元素 $e_k$ 被覆盖时，至少还有 $n-k+1$ 个未覆盖元素
- 贪心选择的子集此时覆盖至少 $\frac{n-k+1}{OPT}$ 个新元素
- 每个元素的"代价" $\leq \frac{OPT}{n-k+1}$
- 总代价 $\leq OPT \cdot \sum_{k=1}^n \frac{1}{k} = OPT \cdot H_n$

### 紧界

存在实例使得贪心算法的解恰好为 $H_n \cdot OPT$，因此该分析是紧的。

## 加权版本

每次选择**单位权重覆盖最多元素**的子集：

$$\text{性价比} = \frac{|S_i \cap U_{\text{uncovered}}|}{w_i}$$

同样得到 $H_n$-近似比。

## 与命中集（Hitting Set）的对偶性

集合覆盖的对偶问题是命中集：选择最少的元素，使得每个子集至少包含一个被选元素。

两者是同一问题的不同表述，LP 松弛的对偶关系揭示了它们之间的联系。

## 实际应用

| 场景 | 全集 | 子集 |
|------|------|------|
| 测试用例选择 | 所有代码行 | 每个测试覆盖的行 |
| 广告投放 | 目标人群 | 每个渠道覆盖的人群 |
| 传感器布置 | 监控区域 | 每个传感器的覆盖范围 |
| 病毒检测 | 人群 | 每次检测可覆盖的群体 |

# 归并排序

## 概述

归并排序（Merge Sort）由 John von Neumann 于 1945 年提出，是典型的**分治算法**。其核心思想是将数组递归地分成两半，分别排序后合并两个有序子数组。

## 算法步骤

1. **分解**：将 $A[low..high]$ 平分为 $A[low..mid]$ 和 $A[mid+1..high]$
2. **解决**：递归地对两个子数组进行归并排序
3. **合并**：将两个有序子数组合并为一个有序数组

## 合并过程（Merge）

给定两个已排序数组 $L$ 和 $R$：

- 初始化指针 $i=0, j=0$
- 比较 $L[i]$ 与 $R[j]$，将较小者放入结果数组
- 当某一数组耗尽后，将另一数组剩余元素全部追加

合并两个总长度为 $n$ 的数组需要 $O(n)$ 时间和 $O(n)$ 辅助空间。

## 复杂度分析

### 时间复杂度

递归式：$T(n) = 2T(n/2) + O(n)$

由主定理（Case 2）：$T(n) = O(n \log n)$

| 场景 | 时间复杂度 |
|------|-----------|
| 最好 | $O(n \log n)$ |
| 平均 | $O(n \log n)$ |
| 最坏 | $O(n \log n)$ |

### 空间复杂度

- 辅助数组：$O(n)$
- 递归栈：$O(\log n)$
- **总计**：$O(n)$

## 稳定性

归并排序是**稳定排序**。合并时，当 $L[i] = R[j]$，优先取 $L[i]$，保证相等元素的原始相对顺序不变。

## 自然归并排序（Natural Merge Sort）

利用输入数组中已有的有序段（run）进行合并：

- 扫描数组识别递增/递减段
- 对递减段原地反转
- 逐层合并相邻段

在部分有序数据上，时间复杂度可接近 $O(n)$。

## 与快速排序的对比

| 特性 | 归并排序 | 快速排序 |
|------|---------|---------|
| 最坏时间 | $O(n \log n)$ ✅ | $O(n^2)$ |
| 空间 | $O(n)$ | $O(\log n)$ ✅ |
| 稳定性 | 稳定 ✅ | 不稳定 |
| 缓存效率 | 需额外扫描 | 原地操作 ✅ |
| 实际速度 | 略慢（内存访问） | 通常更快 |

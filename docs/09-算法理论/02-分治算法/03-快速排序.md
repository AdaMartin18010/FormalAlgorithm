# 快速排序

## 概述

快速排序（Quick Sort）由 Tony Hoare 于 1960 年提出，是基于**分治策略**的经典排序算法。其核心思想是选择一个**枢轴（pivot）**，将数组划分为小于枢轴和大于枢轴的两部分，然后递归地对两部分进行排序。

## 算法步骤

1. **分解**：选择枢轴 $p$，将数组 $A[low..high]$ 重排为 $A[low..q-1] \leq A[q] \leq A[q+1..high]$
2. **解决**：递归排序 $A[low..q-1]$ 和 $A[q+1..high]$
3. **合并**：无需额外合并操作（原地排序）

## 分区算法（Lomuto / Hoare）

### Hoare 分区（更优，减少交换次数）

```
pivot = A[low]
i = low - 1
j = high + 1
loop:
    repeat i = i + 1 until A[i] >= pivot
    repeat j = j - 1 until A[j] <= pivot
    if i >= j: return j
    swap A[i], A[j]
```

## 复杂度分析

| 场景 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好 | $O(n \log n)$ | 每次枢轴恰好为中位数 |
| 平均 | $O(n \log n)$ | 随机输入期望 |
| 最坏 | $O(n^2)$ | 已排序且每次选第一个为枢轴 |
| 空间 | $O(\log n)$ | 递归栈深度 |

## 优化策略

1. **三数取中法**：选择首、中、尾三个元素的中位数作为枢轴
2. **随机化**：随机选择枢轴，期望时间 $O(n \log n)$
3. **小数组切换**：子数组长度小于阈值时切换为插入排序
4. **尾递归优化**：先处理较短子数组，减少栈深度

## 稳定性

快速排序是**不稳定**的排序算法。相等元素的相对顺序可能在分区过程中改变。

## 与其他排序的对比

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
|------|---------|---------|------|--------|
| 快速排序 | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 不稳定 |
| 归并排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 稳定 |
| 堆排序 | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | 不稳定 |

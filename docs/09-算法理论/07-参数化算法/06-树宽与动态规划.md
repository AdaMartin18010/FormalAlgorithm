# 树宽与动态规划（Treewidth & DP）

## 概述

树宽（treewidth）是衡量图"树状程度"的拓扑参数。树宽较小的图（如树、串并联图、外平面图）上，许多 NP-hard 问题可通过树分解上的动态规划在 $O(f(k) \cdot n)$ 时间内高效求解，其中 $k$ 为树宽。

## 树宽定义

**树分解**：图 $G=(V,E)$ 的树分解是树 $T$，其中每个节点（称为 bag）包含 $V$ 的子集，满足：

1. **覆盖**：每个顶点至少出现在一个 bag 中
2. **边覆盖**：每条边的两个端点至少共同出现在一个 bag 中
3. **连通性**：包含顶点 $v$ 的所有 bag 形成 $T$ 的连通子树

**树宽**：所有 bag 大小的最大值减 1。

$$tw(G) = \min_{\text{树分解 } T} \max_{\text{bag } B} |B| - 1$$

## 树宽上界

| 图类 | 树宽 |
|------|------|
| 树 | 1 |
| 串并联图 | $\leq 2$ |
| 外平面图 | $O(\sqrt{n})$ |
| 平面图 | $O(\sqrt{n})$ |
| 网格 $k \times k$ | $k$ |
| 完全图 $K_n$ | $n-1$ |

## 树分解上的动态规划

### 标准模式

对每个 bag，维护一个**状态表**，记录该 bag 内顶点在解中的可能配置。

状态数：$O(2^{tw+1})$ 或 $O((tw+1)^{tw+1})$，取决于问题。

### 示例：Vertex Cover on Tree Decomposition

对每个 bag $B$，状态为 $B$ 中每个顶点是否在覆盖中（0/1）。

转移时确保：

- 边约束满足（边的至少一端在覆盖中）
- bag 间一致性（共享顶点的状态一致）

**复杂度**：$O(2^{tw} \cdot n)$

### 示例：Independent Set

状态为 bag 中哪些顶点在独立集中（需保证 bag 内无边）。

**复杂度**：$O(2^{tw} \cdot n)$

## Courcelle 定理

**定理**：任何可用 MSO$_1$（一元二阶逻辑）表达的图问题，在树宽有界的图上可在 $O(f(k, \phi) \cdot n)$ 时间内求解。

这是参数化复杂性理论中最强大的元定理之一。

## 实际应用

| 领域 | 图类 | 问题 |
|------|------|------|
| 电路设计 | 串并联图 | 布局优化 |
| 进化树 | 树 | 祖先推断 |
| 道路网络 | 低树宽子图 | 路由优化 |
| 概率推理 | 贝叶斯网络 | 精确推断 |

## 代码参考

树分解求解器实现复杂，通常使用专用库（如 LibTW、OGDF）。参数化算法框架（如 PACE 竞赛工具）提供参考实现。

# 一致代价搜索与迭代加深搜索

## 一致代价搜索（Uniform Cost Search, UCS）

### 概述

UCS 是 Dijkstra 算法在搜索问题中的等价形式。它每次扩展从起点出发**路径代价最小**的节点，保证在边代价非负时找到最优解。

### 算法描述

使用优先队列（最小堆）存储待扩展节点，按 $g(n)$（从起点到 $n$ 的累计代价）排序。

```
优先队列 Q 按 g(n) 排序
将起点 s 加入 Q，g(s) = 0
while Q 非空:
    n = Q.pop()
    if n 是目标: 返回路径
    for n 的每个后继 m:
        if m 未访问或找到更短路径:
            g(m) = g(n) + cost(n, m)
            Q.push(m)
```

### 性质

| 性质 | 结论 |
|------|------|
| 完备性 | ✅ 若存在解，必能找到 |
| 最优性 | ✅ 边代价非负时保证最优 |
| 时间复杂度 | $O(b^{1+\lfloor C^*/\epsilon \rfloor})$ |
| 空间复杂度 | $O(b^{1+\lfloor C^*/\epsilon \rfloor})$ |

其中 $C^*$ 为最优解代价，$\epsilon$ 为最小边代价。

### 与 BFS 的关系

- 若所有边代价为 1，UCS 等价于 BFS
- UCS 可处理变代价图（如地形导航中不同地形的移动代价）

## 迭代加深搜索（Iterative Deepening Search, IDS）

### 动机

BFS 空间开销大，DFS 不完备且不最优。IDS 结合两者优势：

- **DFS 的空间效率**：$O(bd)$
- **BFS 的最优性**：找到最短路径

### 算法描述

```
for depth_limit = 0, 1, 2, ...:
    执行深度受限的 DFS，最大深度为 depth_limit
    if 找到解: 返回
```

### 时间复杂度分析

似乎重复做了很多工作，但 overhead 很小：

$$
\begin{aligned}
T(n) &= (d) b^0 + (d-1) b^1 + (d-2) b^2 + \cdots + (1) b^{d-1} \\
&\leq b^d \left( \frac{d}{b^{d-1}} + \frac{d-1}{b^{d-2}} + \cdots \right) \\
&= O(b^d)
\end{aligned}
$$

与 BFS 同阶，但空间仅需 $O(bd)$。

### 迭代加深 A*（IDA*）

将 IDS 与 A* 结合，用 $f(n) = g(n) + h(n)$ 作为深度限制：

- 每次迭代将阈值设为超过当前阈值的最小 $f$ 值
- 空间 $O(d)$，适合内存受限的大规模搜索（如 15 拼图、Rubik's Cube）

## 应用场景对比

| 算法 | 适用场景 |
|------|---------|
| BFS | 无权图、小规模状态空间 |
| UCS | 变代价图、需要最优解 |
| DFS | 空间受限、解深度已知 |
| IDS | 深度未知、需要最优解且空间受限 |
| A* | 有有效启发函数的场景 |
| IDA* | 超大规模状态空间（如棋类游戏）|

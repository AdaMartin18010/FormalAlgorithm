# 双向搜索（Bidirectional Search）

## 概述

双向搜索同时从**起点**和**目标**出发进行搜索，当两个搜索前沿相遇时找到路径。它将搜索空间从 $O(b^d)$ 降至 $O(b^{d/2})$，是求解最短路径和 puzzles 的有效技术。

## 算法描述

```
frontier_s = {start}, frontier_g = {goal}
visited_s = {start}, visited_g = {goal}
while 两个前沿不相交:
    扩展 frontier_s（BFS 或 A*）
    if 新节点在 visited_g 中: 找到路径
    扩展 frontier_g（反向）
    if 新节点在 visited_s 中: 找到路径
```

## 复杂度分析

| 指标 | 单向搜索 | 双向搜索 |
|------|---------|---------|
| 时间 | $O(b^d)$ | $O(b^{d/2})$ |
| 空间 | $O(b^d)$ | $O(b^{d/2})$ |

**假设**：两个方向的 branching factor 相同。

## 关键问题

### 1. 相遇检测

需要高效判断节点是否被另一方向访问过：

- 哈希表：$O(1)$ 查询
- 交替扩展：每次只扩展一层，减少重复检查

### 2. 前端选择

通常扩展**节点数较少**的前沿，保持平衡。

### 3. 最优性保证

- **BFS + BFS**：找到最短路径
- **A* + A***（反向启发式 $h'$）：需满足 $h + h' \geq$ 实际距离，才能保证最优

## 应用场景

### 15/24 拼图

- 单向 BFS：搜索深度 20，节点数 $10^{13}$
- 双向 BFS：每侧深度 10，节点数 $2 \times 10^6$
- 配合不可解性剪枝，可在秒级求解

### 最短路径

- 地图导航：同时从起点和终点运行 Dijkstra / A*
- 数据库连接：在社交网络中找最短人脉链

### 词梯问题（Word Ladder）

从 "hit" 到 "cog"，每次改一个字母：

- hit → hot → dot → dog → cog
- 双向 BFS 在字典中搜索，快速相遇

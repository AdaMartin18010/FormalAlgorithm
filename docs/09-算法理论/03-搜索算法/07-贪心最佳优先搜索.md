# 贪心最佳优先搜索（Greedy Best-First Search）

## 概述

贪心最佳优先搜索是一种有信息搜索算法，它每次扩展**启发函数 $h(n)$ 估计值最小**的节点。与 A* 不同，它完全忽略从起点到当前节点的实际代价 $g(n)$，只关注"看起来离目标最近"的节点。

## 算法描述

使用优先队列，按 $h(n)$ 排序：

```
优先队列 Q 按 h(n) 排序
将起点 s 加入 Q
while Q 非空:
    n = Q.pop()
    if n 是目标: 返回路径
    for n 的每个后继 m:
        if m 未访问:
            Q.push(m)
```

## 性质

| 性质 | 结论 |
|------|------|
| 完备性 | ❌ 可能陷入死循环或局部最优 |
| 最优性 | ❌ 不保证找到最短路径 |
| 时间复杂度 | $O(b^m)$（最坏）|
| 空间复杂度 | $O(b^m)$ |

## 与 A* 的对比

| 特性 | 贪心最佳优先 | A* |
|------|-------------|-----|
| 评价函数 | $f(n) = h(n)$ | $f(n) = g(n) + h(n)$ |
| 完备性 | ❌ | ✅（$h$ 可采纳时）|
| 最优性 | ❌ | ✅（$h$ 可采纳时）|
| 速度 | 通常最快 | 较快 |
| 内存 | 可能浪费在错误路径 | 更均衡 |

## 典型失败案例

```
S --1--> A --100--> G
|                     ^
+--------1------------+
         B
```

贪心算法从 $S$ 出发：

- $h(A) = 1$（估计离目标很近）
- $h(B) = 1$（同样）
- 若优先选 $A$，则实际代价 $1 + 100 = 101$
- 最优路径 $S \to B \to G$ 代价仅 $2$

## 适用场景

- **解的质量要求不高**：只需任意可行解
- **启发函数极其精确**：$h(n) \approx h^*(n)$ 时表现接近 A*
- **内存极度受限**：比 A* 少存储 $g(n)$

## 改进：A* 的权重版本

$$f(n) = g(n) + w \cdot h(n)$$

- $w = 0$：退化为 Dijkstra / UCS
- $w = 1$：标准 A*
- $w \to \infty$：退化为贪心最佳优先

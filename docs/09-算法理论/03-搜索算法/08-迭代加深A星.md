# 迭代加深 A*（IDA*）

## 概述

迭代加深 A*（Iterative Deepening A*, IDA*）由 Korf 于 1985 年提出，它结合了迭代加深搜索（IDS）的空间效率和 A* 的启发式剪枝能力。IDA* 是求解大规模组合优化问题（如 15 拼图、24 拼图、Rubik's Cube）的最有效算法之一。

## 核心思想

用 A* 的评价函数 $f(n) = g(n) + h(n)$ 作为深度限制，而非简单的深度层数：

```
threshold = h(start)
loop:
    result = search(start, 0, threshold)
    if result is solution: return result
    if result == INF: return failure
    threshold = result  // 下一个最小超过阈值的 f 值
```

## 递归搜索

```
search(node, g, threshold):
    f = g + h(node)
    if f > threshold: return f
    if node is goal: return solution
    min_next = INF
    for successor of node:
        t = search(successor, g + cost, threshold)
        if t is solution: return t
        if t < min_next: min_next = t
    return min_next
```

## 空间复杂度

IDA* 不需要存储 Open/Closed 集合，仅需存储当前路径：

$$\text{空间} = O(d)$$

其中 $d$ 为解深度。对 24 拼图（解深度约 100），仅需几百字节栈空间。

## 与 A* 的对比

| 特性 | A* | IDA* |
|------|-----|------|
| 空间 | $O(b^d)$ | $O(d)$ ✅ |
| 时间 | $O(b^d)$ | $O(b^d)$ |
| 重复状态 | 自动处理 | 需额外机制（如转置表）|
| 最优性 | ✅ | ✅ |

## 剪枝优化

### 启发式函数

- **15 拼图**：曼哈顿距离（Manhattan Distance）
- **24 拼图**：线性冲突（Linear Conflict）+ 曼哈顿距离
- **Rubik's Cube**：模式数据库（Pattern Database）

### 转置表（Transposition Table）

缓存已访问状态的最小 $g$ 值，避免重复搜索同一状态的不同路径。由于内存限制，通常使用哈希表或布谷鸟哈希。

### moves 排序

优先尝试 $h$ 值最小的移动，更快找到解，减少阈值迭代次数。

## 历史成就

- **God's Number**：IDA* + 模式数据库证明了 Rubik's Cube 的最坏情况只需 20 步
- **24 拼图**：IDA* 可在数秒内求解随机实例

# 多线程算法（Multithreaded Algorithms）


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

> 对标：CLRS 第4版 Chapter 27

## 一、理论基础

多线程算法研究如何在**共享内存**的多核/多处理器环境中，通过**并行执行**来加速算法。与分布式算法不同，多线程算法假设：

- 所有处理器共享同一地址空间；
- 通信开销极小（通过共享变量）；
- 主要挑战在于**并行度（parallelism）**、**调度（scheduling）**和**竞争条件（race condition）**的避免。

### 核心模型：动态多线程（Dynamic Multithreading）

CLRS 采用动态多线程模型，使用三个原语：

- **parallel**：在调用前加关键字，表示子调用可与父调用并行执行。
- **spawn**：创建一个新的并行线程（逻辑上的）。
- **sync**：等待当前作用域内所有 `spawn` 出的线程完成。

### 性能度量

1. **工作量（Work, $T_1$）**：单处理器上执行的总时间（所有操作串行相加）。
2. **跨度（Span, $T_\infty$）**：无限处理器上由最长依赖链决定的时间（关键路径长度）。
3. **并行度（Parallelism）**：$T_1 / T_\infty$，表示算法可利用的最大处理器数量上限。

### 贪心调度定理

对于 $P$ 个处理器的理想调度器，运行时间 $T_P$ 满足：
$$
T_P \leq \frac{T_1}{P} + T_\infty
$$
这意味着：

- 若 $P \ll T_1 / T_\infty$，加速比接近线性（$\approx P$）；
- 若 $P \gg T_1 / T_\infty$，则额外处理器几乎无益。

## 二、算法设计

### 1. 并行矩阵乘法

将 $n \times n$ 矩阵分成 4 个 $\frac{n}{2} \times \frac{n}{2}$ 子块，递归并行计算 8 个子矩阵乘积：

```
C11 = parallel M11 + M12
C12 = parallel M21 + M22
...
```

- 工作量：$T_1(n) = \Theta(n^3)$（与标准算法相同）
- 跨度：$T_\infty(n) = \Theta(\log^2 n)$（递归深度 $\log n$，每层加法跨度 $\log n$）
- 并行度：$\Theta(n^3 / \log^2 n)$

### 2. 并行归并排序

递归地对左右两半 `spawn` 并行排序，然后 `sync`，最后执行并行归并：

```
spawn merge_sort(A, p, q)
spawn merge_sort(A, q+1, r)
sync
parallel_merge(A, p, q, r)
```

- 工作量：$T_1(n) = \Theta(n \log n)$
- 跨度：$T_\infty(n) = \Theta(\log^3 n)$（若使用朴素并行归并）
- 使用更精巧的并行归并可将跨度降至 $\Theta(\log^2 n)$。

## 三、复杂度分析

| 算法 | 工作量 $T_1$ | 跨度 $T_\infty$ | 并行度 |
|------|--------------|----------------|--------|
| 并行矩阵乘法 | $\Theta(n^3)$ | $\Theta(\log^2 n)$ | $\Theta(n^3 / \log^2 n)$ |
| 并行归并排序 | $\Theta(n \log n)$ | $\Theta(\log^3 n)$ | $\Theta(n \log n / \log^3 n)$ |
| 并行前缀和 | $\Theta(n)$ | $\Theta(\log n)$ | $\Theta(n / \log n)$ |

## 四、形式化验证

### 关键引理：工作量定律与跨度定律

**工作量定律**：$T_P \geq T_1 / P$
（$P$ 个处理器最多将工作量均分）

**跨度定律**：$T_P \geq T_\infty$
（即使无限处理器，关键路径也无法缩短）

**贪心界**：$T_P \leq T_1 / P + T_\infty$

### 竞争条件（Race Condition）

若两个并行线程同时访问同一内存位置，且至少有一个是写操作，则发生**determinacy race**，可能导致非确定性结果。形式化验证中必须证明：

- 所有并行写操作作用于**不相交的内存区域**；
- 或所有共享写操作通过**原子操作/锁**进行互斥保护。

### 并行循环不变式

对于 `parallel for` 结构，循环不变式需在每次迭代中独立成立：

- 迭代 $i$ 的正确性不依赖于同时执行的迭代 $j$ 的中间状态；
- 最终结果由归约操作（如 `+`, `min`）的交换律/结合律保证。

## 五、应用场景

1. **科学计算与深度学习**：矩阵乘法、卷积、反向传播中的大规模张量运算。
2. **图处理**：并行 BFS、PageRank、连通分量计算（如 Ligra 框架）。
3. **排序与搜索**：大规模数据排序、并行数据库查询。
4. **编译器优化**：自动向量化、OpenMP / Cilk / TBB 并行任务调度。

### 现代编程框架

- **OpenMP**：基于编译器指令的共享内存并行（`#pragma omp parallel`）。
- **Cilk / Cilk Plus**：spawn/sync 模型的工业级实现（Intel）。
- **TBB（Intel Threading Building Blocks）**：C++ 任务并行库。
- **Rayon（Rust）**：基于工作窃取（work-stealing）的数据并行库。

---

## 六、扩展变体

- **异步并行（Fork-Join）**：与动态多线程等价，常用于 Java `ForkJoinPool`。
- **GPU 并行（SIMT）**：NVIDIA CUDA / OpenCL，采用单指令多线程模型，适合数据并行。
- **任务图调度（Task Graph）**：将算法表示为 DAG，使用拓扑排序或关键路径调度。
- **工作窃取调度器（Work-Stealing Scheduler）**：如 Cilk 和 TBB 的底层调度器，已被证明在实践中最优。

## 一、规范定义深化

### 1.1 分治范式形式化规约

**定义 1.1 (分治算法)** 分治（Divide and Conquer）是一种算法设计范式，其形式化描述为三元组 $(D, C, M)$：

1. **分解 (Divide)**: $D(I) = \{I_1, I_2, \ldots, I_k\}$，将输入实例 $I$ 分解为 $k \geq 2$ 个不相交子实例
2. **解决 (Conquer)**: $C(\{I_1, \ldots, I_k\}) = \{S_1, \ldots, S_k\}$，递归求解各子实例
3. **合并 (Merge)**: $M(\{S_1, \ldots, S_k\}) = S$，将子解合并为原实例的解

**正确性条件**: $M(C(D(I))) = Solve(I)$，其中 $Solve(I)$ 为实例 $I$ 的真实解。

### 1.2 适用性判定

分治算法在以下条件下有效：

| 条件 | 说明 |
|------|------|
| **子问题独立性** | 子实例间无重叠或依赖（与动态规划的关键区别） |
| **可分解性** | 存在高效算法将原问题分解为子问题 |
| **可合并性** | 存在高效算法将子解合并为完整解 |
| **基例可解** | 小规模实例可直接求解 |

## 二、模型设计深化

### 2.1 递归树模型

分治算法的执行过程对应一棵递归树，其中：

- **根节点**: 原始问题实例，规模为 $n$
- **内部节点**: 规模为 $s$ 的子问题，有 $k$ 个子节点，每个子问题规模为 $s/b$
- **叶节点**: 基例，规模为 $O(1)$

**递归树高度**: $h = \log_b n$

**每层工作量**: 第 $i$ 层有 $k^i$ 个节点，每个节点工作量 $f(n/b^i)$，总工作量 $k^i \cdot f(n/b^i)$

**总工作量**: $T(n) = \sum_{i=0}^{\log_b n} k^i \cdot f(n/b^i)$

### 2.2 主定理 (Master Theorem)

**定理 2.1 (主定理)** 设 $a \geq 1, b > 1$ 为常数，$f(n)$ 为渐近正函数。对于递推式：
$$T(n) = aT(n/b) + f(n)$$

有三种情况：

1. **情况1**: 若 $f(n) = O(n^{\log_b a - \epsilon})$（对某个 $\epsilon > 0$），则 $T(n) = \Theta(n^{\log_b a})$
2. **情况2**: 若 $f(n) = \Theta(n^{\log_b a})$，则 $T(n) = \Theta(n^{\log_b a} \log n)$
3. **情况3**: 若 $f(n) = \Omega(n^{\log_b a + \epsilon})$（对某个 $\epsilon > 0$），且 $af(n/b) \leq cf(n)$（正则条件，$c < 1$），则 $T(n) = \Theta(f(n))$

## 三、数学符号与推导

### 3.1 主定理的严格证明（情况2）

设 $n = b^m$，则：

$$\begin{aligned}
T(n) &= aT(n/b) + cn^{\log_b a} \\
     &= a[aT(n/b^2) + c(n/b)^{\log_b a}] + cn^{\log_b a} \\
     &= a^2 T(n/b^2) + ca(n/b)^{\log_b a} + cn^{\log_b a} \\
     &= a^2 T(n/b^2) + cn^{\log_b a}(a/b^{\log_b a}) + cn^{\log_b a} \\
     &= a^2 T(n/b^2) + 2cn^{\log_b a}
\end{aligned}$$

注意到 $a/b^{\log_b a} = a/a = 1$。继续展开：

$$T(n) = a^m T(1) + m \cdot cn^{\log_b a} = \Theta(n^{\log_b a}) + \Theta(n^{\log_b a} \log n) = \Theta(n^{\log_b a} \log n)$$

### 3.2 矩阵乘法的 Strassen 算法分析

标准矩阵乘法：$T(n) = 8T(n/2) + O(n^2)$，由主定理 $T(n) = \Theta(n^3)$。

Strassen 算法通过代数恒等式将 8 次乘法减少为 7 次：
$$T(n) = 7T(n/2) + O(n^2)$$

这里 $a = 7, b = 2, \log_2 7 \approx 2.81$。$f(n) = O(n^2) = O(n^{\log_2 7 - \epsilon})$（取 $\epsilon \approx 0.81$），适用情况1：
$$T(n) = \Theta(n^{\log_2 7}) \approx \Theta(n^{2.81})$$

## 四、示例性代码

### 4.1 工程级 Rust 实现

```rust
//! 分治算法库 — 工程级实现
//!
//! 包含：归并排序、快速排序、最大子数组、Strassen矩阵乘法

/// 最大子数组问题（分治解法）
///
/// # 问题
/// 给定数组 `arr`，找到连续子数组使其和最大。
///
/// # 复杂度
/// - 时间：O(n log n)
/// - 空间：O(log n)
pub fn max_subarray_divide_conquer(arr: &[i32]) -> (usize, usize, i32) {
    if arr.is_empty() {
        return (0, 0, 0);
    }
    max_subarray_recursive(arr, 0, arr.len() - 1)
}

fn max_subarray_recursive(arr: &[i32], low: usize, high: usize) -> (usize, usize, i32) {
    if low == high {
        return (low, high, arr[low]);
    }
    let mid = low + (high - low) / 2;
    let (left_low, left_high, left_sum) = max_subarray_recursive(arr, low, mid);
    let (right_low, right_high, right_sum) = max_subarray_recursive(arr, mid + 1, high);
    let (cross_low, cross_high, cross_sum) = max_crossing_subarray(arr, low, mid, high);

    if left_sum >= right_sum && left_sum >= cross_sum {
        (left_low, left_high, left_sum)
    } else if right_sum >= left_sum && right_sum >= cross_sum {
        (right_low, right_high, right_sum)
    } else {
        (cross_low, cross_high, cross_sum)
    }
}

fn max_crossing_subarray(arr: &[i32], low: usize, mid: usize, high: usize) -> (usize, usize, i32) {
    let mut left_sum = i32::MIN;
    let mut sum = 0;
    let mut max_left = mid;
    for i in (low..=mid).rev() {
        sum += arr[i];
        if sum > left_sum {
            left_sum = sum;
            max_left = i;
        }
    }

    let mut right_sum = i32::MIN;
    sum = 0;
    let mut max_right = mid + 1;
    for j in (mid + 1)..=high {
        sum += arr[j];
        if sum > right_sum {
            right_sum = sum;
            max_right = j;
        }
    }

    (max_left, max_right, left_sum + right_sum)
}

/// 快速幂（分治）
///
/// 计算 `base^exp mod modulus`
pub fn mod_pow(mut base: u64, mut exp: u64, modulus: u64) -> u64 {
    if modulus == 1 { return 0; }
    let mut result = 1;
    base %= modulus;
    while exp > 0 {
        if exp % 2 == 1 {
            result = (result * base) % modulus;
        }
        exp >>= 1;
        base = (base * base) % modulus;
    }
    result
}

# [cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_max_subarray() {
        let arr = vec![13, -3, -25, 20, -3, -16, -23, 18, 20, -7, 12, -5, -22, 15, -4, 7];
        let (low, high, sum) = max_subarray_divide_conquer(&arr);
        assert_eq!((low, high), (7, 10));
        assert_eq!(sum, 43);
    }

    #[test]
    fn test_mod_pow() {
        assert_eq!(mod_pow(2, 10, 1000), 24);
        assert_eq!(mod_pow(3, 0, 100), 1);
        assert_eq!(mod_pow(5, 3, 13), 8);
    }
}
```

## 五、形式化证明

### 5.1 最大子数组分治算法的正确性

**定理 5.1** `max_subarray_recursive(arr, low, high)` 返回 $arr[low..high]$ 中最大和子数组的索引和最大和。

*证明*（结构归纳法）：

**基例**: $low = high$。唯一子数组为 $arr[low..low]$，算法正确返回。

**归纳假设**: 假设算法对所有规模 $< n$ 的子数组正确。

**归纳步骤**: 对于规模 $n = high - low + 1$，令 $mid = \lfloor (low+high)/2 \rfloor$。

最大子数组有三种可能位置：
1. 完全位于左半部 $arr[low..mid]$：由归纳假设，递归调用返回正确结果
2. 完全位于右半部 $arr[mid+1..high]$：由归纳假设，递归调用返回正确结果
3. 跨越中点：由 `max_crossing_subarray` 正确计算（从中点向两侧分别扩展最大和）

算法取三者最大值，故返回全局最优解。

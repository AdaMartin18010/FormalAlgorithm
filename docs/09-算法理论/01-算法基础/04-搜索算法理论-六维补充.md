## 一、规范定义深化

### 1.1 搜索问题的形式化规约

**定义 1.1 (搜索问题)** 给定一个集合 $S$（或序列 $A$）和一个查询键 $k$，搜索问题要求：

1. **判定问题**: 判定 $k \in S$ 是否成立（成员查询）
2. **检索问题**: 若 $k \in S$，返回与 $k$ 关联的数据记录
3. **范围查询**: 返回所有满足 $l \leq x \leq r$ 的 $x \in S$
4. **前驱/后继查询**: 返回 $\max\{x \in S : x < k\}$ 或 $\min\{x \in S : x > k\}$

### 1.2 搜索算法的分类

| 数据结构 | 搜索类型 | 时间复杂度 | 空间复杂度 | 备注 |
|----------|----------|-----------|-----------|------|
| 无序数组 | 线性搜索 | $O(n)$ | $O(1)$ | 最简单 |
| 有序数组 | 二分搜索 | $O(\log n)$ | $O(1)$ | 需预处理排序 |
| 二叉搜索树 | 树搜索 | $O(h)$ | $O(n)$ | $h$ 为树高，平衡时 $h = O(\log n)$ |
| 散列表 | 哈希搜索 | $O(1)$ 期望 | $O(n)$ | 需良好哈希函数 |
| B树/B+树 | 磁盘搜索 | $O(\log_B n)$ | $O(n)$ | 优化磁盘 I/O |

## 二、模型设计深化

### 2.1 决策树模型与搜索下界

对于基于比较的搜索算法，其决策树模型要求：

- 每个内部节点代表一次键比较
- 叶节点代表搜索结果（找到或不存在）

**定理 2.1 (比较搜索下界)** 在 $n$ 个元素的有序数组中进行基于比较的搜索，最坏情况下至少需要 $\lceil \log_2(n+1) \rceil$ 次比较。

*证明*：$n$ 个元素的有序数组中，搜索可能有 $n+1$ 种结果（找到 $n$ 个位置之一，或未找到）。决策树必须有至少 $n+1$ 个叶节点。高度为 $h$ 的二叉树最多有 $2^h$ 个叶节点，故 $2^h \geq n+1$，即 $h \geq \lceil \log_2(n+1) \rceil$。

### 2.2 哈希模型

哈希搜索将键 $k$ 映射到表地址 $h(k)$。冲突解决策略：

- **链地址法 (Separate Chaining)**: 每个槽位维护一个链表
- **开放寻址法 (Open Addressing)**: 探测序列 $h(k, i) = (h'(k) + f(i)) \mod m$
  - 线性探测: $f(i) = i$
  - 二次探测: $f(i) = c_1 i + c_2 i^2$
  - 双重哈希: $f(i) = i \cdot h_2(k)$

**装载因子**: $\alpha = n/m$（元素数/槽位数）

**定理 2.2 (简单均匀哈希假设, SUHA)** 若 $h$ 将任意键均匀独立地映射到 $m$ 个槽位之一，则链地址法的期望搜索时间为 $O(1 + \alpha)$。

## 三、数学符号与推导

### 3.1 二分搜索的正确性证明

**算法**: 在有序数组 $A[1..n]$ 中搜索键 $k$。

**循环不变式**: 在每次循环开始时，若 $k$ 存在于 $A$ 中，则 $k \in A[l..r]$。

**初始化**: $l = 1, r = n$。若 $k$ 存在，显然 $k \in A[1..n]$。

**保持**: 设 $m = \lfloor (l+r)/2 \rfloor$。

- 若 $A[m] = k$，返回 $m$。
- 若 $A[m] > k$，则对所有 $j \geq m$，$A[j] \geq A[m] > k$。故若 $k$ 存在，必有 $k \in A[l..m-1]$。令 $r = m-1$，不变式保持。
- 若 $A[m] < k$，对称地令 $l = m+1$，不变式保持。

**终止**: 循环终止时 $l > r$。由不变式的逆否命题，$k \notin A$。

**复杂度**: 每次迭代搜索区间减半，故迭代次数为 $\lceil \log_2(n+1) \rceil$，时间 $O(\log n)$。

### 3.2 二叉搜索树操作的摊还分析

对于伸展树 (Splay Tree)，每次操作后将被访问节点旋转至根。使用**势能法**分析：

定义节点 $x$ 的秩 $r(x) = \lfloor \log_2 |T_x| \rfloor$，其中 $|T_x|$ 为以 $x$ 为根的子树大小。整棵树的势能 $\Phi = \sum_x r(x)$。

**定理 3.1 (伸展操作摊还复杂度)** 伸展操作 $splay(x)$ 的摊还时间为 $O(\log n)$。

*证明概要*: 设 $r(x)$ 和 $r'(x)$ 分别为操作前后 $x$ 的秩。通过分情况讨论（zig, zig-zig, zig-zag），可证单次旋转的摊还成本为 $O(r'(x) - r(x))$。累积得总摊还成本为 $O(\log n)$。

## 四、示例性代码

### 4.1 工程级 Rust 实现

```rust
//! 搜索算法库 — 工程级实现
//!
//! 包含：二分搜索、二分搜索树、散列表

use std::cmp::Ordering;
use std::hash::{Hash, Hasher};
use std::collections::hash_map::DefaultHasher;

/// 二分搜索（迭代版）
///
/// # 前置条件
/// - 输入数组必须按非降序排列
///
/// # 返回值
/// - `Some(index)`：若找到
/// - `None`：若未找到
///
/// # 复杂度
/// - 时间：O(log n)
/// - 空间：O(1)
pub fn binary_search<T: Ord>(arr: &[T], key: &T) -> Option<usize> {
    let mut left = 0usize;
    let mut right = arr.len();
    while left < right {
        let mid = left + (right - left) / 2;
        match arr[mid].cmp(key) {
            Ordering::Equal => return Some(mid),
            Ordering::Less => left = mid + 1,
            Ordering::Greater => right = mid,
        }
    }
    None
}

/// 二分搜索的变体：下界（首个 ≥ key 的位置）
pub fn lower_bound<T: Ord>(arr: &[T], key: &T) -> usize {
    let mut left = 0;
    let mut right = arr.len();
    while left < right {
        let mid = left + (right - left) / 2;
        if arr[mid] < *key {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    left
}

/// 二分搜索的变体：上界（首个 > key 的位置）
pub fn upper_bound<T: Ord>(arr: &[T], key: &T) -> usize {
    let mut left = 0;
    let mut right = arr.len();
    while left < right {
        let mid = left + (right - left) / 2;
        if arr[mid] <= *key {
            left = mid + 1;
        } else {
            right = mid;
        }
    }
    left
}

/// 二叉搜索树节点
#[derive(Debug)]
pub struct TreeNode<K: Ord, V> {
    key: K,
    value: V,
    left: Option<Box<TreeNode<K, V>>>,
    right: Option<Box<TreeNode<K, V>>>,
}

impl<K: Ord, V> TreeNode<K, V> {
    pub fn new(key: K, value: V) -> Self {
        Self { key, value, left: None, right: None }
    }
}

/// 简单二叉搜索树（未平衡版）
pub struct BinarySearchTree<K: Ord, V> {
    root: Option<Box<TreeNode<K, V>>>,
    size: usize,
}

impl<K: Ord, V> BinarySearchTree<K, V> {
    pub fn new() -> Self {
        Self { root: None, size: 0 }
    }

    pub fn insert(&mut self, key: K, value: V) {
        self.size += 1;
        Self::insert_node(&mut self.root, key, value);
    }

    fn insert_node(node: &mut Option<Box<TreeNode<K, V>>>, key: K, value: V) {
        match node {
            None => *node = Some(Box::new(TreeNode::new(key, value))),
            Some(n) => match key.cmp(&n.key) {
                Ordering::Less => Self::insert_node(&mut n.left, key, value),
                Ordering::Greater => Self::insert_node(&mut n.right, key, value),
                Ordering::Equal => n.value = value,
            },
        }
    }

    pub fn search(&self, key: &K) -> Option<&V> {
        Self::search_node(&self.root, key)
    }

    fn search_node<'a>(node: &'a Option<Box<TreeNode<K, V>>>, key: &K) -> Option<&'a V> {
        match node {
            None => None,
            Some(n) => match key.cmp(&n.key) {
                Ordering::Equal => Some(&n.value),
                Ordering::Less => Self::search_node(&n.left, key),
                Ordering::Greater => Self::search_node(&n.right, key),
            },
        }
    }
}

/// 散列表（链地址法）
pub struct HashTable<K: Hash + Eq, V> {
    buckets: Vec<Vec<(K, V)>>,
    size: usize,
    capacity: usize,
}

impl<K: Hash + Eq + Clone, V> HashTable<K, V> {
    pub fn new(capacity: usize) -> Self {
        let mut buckets = Vec::with_capacity(capacity);
        buckets.resize_with(capacity, Vec::new);
        Self { buckets, size: 0, capacity }
    }

    fn hash(&self, key: &K) -> usize {
        let mut hasher = DefaultHasher::new();
        key.hash(&mut hasher);
        (hasher.finish() as usize) % self.capacity
    }

    pub fn insert(&mut self, key: K, value: V) {
        let idx = self.hash(&key);
        let bucket = &mut self.buckets[idx];
        for (k, v) in bucket.iter_mut() {
            if *k == key {
                *v = value;
                return;
            }
        }
        bucket.push((key, value));
        self.size += 1;
    }

    pub fn get(&self, key: &K) -> Option<&V> {
        let idx = self.hash(key);
        self.buckets[idx]
            .iter()
            .find(|(k, _)| k == key)
            .map(|(_, v)| v)
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_binary_search_found() {
        let arr = vec![1, 3, 5, 7, 9, 11];
        assert_eq!(binary_search(&arr, &5), Some(2));
        assert_eq!(binary_search(&arr, &1), Some(0));
        assert_eq!(binary_search(&arr, &11), Some(5));
    }

    #[test]
    fn test_binary_search_not_found() {
        let arr = vec![1, 3, 5, 7, 9];
        assert_eq!(binary_search(&arr, &4), None);
        assert_eq!(binary_search(&arr, &0), None);
        assert_eq!(binary_search(&arr, &10), None);
    }

    #[test]
    fn test_bounds() {
        let arr = vec![1, 2, 2, 3, 4];
        assert_eq!(lower_bound(&arr, &2), 1);
        assert_eq!(upper_bound(&arr, &2), 3);
    }

    #[test]
    fn test_bst() {
        let mut bst = BinarySearchTree::new();
        bst.insert(5, "five");
        bst.insert(3, "three");
        bst.insert(7, "seven");
        assert_eq!(bst.search(&5), Some(&"five"));
        assert_eq!(bst.search(&4), None);
    }

    #[test]
    fn test_hash_table() {
        let mut ht = HashTable::new(16);
        ht.insert("key1", 100);
        ht.insert("key2", 200);
        assert_eq!(ht.get(&"key1"), Some(&100));
        assert_eq!(ht.get(&"key3"), None);
    }
}
```

## 五、形式化证明

### 5.1 二分搜索正确性的 Lean4 草图

```lean4
import Mathlib

-- 定义有序性
inductive Sorted {α : Type} [LinearOrder α] : List α → Prop
  | nil : Sorted []
  | single (a : α) : Sorted [a]
  | cons (a b : α) (bs : List α) : a ≤ b → Sorted (b :: bs) → Sorted (a :: b :: bs)

-- 二分搜索的规范定义
def binarySearch {α : Type} [LinearOrder α] (arr : Array α) (key : α)
    (lo hi : Nat) (h : lo ≤ hi) (hs : Sorted arr.toList) :
    Option { i : Nat // lo ≤ i ∧ i < hi ∧ arr[i] = key } ⊕ { // key ∉ arr[lo:hi] } :=
  if hlt : lo < hi then
    let mid := (lo + hi) / 2
    match compare key arr[mid] with
    | .eq => .inl ⟨mid, by omega, by omega, by simp [arr.getElem_mid]⟩
    | .lt => binarySearch arr key lo mid (by omega) hs
    | .gt => binarySearch arr key (mid + 1) hi (by omega) hs
  else
    .inr ⟨by omega⟩
```

## 一、规范定义深化

### 1.1 数据结构的形式化定义

**定义 1.1 (数据结构)** 数据结构是数据对象及其上关系的数学表示，形式化为三元组
$$\mathcal{D} = (D, R, F)$$
其中：

- $D$：数据元素的有限集合
- $R \subseteq D \times D$：元素间的关系集合（如线性序、父子关系、邻接关系）
- $F = \{f_1, f_2, \ldots\}$：施加于 $D$ 上的基本操作集合

**定义 1.2 (抽象数据类型, ADT)** ADT 是从实现中分离出的数据规约，定义为
$$\text{ADT} = (\text{Name}, \text{Domain}, \text{Operations}, \text{Axioms})$$
其中 Axioms 刻画操作间的代数约束。例如，栈 ADT 的核心公理：
$$\text{top}(\text{push}(S, x)) = x, \quad \text{pop}(\text{push}(S, x)) = S$$

### 1.2 结构不变式 (Structural Invariants)

数据结构必须维持的不变式保证其语义正确性：

| 数据结构 | 结构不变式 | 违反后果 |
|----------|-----------|----------|
| **二叉搜索树** | $\forall v \in \text{left}(u): v.\text{key} < u.\text{key}$ | 搜索/插入语义错误 |
| **堆** | $\forall i: A[i] \leq A[2i] \land A[i] \leq A[2i+1]$ | 提取最小值错误 |
| **红黑树** | 红黑性质（根黑、红节点子黑、黑高一致） | 高度退化为 $O(n)$ |
| **AVL树** | $\forall u: |\text{height}(\text{left}(u)) - \text{height}(\text{right}(u))| \leq 1$ | 平衡性破坏 |

## 二、模型设计深化

### 2.1 指针机模型 (Pointer Machine Model)

指针机是分析基于指针的数据结构（链表、树、图）的标准模型：

- **存储单元**: 每个单元存储常数个字段（数据 + $O(1)$ 个指针）
- **操作代价**: 指针跟随、比较、赋值各为 1 单位时间
- **禁止操作**: 随机访问数组单元（区别于 RAM 模型）

**定理 2.1 (排序下界)** 在比较模型中，基于指针机的排序算法时间复杂度下界为 $\Omega(n \log n)$。

### 2.2 持久化数据结构的模型

**定义 2.2 (持久化)** 数据结构是部分持久化 (partially persistent) 的，如果允许查询所有历史版本，但仅可修改最新版本；若所有版本均可修改，则为完全持久化 (fully persistent)。

**路径复制法 (Path Copying)**：修改操作仅复制从根到修改节点的路径，空间开销与路径长度成正比。

## 三、数学符号与推导

### 3.1 摊还分析的势能方法

**定义 3.1 (势能函数)** 设 $D_i$ 为第 $i$ 次操作后的数据结构状态，势能函数 $\Phi: \{D_i\} \to \mathbb{R}_{\geq 0}$ 满足 $\Phi(D_0) = 0$。

第 $i$ 次操作的摊还代价为：
$$\hat{c}_i = c_i + \Phi(D_i) - \Phi(D_{i-1})$$

其中 $c_i$ 为实际代价。对 $n$ 次操作：
$$\sum_{i=1}^n \hat{c}_i = \sum_{i=1}^n c_i + \Phi(D_n) - \Phi(D_0) \geq \sum_{i=1}^n c_i$$

**示例：二叉计数器增量** 设 $\Phi$ 为计数器中 1 的个数。第 $i$ 次增量翻转 $t$ 位（$t-1$ 个 1 变 0，1 个 0 变 1）：
$$c_i = t, \quad \Delta\Phi = 1 - (t-1) = 2 - t$$
$$\hat{c}_i = t + (2 - t) = 2 = O(1)$$

### 3.2 并查集的逆阿克曼函数上界

**定理 3.2 (Tarjan, 1975)** 使用按秩合并与路径压缩的并查集，$m$ 次操作（$n$ 次 Make-Set）的总时间复杂度为 $O(m \cdot \alpha(n))$，其中 $\alpha$ 为逆阿克曼函数。

$$\alpha(n) = \min\{k: A_k(1) \geq n\}$$

阿克曼函数 $A_k(j)$ 增长极快，故 $\alpha(n) \leq 4$ 对所有实际可行的 $n$ 成立。

### 4.1 工程级 Rust 实现

```rust
//! 数据结构理论 — 工程级实现示例
//!
//! 包含：泛型动态数组（Vec 封装，带容量管理）、
//!       持久化栈（函数式，共享子结构）、
//!       并查集（按秩合并 + 路径压缩）。

use std::rc::Rc;

// ============================================================
// 泛型动态数组（工程级封装）
// ============================================================

/// 带显式容量策略的动态数组
///
/// # 不变式
/// - `len <= capacity`
/// - 扩容时 capacity 至少翻倍
#[derive(Debug, Clone)]
pub struct DynamicArray<T> {
    buf: Vec<T>,
}

impl<T> DynamicArray<T> {
    pub fn new() -> Self {
        Self { buf: Vec::new() }
    }

    pub fn with_capacity(cap: usize) -> Self {
        Self { buf: Vec::with_capacity(cap) }
    }

    pub fn len(&self) -> usize { self.buf.len() }
    pub fn is_empty(&self) -> bool { self.buf.is_empty() }

    pub fn push(&mut self, value: T) {
        self.buf.push(value);
    }

    pub fn pop(&mut self) -> Option<T> {
        self.buf.pop()
    }

    pub fn get(&self, index: usize) -> Option<&T> {
        self.buf.get(index)
    }

    /// 摊还 $O(1)$ 尾部插入
    pub fn amortized_push_analysis(&self) -> &'static str {
        // 势能函数 $\Phi = 2 \times \text{len} - \text{capacity}$
        // 当容量充足时：实际代价 1，$\Delta\Phi = 2$，摊还代价 3
        // 扩容时（len == capacity）：实际代价 ~capacity，$\Delta\Phi = 2 - \text{capacity}$，摊还代价 2
        "Amortized O(1) by potential method"
    }
}

// ============================================================
// 持久化栈（函数式，共享子结构）
// ============================================================

#[derive(Debug, Clone)]
pub enum PersistentStack<T> {
    Empty,
    Cons(T, Rc<PersistentStack<T>>),
}

impl<T: Clone> PersistentStack<T> {
    pub fn new() -> Self { PersistentStack::Empty }

    pub fn is_empty(&self) -> bool {
        matches!(self, PersistentStack::Empty)
    }

    /// 压栈：返回新栈，旧栈不变（共享尾部）
    pub fn push(&self, value: T) -> Self {
        PersistentStack::Cons(value, Rc::new(self.clone()))
    }

    /// 弹栈：返回 (Option<首元素>, 新栈)
    pub fn pop(&self) -> (Option<T>, Self) {
        match self {
            PersistentStack::Empty => (None, PersistentStack::Empty),
            PersistentStack::Cons(v, rest) => (Some(v.clone()), rest.as_ref().clone()),
        }
    }

    pub fn peek(&self) -> Option<&T> {
        match self {
            PersistentStack::Empty => None,
            PersistentStack::Cons(v, _) => Some(v),
        }
    }
}

// ============================================================
// 并查集（按秩合并 + 路径压缩）
// ============================================================

pub struct UnionFind {
    parent: Vec<usize>,
    rank: Vec<u32>,
}

impl UnionFind {
    pub fn new(n: usize) -> Self {
        Self {
            parent: (0..n).collect(),
            rank: vec![0; n],
        }
    }

    pub fn find(&mut self, x: usize) -> usize {
        if self.parent[x] != x {
            self.parent[x] = self.find(self.parent[x]);
        }
        self.parent[x]
    }

    pub fn union(&mut self, x: usize, y: usize) -> bool {
        let (rx, ry) = (self.find(x), self.find(y));
        if rx == ry { return false; }
        // 按秩合并
        if self.rank[rx] < self.rank[ry] {
            self.parent[rx] = ry;
        } else if self.rank[rx] > self.rank[ry] {
            self.parent[ry] = rx;
        } else {
            self.parent[ry] = rx;
            self.rank[rx] += 1;
        }
        true
    }

    pub fn connected(&mut self, x: usize, y: usize) -> bool {
        self.find(x) == self.find(y)
    }
}

#[cfg(test)]
mod tests {

    fn test_dynamic_array() {

        arr.push(10);
        arr.push(20);
        assert_eq!(arr.get(0), Some(&10));
        assert_eq!(arr.pop(), Some(20));
        assert_eq!(arr.len(), 1);
    }

    #[test]
    fn test_persistent_stack() {
        let s1 = PersistentStack::new();
        let s2 = s1.push(1);
        let s3 = s2.push(2);
        let (v, s4) = s3.pop();
        assert_eq!(v, Some(2));
        assert!(s1.is_empty());
        assert_eq!(s2.peek(), Some(&1));
    }

    #[test]
    fn test_union_find() {
        let mut uf = UnionFind::new(5);
        assert!(uf.union(0, 1));
        assert!(uf.union(2, 3));
        assert!(!uf.union(0, 1));
        assert!(uf.connected(0, 1));
        assert!(!uf.connected(1, 2));
        uf.union(1, 2);
        assert!(uf.connected(0, 3));
    }
}
```

### 5.2 动态数组摊还 $O(1)$ 的严格证明

**定理 5.1** 采用翻倍扩容策略的动态数组，$n$ 次连续 `push` 操作的总时间为 $O(n)$。

*证明*：定义势能函数 $\Phi(D_i) = 2 \cdot \text{len}_i - \text{cap}_i$。

- 初始时 $\text{len}_0 = \text{cap}_0 = 0$，$\Phi(D_0) = 0$
- 始终有 $\text{len}_i \leq \text{cap}_i$，故 $\Phi(D_i) \geq 0$

**情况 1**（无需扩容）：$\text{len}_{i-1} < \text{cap}_{i-1}$
$$c_i = 1, \quad \Delta\Phi = 2(\text{len}_i - \text{len}_{i-1}) - (\text{cap}_i - \text{cap}_{i-1}) = 2 - 0 = 2$$
$$\hat{c}_i = 1 + 2 = 3$$

**情况 2**（扩容）：$\text{len}_{i-1} = \text{cap}_{i-1} = k$
$$c_i = k + 1 \text{（复制 $k$ 个元素 + 插入新元素）}$$
$$\Delta\Phi = 2(k+1) - 2k - (2k - k) = 2 - k$$
$$\hat{c}_i = (k+1) + (2-k) = 3$$

因此每次 `push` 摊还代价 $\leq 3$，$n$ 次总代价 $\leq 3n = O(n)$。

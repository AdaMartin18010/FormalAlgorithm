## 1. 引言

字符串是计算机科学中的基本数据类型。后缀数组和后缀树是处理字符串问题的强大数据结构，支持高效的模式匹配、重复检测等操作 [Manber 1990]。

## 2. 后缀数组

**定义 2.1 (后缀)**：字符串 $S[0..n-1]$ 的后缀 $S_i$ 是从位置 $i$ 开始的子串 $S[i..n-1]$。

**定义 2.2 (后缀数组)**：后缀数组 $SA$ 是一个排列，满足 $S_{SA[0]} < S_{SA[1]} < \cdots < S_{SA[n-1]}$（字典序）。

**示例**：$S = "banana$"$（$为哨兵字符，字典序最小）

| 索引 | 后缀 | 排序后位置 |
|------|------|-----------|
| 0 | banana$ | 1 |
| 1 | anana$ | 0 |
| 2 | nana$ | 3 |
| 3 | ana$ | 2 |
| 4 | na$ | 5 |
| 5 | a$ | 4 |
| 6 | $ | 6 |

后缀数组：$[6, 5, 3, 1, 0, 4, 2]$（按字典序排序后的起始索引）

### 2.1 $O(n \log n)$ 构造算法

**算法思想**：倍增法

```rust
/// 后缀数组 - O(n log n) 倍增法
pub fn suffix_array_naive(s: &[u8]) -> Vec<usize> {
    let n = s.len();
    let mut sa: Vec<usize> = (0..n).collect();

    // 按长度k倍增排序
    let mut k = 1;
    while k < n {
        sa.sort_by(|&i, &j| {
            let si = &s[i..(i + k).min(n)];
            let sj = &s[j..(j + k).min(n)];
            si.cmp(sj)
        });
        k *= 2;
    }

    sa
}
```

**复杂度分析**：

- 每轮排序：$O(n \log n)$
- 轮数：$O(\log n)$
- 总计：$O(n \log^2 n)$，使用基数排序可优化至 $O(n \log n)$

### 2.2 线性时间构造 (SA-IS)

**算法思想** [Nong 2009]：诱导排序（Induced Sorting）

**字符分类**：

- **S型**：$S[i] < S[i+1]$ 或 ($S[i] = S[i+1]$ 且 $S_{i+1}$ 是S型)
- **L型**：$S[i] > S[i+1]$ 或 ($S[i] = S[i+1]$ 且 $S_{i+1}$ 是L型)

**关键观察**：

- LMS（Left-Most S-type）后缀唯一确定诱导顺序
- 递归缩减问题规模

**定理 2.1 (SA-IS复杂度)**：SA-IS算法时间复杂度为 $O(n)$，空间复杂度为 $O(n)$。

## 3. 最长公共前缀数组 (LCP)

**定义 3.1 (LCP)**：$LCP[i]$ 是后缀 $S_{SA[i]}$ 和 $S_{SA[i-1]}$ 的最长公共前缀长度。

**性质**：

- $LCP[0] = 0$（无前一后缀）
- 可用于高效字符串匹配

### 3.1 Kasai算法

**算法思想** [Kasai 2001]：利用后缀的相邻关系，线性时间计算。

```rust
/// LCP数组 - Kasai算法 O(n)
pub fn lcp_array(s: &[u8], sa: &[usize]) -> Vec<usize> {
    let n = s.len();
    let mut rank = vec![0; n];
    let mut lcp = vec![0; n];

    // 构建rank数组：rank[i] = 后缀i在SA中的位置
    for i in 0..n {
        rank[sa[i]] = i;
    }

    let mut h = 0;
    for i in 0..n {
        if rank[i] > 0 {
            let j = sa[rank[i] - 1];
            // 从当前h开始比较
            while i + h < n && j + h < n && s[i + h] == s[j + h] {
                h += 1;
            }
            lcp[rank[i]] = h;
            if h > 0 {
                h -= 1;
            }
        }
    }

    lcp
}
```

**定理 3.2 (Kasai正确性)**：Kasai算法正确计算LCP数组，时间复杂度 $O(n)$。

## 4. 后缀树

**定义 4.1 (后缀树)**：字符串 $S$ 的后缀树是包含所有后缀的紧凑Trie。

**性质**：

- 边标记为子串
- 内部节点至少有两个子节点
- 总节点数 $O(n)$

### 4.1 从后缀数组构造

**定理 4.1**：给定后缀数组和LCP数组，可在 $O(n)$ 时间构造后缀树。

**算法**：

1. 按SA顺序处理后缀
2. 使用栈维护当前路径
3. 根据LCP值分裂或扩展

### 4.2 Ukkonen算法

**定理 4.2 (Ukkonen 1995)**：存在在线 $O(n)$ 算法构造后缀树。

**关键思想**：

- 活动点（active point）优化
- 后缀链接（suffix link）跳转
- 隐式后缀树逐步扩展

## 5. 应用

### 5.1 最长重复子串

**问题**：找字符串中最长的重复子串。

**解法**：后缀数组中相邻后缀的最大LCP。

```rust
pub fn longest_repeated_substring(s: &[u8]) -> usize {
    let sa = suffix_array_naive(s);
    let lcp = lcp_array(s, &sa);
    *lcp.iter().max().unwrap_or(&0)
}
```

**复杂度**：$O(n \log n)$ 或 $O(n)$（使用线性SA构造）

### 5.2 模式匹配

**问题**：在文本 $T$ 中找模式 $P$ 的所有出现。

**解法**：

1. 对 $T$ 构建后缀数组
2. 二分查找 $P$ 的范围
3. 输出范围内的后缀位置

**复杂度**：$O(|P| \log |T|)$，预处理 $O(|T|)$

### 5.3 最长公共子串

**问题**：找两个字符串的最长公共子串。

**解法**：

1. 连接两个字符串：$S = S_1 + \# + S_2$
2. 构建后缀数组
3. 找来自不同字符串的后缀对的最大LCP

## 6. 复杂度分析

| 算法/结构 | 构造时间 | 空间 | 模式匹配 |
|----------|---------|------|---------|
| 后缀数组 ($O(n \log n)$) | $O(n \log n)$ | $O(n)$ | $O(m \log n)$ |
| 后缀数组 (SA-IS) | $O(n)$ | $O(n)$ | $O(m \log n)$ |
| 后缀树 (Ukkonen) | $O(n)$ | $O(n)$ | $O(m)$ |
| LCP数组 (Kasai) | $O(n)$ | $O(n)$ | - |

## 8. 总结

后缀数组和后缀树是处理字符串问题的核心数据结构：

- 支持高效的子串查询
- 构造算法从 $O(n \log n)$ 优化到 $O(n)$
- 与LCP数组结合解决多种问题

## 一、概念定义 (Concept Definition)

### 1.1 单链表 (Singly Linked List)

**定义 1.1.1** (形式化)

**单链表**是由节点组成的序列，每个节点包含一个数据域和一个指向后继节点的指针域。

$$\text{Node} = (\text{key}, \text{next})$$

其中 $\text{next} \in \text{Node} \cup \{\text{NIL}\}$。

链表 $L$ 由头指针 $head(L)$ 标识。当 $head(L) = \text{NIL}$ 时，链表为空。

### 1.2 双向链表 (Doubly Linked List)

**定义 1.2.1** (形式化)

**双向链表**的每个节点包含前驱指针和后继指针：

$$\text{Node} = (\text{key}, \text{prev}, \text{next})$$

通常维护两个哨兵指针：$head$（指向首节点）和 $tail$（指向尾节点）。

### 1.3 循环链表 (Circular Linked List)

**定义 1.3.1** (形式化)

**循环链表**中，尾节点的 next 指针指向头节点，形成闭环：

$$\text{tail.next} = \text{head}$$

在单向循环链表中，从任一节点出发均可遍历全部节点。

## 二、属性 (Properties)

### 2.1 操作复杂度对比

| 操作 | 单链表 | 双向链表 | 循环链表 |
|------|--------|----------|----------|
| 头插 | $O(1)$ | $O(1)$ | $O(1)$ |
| 尾插 | $O(n)$ | $O(1)^*$ | $O(1)^*$ |
| 头删 | $O(1)$ | $O(1)$ | $O(1)$ |
| 尾删 | $O(n)$ | $O(1)^*$ | $O(1)^*$ |
| 查找 | $O(n)$ | $O(n)$ | $O(n)$ |
| 删除(已知节点) | $O(n)$ | $O(1)$ | $O(1)$ |

*$^*$ 需要维护尾指针

### 2.2 哨兵节点 (Sentinel)

使用哑节点（dummy/sentinel）可简化边界处理：

- **单链表**：dummy head 使空表与非空表操作统一
- **双向链表**：dummy head + dummy tail 避免大量空指针判断
- **循环链表**：自然无需 dummy（尾指向头已是统一结构）

### 3.1 概念关系表

| 源概念 | 目标概念 | 关系类型 | 说明 |
|--------|----------|----------|------|
| 双向链表 | 单链表 | extends | 增加前驱指针 |
| 循环链表 | 单链表 | specializes | 尾节点约束 |

### 3.2 链表与数组对比

| 维度 | 数组 | 链表 |
|------|------|------|
| 内存布局 | 连续 | 离散 |
| 随机访问 | $O(1)$ | $O(n)$ |
| 插入/删除(已知位置) | $O(n)$ | $O(1)$ |
| 缓存命中率 | 高 | 低 |
| 额外空间开销 | 无 | 指针存储 |

## 四、解释 (Explanation)

### 4.1 动机与直观

**为什么需要双向链表？**

想象你在火车站排队：单链表就像只能向前看的人，要找前面第5个人只能一个一个问。双向链表则像每个人都知道前后是谁，不仅可以快速调整队伍，还可以从尾部直接离开。

**循环链表的应用直观**：

操作系统进程调度中的 Round-Robin 算法就是循环链表的典型应用——CPU 时间片轮转，每个进程执行完后"旋转"到队尾等待下一轮。

### 4.2 与已有概念的联系

**链表 ↔ 递归**：链表的定义本身就是递归的——一个链表要么为空，要么是一个节点后跟一个链表。因此链表的许多操作（如反转、合并）天然适合递归实现。

**链表 ↔ 图**：链表可视为一种极特殊的图——每个节点出度为 1（单链表）或 2（双向链表），且无环（除循环链表外）。

## 五、论证 (Argumentation)

### 5.1 单链表反转的正确性

**算法**：迭代反转单链表

```
prev = NIL
current = head
while current != NIL:
    next = current.next
    current.next = prev
    prev = current
    current = next
head = prev
```

**循环不变式**：

- 在第 $k$ 次迭代开始时，$prev$ 指向已反转部分的头节点
- $current$ 指向待反转部分的头节点
- 已反转部分与原链表的后半部分不再相连

**终止**：当 $current = \text{NIL}$ 时，整个链表已反转，$prev$ 为新头节点。$\square$

### 5.2 快慢指针找环

**Floyd 判圈算法**：

- 慢指针每次走 1 步，快指针每次走 2 步
- 若链表有环，两者必在环内相遇
- 若链表无环，快指针先到达 NIL

**证明要点**：
设环长为 $c$，进入环前长度为 $m$。当慢指针进入环时，快指针已在环内某位置。此后相对速度为 1 步/单位时间，快指针最多再追 $c-1$ 步即可与慢指针相遇。$\square$

## 六、形式证明 (Formal Proof)

### 6.1 双向链表删除的 O(1) 正确性

**定理 6.1.1**：在双向链表中，给定指向待删节点的指针，可在 $O(1)$ 时间内完成删除。

**证明**：

设待删节点为 $x$，其前驱为 $x.prev$，后继为 $x.next$。

执行以下指针调整：

1. $x.prev.next \leftarrow x.next$
2. $x.next.prev \leftarrow x.prev$

以上两步各为 $O(1)$。调整完成后，$x$ 不再被任何指针引用，可从内存中释放（垃圾回收语言中由 GC 处理）。

由于无需遍历查找前驱，总时间为 $O(1)$。$\square$

### 6.2 循环链表 rotate 操作的 O(1) 正确性

**定理 6.2.1**：单向循环链表的 rotate 操作（将头节点移到尾部）可在 $O(1)$ 时间内完成。

**证明**：

设循环链表维护尾指针 $tail$，则头节点为 $tail.next$。

rotate 操作只需：
$$tail \leftarrow tail.next$$

即让原来的头节点成为新的尾节点。该操作仅修改一个指针，时间为 $O(1)$。$\square$

## 七、应用场景

| 应用场景 | 推荐链表类型 | 原因 |
|---------|-------------|------|
| LRU 缓存 | 双向链表 + HashMap | O(1) 移动与删除 |
| 内存分配器 | 双向链表/循环链表 | 快速合并空闲块 |
| 音乐播放列表 | 循环链表 | 单曲循环/顺序播放 |
| 多项式运算 | 单链表 | 稀疏存储，项数不定 |
| 撤销操作栈 | 双向链表 | 支持前进/后退 |

## 八、扩展变体

### 8.1 XOR 链表

利用地址的异或运算，用一个指针字段同时存储 prev 和 next 的信息：
$$ptr = \text{prev} \oplus \text{next}$$
空间复杂度从 $2n$ 个指针降到 $n$ 个，但需要顺序遍历，无法随机访问。

### 8.2 跳表 (Skip List)

在有序链表上增加多层"快速通道"，实现期望 $O(\log n)$ 的查找、插入、删除。

### 8.3 非连续链表 (Unrolled Linked List)

每个节点存储一个小数组（如 4~8 个元素），减少指针开销，提高缓存命中率。

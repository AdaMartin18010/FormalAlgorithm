## 一、规范定义深化

### 1.1 排序问题的形式化规约

**定义 1.1 (排序问题)** 给定一个有限序列 $A = \langle a_1, a_2, \ldots, a_n \rangle$，其中每个元素来自全序集 $(S, \preceq)$。排序问题要求输出一个排列 $A' = \langle a'_1, a'_2, \ldots, a'_n \rangle$，使得：

1. **有序性**: $a'_1 \preceq a'_2 \preceq \cdots \preceq a'_n$
2. **稳定性**（可选）: 若 $a_i = a_j$ 且 $i < j$，则在 $A'$ 中 $a_i$ 位于 $a_j$ 之前
3. **排列约束**: $A'$ 是 $A$ 的排列，即存在双射 $\pi: \{1,\ldots,n\} \to \{1,\ldots,n\}$ 使得 $a'_i = a_{\pi(i)}$

### 1.2 排序算法的分类维度

| 维度 | 类型 | 说明 |
|------|------|------|
| **时间复杂度** | 比较排序 / 线性时间排序 | 基于比较 vs 基于数值假设 |
| **空间复杂度** | 原地排序 / 非原地排序 | $O(1)$ 辅助空间 vs $O(n)$ 或更多 |
| **稳定性** | 稳定 / 不稳定 | 相等元素的相对顺序是否保持 |
| **适应性** | 自适应 / 非自适应 | 是否利用输入已有序程度 |
| **在线性** | 在线 / 离线 | 是否必须在输入完全到达前输出 |

## 二、模型设计深化

### 2.1 比较模型 (Comparison Model)

在比较模型中，算法只能通过元素间的比较操作 ($<, \leq, =, \geq, >$) 获取输入信息。该模型的关键性质：

**决策树模型**：任何基于比较的排序算法对应一棵决策树，其中：
- 每个内部节点标记为一次比较 $a_i \leq a_j$
- 每个叶节点标记为一种输出排列
- 从根到叶的路径对应算法在特定输入上的执行轨迹

**定理 2.1 (比较排序下界)** 任何基于比较的排序算法在最坏情况下至少需要 $\Omega(n \log n)$ 次比较。

*证明*：$n$ 个元素有 $n!$ 种排列。高度为 $h$ 的二叉树最多有 $2^h$ 个叶节点。因此：
$$2^h \geq n! \implies h \geq \log(n!) = \sum_{k=1}^n \log k \geq n \log n - n \log e = \Omega(n \log n)$$

### 2.2 非比较模型

当输入满足额外假设时，可突破 $\Omega(n \log n)$ 下界：

- **计数排序**: 元素为 $[0, k]$ 范围内的整数，时间 $O(n + k)$
- **基数排序**: $d$ 位数字，每范围 $[0, k]$，时间 $O(d(n + k))$
- **桶排序**: 元素均匀分布于 $[0, 1)$，期望时间 $O(n)$

## 三、数学符号与推导

### 3.1 插入排序的正确性证明

**循环不变式**: 在第 $j$ 轮迭代开始时，子数组 $A[1..j-1]$ 包含原数组 $A[1..j-1]$ 中的元素，且已按非降序排列。

**初始化**: $j = 2$ 时，$A[1..1]$ 仅含一个元素，显然有序。

**保持**: 假设 $j$ 轮开始时不变式成立。算法将 $A[j]$ 插入到 $A[1..j-1]$ 的正确位置，保持有序性。插入后 $A[1..j]$ 有序。

**终止**: $j = n+1$ 时，$A[1..n]$ 有序，即为完整排序结果。

**复杂度分析**:
- 最坏情况（逆序输入）: $T(n) = \sum_{j=2}^n (j-1) = \frac{n(n-1)}{2} = \Theta(n^2)$
- 最好情况（已有序）: $T(n) = \Theta(n)$
- 平均情况: $T(n) = \Theta(n^2)$

### 3.2 归并排序的递归分析

归并排序满足递推关系：
$$T(n) = 2T(n/2) + \Theta(n)$$

由主定理（Case 2），$a = 2, b = 2, f(n) = \Theta(n)$，$\log_b a = 1$，$f(n) = \Theta(n^{\log_b a})$，故 $T(n) = \Theta(n \log n)$。

**严格推导**（展开法）：
$$\begin{aligned}
T(n) &= 2T(n/2) + cn \\
     &= 2[2T(n/4) + c(n/2)] + cn = 4T(n/4) + 2cn \\
     &= \cdots \\
     &= 2^k T(n/2^k) + kcn
\end{aligned}$$

当 $n/2^k = 1$ 即 $k = \log n$ 时：
$$T(n) = nT(1) + cn\log n = \Theta(n \log n)$$

### 3.3 快速排序的期望分析

假设所有输入排列等概率，快速排序的期望比较次数满足：
$$E[C(n)] = (n+1) + \frac{2}{n}\sum_{q=1}^n E[C(q-1)]$$

通过归纳法可证 $E[C(n)] \leq 2n \ln n = O(n \log n)$。

## 四、示例性代码

### 4.1 工程级 Rust 实现

```rust
//! 排序算法库 — 工程级实现
//!
//! 包含：插入排序、归并排序、堆排序、快速排序、计数排序
//! 所有实现均为泛型，支持稳定/非稳定选项，包含完整单元测试。

use std::cmp::Ordering;

/// 插入排序（稳定、原地）
///
/// # 复杂度
/// - 时间：最坏 O(n²)，平均 O(n²)，最好 O(n)
/// - 空间：O(1)
pub fn insertion_sort<T: Ord>(arr: &mut [T]) {
    for i in 1..arr.len() {
        let mut j = i;
        while j > 0 && arr[j - 1] > arr[j] {
            arr.swap(j - 1, j);
            j -= 1;
        }
    }
}

/// 归并排序（稳定、非原地）
///
/// # 复杂度
/// - 时间：最坏 O(n log n)，平均 O(n log n)，最好 O(n log n)
/// - 空间：O(n)
pub fn merge_sort<T: Ord + Clone>(arr: &[T]) -> Vec<T> {
    if arr.len() <= 1 {
        return arr.to_vec();
    }
    let mid = arr.len() / 2;
    let left = merge_sort(&arr[..mid]);
    let right = merge_sort(&arr[mid..]);
    merge(&left, &right)
}

fn merge<T: Ord + Clone>(left: &[T], right: &[T]) -> Vec<T> {
    let mut result = Vec::with_capacity(left.len() + right.len());
    let (mut i, mut j) = (0, 0);
    while i < left.len() && j < right.len() {
        if left[i] <= right[j] {
            result.push(left[i].clone());
            i += 1;
        } else {
            result.push(right[j].clone());
            j += 1;
        }
    }
    result.extend_from_slice(&left[i..]);
    result.extend_from_slice(&right[j..]);
    result
}

/// 堆排序（非稳定、原地）
///
/// # 复杂度
/// - 时间：最坏 O(n log n)
/// - 空间：O(1)
pub fn heap_sort<T: Ord>(arr: &mut [T]) {
    let n = arr.len();
    if n < 2 { return; }

    // 建堆：从最后一个非叶节点开始下沉
    for i in (0..n / 2).rev() {
        sift_down(arr, i, n);
    }

    // 依次将堆顶与末尾交换，然后下沉
    for end in (1..n).rev() {
        arr.swap(0, end);
        sift_down(arr, 0, end);
    }
}

fn sift_down<T: Ord>(arr: &mut [T], mut root: usize, end: usize) {
    loop {
        let left = 2 * root + 1;
        let right = 2 * root + 2;
        let mut largest = root;
        if left < end && arr[left] > arr[largest] {
            largest = left;
        }
        if right < end && arr[right] > arr[largest] {
            largest = right;
        }
        if largest == root { break; }
        arr.swap(root, largest);
        root = largest;
    }
}

/// 快速排序（非稳定、原地、随机化）
///
/// # 复杂度
/// - 时间：期望 O(n log n)，最坏 O(n²)
/// - 空间：O(log n)（递归栈）
use rand::seq::SliceRandom;
use rand::thread_rng;

pub fn quick_sort<T: Ord>(arr: &mut [T]) {
    if arr.len() <= 1 { return; }
    arr.shuffle(&mut thread_rng());
    qsort(arr);
}

fn qsort<T: Ord>(arr: &mut [T]) {
    if arr.len() <= 1 { return; }
    let pivot_idx = partition(arr);
    qsort(&mut arr[..pivot_idx]);
    qsort(&mut arr[pivot_idx + 1..]);
}

fn partition<T: Ord>(arr: &mut [T]) -> usize {
    let len = arr.len();
    let mut i = 0;
    for j in 0..len - 1 {
        if arr[j] <= arr[len - 1] {
            arr.swap(i, j);
            i += 1;
        }
    }
    arr.swap(i, len - 1);
    i
}

/// 计数排序（稳定、非原地）
///
/// # 前提
/// 输入元素为 `usize`，且值域为 `[0, k]`。
///
/// # 复杂度
/// - 时间：O(n + k)
/// - 空间：O(n + k)
pub fn counting_sort(arr: &[usize], k: usize) -> Vec<usize> {
    let mut count = vec![0usize; k + 1];
    for &x in arr {
        count[x] += 1;
    }
    for i in 1..=k {
        count[i] += count[i - 1];
    }
    let mut output = vec![0; arr.len()];
    for &x in arr.iter().rev() {
        count[x] -= 1;
        output[count[x]] = x;
    }
    output
}

#[cfg(test)]
mod tests {
    use super::*;

    fn is_sorted<T: Ord>(arr: &[T]) -> bool {
        arr.windows(2).all(|w| w[0] <= w[1])
    }

    #[test]
    fn test_insertion_sort() {
        let mut arr = vec![64, 34, 25, 12, 22, 11, 90];
        insertion_sort(&mut arr);
        assert!(is_sorted(&arr));
    }

    #[test]
    fn test_merge_sort() {
        let arr = vec![64, 34, 25, 12, 22, 11, 90];
        let sorted = merge_sort(&arr);
        assert!(is_sorted(&sorted));
        assert_eq!(sorted.len(), arr.len());
    }

    #[test]
    fn test_heap_sort() {
        let mut arr = vec![64, 34, 25, 12, 22, 11, 90];
        heap_sort(&mut arr);
        assert!(is_sorted(&arr));
    }

    #[test]
    fn test_quick_sort() {
        let mut arr = vec![64, 34, 25, 12, 22, 11, 90];
        quick_sort(&mut arr);
        assert!(is_sorted(&arr));
    }

    #[test]
    fn test_counting_sort() {
        let arr = vec![4, 2, 2, 8, 3, 3, 1];
        let sorted = counting_sort(&arr, 8);
        assert!(is_sorted(&sorted));
        assert_eq!(sorted, vec![1, 2, 2, 3, 3, 4, 8]);
    }

    #[test]
    fn test_all_sorts_on_empty_and_single() {
        let mut empty: Vec<i32> = vec![];
        insertion_sort(&mut empty);
        assert!(empty.is_empty());

        let mut single = vec![42];
        heap_sort(&mut single);
        assert_eq!(single, vec![42]);
    }
}
```

## 五、形式化证明

### 5.1 归并排序正确性的 Lean4 草图

```lean4
-- 定义有序性谓词
inductive Sorted {α : Type} [LinearOrder α] : List α → Prop
  | nil : Sorted []
  | single (a : α) : Sorted [a]
  | cons (a b : α) (bs : List α) :
      a ≤ b → Sorted (b :: bs) → Sorted (a :: b :: bs)

-- 定义排列关系（多重集相等）
def IsPermutation (xs ys : List α) : Prop :=
  xs ~ ys  -- Lean 标准库中的排列符号

-- 归并的正确性定理
theorem merge_sorted {α : Type} [LinearOrder α] (xs ys : List α)
    (h1 : Sorted xs) (h2 : Sorted ys) :
    Sorted (merge xs ys) ∧ IsPermutation (merge xs ys) (xs ++ ys) := by
  induction xs, ys using merge.induct with
  | case1 => simp [merge, Sorted.single]
  | case2 => simp [merge, Sorted.single]
  | case3 x xs y ys ih =>
      by_cases h : x ≤ y
      · simp [merge, h, Sorted.cons, ih]
      · simp [merge, h, Sorted.cons, ih]
```

### 5.2 快速排序的期望复杂度证明

**定理 5.1** 随机化快速排序的期望比较次数为 $O(n \log n)$。

*证明框架*：
设 $X$ 为总比较次数。定义指示器随机变量 $X_{ij} = \mathbb{1}\{z_i \text{ 与 } z_j \text{ 被比较}\}$，其中 $z_i, z_j$ 为排序后第 $i, j$ 小的元素。

则 $X = \sum_{i=1}^{n-1}\sum_{j=i+1}^n X_{ij}$。

$z_i$ 与 $z_j$ 被比较当且仅当在 $z_i, z_{i+1}, \ldots, z_j$ 中，$z_i$ 或 $z_j$ 率先被选为枢轴。该事件概率为 $\frac{2}{j-i+1}$。

因此：
$$\begin{aligned}
E[X] &= \sum_{i=1}^{n-1}\sum_{j=i+1}^n \frac{2}{j-i+1} \\
     &= \sum_{i=1}^{n-1}\sum_{k=1}^{n-i} \frac{2}{k+1} \\
     &\leq \sum_{i=1}^{n-1}\sum_{k=1}^{n} \frac{2}{k} \\
     &= 2(n-1)H_n = O(n \log n)
\end{aligned}$$

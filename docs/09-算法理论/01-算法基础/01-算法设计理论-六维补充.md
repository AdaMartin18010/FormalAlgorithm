## 一、规范定义深化

### 1.1 算法的形式化定义

**定义 1.1 (算法)** 算法是一个良定义的计算过程，它将一组输入值映射到一组输出值。形式化地，算法可表示为五元组 $\mathcal{A} = (\Sigma, S, s_0, \delta, H)$，其中：

1. **状态集** $\Sigma$：算法所有可能状态的集合
2 **存储集** $S$：有限或可数无限的存储配置集合
2. **初始状态** $s_0 \in \Sigma$：对应输入实例的初始配置
3. **转移函数** $\delta: \Sigma \times S \to \Sigma \times S$：单步状态转移规则
4. **停机条件** $H \subseteq \Sigma$：接受或拒绝状态集合

**定义 1.2 (算法正确性)** 算法 $\mathcal{A}$ 对问题 $\Pi$ 是正确的，当且仅当：

- **完全性 (Soundness)**：对任意输入 $x \in \Pi$，若 $\mathcal{A}(x)$ 输出"是"，则 $x$ 确实是"是"实例
- **完备性 (Completeness)**：对任意"是"实例 $x \in \Pi$，$\mathcal{A}(x)$ 必输出"是"

### 1.2 算法设计范式的分类体系

算法设计范式可按以下维度组织：

| 范式 | 核心思想 | 代表问题 | 时间复杂度特征 |
|------|----------|----------|----------------|
| **蛮力法** (Brute Force) | 枚举所有候选解 | 子集和问题 | 指数级 $O(2^n)$ |
| **贪心法** (Greedy) | 局部最优选择 | 最小生成树、活动选择 | 多项式级 |
| **分治法** (Divide & Conquer) | 分解-解决-合并 | 归并排序、FFT | $O(n \log n)$ 或类似 |
| **动态规划** (DP) | 重叠子问题 + 最优子结构 | 最短路径、背包 | 多项式级 |
| **回溯法** (Backtracking) | 深度优先 + 剪枝 | N皇后、图着色 | 最坏指数，实际剪枝 |
| **分支限界** (Branch & Bound) | 广度优先/最优优先 + 界 | TSP、整数规划 | 最坏指数，界可剪枝 |
| **随机算法** (Randomized) | 随机选择辅助决策 | 快速排序、素性测试 | 期望多项式 |
| **近似算法** (Approximation) | 牺牲最优性换取效率 | 顶点覆盖、集合覆盖 | 多项式，有近似比 |

## 二、模型设计深化

### 2.1 计算模型层次

算法设计依赖于底层计算模型的假设。常见模型层次如下：

**RAM 模型 (Random Access Machine)**：

- 基本操作（算术、比较、赋值）耗时 $O(1)$
- 内存访问为随机访问
- 现代算法分析的标准模型

**字 RAM 模型 (Word RAM)**：

- 机器字长为 $w = \Theta(\log n)$ 位
- 支持字级并行操作
- 适用于基数排序、哈希表等算法分析

**外部存储模型 (External Memory Model)**：

- 两级存储：快速内存 $M$，慢速磁盘
- 以块 (block) 为单位传输，块大小 $B$
- 分析目标：最小化 I/O 次数

### 2.2 问题归约与设计模式

**定义 2.1 (多项式时间归约)** 问题 $A$ 可多项式时间归约到问题 $B$（记作 $A \leq_p B$），若存在多项式时间可计算函数 $f$，使得对任意实例 $x$：
$$x \in A \iff f(x) \in B$$

归约不仅是复杂度分析工具，也是算法设计方法。通过将新问题归约到已知问题的解法，可快速得到新算法。

**设计模式模板**：

```
蛮力模板:
  生成所有候选解 C
  for each c in C:
    if valid(c): return c
  return "无解"

贪心模板:
  初始化解 S = ∅
  while 未满足终止条件:
    按贪心标准选择当前最优元素 e
    if feasible(S ∪ {e}): S = S ∪ {e}
  return S

分治模板:
  函数 D&C(P):
    if |P| ≤ 基例规模: return 直接求解(P)
    将 P 分解为 {P₁, ..., Pₖ}
    for i = 1 to k: Sᵢ = D&C(Pᵢ)
    return Merge(S₁, ..., Sₖ)
```

## 三、数学符号与推导

### 3.1 算法复杂度分析的形式化

**定义 3.1 (最坏情况时间复杂度)**
$$T_{\text{worst}}(n) = \max_{|x| = n} \text{time}_\mathcal{A}(x)$$

**定义 3.2 (期望时间复杂度，随机算法)**
$$T_{\text{exp}}(n) = \max_{|x| = n} \mathbb{E}[\text{time}_\mathcal{A}(x)]$$

**定义 3.3 (摊还分析)** 对于操作序列 $\sigma = \langle o_1, o_2, \ldots, o_m \rangle$，摊还代价满足：
$$\sum_{i=1}^m \hat{c}_i \geq \sum_{i=1}^m c_i$$
其中 $\hat{c}_i$ 为摊还代价，$c_i$ 为实际代价。常用方法包括：聚集分析、记账方法、势能方法。

**势能方法形式化**：
设 $D_i$ 为第 $i$ 次操作后的数据结构状态，$\Phi(D_i)$ 为势能函数。则：
$$\hat{c}_i = c_i + \Phi(D_i) - \Phi(D_{i-1})$$

若要求对所有 $i$ 有 $\Phi(D_i) \geq \Phi(D_0)$，则总摊还代价给出了总实际代价的上界。

### 3.2 贪心算法正确性证明框架

贪心算法的正确性通常依赖两个核心性质：

**性质 1 (贪心选择性质)**：存在某个最优解包含贪心算法做出的第一个选择。

**性质 2 (最优子结构)**：在做出贪心选择后，剩余子问题仍存在最优解，且该最优解与贪心选择组合构成原问题的最优解。

**定理 3.1** 若问题同时满足贪心选择性质和最优子结构，则贪心算法总能得到最优解。

*证明框架*（归纳法）：

- **基例**：问题规模最小的情况显然成立
- **归纳假设**：假设贪心选择对规模小于 $n$ 的所有实例均能得到最优解
- **归纳步骤**：由贪心选择性质，存在一个最优解包含当前贪心选择；移除该选择后，剩余子问题规模减小，由归纳假设贪心算法对子问题亦最优；由最优子结构，整体解最优

## 四、示例性代码（Rust，含单元测试）

### 4.1 多范式算法设计库

```rust
//! 算法设计范式 — 工程级 Rust 实现
//!
//! 包含：活动选择（贪心）、矩阵链乘（DP）、N皇后（回溯）

/// 活动选择问题 — 贪心算法
///
/// 给定活动集合，每个活动有开始时间和结束时间，选择最大数量的互不冲突活动。
///
/// # 复杂度
/// - 时间：O(n log n)（排序主导）
/// - 空间：O(1) 额外空间
pub fn activity_selection(start: &[usize], finish: &[usize]) -> Vec<usize> {
    let n = start.len();
    if n == 0 {
        return vec![];
    }

    // 创建索引数组并按结束时间排序
    let mut indices: Vec<usize> = (0..n).collect();
    indices.sort_by_key(|&i| finish[i]);

    let mut result = vec![indices[0]];
    let mut last_finish = finish[indices[0]];

    for &i in &indices[1..] {
        if start[i] >= last_finish {
            result.push(i);
            last_finish = finish[i];
        }
    }

    result
}

/// 矩阵链乘法 — 动态规划
///
/// 给定矩阵维度序列 dims，求最优括号化方案的最小乘积次数。
/// 矩阵 A_i 的维度为 dims[i] × dims[i+1]。
///
/// # 复杂度
/// - 时间：O(n³)
/// - 空间：O(n²)
pub fn matrix_chain_order(dims: &[usize]) -> (usize, Vec<Vec<usize>>) {
    let n = dims.len() - 1; // 矩阵数量
    if n == 0 {
        return (0, vec![]);
    }

    let mut dp = vec![vec![0; n]; n];
    let mut split = vec![vec![0; n]; n];

    for chain_len in 2..=n {
        for i in 0..=n - chain_len {
            let j = i + chain_len - 1;
            dp[i][j] = usize::MAX;
            for k in i..j {
                let cost = dp[i][k]
                    .saturating_add(dp[k + 1][j])
                    .saturating_add(dims[i] * dims[k + 1] * dims[j + 1]);
                if cost < dp[i][j] {
                    dp[i][j] = cost;
                    split[i][j] = k;
                }
            }
        }
    }

    (dp[0][n - 1], split)
}

/// N 皇后问题 — 回溯算法
///
/// 在 n×n 棋盘上放置 n 个皇后，使其互不攻击。
/// 返回所有可行解的集合，每个解为列位置向量（第 i 行皇后在第 solution[i] 列）。
///
/// # 复杂度
/// - 时间：最坏 O(n!)，实际剪枝后远小于此
/// - 空间：O(n)（递归栈）
pub fn n_queens(n: usize) -> Vec<Vec<usize>> {
    let mut solutions = Vec::new();
    let mut board = vec![0usize; n];
    backtrack(&mut board, 0, n, &mut solutions);
    solutions
}

fn backtrack(board: &mut [usize], row: usize, n: usize, solutions: &mut Vec<Vec<usize>>) {
    if row == n {
        solutions.push(board.to_vec());
        return;
    }

    for col in 0..n {
        if is_safe(board, row, col) {
            board[row] = col;
            backtrack(board, row + 1, n, solutions);
        }
    }
}

fn is_safe(board: &[usize], row: usize, col: usize) -> bool {
    for i in 0..row {
        if board[i] == col
            || board[i].abs_diff(col) == (row - i)
        {
            return false;
        }
    }
    true
}

/// 快速幂 — 分治思想
///
/// 计算 base^exp mod modulus，使用分治将 O(exp) 降至 O(log exp)。
pub fn fast_pow(mut base: u64, mut exp: u64, modulus: u64) -> u64 {
    if modulus == 1 {
        return 0;
    }
    let mut result = 1;
    base %= modulus;
    while exp > 0 {
        if exp & 1 == 1 {
            result = (result * base) % modulus;
        }
        exp >>= 1;
        base = (base * base) % modulus;
    }
    result
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_activity_selection() {
        let start = vec![1, 3, 0, 5, 8, 5];
        let finish = vec![2, 4, 6, 7, 9, 9];
        let selected = activity_selection(&start, &finish);
        // 期望选择活动 0, 1, 3, 4（索引）
        assert_eq!(selected, vec![0, 1, 3, 4]);
    }

    #[test]
    fn test_matrix_chain_order() {
        // A1: 10×30, A2: 30×5, A3: 5×60
        let dims = vec![10, 30, 5, 60];
        let (cost, _) = matrix_chain_order(&dims);
        assert_eq!(cost, 4500); // (A1×A2)×A3 = 10×30×5 + 10×5×60 = 1500 + 3000 = 4500
    }

    #[test]
    fn test_n_queens_4() {
        let solutions = n_queens(4);
        assert_eq!(solutions.len(), 2);
        for sol in &solutions {
            assert_eq!(sol.len(), 4);
        }
    }

    #[test]
    fn test_n_queens_1() {
        let solutions = n_queens(1);
        assert_eq!(solutions, vec![vec![0]]);
    }

    #[test]
    fn test_fast_pow() {
        assert_eq!(fast_pow(2, 10, 1000), 24);
        assert_eq!(fast_pow(3, 0, 100), 1);
        assert_eq!(fast_pow(5, 3, 13), 8);
        assert_eq!(fast_pow(7, 2, 49), 0);
    }

    #[test]
    fn test_fast_pow_large() {
        assert_eq!(fast_pow(2, 63, u64::MAX), 1u64.wrapping_shl(63));
    }
}
```

## 五、形式化证明（Lean4 草图或严格推导）

### 5.1 活动选择贪心算法的正确性

**定理 5.1** 上述 `activity_selection` 算法返回的活动集合是具有最大基数的不冲突活动子集。

*证明*（贪心选择性质 + 归纳法）：

设活动按结束时间排序为 $a_1, a_2, \ldots, a_n$，满足 $f_1 \leq f_2 \leq \cdots \leq f_n$。

**引理 5.2 (贪心选择性质)**：存在一个最优解包含活动 $a_1$（结束时间最早的活动）。

*证明*：设 $S^*$ 为任意最优解。若 $a_1 \in S^*$，得证。否则，设 $a_k$ 为 $S^*$ 中结束时间最早的活动。由于 $f_1 \leq f_k$，用 $a_1$ 替换 $a_k$ 不会增加冲突（$a_1$ 结束不晚于 $a_k$）。新集合大小不变，仍最优且包含 $a_1$。∎

**归纳证明**：

- **基例**：$n = 1$ 时显然成立
- **归纳假设**：假设算法对所有活动数小于 $n$ 的实例返回最优解
- **归纳步骤**：由引理 5.2，存在包含 $a_1$ 的最优解。移除 $a_1$ 及所有与 $a_1$ 冲突的活动后，剩余活动互不冲突且结束时间均晚于 $f_1$。对剩余子问题应用归纳假设，可知贪心算法对子问题返回最优解。结合 $a_1$，整体最优。

-- 定义矩阵乘法代价
noncomputable def mat_mul_cost (dims : List ℕ) (i j : ℕ) : ℕ :=
  if i >= j then 0
  else List.Ico i j |>.map (fun k =>
    mat_mul_cost dims i k + mat_mul_cost dims (k+1) j +
    dims[i]! * dims[k+1]! * dims[j+1]!
  ) |>.minimum?.getD 0

-- 矩阵链乘满足最优子结构
theorem matrix_chain_optimal_substructure (dims : List ℕ)

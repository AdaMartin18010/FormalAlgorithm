## 一、理论基础 (Theoretical Foundation)

### 1.1 选择问题

**定义 1.1.1** (第 $i$ 顺序统计量) [CLRS2022, Ch.9]

给定一个包含 $n$ 个互异元素的集合，将其按非递减顺序排列后，第 $i$ 个元素称为第 $i$ **顺序统计量**（$i$-th Order Statistic）。特别地：

- $i = 1$: 最小值（minimum）
- $i = n$: 最大值（maximum）
- $i = \lfloor (n+1)/2 \rfloor$: 中位数（median）

### 1.2 选择问题的形式化定义

**定义 1.2.1** (选择问题)

**输入**: 一个包含 $n$ 个（可比较）元素的数组 $A$ 和一个整数 $i$，其中 $1 \leq i \leq n$。

**输出**: 数组 $A$ 中恰好大于 $i-1$ 个其他元素的那个元素。

## 二、算法设计 (Algorithm Design)

### 2.1 QuickSelect 算法

QuickSelect 由 Tony Hoare 于 1961 年提出，是快速排序的分区思想在选择问题上的应用。

```
算法: RANDOMIZED-SELECT(A, p, r, i)
1. if p == r
2.     return A[p]
3. q = RANDOMIZED-PARTITION(A, p, r)
4. k = q - p + 1
5. if i == k
6.     return A[q]
7. else if i < k
8.     return RANDOMIZED-SELECT(A, p, q-1, i)
9. else
10.    return RANDOMIZED-SELECT(A, q+1, r, i-k)
```

### 2.2 Median of Medians 算法

Median of Medians（中位数的中位数）算法由 Blum、Floyd、Pratt、Rivest 和 Tarjan 于 1973 年提出，是第一个确定性线性时间选择算法。

```
算法: SELECT(A, n, i)
1. 将 A 中元素划分为 ⌈n/5⌉ 组，每组 5 个元素
2. 找出每组的中位数，构成集合 M
3. 递归调用 SELECT 找出 M 的中位数 x
4. 用 x 作为 pivot 对 A 进行分区，得到分区点 q
5. 设 k = q 左侧元素个数 + 1
6. if i == k, return x
7. else if i < k, 在左侧递归 SELECT
8. else 在右侧递归 SELECT(i - k)
```

### 2.3 算法设计对比

| 特性 | QuickSelect | Median of Medians |
|------|-------------|-------------------|
| 主元选择 | 随机 | 中位数的中位数 |
| 期望时间 | $O(n)$ | $O(n)$ |
| 最坏时间 | $O(n^2)$ | $O(n)$ |
| 常数因子 | 小 | 较大 |
| 实用性 | 广泛用于实际系统 | 主要用于理论证明 |

## 三、复杂度分析 (Complexity Analysis)

### 3.1 QuickSelect 期望复杂度

**定理 3.1.1** (QuickSelect 期望时间复杂度) [CLRS2022, Theorem 9.2]

在假设分区点是均匀随机的情况下，QuickSelect 的期望时间复杂度为 $O(n)$。

**证明概要**:

设 $T(n)$ 为处理规模为 $n$ 的问题时的期望运行时间。由于只递归处理一侧：

$$T(n) \leq \frac{1}{n}\sum_{k=1}^{n} T(\max(k-1, n-k)) + O(n)$$

注意到 $\max(k-1, n-k)$ 在 $k > n/2$ 时为 $k-1$，否则为 $n-k$。通过对称性：

$$T(n) \leq \frac{2}{n}\sum_{k=\lfloor n/2 \rfloor}^{n-1} T(k) + O(n)$$

用替换法可证 $T(n) = O(n)$。$\square$

### 3.2 Median of Medians 最坏复杂度

**定理 3.2.1** (Median of Medians 最坏时间复杂度) [CLRS2022, Theorem 9.3]

Median of Medians 算法在最坏情况下的时间复杂度为 $O(n)$。

**证明概要**:

关键引理：以 $x$（中位数的中位数）作为 pivot，至少可以保证 $3n/10 - 6$ 个元素小于 $x$，同时至少 $3n/10 - 6$ 个元素大于 $x$。

这意味着每次递归的问题规模最多为 $7n/10 + 6$。

递归式为：

$$T(n) \leq T(\lceil n/5 \rceil) + T(7n/10 + 6) + O(n)$$

用替换法可证 $T(n) = O(n)$。$\square$

### 3.3 下界定理

**定理 3.3.1** (选择问题下界)

任何基于比较的算法，在最坏情况下找到中位数至少需要 $\Omega(n)$ 次比较。

**证明概要**: 每个元素至少必须参与一次比较，否则无法确定其是否为中位数。$\square$

**定理 3.3.2** (精确下界)

找到中位数的比较次数下界为 $2n - o(n)$ [Dor & Zwick 1999]，上界约为 $2.95n$ [Dor & Zwick 2001]。

## 四、形式化验证 (Formal Verification)

### 4.1 QuickSelect 正确性

**定理 4.1.1** (QuickSelect 正确性)

`RANDOMIZED-SELECT(A, p, r, i)` 返回数组 $A[p..r]$ 中第 $i$ 小的元素。

### 4.2 归纳证明

**证明** (数学归纳法):

**基础情况**: $p = r$ 时，子数组只有一个元素，它必然是第 1 小的元素，返回正确。

**归纳步骤**: 设分区点 $q$ 将数组分为 $A[p..q-1]$ 和 $A[q+1..r]$，$k = q - p + 1$。

- 若 $i = k$，则 $A[q]$ 就是第 $i$ 小元素，正确
- 若 $i < k$，则第 $i$ 小元素在左半部分，递归调用规模更小，由归纳假设正确
- 若 $i > k$，则第 $i$ 小元素在右半部分，其相对排名为 $i - k$，由归纳假设正确

**终止性**: 每次递归至少排除 pivot 本身，子问题规模严格减小，因此必然终止。$\square$

### 4.3 Median of Medians 正确性

Median of Medians 的正确性同样可以通过归纳法证明，关键在于 pivot $x$ 保证了对两侧子问题规模的限制，确保递归始终向更小的子问题推进。

## 五、应用场景 (Application Scenarios)

### 5.1 典型应用

| 应用 | 算法选择 | 说明 |
|------|----------|------|
| 找中位数 | QuickSelect | 最简单直接的实现 |
| 找 Top-k | QuickSelect / 堆 | $O(n)$ 期望 vs $O(n \log k)$ 确定 |
| 分位数计算 | QuickSelect | 统计学中的四分位数、百分位数 |
| 异常检测 | Median of Medians | 需要最坏情况保证 |
| 图像处理 | QuickSelect | 中值滤波的核心操作 |

### 5.2 工程实现建议

- 对于**大多数实际应用**，随机化 QuickSelect 足够高效
- 对于**安全性关键系统**（如需要防御 DoS 攻击的在线服务），应考虑使用 Median of Medians 或其变种
- 对于**找 Top-k 且 $k$ 很小**的情况，使用大小为 $k$ 的最大堆可能更实用

## 六、扩展变体 (Extensions and Variants)

### 6.1 IntroSelect

**IntroSelect** 是 C++ `std::nth_element` 的实现策略：

- 首先使用 QuickSelect
- 若递归深度超过阈值，则切换到 Median of Medians
- 结合了 QuickSelect 的平均快速性和 Median of Medians 的最坏情况保证

### 6.2 加权中位数

**定义 6.2.1** (加权中位数)

给定元素 $x_i$ 和权重 $w_i$，加权中位数是满足 $\sum_{x_i < x} w_i \leq 1/2$ 且 $\sum_{x_i > x} w_i \leq 1/2$ 的元素 $x$。

加权中位数在设施选址问题（如一维 k-中位数）中有重要应用。

### 6.3 流式选择

对于数据流场景（只能扫描一次，内存有限），可以使用**蓄水池采样**（Reservoir Sampling）或**Munro-Paterson 算法**来近似选择顺序统计量。

### 6.4 并行选择

在 PRAM 模型中，选择问题可以在 $O(\log n)$ 并行时间和 $O(n)$ 工作量内解决，通过并行Median of Medians 和并行分区实现。

## 1. 引言

数据流算法处理大规模、高速到达的数据，只能在有限内存中维护摘要信息 [Muthukrishnan 2005]。典型应用包括网络监控、日志分析、实时推荐等。

**数据流模型**：

- 输入：元素序列 $a_1, a_2, \cdots, a_m$，每个 $a_i \in [n]$
- 约束：内存 $o(\min(m, n))$，通常 $O(\log n)$ 或 $O(\sqrt{n})$
- 目标：计算统计量或回答查询

## 2. 频繁项检测

### 2.1 Misra-Gries算法

**问题**：找出现频率超过 $\phi$ 的元素。

**算法**：

- 维护 $k = 1/\phi$ 个候选及其计数
- 新元素：若候选未满，加入；否则所有计数减1

**定理 2.1**：Misra-Gries报告的元素频率至少为 $f - \phi m$，其中 $f$ 是真实频率。

### 2.2 Count-Min Sketch

**数据结构** [Cormode 2005]：

- $w \times d$ 的计数器数组
- $d$ 个独立哈希函数 $h_1, \cdots, h_d: [n] \to [w]$

**更新**：对每个哈希函数，相应计数器加1
**查询**：取所有哈希位置的最小值

```rust
/// Count-Min Sketch
pub struct CountMinSketch {
    width: usize,
    depth: usize,
    counters: Vec<Vec<u64>>,
    hashes: Vec<Box<dyn Fn(usize) -> usize>>,
}

impl CountMinSketch {
    pub fn update(&mut self, item: usize, count: u64) {
        for i in 0..self.depth {
            let j = (self.hashes[i])(item) % self.width;
            self.counters[i][j] += count;
        }
    }

    pub fn query(&self, item: usize) -> u64 {
        let mut min_count = u64::MAX;
        for i in 0..self.depth {
            let j = (self.hashes[i])(item) % self.width;
            min_count = min_count.min(self.counters[i][j]);
        }
        min_count
    }
}
```

**定理 2.2**：设 $\epsilon = 2/w$，$\delta = 1/2^d$，则以概率 $1-\delta$：
$$f_i \leq \hat{f}_i \leq f_i + \epsilon m$$

**空间复杂度**：$O(\frac{1}{\epsilon} \ln \frac{1}{\delta})$

## 3. 基数估计

### 3.1 HyperLogLog

**问题**：估计不同元素的个数（基数）。

**算法思想** [Flajolet 2007]：

- 观察哈希值的二进制前缀
- 前缀中前导零的最大数量与基数对数相关

**算法**：

1. 将元素哈希为二进制串
2. 根据前缀分桶
3. 每桶记录最大前导零数 $R$
4. 基数估计：$\alpha_m \cdot m^2 \cdot (\sum_{j=1}^m 2^{-R_j})^{-1}$

**定理 3.1**：HyperLogLog使用 $O(\epsilon^{-2} \log\log n + \log n)$ 比特空间，达到标准误差 $1.04/\sqrt{m}$。

## 4. 矩估计

### 4.1 AMS Sketch

**问题**：估计 $F_k = \sum_i f_i^k$（第 $k$ 阶矩）。

**特别地**：$F_2$（自连接大小）用于数据库查询优化。

**算法** [Alon 1996]：

- 使用4-独立哈希函数 $h: [n] \to \{-1, +1\}$
- 维护 $Z = \sum_i f_i \cdot h(i)$
- 估计：$\hat{F}_2 = Z^2$

**定理 4.1**：$E[\hat{F}_2] = F_2$，$Var[\hat{F}_2] \leq 2F_2^2$。

## 5. 采样技术

### 5.1 水库采样

**问题**：从流中均匀随机采样 $k$ 个元素。

**算法**：

- 前 $k$ 个元素直接存入水库
- 对于第 $i$ 个元素 ($i > k$)：以概率 $k/i$ 替换水库中随机元素

**定理 5.1**：水库采样保证每个元素最终在水库中的概率相等，为 $k/n$。

### 5.2 滑动窗口

**问题**：统计最近 $W$ 个元素中的频繁项。

**算法**：

- 指数直方图（Exponential Histogram）
- 维护桶的层次结构
- 近似计数，误差可控

## 6. 复杂度对比

| 问题 | 算法 | 空间 | 误差 | 备注 |
|------|------|------|------|------|
| 频繁项 | Count-Min | $O(\frac{1}{\epsilon}\ln\frac{1}{\delta})$ | $\epsilon m$ | 高估 |
| 基数 | HyperLogLog | $O(\epsilon^{-2}\log\log n)$ | $1.04/\sqrt{m}$ | 概率 |
| 二阶矩 | AMS | $O(\frac{1}{\epsilon^2}\ln\frac{1}{\delta})$ | $\epsilon F_2$ | 概率 |
| 采样 | 水库 | $O(k)$ | 精确 | 均匀 |

## 7. 应用案例

### 7.1 网络流量分析

**场景**：路由器统计流量中目的IP的分布。
**算法**：Count-Min Sketch。
**效果**：实时检测DDoS攻击。

### 7.2 数据库查询优化

**场景**：估计连接操作的结果大小。
**算法**：AMS Sketch估计$F_2$。
**效果**：选择最优查询计划。

### 7.3 实时推荐

**场景**：统计用户最近的行为偏好。
**算法**：滑动窗口频繁项。
**效果**：动态调整推荐。

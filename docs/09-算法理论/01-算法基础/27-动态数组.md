## 一、概念定义 (Concept Definition)

### 1.1 动态数组

**定义 1.1.1** (形式化)

**动态数组**是维护一个连续内存块的数据结构，支持在尾部进行均摊 O(1) 的插入与删除，并自动调整容量以适应元素数量的变化。

形式化地，动态数组 $A$ 可表示为三元组 $(data, n, cap)$，其中：

- $data$：底层连续存储区
- $n$：当前有效元素个数
- $cap$：当前总容量

### 1.2 扩容策略 (Expansion Policy)

当 $n = cap$ 时触发**扩容**：

$$cap' = \max(1, r \cdot cap)$$

其中 $r > 1$ 为增长因子。常见取值为：

| 增长因子 | 实现 | 特点 |
|---------|------|------|
| $r = 2$ | C++ `std::vector` (传统) | 实现简单，均摊分析直观 |
| $r = 1.5$ | Python `list` | 空间利用率更高 |
| $r = 1.125$ | 某些 JVM 实现 | 减少内存峰值 |

**扩容代价**：分配新内存 + 复制全部 $n$ 个元素 = $\Theta(n)$

### 1.3 缩容策略 (Contraction Policy)

为避免频繁扩缩容震荡，缩容阈值应**严格低于**扩容阈值。常见策略：

$$\text{当 } n \leq \frac{cap}{4} \text{ 时，缩容至 } cap' = \max(1, \frac{cap}{2})$$

**引理 1.3.1**：使用 $1/4$ 缩容阈值可保证连续两次 resize 之间至少经历 $\Theta(cap)$ 次操作。

## 二、属性 (Properties)

### 2.1 操作复杂度

| 操作 | 最坏情况 | 均摊情况 | 空间复杂度 |
|------|---------|---------|-----------|
| push_back | $\Theta(n)$ | $\Theta(1)$ | $O(1)$ |
| pop_back | $\Theta(n)$ | $\Theta(1)$ | $O(1)$ |
| get/set | $\Theta(1)$ | $\Theta(1)$ | $O(1)$ |
| insert(at) | $\Theta(n)$ | $\Theta(n)$ | $O(1)$ |
| remove(at) | $\Theta(n)$ | $\Theta(n)$ | $O(1)$ |

### 2.2 关键不变式

**容量不变式**：对于使用 $r=2$ 扩缩容策略的动态数组，始终满足：

$$\frac{cap}{4} \leq n \leq cap \quad (\text{当 } n > 0)$$

**内存浪费上界**：未使用空间比例 $< 75\%$

## 四、解释 (Explanation)

### 4.1 动机与直观

**为什么动态数组的 push_back 是均摊 O(1)？**

想象你在买电影票：大多数时候直接入座（$O(1)$），但偶尔影院满座时需要整体搬迁到更大的影厅（$O(n)$）。由于每次搬迁后容量翻倍，搬迁的间隔也越来越长。把搬迁的巨大成本均摊到之前的多次快速入座上，每次平均成本仍是常数。

### 4.2 与摊还分析的联系

动态数组是理解**摊还分析**最经典的教学案例。三种分析方法均适用：

- **聚合分析**：$n$ 次操作总成本 $< 3n$
- **势函数法**：$\Phi = 2n - cap$
- **记账方法**：为每个元素预存 "搬迁基金"

## 五、论证 (Argumentation)

### 5.1 势函数法证明 push_back 均摊 O(1)

**定理 5.1.1**：使用 $r=2$ 扩容策略时，push_back 的均摊时间复杂度为 $O(1)$。

**证明** (势函数法)：

定义势函数：
$$\Phi(D_i) = 2n_i - cap_i$$

其中 $n_i$ 为第 $i$ 次操作后的元素个数，$cap_i$ 为容量。

**情况 1**：无需扩容

- 实际成本 $c_i = 1$
- $n_i = n_{i-1} + 1$，$cap_i = cap_{i-1}$
- 势变化：$\Delta\Phi = 2(n_{i-1}+1) - cap_{i-1} - (2n_{i-1} - cap_{i-1}) = 2$
- 均摊成本：$\hat{c}_i = 1 + 2 = 3$

**情况 2**：需要扩容（$n_{i-1} = cap_{i-1}$）

- 实际成本 $c_i = n_{i-1} + 1$（复制 $n_{i-1}$ 个元素 + 插入 1 个）
- $cap_i = 2 \cdot cap_{i-1} = 2n_{i-1}$
- 势变化：$\Delta\Phi = 2(n_{i-1}+1) - 2n_{i-1} - (2n_{i-1} - n_{i-1}) = 2 - n_{i-1}$
- 均摊成本：$\hat{c}_i = (n_{i-1} + 1) + (2 - n_{i-1}) = 3$

综上，每次 push_back 的均摊成本 $\leq 3 = O(1)$。$\square$

### 5.2 缩容策略的震荡避免

**反例 5.2.1**：若缩容阈值设为 $cap/2$（与扩容对称）

考虑交替执行 push_back 到 $n=cap$ 然后 pop_back 到 $n=cap/2$：

- 每次 push_back 到满 → 扩容 $\Theta(cap)$
- 每次 pop_back 到半 → 缩容 $\Theta(cap)$
- 连续 $m$ 次操作总成本 $\Theta(m \cdot cap)$

这导致均摊成本退化到 $\Theta(cap)$。因此缩容阈值必须**严格小于** $cap/2$。

## 六、形式证明 (Formal Proof)

### 6.1 n 次 push_back 的聚合分析

**定理 6.1.1**：对空动态数组执行 $n$ 次 push_back，总时间复杂度为 $O(n)$。

**证明**：

设扩容发生在元素个数为 $1, 2, 4, \ldots, 2^k$ 时，其中 $2^k \leq n < 2^{k+1}$。

总成本：
$$T(n) = \underbrace{n}_{\text{每次插入}} + \underbrace{\sum_{j=0}^{k} 2^j}_{\text{扩容复制}}$$

$$= n + (2^{k+1} - 1) < n + 2n = 3n = O(n)$$

因此单次均摊成本为 $O(1)$。$\square$

### 6.2 包含 pop_back 的均摊分析

**定理 6.2.1**：使用 $1/4$ 缩容阈值时，任意 $n$ 次 push_back/pop_back 序列的均摊时间复杂度为 $O(1)$。

**证明**：

扩展势函数：
$$\Phi = \begin{cases}
2n - cap & \text{if } \alpha \geq 1/2 \\
cap/2 - n & \text{if } \alpha < 1/2
\end{cases}$$

其中 $\alpha = n/cap$ 为负载因子。

可以验证（CLRS 习题 17.4-3）：
- 当 $\alpha \geq 1/2$ 时，push_back 均摊成本 $\leq 3$
- 当 $\alpha < 1/2$ 时，pop_back 均摊成本 $\leq 2$
- 跨越阈值的操作均摊成本仍为 $O(1)$

因此任意操作序列的均摊成本为 $O(1)$。$\square$

## 七、应用场景

| 应用场景 | 使用原因 | 代表实现 |
|---------|---------|---------|
| 通用顺序容器 | 缓存友好，随机访问快 | `std::vector`, Python `list` |
| 动态字符串 | 字符追加频繁 | `std::string`, Java `StringBuilder` |
| 缓冲区/流 | 数据量不确定 | I/O buffer, network packet buffer |
| 栈/队列底层 | 利用连续内存优势 | `VecDeque` in Rust |

## 八、扩展变体

### 8.1  ropes / gap buffers
针对频繁中间插入的文本编辑器场景，使用平衡树或间隙缓冲区替代纯动态数组。

### 8.2 变长数组 (VLA)
C99 引入的栈上变长数组，无自动扩容能力，与动态数组有本质区别。

### 8.3 分块数组 (Unrolled Linked List)
结合链表与数组优点：每个节点存储一个小数组，减少指针开销同时保持插入灵活性。

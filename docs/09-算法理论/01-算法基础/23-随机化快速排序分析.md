# 随机化快速排序分析


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

> 对标：CLRS 第4版 Chapter 7.3

## 一、理论基础

快速排序（Quicksort）的平均时间复杂度为 $O(n \log n)$，但最坏情况为 $O(n^2)$（当输入已排序且总是选第一个/最后一个作为 pivot 时）。

**随机化快速排序（Randomized Quicksort）**通过**随机选择 pivot**来消除对输入分布的依赖：无论对手如何构造输入，期望运行时间始终为 $O(n \log n)$。

### 随机化策略

1. **随机采样 pivot**：在待排序区间 $A[p..r]$ 中，均匀随机选取一个元素作为 pivot；
2. 执行标准 partition；
3. 递归排序左右子区间。

由于 pivot 的选择与输入无关，任何特定元素成为 pivot 的概率均等，从而避免了系统性的最坏情况。

## 二、算法设计

### 标准 partition（Lomuto / Hoare）

以随机 pivot 为例：

```text
randomized_partition(A, p, r):
    i = random(p, r)
    swap A[r] with A[i]
    return partition(A, p, r)
```

### 随机化快速排序递归框架

```text
randomized_quicksort(A, p, r):
    if p < r:
        q = randomized_partition(A, p, r)
        randomized_quicksort(A, p, q-1)
        randomized_quicksort(A, q+1, r)
```

### 三路划分（Dutch National Flag）

当数组中存在大量重复元素时，可采用三路划分将数组分为 `< pivot`、`== pivot`、`> pivot` 三部分，避免重复元素导致的二次退化。

## 三、复杂度分析

### 期望比较次数

设 $X$ 为算法执行的总比较次数。对任意两个元素 $z_i, z_j$（设 $z_i$ 为排序后第 $i$ 小的元素），定义指示器随机变量：

$$
X_{ij} = \mathbb{I}\{z_i \text{ 与 } z_j \text{ 在某次 partition 中被比较}\}
$$

则：

$$
X = \sum_{i=1}^{n-1} \sum_{j=i+1}^{n} X_{ij}
$$

$z_i$ 与 $z_j$ 被比较，当且仅当在区间 $Z_{ij} = \{z_i, z_{i+1}, \dots, z_j\}$ 中，**第一个被选为 pivot 的元素是 $z_i$ 或 $z_j$**。由于 pivot 均匀随机：

$$
\Pr(X_{ij} = 1) = \frac{2}{j - i + 1}
$$

因此期望比较次数为：

$$
\mathbb{E}[X] = \sum_{i=1}^{n-1} \sum_{j=i+1}^{n} \frac{2}{j - i + 1}
$$

令 $k = j - i$，则：

$$
\mathbb{E}[X] = \sum_{i=1}^{n-1} \sum_{k=1}^{n-i} \frac{2}{k+1} < \sum_{i=1}^{n-1} \sum_{k=1}^{n} \frac{2}{k} = 2n H_n = O(n \log n)
$$

### 时间复杂度总结

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最坏情况 | $O(n^2)$ | 极罕见（需要每次随机都选到最差 pivot） |
| 期望情况 | **$O(n \log n)$** | 对任意输入成立 |
| 最好情况 | $O(n \log n)$ | 每次 pivot 都接近中位数 |

### 空间复杂度

- 递归栈深度期望为 $O(\log n)$；
- 最坏情况下递归栈深度为 $O(n)$（概率极低）。

## 四、形式化验证

### 引理 7.1（比较概率）

在随机化快速排序中，排序后第 $i$ 小与第 $j$ 小的元素被比较的概率为 $2/(j-i+1)$。

**证明**：$z_i$ 和 $z_j$ 被比较当且仅当在集合 $Z_{ij}$ 中第一个被选为 pivot 的元素恰好是 $z_i$ 或 $z_j$。由于 pivot 选择是均匀的，该概率为 $2 / |Z_{ij}| = 2/(j-i+1)$。

### 定理 7.2（随机化快速排序期望时间）

随机化快速排序的期望运行时间为 $O(n \log n)$。

**证明**：由引理 7.1 和期望的线性性，总期望比较次数为 $O(n \log n)$。由于 partition 的额外开销为 $O(n)$ 且每层递归的总 partition 工作为 $O(n)$，总期望时间为 $O(n \log n)$。

### 高概率界

利用 Chernoff 界或鞅方法可以进一步证明：随机化快速排序的运行时间以**高概率**（w.h.p.）落在 $O(n \log n)$ 内。

## 五、应用场景

1. **通用排序库**：C++ `std::sort`（Introsort）、Java `Arrays.sort`（Dual-Pivot Quicksort）均以快速排序为核心。
2. **大数据排序**：Hadoop MapReduce、Spark 等分布式框架中的 Shuffle 排序阶段广泛使用快速排序变体。
3. **选择算法（Quickselect）**：基于快速排序 partition 思想，可在期望 $O(n)$ 内找到第 $k$ 小元素。
4. **字符串排序**：三向字符串快速排序（Three-way String Quicksort）在字符串集合上表现优异。

---

## 六、扩展变体

- **Introsort**：结合快速排序、堆排序和插入排序。当递归深度超过阈值时切换到堆排序，保证最坏情况 $O(n \log n)$。
- **Dual-Pivot Quicksort**：使用两个 pivot 将数组分为三部分，Java 7+ 的默认排序算法。
- **Median-of-Medians**：确定性选择中位数作为 pivot，保证最坏 $O(n \log n)$，但常数因子较大，工程中少用。
- **并行快速排序**：对左右子数组 `spawn` 并行递归，是共享内存多核系统中的高效排序实现。

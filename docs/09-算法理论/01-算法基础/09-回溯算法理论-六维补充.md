## 一、规范定义深化

### 1.1 回溯的形式化规约

**定义 1.1 (状态空间)** 组合优化问题的状态空间是一个有根树 $T = (V, E, r)$，其中：
- **根节点 $r$**: 空解或初始状态
- **内部节点**: 部分解（partial solution）
- **叶节点**: 完整解（complete solution）或死胡同（dead end）

**定义 1.2 (回溯算法)** 回溯是一种系统性地遍历状态空间树的深度优先搜索策略，其形式化为：

$$
\text{Backtrack}(v) =
\begin{cases}
\text{输出解} & \text{if } v \text{ 是可行叶节点} \\
\text{剪枝} & \text{if } v \text{ 不可行} \\
\bigcup_{u \in \text{Children}(v)} \text{Backtrack}(u) & \text{otherwise}
\end{cases}
$$

**剪枝函数 (Pruning Function)**: $P: V \to \{\text{继续}, \text{剪枝}\}$，用于在节点 $v$ 处判断其子树是否包含可行解。

### 1.2 适用问题特征

回溯适用于满足以下特征的问题：

| 特征 | 说明 | 典型示例 |
|------|------|----------|
| **解向量为元组** | 解可表示为 $(x_1, x_2, \ldots, x_n)$ | N皇后、子集和 |
| **约束可局部检验** | 部分解的违规可被早期发现 | 图着色、数独 |
| **解空间为笛卡尔积子集** | $X_1 \times X_2 \times \cdots \times X_n$ 的子集 | 哈密顿回路、旅行商 |

## 二、模型设计深化

### 2.1 状态空间树的复杂度模型

对于解向量长度为 $n$、每个分量有 $m$ 种选择的问题：

- **完整状态空间大小**: $m^n$（最坏情况需遍历所有节点）
- **有效剪枝后**: 取决于剪枝函数 $P$ 的强度
- **理想情况**（强剪枝）: $O(n \cdot \text{poly}(m))$

**定理 2.1 (回溯时间复杂度上界)** 若剪枝函数 $P$ 在每个节点以概率 $p$ 剪枝，则期望访问节点数为：
$$E[N] = \sum_{i=0}^n m^i (1-p)^i = \frac{(m(1-p))^{n+1} - 1}{m(1-p) - 1}$$

当 $m(1-p) < 1$ 时，$E[N] = O(1)$（与 $n$ 无关的常数）。

### 2.2 与动态规划的对比

| 维度 | 回溯 | 动态规划 |
|------|------|----------|
| **子问题重叠** | 不利用 | 核心利用 |
| **最优子结构** | 不必须 | 必须 |
| **解空间遍历** | 深度优先 | 自底向上填表 |
| **适用问题** | 计数、判定、枚举 | 最优化 |
| **典型复杂度** | 指数（最坏）| 多项式 |

## 三、数学符号与推导

### 3.1 N皇后问题的解空间分析

**问题**: 在 $n \times n$ 棋盘上放置 $n$ 个皇后，使其互不攻击。

**状态空间树**: 第 $i$ 层决定第 $i$ 行皇后的列位置， branching factor 为 $n$。

**剪枝策略**: 
- 列冲突: $\forall j < i: x_j \neq x_i$
- 主对角线冲突: $\forall j < i: x_j - j \neq x_i - i$
- 副对角线冲突: $\forall j < i: x_j + j \neq x_i + i$

**解的数量**: 记 $Q(n)$ 为 $n$ 皇后问题的解数。已知 $Q(8) = 92$，$Q(n)$ 无闭式公式，但满足渐近估计 [Bell2009]：
$$Q(n) \sim n! / e^n$$

### 3.2 子集和问题的回溯分析

**问题**: 给定集合 $S = \{s_1, \ldots, s_n\}$ 和目标 $t$，判定是否存在子集和为 $t$。

**回溯算法**: 对每个元素选择"包含"或"不包含"，并维护当前和 $sum$ 与剩余元素和 $remaining$。

**剪枝条件**:
- 若 $sum > t$，剪枝（假设所有 $s_i > 0$）
- 若 $sum + remaining < t$，剪枝

**复杂度**: 最坏 $O(2^n)$，但强剪枝后在实践中远小于此。

## 四、示例性代码

### 4.1 工程级 Rust 实现

```rust
//! 回溯算法库 — 工程级实现
//!
//! 包含：N皇后、子集和、全排列、组合

/// N皇后问题求解
///
/// # 返回值
/// 所有互不攻击的皇后布局，每个布局为列位置向量
///
/// # 复杂度
/// - 时间：O(n!)（最坏，无剪枝）
/// - 空间：O(n)（递归栈）
pub fn solve_n_queens(n: usize) -> Vec<Vec<usize>> {
    let mut solutions = Vec::new();
    let mut board = vec![0; n];
    backtrack_n_queens(&mut board, 0, n, &mut solutions);
    solutions
}

fn backtrack_n_queens(board: &mut Vec<usize>, row: usize, n: usize, solutions: &mut Vec<Vec<usize>>) {
    if row == n {
        solutions.push(board.clone());
        return;
    }
    for col in 0..n {
        if is_safe(board, row, col) {
            board[row] = col;
            backtrack_n_queens(board, row + 1, n, solutions);
            // 回溯：自动通过覆盖 board[row] 实现
        }
    }
}

fn is_safe(board: &[usize], row: usize, col: usize) -> bool {
    for i in 0..row {
        if board[i] == col
            || board[i] as isize - i as isize == col as isize - row as isize
            || board[i] + i == col + row
        {
            return false;
        }
    }
    true
}

/// 子集和问题（判定版本）
///
/// 判定是否存在子集和等于 target
pub fn subset_sum_exists(nums: &[i32], target: i32) -> bool {
    let total: i32 = nums.iter().sum();
    backtrack_subset_sum(nums, 0, target, total)
}

fn backtrack_subset_sum(nums: &[i32], idx: usize, target: i32, remaining: i32) -> bool {
    if target == 0 {
        return true;
    }
    if idx >= nums.len() || target < 0 || target > remaining {
        return false;
    }
    // 包含 nums[idx]
    if backtrack_subset_sum(nums, idx + 1, target - nums[idx], remaining - nums[idx]) {
        return true;
    }
    // 不包含 nums[idx]
    backtrack_subset_sum(nums, idx + 1, target, remaining - nums[idx])
}

/// 全排列生成
pub fn permutations<T: Clone>(nums: &[T]) -> Vec<Vec<T>> {
    let mut result = Vec::new();
    let mut current = nums.to_vec();
    backtrack_permute(&mut current, 0, &mut result);
    result
}

fn backtrack_permute<T: Clone>(arr: &mut [T], start: usize, result: &mut Vec<Vec<T>>) {
    if start == arr.len() {
        result.push(arr.to_vec());
        return;
    }
    for i in start..arr.len() {
        arr.swap(start, i);
        backtrack_permute(arr, start + 1, result);
        arr.swap(start, i); // 回溯
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_n_queens_4() {
        let sols = solve_n_queens(4);
        assert_eq!(sols.len(), 2);
    }

    #[test]
    fn test_n_queens_8() {
        let sols = solve_n_queens(8);
        assert_eq!(sols.len(), 92);
    }

    #[test]
    fn test_subset_sum() {
        let nums = vec![3, 34, 4, 12, 5, 2];
        assert!(subset_sum_exists(&nums, 9));
        assert!(!subset_sum_exists(&nums, 30));
    }

    #[test]
    fn test_permutations() {
        let nums = vec![1, 2, 3];
        let perms = permutations(&nums);
        assert_eq!(perms.len(), 6);
    }
}
```

## 五、形式化证明

### 5.1 N皇后回溯算法的正确性

**定理 5.1** `solve_n_queens(n)` 返回所有且仅所有有效的 $n$ 皇后布局。

*证明*（双重归纳法）：

**完备性**（所有返回的布局都有效）：
- 基例: $row = n$ 时，算法通过 `is_safe` 检查了所有行对，保证无冲突。
- 归纳: 假设前 $row$ 行放置有效。第 $row+1$ 行仅在 `is_safe` 通过时放置，保持无冲突。

**可靠性**（所有有效布局都被返回）：
- 对任意有效布局 $(c_1, \ldots, c_n)$，在第 $row$ 层递归中，算法会尝试 $c_{row}$ 列。
- 由于前 $row-1$ 行与有效布局一致，`is_safe(board, row, c_row)` 必然通过。
- 由归纳，算法会沿着该路径到达叶节点并记录布局。

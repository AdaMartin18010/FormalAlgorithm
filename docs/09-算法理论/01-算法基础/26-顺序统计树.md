# 顺序统计树（Order-Statistic Tree）


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

> 对标：CLRS 第4版 Chapter 14

## 一、理论基础

顺序统计树是**红黑树的扩展（Augmented Red-Black Tree）**，通过在节点中维护**子树大小（size）**信息，支持以下操作：

- `select(i)`：查找第 $i$ 小的元素（顺序统计量），$O(\log n)$；
- `rank(x)`：查找元素 $x$ 的排名（即在排序序列中的索引），$O(\log n)$。

顺序统计树是普通动态集合在需要**按秩查询**场景下的自然扩展。

### 与静态顺序统计的区别

- **静态数组**：排序后用 $O(1)$ 查询第 $i$ 小，但插入/删除为 $O(n)$；
- **顺序统计树**：动态插入/删除仍为 $O(\log n)$，同时支持按秩查询。

---

## 二、算法设计

### 节点扩展

在红黑树节点中新增字段：

```text
size = 1 + size(left) + size(right)
```

### 1. 选择操作（select）

```text
select(node, i):
    r = size(node.left) + 1
    if i == r:
        return node
    else if i < r:
        return select(node.left, i)
    else:
        return select(node.right, i - r)
```

### 2. 排名操作（rank）

```text
rank(node, x):
    r = size(node.left) + 1
    if x == node.key:
        return r
    else if x < node.key:
        return rank(node.left, x)
    else:
        return r + rank(node.right, x)
```

### 3. 维护 size 信息

在插入、删除、旋转后，需要更新受影响节点的 `size`：

```text
update_size(node):
    node.size = 1 + size(node.left) + size(node.right)
```

由于红黑树的旋转仅涉及常数个节点，`update_size` 的额外开销为 $O(1)$，不影响整体 $O(\log n)$ 的时间复杂度。

---

## 三、复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 插入 | $O(\log n)$ | 含 size 维护 |
| 删除 | $O(\log n)$ | 含 size 维护 |
| 查找 | $O(\log n)$ | 标准 BST 查找 |
| `select(i)` | $O(\log n)$ | 第 $i$ 顺序统计量 |
| `rank(x)` | $O(\log n)$ | 元素排名 |

## 四、形式化验证

### 不变式

对每个节点 $x$，其 `size` 等于以 $x$ 为根的子树中的节点总数：

$$
\text{size}(x) = 1 + \text{size}(\text{left}(x)) + \text{size}(\text{right}(x))
$$

### select 正确性

设 $r = \text{size}(\text{left}(x)) + 1$，则 $r$ 恰好是 $x$ 在子树中序遍历序列中的秩。

- 若 $i = r$，则 $x$ 即为所求；
- 若 $i < r$，则第 $i$ 小元素在左子树中；
- 若 $i > r$，则第 $i$ 小元素在右子树中，且其在右子树中的相对秩为 $i - r$。

由结构归纳法，`select` 正确。

### rank 正确性

若 $x$ 在左子树中，其全局秩等于在左子树中的秩；
若 $x$ 在右子树中，其全局秩等于 $r$ 加上在右子树中的秩。

同样由结构归纳法可证。

## 五、应用场景

1. **动态中位数维护**：在线数据流中实时查询当前序列的中位数（`select(n/2)`）。
2. **数据库排名查询**：例如“查找销量排名第 $k$ 的产品”。
3. **游戏排行榜**：玩家分数动态变化时快速查询玩家排名。
4. **区间树（Interval Tree）**：顺序统计树的扩展思想也可用于区间查询（如线段树、树状数组）。

---

## 六、扩展变体

- **Treap / Cartesian Tree**：结合堆性质与 BST 性质，通过随机优先级保证平衡，天然支持 size 维护和顺序统计。
- **Fenwick Tree / Segment Tree 上的顺序统计**：若键域为有限整数范围，可用树状数组或线段树在 $O(\log n)$ 内实现 `select` 和 `rank`。
- **持久化顺序统计树**：支持查询历史版本中某个元素的排名，常用于离线算法与可持久化数据结构。
- **Wavelet Tree**：一种更高级的顺序统计结构，支持在子区间上查询第 $k$ 小，适用于字符串和序列处理。

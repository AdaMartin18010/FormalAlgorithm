## 1. 斐波那契堆 (Fibonacci Heap)

斐波那契堆是由 Fredman 与 Tarjan 于 1987 年提出的摊还数据结构 [FredmanTarjan1987]，它在支持 **decrease-key** 操作的表现上显著优于二叉堆，从而将 Dijkstra 单源最短路径算法与 Prim 最小生成树算法的时间复杂度优化到 $O(V \log V + E)$ 与 $O(E + V \log V)$。

### 1.1 结构定义

**定义 1.1.1** (斐波那契堆)

斐波那契堆 $H$ 是由一组**根链表**（root list）连接的**有根树**组成的集合。每棵树都满足**最小堆序**（min-heap order）：每个节点的关键字不大于其所有子节点的关键字。堆的**最小节点** $H.min$ 指向根链表中关键字最小的节点。

每个节点 $x$ 维护以下属性：

| 属性 | 含义 |
|------|------|
| $x.key$ | 节点关键字 |
| $x.degree$ | 子节点的数量 |
| $x.parent$ | 父节点指针（根节点为 `NIL`）|
| $x.child$ | 指向某个子节点 |
| $x.left$, $x.right$ | 双向兄弟链表指针 |
| $x.mark$ | **标记域**（布尔值）|

**标记域** $x.mark$ 的语义：若 $x$ 在成为某节点的子节点后**失去了恰好一个子节点**（由于 cut 操作），则 $x.mark = \text{true}$；否则为 $\text{false}$。标记域是控制摊还成本、保证树形状接近斐波那契数列的关键。

**根链表**（Root List）：所有根节点通过 $left$ 与 $right$ 指针构成一个循环双向链表。$H.min$ 始终指向根链表中 $key$ 最小的节点。

**最大度数界**（Maximum Degree）：对于包含 $n$ 个节点的斐波那契堆，任意节点的度数上界为 $D(n) \leq \lfloor \log_\phi n \rfloor$，其中 $\phi = (1 + \sqrt{5}) / 2$ 为黄金比例。该界得名于斐波那契数列在分析中的核心作用。

### 1.2 势函数与摊还分析

**势函数**（Potential Function）定义 [CLRS2022]：

$$\Phi(H) = t(H) + 2 \cdot m(H)$$

其中 $t(H)$ 为根链表中树的棵数，$m(H)$ 为标记节点的数量。

#### 1.2.1 核心操作的摊还成本

**定理 1.2.1** (斐波那契堆操作摊还复杂度)

设 $H$ 为包含 $n$ 个节点的斐波那契堆，则：

- $\text{insert}(x)$ 的摊还成本为 $O(1)$
- $\text{find-min}$ 的摊还成本为 $O(1)$
- $\text{union}(H_1, H_2)$ 的摊还成本为 $O(1)$
- $\text{extract-min}$ 的摊还成本为 $O(\log n)$
- $\text{decrease-key}(x, k)$ 的摊还成本为 $O(1)$
- $\text{delete}(x)$ 的摊还成本为 $O(\log n)$

**证明概要**：

1. **insert**：将新节点直接插入根链表，更新 $H.min$。实际成本 $O(1)$，势增量 $\Delta\Phi = +1$，摊还成本 $\hat{c} = O(1) + 1 = O(1)$。

2. **extract-min**：抽取最小节点后，将其子节点全部移入根链表，然后执行**级联合并**（cascading consolidation），将同度数的根两两链接，直到根链表中各根度数互异。设实际成本为 $O(t(H) + D(n))$。合并后根树数 $t'(H) \leq D(n) + 1$，势变化满足：
   $$\Delta\Phi \leq (D(n) + 1) - t(H)$$
   因此摊还成本：
   $$\hat{c} \leq c \cdot (t(H) + D(n)) + (D(n) + 1 - t(H)) = O(D(n)) = O(\log n)$$

3. **decrease-key**：将节点 $x$ 的关键字减小。若破坏堆序，则执行 **cut**($H, x, y$)：将 $x$ 从其父 $y$ 处切下，移入根链表，并清除标记。若 $y$ 已被标记，则对 $y$ 递归执行 **cascading-cut**，形成级联切割链。

   每次 cut 的实际成本为 $O(1)$。设共执行 $c$ 次 cut，势变化为：
   $$\Delta\Phi = (t(H) + c) + 2(-(c - 1) + 1) - t(H) = t(H) + c - 2c + 4 - t(H) = 4 - c$$
   （每切下一个节点增加一棵根树 $+1$，同时可能减少标记数。）因此摊还成本：
   $$\hat{c} = c \cdot O(1) + (4 - c) = O(1)$$

**定理 1.2.2** (最大度数界)

对于任意节点 $x$，设其度数为 $k$，以 $x$ 为根的子树至少包含 $F_{k+2}$ 个节点，其中 $F_k$ 为第 $k$ 个斐波那契数（$F_0 = 0, F_1 = 1$）。因此：

$$n \geq F_{k+2} \geq \phi^k \quad \Rightarrow \quad k \leq \log_\phi n$$

**证明**（归纳法）：利用标记域规则，度数为 $k$ 的节点的第 $i$ 个子节点（按加入顺序）的度数至少为 $i - 2$。子树大小满足递推 $S_k \geq 2 + \sum_{i=0}^{k-2} S_i$，这正是斐波那契数列的递推形式。∎

### 1.3 在 Dijkstra 与 Prim 中的应用

**定理 1.3.1** (Dijkstra 算法优化)

使用斐波那契堆实现的 Dijkstra 单源最短路径算法，在稠密图与稀疏图上的时间复杂度均为：

$$T(V, E) = O(V \log V + E)$$

**证明**：算法执行 $V$ 次 $\text{extract-min}$（总计 $O(V \log V)$）和 $E$ 次 $\text{decrease-key}$（总计 $O(E)$）。∎

**定理 1.3.2** (Prim 算法优化)

使用斐波那契堆实现的 Prim 最小生成树算法的时间复杂度为：

$$T(V, E) = O(E + V \log V)$$

**证明**：与 Dijkstra 类似，$V$ 次 $\text{extract-min}$ 与 $E$ 次 $\text{decrease-key}$。∎

| 数据结构 | Dijkstra | Prim |
|----------|----------|------|
| 二叉堆 | $O(E \log V)$ | $O(E \log V)$ |
| 斐波那契堆 | $O(V \log V + E)$ | $O(E + V \log V)$ |
| 适用场景 | $E = o(V \log V)$ 时显著优于二叉堆 | 稀疏图尤佳 |

## 2. van Emde Boas 树 (vEB Tree)

van Emde Boas 树由 Peter van Emde Boas 于 1975 年提出 [Boas1975]，是一种针对**有界整数域** $[0, u-1]$ 设计的递归数据结构，支持 $O(\log \log u)$ 时间的插入、删除、前驱与后继查询。

### 2.1 递归结构

**定义 2.1.1** (vEB 树)

设全域大小（universe size）为 $u = 2^{2^k}$（$k \geq 0$）。vEB 树 $V$ 递归定义如下：

- **基础情况** ($u = 2$)：仅维护一个长度为 2 的位数组，直接存储两个元素是否存在。
- **递归情况** ($u > 2$)：
  - $V.min$ 与 $V.max$：分别存储当前集合中的最小与最大元素（若集合为空则为 `NIL`）。
  - $V.cluster[0 \ldots \sqrt{u} - 1]$：一个包含 $\sqrt{u}$ 个子 vEB 树的数组，每个子树的全域大小为 $\sqrt{u}$。
  - $V.summary$：一棵额外的 vEB 树，全域大小为 $\sqrt{u}$，记录哪些 cluster 非空。

**聚类索引**：对于元素 $x \in [0, u-1]$，定义：

- 高半部（cluster 索引）：$\text{high}(x) = \lfloor x / \sqrt{u} \rfloor$
- 低半部（偏移量）：$\text{low}(x) = x \bmod \sqrt{u}$
- 重构：$\text{index}(h, l) = h \cdot \sqrt{u} + l$

### 2.2 操作与复杂度

**定理 2.2.1** (vEB 树操作复杂度)

对于全域大小为 $u$ 的 vEB 树，以下操作均可在 $O(\log \log u)$ 时间内完成：

- **insert**($V, x$)
- **delete**($V, x$)
- **minimum**($V$) / **maximum**($V$)
- **successor**($V, x$) / **predecessor**($V, x$)
- **member**($V, x$)

**证明概要**：

每次递归调用都将问题规模从 $u$ 缩小到 $\sqrt{u}$。设 $T(u)$ 为操作时间，则递归关系为：

$$T(u) = T(\sqrt{u}) + O(1)$$

令 $u = 2^m$，则 $T(2^m) = T(2^{m/2}) + O(1)$。由主定理或变量替换法：

$$T(u) = O(\log \log u)$$

**Successor 查询伪代码**：

```
function SUCCESSOR(V, x):
    if x < V.min then return V.min
    max_low = MAXIMUM(V.cluster[high(x)])
    if max_low ≠ NIL and low(x) < max_low then
        return index(high(x), SUCCESSOR(V.cluster[high(x)], low(x)))
    else
        succ_cluster = SUCCESSOR(V.summary, high(x))
        if succ_cluster == NIL then return NIL
        return index(succ_cluster, MINIMUM(V.cluster[succ_cluster]))
```

### 2.3 应用场景

vEB 树的主要价值在于**整数优先队列**和**稀疏整数集合**的管理：

1. **整数优先队列**：当关键字范围 $u$ 相对于元素数 $n$ 较小时（$u = \text{poly}(n)$），vEB 树的 $O(\log \log u) = O(\log \log n)$ 操作远快于二叉堆的 $O(\log n)$。
2. **图算法的边松弛**：在边权为整数且范围受限的最短路径问题中，可用 vEB 树优化 Dijkstra 算法。
3. **范围查询与顺序统计**：前驱/后继查询天然支持动态顺序统计。

## 3. Link-Cut 树

Link-Cut 树由 Sleator 与 Tarjan 于 1983 年提出 [SleatorTarjan1983]，是处理**动态森林**（dynamic forest）问题的经典数据结构，支持在均摊 $O(\log n)$ 时间内完成路径查询、路径修改、连接（link）与切断（cut）操作。

### 3.1 动态树问题定义

**定义 3.1.1** (动态树问题)

维护一个由若干棵有根树组成的动态森林 $F$，支持以下操作：

| 操作 | 语义 |
|------|------|
| $\text{find-root}(v)$ | 返回节点 $v$ 所在树的根 |
| $\text{link}(v, w)$ | 假设 $v$ 是某棵树的根，将 $v$ 的父节点设为 $w$ |
| $\text{cut}(v)$ | 切断 $v$ 与其父节点之间的边 |
| $\text{path-aggregate}(v)$ | 返回从根到 $v$ 的路径上的聚合信息（如最小值、和） |
| $\text{path-update}(v, x)$ | 对根到 $v$ 的路径施加更新操作 |

### 3.2 Preferred Path 与 Splay 操作

Link-Cut 树的核心思想是将每棵有根树分解为若干条**preferred path **，每条 preferred path 用一棵**辅助 splay 树**（auxiliary splay tree）维护。

**Preferred Child**：对于节点 $v$，若从根到 $v$ 的最后访问路径经过其子节点 $c$，则 $c$ 是 $v$ 的 preferred child。每个节点最多有一个 preferred child。

**Preferred Path**：由 preferred child 关系形成的一条从某个节点向下延伸到叶子（或 preferred child 为空的节点）的路径。整棵树被划分为若干条互不相交的 preferred path。

**辅助树**（Auxiliary Tree）：每条 preferred path 上的节点按键值（在原树中的深度）组织成一棵 splay 树。在辅助树中，中序遍历对应路径上从浅到深的节点顺序。

**关键操作**：

1. **access($v$)**：将从根到 $v$ 的路径变为新的 preferred path。具体操作：
   - 对 $v$ 执行 splay，将其提为辅助树根。
   - 将 $v$ 的右子树（对应 preferred path 中比 $v$ 更深的部分）断开，作为新的独立 preferred path。
   - 沿父指针（path-parent pointer）向上遍历，重复上述过程直到根。

2. **make-root($v$)**：先将 $v$ access 到根，然后反转整棵辅助树（利用懒标记翻转），使 $v$ 成为整棵树的根。

3. **link($v, w$)**：make-root($v$) 后，将 $v$ 的父指针设为 $w$。

4. **cut($v$)**：access($v$) 后，$v$ 的左子树即为其原父节点方向的路径，直接断开左子树即可。

### 3.3 均摊复杂度分析

**定理 3.3.1** (Link-Cut 树操作均摊复杂度)

对于包含 $n$ 个节点的 Link-Cut 树，$\text{access}$、$\text{link}$、$\text{cut}$、$\text{path-aggregate}$ 等操作的均摊时间复杂度均为 $O(\log n)$。

**证明概要** [SleatorTarjan1983]：

将 splay 树的势能分析扩展到 Link-Cut 树的辅助森林。定义节点 $v$ 的势能为 $\phi(v) = \log s(v)$，其中 $s(v)$ 为以 $v$ 为根的辅助 splay 子树的大小。整个 Link-Cut 树的势能为所有辅助树中所有节点势能之和。

每次 $\text{access}(v)$ 涉及若干次 splay 操作。由 splay 树的摊还分析，单次 splay 的摊还成本为 $O(\log n)$。在 $\text{access}$ 过程中，对沿 path-parent 链上的每个节点执行一次 splay，通过**加权分析**（weighted analysis）可以证明，整条链上所有 splay 操作的总摊还成本为 $O(\log n)$。

具体地，设 $w$ 为 $v$ 的某个祖先，在将 $w$ splay 到其辅助树根的过程中，势变化被 $O(\log n)$ 的界控制。由于 preferred path 的重构只会减少（或有限增加）节点所在辅助树的大小，望远镜求和后总摊还成本为 $O(\log n)$。∎

## 4. Rust 实现框架

以下提供三种数据结构的 Rust 类型定义与核心操作伪代码框架，展示所有权与内存安全的设计思路。

### 4.1 斐波那契堆核心框架

```rust
use std::ptr::NonNull;
use std::cmp::Ordering;

pub struct FibNode<K: Ord, V> {
    key: K,
    value: V,
    degree: usize,
    mark: bool,
    parent: Option<NonNull<FibNode<K, V>>>,
    child: Option<NonNull<FibNode<K, V>>>,
    left: NonNull<FibNode<K, V>>,
    right: NonNull<FibNode<K, V>>,
}

pub struct FibHeap<K: Ord, V> {
    min: Option<NonNull<FibNode<K, V>>>,
    n: usize,
}

impl<K: Ord, V> FibHeap<K, V> {
    pub fn new() -> Self {
        FibHeap { min: None, n: 0 }
    }

    pub fn insert(&mut self, key: K, value: V) -> NonNull<FibNode<K, V>> {
        let node = Box::into_raw(Box::new(FibNode {
            key, value, degree: 0, mark: false,
            parent: None, child: None,
            left: NonNull::dangling(), right: NonNull::dangling(),
        }));
        let mut node = unsafe { NonNull::new_unchecked(node) };
        unsafe {
            (*node.as_ptr()).left = node;
            (*node.as_ptr()).right = node;
        }
        self.min = match self.min {
            None => Some(node),
            Some(min) => {
                self.concat_root_list(min, node);
                if key < unsafe { (*min.as_ptr()).key } { Some(node) } else { Some(min) }
            }
        };
        self.n += 1;
        node
    }

    fn concat_root_list(&mut self, a: NonNull<FibNode<K, V>>, b: NonNull<FibNode<K, V>>) {
        unsafe {
            let a_right = (*a.as_ptr()).right;
            let b_left = (*b.as_ptr()).left;
            (*a.as_ptr()).right = b;
            (*b.as_ptr()).left = a;
            (*a_right.as_ptr()).left = b_left;
            (*b_left.as_ptr()).right = a_right;
        }
    }

    // extract_min, decrease_key, consolidate 等操作省略...
}
```

### 4.2 vEB 树核心框架

```rust
#[derive(Clone)]
pub struct VEBTree {
    u: usize,
    min: Option<usize>,
    max: Option<usize>,
    summary: Option<Box<VEBTree>>,
    cluster: Vec<Option<Box<VEBTree>>>,
}

impl VEBTree {
    pub fn new(u: usize) -> Self {
        if u == 2 {
            VEBTree { u, min: None, max: None, summary: None, cluster: vec![] }
        } else {
            let sqrt = (u as f64).sqrt() as usize;
            VEBTree {
                u,
                min: None,
                max: None,
                summary: Some(Box::new(VEBTree::new(sqrt))),
                cluster: (0..sqrt).map(|_| Some(Box::new(VEBTree::new(sqrt)))).collect(),
            }
        }
    }

    fn high(&self, x: usize) -> usize { x / ((self.u as f64).sqrt() as usize) }
    fn low(&self, x: usize) -> usize { x % ((self.u as f64).sqrt() as usize) }
    fn index(&self, h: usize, l: usize) -> usize { h * ((self.u as f64).sqrt() as usize) + l }

    pub fn insert(&mut self, x: usize) {
        if let (Some(min), Some(max)) = (self.min, self.max) {
            if x < min { self.min = Some(x); }
            if x > max { self.max = Some(x); }
            let h = self.high(x);
            let l = self.low(x);
            if let Some(ref mut c) = self.cluster[h] {
                if c.min.is_none() {
                    self.summary.as_mut().unwrap().insert(h);
                }
                c.insert(l);
            }
        } else {
            self.min = Some(x);
            self.max = Some(x);
        }
    }

    // successor, predecessor, delete 等操作省略...
}
```

### 4.3 Link-Cut 树核心框架

```rust
pub struct LCTNode {
    left: Option<Box<LCTNode>>,
    right: Option<Box<LCTNode>>,
    parent: Option<NonNull<LCTNode>>,
    path_parent: Option<NonNull<LCTNode>>,
    rev: bool,
    val: i64,
    sum: i64,
}

impl LCTNode {
    fn is_root(&self) -> bool {
        // 在辅助树中是否为根（无父节点或不是父节点的 preferred child）
        true // 简化占位
    }

    fn push(&mut self) {
        if self.rev {
            std::mem::swap(&mut self.left, &mut self.right);
            if let Some(ref mut l) = self.left { unsafe { (*l).rev ^= true; } }
            if let Some(ref mut r) = self.right { unsafe { (*r).rev ^= true; } }
            self.rev = false;
        }
    }

    fn update(&mut self) {
        self.sum = self.val;
        if let Some(ref l) = self.left { self.sum += unsafe { (*l.as_ptr()).sum }; }
        if let Some(ref r) = self.right { self.sum += unsafe { (*r.as_ptr()).sum }; }
    }
}
```

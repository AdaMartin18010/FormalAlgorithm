# 有根树的表示法（Representation of Rooted Trees）


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

> 对标：CLRS 4th Ed. Chapter 10.4

## 一、理论基础

有根树（Rooted Tree）是计算机科学中最基础的数据结构之一。与二叉树不同，有根树的每个节点可以有**任意数量**的子节点。因此，如何高效地表示和遍历有根树是一个重要的工程问题。

### 1.1 基本定义

- **根节点（Root）**：没有父节点的节点，是树的入口。
- **子节点（Child）**：直接连接到某个节点下方的节点。
- **父节点（Parent）**：直接连接到某个节点上方的节点。
- **兄弟节点（Sibling）**：具有相同父节点的节点。
- **叶子节点（Leaf）**：没有子节点的节点。

### 1.2 常见的表示方法

| 表示法 | 空间复杂度 | 查找父节点 | 查找所有子节点 | 适用场景 |
|--------|-----------|-----------|---------------|----------|
| **父指针表示法** | $O(n)$ | $O(1)$ | $O(n)$ | 并查集、向上遍历 |
| **孩子链表表示法** | $O(n)$ | $O(1)$（若存父指针） | $O(k)$ | 通用有根树 |
| **左孩子右兄弟表示法（LCRS）** | $O(n)$ | $O(1)$ | $O(k)$ | 二叉树转换、节省空间 |
| **邻接表表示法** | $O(n)$ | $O(d)$ | $O(k)$ | 图论中的树 |

其中 $n$ 为节点总数，$k$ 为子节点数，$d$ 为节点的度数。

## 二、算法设计

### 2.1 父指针表示法（Parent-Pointer Representation）

每个节点只存储其父节点的引用（或索引）。

```rust
struct Node {
    data: i32,
    parent: Option<usize>, // 父节点索引，None 表示根
}
```

**特点**：

- 极省空间，每个节点只需一个额外指针。
- 只能高效地**向上遍历**（到根节点）。
- **并查集（Union-Find）** 是父指针表示法的经典应用。

### 2.2 孩子链表表示法（Child-List Representation）

每个节点维护一个指向其**第一个孩子**的指针，以及一个指向其**下一个兄弟**的指针。这就是**左孩子右兄弟表示法（Left-Child Right-Sibling, LCRS）**。

```rust
struct LcrsNode<T> {
    value: T,
    left_child: Option<Box<LcrsNode<T>>>,   // 第一个子节点
    right_sibling: Option<Box<LcrsNode<T>>>, // 下一个兄弟节点
}
```

**特点**：

- 将 $k$ 叉树转化为**二叉树**的等价表示。
- 每个节点固定只有 **2 个指针**，空间复杂度 $O(n)$。
- 遍历所有子节点：沿着 `left_child` 找到第一个孩子，再沿着 `right_sibling` 遍历所有兄弟。

### 2.3 邻接表表示法（Adjacency List）

在图论中，树是无向图的一种特殊形式。使用邻接表表示树时，每个节点维护一个邻居列表。

```rust
struct TreeAdj {
    n: usize,
    adj: Vec<Vec<usize>>,
}
```

**特点**：

- 适合存储**无根树**，可以方便地进行 BFS/DFS。
- 若需要表示有根树，通常额外指定一个根节点，并在遍历时记录父节点以避免回溯。

## 三、正确性证明

### 定理 1：LCRS 表示法可以唯一表示任意有根树

**证明**：

- 对于任意有根树中的节点 $v$，设其第一个子节点为 $c_1$，其余子节点为 $c_2, \dots, c_k$。
- 在 LCRS 表示中，$v$ 的 `left_child` 指向 $c_1$，$c_i$ 的 `right_sibling` 指向 $c_{i+1}$。
- 这种映射是**一一对应**的：给定原始有根树，可以构造唯一的 LCRS 表示；反之，给定 LCRS 表示，也可以唯一恢复原始的有根树结构（忽略兄弟顺序时）。
- 因此，LCRS 是有根树的有效表示。

### 定理 2：LCRS 表示法的空间复杂度为 $O(n)$

**证明**：

- $n$ 个节点，每个节点恰好有 2 个指针（`left_child` 和 `right_sibling`）。
- 总指针数为 $2n = O(n)$。
- 因此空间复杂度为 $O(n)$。

## 四、复杂度分析

| 操作 | 父指针法 | 孩子链表法 | LCRS | 邻接表法 |
|------|---------|-----------|------|---------|
| 查找父节点 | $O(1)$ | $O(1)$ | $O(1)$ | $O(d)$ |
| 查找第 $i$ 个子节点 | $O(n)$ | $O(i)$ | $O(i)$ | $O(1)$ |
| 遍历所有子节点 | $O(n)$ | $O(k)$ | $O(k)$ | $O(k)$ |
| 添加子节点 | $O(1)$ | $O(1)$ | $O(1)$ | $O(1)$ |
| 空间复杂度 | $O(n)$ | $O(n)$ | $O(n)$ | $O(n)$ |

## 五、类型理论视角

从类型论角度看，有根树可以表示为**递归类型（Recursive Type）**：

```haskell
Tree a = Node a [Tree a]
```

即一个节点包含一个值和一组子树列表。LCRS 表示法则对应于**二叉树类型**：

```haskell
LCRS a = Empty | Node a (LCRS a) (LCRS a)
```

其中第二个参数表示左孩子（子树），第三个参数表示右兄弟（同一棵树的另一棵子树）。这种对应关系在函数式编程和编译器实现中非常常见（例如抽象语法树的表示）。

## 六、工程实现要点

### 6.1 LCRS 遍历示例

```rust
fn print_lcrs_tree<T: Debug>(node: &LcrsNode<T>, depth: usize) {
    println!("{:indent$}{:?}", "", node.value, indent = depth * 2);

    // 遍历所有子节点
    let mut child = node.left_child.as_ref();
    while let Some(c) = child {
        print_lcrs_tree(c, depth + 1);
        child = c.right_sibling.as_ref();
    }
}
```

### 6.2 从孩子链表到 LCRS 的转换

将一个孩子链表表示的有根树转换为 LCRS 表示：

1. 对于每个节点，取其第一个子节点作为 `left_child`。
2. 对于其余子节点，依次链接为 `right_sibling` 链。
3. 递归处理所有子节点。

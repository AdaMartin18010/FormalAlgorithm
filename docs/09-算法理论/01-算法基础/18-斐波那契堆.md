# 斐波那契堆（Fibonacci Heap）


> **版本**: 1.0
> **创建日期**: 2026-04-19
> **最后更新**: 2026-04-19

> 对标：CLRS 第4版 Chapter 19

## 一、理论基础

斐波那契堆是一种**可合并堆（Mergeable Heap）**，由 Michael L. Fredman 与 Robert E. Tarjan 于 1987 年提出。它在**均摊意义**下支持以下操作的**最优**时间复杂度：

- `insert`：O(1)
- `extract-min`：O(log n)
- `decrease-key`：O(1) 均摊
- `merge`（union）：O(1)

正是由于 `decrease-key` 的常数均摊代价，斐波那契堆成为 **Dijkstra 算法** 和 **Prim 算法** 的**理论最优**数据结构，可将它们的时间复杂度分别优化至 **O(V log V + E)** 和 **O(E log V)**。

### 核心思想

斐波那契堆由一组**无序的根链表（root list）**组成，每个根节点是一棵**最小堆有序（min-heap-ordered）树**。树的度数可以任意大，但通过**级联剪切（Cascading Cut）**机制保证：

- 任意节点的度数上界为 $O(\log n)$；
- 树的结构足够“懒惰（lazy）”，使得大多数操作只需局部调整。

### 关键结构

| 字段 | 含义 |
|------|------|
| `key` | 节点键值 |
| `degree` | 子节点数量 |
| `mark` | 布尔标记，记录该节点是否已失去过一个子节点 |
| `parent`, `child` | 父指针、任一子指针 |
| `left`, `right` | 双向循环链表指针 |

## 二、算法设计

### 1. 插入（insert）

将新节点加入根链表，并更新最小指针。不合并相同度数的树。

```text
时间复杂度：O(1)
```

### 2. 合并两个堆（union）

将两个斐波那契堆的根链表拼接，取较小者为新的最小指针。

```text
时间复杂度：O(1)
```

### 3. 抽取最小值（extract-min）

1. 将最小节点的所有子节点提升至根链表；
2. 从根链表中移除最小节点；
3. **合并（consolidate）**根链表：按度数分组，将同度数的树两两链接（link），直到所有根节点的度数各不相同；
4. 重新确定最小指针。

```text
时间复杂度：O(log n) 均摊
```

### 4. 关键字减值（decrease-key）

1. 若新键值小于父节点键值，执行 **剪切（cut）**：将该节点从父节点处切下，移至根链表，并清除 `mark`；
2. 若父节点此前已失去过一个子节点（`mark == true`），则对父节点继续执行级联剪切（cascading cut），向上递归。

```text
时间复杂度：O(1) 均摊
```

### 5. 删除节点（delete）

执行 `decrease-key` 将该节点键值降至 $-\infty$，再执行 `extract-min`。

```text
时间复杂度：O(log n) 均摊
```

## 三、复杂度分析

| 操作 | 最坏情况 | 均摊复杂度 |
|------|----------|-----------|
| `make-heap` | O(1) | O(1) |
| `insert` | O(1) | O(1) |
| `minimum` | O(1) | O(1) |
| `extract-min` | O(n) | O(log n) |
| `union` | O(1) | O(1) |
| `decrease-key` | O(n) | **O(1)** |
| `delete` | O(n) | O(log n) |

### 度数上界证明（Fibonacci 数列来源）

设 $F_k$ 为第 $k$ 个斐波那契数。可以证明：

- 任意度数为 $k$ 的斐波那契堆节点，其子树至少包含 $F_{k+2}$ 个节点；
- 由于 $F_{k+2} \approx \phi^{k}$（其中 $\phi = \frac{1+\sqrt{5}}{2}$ 为黄金比例），故最大度数 $D(n) = O(\log n)$。

这也是“斐波那契堆”名称的由来。

---

## 四、形式化验证

### 关键不变式（Invariants）

1. **最小堆有序性**：任意节点的键值不大于其所有子节点的键值。
2. **根链表无序性**：根链表中的树无需保持特定顺序。
3. **度数唯一性（consolidate 后）**：每次 `extract-min` 结束后，根链表中任意两棵树的度数不同。
4. **Mark 规则**：一个节点被标记（marked）当且仅当它已失去恰好一个子节点；若失去第二个子节点，则被级联剪切。

### 均摊分析（势能法）

定义势函数：
$$
\Phi(H) = t(H) + 2 \cdot m(H)
$$
其中 $t(H)$ 为根链表中树的棵数，$m(H)$ 为被标记节点数。

- `insert`：实际代价 $O(1)$，势能增加 $1$，均摊代价 $O(1)$。
- `extract-min`：实际代价 $O(t(H) + D(n))$，势能下降至少 $t(H) - D(n) - 1$，均摊代价 $O(D(n)) = O(\log n)$。
- `decrease-key`：实际代价 $O(c)$（$c$ 为级联剪切次数），势能下降至少 $c - 1$，均摊代价 $O(1)$。

## 五、应用场景

1. **Dijkstra 最短路径**：使用斐波那契堆可将复杂度从 $O(E \log V)$ 优化至 $O(V \log V + E)$，在稠密图上具有理论优势。
2. **Prim 最小生成树**：同理可优化至 $O(E \log V)$ 或 $O(E + V \log V)$。
3. **网络流与组合优化**：作为可合并堆，在需要频繁 `decrease-key` 的算法中具有理论最优性。

### 工程实践中的取舍

尽管理论优势明显，斐波那契堆的**常数因子较大**，且实现复杂（指针操作、级联剪切）。在实际工程中，**二叉堆**或**配对堆（Pairing Heap）**往往表现更好。因此，斐波那契堆更多作为**理论基准**存在。

---

## 六、扩展变体

- **严格斐波那契堆（Strict Fibonacci Heap）**：由 Brodal 于 2012 年提出，首次实现了所有操作同时满足**最坏情况** $O(1)$（`extract-min` 除外，为 $O(\log n)$）。
- **配对堆（Pairing Heap）**：被认为是斐波那契堆的实用替代品，实验性能优异，但均摊分析尚未完全解决。
- **2-3 堆、斜堆（Skew Heap）**：其他可合并堆的变体。

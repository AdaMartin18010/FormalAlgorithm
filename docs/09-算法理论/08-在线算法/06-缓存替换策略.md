# 缓存替换策略

## 概述

缓存是计算机系统中最重要的性能优化技术之一。当缓存容量有限而请求序列在线到达时，需要决定替换哪个已有项。缓存替换问题是在线算法的经典应用场景。

## 问题定义

- **缓存容量**：$k$ 个槽位
- **请求序列**：$r_1, r_2, r_3, ...$，$r_i \in \mathcal{U}$（全集）
- **命中（hit）**：请求项已在缓存中，无代价
- **未命中（miss）**：请求项不在缓存中，产生代价 1，需替换某一项
- **目标**：最小化未命中次数

## 经典在线策略

### FIFO（First-In-First-Out）

替换最早进入缓存的项。

**竞争比**：$k$

### LRU（Least Recently Used）

替换最久未被访问的项。

**竞争比**：$k$

**实际表现**：在局部性良好的序列上接近最优，是现代 CPU 缓存、操作系统页面置换的标准策略。

### LFU（Least Frequently Used）

替换访问频次最低的项。

**缺点**：对新项不友好，需要维护计数器。

### Random

随机替换。

**竞争比**：$k$（期望）

## 最优离线算法：Belady's MIN

知道未来请求序列时，替换**最远将来被使用**的项（或永不被使用）。

这是最优的，但无法在线实现。

## 竞争比分析

**定理**：任何确定性在线缓存算法的竞争比 $\geq k$。

**证明思路**：构造请求序列，使算法每次未命中，而 MIN 仅每 $k$ 次未命中一次。

**LRU 和 FIFO 达到此下界**。

## 随机化改进

| 算法 | 竞争比 | 特点 |
|------|--------|------|
| Marking 算法 | $2H_k \approx 2\ln k$ | 阶段式标记 |
| Partition 算法 | $H_k$ | 更精细的随机化 |
| 下界 | $\Omega(\log k)$ | 任何随机化算法 |

## 实际系统应用

| 系统/层级 | 策略 | 容量 |
|-----------|------|------|
| CPU L1/L2/L3 | LRU 变种 | 32KB - 64MB |
| 操作系统页面 | LRU / Clock | 4KB 页面，GB 级 |
| 数据库缓冲池 | LRU / 2Q | 数百 MB - GB |
| CDN / Web 缓存 | LRU / LFU 混合 | TB 级 |
| Redis | 可配置（volatile-lru, allkeys-lru 等）| 内存限制 |

## 效果评估

- 典型工作负载的缓存命中率：80-95%
- 每提升 1% 命中率，系统响应时间降低 5-10%
- LRU 在大多数实际负载上优于 FIFO 和 LFU

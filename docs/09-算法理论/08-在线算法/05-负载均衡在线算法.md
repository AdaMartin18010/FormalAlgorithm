# 负载均衡在线算法

## 概述

在云计算、内容分发网络（CDN）和分布式系统中，任务（请求、作业）以在线方式到达，需要实时分配到多个服务器上。目标是最小化最大负载（makespan），而不知道未来的任务到达序列。

## 问题定义

- **输入**：$m$ 台相同的机器，任务序列 $j_1, j_2, ...$，任务 $j_i$ 处理时间为 $p_i$
- **输出**：将每个任务分配到某台机器
- **目标**：最小化 $\max_{i} L_i$，其中 $L_i$ 为机器 $i$ 的总负载

## 经典在线算法

### List Scheduling（Graham, 1966）

将每个到达的任务分配到**当前负载最小**的机器。

**竞争比**：$2 - \frac{1}{m}$

**证明概要**：

- 设最优 makespan 为 $OPT$
- 最后一项任务 $p_n$ 分配到机器 $k$ 前，$L_k \leq \frac{1}{m} \sum_{i<n} p_i \leq OPT$
- 因此 $ALG = L_k + p_n \leq OPT + OPT = 2 \cdot OPT$
- 更精细分析得 $2 - 1/m$

### 改进：按大小排序（离线）

若任务可排序（离线），Graham 证明 LPT（Longest Processing Time first）的近似比为 $\frac{4}{3} - \frac{1}{3m}$。

### 在线改进

| 算法 | 竞争比 | 特点 |
|------|--------|------|
| List Scheduling | $2 - 1/m$ | 简单，无需预知 |
| Bartal et al. | $1.986$ | 首个突破 2 |
| Karger et al. | $1.945$ | 随机化 |
| Albers | $1.923$ | 当前最好确定性 |
| 下界 | $1.853$ | 任何在线算法 |

## 相关模型

### 相关机器（Related Machines）

机器速度不同，$L_i = \frac{\text{分配到 } i \text{ 的任务和}}{s_i}$

### 带约束的分配

- 任务只能分配到特定子集（如数据局部性约束）
- 需要用网络流或匹配模型

## 实际系统

| 系统 | 策略 | 说明 |
|------|------|------|
| Nginx / HAProxy | Round Robin / Least Connections | 简单负载均衡 |
| Kubernetes | 资源请求 + 节点亲和性 | 调度器评分机制 |
| MapReduce | 任务窃取（Work Stealing）| 动态重平衡 |
| Spark | Delay Scheduling | 等待数据局部性 |

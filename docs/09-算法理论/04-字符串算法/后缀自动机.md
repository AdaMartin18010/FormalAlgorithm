## 一、理论基础

### 1.1 定义

**后缀自动机（Suffix Automaton, SAM）** 是一个接受字符串 $s$ 的所有后缀的**最小确定性有限自动机（DFA）**。由于 $s$ 的任意子串都是某个后缀的前缀，因此 SAM 也接受 $s$ 的所有子串。

形式化地，对于字符串 $s$，其后缀自动机 $\text{SAM}(s)$ 是一个五元组 $(Q, \Sigma, \delta, q_0, F)$，其中：

- $Q$：状态集合
- $\Sigma$：字符集
- $\delta$：转移函数
- $q_0$：初始状态
- $F$：接受状态集合（对应所有后缀）

### 1.2 核心性质

后缀自动机之所以强大，在于其**极度紧凑**的表示：

| 性质 | 上界 | 说明 |
|------|------|------|
| 状态数 | $2n - 1$ | $n = |s|$ |
| 转移数 | $3n - 4$ | 实际中通常更少 |
| 构造时间 | $O(n)$ | 在线构造算法 |

**endpos 等价类**：SAM 的每个状态对应 $s$ 的一组子串的结束位置集合（endpos 集合）。两个子串属于同一个状态，当且仅当它们在 $s$ 中的出现结束位置集合相同。

**link（后缀链接）**：每个状态 $v$（除初始状态外）都有一个后缀链接 $\text{link}(v)$，指向代表 $v$ 的最长子串的最长真后缀所在的状态。后缀链接构成一棵以初始状态为根的树，称为 **link 树**。

**len 数组**：状态 $v$ 的 $\text{len}(v)$ 表示该状态对应的最长子串的长度。

### 1.3 状态与 endpos 的关系

对于状态 $v$，设其对应的最长子串长度为 $\text{len}(v)$，其后缀链接对应的最长子串长度为 $\text{len}(\text{link}(v))$。则状态 $v$ 代表的所有子串长度范围为：

$$
(\text{len}(\text{link}(v)), \text{ len}(v)]
$$

这意味着一个状态可以代表**多个**不同长度的子串。

## 二、算法设计

### 2.1 在线构造算法

后缀自动机可以通过逐个添加字符的方式在线构造，时间复杂度为 $O(n)$。

#### 核心思想

当在字符串末尾添加新字符 $c$ 时：

1. 创建一个新状态 `cur`，代表整个新字符串的后缀。
2. 从上一个完整字符串的最后一个状态 `last` 开始，沿着后缀链接向上跳转，为这些状态添加指向 `cur` 的 $c$ 转移。
3. 如果某个状态已经有 $c$ 转移，则需要进行**分裂（clone）**操作：创建该状态的克隆，调整长度和链接，以保证自动机的最小性。

#### 伪代码

```text
function sa_extend(c):
    p = last
    cur = new_state(len = len(p) + 1)

    while p != -1 and p has no transition by c:
        add transition p --c--> cur
        p = link(p)

    if p == -1:
        link(cur) = root
    else:
        q = transition(p, c)
        if len(p) + 1 == len(q):
            link(cur) = q
        else:
            clone = clone_state(q)
            len(clone) = len(p) + 1
            while p != -1 and transition(p, c) == q:
                redirect p --c--> clone
                p = link(p)
            link(q) = link(cur) = clone

    last = cur
```

### 2.2 复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 构造 SAM | $O(n)$ | 均摊分析，每个状态/转移只被创建有限次 |
| 查询子串 | $O(m)$ | $m$ 为子串长度，沿转移边走即可 |
| 统计不同子串 | $O(|Q|)$ | DP 在 SAM 上进行 |
| 最长公共子串 | $O(|s_1| + |s_2|)$ | 先用 $s_1$ 建 SAM，再用 $s_2$ 在上面走 |

## 三、正确性证明

### 定理 1：SAM 的状态数不超过 $2n - 1$

**证明**：

- 每次 `sa_extend` 至少创建一个状态 `cur`。
- 只有在 `len(p) + 1 != len(q)` 时才创建克隆状态。
- 可以证明，每次扩展最多创建一个克隆状态，且每个原始状态最多被克隆一次。
- 因此总状态数 $\leq 2n - 1$。

### 定理 2：SAM 的转移数不超过 $3n - 4$

**证明**：基于对自动机结构的标准分析，每次扩展最多增加 2 个转移，结合状态数上界可证得转移数上界为 $3n - 4$。

### 定理 3：在线构造算法的时间复杂度为 $O(n)$

**证明**：

- 均摊分析：在 `sa_extend` 中，while 循环沿着后缀链接向上走。每次创建新转移或重定向转移都会"消耗"一个状态的后缀链接跳转机会。
- 由于后缀链接树的高度和状态总数都是 $O(n)$，且每个转移最多被创建/修改一次，因此均摊时间复杂度为 $O(n)$。

## 五、类型理论视角

后缀自动机可以被视为对字符串的**正则语言**（所有子串构成的语言）的最小 DFA 表示。从类型论角度看：

- 每个状态对应一个**等价类**（endpos 等价类）。
- 转移函数 $\delta(v, c)$ 对应于在类型 $v$ 上应用构造子 $c$ 得到的新类型。
- 后缀链接构成一个**偏序关系**（由子串包含关系诱导），link 树是该偏序的 Hasse 图的一种表示。

## 六、工程实现要点

### 6.1 数据结构

```rust
struct State {
    next: HashMap<char, usize>, // 转移
    link: isize,                // 后缀链接，-1 表示根
    len: usize,                 // 最长子串长度
}

struct SuffixAutomaton {
    st: Vec<State>,
    last: usize,
}
```

### 6.2 关键技巧

- **克隆状态**：必须正确复制原状态的所有转移，并调整 `len`。
- **DP 顺序**：按 `len` 降序遍历状态（拓扑排序），可以方便地在 link 树上做 DP。
- **出现次数统计**：在每个 `cur` 状态的 `occ` 上标记 1，然后按 `len` 降序将 `occ` 累加到 `link` 上。

### 6.3 常见应用

1. **判断子串**：从初始状态出发，尝试沿子串字符走转移，看能否走完。
2. **不同子串个数**：对每个状态 $v$，其贡献为 $\text{len}(v) - \text{len}(\text{link}(v))$。
3. **第 $k$ 小子串**：在 SAM 上按字典序做 DP。
4. **最长公共子串**：对第二个串的每个前缀，在 SAM 上尽量走，记录最长匹配长度。
5. **最小循环移位**：将字符串加倍后建 SAM，找长度为 $n$ 的最小字典序路径。

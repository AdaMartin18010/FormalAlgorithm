## 04-字符串算法. KMP 算法 / Knuth-Morris-Pratt Algorithm

### 1. 理论基础

KMP 算法由 Knuth、Morris 与 Pratt 于 1977 年提出 [Knuth et al., 1977]。其核心在于利用**模式串自身的结构信息**，在失配时避免主串指针回退。定义**前缀函数**（$ ext{prefix function}$，亦称 failure function）

$$
\pi[i] \;=\; \max\{k \mid 0 \le k < i+1,\; P[0..k-1] = P[i-k+1..i]\},
$$

即子串 $P[0..i]$ 的最长相等真前缀与真后缀的长度。一旦 $ ext{prefix function}$ 被正确计算，匹配阶段即可在 $O(n)$ 时间内完成扫描。

### 2. 算法设计

KMP 算法分为两个阶段：

1. **预处理阶段**：自匹配计算 $\pi$ 数组。维护变量 $k$ 表示当前最长公共前缀长度，遍历模式串 $P$：
   - 若 $P[k] \neq P[q]$，则回退 $k = \pi[k-1]$，直至匹配或 $k=0$；
   - 若 $P[k] = P[q]$，则 $k \leftarrow k+1$，并令 $\pi[q] = k$。

2. **匹配阶段**：用文本指针 $i$（单调递增）与模式指针 $q$ 同步扫描文本 $T$：
   - 失配时回退 $q = \pi[q-1]$；
   - 匹配时 $q \leftarrow q+1$；
   - 当 $q = m$ 时报告一次完整匹配，随后令 $q = \pi[m-1]$ 继续寻找重叠匹配。

项目实现 additionally 提供了流式匹配器 `KmpMatcher`，可在无法一次性加载全部文本的场景下逐字节处理。

### 3. 复杂度分析

| 阶段 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 前缀函数计算 | $O(m)$ | $O(m)$ |
| 主串匹配 | $O(n)$ | $O(m)$（仅存储 $\pi$ 数组） |
| **总计** | **$O(n+m)$** | **$O(m)$** |

其中 $n$ 为文本长度，$m$ 为模式长度。

### 4. 形式化验证 / 正确性论证

**引理 1（前缀函数正确性）**：对任意位置 $q$，算法输出的 $\pi[q]$ 等于 $P[0..q]$ 的最长相等真前缀-真后缀长度。

*证明*：对 $q$ 进行归纳。基例 $q=0$ 时 $\pi[0]=0$ 显然成立。归纳假设对所有位置 $<q$ 成立。设 $k$ 为进入循环前的值，若 $P[k] \neq P[q]$，则根据归纳假设，$\pi[k-1]$ 是次长的候选，回退过程恰好枚举了所有可能前缀长度。最终 $k$ 为最大可行值，故 $\pi[q]=k$ 正确。$\square$

**定理（KMP 线性时间）**：对任意文本 $T$ 与模式 $P$，KMP 算法的总比较次数不超过 $2n + 2m$。

*证明*：预处理阶段 $q$ 从 $1$ 扫描到 $m-1$，且每次回退至少使 $k$ 减少 $1$，而 $k$ 的总增量不超过 $m$，故比较次数 $O(m)$。匹配阶段文本指针 $i$ 只增不减，模式指针 $q$ 的增量不超过 $n$，回退总量受 $n$ 限制，故比较次数 $O(n)$。$\square$

### 5. 应用场景

- **文本编辑器查找**：在大型文档中实现 $O(n)$ 的单模式搜索。
- **生物信息学**：DNA、RNA 序列中的基因片段定位。
- **网络包过滤**：单模式深度包检测（DPI）的底层匹配引擎。
- **流式处理**：`KmpMatcher` 适用于流式字节流，无需预加载完整数据。

### 6. 扩展变体

| 变体 | 说明 |
|------|------|
| **多模式 KMP** | 为每个模式独立维护前缀函数，逐个扫描文本（Aho-Corasick 的前身）。 |
| **双向匹配（Two-Way Algorithm）** | 结合前缀函数与后缀函数，从模式两端同时扫描，常数因子更优。 |
| **流式 KMP** | 将状态机封装为 `KmpMatcher`，支持逐字节输入与实时匹配。 |

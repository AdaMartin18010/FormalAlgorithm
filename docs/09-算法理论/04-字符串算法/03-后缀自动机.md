# 后缀自动机（Suffix Automaton）

## 概述

后缀自动机（SAM）是一种强大的线性时间字符串数据结构，由 Anisimov 和 Knuth 提出，后由 Blumer 等人完善。它能以 $O(n)$ 的时间和空间复杂度，编码字符串的所有子串信息。

## 结构定义

后缀自动机是一个有向无环图（DAG），其中：

- **状态（State）**：代表一类子串的结束位置集合（endpos equivalence class）
- **转移（Transition）**：$\delta(v, c) = u$ 表示状态 $v$ 通过字符 $c$ 转移到状态 $u$
- **初始状态**：$t_0$
- **终止状态**：所有对应后缀的状态集合

## 关键性质

### 1. 状态数上界

对于长度为 $n$ 的字符串，SAM 的状态数不超过 $2n - 1$，转移数不超过 $3n - 4$。

### 2. endpos 等价类

两个子串 $s_1, s_2$ 属于同一状态，当且仅当它们在原串中的所有出现结束位置集合相同。

### 3. link 树（Suffix Link Tree）

每个状态（除初始状态外）有一个后缀链接 $link(v)$，指向其最长的真后缀所对应的状态。这些链接形成一棵树。

## 构建算法（在线构造）

逐个字符扩展 SAM，维护最后一个状态 $last$：

```
for each character c:
    create new state cur
    p = last
    while p ≠ t0 and no transition from p by c:
        add transition p --c--> cur
        p = link(p)
    if no transition from p by c:
        link(cur) = t0
    else:
        q = transition from p by c
        if len(q) = len(p) + 1:
            link(cur) = q
        else:
            clone q into clone
            len(clone) = len(p) + 1
            while p ≠ t0 and transition p --c--> q:
                redirect to clone
                p = link(p)
            link(q) = link(cur) = clone
    last = cur
```

时间复杂度：均摊 $O(n)$

## 经典应用

| 问题 | SAM 解法 |
|------|---------|
| 不同子串个数 | 统计所有状态的 $len(v) - len(link(v))$ 之和 |
| 第 $k$ 字典序子串 | 在 DAG 上按拓扑序 DP |
| 最长公共子串 | 对两个串构建 SAM，用第二个串在第一个的 SAM 上行走 |
| 出现次数查询 | 在 link 树上统计 endpos 集合大小 |
| 最短未出现子串 | 在 SAM 上找最短的不可行路径 |

## 与后缀数组的对比

| 特性 | 后缀自动机 | 后缀数组 |
|------|-----------|---------|
| 空间 | $O(n)$（常数较大）| $O(n)$ |
| 构建时间 | $O(n)$ | $O(n \log n)$ 或 $O(n)$ |
| 子串查询 | $O(|pattern|)$ | $O(|pattern| \log n)$ |
| 实现难度 | 较高 | 中等 |
| 出现次数 | 天然支持 | 需结合 LCP |

## 一、概念定义 (Concept Definition)

### 1.1 后缀数组定义

**定义 1.1.1** (后缀数组 / Suffix Array)

给定字符串 $S$ 长度为 $n$，其后缀数组 $SA$ 是一个长度为 $n$ 的排列，满足：

$$S[SA[0]..n-1] < S[SA[1]..n-1] < \cdots < S[SA[n-1]..n-1]$$

即 $SA[i]$ 表示第 $i$ 小的后缀的起始位置。

**示例**：对于 $S = "banana"$

- 后缀："banana", "anana", "nana", "ana", "na", "a"
- 排序后："a", "ana", "anana", "banana", "na", "nana"
- $SA = [5, 3, 1, 0, 4, 2]$

---

### 1.2 排名数组

**定义 1.2.1** (排名数组 / Rank Array)

排名数组 $Rank[i]$ 表示从位置 $i$ 开始的后缀在后缀数组中的排名：

$$Rank[SA[i]] = i$$

$Rank$ 是 $SA$ 的逆排列，满足 $SA[Rank[i]] = i$。

---

### 1.3 LCP数组

**定义 1.3.1** (LCP数组 / Longest Common Prefix Array)

LCP数组 $LCP[i]$ 表示后缀 $SA[i]$ 与 $SA[i-1]$ 的最长公共前缀长度：

$$LCP[i] = \text{LCP}(S[SA[i]..], S[SA[i-1]..])$$

特别地，$LCP[0] = 0$。

**示例**：$S = "banana"$

- $SA = [5, 3, 1, 0, 4, 2]$
- $LCP = [0, 1, 3, 0, 0, 2]$

---

### 1.4 关键性质

**定理 1.4.1** (LCP区间与公共前缀)

对于任意 $i < j$，后缀 $SA[i]$ 和 $SA[j]$ 的最长公共前缀长度为：

$$\text{LCP}(SA[i], SA[j]) = \min_{k=i+1}^{j} LCP[k]$$

## 二、属性 (Properties)

### 2.1 后缀数组构建算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 特点 | 适用场景 |
|------|-----------|-----------|------|---------|
| **朴素** | $O(n^2 \log n)$ | $O(n^2)$ | 直接排序后缀字符串 | 教学演示 |
| **倍增** | $O(n \log n)$ | $O(n)$ | 基于倍增+基数排序 | 竞赛实现 |
| **DC3/Skew** | $O(n)$ | $O(n)$ | 递归分治 | 理论最优 |
| **SA-IS** | $O(n)$ | $O(n)$ | 诱导排序 | 实际最快 |

### 2.2 LCP数组应用

| 问题 | 解决方法 | 复杂度 |
|------|---------|--------|
| 最长重复子串 | $\max(LCP)$ | $O(n)$ |
| 不同子串计数 | $\frac{n(n+1)}{2} - \sum LCP$ | $O(n)$ |
| 模式匹配 | 二分查找+LCP优化 | $O(m \log n)$ |
| 最长公共子串 | 拼接字符串+LCP | $O(n)$ |

### 2.3 与其他数据结构对比

| 结构 | 构建时间 | 空间 | 子串查询 | 后缀查询 |
|------|---------|------|---------|---------|
| **后缀数组** | $O(n)$ / $O(n \log n)$ | $O(n)$ | $O(m \log n)$ | $O(1)$ |
| **后缀树** | $O(n)$ | $O(n)$ | $O(m)$ | $O(m)$ |
| **后缀自动机** | $O(n)$ | $O(n)$ | $O(m)$ | $O(m)$ |
| **Trie** | $O(n^2)$ | $O(n^2)$ | $O(m)$ | $O(m)$ |

## 四、解释 (Explanation)

### 4.1 倍增算法核心思想

**关键洞察**: 利用已排序的 $k$-前缀信息，在 $O(n)$ 时间内排序 $2k$-前缀。

```
第k轮: 比较 S[i..i+k-1] 和 S[j..j+k-1]
        ↓
使用上一轮排名 (rank[i], rank[i+k]) vs (rank[j], rank[j+k])
        ↓
基数排序，得到新的排名
        ↓
k *= 2，直到 k >= n
```

**复杂度分析**:

- 每轮排序：$O(n)$（基数排序）
- 轮数：$O(\log n)$（$k$ 倍增）
- 总复杂度：$O(n \log n)$

### 4.2 Kasai算法直观理解

**核心观察**: 如果两个后缀的LCP为 $h$，则它们去掉首字符后的LCP至少为 $h-1$。

```
位置i的后缀:  banana
位置j的后缀:  anana
LCP = 0

去掉首字符:
位置i+1: anana
位置j+1: nana
LCP = 0 (符合 ≥ 0-1 = -1，即0)
```

利用这个性质，可以从左到右在 $O(n)$ 时间内计算所有LCP。

### 4.3 不同子串计数原理

字符串 $S$ 的所有子串数量：$\frac{n(n+1)}{2}$

每个LCP值 $h$ 表示两个后缀共享了 $h$ 个子串（以这些前缀开头的子串重复）。

因此不同子串数：

$$\text{Distinct} = \frac{n(n+1)}{2} - \sum_{i=1}^{n-1} LCP[i]$$

## 五、形式证明 (Formal Proofs)

### 5.1 倍增算法正确性证明

**定理**: 倍增算法正确构建后缀数组。

**证明**:

**归纳基础**: $k=1$ 时，按首字符排序，正确。

**归纳假设**: 假设第 $k$ 轮后，所有长度为 $k$ 的子串排名正确。

**归纳步骤**: 第 $2k$ 轮，比较 $(rank[i], rank[i+k])$ 和 $(rank[j], rank[j+k])$。

- 若 $rank[i] \neq rank[j]$，则 $S[i..i+k-1] \neq S[j..j+k-1]$，排序正确。
- 若 $rank[i] = rank[j]$，则比较 $rank[i+k]$ 和 $rank[j+k]$，即比较 $S[i+k..i+2k-1]$ 和 $S[j+k..j+2k-1]$。

由归纳假设，两部分比较都正确，因此整体正确。

**终止条件**: 当 $k \geq n$ 时，所有后缀都被完整比较。

∎

### 5.2 Kasai算法线性时间证明

**定理**: Kasai算法在 $O(n)$ 时间内计算LCP数组。

**证明**:

算法伪代码：

```
h = 0
for i = 0 to n-1:
    if Rank[i] > 0:
        j = SA[Rank[i] - 1]
        while S[i+h] == S[j+h]:
            h += 1
        LCP[Rank[i]] = h
        if h > 0: h -= 1
```

**关键观察**:

1. $h$ 只增不减（除了每次减1）
2. $h$ 最多增加 $n$ 次（每次比较成功）
3. $h$ 总共减少不超过 $n$ 次（每次循环最多减1）

因此while循环总共执行 $O(n)$ 次。

∎

### 5.3 LCP区间最小值性质证明

**定理**: 对于 $i < j$，$\text{LCP}(SA[i], SA[j]) = \min_{k=i+1}^{j} LCP[k]$。

**证明**:

设 $m$ 是 $[i+1, j]$ 中 $LCP$ 最小的位置，$h = LCP[m]$。

**上界**: $LCP[m]$ 表示 $SA[m]$ 和 $SA[m-1]$ 的公共前缀长度。由于后缀数组按字典序排列，$SA[i]$ 和 $SA[j]$ 的公共前缀不能超过 $h$（否则 $SA[m]$ 的位置会不同）。

**下界**: 所有位置在 $[i, j]$ 的后缀都以相同的 $h$ 个字符开头（因为相邻的都有至少 $h$ 的LCP，由传递性可得）。

因此等式成立。

∎

## 六、实现参考

### 6.1 倍增算法伪代码

```rust
fn build_sa(s: &str) -> Vec<usize> {
    let n = s.len();
    let mut sa: Vec<usize> = (0..n).collect();
    let mut rank: Vec<usize> = s.bytes().map(|b| b as usize).collect();
    let mut k = 1;

    while k < n {
        // 按第二关键字(rank[i+k])，再按第一关键字(rank[i])排序
        sa.sort_by(|&i, &j| {
            let ri = if i + k < n { rank[i + k] } else { 0 };
            let rj = if j + k < n { rank[j + k] } else { 0 };
            (ri, rank[i]).cmp(&(rj, rank[j]))
        });

        // 重新计算rank
        let mut new_rank = vec![0; n];
        new_rank[sa[0]] = 0;
        for i in 1..n {
            new_rank[sa[i]] = new_rank[sa[i-1]] +
                if (rank[sa[i]], rank[sa[i]+k]) != (rank[sa[i-1]], rank[sa[i-1]+k]) { 1 } else { 0 };
        }
        rank = new_rank;
        k *= 2;
    }
    sa
}
```

### 6.2 Kasai算法伪代码

```rust
fn build_lcp(s: &str, sa: &[usize], rank: &[usize]) -> Vec<usize> {
    let n = s.len();
    let mut lcp = vec![0; n];
    let mut h = 0;
    let bytes = s.as_bytes();

    for i in 0..n {
        if rank[i] == 0 { continue; }
        let j = sa[rank[i] - 1];
        while i + h < n && j + h < n && bytes[i + h] == bytes[j + h] {
            h += 1;
        }
        lcp[rank[i]] = h;
        if h > 0 { h -= 1; }
    }
    lcp
}
```

# 随机选择（Randomized Select）

## 概述

随机选择算法用于在未完全排序的数组中查找第 $k$ 小（或大）的元素，期望时间复杂度为 $O(n)$。它是快速排序随机化思想的直接应用，由 Hoare 的 Quickselect 算法演化而来。

## 算法描述

```
RandomizedSelect(A, p, r, k):
    if p == r: return A[p]
    q = RandomizedPartition(A, p, r)
    i = q - p + 1  // A[q] 是第 i 小的元素
    if k == i: return A[q]
    if k < i: return RandomizedSelect(A, p, q-1, k)
    else: return RandomizedSelect(A, q+1, r, k-i)
```

## 期望时间复杂度分析

### 指示器随机变量法

设 $X_i$ 为指示器变量，表示第 $i$ 小的元素是否被选为枢轴。

$$E[T(n)] \leq \sum_{i=1}^{n} \frac{1}{n} \cdot E[T(\max(i-1, n-i))] + O(n)$$

### 关键观察

对于一半的元素，$\max(i-1, n-i) \leq 3n/4$：

$$
\begin{aligned}
E[T(n)] &\leq \frac{2}{n} \sum_{i=\lfloor n/2 \rfloor}^{n-1} E[T(i)] + O(n) \\
&\leq \frac{2}{n} \cdot n \cdot E[T(3n/4)] + O(n) \\
&= 2E[T(3n/4)] + O(n)
\end{aligned}
$$

由递归树法或代入法：$E[T(n)] = O(n)$

## 与确定性选择的对比

| 算法 | 最坏时间 | 期望时间 | 常数因子 | 实现复杂度 |
|------|---------|---------|---------|-----------|
| 随机选择 | $O(n^2)$ | $O(n)$ | 小 | 简单 |
| Median-of-Medians | $O(n)$ | $O(n)$ | 大 | 较复杂 |

## 应用

- **Top-K 问题**：查找最大的 $k$ 个元素（期望 $O(n + k \log k)$）
- **中位数查找**：$O(n)$ 期望时间
- **快速排序优化**：用 Quickselect 找中位数作为枢轴，保证 $O(n \log n)$ 最坏时间

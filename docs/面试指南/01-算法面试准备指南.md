## 💡 面试解题技巧

### 面试答题四步法

```
1. 理解题目 (2-3分钟)
   ├── 复述题目要求
   ├── 询问边界条件
   ├── 确认输入输出格式
   └── 讨论几个示例

2. 思考方案 (3-5分钟)
   ├── 暴力解法（先说出来）
   ├── 分析时间和空间复杂度
   ├── 思考优化方向
   └── 确定最优方案

3. 编码实现 (10-15分钟)
   ├── 先写主逻辑框架
   ├── 边写边解释思路
   ├── 处理边界情况
   └── 测试关键用例

4. 测试与优化 (3-5分钟)
   ├── 自测正常情况
   ├── 测试边界情况
   ├── 分析复杂度
   └── 讨论可能的优化
```

### 常见错误与避免

| 错误类型 | 示例 | 解决方案 |
|---------|------|---------|
| 空指针 | 访问 null->next | 先判空 |
| 数组越界 | arr[n] 越界 | 检查索引范围 |
| 整数溢出 | mid = (l+r)/2 | mid = l + (r-l)/2 |
| 死循环 | while (l < r) 边界 | 明确循环终止条件 |
| 忘记回溯 | 递归后未恢复状态 | 成对操作 |
| 边处理遗漏 | 首尾元素特殊处理 | 单独讨论边界 |

## 🎤 面试沟通技巧

### 思考 aloud（边说边想）

**推荐表达：**

```
"我首先想到一个暴力解法..."
"让我分析一下复杂度..."
"我注意到这里有重叠子问题，可以用动态规划..."
"边界情况需要特殊处理，比如当数组为空时..."
"让我用几个例子验证一下..."
```

**避免：**

- 完全沉默超过 30 秒
- 直接写代码不解释
- 坚持错误方向不调整

### 遇到不会的题目

1. **请求提示**："能否给一点提示？"
2. **分解问题**：尝试简化版本
3. **换个角度**：从其他思路尝试
4. **诚实表达**："这部分我不太确定，但我会尝试..."

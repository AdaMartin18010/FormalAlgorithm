| 指标 | 说明 | 优化方向 |

**一致性哈希（算法工程师重点）**

```python
# 一致性哈希简化实现
import hashlib
import bisect

class ConsistentHash:
    def __init__(self, replicas=3):
        self.replicas = replicas  # 虚拟节点数
        self.ring = []            # 哈希环
        self.nodes = {}           # 节点映射

    def _hash(self, key):
        return int(hashlib.md5(key.encode()).hexdigest(), 16)

    def add_node(self, node):
        for i in range(self.replicas):
            key = self._hash(f"{node}:{i}")
            bisect.insort(self.ring, key)
            self.nodes[key] = node

    def remove_node(self, node):
        for i in range(self.replicas):
            key = self._hash(f"{node}:{i}")
            self.ring.remove(key)
            del self.nodes[key]

    def get_node(self, key):
        if not self.ring:
            return None
        hash_key = self._hash(key)
        idx = bisect.bisect_right(self.ring, hash_key)
        if idx == len(self.ring):
            idx = 0
        return self.nodes[self.ring[idx]]
```

**缓存算法（LRU）**

```cpp
// LRU Cache 实现
class LRUCache {
    int capacity;
    list<pair<int, int>> cache;
    unordered_map<int, list<pair<int, int>>::iterator> map;

public:
    LRUCache(int capacity) : capacity(capacity) {}

    int get(int key) {
        auto it = map.find(key);
        if (it == map.end()) return -1;
        // 移动到头部
        cache.splice(cache.begin(), cache, it->second);
        return it->second->second;
    }

    void put(int key, int value) {
        auto it = map.find(key);
        if (it != map.end()) {
            cache.erase(it->second);
        }
        cache.push_front({key, value});
        map[key] = cache.begin();

        if (cache.size() > capacity) {
            map.erase(cache.back().first);
            cache.pop_back();
        }
    }
};
```

### 3. 限流算法

```python
# 令牌桶算法
import time
import threading

class TokenBucket:
    def __init__(self, rate, capacity):
        self.rate = rate          # 每秒产生的令牌数
        self.capacity = capacity  # 桶容量
        self.tokens = capacity    # 当前令牌数
        self.last_time = time.time()
        self.lock = threading.Lock()

    def allow_request(self, tokens=1):
        with self.lock:
            now = time.time()
            # 计算新增令牌
            self.tokens = min(
                self.capacity,
                self.tokens + self.rate * (now - self.last_time)
            )
            self.last_time = now

            if self.tokens >= tokens:
                self.tokens -= tokens
                return True
            return False
```

## 4. 布隆过滤器

### 4. 布隆过滤器

```cpp
// 布隆过滤器
class BloomFilter {
    vector<bool> bits;
    int size;
    vector<function<int(string)>> hashFunctions;

public:
    BloomFilter(int n, int k) : size(n) {
        bits.resize(n);
        // 使用 k 个哈希函数
        for (int i = 0; i < k; i++) {
            hashFunctions.push_back([i, n](string key) {
                hash<string> h;
                return (h(key) + i * h(key + "salt")) % n;
            });
        }
    }

    void add(string key) {
        for (auto& hf : hashFunctions) {
            bits[hf(key)] = true;
        }
    }

    bool mightContain(string key) {
        for (auto& hf : hashFunctions) {
            if (!bits[hf(key)]) return false;
        }
        return true;  // 可能存在（有假阳性）
    }
};
```

## 📊 系统设计答题框架

### 4S 框架

```
1. Scenario（场景分析）
   ├── 功能需求：系统要解决什么问题
   ├── 非功能需求：QPS、延迟、数据规模
   └── 用户规模：DAU、峰值流量

2. Service（服务划分）
   ├── 模块拆分：按功能或业务拆分
   ├── 接口设计：API 定义
   └── 数据流：请求如何流转

3. Storage（存储设计）
   ├── 数据建模：Schema 设计
   ├── 存储选型：SQL/NoSQL/Cache
   └── 索引设计：查询优化

4. Scale（扩展设计）
   ├── 高可用：故障转移、降级
   ├── 高性能：缓存、异步、批处理
   └── 高扩展：水平扩展、分片
```

### 答题时间分配

| 阶段 | 时间 | 产出 |
|-----|------|------|
| 需求分析 | 2-3 min | 明确范围、约束 |
| 高层设计 | 10-15 min | 系统架构图 |
| 详细设计 | 15-20 min | 关键组件设计 |
| 扩展讨论 | 5-10 min | 优化、瓶颈、监控 |

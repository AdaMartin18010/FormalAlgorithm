## 一、核心概念

### 1.1 定义

**正式定义**:
安全多方计算（Secure Multi-Party Computation, MPC）允许 $n$ 个参与方 $P_1, \ldots, P_n$ 在各自持有私有输入 $x_1, \ldots, x_n$ 的情况下，共同计算一个公共函数 $f(x_1, \ldots, x_n) = (y_1, \ldots, y_n)$，其中 $P_i$ 仅获得输出 $y_i$，且在计算过程中不泄露超出函数输出所隐含的输入信息。

**安全定义（真实/理想世界范式）**:
协议 $\Pi$ 安全实现函数 $f$，如果对于任何真实世界中的敌手 $\mathcal{A}$，存在理想世界中的模拟器 $\mathcal{S}$，使得对于所有输入：
$$\text{Real}_{\Pi, \mathcal{A}} \approx \text{Ideal}_{f, \mathcal{S}}$$

**被动安全（半诚实模型）**: 敌手遵循协议但试图从视图中学习额外信息
**主动安全（恶意模型）**: 敌手可任意偏离协议

**直观解释**:
想象一群亿万富翁想知道谁最富有，但不想透露各自的财富。MPC允许他们"合作计算"比较函数，每个人只知道比较结果（如"A最富有"），而不知道其他人的具体财富值。

**关键要点**:

- **隐私保护**: 中间计算不泄露输入信息
- **正确性**: 输出结果与可信第三方计算一致
- **输入独立性**: 各方独立选择输入
- **公平性**: 所有诚实方获得输出（主动安全下）

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 隐私性 | 中间视图不泄露超出输出的信息 | 由模拟器证明 |
| 正确性 | 输出等于指定函数值 | 对抗恶意行为 |
| 独立性 | 输入选择不受其他方影响 | 防止强制性泄露 |
| 公平性 | 所有诚实方获得输出 | 可放宽为带中止的安全 |

**性质总结**:

1. **通用性**: 任何函数都可安全计算（Yao、GMW定理）
2. **门限性**: $t < n/2$ 被动腐败可容忍，$t < n/3$ 主动腐败可容忍
3. **通信复杂度**: 依赖于电路深度和宽度
4. **计算复杂度**: 通常比明文计算慢 $10^3$-$10^6$ 倍

### 1.3 变体

**姚氏混淆电路（Garbled Circuits）**:

- 定义: 将布尔电路加密，使得计算者不知中间值
- 与GMW的区别: 常数轮通信，适合两方
- 适用场景: 两方安全计算、隐私集合求交

**GMW协议**:

- 定义: 基于秘密共享的MPC，逐门计算
- 与Yao的区别: 多方参与，轮数与电路深度成正比
- 适用场景: 多方计算、在线/离线分离

**SPDZ系列**:

- 定义: 基于消息认证码（MAC）的预处理模型协议
- 与传统协议的区别: 昂贵密码操作移到预处理阶段
- 适用场景: 算术电路、机器学习训练

## 三、形式化证明

### 3.1 核心定理

**定理 1 (Yao, 1986)**: 对于任意多项式时间可计算的函数 $f$，存在一个常数轮的两方安全计算协议，在半诚实敌手下安全实现 $f$。

**证明**:

构造基于混淆电路的协议：

**构造**: 混淆电路 $GC$ 由以下组成：

- 对每根导线 $w$，两个混淆值 $\tilde{w}_0, \tilde{w}_1$，满足 $\tilde{w}_b = (k_b, b \oplus \pi_w)$，其中 $k_b$ 是密钥，$\pi_w$ 是混淆位
- 对每个门 $g$，混淆表 $T_g$，使得给定输入混淆值，可计算输出混淆值

**协议**:

1. Alice（电路生成者）构造 $GC$ 并发送给 Bob
2. Alice 通过OT向 Bob 传输对应其输入的混淆值
3. Alice 直接发送其输入对应的混淆值
4. Bob 评估电路，获得输出混淆值，映射为实际输出
5. Bob 发送输出给 Alice（或保持私有）

**安全性**:

- **Alice安全**: Bob仅看到混淆值，与随机不可区分（基于PRF/PRG）
- **Bob安全**: Alice仅从OT获得信息，由OT安全性保证

模拟器构造：

- 对Bob的模拟器：给定Bob的输出 $y$，构造随机混淆值使其计算得到 $y$
- 对Alice的模拟器：由OT模拟器保证

∎

**证明要点分析**:

1. 混淆电路隐藏真实输入值，只暴露函数输出
2. OT保证Bob无法获知Alice输入，Alice无法获知Bob输入选择
3. 安全性归约到PRG/PRF和OT的安全性

**定理 2 (GMW, 1987)**: 对于任意 $n$ 方函数 $f$，若 $t < n/2$ 方被腐败，存在半诚实安全的MPC协议。

*证明概要*: 使用Shamir秘密共享 $(t, n)$-门限方案，各门计算通过本地运算和通信完成。∎

### 3.2 辅助引理

**引理 1 (Free-XOR优化)**: 若混淆电路使用特定的混淆值结构（$\tilde{w}_1 = \tilde{w}_0 \oplus R$），则XOR门可无需混淆表（free）评估。

*证明*: 设 $\tilde{w} = k_w \| (w \oplus \pi_w)$，定义 $\tilde{w}_1 = \tilde{w}_0 \oplus (R \| 1)$。对XOR门输出 $c = a \oplus b$：
$$\tilde{c}_0 \oplus \tilde{c}_1 = (k_a \oplus k_b) \| ((a \oplus b) \oplus (\pi_a \oplus \pi_b))$$
可由 $\tilde{a}_v \oplus \tilde{b}_w$ 直接计算。∎

**引理 2 (Cut-and-Choose)**: 通过让生成者构造多个电路并打开部分验证，可将半诚实安全协议转化为恶意安全协议。

*证明概要*: 设生成 $s$ 个电路，随机打开 $s/2$ 个验证正确性。若存在错误电路，被检测概率 $\geq 1 - 2^{-s/2}$。∎

## 四、算法/方法详解

### 4.1 算法描述

**姚氏混淆电路协议**:

```
协议: Yao-GC-2PC
参与方: Alice (输入 x), Bob (输入 y)
函数: f(x, y) 表示为布尔电路 C

Alice (电路生成者):
1. 为 C 的每根导线 w 生成混淆密钥对 (k_w^0, k_w^1)
2. 为每根导线生成混淆位 π_w ∈ {0,1}
3. 对 C 的每个门 g，构造混淆表:
   - 对每种输入组合 (a,b) ∈ {0,1}^2
   - 输出混淆值: Enc_{k_u^a, k_v^b}(k_w^{g(a,b)})
4. 发送混淆电路 GC 给 Bob
5. 通过OT向 Bob 发送 x 对应的输入混淆值
6. 发送自己的输入混淆值 (直接暴露给Bob)

Bob (电路评估者):
1. 通过OT从 Alice 获取 y 对应的输入混淆值
2. 接收 Alice 的输入混淆值
3. 拓扑序评估每个门:
   - 用输入密钥解密混淆表项
   - 获得输出混淆值
4. 从输出混淆值映射到实际输出
5. （可选）将输出发送给 Alice
```

**GMW协议（AND门）**:

```
协议: GMW-AND
输入: P1 有 a1, P2 有 a2, 共享 AND 结果

预处理:
1. P1 生成随机比特 r1, P2 生成随机比特 r2
2. 双方计算 r = r1 ⊕ r2（通过OT或共享随机源）
3. 计算乘法三元组 (u1,u2), (v1,v2), (w1,w2) 满足 (u1⊕u2)∧(v1⊕v2) = w1⊕w2

在线阶段:
1. P1 设置 x1 = a1 ⊕ u1, P2 设置 x2 = a2 ⊕ u2
2. 双方交换 x1, x2（公开）
3. P1 计算 c1 = x1∧x2 ⊕ x1∧v1 ⊕ x2∧u1 ⊕ w1
4. P2 类似计算 c2
5. c1 ⊕ c2 = a1∧a2
```

**流程图**:

```
Yao协议:               GMW协议:
Alice          Bob     P1          P2
  |              |       |          |
  |-- GC ------->|       |--共享值-->| (AND门)
  |              |       |          |
  |-- OT(x) ---->|       |<--公开值--|
  |              |       |          |
  |-- 她的输入->|       |--本地计算  |
  |              |       |          |
  |              |       |--结果共享->|
  |              |       |          |
评估完成          结果重构
```

### 4.2 正确性分析

**混淆电路的正确性**:

- 每个混淆表项用输入密钥加密正确的输出密钥
- 评估者只能用正确的输入密钥解密正确的输出行
- 通过混淆位映射确保输出值正确解码

**GMW AND门的正确性**:
设 $a = a_1 \oplus a_2$，$b = b_1 \oplus b_2$，乘法三元组满足 $uv = w$。

展开 $c_1 \oplus c_2$ 的计算可验证等于 $ab$。

### 4.3 复杂度分析

**Yao混淆电路**:

- 通信: $O(|C| \cdot \lambda)$，$\lambda$ 为安全参数（通常128或256）
- 轮数: $O(1)$（常数轮）
- 计算: $O(|C|)$ 加密操作

**GMW协议**:

- 通信: $O(|C| \cdot n^2 \cdot \lambda)$ 对于 $n$ 方
- 轮数: $O(d)$，$d$ 为电路深度
- 计算: $O(|C|)$ 本地运算 + 通信

**SPDZ协议**:

- 离线: $O(|C|)$ 同态加密操作（昂贵但可并行）
- 在线: $O(|C|)$ 轻量运算，$O(1)$ 轮

## 五、示例与实例

### 5.1 标准示例

**示例 1: 两方百万富翁问题**

**问题**: Alice有财富 $a$，Bob有财富 $b$，双方想知道谁更富有但不透露具体数值。

**电路设计**:

```
输入: a (n位), b (n位)
输出: a > b ?

比较器电路:
- 从高位到低位逐位比较
- 若高位不同，决定结果
- 若高位相同，比较下一位
```

**协议执行**:

1. Alice构造比较电路的混淆版本
2. Alice通过OT向Bob发送她的输入混淆值
3. Bob评估电路
4. Bob将结果（或混淆结果）发送给Alice

**安全分析**:

- Alice从OT只学习Bob的输入长度
- Bob从混淆电路只学习比较结果

**示例 2: 隐私集合求交（PSI）**

Alice有集合 $A = \{a_1, \ldots, a_n\}$，Bob有 $B = \{b_1, \ldots, b_m\}$。

目标: 共同获得 $A \cap B$，但不泄露其他元素。

**协议**:

1. 将元素哈希为布隆过滤器
2. 使用OT或混淆电路比较
3. 或用Diffie-Hellman风格协议:
   - Alice发送 $\{H(a_i)^x\}$
   - Bob发送 $\{H(b_j)^y\}$ 和 $\{H(b_j)^{xy}\}$
   - Alice计算 $\{H(a_i)^{xy}\}$ 并比较

### 5.2 代码实现

**Python实现 (简单两方AND门)**:

```python
import random
import hashlib
from typing import Tuple, List

class SimpleYaoGC:
    """简化的姚氏混淆电路演示"""

    def __init__(self, security_param: int = 128):
        self.lambda_ = security_param // 8  # 字节数

    def _prf(self, key: bytes, input_val: int) -> bytes:
        """伪随机函数"""
        return hashlib.sha256(key + input_val.to_bytes(4, 'big')).digest()

    def garble_and_gate(self) -> Tuple[List[bytes], List[bytes], dict]:
        """
        混淆AND门
        输入导线: u, v; 输出导线: w
        返回: (混淆表, 输出混淆值, 解密映射)
        """
        # 生成导线密钥
        keys = {
            'u': (random.randbytes(self.lambda_), random.randbytes(self.lambda_)),
            'v': (random.randbytes(self.lambda_), random.randbytes(self.lambda_)),
            'w': (random.randbytes(self.lambda_), random.randbytes(self.lambda_))
        }

        # 混淆位
        pi = {
            'u': random.randint(0, 1),
            'v': random.randint(0, 1),
            'w': random.randint(0, 1)
        }

        # 构造混淆表（4行，每行对应一种输入组合）
        garbled_table = []
        dec_map = {}  # 解密映射

        for a in [0, 1]:
            for b in [0, 1]:
                # AND门输出
                c = a & b

                # 混淆输出值
                c_tilde = c ^ pi['w']

                # 加密: 用输入密钥加密输出密钥和混淆位
                key_a = keys['u'][a]
                key_b = keys['v'][b]
                encrypted = bytes(x ^ y ^ z for x, y, z in zip(
                    self._prf(key_a, 0)[:self.lambda_],
                    self._prf(key_b, 1)[:self.lambda_],
                    keys['w'][c] + bytes([c_tilde])
                ))

                garbled_table.append(encrypted)
                dec_map[(a, b)] = keys['w'][c], c_tilde

        # 随机打乱表项顺序
        random.shuffle(garbled_table)

        # 输出混淆值（用于解码）
        output_wires = [keys['w'][0] + bytes([0 ^ pi['w']]),
                       keys['w'][1] + bytes([1 ^ pi['w']])]

        return garbled_table, output_wires, keys, pi

    def evaluate_and_gate(self, garbled_table: List[bytes],
                         input_key_u: bytes, input_key_v: bytes) -> bytes:
        """评估混淆AND门"""
        for entry in garbled_table:
            # 尝试解密
            decrypted = bytes(x ^ y ^ z for x, y, z in zip(
                entry,
                self._prf(input_key_u, 0)[:self.lambda_],
                self._prf(input_key_v, 1)[:self.lambda_]
            ))
            # 验证解密是否有效（简化：检查格式）
            if len(decrypted) == self.lambda_ + 1:
                return decrypted
        raise ValueError("解密失败")

# GMW协议简单演示
class SimpleGMW:
    """GMW协议简化版（两方AND门）"""

    @staticmethod
    def share_secret(secret: int) -> Tuple[int, int]:
        """加法共享秘密"""
        share1 = random.randint(0, 1)
        share2 = secret ^ share1
        return share1, share2

    @staticmethod
    def and_gate_p1(a1: int, b1: int, d: int) -> int:
        """P1的AND门计算"""
        # d = a2 & b2 (从P2获取的掩码)
        return a1 & b1 ^ a1 & d[0] ^ b1 & d[1] ^ d[2]

    @staticmethod
    def and_gate_p2(a2: int, b2: int) -> Tuple[int, int, int]:
        """P2的AND门计算，生成掩码"""
        return a2, b2, a2 & b2

# 使用示例
if __name__ == "__main__":
    print("=== Yao混淆电路演示 ===")
    gc = SimpleYaoGC(security_param=128)
    table, outputs, keys, pi = gc.garble_and_gate()
    print(f"混淆表项数: {len(table)}")

    # 模拟评估
    result = gc.evaluate_and_gate(table, keys['u'][1], keys['v'][1])
    print(f"评估结果: {result.hex()}")

    print("\n=== GMW协议演示 ===")
    # Alice输入1, Bob输入1
    a, b = 1, 1
    a1, a2 = SimpleGMW.share_secret(a)
    b1, b2 = SimpleGMW.share_secret(b)
    print(f"Alice共享: a1={a1}, b1={b1}")
    print(f"Bob共享: a2={a2}, b2={b2}")

    # 模拟AND计算（简化，实际需要交互）
    print(f"实际AND结果: {a & b}")
```

**Rust实现**:

```rust
use rand::Rng;
use sha2::{Sha256, Digest};

/// 简化的混淆电路
pub struct GarbledCircuit {
    security_param: usize,
}

impl GarbledCircuit {
    pub fn new(security_param: usize) -> Self {
        Self { security_param }
    }

    fn prf(&self, key: &[u8], input: u32) -> Vec<u8> {
        let mut hasher = Sha256::new();
        hasher.update(key);
        hasher.update(input.to_be_bytes());
        hasher.finalize().to_vec()
    }

    /// 生成随机密钥对
    pub fn generate_wire_keys(&self) -> (Vec<u8>, Vec<u8>) {
        let mut rng = rand::thread_rng();
        let key0: Vec<u8> = (0..self.security_param).map(|_| rng.gen()).collect();
        let key1: Vec<u8> = (0..self.security_param).map(|_| rng.gen()).collect();
        (key0, key1)
    }
}

fn main() {
    let gc = GarbledCircuit::new(16);
    let (k0, k1) = gc.generate_wire_keys();
    println!("密钥0: {:?}", k0);
    println!("密钥1: {:?}", k1);
}
```

## 5.3 反例

### 5.3 反例

**常见错误**:

**错误1: 混淆表顺序可预测**

```python
# 错误：混淆表按输入顺序排列
garbled_table = []
for a in [0, 1]:
    for b in [0, 1]:
        garbled_table.append(encrypt(keys_u[a], keys_v[b], output))
# 评估者从解密位置推断输入！
```

**错误原因**: 表项位置泄露输入信息
**正确做法**: 随机打乱表项顺序

**错误2: 重复使用OT基**

```python
# 错误：多次使用相同的OT选择
for round in range(100):
    ot_send(choices[round], same_base_keys)
# 敌手可组合多次观察攻击
```

**错误原因**: 破坏了OT的独立安全性
**正确做法**: 每次OT使用独立参数

## 六、习题

### 6.1 理解题

1. **解释**: 为什么Yao的混淆电路协议在半诚实模型下是安全的，但在恶意模型下需要修改？

   <details>
   <summary>解答</summary>

   半诚实模型中，Alice和Bob都遵循协议：
   - Alice诚实构造正确的混淆电路
   - Bob诚实评估并报告结果

   恶意模型中的攻击：
   1. Alice可构造错误的电路（如泄露Bob输入）
   2. Bob可恶意选择OT输入（虽然OT本身安全，但组合攻击可能）

   恶意安全加固：
   - Cut-and-choose: Alice构造多个电路，打开部分验证
   - 输入一致性: 确保Alice的输入在所有电路中一致
   - 输出验证: 防止Bob报告错误输出
   </details>

### 6.2 应用题

1. **设计**: 设计一个隐私保护的机器学习推理协议，客户端有输入数据，服务器有训练好的模型，双方协作计算预测结果。

   <details>
   <summary>解答</summary>

   使用混合方法：

   1. **线性层**: 使用同态加密
      - 客户端加密输入 $[x]$
      - 服务器计算 $[Wx + b]$
      - 客户端解密

   2. **非线性激活**: 使用混淆电路或函数秘密共享
      - ReLU: 比较电路
      - Sigmoid: 分段近似 + 混淆电路

   3. **优化**:
      - 批量处理减少开销
      - 使用FHE的SIMD操作
      - 通信与计算trade-off

   具体协议（Delphi系统）：
   - 预处理：使用MPC生成乘法三元组
   - 在线：客户端输入，双方协作评估神经网络
   </details>

### 6.3 证明题

1. **证明**: 若OT协议是安全的，则基于混淆电路的两方计算协议在半诚实模型下安全。

   <details>
   <summary>解答</summary>

   构造模拟器：

   **模拟Bob的视图**:
   1. 模拟器知道Bob的输入 $y$ 和输出 $f(x, y)$
   2. 生成随机混淆值代替真实密钥
   3. 构造混淆表使得评估结果为 $f(x, y)$
   4. 由OT模拟器生成Bob的OT视图

   **混淆值的伪随机性**: 基于PRF的安全性，混淆值与随机不可区分
   **混淆表的正确性**: 每一行用正确的输入密钥加密正确的输出
   **OT安全性**: 由OT的半诚实安全保证

   由于模拟器仅使用输出 $f(x, y)$ 就能产生与真实执行不可区分的视图，协议是安全的。
   </details>

### 7.2 实际案例

**案例**: 苹果隐私保护数据收集

**背景**: 收集用户数据进行分析但保护个人隐私

**应用方式**:

1. 使用局部差分隐私（Local DP）在设备上扰动数据
2. 使用安全聚合（Secure Aggregation）聚合数据
3. 服务器只能看到聚合结果，无法恢复个体数据

**效果**: 获得统计洞察同时保护用户隐私

### 7.3 跨领域联系

**与差分隐私的联系**:
MPC保护计算过程中的隐私，差分隐私保护输出的隐私，两者可结合使用。

**与区块链的联系**:
门限签名用于多签钱包，MPC用于隐私交易（如Zcash的扩展）。

## 八、多维对比

### 8.1 方法对比

| 维度 | Yao GC | GMW | SPDZ | FHE |
|------|--------|-----|------|-----|
| 适用方数 | 2方 | 多方 | 多方 | 单方/多方 |
| 通信轮数 | 常数 | O(深度) | O(1) | O(1) |
| 通信量 | 大 | 中等 | 小(在线) | 小 |
| 计算量 | 中等 | 小 | 大(离线) | 极大 |
| 恶意安全 | 需强化 | 需强化 | 内置 | 天然 |
| 适用电路 | 布尔 | 布尔 | 算术 | 任意 |

### 8.2 决策建议

**何时选择Yao混淆电路**:

- 两方计算场景
- 低延迟要求
- 布尔电路为主

**何时选择SPDZ**:

- 多方计算
- 算术电路（机器学习）
- 可接受预处理开销

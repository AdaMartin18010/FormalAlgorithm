## 一、核心概念

### 1.1 定义

**正式定义**:
快速排序是一种基于**分治策略**的比较排序算法，由Tony Hoare于1960年提出。其核心操作是**分区**（Partition）：选择一个**枢轴元素**（pivot），将数组重新排列使得：

- 枢轴左侧的所有元素 ≤ 枢轴
- 枢轴右侧的所有元素 ≥ 枢轴

**递归定义**:

```
QuickSort(A, p, r):
    if p < r then
        q = Partition(A, p, r)
        QuickSort(A, p, q-1)
        QuickSort(A, q+1, r)
```

**直观解释**:
想象整理一副扑克牌：

1. 随机抽一张牌作为"基准"
2. 将比它小的牌放左边，大的放右边
3. 对左右两堆分别重复上述过程
4. 最终得到有序序列

**关键要点**:

- **原地排序**：仅需$O(\log n)$额外空间（递归栈）
- **不稳定排序**：相等元素的相对顺序可能改变
- **枢轴选择**决定性能：最好/平均$O(n \log n)$，最坏$O(n^2)$

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 时间复杂度（最好） | $O(n \log n)$ | 每次分区均衡 |
| 时间复杂度（平均） | $O(n \log n)$ | 随机输入的期望性能 |
| 时间复杂度（最坏） | $O(n^2)$ | 已排序且选首尾为枢轴 |
| 空间复杂度 | $O(\log n)$ | 递归调用栈深度 |
| 稳定性 | 不稳定 | 相等元素可能交换位置 |
| 适应性 | 不具备 | 对已排序数据性能差 |
| 比较次数（平均） | $\approx 1.39n\ln n$ | 比归并排序多约39% |

**性质总结**:

1. **缓存友好性**: 顺序访问内存，缓存命中率高
2. **原地操作**: 不需要额外存储空间存储临时数组
3. **尾递归优化**: 可优化递归深度，保证$O(\log n)$栈空间

### 1.3 变体

**随机化快速排序**:

- 定义: 随机选择枢轴元素
- 与标准形式的区别: 消除对输入分布的依赖
- 适用场景: 输入分布未知或可能被恶意构造
- 期望时间复杂度: $O(n \log n)$，概率极高

**三数取中法**:

- 定义: 选择首、中、尾三个元素的中位数作为枢轴
- 与标准形式的区别: 更好处理已排序/逆序输入
- 适用场景: 部分有序数据

**三路快速排序 (Dutch National Flag)**:

- 定义: 将数组分为三部分：< pivot, = pivot, > pivot
- 与标准形式的区别: 高效处理大量重复元素
- 适用场景: 含大量重复键的数据

**双轴快速排序**:

- 定义: 使用两个枢轴将数组分为三部分
- 与标准形式的区别: Java 7+中Arrays.sort()的实现
- 适用场景: 原始类型数组排序

## 三、形式化证明

### 3.1 核心定理

**定理 1** (快速排序正确性): 快速排序算法对任意输入数组A[p..r]都能正确排序。

**证明**（使用数学归纳法）:

```
基例: 当r ≤ p时，子数组长度为0或1，已经有序。

归纳假设: 假设QuickSort对所有长度小于n的子数组都能正确排序。

归纳步骤:
考虑长度为n = r - p + 1的子数组A[p..r]。

1. 调用Partition(A, p, r)返回q，满足：
   - ∀i ∈ [p, q-1]: A[i] ≤ A[q]
   - ∀j ∈ [q+1, r]: A[j] ≥ A[q]

2. 递归调用QuickSort(A, p, q-1):
   - 子数组长度为q-p < n（因为至少枢轴被排除）
   - 由归纳假设，A[p..q-1]被正确排序
   - 且A[p..q-1]中所有元素 ≤ A[q]

3. 递归调用QuickSort(A, q+1, r):
   - 子数组长度为r-q < n
   - 由归纳假设，A[q+1..r]被正确排序
   - 且A[q+1..r]中所有元素 ≥ A[q]

4. 综合1-3：
   - A[p..q-1]有序且元素 ≤ A[q]
   - A[q]处于正确位置
   - A[q+1..r]有序且元素 ≥ A[q]

因此A[p..r]整体有序。∎
```

**证明要点分析**:

1. **分区不变式**: 证明Partition正确建立分区条件
2. **规模减小**: 每次递归至少排除一个元素（枢轴）
3. **组合正确性**: 分区的性质保证子问题解可组合

**直觉理解**:
分区操作如同"各就各位"——枢轴元素被放置到其最终排序位置。递归确保所有元素都找到正确位置。

### 3.2 复杂度分析证明

**定理 2** (平均时间复杂度): 随机化快速排序的期望比较次数为$O(n \log n)$。

**证明**:

```
设X为算法执行的总比较次数，需证E[X] = O(n log n)。

定义指示随机变量：
X_{ij} = I{z_i与z_j被比较}，其中z_i, z_j是第i小和第j小的元素

则X = Σ_{i=1}^{n-1} Σ_{j=i+1}^{n} X_{ij}

由期望线性性：
E[X] = Σ_{i=1}^{n-1} Σ_{j=i+1}^{n} E[X_{ij}] = Σ_{i=1}^{n-1} Σ_{j=i+1}^{n} Pr{z_i与z_j被比较}

关键观察：
两个元素被比较当且仅当其中一个在另一个之前被选为枢轴时，
它们还在同一子数组中。

考虑集合Z_{ij} = {z_i, z_{i+1}, ..., z_j}：
- z_i和z_j被比较 ⇔ 在Z_{ij}中，z_i或z_j第一个被选为枢轴
- 由于枢轴随机均匀选择，这个概率 = 2/(j-i+1)

因此：
E[X] = Σ_{i=1}^{n-1} Σ_{j=i+1}^{n} 2/(j-i+1)
     = Σ_{i=1}^{n-1} Σ_{k=1}^{n-i} 2/(k+1)  （令k=j-i）
     ≤ Σ_{i=1}^{n-1} Σ_{k=1}^{n} 2/k
     = Σ_{i=1}^{n-1} O(log n)
     = O(n log n) ∎
```

**定理 3** (最坏情况): 快速排序最坏情况时间复杂度为$\Theta(n^2)$。

**证明**:

```
最坏情况发生在每次分区极度不均衡时，
例如数组已排序且总是选择第一个/最后一个作为枢轴。

此时递归关系：
T(n) = T(n-1) + T(0) + Θ(n)
     = T(n-1) + Θ(n)

展开：
T(n) = Θ(n) + Θ(n-1) + ... + Θ(1)
     = Θ(n²) ∎
```

## 四、算法/方法详解

### 4.1 算法描述

**基础分区算法 (Lomuto分区)**:

```
算法: Lomuto-Partition(A, p, r)
输入: 数组A，索引p到r
输出: 枢轴最终位置q

1. x = A[r]              // 选择最后一个元素为枢轴
2. i = p - 1             // i指向≤枢轴区域的最后一个
3. for j = p to r-1 do
4.     if A[j] ≤ x then
5.         i = i + 1
6.         exchange A[i] with A[j]
7.     end if
8. end for
9. exchange A[i+1] with A[r]
10. return i+1
```

**Hoare分区方案**:

```
算法: Hoare-Partition(A, p, r)
输入: 数组A，索引p到r
输出: 分区边界

1. x = A[p]              // 选择第一个元素为枢轴
2. i = p - 1
3. j = r + 1
4. while True do
5.     repeat j = j - 1 until A[j] ≤ x
6.     repeat i = i + 1 until A[i] ≥ x
7.     if i < j then
8.         exchange A[i] with A[j]
9.     else
10.        return j
11.    end if
12. end while
```

**流程图**:

```
开始
  │
  ▼
选择枢轴
  │
  ▼
分区操作
  │
  ├──► 左子数组递归排序
  │
  └──► 右子数组递归排序
  │
  ▼
结束
```

### 4.2 正确性分析

**Lomuto分区不变式**:
在for循环的每次迭代开始时（第3行），以下不变式成立：

1. **A[p..i] ≤ 枢轴**（已处理的小于等于区域）
2. **A[i+1..j-1] > 枢轴**（已处理的大于区域）
3. **A[j..r-1]尚未处理**
4. **A[r] = 枢轴**

**证明**:

- **初始化**: i = p-1，A[p..i]为空，满足≤枢轴；A[i+1..j-1]也为空
- **保持**:
  - 若A[j] ≤ x，i增加后交换，A[i]变为原A[j] ≤ x，不变式1保持
  - 若A[j] > x，不交换，j-1位置的大于区域扩展，不变式2保持
- **终止**: j=r，所有非枢轴元素已处理，交换A[i+1]和A[r]完成分区

### 4.3 复杂度分析

**时间复杂度**:

| 情况 | 复杂度 | 条件 |
|------|--------|------|
| 最好 | $O(n \log n)$ | 每次分区平衡 |
| 平均 | $O(n \log n)$ | 随机输入或随机化 |
| 最坏 | $O(n^2)$ | 每次分区极度不平衡 |

**空间复杂度**:

- 额外空间: $O(\log n)$（递归栈深度）
- 最坏情况: $O(n)$（递归链长度为n）

**复杂度证明**:
平均情况递归树高度为$O(\log n)$，每层工作量$O(n)$，总计$O(n \log n)$。

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 对数组 [8, 3, 7, 4, 1, 9, 2] 进行快速排序

**问题描述**:
展示快速排序的完整执行过程，使用Lomuto分区方案。

**解决过程**:

```
初始: [8, 3, 7, 4, 1, 9, 2]

第1轮分区 (pivot=2):
  j=0: 8>2, 不交换, i=-1
  j=1: 3>2, 不交换
  j=2: 7>2, 不交换
  j=3: 4>2, 不交换
  j=4: 1≤2, i=0, swap A[0]↔A[4]: [1, 3, 7, 4, 8, 9, 2]
  j=5: 9>2, 不交换
  结束: swap A[1]↔A[6]: [1, 2, 7, 4, 8, 9, 3]
  返回 q=1

左子数组 [1] 已有序
右子数组 [7, 4, 8, 9, 3] 递归排序...

完整递归树:
[8,3,7,4,1,9,2]
    ├── [1]  (已排序)
    └── [7,4,8,9,3]
         ├── [3,4]
         │    ├── [3]
         │    └── [4]
         └── [7,8,9]
              ├── [7,8]
              └── [9]

最终结果: [1, 2, 3, 4, 7, 8, 9]
```

**结果**: [1, 2, 3, 4, 7, 8, 9]

### 5.2 代码实现

**Python实现**:

```python
"""
快速排序完整实现
包含基础版、随机化版和三路快速排序
"""

import random
from typing import List, Tuple


def quicksort_basic(arr: List[int]) -> List[int]:
    """基础快速排序（非原地，用于教学演示）"""
    if len(arr) <= 1:
        return arr

    pivot = arr[-1]
    left = [x for x in arr[:-1] if x <= pivot]
    right = [x for x in arr[:-1] if x > pivot]

    return quicksort_basic(left) + [pivot] + quicksort_basic(right)


def quicksort_inplace(arr: List[int], low: int = 0, high: int = None) -> None:
    """原地快速排序（标准实现）"""
    if high is None:
        high = len(arr) - 1

    if low < high:
        # 分区操作
        pivot_index = _partition_lomuto(arr, low, high)
        # 递归排序左右子数组
        quicksort_inplace(arr, low, pivot_index - 1)
        quicksort_inplace(arr, pivot_index + 1, high)


def _partition_lomuto(arr: List[int], low: int, high: int) -> int:
    """Lomuto分区方案"""
    pivot = arr[high]
    i = low - 1

    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]

    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1


def _partition_hoare(arr: List[int], low: int, high: int) -> int:
    """Hoare分区方案（更高效，更少交换）"""
    pivot = arr[low]
    i = low - 1
    j = high + 1

    while True:
        # 从右向左找≤pivot的元素
        j -= 1
        while arr[j] > pivot:
            j -= 1

        # 从左向右找≥pivot的元素
        i += 1
        while arr[i] < pivot:
            i += 1

        if i < j:
            arr[i], arr[j] = arr[j], arr[i]
        else:
            return j


def quicksort_randomized(arr: List[int], low: int = 0, high: int = None) -> None:
    """随机化快速排序"""
    if high is None:
        high = len(arr) - 1

    if low < high:
        # 随机选择枢轴并与末尾交换
        pivot_idx = random.randint(low, high)
        arr[pivot_idx], arr[high] = arr[high], arr[pivot_idx]

        pivot_index = _partition_lomuto(arr, low, high)
        quicksort_randomized(arr, low, pivot_index - 1)
        quicksort_randomized(arr, pivot_index + 1, high)


def quicksort_3way(arr: List[int], low: int = 0, high: int = None) -> None:
    """三路快速排序（处理大量重复元素）"""
    if high is None:
        high = len(arr) - 1

    if low >= high:
        return

    # 随机选择枢轴
    pivot_idx = random.randint(low, high)
    arr[low], arr[pivot_idx] = arr[pivot_idx], arr[low]
    pivot = arr[low]

    # lt: < pivot区域的右边界
    # gt: > pivot区域的左边界
    # i: 当前扫描位置
    lt = low
    gt = high
    i = low + 1

    while i <= gt:
        if arr[i] < pivot:
            arr[lt], arr[i] = arr[i], arr[lt]
            lt += 1
            i += 1
        elif arr[i] > pivot:
            arr[i], arr[gt] = arr[gt], arr[i]
            gt -= 1
        else:
            i += 1

    quicksort_3way(arr, low, lt - 1)
    quicksort_3way(arr, gt + 1, high)


def quickselect(arr: List[int], k: int) -> int:
    """
    快速选择算法 - 找第k小的元素（0-based）
    平均时间复杂度O(n)
    """
    if not 0 <= k < len(arr):
        raise ValueError("k out of bounds")

    arr = arr.copy()  # 不修改原数组

    def select(low: int, high: int, k: int) -> int:
        if low == high:
            return arr[low]

        # 随机化分区
        pivot_idx = random.randint(low, high)
        arr[pivot_idx], arr[high] = arr[high], arr[pivot_idx]

        pivot_final = _partition_lomuto(arr, low, high)

        if k == pivot_final:
            return arr[k]
        elif k < pivot_final:
            return select(low, pivot_final - 1, k)
        else:
            return select(pivot_final + 1, high, k)

    return select(0, len(arr) - 1, k)


# 测试代码
if __name__ == "__main__":
    # 测试各种实现
    test_cases = [
        [8, 3, 7, 4, 1, 9, 2],
        [5, 5, 5, 5, 5],  # 大量重复元素
        [1, 2, 3, 4, 5],  # 已排序
        [5, 4, 3, 2, 1],  # 逆序
        [3],              # 单元素

    ]

    print("=" * 60)
    print("快速排序测试")

    for i, test in enumerate(test_cases):
        print(f"\n测试用例 {i+1}: {test}")

        # 基础版
        sorted_basic = quicksort_basic(test)
        print(f"基础版结果: {sorted_basic}")

        # 原地版
        arr_inplace = test.copy()
        quicksort_inplace(arr_inplace)
        print(f"原地版结果: {arr_inplace}")

        # 三路快速排序（重复元素测试）
        arr_3way = test.copy()
        quicksort_3way(arr_3way)
        print(f"三路版结果: {arr_3way}")

        # 验证正确性
        assert sorted_basic == sorted(test)
        assert arr_inplace == sorted(test)

    # 测试快速选择
    print("\n" + "=" * 60)
    print("快速选择测试 (找第k小元素)")
    print("=" * 60)

    arr = [8, 3, 7, 4, 1, 9, 2]
    sorted_arr = sorted(arr)
    print(f"原数组: {arr}")
    print(f"排序后: {sorted_arr}")

    for k in range(len(arr)):
        result = quickselect(arr, k)
        print(f"第{k}小的元素: {result} (期望: {sorted_arr[k]})")
        assert result == sorted_arr[k]

    print("\n✓ 所有测试通过!")
```

```rust
/// 快速排序Rust实现
pub fn quicksort<T: Ord>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    let pivot_idx = partition(arr);
    let (left, right) = arr.split_at_mut(pivot_idx);

    quicksort(left);
    quicksort(&mut right[1..]); // 跳过枢轴
}

fn partition<T: Ord>(arr: &mut [T]) -> usize {
    let len = arr.len();
    let pivot_idx = len / 2; // 简单选择中间元素

    // 将枢轴移到末尾
    arr.swap(pivot_idx, len - 1);

    let mut i = 0;
    for j in 0..len - 1 {
        if arr[j] <= arr[len - 1] {
            arr.swap(i, j);
            i += 1;
        }
    }

    arr.swap(i, len - 1);
    i
}

/// 迭代实现（避免栈溢出）
pub fn quicksort_iterative<T: Ord>(arr: &mut [T]) {
    let mut stack = vec![(0, arr.len())];

    while let Some((low, high)) = stack.pop() {
        if high - low <= 1 {
            continue;
        }

        let pivot = partition_slice(arr, low, high);

        // 先处理较小的子数组（保证栈深度O(log n)）
        if pivot - low < high - pivot - 1 {
            stack.push((pivot + 1, high));
            stack.push((low, pivot));
        } else {
            stack.push((low, pivot));
            stack.push((pivot + 1, high));
        }
    }
}

fn partition_slice<T: Ord>(arr: &mut [T], low: usize, high: usize) -> usize {
    let pivot_idx = low + (high - low) / 2;
    arr.swap(pivot_idx, high - 1);

    let mut i = low;
    for j in low..high - 1 {
        if arr[j] <= arr[high - 1] {
            arr.swap(i, j);
            i += 1;
        }
    }

    arr.swap(i, high - 1);
    i
}
```

## 5.3 反例

### 5.3 反例

**常见错误1**: 忽略递归终止条件

```python
# 错误代码
def bad_quicksort(arr):
    pivot = arr[0]  # 如果arr为空会出错！
    left = [x for x in arr[1:] if x <= pivot]
    right = [x for x in arr[1:] if x > pivot]
    return bad_quicksort(left) + [pivot] + bad_quicksort(right)
    # 缺少基本情况处理，空数组会无限递归
```

**错误原因**: 没有处理空数组或单元素数组的情况
**正确做法**: 始终添加`if len(arr) <= 1: return arr`

**常见错误2**: 枢轴选择导致栈溢出

```python
# 对于已排序数组，总是选择第一个元素为枢轴
def worst_quicksort(arr, low=0, high=None):
    if high is None:
        high = len(arr) - 1
    if low < high:
        # 总是选择arr[low]作为枢轴
        pivot = arr[low]  # 对于已排序数组，分区极度不均衡
        # ... 导致递归深度O(n)，栈溢出
```

**错误原因**: 对于已排序输入，递归深度为n而非log n
**正确做法**: 使用随机化或三数取中法

## 六、习题

### 6.1 理解题

1. **分区不变式** [难度⭐]

   证明Lomuto分区方案中的循环不变式：在每次for循环迭代开始时，
   - A[p..i] ≤ 枢轴
   - A[i+1..j-1] > 枢轴

   <details>
   <summary>解答</summary>

   **初始化**: 在第一次迭代前，i = p-1，A[p..i]为空，满足≤枢轴；A[i+1..j-1] = A[p..p-1]也为空，满足>枢轴。

   **保持**:
   - 若A[j] ≤ 枢轴：i增加后，A[i]被设为A[j]（≤枢轴），不变式1保持
   - 若A[j] > 枢轴：不交换，j增加后A[i+1..j-1]包含原A[j]（>枢轴），不变式2保持

   **终止**: j = r时，A[p..i] ≤ 枢轴，A[i+1..r-1] > 枢轴。交换A[i+1]和A[r]后，枢轴位于正确位置。

   </details>

### 6.2 应用题

1. **找第k大元素** [难度⭐⭐]

   使用快速选择算法在给定数组中找到第k大的元素。要求平均时间复杂度$O(n)$，空间复杂度$O(\log n)$。

   <details>
   <summary>解答</summary>

   使用快速选择算法（QuickSelect）：

   ```python
   def find_kth_largest(arr, k):
       def partition(left, right, pivot_idx):
           pivot = arr[pivot_idx]
           arr[pivot_idx], arr[right] = arr[right], arr[pivot_idx]
           store_idx = left
           for i in range(left, right):
               if arr[i] > pivot:  # 找第k大，所以用>
                   arr[store_idx], arr[i] = arr[i], arr[store_idx]
                   store_idx += 1
           arr[right], arr[store_idx] = arr[store_idx], arr[right]
           return store_idx

       def select(left, right, k_smallest):
           if left == right:
               return arr[left]
           pivot_idx = random.randint(left, right)
           pivot_idx = partition(left, right, pivot_idx)
           if k_smallest == pivot_idx:
               return arr[k_smallest]
           elif k_smallest < pivot_idx:
               return select(left, pivot_idx - 1, k_smallest)
           else:
               return select(pivot_idx + 1, right, k_smallest)

       return select(0, len(arr) - 1, k - 1)  # 第k大 = 第(n-k+1)小
   ```

   </details>

### 6.3 证明题

1. **快速排序递归深度** [难度⭐⭐⭐]

   证明：使用尾递归优化的快速排序，最坏情况下递归栈深度为$O(\log n)$。

   <details>
   <summary>解答</summary>

   **尾递归优化策略**: 总是先递归处理较小的子数组，较大的子数组用尾递归（迭代）处理。

   **证明**:

   ```
   设栈深度为d(n)，T(n)为处理n个元素的最大栈深度。

   算法流程:
   1. 分区得到两个子数组，大小分别为k和n-k-1
   2. 设k ≤ n-k-1（较小的一半）
   3. 递归处理较小的（大小k），迭代处理较大的

   递归关系:
   T(n) ≤ 1 + T(k)，其中k ≤ n/2

   展开:
   T(n) ≤ 1 + T(n/2)
        ≤ 1 + 1 + T(n/4)
        ≤ ...
        ≤ log₂n + T(1)
        = O(log n)

   因此栈深度最坏为O(log n)。∎
   ```

   </details>

## 八、多维对比

### 8.1 排序算法对比

| 维度 | 快速排序 | 归并排序 | 堆排序 | 插入排序 |
|------|----------|----------|--------|----------|
| 最好时间 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ |
| 平均时间 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ |
| 最坏时间 | $O(n^2)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ |
| 空间复杂度 | $O(\log n)$ | $O(n)$ | $O(1)$ | $O(1)$ |
| 稳定性 | 不稳定 | 稳定 | 不稳定 | 稳定 |
| 缓存性能 | 优秀 | 良好 | 较差 | 优秀 |
| 适用数据规模 | 大规模 | 大规模/链表 | 大规模 | 小规模 |

### 8.2 决策建议

**何时选择快速排序**:

- 需要原地排序（内存受限）
- 数据随机分布，需要平均最优性能
- 缓存性能重要（数值计算）
- 不需要稳定性

**何时选择其他排序**:

- **归并排序**: 需要稳定性，或处理链表
- **堆排序**: 需要$O(n \log n)$最坏保证
- **插入排序**: 数据规模很小(< 20)或几乎有序
- **计数/基数排序**: 数据范围有限且为整数

**决策流程图**:

```
选择排序算法
├── 数据规模 < 20?
│   └── 是 → 插入排序
├── 需要稳定性?
│   └── 是 → 归并排序或TimSort
├── 内存极度受限?
│   └── 是 → 堆排序
├── 数据范围有限整数?
│   └── 是 → 计数/基数排序
└── 默认 → 快速排序（随机化/内省）
```

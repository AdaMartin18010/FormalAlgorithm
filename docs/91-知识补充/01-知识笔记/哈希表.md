## 一、核心概念

### 1.1 定义

**正式定义**:
哈希表是一种实现**字典**抽象数据类型(Dictionary ADT)的数据结构，支持平均$O(1)$时间的插入、删除和查找操作。

给定键空间$U$（可能非常大），哈希表使用哈希函数$h: U \to \{0, 1, \ldots, m-1\}$将键映射到大小为$m$的表中的位置。

**核心组件**:
1. **哈希函数**: $h(k) \to \{0, 1, \ldots, m-1\}$，将键映射到槽位
2. **冲突解决**: 处理不同键映射到同一槽位的情况
3. **装载因子**: $\alpha = n/m$，元素数/槽位数

**直观解释**:
哈希表就像图书馆的索引系统：
- 书名 = 键(key)
- 书架位置 = 哈希值
- 哈希函数 = 按首字母分类的规则
- 冲突 = 多本书首字母相同，需要同一书架

**关键要点**:
- 哈希表的核心是**用空间换时间**，期望O(1)操作
- 冲突不可避免（鸽巢原理：|U| > m）
- 好的哈希函数应使键均匀分布
- 装载因子决定性能，通常需要扩容

### 1.2 属性

| 属性 | 描述 | 公式/备注 |
|------|------|-----------|
| 装载因子 | 表满程度 | $\alpha = n/m$，通常保持$\alpha < 0.75$ |
| 冲突率 | 新键发生冲突的概率 | 约$\alpha$（简单均匀哈希假设） |
| 聚集 | 连续被占用的槽位 | 影响线性探测的性能 |
| 均匀性 | 键分布的均匀程度 | 好的哈希函数的关键指标 |

**性质总结**:
1. **生日悖论**: 即使$m \gg n$，冲突概率也比直觉高
2. **均匀哈希假设**: 简单分析假设哈希函数将键均匀随机映射
3. **全域哈希**: 随机选择哈希函数，保证期望性能

### 1.3 变体

**链地址法(Chaining)**:
- 定义: 每个槽位维护一个链表（或其他结构）存储冲突元素
- 与标准形式的区别: 表大小可小于元素数（$\alpha$可>1）
- 适用场景: 通用场景，实现简单

**开放寻址法(Open Addressing)**:
- 定义: 所有元素存储在表中，冲突时按探测序列找下一个空槽
- 与标准形式的区别: 必须有$\alpha < 1$，删除较复杂
- 适用场景: 内存受限，需要更好的缓存局部性

**完美哈希**:
- 定义: 保证O(1)最坏情况，无冲突
- 与标准形式的区别: 需要静态键集，两级哈希
- 适用场景: 只读数据集（如编译器关键字表）

## 三、形式化证明

### 3.1 核心定理

**定理 1** (简单均匀哈希的期望分析): 在简单均匀哈希假设下，使用链地址法的哈希表中，不成功查找的期望时间为$\Theta(1 + \alpha)$。

**证明**:
```
设表有m个槽位，存储n个元素，装载因子α = n/m。

不成功查找：键k不在表中，需要遍历h(k)对应的整个链表。

设链表T[j]的长度为n_j，则Σn_j = n。

在简单均匀哈希假设下，h(k)均匀随机选择槽位：

E[查找时间] = E[链表长度]
            = Σ(j=0 to m-1) Pr[h(k)=j] · n_j
            = Σ(j=0 to m-1) (1/m) · n_j
            = (1/m) · Σn_j
            = n/m
            = α

加上计算哈希的O(1)时间，总期望时间为Θ(1 + α)。∎
```

**证明要点分析**:
1. **均匀假设**: 每个槽位被选中的概率相等
2. **期望线性性**: 无条件期望可分解为各槽位贡献之和
3. **装载因子意义**: α直接决定了期望查找长度

**直觉理解**:
如果键均匀分布，每个链表平均有n/m个元素，遍历这些元素就是期望成本。

### 3.2 辅助引理

**引理 1** (生日问题): 在大小为m的哈希表中，插入n个随机键，至少一次冲突的概率为：
$$P(\text{冲突}) = 1 - \frac{m!}{(m-n)! \cdot m^n} \approx 1 - e^{-n^2/(2m)}$$

*直观*: 当$n \approx \sqrt{2m \ln 2} \approx 1.177\sqrt{m}$时，冲突概率超过50%。

**引理 2** (线性探测的聚集): 使用线性探测时，连续被占用的槽位形成"簇"，期望簇长度为$\Theta(1/(1-\alpha)^2)$。

## 四、算法/方法详解

### 4.1 算法描述

**链地址法实现**:
```
算法: CHAINED-HASH-INSERT(T, x)
输入: 哈希表T，元素x
1. 将x插入链表T[h(x.key)]的头部

算法: CHAINED-HASH-SEARCH(T, k)
输入: 哈希表T，键k
输出: 键为k的元素，或NIL
1. 在链表T[h(k)]中搜索键k

算法: CHAINED-HASH-DELETE(T, x)
输入: 哈希表T，元素x
1. 从链表T[h(x.key)]中删除x
```

**开放寻址法实现（线性探测）**:
```
算法: HASH-INSERT(T, k)
输入: 表T，键k
1. i = 0
2. repeat
3.     j = h(k, i)  // 探测序列: (h(k) + i) mod m
4.     if T[j] == NIL then
5.         T[j] = k
6.         return j
7.     else
8.         i = i + 1
9. until i == m
10. error "hash table overflow"

算法: HASH-SEARCH(T, k)
输入: 表T，键k
输出: 槽位j，或NIL
1. i = 0
2. repeat
3.     j = h(k, i)
4.     if T[j] == k then
5.         return j
6.     i = i + 1
7. until T[j] == NIL or i == m
8. return NIL
```

**流程图**:
```
插入键k
    │
    ▼
计算h(k)
    │
    ▼
槽位空闲？
    ├── 是 → 插入，完成
    └── 否 → 冲突！
              │
              ├── 链地址法: 加入链表
              │
              └── 开放寻址: 探测下一个槽位
                    直到找到空槽
```

### 4.2 正确性分析

**链地址法正确性**:

**不变式**: 元素x始终在链表T[h(x.key)]中。

**证明**:
- **初始化**: 插入时加入对应链表，满足
- **保持**: 查找不改变结构；删除从链表移除，仍满足位置不变式

### 4.3 复杂度分析

**链地址法**:

| 操作 | 平均情况 | 最坏情况 |
|------|----------|----------|
| 插入 | $O(1)$ | $O(n)$ |
| 删除 | $O(1)$ | $O(n)$ |
| 查找 | $O(1 + \alpha)$ | $O(n)$ |

**开放寻址法**:

| 探测方式 | 成功查找 | 不成功查找 |
|----------|----------|------------|
| 线性探测 | $\frac{1}{2}(1 + \frac{1}{1-\alpha})$ | $\frac{1}{2}(1 + \frac{1}{(1-\alpha)^2})$ |
| 双重哈希 | $\frac{1}{\alpha}\ln\frac{1}{1-\alpha}$ | $\frac{1}{1-\alpha}$ |

## 五、示例与实例

### 5.1 标准示例

**示例 1**: 简单哈希表操作

**问题描述**:
哈希表大小m=7，使用除法法h(k)=k mod 7，插入键[10, 22, 31, 4, 15, 28]。

**解决过程**:
```
h(10) = 10 mod 7 = 3
h(22) = 22 mod 7 = 1
h(31) = 31 mod 7 = 3  // 冲突！
h(4)  = 4 mod 7  = 4
h(15) = 15 mod 7 = 1  // 冲突！
h(28) = 28 mod 7 = 0

链地址法结果:
槽0: [28]
槽1: [15] → [22]  或 [22] → [15]
槽2: []
槽3: [31] → [10]  或 [10] → [31]
槽4: [4]
槽5: []
槽6: []
```

**结果**: 装载因子α = 6/7 ≈ 0.86

**示例 2**: 两数之和（哈希表优化）

**问题描述**:
给定数组和目标值，找出数组中两个数之和等于目标值。

**解决过程**:
```
使用哈希表存储已遍历的元素：
1. 初始化空哈希表
2. 遍历数组，对于当前元素x：
3.     if (target - x) in hash_table:
4.         return (target-x, x)
5.     insert x into hash_table
```

**结果**: 时间O(n)，空间O(n)，优于暴力O(n²)

### 5.2 代码实现

**语言**: Python

```python
class HashTable:
    """链地址法哈希表"""
    def __init__(self, size=10):
        self.size = size
        self.table = [[] for _ in range(size)]
        self.count = 0
    
    def _hash(self, key):
        """除法哈希"""
        return hash(key) % self.size
    
    def insert(self, key, value):
        """插入键值对"""
        index = self._hash(key)
        bucket = self.table[index]
        
        # 检查是否已存在
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # 更新
                return
        
        bucket.append((key, value))
        self.count += 1
        
        # 扩容检查
        if self.count / self.size > 0.75:
            self._resize(self.size * 2)
    
    def get(self, key):
        """查找键对应的值"""
        index = self._hash(key)
        bucket = self.table[index]
        
        for k, v in bucket:
            if k == key:
                return v
        raise KeyError(key)
    
    def delete(self, key):
        """删除键值对"""
        index = self._hash(key)
        bucket = self.table[index]
        
        for i, (k, v) in enumerate(bucket):
            if k == key:
                del bucket[i]
                self.count -= 1
                return
        raise KeyError(key)
    
    def _resize(self, new_size):
        """扩容/缩容"""
        old_table = self.table
        self.size = new_size
        self.table = [[] for _ in range(new_size)]
        self.count = 0
        
        for bucket in old_table:
            for key, value in bucket:
                self.insert(key, value)
    
    def __contains__(self, key):
        try:
            self.get(key)
            return True
        except KeyError:
            return False
    
    def __getitem__(self, key):
        return self.get(key)
    
    def __setitem__(self, key, value):
        self.insert(key, value)


# 开放寻址法（线性探测）
class OpenAddressHashTable:
    def __init__(self, size=10):
        self.size = size
        self.table = [None] * size
        self.DELETED = object()  # 标记已删除
        self.count = 0
    
    def _hash(self, key, i=0):
        """线性探测"""
        return (hash(key) + i) % self.size
    
    def insert(self, key, value):
        for i in range(self.size):
            j = self._hash(key, i)
            if self.table[j] is None or self.table[j] is self.DELETED:
                self.table[j] = (key, value)
                self.count += 1
                return
            if self.table[j][0] == key:
                self.table[j] = (key, value)  # 更新
                return
        raise Exception("Hash table is full")
    
    def get(self, key):
        for i in range(self.size):
            j = self._hash(key, i)
            if self.table[j] is None:
                raise KeyError(key)
            if self.table[j] is not self.DELETED and self.table[j][0] == key:
                return self.table[j][1]
        raise KeyError(key)


# Rust实现
'''
pub struct HashTable<K, V> {
    buckets: Vec<Vec<(K, V)>>,
    size: usize,
}

impl<K: Eq + std::hash::Hash, V> HashTable<K, V> {
    pub fn new(size: usize) -> Self {
        HashTable {
            buckets: vec![Vec::new(); size],
            size,
        }
    }
    
    fn hash(&self, key: &K) -> usize {
        use std::collections::hash_map::DefaultHasher;
        use std::hash::{Hash, Hasher};
        
        let mut hasher = DefaultHasher::new();
        key.hash(&mut hasher);
        (hasher.finish() as usize) % self.size
    }
    
    pub fn insert(&mut self, key: K, value: V) {
        let idx = self.hash(&key);
        let bucket = &mut self.buckets[idx];
        
        for (k, v) in bucket.iter_mut() {
            if *k == key {
                *v = value;
                return;
            }
        }
        bucket.push((key, value));
    }
    
    pub fn get(&self, key: &K) -> Option<&V> {
        let idx = self.hash(key);
        self.buckets[idx]
            .iter()
            .find(|(k, _)| k == key)
            .map(|(_, v)| v)
    }
}
'''
```

## 5.3 反例
### 5.3 反例

**常见错误1**: 可变对象作为键
```python
# 错误代码
hash_table = {}
key = [1, 2, 3]  # 列表是可变的！
hash_table[key] = "value"  # TypeError: unhashable type: 'list'
```
**错误原因**: 可变对象改变后哈希值变化，导致无法查找
**正确做法**: 使用元组`tuple`或不可变对象作为键

**常见错误2**: 忘记处理哈希冲突
```python
# 错误代码
class BadHashTable:
    def __init__(self, size):
        self.table = [None] * size
    
    def insert(self, key, value):
        self.table[hash(key) % len(self.table)] = value  # 直接覆盖！
```
**错误原因**: 冲突时直接覆盖旧值，导致数据丢失
**正确做法**: 使用链地址法或开放寻址法处理冲突

## 六、习题

### 6.1 理解题 (L1)

1. **装载因子影响** [难度⭐]
   
   哈希表大小为100，已存储50个元素。此时装载因子是多少？平均链长度是多少（链地址法）？
   
   <details>
   <summary>解答</summary>
   
   装载因子α = n/m = 50/100 = 0.5
   
   链地址法平均链长度 = α = 0.5
   
   </details>

2. **冲突不可避免性** [难度⭐]
   
   为什么哈希表一定会有冲突？如何避免最坏情况？
   
   <details>
   <summary>解答</summary>
   
   **冲突不可避免**（鸽巢原理）：
   键空间通常远大于表大小（|U| >> m），n个键放入m个槽，必有冲突。
   
   **避免最坏情况**：
   - 好的哈希函数减少冲突概率
   - 适当的装载因子（通常<0.75）
   - 冲突解决机制（链地址、探测）
   - 动态扩容避免过长链
   
   </details>

### 6.2 应用题 (L2-L3)

1. **LRU缓存** [难度⭐⭐]
   
   使用哈希表+双向链表实现LRU（最近最少使用）缓存。
   
   <details>
   <summary>解答</summary>
   
   ```python
   from collections import OrderedDict
   
   class LRUCache:
       def __init__(self, capacity):
           self.capacity = capacity
           self.cache = OrderedDict()
       
       def get(self, key):
           if key not in self.cache:
               return -1
           # 移动到末尾（最近使用）
           self.cache.move_to_end(key)
           return self.cache[key]
       
       def put(self, key, value):
           if key in self.cache:
               self.cache.move_to_end(key)
           self.cache[key] = value
           if len(self.cache) > self.capacity:
               # 弹出最久未使用的（开头）
               self.cache.popitem(last=False)
   ```
   
   哈希表提供O(1)查找，有序字典/链表维护使用顺序。
   
   </details>

2. **异位词分组** [难度⭐⭐]
   
   给定字符串数组，将异位词（字母相同排列不同）分组。
   
   <details>
   <summary>解答</summary>
   
   ```python
   def group_anagrams(strs):
       groups = {}
       for s in strs:
           # 排序后的字符串作为键
           key = ''.join(sorted(s))
           if key not in groups:
               groups[key] = []
           groups[key].append(s)
       return list(groups.values())
   ```
   
   时间复杂度：O(N·K·logK)，N为字符串数，K为最大字符串长度。
   
   </details>

### 6.3 证明题 (L4-L5)

1. **全域哈希的期望分析** [难度⭐⭐⭐]
   
   证明使用全域哈希函数族时，任意一对键冲突的概率为1/m。
   
   <details>
   <summary>解答</summary>
   
   **定义**：哈希函数族$\mathcal{H}$是全域的，如果对所有$x \neq y$：
   $$Pr_{h \in \mathcal{H}}[h(x) = h(y)] = \frac{1}{m}$$
   
   **证明**：
   
   对固定键x，恰好有$|\mathcal{H}|/m$个函数将x映射到任意槽j（由全域定义均匀性）。
   
   对$x \neq y$，$h(x) = h(y)$当且仅当两者映射到同一槽。
   
   对所有m个槽求和：
   $$Pr[h(x) = h(y)] = \sum_{j=0}^{m-1} Pr[h(x)=j \land h(y)=j]$$
   
   由全域性，$Pr[h(x)=j \land h(y)=j] = \frac{1}{m} \cdot \frac{1}{m} \cdot m = \frac{1}{m^2}$（独立均匀选择）
   
   实际上标准定义直接要求$Pr[h(x)=h(y)] = 1/m$。
   
   因此期望链长度 = 1 + (n-1)/m ≈ 1 + α。∎
   
   </details>

### 7.2 实际案例

**案例**: Python字典实现

**背景**:
Python的dict是世界上最广泛使用的哈希表实现之一。

**应用方式**:
- 开放寻址法+伪随机探测
- 优化过的哈希函数（针对字符串优化）
- 动态扩容（装载因子>2/3时扩容）

**效果**:
- 平均O(1)操作
- 缓存局部性良好
- 节省内存（相比链地址法）

### 7.3 跨领域联系

**与密码学的联系**:
加密哈希（SHA-256）与数据结构哈希不同，追求计算不可逆性和抗碰撞性。

**与分布式系统的联系**:
一致性哈希解决分布式缓存中的节点动态增删问题。

## 八、多维对比

### 8.1 方法对比

| 方法 | 平均查找 | 最坏查找 | 空间 | 删除 | 适用场景 |
|------|----------|----------|------|------|----------|
| 链地址法 | O(1) | O(n) | O(n+m) | 简单 | 通用 |
| 线性探测 | O(1) | O(n) | O(m) | 需标记 | 内存受限 |
| 二次探测 | O(1) | O(n) | O(m) | 需标记 | 减少聚集 |
| 双重哈希 | O(1) | O(n) | O(m) | 需标记 | 均匀分布 |
| 完美哈希 | O(1) | O(1) | O(n) | - | 静态数据集 |

### 8.2 决策建议

**何时选择链地址法**:
- 通用场景
- 装载因子可能>1
- 实现简单优先

**何时选择开放寻址法**:
- 内存受限
- 装载因子<0.5能保证
- 需要更好的缓存性能

**哈希函数选择**:
- 整数键：除法法、乘法法
- 字符串：多项式滚动哈希
- 安全场景：加密哈希

**决策流程图**:
```
数据是否静态？
├── 是 → 完美哈希（确定性O(1)）
└── 否 → 装载因子预期多少？
          ├── > 0.75 → 链地址法
          └── < 0.5 → 开放寻址法
                        聚集敏感？
                        ├── 是 → 双重哈希
                        └── 否 → 线性探测
```

## 一、核心概念

### 1.1 定义

**正式定义**:
**流网络**是一个有向图$G=(V, E)$，其中：

- 每条边$(u,v) \in E$有**容量**$c(u,v) \geq 0$
- 有**源点**$s \in V$和**汇点**$t \in V$，$s \neq t$
- **流**是函数$f: V \times V \to \mathbb{R}$，满足：
  1. **容量约束**: $0 \leq f(u,v) \leq c(u,v)$
  2. **流量守恒**: 对$v \notin \{s,t\}$，$\sum_u f(u,v) = \sum_w f(v,w)$

**流值**: $|f| = \sum_v f(s,v) - \sum_v f(v,s)$

**最大流问题**: 找到从$s$到$t$的最大流值。

**残差网络**: 对于流$f$，定义残差容量：
$$c_f(u,v) = \begin{cases} c(u,v) - f(u,v) & \text{if } (u,v) \in E \\ f(v,u) & \text{if } (v,u) \in E \\ 0 & \text{otherwise} \end{cases}$$

**增广路径**: 残差网络$G_f$中从$s$到$t$的路径。

**s-t割**: 将$V$划分为$S$和$T=V-S$，其中$s \in S$，$t \in T$。
割的容量：$c(S,T) = \sum_{u \in S, v \in T} c(u,v)$

**直观解释**:
网络流模型将图视为管道网络，边是管道，容量是管道的最大流量，流是实际传输的流量。最大流问题类似于问：从水源到目的地，最多能输送多少水？残差网络允许我们"撤销"之前的流量分配，这是找到最优解的关键。

### 1.2 属性

| 属性 | 描述 | 备注 |
|------|------|------|
| 最大流最小割定理 | 最大流值 = 最小割容量 | 核心对偶定理 |
| 整数性定理 | 整数容量必有整数最大流 | 实际应用重要 |
| 流分解 | 任何流可分解为$\leq |E|$条路径流 | 分析工具 |
| 唯一性 | 最大流不唯一，最大流值唯一 | 构造反例时注意 |

**性质总结**:

1. **弱对偶性**: 任意流值 $\leq$ 任意割容量
2. **强对偶性**: 最大流值 = 最小割容量
3. **整数性**: 若容量为整数，则存在整数最大流

### 1.3 变体

**多源多汇网络**:

- 定义: 多个源点和多个汇点
- 转化: 添加超级源点$s'$和超级汇点$t'$，边容量$\infty$

**节点容量限制**:

- 定义: 节点$v$有通过容量$c(v)$
- 转化: 将$v$拆分为$v_{in}$和$v_{out}$，中间边容量$c(v)$

**最小费用流**:

- 定义: 边有单位费用，求流值为$F$的最小费用流
- 算法: 连续最短路、消圈算法、网络单纯形

**上下界网络流**:

- 定义: 边有下界$l(u,v)$和上界$c(u,v)$
- 转化: 转化为无下界问题求解

## 三、形式化证明

### 3.1 核心定理

**定理 1** (最大流最小割定理): 对于任何流网络，最大流的值等于最小割的容量。

**证明**:

```
第一部分：弱对偶性
对于任意流f和任意s-t割(S,T)：
    |f| = ∑_{v} f(s,v) - ∑_{v} f(v,s)
        = ∑_{u∈S} [∑_{v} f(u,v) - ∑_{v} f(v,u)]  （流量守恒）
        = ∑_{u∈S} ∑_{v∈T} f(u,v) - ∑_{u∈S} ∑_{v∈T} f(v,u)
        ≤ ∑_{u∈S} ∑_{v∈T} c(u,v) = c(S,T)

因此 |f| ≤ c(S,T)，即最大流 ≤ 最小割。

第二部分：强对偶性
设f*是Ford-Fulkerson得到的流，则G_f*中无增广路。

定义 S = {v | G_f*中存在s到v的路径}
定义 T = V - S

则s ∈ S，t ∈ T（否则存在增广路），(S,T)是一个割。

对于任意u ∈ S，v ∈ T：
- 若(u,v) ∈ E，则f*(u,v) = c(u,v)（否则v ∈ S）
- 若(v,u) ∈ E，则f*(v,u) = 0（否则v ∈ S）

因此：
    |f*| = ∑_{u∈S} ∑_{v∈T} f*(u,v) - ∑_{u∈S} ∑_{v∈T} f*(v,u)
         = ∑_{u∈S} ∑_{v∈T} c(u,v) = c(S,T)

即流值等于割容量。由弱对偶性，(S,T)是最小割，|f*|是最大流。∎
```

**证明要点分析**:

1. **流量分解**: 将流值表示为跨割的流量差
2. **残差网络**: 无增广路时构造最小割
3. **紧性**: 流饱和了割的所有边

### 3.2 辅助引理

**引理 2** (整数性): 若所有容量为整数，则存在整数最大流。

*证明*:
Ford-Fulkerson每次增广整数流量，维护整数流。由于最大流存在，整数最大流存在。∎

**引理 3** (Edmonds-Karp复杂度): Edmonds-Karp算法的时间复杂度为$O(VE^2)$。

*证明*:

- 每次增广最短路径
- 关键边（在残差网络中从满到不满或从空到非空）最多被$O(V)$次作为关键边
- 总共$O(VE)$次增广，每次BFS $O(E)$
- 总时间$O(VE^2)$。∎

## 四、算法/方法详解

### 4.1 Dinic算法

**算法思想**:

1. 构建分层图（BFS确定每个节点到s的距离）
2. 在分层图上找阻塞流（无法再增广的流）
3. 重复直到汇点不可达

**伪代码**:

```
算法: DINIC(G, s, t)
输入: 流网络G，源点s，汇点t
输出: 最大流值

1.  flow = 0
2.  while BFS构建分层图成功 do
3.      初始化当前弧指针cur[u]为head[u]
4.      while (pushed = DFS(s, INF)) > 0 do
5.          flow += pushed
6.  return flow

DFS(u, pushed):
1.  if u == t then return pushed
2.  for v in adj[u] starting from cur[u] do
3.      if level[v] == level[u] + 1 and c_f(u,v) > 0 then
4.          tr = DFS(v, min(pushed, c_f(u,v)))
5.          if tr > 0 then
6.              更新残差容量
7.              return tr
8.      cur[u] = next edge
9.  return 0
```

**流程图**:

```
构建分层图(BFS)
    ↓
是否存在到t的路径?
    ├── 否 → 返回最大流
    └── 是 → 找阻塞流(DFS)
              ↓
        增广直到无法增广
              ↓
        返回构建分层图
```

### 4.2 正确性分析

**不变式**: 每次分层后，到汇点的最短距离严格增加。

**证明**:

- **初始化**: 初始分层正确
- **保持**: 阻塞流饱和了某层到下一层的所有边，下次BFS距离必增加
- **终止**: 最多$O(V)$次分层，算法终止

### 4.3 复杂度分析

**时间复杂度**:

- 最坏情况: $O(V^2 E)$（一般图）
- 单位容量: $O(\min(V^{2/3}, E^{1/2}) \cdot E)$
- 二分图匹配: $O(E \sqrt{V})$

**空间复杂度**: $O(V + E)$

**复杂度证明**:

```
Dinic复杂度分析：

1. 分层次数：每次分层后s-t距离至少+1，最多O(V)次

2. 每次分层的DFS复杂度：
   - 使用当前弧优化，每条边最多被遍历一次
   - 总复杂度O(E)每层

3. 总复杂度：O(V²E)

单位容量图的优化分析：
- 经过O(√E)次分层后，剩余流可以被限制
- 使用阻塞流性质，得到O(E√V)或O(V^{2/3}E)
```

# Python: Dinic算法完整实现
from typing import List, Optional
from collections import deque

class Edge:
    """残差网络中的边"""
    def __init__(self, to: int, rev: int, cap: int):
        self.to = to          # 目标节点
        self.rev = rev        # 反向边在目标节点邻接表中的索引
        self.cap = cap        # 残差容量

class Dinic:
    """
    Dinic最大流算法实现
    时间复杂度: O(V²E)一般图, O(E√V)单位容量
    """

    def __init__(self, n: int):
        self.n = n
        self.graph = [[] for _ in range(n)]

    def add_edge(self, fr: int, to: int, cap: int):
        """添加有向边及其反向边"""
        forward = Edge(to, len(self.graph[to]), cap)
        backward = Edge(fr, len(self.graph[fr]), 0)
        self.graph[fr].append(forward)
        self.graph[to].append(backward)

    def bfs(self, s: int, t: int) -> bool:
        """构建分层图"""
        self.level = [-1] * self.n
        self.level[s] = 0
        queue = deque([s])

        while queue:
            u = queue.popleft()
            for e in self.graph[u]:
                if e.cap > 0 and self.level[e.to] < 0:
                    self.level[e.to] = self.level[u] + 1
                    queue.append(e.to)

        return self.level[t] >= 0

    def dfs(self, u: int, t: int, f: int) -> int:
        """在分层图上找增广路（当前弧优化）"""
        if u == t:
            return f

        for i in range(self.iter[u], len(self.graph[u])):
            e = self.graph[u][i]
            if e.cap > 0 and self.level[u] < self.level[e.to]:
                d = self.dfs(e.to, t, min(f, e.cap))
                if d > 0:
                    e.cap -= d
                    self.graph[e.to][e.rev].cap += d
                    return d
            self.iter[u] += 1

        return 0

    def max_flow(self, s: int, t: int) -> int:
        """计算从s到t的最大流"""
        flow = 0
        INF = float('inf')

        while self.bfs(s, t):
            self.iter = [0] * self.n
            while True:
                f = self.dfs(s, t, INF)
                if f == 0:
                    break
                flow += f

        return flow

    def min_cut(self, s: int) -> List[int]:
        """返回最小割的S集合（在残差网络中从s可达的节点）"""
        visited = [False] * self.n
        stack = [s]
        visited[s] = True

        while stack:
            u = stack.pop()
            for e in self.graph[u]:
                if e.cap > 0 and not visited[e.to]:
                    visited[e.to] = True
                    stack.append(e.to)

        return [i for i in range(self.n) if visited[i]]


# 最小费用最大流（连续最短路算法）
import heapq

class MinCostMaxFlow:
    """最小费用最大流 - SPFA/ dijkstra + 势函数优化"""

    def __init__(self, n: int):
        self.n = n
        self.graph = [[] for _ in range(n)]

    def add_edge(self, fr: int, to: int, cap: int, cost: int):
        """添加边，cost是单位流量费用"""
        forward = [to, cap, cost, None]  # [to, cap, cost, rev]
        backward = [fr, 0, -cost, None]
        forward[3] = len(self.graph[to])
        backward[3] = len(self.graph[fr])
        self.graph[fr].append(forward)
        self.graph[to].append(backward)

    def flow(self, s: int, t: int, maxf: int = float('inf')):
        """返回 (最大流, 最小费用) """
        n = self.n
        res = 0
        h = [0] * n  # 势函数
        prevv = [0] * n
        preve = [0] * n

        INF = float('inf')
        flow = 0

        while flow < maxf:
            # Dijkstra找最短路
            dist = [INF] * n
            dist[s] = 0
            pq = [(0, s)]

            while pq:
                d, v = heapq.heappop(pq)
                if dist[v] < d:
                    continue
                for i, e in enumerate(self.graph[v]):
                    if e[1] > 0 and dist[e[0]] > dist[v] + e[2] + h[v] - h[e[0]]:
                        dist[e[0]] = dist[v] + e[2] + h[v] - h[e[0]]
                        prevv[e[0]] = v
                        preve[e[0]] = i
                        heapq.heappush(pq, (dist[e[0]], e[0]))

            if dist[t] == INF:
                break

            for v in range(n):
                h[v] += dist[v] if dist[v] < INF else 0

            # 沿最短路增广
            d = maxf - flow
            v = t
            while v != s:
                d = min(d, self.graph[prevv[v]][preve[v]][1])
                v = prevv[v]

            flow += d
            res += d * h[t]

            v = t
            while v != s:
                e = self.graph[prevv[v]][preve[v]]
                e[1] -= d
                self.graph[v][e[3]][1] += d
                v = prevv[v]

        return flow, res

// Rust: 高性能Dinic实现
pub struct Dinic {
    n: usize,
    graph: Vec<Vec<Edge>>,
    level: Vec<i32>,
    iter: Vec<usize>,
}

struct Edge {
    to: usize,
    rev: usize,
    cap: i64,
}

impl Dinic {
    pub fn new(n: usize) -> Self {
        Dinic {
            n,
            graph: vec![Vec::new(); n],
            level: vec![0; n],
            iter: vec![0; n],
        }
    }

    pub fn add_edge(&mut self, fr: usize, to: usize, cap: i64) {
        let forward = Edge { to, rev: self.graph[to].len(), cap };
        let backward = Edge { to: fr, rev: self.graph[fr].len(), cap: 0 };
        self.graph[fr].push(forward);
        self.graph[to].push(backward);
    }

    fn bfs(&mut self, s: usize, t: usize) -> bool {
        self.level.fill(-1);
        self.level[s] = 0;
        let mut queue = std::collections::VecDeque::new();
        queue.push_back(s);

        while let Some(u) = queue.pop_front() {
            for e in &self.graph[u] {
                if e.cap > 0 && self.level[e.to] < 0 {
                    self.level[e.to] = self.level[u] + 1;
                    queue.push_back(e.to);
                }
            }
        }

        self.level[t] >= 0
    }

    fn dfs(&mut self, u: usize, t: usize, f: i64) -> i64 {
        if u == t {
            return f;
        }

        let i = self.iter[u];
        for j in i..self.graph[u].len() {
            let e = &self.graph[u][j];
            if e.cap > 0 && self.level[u] < self.level[e.to] {
                let d = self.dfs(e.to, t, f.min(e.cap));
                if d > 0 {
                    let to = e.to;
                    let rev = e.rev;
                    self.graph[u][j].cap -= d;
                    self.graph[to][rev].cap += d;
                    return d;
                }
            }
            self.iter[u] = j + 1;
        }

        0
    }

    pub fn max_flow(&mut self, s: usize, t: usize) -> i64 {
        let mut flow = 0;
        const INF: i64 = i64::MAX;

        while self.bfs(s, t) {
            self.iter.fill(0);
            loop {
                let f = self.dfs(s, t, INF);
                if f == 0 {
                    break;
                }
                flow += f;
            }
        }

        flow
    }
}

## 5.3 反例

### 5.3 反例

**常见错误1**: 忘记添加反向边

```python
# 错误：只添加正向边
graph[fr].append(Edge(to, cap))

# 正确：同时添加反向边
forward = Edge(to, len(graph[to]), cap)
backward = Edge(fr, len(graph[fr]), 0)
graph[fr].append(forward)
graph[to].append(backward)
```

**错误原因**: 残差网络需要反向边来允许流量撤销
**正确做法**: 同时添加容量为0的反向边

**常见错误2**: DFS中不更新当前弧指针

```python
# 错误：每次从头遍历
for e in graph[u]:  # 重复检查已饱和的边

# 正确：使用当前弧优化
for i in range(iter[u], len(graph[u])):
    iter[u] = i  # 记录当前检查位置
```

**错误原因**: 导致重复检查，复杂度退化
**正确做法**: 使用当前弧优化记录已检查的边

## 六、习题

### 6.1 理解题 (L1)

1. **残差网络的作用** [难度⭐]

   解释为什么需要残差网络。

   <details>
   <summary>解答</summary>

   残差网络允许"撤销"之前的流量分配。通过反向边，算法可以调整之前的决策，这是达到最优解的关键。

   形式化：若流f不是最大流，则残差网络中存在增广路径，可以增加流值。

   </details>

2. **最大流最小割直观理解** [难度⭐]

   用"管道网络"的比喻解释最大流最小割定理。

   <details>
   <summary>解答</summary>

   想象从水源s到目的地t的管道网络。最大流是从s能输送到t的最大水量。

   最小割是切断某些管道使s和t断开的最小代价。显然，最大输送量不可能超过任何割断方案的容量。

   定理说明：存在一个割，其容量恰好等于最大输送量。

   </details>

### 6.2 应用题 (L2-L3)

1. **二分图最大匹配** [难度⭐⭐]

   如何将二分图最大匹配转化为最大流问题？

   <details>
   <summary>解答</summary>

   构造：
   1. 添加超级源点s，连接左部所有点，容量1
   2. 添加超级汇点t，右部所有点连接t，容量1
   3. 原图中的边从左指向右，容量1

   最大流值 = 最大匹配数

   原因：
   - 容量1保证每个点最多匹配一次
   - 流从s到左部到右部到t，对应一个匹配

   </details>

2. **网络连通性** [难度⭐⭐⭐]

   给定无向图，求从s到t的边不相交路径的最大数量。

   <details>
   <summary>解答</summary>

   转化为最大流问题：
   1. 将每条无向边$(u,v)$替换为两条有向边$(u,v)$和$(v,u)$，容量1
   2. 求s到t的最大流

   由整数性定理，最大流对应边不相交路径的最大数量（Menger定理）。

   时间复杂度：$O(V^{1/2}E)$使用Dinic在单位容量图上。

   </details>

### 6.3 证明题 (L4-L5)

1. **Dinic复杂度证明** [难度⭐⭐⭐⭐]

   证明Dinic算法在单位容量图上的复杂度为$O(E \sqrt{V})$。

   <details>
   <summary>解答</summary>

   关键引理：经过$O(\sqrt{E})$个阶段后，剩余流的值小于$\sqrt{E}$。

   证明：
   - 设当前距离为d，则s-t最短路径长度至少为d
   - 残差网络中s-t路径数最多为$E/d$
   - 每阶段为阻塞流，至少饱和一条边
   - 经过$O(\sqrt{E})$阶段后，剩余流$< \sqrt{E}$
   - 再用$O(\sqrt{E})$次增广完成

   总时间：$O((\sqrt{E} + \sqrt{E}) \cdot E) = O(E\sqrt{E}) = O(E\sqrt{V})$（对于简单图$E \leq V^2$）

   </details>

2. **Gale-Ryser定理证明** [难度⭐⭐⭐⭐⭐]

   证明二分图度序列可实现的充要条件（用网络流）。

   <details>
   <summary>解答</summary>

   构造流网络：
   - 左部n个点，度数序列$r_1 \geq \ldots \geq r_n$
   - 右部m个点，度数序列$c_1 \geq \ldots \geq c_m$
   - s连接左部点i，容量$r_i$
   - 右部点j连接t，容量$c_j$
   - 所有左-右边容量1

   度序列可实现当且仅当存在流值为$\sum r_i = \sum c_j$的可行流。

   由最大流最小割定理，等价于对所有$X \subseteq$左部，$Y \subseteq$右部：
   $$\sum_{i \in X} r_i \leq |X| \cdot |Y| + \sum_{j \notin Y} c_j$$

   简化得Gale-Ryser条件。

   </details>

### 7.2 实际案例

**案例**: 图像分割（Graph Cut）

**背景**:
图像分割需要将像素划分为前景和背景。GrabCut等算法使用最小割。

**应用方式**:

- 每个像素是一个节点
- 相邻像素之间有边（鼓励相似像素同标签）
- 源点代表前景，汇点代表背景
- 像素到s/t的边表示属于前景/背景的代价

**效果**:

- 最小割对应最优分割
- 能量函数最小化
- 广泛应用在图像编辑软件中

### 7.3 跨领域联系

**与线性规划**:
最大流是LP的特殊形式（全单模矩阵），因此有整数最优解。

**与组合优化**:
拟阵交、次模函数最小化都可以归约到网络流。

**与机器学习**:
图割用于MRF推断，是计算机视觉的基础算法。

## 八、多维对比

### 8.1 方法对比

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|------------|------------|----------|
| Ford-Fulkerson | $O(E \cdot |f^*|)$ | $O(V+E)$ | 整数容量，小流量 |
| Edmonds-Karp | $O(VE^2)$ | $O(V+E)$ | 通用，简单实现 |
| Dinic | $O(V^2E)$ | $O(V+E)$ | 稠密图，二分图匹配 |
| Push-Relabel | $O(V^3)$或$O(V^2\sqrt{E})$ | $O(V^2)$ | 实践中通常最快 |
| MPM | $O(V^3)$ | $O(V^2)$ | 理论上有保证 |

### 8.2 决策建议

**选择Edmonds-Karp当**:

- 需要简单实现
- 对最坏情况有要求
- 教学或原型验证

**选择Dinic当**:

- 二分图匹配问题
- 单位容量图
- 需要较好的理论保证

**选择Push-Relabel当**:

- 追求实际运行速度
- 可以承受较高空间开销
- 稠密图

**决策流程图**:

```
图类型：
    ↓
二分图匹配？
    ├── 是 → Dinic（O(E√V)）
    └── 否 → 容量是否为单位？
              ├── 是 → Dinic或Push-Relabel
              └── 否 → 追求速度？
                        ├── 是 → Push-Relabel
                        └── 否 → Dinic
```

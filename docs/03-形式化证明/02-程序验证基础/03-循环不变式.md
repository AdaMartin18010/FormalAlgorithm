# 循环不变式 / Loop Invariants

> **摘要**：循环不变式（Loop Invariant）是程序验证中连接霍尔逻辑与实用算法正确性证明的核心工具。本文从形式化定义出发，系统阐述不变式的三要素（初始化、保持、终止），提供面向LeetCode算法面试的构造启发式与典型案例分析，并建立从自然语言不变式到Lean 4形式化的映射路径。

---

## 术语 / Terminology

| 术语 | 英文 | 说明 |
|------|------|------|
| 循环不变式 | Loop Invariant | 在循环的每次迭代前后均保持为真的断言 |
| 初始化 | Initialization | 循环开始前不变式成立 |
| 保持 | Maintenance | 若某次迭代前不变式成立，则迭代后仍成立 |
| 终止 | Termination | 循环终止时，不变式与循环条件的否定共同蕴含后条件 |
| 秩函数 | Ranking Function | 用于证明循环终止的良基度量 |
| 部分正确性 | Partial Correctness | 若程序终止，则结果正确 |
| 完全正确性 | Total Correctness | 程序必然终止，且结果正确 |

## 一、形式化定义 / Formal Definition

### 定义 3.1（循环不变式）

设 $P$ 为循环的前置条件，$Q$ 为后置条件，循环形式为：

$$\mathbf{while}\ B\ \mathbf{do}\ S\ \mathbf{end}$$

一个断言 $I$ 称为该循环的**循环不变式**，当且仅当满足以下三个条件：

**1. 初始化（Initialization）**：
$$\{P\}\ \Rightarrow\ \{I\}$$
即循环开始前，前置条件蕴含不变式成立。

**2. 保持（Maintenance）**：
$$\{I \land B\}\ S\ \{I\}$$
即若某次迭代开始时 $I$ 与循环条件 $B$ 同时成立，执行循环体 $S$ 后 $I$ 仍然成立。

**3. 终止（Termination）**：
$$I \land \neg B\ \Rightarrow\ Q$$
即当循环终止时（$\neg B$ 成立），不变式与循环条件否定共同蕴含后置条件 $Q$。

---

### 定理 3.1（循环不变式与部分正确性）

若断言 $I$ 满足上述三要素，则霍尔三元组：

$$\{P\}\ \mathbf{while}\ B\ \mathbf{do}\ S\ \mathbf{end}\ \{Q\}$$

在**部分正确性**意义下成立。

**证明概要**：
由初始化，$I$ 在循环开始前成立。由保持性，若第 $k$ 次迭代前 $I$ 成立，则第 $k+1$ 次迭代前 $I$ 仍成立。根据数学归纳法，$I$ 在每次迭代前均成立。当循环终止时，$\neg B$ 成立，结合终止性条件即得 $Q$。$\square$

---

### 定理 3.2（完全正确性的充分条件）

若 $I$ 是循环不变式，且存在**秩函数**（Ranking Function）$r: \text{State} \to \mathbb{N}$ 满足：
- (R1) $I \land B \Rightarrow r \geq 0$
- (R2) $\{I \land B \land r = t\}\ S\ \{r < t\}$

则循环在**完全正确性**意义下终止。

## 二、不变式的三要素 / The Three Properties

### 2.1 初始化：从零开始

不变式必须在进入循环前成立。这通常意味着不变式描述的是"空范围"或"初始状态"下的正确性质。

**直觉**：如果算法维护"已处理前缀的正确性"，那么在处理0个元素时，这个性质必须平凡成立。

### 2.2 保持：归纳的核心

保持性是不变式证明中最关键的步骤，本质上是一次**数学归纳法**：
- **归纳基**：初始化已证 $I$ 在第0次迭代前成立
- **归纳步**：假设第 $k$ 次迭代前 $I$ 成立，证明执行 $S$ 后 $I$ 仍成立

### 2.3 终止：导出答案

循环终止时，我们已知：
- $I$ 成立（由保持性）
- $\neg B$ 成立（循环终止的定义）

两者的合取必须足够强，能够推出我们想要的后置条件 $Q$。

## 三、构造启发式 / Construction Heuristics

循环不变式没有通用的自动生成算法（这是程序验证的核心难点之一），但存在行之有效的构造启发式：

### 启发式 1：从后置条件中"删除"循环变量

**策略**：将后置条件 $Q$ 中对循环计数器 $i$ 的依赖弱化，使其在循环过程中也成立。

**示例**：求数组和的后置条件是 $\text{sum} = \sum_{j=0}^{n-1} a[j]$。
不变式可构造为：$\text{sum} = \sum_{j=0}^{i-1} a[j]$，其中 $i$ 是当前循环索引。

### 启发式 2：范围归纳（Prefix/Suffix Invariant）

**策略**：维护"已处理范围满足某性质，未处理范围保持另一性质"。

**典型模式**：
- **前缀不变式**：$\forall j \in [0, i).\ P(a[j])$ — 已处理的所有元素满足性质 $P$
- **后缀不变式**：$\forall j \in (i, n].\ Q(a[j])$ — 未处理的所有元素满足性质 $Q$

### 启发式 3：边界缩小法（用于二分查找等）

**策略**：维护搜索空间的边界 $[lo, hi]$，并保证"若目标存在，则一定在当前边界内"。

**典型不变式**：$\text{target} \notin (-\infty, lo) \land \text{target} \notin (hi, +\infty)$

### 启发式 4：双指针的分离不变式

**策略**：两个指针 $left$ 和 $right$ 从两端向中间移动，分别维护各自方向的性质。

**典型模式**：$\forall j \leq left.\ P(j) \land \forall j \geq right.\ Q(j)$

## 四、LeetCode语境案例集 / LeetCode Case Studies

### 案例 1：线性扫描 —— Two Sum 暴力解法

**题目**：LeetCode 1. Two Sum

**算法**：遍历数组，对每个元素检查其补数是否在已遍历部分。

**循环不变式**：
> $I \triangleq \forall j \in [0, i).\ a[j] + a[k] \neq \text{target}$ 对于所有 $k < j$ 的已检查组合

简化版："已遍历的所有元素对中，不存在和为target的组合"

**三要素验证**：
- **初始化**：$i = 0$，范围为空，性质平凡成立
- **保持**：第 $i$ 次迭代检查了所有 $(i, j)$ 对（$j < i$），若均未找到解，则范围扩大后性质仍成立
- **终止**：若找到解则立即返回；若循环结束仍未找到，则所有可能组合均已检查，返回无解

---

### 案例 2：二分查找 —— 边界缩小不变式

**题目**：LeetCode 704. Binary Search

**算法**（左闭右开版本）：
```rust
let mut lo = 0;
let mut hi = nums.len();
while lo < hi {
    let mid = lo + (hi - lo) / 2;
    if nums[mid] == target { return mid as i32; }
    else if nums[mid] < target { lo = mid + 1; }
    else { hi = mid; }
}
return -1;
```

**循环不变式**：
> $I \triangleq (\text{target} \notin \text{nums}[0..\text{lo})) \land (\text{target} \notin \text{nums}[\text{hi}..n))$

**文字表述**："若target存在于数组中，则它一定在$[lo, hi)$范围内"

**三要素验证**：
- **初始化**：$lo = 0, hi = n$，$[0, n)$ 覆盖整个数组，性质成立
- **保持**：
  - 若 $nums[mid] < target$，则 $target$ 不可能在 $[0, mid]$ 中，设 $lo = mid + 1$ 后性质仍成立
  - 若 $nums[mid] > target$，则 $target$ 不可能在 $[mid, n)$ 中，设 $hi = mid$ 后性质仍成立
  - 若 $nums[mid] = target$，直接返回，无需保持
- **终止**：当 $lo \geq hi$ 时，搜索范围为空，结合不变式可知 $target$ 不在数组中

```mermaid

    A["初始化: lo=0, hi=n<br/>I: target ∈ [lo, hi)"] --> B["计算 mid"]
    B --> C{"nums[mid] vs target"}

```

### 案例 3：滑动窗口 —— 最长无重复子串

**题目**：LeetCode 3. Longest Substring Without Repeating Characters

**算法**：双指针维护窗口，右指针扩展，左指针在发现重复时收缩。

**循环不变式**（双指针分离模式）：
> $I \triangleq (\text{window} = \text{nums}[left..right)) \land (\text{window 中无重复字符}) \land (\text{result} = \max_{j \leq right} \text{合法窗口长度})$

**三要素验证**：
- **初始化**：$left = 0, right = 0$，窗口为空，无重复，性质成立
- **保持**：
  - 右指针右移：若新字符不在窗口中，窗口仍无重复，更新result
  - 若新字符在窗口中：左指针右移直到重复消除，窗口恢复无重复状态
- **终止**：$right = n$，所有以任意位置结尾的最大无重复窗口均已考察，result即为全局最优

## 五、常见错误与反模式 / Common Mistakes

### 错误 1：过强的不变式

**症状**：试图让不变式同时描述太多信息，导致保持性无法证明。

**示例**：在二分查找中要求"$lo$ 指向小于target的最大元素，$hi$ 指向大于等于target的最小元素"。这个不变式在初始化时不成立（$lo=0$ 可能并不小于target）。

**修复**：弱化不变式为"若target存在，则在$[lo, hi)$内"。

### 错误 2：过弱的不变式

**症状**：不变式在终止时无法推出后置条件。

**示例**：仅维护"$lo \leq hi$"。循环终止时无法得出任何关于target位置的结论。

### 错误 3：忘记边界情况

**症状**：不变式在空数组、单元素数组、全等数组等边界情况下不成立。

**修复**：初始化验证时，显式检查空范围是否满足不变式。

### 错误 4：将循环条件与不变式混淆

**症状**：把"$i < n$"当作不变式的一部分。

**澄清**：循环条件是控制流机制，不变式是语义性质。两者应分离：不变式描述"什么为真"，循环条件描述"何时继续"。

## 六、到Lean 4的映射 / Lean 4 Formalization

### 6.1 自然语言不变式 → Lean `invariant` tactic

在Lean 4中，循环不变式可以通过`while`的组合子配合`invariant`关键字表达：

```lean
-- 伪代码示意（Lean 4 while循环的组合子表达）
while b do
  invariant I  -- 声明循环不变式
  body
```

### 6.2 二分查找不变式的Lean表达

```lean
def binary_search_invariant (nums : Array Int) (target : Int) (lo hi : Nat) : Prop :=
  -- 若target存在于数组中，则一定在[lo, hi)范围内
  (∀ i, nums[i]! = target → lo ≤ i ∧ i < hi) ∨
  -- 或者target不存在，此时lo ≥ hi
  (lo ≥ hi)
```

### 6.3 从不变式导出证明义务

使用Lean的`wp`（最弱前置条件）策略时，循环会产生三个证明义务，恰好对应不变式三要素：

```lean
-- 证明义务1：初始化
⊢ P → I

-- 证明义务2：保持
⊢ I ∧ B → wp(body, I)

-- 证明义务3：终止
⊢ I ∧ ¬B → Q
```

## 七、总结 / Summary

循环不变式是连接**程序直觉**与**形式化证明**的桥梁：

1. **三要素**（初始化、保持、终止）是不变式的充要条件
2. **构造启发式**（删除循环变量、范围归纳、边界缩小、双指针分离）覆盖了LeetCode绝大多数算法模式
3. **LeetCode面试语境**下，清晰表述循环不变式是区分"会写代码"与"理解代码"的关键标志
4. **到Lean的映射**表明，自然语言的不变式可以直接转化为机器可检验的证明义务

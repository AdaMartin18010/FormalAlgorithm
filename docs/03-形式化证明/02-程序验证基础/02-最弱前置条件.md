# 最弱前置条件 / Weakest Preconditions

> **摘要**：最弱前置条件（Weakest Precondition, wp）是由Dijkstra提出的谓词转换器语义核心概念，它通过从后条件向前的反向推理，为程序验证提供了系统化的数学基础。本文从形式化定义出发，建立wp的公理化体系，展示其与霍尔逻辑的等价性，并给出面向LeetCode算法的推导示例。

| 最弱前置条件 | Weakest Precondition | 使程序满足给定后条件的最宽松前置条件 |
| 谓词转换器 | Predicate Transformer | 将后条件（谓词）映射到前置条件的函数 |

| 最强后条件 | Strongest Postcondition | 给定前置条件和程序，能推出的最强断言 |

## 一、形式化定义 / Formal Definition

### 定义 2.1（最弱前置条件）

给定程序语句 $S$ 和后条件谓词 $Q$，$S$ 关于 $Q$ 的**最弱前置条件**记为 $\text{wp}(S, Q)$，是满足以下两个性质的最弱断言：

**1. 充分性（Sufficiency）**：
$$\{\text{wp}(S, Q)\}\ S\ \{Q\}$$
即若 $\text{wp}(S, Q)$ 成立，执行 $S$ 后 $Q$ 必然成立。

**2. 最弱性（Weakest）**：
$$\forall P.\ (\{P\}\ S\ \{Q\}) \Rightarrow (P \Rightarrow \text{wp}(S, Q))$$
即任何其他能保证 $S$ 执行后 $Q$ 成立的前置条件 $P$，都蕴含 $\text{wp}(S, Q)$。

---

### 定理 2.1（wp与霍尔逻辑的等价性）

霍尔三元组 $\{P\}\ S\ \{Q\}$ 有效，当且仅当：

$$P \Rightarrow \text{wp}(S, Q)$$

**证明**：

- ($\Rightarrow$)：若 $\{P\}\ S\ \{Q\}$ 有效，由wp的最弱性，$P \Rightarrow \text{wp}(S, Q)$。
- ($\Leftarrow$)：若 $P \Rightarrow \text{wp}(S, Q)$，由wp的充分性，$\{\text{wp}(S, Q)\}\ S\ \{Q\}$。根据霍尔逻辑的推论规则（强化前置条件），$\{P\}\ S\ \{Q\}$ 有效。$\square$

## 二、wp的公理化定义 / Axiomatic Definition

### 2.1 基本命令的wp

**赋值语句（Assignment）**：
$$\text{wp}(x := e, Q) = Q[x \mapsto e]$$
即把 $Q$ 中所有自由出现的 $x$ 替换为表达式 $e$。

**顺序组合（Sequential Composition）**：
$$\text{wp}(S_1; S_2, Q) = \text{wp}(S_1, \text{wp}(S_2, Q))$$

**条件语句（Conditional）**：
$$\text{wp}(\mathbf{if}\ B\ \mathbf{then}\ S_1\ \mathbf{else}\ S_2, Q) = (B \Rightarrow \text{wp}(S_1, Q)) \land (\neg B \Rightarrow \text{wp}(S_2, Q))$$

**循环语句（While Loop）**：
$$\text{wp}(\mathbf{while}\ B\ \mathbf{do}\ S, Q) = \exists k.\ H_k$$
其中 $H_0 = \neg B \Rightarrow Q$，$H_{k+1} = H_k \lor (B \land \text{wp}(S, H_k))$。

实际证明中，通常使用**循环不变式**代替直接的wp展开。

## 三、Dijkstra的守卫命令语言 / Guarded Commands

Dijkstra在《A Discipline of Programming》中提出了守卫命令语言（GCL），其wp定义如下：

**跳过（Skip）**：
$$\text{wp}(\mathbf{skip}, Q) = Q$$

**中止（Abort）**：
$$\text{wp}(\mathbf{abort}, Q) = \mathbf{false}$$

**守卫选择（Guarded Alternative）**：
$$\text{wp}(\mathbf{if}\ B_1 \to S_1 \parallel B_2 \to S_2 \parallel \dots \parallel B_n \to S_n\ \mathbf{fi}, Q) = \bigwedge_{i=1}^{n} (B_i \Rightarrow \text{wp}(S_i, Q)) \land \bigvee_{i=1}^{n} B_i$$

**守卫迭代（Guarded Iteration）**：
$$\text{wp}(\mathbf{do}\ B_1 \to S_1 \parallel \dots \parallel B_n \to S_n\ \mathbf{od}, Q) = \exists k.\ H_k$$

## 四、LeetCode语境示例 / LeetCode Examples

### 示例 1：数组元素交换的正确性

**程序**：

```
temp := a[i];
a[i] := a[j];
a[j] := temp;
```

**后条件**：$Q \triangleq a[i] = a_{old}[j] \land a[j] = a_{old}[i] \land \forall k \notin \{i, j\}.\ a[k] = a_{old}[k]$

**wp推导**（从后向前）：

$$\begin{aligned}
\text{wp}(a[j] := \text{temp}, Q) &= Q[a[j] \mapsto \text{temp}] \\
&= a[i] = a_{old}[j] \land \text{temp} = a_{old}[i] \land \dots \\
\text{wp}(a[i] := a[j], \dots) &= a[j] = a_{old}[j] \land \text{temp} = a_{old}[i] \land \dots \\
\text{wp}(\text{temp} := a[i], \dots) &= \text{true} \land \text{true} \land \dots = \mathbf{true}
\end{aligned}$$

结论：交换程序在**任何前置条件**下都满足后置条件（只要索引 $i, j$ 在范围内）。

### 示例 2：累加和程序

**程序**：
```
sum := 0;
i := 0;
while i < n do
  sum := sum + a[i];
  i := i + 1;
od
```

**后条件**：$Q \triangleq \text{sum} = \sum_{j=0}^{n-1} a[j]$

使用不变式 $I \triangleq \text{sum} = \sum_{j=0}^{i-1} a[j] \land 0 \leq i \leq n$ 作为wp的近似，可证明程序正确性。

## 五、wp与最强后条件的对偶 / Duality with Strongest Postcondition

| 维度 | 最弱前置条件 wp | 最强后条件 sp |
|------|----------------|---------------|
| 推理方向 | 后向（Backward） | 前向（Forward） |
| 输出 | 最宽松的前置条件 | 最强的后置条件 |
| 公式 | $\text{wp}(S, Q)$ | $\text{sp}(P, S)$ |
| 关系 | $P \Rightarrow \text{wp}(S, Q) \iff \text{sp}(P, S) \Rightarrow Q$ | |

## 六、总结 / Summary

最弱前置条件提供了程序验证的**系统性框架**：

1. **反向推理**：从目标（后条件）出发，逐步推导所需的前提条件
2. **模块化**：复杂程序的wp可分解为简单命令wp的组合
3. **与霍尔逻辑等价**：$\{P\}\ S\ \{Q\} \iff P \Rightarrow \text{wp}(S, Q)$
4. **实用价值**：在编译器优化、静态分析工具（如Facebook的Infer）中有广泛应用

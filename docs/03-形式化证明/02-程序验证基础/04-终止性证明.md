# 终止性证明 / Termination Proofs

> **摘要**：终止性证明是程序完全正确性的核心组成部分。本文系统阐述良基关系、秩函数、结构递归等终止性证明方法，提供LeetCode典型算法（for循环、二分查找、快速排序）的终止性分析，并给出Lean 4中`termination_by`策略的使用方法。

## 一、形式化定义 / Formal Definition

### 定义 4.1（良基关系）

集合 $W$ 上的二元关系 $\prec$ 称为**良基的**（Well-Founded），当且仅当 $W$ 中不存在无限的严格下降链：

$$\neg \exists (w_0, w_1, w_2, \dots).\ \forall i.\ w_{i+1} \prec w_i$$

**等价表述**：$W$ 的每个非空子集都有关于 $\prec$ 的极小元。

**典型良基集**：

- $(\mathbb{N}, <)$ — 自然数上的标准序
- $(\text{String}, \text{proper-prefix})$ — 字符串的真前缀关系
- $(\text{Tree}, \text{subtree})$ — 树的真子树关系

---

### 定义 4.2（秩函数）

给定循环 $\mathbf{while}\ B\ \mathbf{do}\ S$，函数 $r: \text{State} \to W$ 称为该循环的**秩函数**，如果满足：

**R1（有界性）**：$\forall s.\ (B(s) \Rightarrow r(s) \in W)$

**R2（递减性）**：$\forall s, s'.\ (B(s) \land \langle S, s \rangle \to s') \Rightarrow r(s') \prec r(s)$

---

### 定理 4.1（秩函数蕴含终止性）

若循环存在秩函数 $r: \text{State} \to (W, \prec)$，且 $(W, \prec)$ 良基，则该循环必然终止。

**证明概要**（反证法）：
假设循环不终止，则存在无限执行序列 $s_0 \to s_1 \to s_2 \to \dots$。
由R2，$r(s_0) \succ r(s_1) \succ r(s_2) \succ \dots$ 构成无限严格下降链，与 $(W, \prec)$ 良基矛盾。$\square$

## 二、常见终止性证明模式 / Proof Patterns

### 模式 1：计数器递减（for循环）

**结构**：

```
for i := 0 to n-1 do S od
```

**秩函数**：$r = n - i$

- 初始 $r = n \geq 0$
- 每次迭代 $i$ 增加1，$r$ 严格递减
- 当 $r = 0$ 时循环终止

---

### 模式 2：区间缩小（二分查找）

**结构**：

```
while lo < hi do
  mid := (lo + hi) / 2;
  if ... then lo := mid + 1 else hi := mid;
od
```

**秩函数**：$r = hi - lo$

- 初始 $r = n > 0$
- 若 $lo := mid + 1$，则新 $r' = hi - (mid + 1) < hi - lo = r$（因 $mid \geq lo$）
- 若 $hi := mid$，则新 $r' = mid - lo < hi - lo = r$（因 $mid < hi$）
- 每次至少减少1，最多 $n$ 次迭代后 $r = 0$，循环终止

---

### 模式 3：子问题规模递减（快速排序递归）

**结构**：

```
quicksort(A, lo, hi):
  if lo < hi then
    p := partition(A, lo, hi);
    quicksort(A, lo, p-1);
    quicksort(A, p+1, hi);
```

**秩函数**：$r = hi - lo$

- 分区后，$p \in [lo, hi]$，两个子问题规模均小于原问题
- 递归深度受限于 $n$，每层总工作量 $O(n)$
- 注意：最坏情况下 $r$ 每次仅减1，导致 $O(n^2)$ 时间，但仍终止

---

### 模式 4：结构递归（递归数据结构）

**结构**：对树/列表的子结构进行递归。

**秩函数**：数据结构的大小或高度。

**示例**：链表求和

```
sum(list):
  if list == null then return 0;
  else return list.val + sum(list.next);
```

**秩函数**：$r = \text{list.length}$

- 每次递归 $r$ 减1
- 良基集 $(\mathbb{N}, <)$ 保证终止

---

## 三、LeetCode语境示例 / LeetCode Examples

### 示例 1：线性扫描的终止性

**程序**：遍历数组找target

```rust
for i in 0..nums.len() {
    if nums[i] == target { return i as i32; }
}
return -1;
```

**终止性论证**：

- 秩函数 $r = n - i$，其中 $n = \text{nums.len}()$
- $i$ 从0开始，每次严格递增
- $r$ 为非负整数且严格递减
- 最多 $n$ 次迭代后 $r = 0$，循环终止

---

### 示例 2：二分查找的终止性

**程序**：

```rust
while lo < hi {
    let mid = lo + (hi - lo) / 2;
    if nums[mid] == target { return mid as i32; }
    else if nums[mid] < target { lo = mid + 1; }
    else { hi = mid; }
}
```

**终止性论证**：

- 秩函数 $r = hi - lo$
- 初始 $r = n \geq 0$
- **情况1**（$lo = mid + 1$）：$mid = lo + \lfloor r/2 \rfloor \geq lo$，故 $mid + 1 > lo$，新 $r' = hi - (mid+1) < hi - lo = r$
- **情况2**（$hi = mid$）：$mid = lo + \lfloor r/2 \rfloor < lo + r = hi$（因 $r \geq 1$ 且 $B = lo < hi$），故新 $r' = mid - lo < r$
- 每次至少减1，最多 $n$ 次迭代终止

---

### 示例 3：不终止的反面案例

**程序**：

```rust
let mut i = 0;
while i < n {
    // 忘记 i += 1
}
```

**终止性分析**：

- 秩函数 $r = n - i$ 不递减（$i$ 不变）
- R2条件不满足
- 若 $n > 0$，循环永不终止

## 四、Lean 4中的终止性表达 / Termination in Lean 4

### 4.1 `partial` vs `termination_by`

Lean 4默认要求所有递归函数必须可证明终止：

```lean
-- 结构递归：自动识别
 def sum_list (lst : List Nat) : Nat :=
  match lst with
  | [] => 0
  | x :: xs => x + sum_list xs  -- Lean自动识别为结构递归

-- 非结构递归：需要显式证明
 def binary_search (nums : Array Int) (target : Int) (lo hi : Nat) : Int :=
  if h : lo < hi then
    let mid := (lo + hi) / 2;
    if nums[mid]! = target then mid
    else if nums[mid]! < target then
      binary_search nums target (mid + 1) hi
    else
      binary_search nums target lo mid
  else -1
  termination_by hi - lo  -- 显式声明秩函数
  decreasing_by
    simp_wf
    -- 证明 hi - lo 严格递减
    omega
```

### 4.2 `decreasing_by` 策略

当Lean无法自动证明终止时，使用 `decreasing_by` 子句：

```lean
def foo (n : Nat) : Nat :=
  if n > 0 then
    foo (n - 1)
  else 0
  termination_by n
  decreasing_by
    simp_wf
    omega  -- 自动证明 n-1 < n
```

## 五、总结 / Summary

终止性证明的要点：

1. **良基关系是核心**：$(\mathbb{N}, <)$ 覆盖了绝大多数算法场景
2. **秩函数是桥梁**：将程序状态映射到良基集，证明每次迭代严格递减
3. **LeetCode常见模式**：计数器递减、区间缩小、子问题规模递减
4. **Lean 4支持**：`termination_by` 声明秩函数，`decreasing_by` 提供证明

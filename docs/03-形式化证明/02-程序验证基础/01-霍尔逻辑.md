## 02.01 霍尔逻辑 / Hoare Logic

### 摘要 / Executive Summary

本文档系统阐述**霍尔逻辑**（Hoare Logic）——命令式程序验证的基石公理系统。内容涵盖霍尔三元组的语义解释、完整推理规则表、各规则的可靠性证明思路，以及 Cook 相对完备性定理。与纯理论阐述不同，本文档在每个核心规则后配备**LeetCode 语境下的最小可理解示例**（如变量交换、累加和），使算法工程师能将霍尔逻辑直接应用于日常代码的正确性推理。

## 1. 霍尔三元组语义

**定义 1.0.1** (霍尔三元组 [Hoare1969])

设 $P$ 和 $Q$ 为关于程序状态的谓词（断言），$C$ 为命令（程序语句），则**霍尔三元组**（Hoare Triple）记为：

$$\{P\} \; C \; \{Q\}$$

其直观含义为：若程序执行前状态满足前置条件 $P$，且程序 $C$ 成功终止，则终止后的状态满足后置条件 $Q$。

### 1.1 部分正确性

**定义 1.1.1** (部分正确性 / Partial Correctness)

霍尔三元组 $\{P\} C \{Q\}$ 在部分正确性意义下有效，记作 $\models_{\text{par}} \{P\} C \{Q\}$，当且仅当对所有状态 $\sigma$：

$$\sigma \models P \;\; \Rightarrow \;\; \big( \mathcal{M}\llbracket C \rrbracket(\sigma) \downarrow \; \Rightarrow \; \mathcal{M}\llbracket C \rrbracket(\sigma) \models Q \big)$$

其中 $\mathcal{M}\llbracket C \rrbracket$ 为命令 $C$ 的指称语义（状态到状态的偏函数），$\downarrow$ 表示有定义（即终止）。

> **关键点**：部分正确性**不保证**程序一定终止，仅保证"若终止，则后置条件成立"。在分析 LeetCode 题解时，部分正确性常用于先验证"结果若返回则必正确"，再单独验证终止性。

### 1.2 完全正确性

**定义 1.2.1** (完全正确性 / Total Correctness)

完全正确性要求程序不仅满足部分正确性，而且**必定终止**。通常用方括号记法：

$$[P] \; C \; [Q]$$

形式化定义为：

$$\models_{\text{tot}} [P] C [Q] \;\; \equiv \;\; \forall \sigma. \; \sigma \models P \Rightarrow \big( \mathcal{M}\llbracket C \rrbracket(\sigma) \downarrow \; \land \; \mathcal{M}\llbracket C \rrbracket(\sigma) \models Q \big)$$

等价地：

$$\models_{\text{tot}} [P] C [Q] \;\; \equiv \;\; \models_{\text{par}} \{P\} C \{Q\} \; \land \; \text{Term}(C, P)$$

其中 $\text{Term}(C, P)$ 表示：在所有满足 $P$ 的输入状态下，$C$ 均终止。

## 2. 核心推理规则

以下给出针对命令式语言 IMP（算术表达式、布尔表达式、赋值、顺序组合、条件、循环）的完整霍尔逻辑公理系统 [Hoare1969, AptOlderog2019]。记 $\vdash \{P\} C \{Q\}$ 表示该三元组在公理系统中可推导。

### 2.1 空语句与赋值

| 规则名称 | 规则形式 | 说明 |
|----------|----------|------|
| **空语句 (Skip)** | $$\overline{\{P\} \; \mathbf{skip} \; \{P\}}$$ | `skip` 不改变状态。 |
| **赋值公理 (Assignment)** | $$\overline{\{Q[e/x]\} \; x := e \; \{Q\}}$$ | 后置条件 $Q$ 中所有自由出现的 $x$ 被表达式 $e$ 替换。 |

**示例 2.1.1** (LeetCode 语境：初始化计数器)

在 LeetCode 题解中，常见语句 `sum := 0`。若后置条件为 $\{sum = 0\}$，则前置条件为将 $0$ 代入 $sum$ 后的结果，即恒真 $\{\text{true}\}$：

$$\overline{\{\text{true}\} \; sum := 0 \; \{sum = 0\}}$$

### 2.2 顺序组合

**顺序组合规则 (Sequence)**：

$$\frac{\{P\} \; C_1 \; \{R\} \quad \{R\} \; C_2 \; \{Q\}}{\{P\} \; C_1; C_2 \; \{Q\}}$$

中间断言 $R$ 作为桥梁连接两个命令。

**示例 2.2.1** (LeetCode 语境：先赋值后累加)

证明 `sum := 0; sum := sum + a[i]` 满足 $\{sum = a[i]\}$：

1. 由赋值公理：$\{\text{true}\} \; sum := 0 \; \{sum = 0\}$
2. 由赋值公理：$\{sum = 0\} \; sum := sum + a[i] \; \{sum = a[i]\}$
   - 注意：$Q[e/x]$ 此处 $Q = (sum = a[i])$，$e = sum + a[i]$，替换后得 $(sum + a[i] = a[i])$，即 $sum = 0$。
3. 由顺序组合，得：$\{\text{true}\} \; sum := 0; sum := sum + a[i] \; \{sum = a[i]\}$。

### 2.3 条件规则

**条件规则 (Conditional)**：

$$\frac{\{P \land B\} \; C_1 \; \{Q\} \quad \{P \land \neg B\} \; C_2 \; \{Q\}}{\{P\} \; \mathbf{if} \; B \; \mathbf{then} \; C_1 \; \mathbf{else} \; C_2 \; \{Q\}}$$

两个分支的后置条件必须一致。

**示例 2.3.1** (LeetCode 语境：取最大值)

证明 `if x > y then max := x else max := y` 满足 $\{max = \max(x, y)\}$：

- **then 分支**：前置条件 $\{x > y\}$，执行 `max := x`。由赋值公理，后置条件为 $\{x = \max(x, y)\}$。由于 $x > y$，此断言成立。
- **else 分支**：前置条件 $\{\neg(x > y)\} = \{x \leq y\}$，执行 `max := y`。后置条件为 $\{y = \max(x, y)\}$。由于 $x \leq y$，此断言成立。
- 由条件规则，得证。

### 2.4 循环规则

**循环规则 (While Loop)** [Hoare1969]：

$$\frac{\{I \land B\} \; C \; \{I\}}{\{I\} \; \mathbf{while} \; B \; \mathbf{do} \; C \; \{I \land \neg B\}}$$

其中 $I$ 为**循环不变式**（Loop Invariant）。该规则是霍尔逻辑中最具实践价值的规则，也是 LeetCode 迭代算法正确性证明的核心工具。

> 循环不变式的三性质（将在 `03-循环不变式.md` 中系统展开）：
>
> 1. **初始化**（Initialization）：进入循环前 $I$ 成立；
> 2. **保持**（Maintenance）：若 $I \land B$ 成立，执行 $C$ 后 $I$ 仍成立；
> 3. **终止**（Termination）：当 $I \land \neg B$ 成立时，可推出期望的后置条件。

### 2.5 推论规则

**推论规则 (Rule of Consequence)**：

$$\frac{P \Rightarrow P' \quad \{P'\} \; C \; \{Q'\} \quad Q' \Rightarrow Q}{\{P\} \; C \; \{Q\}}$$

该规则允许在执行核心推导前**强化前置条件**（strengthen precondition）或**弱化后置条件**（weaken postcondition），是实际验证中不可或缺的技术。

**示例 2.5.1** (LeetCode 语境：数组边界)

已知 `arr[i] := x` 在索引有效时满足某后置条件。若前置条件包含 $0 \leq i < n$，而规则要求的前置条件是 $i \geq 0 \land i < \text{len}(arr)$，由于 $0 \leq i < n \Rightarrow i \geq 0 \land i < \text{len}(arr)$（当 $n = \text{len}(arr)$ 时），可由推论规则应用该赋值公理。

## 3. 规则可靠性证明

**定理 3.0.1** (霍尔逻辑的可靠性 / Soundness)

若 $\vdash \{P\} C \{Q\}$，则 $\models_{\text{par}} \{P\} C \{Q\}$。即，公理系统中可推导出的所有霍尔三元组在操作语义下都是部分正确的。

**证明策略**：对推导树的高度进行结构归纳法。只需验证每条公理和推理规则在操作语义下保持有效性。

### 3.1 赋值公理的可靠性

**引理 3.1.1**：赋值公理 $\{Q[e/x]\} \; x := e \; \{Q\}$ 是可靠的。

**证明**：设状态 $\sigma$ 满足 $Q[e/x]$，即 $\sigma \models Q[e/x]$。执行 $x := e$ 后，新状态为 $\sigma' = \sigma[x \mapsto \mathcal{A}\llbracket e \rrbracket \sigma]$。根据替换引理（Substitution Lemma），$\sigma \models Q[e/x]$ 当且仅当 $\sigma[x \mapsto \mathcal{A}\llbracket e \rrbracket \sigma] \models Q$。因此 $\sigma' \models Q$。∎

### 3.2 循环规则的可靠性

**引理 3.2.1** (While 规则)：若 $\models_{\text{par}} \{I \land B\} C \{I\}$，则 $\models_{\text{par}} \{I\} \mathbf{while} \; B \; \mathbf{do} \; C \; \{I \land \neg B\}$。

**证明**：对操作语义推导树的高度（即循环迭代次数）进行归纳。

- **基例**（0 次迭代）：若 $\sigma \models I$ 且 $\sigma \models \neg B$，则循环立即终止，终止状态显然满足 $I \land \neg B$。
- **归纳步**：假设对 $k$ 次迭代成立。考虑 $k+1$ 次迭代：由于 $\sigma \models I \land B$，执行循环体 $C$ 后得到状态 $\sigma_1$。由前提，$\sigma_1 \models I$。从 $\sigma_1$ 开始的剩余 $k$ 次迭代由归纳假设保证终止于满足 $I \land \neg B$ 的状态。∎

## 4. Cook 相对完备性

**定理 4.0.1** (Cook 相对完备性 [Cook1978])

设 $\mathcal{L}$ 为一阶逻辑语言，$T$ 为关于程序状态的充分表达理论（adequately expressive theory）。对于 IMP 程序 $C$ 和断言 $P, Q \in \mathcal{L}$：

$$\models_{\text{par}} \{P\} C \{Q\} \quad \Rightarrow \quad T \vdash \{P\} C \{Q\}$$

换言之，若一阶理论 $T$ 足够强（例如包含皮亚诺算术 PA），则所有语义上有效的部分正确性三元组在霍尔逻辑中都是可证明的（相对于 $T$ 中一阶定理的可证明性）。

**证明概要**：核心在于构造**最弱前置条件**（Weakest Precondition, wp）与**最强后置条件**（Strongest Postcondition, sp），并证明它们可以在充分表达的理论 $T$ 中被一阶公式定义。详见 `02-最弱前置条件.md` §4。

**推论** (哥德尔不完备性的影响 [Gödel1931])：由于 while 循环的不变式可能涉及任意复杂的算术性质，且 PA 本身不完备，因此**不存在机械化的、总能够找到正确循环不变式的算法**。程序验证中循环不变式的构造本质上需要人类洞察力（或强大的自动推理启发式）。这也是 LeetCode 题解中"正确性证明"仍具价值的根本原因——算法工程师的直觉与形式化训练缺一不可。

## 5. LeetCode 语境示例集

本节提供三个从简单到复杂的示例，展示如何将霍尔逻辑规则组合使用，验证 LeetCode 常见代码模式。

### 5.1 示例：交换两个变量

**场景**：LeetCode 数组题中经常需要交换 `arr[i]` 与 `arr[j]`。

**代码片段**：

```
t := x;
x := y;
y := t;
```

**目标**：证明 $\{x = a \land y = b\} \; C \; \{x = b \land y = a\}$。

**霍尔逻辑推导**：

1. `t := x`：前置条件 $\{x = a \land y = b\}$，后置条件 $\{t = a \land x = a \land y = b\}$（由赋值公理）。
2. `x := y`：前置条件 $\{t = a \land x = a \land y = b\}$，后置条件 $\{t = a \land x = b \land y = b\}$。
3. `y := t`：前置条件 $\{t = a \land x = b \land y = b\}$，后置条件 $\{t = a \land x = b \land y = a\}$。
4. 由顺序组合（两次应用），将三个赋值组合起来。
5. 由推论规则，弱化后置条件：$\{t = a \land x = b \land y = a\} \Rightarrow \{x = b \land y = a\}$。

因此：

$$\vdash \{x = a \land y = b\} \; t := x; x := y; y := t \; \{x = b \land y = a\}$$

### 5.2 示例：累加和计算

**场景**：LeetCode 中统计数组元素和（如 53. Maximum Subarray 的前置子问题）。

**代码片段**：

```
sum := 0;
i := 0;
while i < n do
    sum := sum + a[i];
    i := i + 1;
end
```

**目标**：证明后置条件 $\{sum = \sum_{k=0}^{n-1} a[k]\}$。

**循环不变式**：$I \;\equiv\; sum = \sum_{k=0}^{i-1} a[k] \;\land\; 0 \leq i \leq n$。

**验证三性质**：

| 性质 | 验证 |
|------|------|
| **初始化** | 进入循环前 $sum = 0, i = 0$，故 $\sum_{k=0}^{-1} a[k] = 0$，$I$ 成立。 |
| **保持** | 假设 $I \land (i < n)$ 成立。执行 `sum := sum + a[i]; i := i + 1` 后，新的 $sum' = \sum_{k=0}^{i-1} a[k] + a[i] = \sum_{k=0}^{i} a[k]$，新的 $i' = i + 1$，故 $I$ 对 $(sum', i')$ 成立。 |
| **终止** | 循环结束时 $I \land \neg(i < n) \equiv I \land i \geq n$。结合 $i \leq n$，得 $i = n$，故 $sum = \sum_{k=0}^{n-1} a[k]$。 |

由循环规则与顺序组合，得证。

### 5.3 示例：二分查找的循环不变式框架

**场景**：LeetCode 704. Binary Search。

**代码片段（左闭右开区间版）**：

```
lo := 0;
hi := n;
while lo < hi do
    mid := lo + (hi - lo) / 2;
    if a[mid] < target then
        lo := mid + 1;
    else
        hi := mid;
    end
end
// result: lo (or hi)
```

**循环不变式**：$I \;\equiv\; 0 \leq lo \leq hi \leq n \;\land\; \big( \text{if } \exists k. a[k] = target \text{ then } lo \leq k < hi \big)$。

**在霍尔逻辑中的角色**：

- 初始化：$lo = 0, hi = n$，显然 $0 \leq 0 \leq n \leq n$，且若目标存在则必在 $[0, n)$ 内。
- 保持：分两种分支，由条件规则分别验证。`if` 分支排除 $mid$ 左侧（因 $a[mid] < target$），`else` 分支收缩右边界至 $mid$。
- 终止：$lo = hi$，结合不变式中"目标若在则必在 $[lo, hi)$ 内"，当区间为空时推出目标不存在于数组中，或 $lo$ 即目标位置。

> **完整证明**：见 `docs/03-形式化证明/04-算法正确性证明案例/02-LeetCode-704-Binary-Search.md`。

1. **[Hoare1969]** **Hoare, C. A. R.** An Axiomatic Basis for Computer Programming. *Communications of the ACM*, 12(10), 1969, 576–580. DOI: 10.1145/363235.363259.

   - 提出程序流程图上的断言方法，是循环不变式与变式函数思想的先驱。本文档 §2.4 参考此论文。

3. **[Cook1978]** **Cook, S. A.** Soundness and Completeness of an Axiom System for Program Verification. *SIAM Journal on Computing*, 7(1), 1978, 70–90. DOI: 10.1137/0207005.
   - 证明霍尔逻辑相对于充分表达一阶理论的相对完备性定理。本文档 §4 的核心内容基于此论文。

## 学习目标

完成本章节后，读者将能够：

1. 解释霍尔三元组 $\{P\}C\{Q\}$ 的部分正确性与完全正确性语义区别；
2. 使用赋值公理、顺序组合、条件规则和循环规则手动推导简单命令式程序的霍尔逻辑证明；
3. 为 LeetCode 常见代码模式（累加、交换、二分查找框架）写出循环不变式并验证其三性质；
4. 理解 Cook 相对完备性定理的含义，以及为何循环不变式构造无法完全自动化；
5. 将自然语言的"代码正确性直觉"转化为形式化的霍尔逻辑推导树。

# 案例 03：三数之和 / Case 03：3Sum

> **LeetCode**: [15. 3Sum](https://leetcode.com/problems/3sum/)
> **难度**: Medium
> **范式**: Two Pointers / 双指针

---

## 一、题目规约 / Problem Specification

### 1.1 自然语言描述

给你一个整数数组 `nums`，判断是否存在三元组 $[\text{nums}[i], \text{nums}[j], \text{nums}[k]]$ 满足：
- $i \neq j$，$i \neq k$，$j \neq k$
- $\text{nums}[i] + \text{nums}[j] + \text{nums}[k] = 0$

返回所有和为 0 且不重复的三元组。

### 1.2 形式化规约

**前置条件** $P$：
$$\text{nums} \in \text{Array}(\mathbb{Z}) \land 3 \leq |\text{nums}| \leq 3000$$

**后置条件** $Q$：
$$\text{result} = \{(a, b, c) \mid \exists i,j,k.\ \text{nums}[i]=a \land \text{nums}[j]=b \land \text{nums}[k]=c \land a+b+c=0 \land |\{i,j,k\}|=3\}$$

**去重约束**：结果中不包含重复的三元组（值相等即视为重复）。

## 二、算法描述 / Algorithm Description

### 2.1 排序 + 双指针

```rust
use std::cmp::Ordering;

pub fn three_sum(mut nums: Vec<i32>) -> Vec<Vec<i32>> {
    let mut result = Vec::new();
    nums.sort_unstable();
    let n = nums.len();
    
    for i in 0..n {
        if i > 0 && nums[i] == nums[i-1] { continue; }  // 去重
        let target = -nums[i];
        let mut left = i + 1;
        let mut right = n - 1;
        
        while left < right {
            match (nums[left] + nums[right]).cmp(&target) {
                Ordering::Equal => {
                    result.push(vec![nums[i], nums[left], nums[right]]);
                    while left < right && nums[left] == nums[left+1] { left += 1; }
                    while left < right && nums[right] == nums[right-1] { right -= 1; }
                    left += 1;
                    right -= 1;
                }
                Ordering::Less => left += 1,
                Ordering::Greater => right -= 1,
            }
        }
    }
    result
}
```

## 三、正确性证明 / Correctness Proof

### 3.1 外层循环不变式

固定 $i$ 后，问题转化为 Two Sum：在 $\text{nums}[i+1..n)$ 中找和为 $-\text{nums}[i]$ 的两个数。

> $I_{outer}(i) \triangleq \text{result 中已包含所有以} \leq i \text{为第一个元素的不重复三元组}$

### 3.2 内层双指针不变式

对于固定的 $i$，设搜索区间为 $[\text{left}, \text{right}]$：

> $I_{inner} \triangleq \text{所有满足 } j \in [i+1, \text{left}), k \in (\text{right}, n) \text{ 的合法对都已被检查}$

**等价表述**：若存在解 $(j, k)$ 满足 $j < k$ 且 $\text{nums}[j] + \text{nums}[k] = -\text{nums}[i]$，则 $j \in [\text{left}, \text{right}]$。

### 3.3 初始化

- 外层：$i = 0$，尚未处理任何三元组，空真成立
- 内层：$\text{left} = i+1, \text{right} = n-1$，覆盖所有可能的 $(j, k)$ 对

### 3.4 保持

假设内层迭代前 $I_{inner}$ 成立，考虑 $\text{sum} = \text{nums}[\text{left}] + \text{nums}[\text{right}]$：

**情况A**：$\text{sum} = \text{target}$

找到一组解。去重后收缩两端：$\text{left}++, \text{right}--$。所有包含当前 $\text{left}$ 或 $\text{right}$ 的可能组合已检查，保持成立。

**情况B**：$\text{sum} < \text{target}$

由于数组升序，对于固定的 $\text{right}$，所有 $j \leq \text{left}$ 都有 $\text{nums}[j] + \text{nums}[\text{right}] \leq \text{sum} < \text{target}$。

因此不存在以当前 $\text{right}$ 和任何 $\leq \text{left}$ 的 $j$ 组成的解。$\text{left}++$ 后保持成立。

**情况C**：$\text{sum} > \text{target}$

对称地，对于固定的 $\text{left}$，所有 $k \geq \text{right}$ 都有 $\text{nums}[\text{left}] + \text{nums}[k] \geq \text{sum} > \text{target}$。

因此不存在以当前 $\text{left}$ 和任何 $\geq \text{right}$ 的 $k$ 组成的解。$\text{right}--$ 后保持成立。

### 3.5 终止

内层循环终止时 $\text{left} \geq \text{right}$，搜索区间为空。由 $I_{inner}$，所有可能的 $(j, k)$ 对均已检查。

外层循环遍历所有可能的 $i$，最终所有不重复三元组均被收集。

## 四、复杂度分析 / Complexity Analysis

**时间复杂度 O(n²)**：
- 排序：$O(n \log n)$
- 外层循环 $n$ 次，内层双指针每次 $O(n)$
- 总时间：$O(n \log n + n^2) = O(n^2)$

**空间复杂度 O(1)**（不计输出）：
- 排序可能是 $O(\log n)$（递归栈）或 $O(1)$（非递归）
- 双指针仅使用常数额外空间

## 五、去重正确性论证 / Deduplication Correctness

### 5.1 外层去重

```rust
if i > 0 && nums[i] == nums[i-1] { continue; }
```

**论证**：若 $\text{nums}[i] = \text{nums}[i-1]$，则以 $i$ 为第一个元素的所有三元组必然与以 $i-1$ 为第一个元素的某三元组重复。跳过 $i$ 不会遗漏任何独特解。

### 5.2 内层去重

```rust
while left < right && nums[left] == nums[left+1] { left += 1; }
while left < right && nums[right] == nums[right-1] { right -= 1; }
```

**论证**：找到一组解后，连续相等的元素会产生重复的三元组。跳过这些重复元素确保每个独特的值组合只被记录一次。

## 六、常见错误 / Common Mistakes

| 错误 | 现象 | 正确做法 |
|------|------|---------|
| 去重时机错误 | 漏解或重复 | 先收集解，再去重，最后收缩指针 |
| 未排序直接双指针 | 结果错误 | 必须先排序 |
| 去重条件写错 | 如 `nums[i] == nums[i+1]` | 应为 `nums[i] == nums[i-1]` |
| 边界溢出 | `i+1` 越界 | 确保 $n \geq 3$ |

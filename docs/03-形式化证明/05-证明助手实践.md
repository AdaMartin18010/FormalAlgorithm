### 摘要 / Executive Summary

### 关键术语与符号 / Glossary

- 证明助手（Proof Assistant）：基于类型论的交互式定理证明环境。
- 依赖类型（Dependent Type）：类型可依赖值的类型系统，支撑 Curry-Howard 同构。
- 策略（Tactic）：面向目标的证明指令，如 `intro`、`rewrite`、`induction`。
- 项（Term）：直接书写的证明对象，对应λ演算中的正规形式。

## 1. 概述

证明助手（Proof Assistant）是一类基于严格数理逻辑和类型论的软件工具，用于辅助人类构造和机器检查数学证明与程序正确性证明 [Geuvers2009]。
它们的核心理论基础是 **Curry-Howard 同构**：命题即类型，证明即程序 [SorensenUrzyczyn2006]。

当前最具影响力的证明助手包括：

- **Lean 4**（Microsoft Research）：强调数学形式化与程序验证的统一，拥有活跃的数学库 Mathlib [AvigadMassot2024]。
- **Coq / Rocq**（INRIA）：构造演算（Calculus of Constructions）的实现，Software Foundations 系列课程的基石 [Pierce2010]。
- **Agda**（Chalmers）：依赖类型编程语言，直接以项（term）方式书写证明，强调证明即程序 [Norell2007]。

## 2. Lean 4

### 2.1 简介与设计哲学

Lean 是由 Leonardo de Moura 在 Microsoft Research 主导开发的定理证明器和编程语言 [deMoura2021]。
Lean 4 在保持强大证明能力的同时，具备高效的编译执行性能，其设计目标包括：

1. **数学形式化**：支持大型数学项目（如 Mathlib、Liquid Tensor Experiment）。
2. **程序验证**：通过依赖类型和 tactics 验证程序正确性。
3. **自动化**：内置强大的元编程（metaprogramming）和求解器接口。

### 2.2 环境安装

**方式一：使用 `elan`（推荐）**

`elan` 是 Lean 的版本管理器，类似于 Rust 的 `rustup`：

```bash
# Linux / macOS
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

# Windows (PowerShell)
# 参见 https://leanprover-community.github.io/install/windows.html
```

安装完成后，创建新项目：

```bash
lake new MyFirstProof math
```

**方式二：使用在线编辑器**

访问 [Lean 4 Web Editor](https://live.lean-lang.org/) 即可在浏览器中编写和运行 Lean 代码。

### 2.3 基本类型与函数

Lean 4 是一门纯函数式语言，带有依赖类型。以下是完整的可运行代码片段，文件位于 `examples/lean_proofs/BasicTypes.lean`：

```lean
namespace BasicTypes

-- 简单函数
def square (n : Nat) : Nat := n * n

-- 多态函数
def identity {α : Type} (x : α) : α := x

def compose {α β γ : Type} (f : β → γ) (g : α → β) (x : α) : γ := f (g x)

-- 自定义归纳类型：列表
inductive MyList (α : Type) where
  | nil : MyList α
  | cons : α → MyList α → MyList α

deriving Repr

-- 列表长度
def myLength {α : Type} : MyList α → Nat
  | MyList.nil => 0
  | MyList.cons _ t => 1 + myLength t

-- 依赖类型示例：定长向量
inductive Vect (α : Type) : Nat → Type where
  | nil : Vect α 0
  | cons : α → Vect α n → Vect α (n + 1)

def vectHead {α : Type} {n : Nat} : Vect α (n + 1) → α
  | Vect.cons x _ => x

end BasicTypes
```

### 2.4 第一个证明：加法单位元

在 Lean 4 中，证明通常以 `theorem` 或 `lemma` 开始，后接证明策略（tactics）。文件位于 `examples/lean_proofs/NatInduction.lean`：

```lean
namespace NatInduction
open Nat

-- 定理：0 是加法的右单位元
-- 这是 Lean 教程中的经典第一个证明 [AvigadMassot2024, 第3章]

theorem add_zero_right (n : Nat) : n + 0 = n := by
  induction n with
  | zero =>
    -- 基础情况：0 + 0 = 0
    rfl
  | succ n ih =>
    -- 归纳步骤：假设 n + 0 = n，证明 (n + 1) + 0 = n + 1
    calc
      (n + 1) + 0 = (n + 0) + 1 := by rfl
      _ = n + 1 := by rw [ih]

-- 加法交换律
theorem add_comm (n m : Nat) : n + m = m + n := by
  induction n with
  | zero =>
    rw [add_zero_right]
    rfl
  | succ n ih =>
    calc
      (n + 1) + m = (n + m) + 1 := by rfl
      _ = (m + n) + 1 := by rw [ih]
      _ = m + (n + 1) := by rw [Nat.add_succ]

end NatInduction
```

**证明解析**：

1. `induction n with` 启动对自然数 `n` 的结构归纳。
2. `zero` 分支处理基础情况，使用 `rfl`（reflexivity）证明 `0 + 0 = 0`。
3. `succ n ih` 分支处理归纳步骤，`ih` 是归纳假设 `n + 0 = n`。
4. `calc` 用于等式链推导，`rw [ih]` 使用归纳假设进行重写。

### 2.5 归纳证明：列表反转

下面证明列表反转的对合性质（reverse 是自身的逆）：`reverse (reverse xs) = xs`。文件位于 `examples/lean_proofs/ListReverse.lean`：

```lean
namespace ListReverse

inductive MyList (α : Type) where
  | nil : MyList α
  | cons : α → MyList α → MyList α

deriving Repr

def myReverse {α : Type} : MyList α → MyList α
  | MyList.nil => MyList.nil
  | MyList.cons x xs => myAppend (myReverse xs) (MyList.cons x MyList.nil)
where
  myAppend : MyList α → MyList α → MyList α
    | MyList.nil, ys => ys
    | MyList.cons x xs, ys => MyList.cons x (myAppend xs ys)

-- 关键引理：reverse 保持 append 的分配律
theorem reverse_append_distrib {α : Type} (xs ys : MyList α) :
  myReverse (myReverse.myAppend xs ys) =
    myReverse.myAppend (myReverse ys) (myReverse xs) := by
  induction xs with
  | nil =>
    simp [myReverse, myReverse.myAppend]
  | cons x xs ih =>
    simp [myReverse, myReverse.myAppend]
    rw [ih]
    simp [myReverse.myAppend]

-- 主定理：reverse 是对合的
theorem reverse_involutive {α : Type} (xs : MyList α) :
  myReverse (myReverse xs) = xs := by
  induction xs with
  | nil =>
    simp [myReverse]
  | cons x xs ih =>
    simp [myReverse]
    rw [reverse_append_distrib]
    simp [myReverse, myReverse.myAppend]
    rw [ih]

end ListReverse
```

**证明要点**：

- 直接证明 `reverse (reverse xs) = xs` 会遇到 `reverse` 与 `append` 的嵌套问题。
- 因此先证明辅助引理 `reverse (xs ++ ys) = reverse ys ++ reverse xs`。
- `simp` 策略会自动展开定义并进行等式化简，大幅简化证明过程。

## 3. Coq / Rocq

### 3.1 简介与历史

Coq（自 2023 年底起官方更名为 **Rocq**）是基于构造演算（Calculus of Constructions）和归纳构造（Calculus of Inductive Constructions）的交互式定理证明器 [CoquandHuet1988][BertotCasteran2004]。
它是形式化验证领域历史最悠久、生态系统最成熟的工具之一，著名的应用包括 CompCert 编译器验证和四色定理的形式化证明 [Gonthier2008]。

### 3.2 环境安装

**通过包管理器安装（Linux / macOS）**：

```bash
# Ubuntu / Debian
sudo apt-get install coq

# macOS (Homebrew)
brew install coq

# 或使用 opam
opam install coq
```

**推荐学习资源**：

- *Software Foundations* [Pierce2010]：Coq 的权威教材，从逻辑基础到程序验证系统递进。
- *Certified Programming with Dependent Types* [Chlipala2013]：高级 Coq 编程技巧。

### 3.3 Gallina 基础

Coq 的规范语言称为 Gallina。以下是基本语法示例：

```coq
(* 定义简单函数 *)
Definition square (n : nat) : nat := n * n.

(* 归纳数据类型 *)
Inductive natlist : Type :=
  | nil
  | cons (n : nat) (l : natlist).

(* 递归函数 *)
Fixpoint length (l : natlist) : nat :=
  match l with
  | nil => 0
  | cons _ t => 1 + length t
  end.

(* 多态列表 *)
Inductive list (A : Type) : Type :=
  | nil
  | cons (x : A) (l : list A).
```

### 3.4 第一个证明

以下是一个完整的、可直接在 Coq 中运行的脚本，证明自然数加法的交换律：

```coq
(* 引入标准库 *)
Require Import Arith.

(* 定理：0 是加法的右单位元 *)
Theorem add_0_r : forall n : nat, n + 0 = n.
Proof.
  intros n.
  induction n as [| n' IHn'].
  - (* n = 0 *)
    reflexivity.
  - (* n = S n' *)
    simpl.
    rewrite IHn'.
    reflexivity.
Qed.

(* 定理：加法交换律 *)
Theorem add_comm : forall n m : nat, n + m = m + n.
Proof.
  intros n m.
  induction n as [| n' IHn'].
  - (* n = 0 *)
    rewrite add_0_r.
    reflexivity.
  - (* n = S n' *)
    simpl.
    rewrite IHn'.
    rewrite plus_n_Sm.
    reflexivity.
Qed.
```

**证明解析**：

1. `intros n` 将全称量词引入上下文。
2. `induction n as [| n' IHn']` 对 `n` 进行归纳，`IHn'` 是归纳假设。
3. `-` 标记子目标，`reflexivity` 证明自反等式。
4. `rewrite IHn'` 使用归纳假设重写目标。
5. `simpl` 化简表达式，`plus_n_Sm` 是库中关于后继的引理。

## 4. Agda

### 4.1 简介与特点

Agda 是一门基于直觉主义类型论的依赖类型函数式编程语言，由 Ulf Norell 在 Chalmers 开发 [Norell2007]。与 Lean 和 Coq 不同，Agda **没有独立的 tactics 语言**——证明直接以项（term）的形式书写，强调"证明即程序"的理念。Agda 的 Unicode 支持极佳，数学符号可直接在代码中使用。

### 4.2 环境安装

**通过 cabal / stack 安装**：

```bash
cabal install Agda
```

或使用 Emacs 的 Agda 模式进行交互式开发。

### 4.3 依赖类型入门

以下是 Agda 的基本语法示例：

```agda
module AgdaIntro where

-- 自然数定义
data ℕ : Set where
  zero : ℕ
  suc  : ℕ → ℕ

-- 加法
_+_ : ℕ → ℕ → ℕ
zero  + n = n
suc m + n = suc (m + n)

-- 多态恒等函数
id : {A : Set} → A → A
id x = x

-- 定长向量
data Vec (A : Set) : ℕ → Set where
  []  : Vec A zero
  _∷_ : {n : ℕ} → A → Vec A n → Vec A (suc n)

-- 安全取头元素
head : {A : Set} {n : ℕ} → Vec A (suc n) → A
head (x ∷ xs) = x
```

### 4.4 第一个证明

在 Agda 中，证明就是写一个返回特定等式类型的函数。以下证明 `0` 是加法的右单位元：

```agda
module AgdaProof where

open import Relation.Binary.PropositionalEquality
open ≡-Reasoning

-- 自然数和加法（复用上述定义或标准库）
open import Data.Nat
open import Data.Nat.Properties

-- 定理：0 是加法的右单位元
+-identityʳ : ∀ n → n + 0 ≡ n
+-identityʳ zero    = refl
+-identityʳ (suc n) = cong suc (+-identityʳ n)

-- 定理：加法交换律
+-comm : ∀ m n → m + n ≡ n + m
+-comm zero    n = sym (+-identityʳ n)
+-comm (suc m) n =
  begin
    suc m + n
  ≡⟨ refl ⟩
    suc (m + n)
  ≡⟨ cong suc (+-comm m n) ⟩
    suc (n + m)
  ≡⟨ sym (+-suc n m) ⟩
    n + suc m
  ∎
```

**证明解析**：

1. `+-identityʳ zero = refl`：基础情况，`0 + 0 ≡ 0` 由自反性 `refl` 直接证明。
2. `+-identityʳ (suc n) = cong suc (+-identityʳ n)`：归纳步骤，对 `suc` 应用同态（congruence）。
3. `≡-Reasoning` 提供了 `begin ... ≡⟨ ... ⟩ ... ∎` 的等式链语法，使证明更易读。
4. `sym` 表示对称性，`+-suc` 是标准库中关于后继的引理。

## 5. 三种证明助手对比

| 维度 | Lean 4 | Coq / Rocq | Agda |
|------|--------|------------|------|
| **核心演算** | 依赖类型理论 (DTT) | 归纳构造演算 (CIC) | 直觉主义类型论 (ITT) |
| **证明风格** | Tactics + Term | Tactics (Ltac/Ltac2) + Term | 纯 Term |
| **编程体验** | 高性能编译，类Rust | 提取到 OCaml/Haskell | 解释执行，交互编辑 |
| **数学库** | Mathlib (非常活跃) | Mathematical Components | 标准库 + 社区包 |
| **工业应用** | 数学形式化、验证 | 编译器验证、密码学 | 类型论研究、教学 |
| **学习曲线** | 中等 | 较陡 | 中等 |

**选择建议**：

- 若目标是**现代数学形式化**（代数、分析、拓扑），优先选择 **Lean 4** [AvigadMassot2024]。
- 若目标是**软件验证与编译器验证**，优先选择 **Coq / Rocq** [Pierce2010]。
- 若偏好**直接以程序方式书写证明**，或研究**同伦类型论**（HoTT），优先选择 **Agda** [Norell2007]。

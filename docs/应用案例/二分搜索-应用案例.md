# 二分搜索实际应用案例

## 案例概述

**算法**: 二分搜索 (Binary Search)  
**应用领域**: 数据库索引、版本控制系统、游戏开发  
**案例来源**: Git版本控制 / MySQL B+树索引 / Unity游戏物理引擎

## 应用场景描述

### 背景

二分搜索虽然概念简单，但在工业界有极其广泛的应用：
- **数据库索引**: B+树索引的核心查找操作
- **版本控制**: Git的commit历史和文件diff
- **游戏开发**: 物理碰撞检测、动画关键帧插值
- **系统编程**: 内核内存管理、虚拟地址映射

### 问题定义

**场景1 - Git Blame**: 查找某行代码的最后修改者
- 输入: 文件历史版本列表 + 目标行号
- 输出: 最后修改该行的commit

**场景2 - 数据库范围查询**: 查找某区间内的记录
- 输入: 有序索引 + 查询范围 [L, R]
- 输出: 范围内的所有记录ID

### 为什么选择二分搜索

| 特性 | 二分搜索优势 | 对比 |
|------|-----------|------|
| 时间复杂度 | ✅ $O(\log n)$ 极快 | 线性扫描 $O(n)$ |
| 空间复杂度 | ✅ $O(1)$ 原地 | 哈希表额外空间 |
| 适用范围 | ✅ 有序序列即可 | 哈希需要离散键 |
| 范围查询 | ✅ 天然支持 | 哈希表范围查询困难 |

## 算法实现

### 问题建模

**Git Blame二分查找**:

```
问题: 文件F在第N行在commit C中仍然存在，找出最后修改该行的commit

策略: 在commit历史上二分查找
- 假设commit历史按时间有序: C1, C2, ..., Cn
- 二分查找最后一个使第N行发生变化的commit

关键观察: 设f(C) = "在commit C中第N行与C0中相同"
则f(C)是单调的：一旦不同，后续commit都不同
```

use std::cmp::Ordering;

/// ==================== Git Blame实现 ====================

/// 简化版的Git提交
#[derive(Clone, Debug)]
struct Commit {
    id: String,
    parent_id: Option<String>,
    timestamp: u64,
    changes: Vec<FileChange>,  // 本次提交的文件变更
}

/// 文件变更
#[derive(Clone, Debug)]
struct FileChange {
    file_path: String,
    line_changes: Vec<LineChange>,
}

/// 行变更
#[derive(Clone, Debug)]
enum LineChange {
    Add { line_num: usize, content: String },
    Delete { line_num: usize },
    Modify { line_num: usize, new_content: String },
}

/// Git Blame查找器
pub struct GitBlameFinder {
    commits: Vec<Commit>,  // 按时间排序的提交历史
}

impl GitBlameFinder {
    pub fn new(commits: Vec<Commit>) -> Self {
        Self { commits }
    }

    /// 二分查找最后修改指定行的commit
    pub fn blame(&self, file_path: &str, target_line: usize) -> Option<&Commit> {
        if self.commits.is_empty() {
            return None;
        }
        
        let mut left = 0;
        let mut right = self.commits.len() - 1;
        let mut result = None;
        
        while left <= right {
            let mid = left + (right - left) / 2;
            let commit = &self.commits[mid];
            
            // 检查在第mid个commit时目标行是否存在且内容不变
            if self.line_exists_at(file_path, target_line, mid) {
                // 继续向左查找更早的修改
                result = Some(commit);
                if mid == 0 {
                    break;
                }
                right = mid - 1;
            } else {
                // 目标行不存在或内容已变，向右查找
                left = mid + 1;
            }
        }
        
        result
    }

    /// 模拟：检查在某commit索引位置，目标行是否存在且内容一致
    fn line_exists_at(&self, file_path: &str, target_line: usize, commit_idx: usize) -> bool {
        // 简化实现：遍历到该commit的所有变更，重建文件状态
        let mut current_content = String::new();
        
        for i in 0..=commit_idx {
            if let Some(change) = self.find_file_change(&self.commits[i], file_path) {
                self.apply_change(&mut current_content, change);
            }
        }
        
        // 检查目标行是否存在
        current_content.lines().nth(target_line.saturating_sub(1)).is_some()
    }

    fn find_file_change<'a>(&self, commit: &'a Commit, path: &str) -> Option<&'a FileChange> {
        commit.changes.iter().find(|c| c.file_path == path)
    }

    fn apply_change(&self, content: &mut String, change: &FileChange) {
        // 简化实现
        for lc in &change.line_changes {
            match lc {
                LineChange::Add { line_num, content: new_line } => {
                    // 在指定行插入
                    let lines: Vec<_> = content.lines().collect();
                    let mut result = String::new();
                    for (i, line) in lines.iter().enumerate() {
                        if i + 1 == *line_num {
                            result.push_str(new_line);
                            result.push('\n');
                        }
                        result.push_str(line);
                        result.push('\n');
                    }
                    *content = result;
                }
                _ => {} // 简化处理
            }
        }
    }
}

/// ==================== 数据库索引范围查询 ====================

/// B+树索引节点（简化版）
#[derive(Clone, Debug)]
pub struct BPlusTreeNode {
    keys: Vec<i64>,           // 索引键值
    values: Vec<u64>,         // 记录ID
    children: Vec<Box<BPlusTreeNode>>, // 子节点（非叶子节点）
    is_leaf: bool,
    next: Option<Box<BPlusTreeNode>>,  // 叶子节点的链表指针
}

impl BPlusTreeNode {
    pub fn new_leaf() -> Self {
        Self {
            keys: Vec::new(),
            values: Vec::new(),
            children: Vec::new(),
            is_leaf: true,
            next: None,
        }
    }

    /// 二分查找：找到第一个大于等于key的位置
    fn lower_bound(&self, key: i64) -> usize {
        let mut left = 0;
        let mut right = self.keys.len();
        
        while left < right {
            let mid = left + (right - left) / 2;
            match self.keys[mid].cmp(&key) {
                Ordering::Less => left = mid + 1,
                _ => right = mid,
            }
        }
        left
    }

    /// 二分查找：找到第一个大于key的位置
    fn upper_bound(&self, key: i64) -> usize {
        let mut left = 0;
        let mut right = self.keys.len();
        
        while left < right {
            let mid = left + (right - left) / 2;
            match self.keys[mid].cmp(&key) {
                Ordering::Greater => right = mid,
                _ => left = mid + 1,
            }
        }
        left
    }

    /// 范围查询 [start, end]
    pub fn range_query(&self, start: i64, end: i64) -> Vec<u64> {
        let mut result = Vec::new();
        
        if self.is_leaf {
            // 叶子节点：使用二分查找确定范围
            let start_idx = self.lower_bound(start);
            let end_idx = self.upper_bound(end);
            
            result.extend_from_slice(&self.values[start_idx..end_idx]);
            
            // 如果范围跨越多个叶子节点，继续遍历链表
            if end_idx == self.keys.len() {
                if let Some(ref next) = self.next {
                    result.extend(next.range_query(start, end));
                }
            }
        } else {
            // 内部节点：找到可能包含范围的子节点
            let idx = self.lower_bound(start);
            if idx < self.children.len() {
                result.extend(self.children[idx].range_query(start, end));
            }
        }
        
        result
    }

    /// 精确查找
    pub fn find(&self, key: i64) -> Option<u64> {
        if self.is_leaf {
            let idx = self.lower_bound(key);
            if idx < self.keys.len() && self.keys[idx] == key {
                Some(self.values[idx])
            } else {
                None
            }
        } else {
            let idx = self.lower_bound(key);
            if idx < self.children.len() {
                self.children[idx].find(key)
            } else {
                None
            }
        }
    }
}

/// ==================== 游戏动画关键帧插值 ====================

/// 关键帧
#[derive(Clone, Copy, Debug)]
struct Keyframe {
    time: f32,      // 时间戳
    value: f32,     // 值
    easing: Easing, // 缓动函数
}

#[derive(Clone, Copy, Debug)]
enum Easing {
    Linear,
    EaseIn,
    EaseOut,
    EaseInOut,
}

/// 动画曲线
pub struct AnimationCurve {
    keyframes: Vec<Keyframe>,  // 按时间排序
}

impl AnimationCurve {
    pub fn new(mut keyframes: Vec<Keyframe>) -> Self {
        keyframes.sort_by(|a, b| a.time.partial_cmp(&b.time).unwrap());
        Self { keyframes }
    }

    /// 二分查找当前时间所在的关键帧区间
    pub fn evaluate(&self, time: f32) -> f32 {
        if self.keyframes.is_empty() {
            return 0.0;
        }
        
        if time <= self.keyframes[0].time {
            return self.keyframes[0].value;
        }
        
        if time >= self.keyframes.last().unwrap().time {
            return self.keyframes.last().unwrap().value;
        }
        
        // 二分查找包含time的关键帧区间
        let idx = self.find_keyframe_interval(time);
        let k1 = &self.keyframes[idx];
        let k2 = &self.keyframes[idx + 1];
        
        // 线性插值
        let t = (time - k1.time) / (k2.time - k1.time);
        k1.value + (k2.value - k1.value) * self.apply_easing(t, k1.easing)
    }

    fn find_keyframe_interval(&self, time: f32) -> usize {
        let mut left = 0;
        let mut right = self.keyframes.len() - 1;
        
        while left < right {
            let mid = left + (right - left) / 2;
            if self.keyframes[mid].time <= time {
                if self.keyframes[mid + 1].time > time {
                    return mid;
                }
                left = mid + 1;
            } else {
                right = mid;
            }
        }
        left
    }

    fn apply_easing(&self, t: f32, easing: Easing) -> f32 {
        match easing {
            Easing::Linear => t,
            Easing::EaseIn => t * t,
            Easing::EaseOut => 1.0 - (1.0 - t) * (1.0 - t),
            Easing::EaseInOut => {
                if t < 0.5 {
                    2.0 * t * t
                } else {
                    1.0 - (-2.0 * t + 2.0).powi(2) / 2.0
                }
            }
        }
    }
}

/// ==================== 浮点数二分搜索（数值计算） ====================

/// 浮点数二分搜索：求函数的根
pub fn find_root<F>(f: F, left: f64, right: f64, epsilon: f64) -> Option<f64>
where
    F: Fn(f64) -> f64,
{
    let mut l = left;
    let mut r = right;
    
    // 检查区间是否有根
    if f(l) * f(r) > 0.0 {
        return None;
    }
    
    while (r - l).abs() > epsilon {
        let mid = (l + r) / 2.0;
        let f_mid = f(mid);
        
        if f_mid.abs() < epsilon {
            return Some(mid);
        }
        
        if f(l) * f_mid < 0.0 {
            r = mid;
        } else {
            l = mid;
        }
    }
    
    Some((l + r) / 2.0)
}

/// 浮点数二分搜索：求平方根
pub fn sqrt_binary(x: f64, epsilon: f64) -> f64 {
    if x < 0.0 {
        return f64::NAN;
    }
    if x == 0.0 {
        return 0.0;
    }
    
    let mut l = 0.0;
    let mut r = x.max(1.0);
    
    while (r - l) > epsilon {
        let mid = (l + r) / 2.0;
        if mid * mid < x {
            l = mid;
        } else {
            r = mid;
        }
    }
    
    (l + r) / 2.0
}

/// 整数二分搜索：避免溢出的mid计算
pub fn safe_binary_search(arr: &[i32], target: i32) -> Option<usize> {
    let mut left = 0i32;
    let mut right = arr.len() as i32 - 1;
    
    while left <= right {
        // 避免 (left + right) / 2 溢出
        let mid = left + (right - left) / 2;
        let mid_idx = mid as usize;
        
        match arr[mid_idx].cmp(&target) {
            Ordering::Equal => return Some(mid_idx),
            Ordering::Less => left = mid + 1,
            Ordering::Greater => right = mid - 1,
        }
    }
    None
}

/// 二分搜索泛型实现（支持自定义比较器）
pub fn binary_search_by<T, F>(arr: &[T], mut f: F) -> Result<usize, usize>
where
    F: FnMut(&T) -> Ordering,
{
    let mut left = 0;
    let mut right = arr.len();
    
    while left < right {
        let mid = left + (right - left) / 2;
        match f(&arr[mid]) {
            Ordering::Equal => return Ok(mid),
            Ordering::Less => left = mid + 1,
            Ordering::Greater => right = mid,
        }
    }
    Err(left) // 返回应该插入的位置
}

### 经验总结

1. **防溢出**: 使用 `mid = left + (right - left) / 2` 而非 `(left + right) / 2`
2. **闭开区间**: 明确[left, right]还是[left, right)避免死循环
3. **lower_bound**: 第一个>=target的位置，用于插入和范围查询
4. **浮点精度**: 设置合适的epsilon终止条件

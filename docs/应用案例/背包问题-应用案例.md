## 案例概述

**算法**: 0/1背包问题 (0/1 Knapsack) / 完全背包
**应用领域**: 资源分配、装载优化、投资组合、云计算调度
**案例来源**: 云服务器资源调度 / 物流装箱 / 广告投放优化

## 应用场景描述

### 背景

背包问题是动态规划最经典的应用：

- **云计算**: 虚拟机调度到物理服务器
- **物流**: 卡车/集装箱装箱优化
- **广告**: 预算限制下的广告组合
- **金融**: 风险约束下的投资组合

### 问题定义

**场景 - 云服务器资源调度**:

**输入**:

- 物理服务器资源（CPU、内存、磁盘）
- 待调度虚拟机列表（资源需求、优先级）
- 目标：最大化放置的虚拟机数量或优先级总和

### 为什么选择动态规划

| 特性 | DP优势 | 对比 |
|------|--------|------|
| 最优性 | ✅ 保证最优解 | 贪心只能近似 |
| 灵活性 | ✅ 支持多种约束 | - |
| 可扩展 | ✅ 可加入新约束 | - |

## 算法实现

```rust
/// 0/1背包问题
pub struct KnapsackItem {
    pub weight: usize,
    pub value: usize,
    pub id: usize,
}

/// 标准0/1背包DP
pub fn knapsack(items: &[KnapsackItem], capacity: usize) -> (usize, Vec<usize>) {
    let n = items.len();

    // dp[i][w] = 前i个物品，容量w时的最大价值
    let mut dp = vec![vec![0; capacity + 1]; n + 1];

    for i in 1..=n {
        let item = &items[i - 1];
        for w in 0..=capacity {
            if item.weight <= w {
                dp[i][w] = dp[i - 1][w].max(dp[i - 1][w - item.weight] + item.value);
            } else {
                dp[i][w] = dp[i - 1][w];
            }
        }
    }

    // 回溯找选了哪些物品
    let mut selected = Vec::new();
    let mut w = capacity;
    for i in (1..=n).rev() {
        if dp[i][w] != dp[i - 1][w] {
            selected.push(items[i - 1].id);
            w -= items[i - 1].weight;
        }
    }

    (dp[n][capacity], selected)
}

/// 空间优化版（一维DP）
pub fn knapsack_optimized(items: &[KnapsackItem], capacity: usize) -> usize {
    let mut dp = vec![0; capacity + 1];

    for item in items {
        // 逆序遍历，避免重复选择
        for w in (item.weight..=capacity).rev() {
            dp[w] = dp[w].max(dp[w - item.weight] + item.value);
        }
    }

    dp[capacity]
}

/// 多维背包（云计算资源调度）
pub struct VMRequest {
    pub id: usize,
    pub cpu: usize,     // CPU核心数
    pub memory: usize,  // 内存GB
    pub disk: usize,    // 磁盘GB
    pub priority: usize,
}

pub struct Server {
    pub cpu: usize,
    pub memory: usize,
    pub disk: usize,
}

/// 多维背包简化版：按主要资源（CPU）做DP，其他资源做约束检查
pub fn schedule_vms(vms: &[VMRequest], server: &Server) -> (usize, Vec<usize>) {
    // 过滤掉超规格的VM
    let feasible_vms: Vec<_> = vms.iter()
        .filter(|vm| vm.cpu <= server.cpu && vm.memory <= server.memory && vm.disk <= server.disk)
        .cloned()
        .collect();

    // 先按CPU做背包，再验证其他约束
    // 简化：使用贪心+回溯的混合策略
    let mut selected = Vec::new();
    let mut remaining = Server {
        cpu: server.cpu,
        memory: server.memory,
        disk: server.disk,
    };

    // 按优先级/资源比排序
    let mut sorted_vms: Vec<_> = feasible_vms.iter().enumerate().collect();
    sorted_vms.sort_by(|(_, a), (_, b)| {
        let ratio_a = a.priority as f64 / (a.cpu + a.memory + a.disk) as f64;
        let ratio_b = b.priority as f64 / (b.cpu + b.memory + b.disk) as f64;
        ratio_b.partial_cmp(&ratio_a).unwrap()
    });

    let mut total_priority = 0;

    for (idx, vm) in sorted_vms {
        if vm.cpu <= remaining.cpu && vm.memory <= remaining.memory && vm.disk <= remaining.disk {
            selected.push(vm.id);
            total_priority += vm.priority;
            remaining.cpu -= vm.cpu;
            remaining.memory -= vm.memory;
            remaining.disk -= vm.disk;
        }
    }

    (total_priority, selected)
}

/// 完全背包（无限物品）
pub fn unbounded_knapsack(items: &[KnapsackItem], capacity: usize) -> usize {
    let mut dp = vec![0; capacity + 1];

    for item in items {
        for w in item.weight..=capacity {
            dp[w] = dp[w].max(dp[w - item.weight] + item.value);
        }
    }

    dp[capacity]
}
```

### 复杂度分析

| 问题类型 | 时间复杂度 | 空间复杂度 |
|---------|-----------|-----------|
| 0/1背包 | $O(n \cdot W)$ | $O(W)$ |
| 完全背包 | $O(n \cdot W)$ | $O(W)$ |
| 多维背包 | $O(n \cdot W_1 \cdot W_2 \cdots)$ | 指数级 |

# 快速排序实际应用案例

## 案例概述

**算法**: 快速排序 (Quick Sort) + 快速选择 (Quick Select)  
**应用领域**: 通用排序库、实时数据分析、Top-K问题  
**案例来源**: 实时排行榜系统 / C++ std::sort 实现参考

## 应用场景描述

### 背景

在游戏、电商和社交平台中，实时排行榜是核心功能之一。例如：
- 手游实时战力排行榜（百万在线用户）
- 直播间实时礼物榜（每秒数千条礼物记录）
- 股票交易实时涨跌幅排行

这些场景需要**高频更新** + **快速查询**的排序能力。

### 问题定义

**输入**: 
- $N$ 个用户分数记录（$N$ 在10万到1000万之间）
- 频繁的分数更新（每秒1000-10000次更新）
- 查询请求：获取Top-K（如前100名）

**约束**:
- 查询响应时间 < 10ms
- 更新操作不能阻塞查询
- 内存限制在几GB内

**输出**: 实时Top-K排行榜

### 为什么选择快速排序/快速选择

| 特性 | 快速排序优势 | 对比 |
|------|-----------|------|
| 平均性能 | ✅ $O(n \log n)$，常数极小 | 归并排序额外空间开销大 |
| 缓存友好 | ✅ 原地操作，局部性好 | 堆排序跳跃访问缓存不友好 |
| 快速选择 | ✅ 平均 $O(n)$ 找第K大 | 全排序后再取Top-K浪费 |
| 工程优化 | ✅ introsort等混合策略成熟 | 其他算法优化空间有限 |

## 算法实现

### 问题建模

采用**增量维护 + 快速选择**策略：

1. **数据结构**: 使用无序数组/向量存储所有用户分数
2. **更新操作**: 直接修改对应用户分数，$O(1)$
3. **查询操作**: 使用快速选择找Top-K，$O(n)$ 平均，或定期全排序维护有序区

```
优化策略: 双缓冲 + 惰性排序
- Buffer A: 当前服务缓冲区（可能无序）
- Buffer B: 后台排序缓冲区
- 查询时：若A无序，用快速选择取Top-K；若A有序，直接返回
```

### 关键代码

```rust
use std::sync::{Arc, RwLock};
use std::thread;
use std::time::{Duration, Instant};
use rand::Rng;

/// 用户分数记录
#[derive(Clone, Copy, Debug)]
struct UserScore {
    user_id: u64,
    score: i64,      // 可能为负数（扣分场景）
    timestamp: u64,  // 用于同分时的排序
}

/// 实时排行榜系统
pub struct RealtimeLeaderboard {
    scores: Arc<RwLock<Vec<UserScore>>>,
    top_k: usize,  // 排行榜大小，如100
}

impl RealtimeLeaderboard {
    pub fn new(top_k: usize) -> Self {
        Self {
            scores: Arc::new(RwLock::new(Vec::with_capacity(1_000_000))),
            top_k,
        }
    }

    /// 更新用户分数
    pub fn update_score(&self, user_id: u64, new_score: i64) {
        let mut scores = self.scores.write().unwrap();
        
        // 查找用户
        if let Some(pos) = scores.iter().position(|s| s.user_id == user_id) {
            scores[pos].score = new_score;
            scores[pos].timestamp = Self::current_timestamp();
        } else {
            // 新用户
            scores.push(UserScore {
                user_id,
                score: new_score,
                timestamp: Self::current_timestamp(),
            });
        }
    }

    /// 批量更新（更高效率）
    pub fn batch_update(&self, updates: Vec<(u64, i64)>) {
        let mut scores = self.scores.write().unwrap();
        
        for (user_id, new_score) in updates {
            if let Some(pos) = scores.iter().position(|s| s.user_id == user_id) {
                scores[pos].score = new_score;
                scores[pos].timestamp = Self::current_timestamp();
            } else {
                scores.push(UserScore {
                    user_id,
                    score: new_score,
                    timestamp: Self::current_timestamp(),
                });
            }
        }
    }

    /// 获取Top-K（使用快速选择优化）
    pub fn get_top_k(&self) -> Vec<UserScore> {
        let scores = self.scores.read().unwrap();
        let n = scores.len();
        
        if n <= self.top_k {
            // 数据量小，直接排序
            let mut result = scores.clone();
            result.sort_by(|a, b| {
                b.score.cmp(&a.score)  // 降序
                    .then_with(|| a.timestamp.cmp(&b.timestamp))
            });
            return result;
        }
        
        // 快速选择找Top-K
        let mut working: Vec<UserScore> = scores.clone();
        self.quick_select_top_k(&mut working, self.top_k);
        
        // 前K个元素无序，需要排序
        working[..self.top_k].sort_by(|a, b| {
            b.score.cmp(&a.score)
                .then_with(|| a.timestamp.cmp(&b.timestamp))
        });
        
        working[..self.top_k].to_vec()
    }

    /// 快速选择算法：将Top-K元素放到数组前K个位置
    fn quick_select_top_k(&self, arr: &mut [UserScore], k: usize) {
        if k == 0 || arr.len() <= k {
            return;
        }
        
        Self::quick_select_helper(arr, k, 0, arr.len() - 1);
    }

    fn quick_select_helper(arr: &mut [UserScore], k: usize, left: usize, right: usize) {
        if left >= right {
            return;
        }
        
        // 三数取中法选择pivot
        let pivot_idx = Self::median_of_three(arr, left, right);
        arr.swap(pivot_idx, right);
        
        let pivot_pos = Self::partition(arr, left, right);
        
        // 计算大于pivot的元素数量
        let larger_count = right - pivot_pos + 1;
        
        if larger_count == k {
            // 正好找到Top-K
            return;
        } else if larger_count > k {
            // Top-K全在右半部分，继续递归右半
            if pivot_pos + 1 < right {
                Self::quick_select_helper(arr, k, pivot_pos + 1, right);
            }
        } else {
            // 需要从左半部分补充 (k - larger_count) 个
            if left + 1 < pivot_pos {
                Self::quick_select_helper(arr, k - larger_count, left, pivot_pos - 1);
            }
        }
    }

    fn partition(arr: &mut [UserScore], left: usize, right: usize) -> usize {
        let pivot = arr[right];
        let mut i = left;
        
        for j in left..right {
            // 降序排列：大的放右边
            if Self::compare_scores(&arr[j], &pivot) == std::cmp::Ordering::Less {
                arr.swap(i, j);
                i += 1;
            }
        }
        
        arr.swap(i, right);
        i
    }

    /// 三数取中
    fn median_of_three(arr: &mut [UserScore], left: usize, right: usize) -> usize {
        let mid = left + (right - left) / 2;
        
        if Self::compare_scores(&arr[left], &arr[mid]) == std::cmp::Ordering::Greater {
            arr.swap(left, mid);
        }
        if Self::compare_scores(&arr[left], &arr[right]) == std::cmp::Ordering::Greater {
            arr.swap(left, right);
        }
        if Self::compare_scores(&arr[mid], &arr[right]) == std::cmp::Ordering::Greater {
            arr.swap(mid, right);
        }
        
        mid
    }

    fn compare_scores(a: &UserScore, b: &UserScore) -> std::cmp::Ordering {
        a.score.cmp(&b.score)
            .then_with(|| b.timestamp.cmp(&a.timestamp))
    }

    fn current_timestamp() -> u64 {
        use std::time::SystemTime;
        SystemTime::now()
            .duration_since(SystemTime::UNIX_EPOCH)
            .unwrap()
            .as_millis() as u64
    }
}

/// 使用Introsort的完整排序（当需要全量排序时）
pub struct IntrosortLeaderboard {
    scores: Arc<RwLock<Vec<UserScore>>>,
}

impl IntrosortLeaderboard {
    /// Introsort: 快速排序 + 堆排序 + 插入排序的混合
    pub fn introsort(&self, arr: &mut [UserScore]) {
        let max_depth = 2 * (arr.len() as f64).log2() as usize;
        Self::introsort_helper(arr, max_depth);
    }

    fn introsort_helper(arr: &mut [UserScore], max_depth: usize) {
        const INSERTION_THRESHOLD: usize = 16;
        
        if arr.len() <= INSERTION_THRESHOLD {
            Self::insertion_sort(arr);
            return;
        }
        
        if max_depth == 0 {
            // 递归过深，切换为堆排序
            Self::heap_sort(arr);
            return;
        }
        
        // 快速排序
        let pivot = Self::partition(arr);
        let len = arr.len();
        
        Self::introsort_helper(&mut arr[..pivot], max_depth - 1);
        Self::introsort_helper(&mut arr[pivot + 1..], max_depth - 1);
    }

    fn insertion_sort(arr: &mut [UserScore]) {
        for i in 1..arr.len() {
            let mut j = i;
            while j > 0 && Self::compare(&arr[j - 1], &arr[j]) == std::cmp::Ordering::Greater {
                arr.swap(j - 1, j);
                j -= 1;
            }
        }
    }

    fn heap_sort(arr: &mut [UserScore]) {
        let n = arr.len();
        
        // 建堆
        for i in (0..n / 2).rev() {
            Self::heapify(arr, n, i);
        }
        
        // 提取元素
        for i in (1..n).rev() {
            arr.swap(0, i);
            Self::heapify(arr, i, 0);
        }
    }

    fn heapify(arr: &mut [UserScore], n: usize, i: usize) {
        let mut largest = i;
        let left = 2 * i + 1;
        let right = 2 * i + 2;
        
        if left < n && Self::compare(&arr[left], &arr[largest]) == std::cmp::Ordering::Greater {
            largest = left;
        }
        if right < n && Self::compare(&arr[right], &arr[largest]) == std::cmp::Ordering::Greater {
            largest = right;
        }
        
        if largest != i {
            arr.swap(i, largest);
            Self::heapify(arr, n, largest);
        }
    }

    fn partition(arr: &mut [UserScore]) -> usize {
        let len = arr.len();
        let pivot_idx = len / 2;
        arr.swap(pivot_idx, len - 1);
        
        let mut i = 0;
        for j in 0..len - 1 {
            if Self::compare(&arr[j], &arr[len - 1]) == std::cmp::Ordering::Less {
                arr.swap(i, j);
                i += 1;
            }
        }
        arr.swap(i, len - 1);
        i
    }

    fn compare(a: &UserScore, b: &UserScore) -> std::cmp::Ordering {
        b.score.cmp(&a.score)
            .then_with(|| a.timestamp.cmp(&b.timestamp))
    }
}

#[cfg(test)]
mod tests {
    use super::*;
    
    #[test]
    fn test_quick_select_performance() {
        let mut rng = rand::thread_rng();
        let n = 1_000_000;
        let k = 100;
        
        // 生成随机数据
        let mut data: Vec<UserScore> = (0..n)
            .map(|i| UserScore {
                user_id: i as u64,
                score: rng.gen_range(0..1_000_000),
                timestamp: i as u64,
            })
            .collect();
        
        // 测试快速选择
        let start = Instant::now();
        let lb = RealtimeLeaderboard::new(k);
        lb.quick_select_top_k(&mut data, k);
        let quick_select_time = start.elapsed();
        
        // 测试全排序
        let start = Instant::now();
        data.sort_by(|a, b| b.score.cmp(&a.score));
        let full_sort_time = start.elapsed();
        
        println!("数据集大小: {}", n);
        println!("Top-K数量: {}", k);
        println!("快速选择时间: {:?}", quick_select_time);
        println!("全排序时间: {:?}", full_sort_time);
        println!("加速比: {:.2}x", full_sort_time.as_secs_f64() / quick_select_time.as_secs_f64());
    }
}

fn main() {
    let lb = RealtimeLeaderboard::new(100);
    
    // 模拟100万用户
    println!("初始化100万用户...");
    for i in 0..1_000_000 {
        lb.update_score(i as u64, (i * 10) as i64);
    }
    
    // 模拟高频更新
    println!("开始模拟高频更新和查询...");
    let start = Instant::now();
    let mut total_query_time = Duration::ZERO;
    
    for _ in 0..1000 {
        // 批量更新
        let updates: Vec<_> = (0..100)
            .map(|_| (rand::random::<u64>() % 1_000_000, rand::random::<i64>() % 1000000))
            .collect();
        lb.batch_update(updates);
        
        // 查询Top-K
        let query_start = Instant::now();
        let _top = lb.get_top_k();
        total_query_time += query_start.elapsed();
    }
    
    let elapsed = start.elapsed();
    println!("总耗时: {:?}", elapsed);
    println!("平均查询耗时: {:?}", total_query_time / 1000);
}
```

### 复杂度分析

**快速选择**:
- 平均时间复杂度: $O(n)$
- 最坏情况: $O(n^2)$（可通过随机化或中位数优化避免）
- 空间复杂度: $O(1)$ 原地操作

**快速排序（Introsort）**:
- 最坏时间复杂度: $O(n \log n)$（限制递归深度后）
- 平均时间复杂度: $O(n \log n)$
- 空间复杂度: $O(\log n)$ 递归栈深度

### 经验总结

1. **混合策略**: 数据量<10K时全排序更快，>100K时快速选择优势明显
2. **线程安全**: 使用RwLock实现读写分离，更新不阻塞查询
3. **缓存优化**: 预分配Vec容量，避免频繁扩容
4. **三数取中**: 相比随机pivot，减少最坏情况概率
5. **定期全排序**: 后台线程定期维护有序副本，进一步降低查询延迟

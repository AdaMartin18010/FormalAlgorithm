# 归并排序实际应用案例

## 案例概述

**算法**: 归并排序 (Merge Sort)  
**应用领域**: 数据库系统、大数据处理  
**案例来源**: 关系型数据库管理系统中的外部排序实现

## 应用场景描述

### 背景

在大数据处理场景中，经常需要对超出内存容量的数据进行排序。例如，一个大型电商平台需要对数亿条订单记录按时间戳排序进行数据分析，或者数据库系统需要为没有索引的字段生成有序结果。

### 问题定义

**输入**: 存储在磁盘上的 $N$ 条记录（$N$ 可能达到数十亿，远超内存容量）  
**约束**: 
- 内存有限（如 8GB RAM）
- 磁盘 I/O 成本高昂
- 需要稳定排序保持相等记录的原始顺序

**输出**: 完全有序的 $N$ 条记录序列

### 为什么选择归并排序

| 特性 | 归并排序优势 | 其他算法对比 |
|------|------------|------------|
| 稳定性 | ✅ 天然稳定，保持相等元素相对顺序 | 快排不稳定 |
| 时间复杂度 | ✅ 稳定的 $O(n \log n)$ | 冒泡/选择排序 $O(n^2)$ |
| 外部排序适应性 | ✅ 天然适合多路归并 | 堆排序难以并行化 |
| 顺序访问模式 | ✅ 适合磁盘块读取 | 快排随机访问性能差 |

## 算法实现

### 问题建模

将外部排序问题抽象为**多阶段归并模型**:

1. **阶段1 - 生成初始归并段**: 将大文件分成若干块，每块能装入内存，分别排序后写回磁盘
2. **阶段2 - 多路归并**: 使用 $k$ 路归并将初始归并段合并为最终有序文件

```
总数据量: N = 100GB
内存容量: M = 8GB
块大小: B = 64MB

初始归并段数: n_runs = N / M = 12.5 → 向上取整为 13 段
每段大小: M = 8GB
```

### 关键代码

```rust
use std::fs::{File, OpenOptions};
use std::io::{BufReader, BufWriter, Read, Write, Seek, SeekFrom};
use std::cmp::Ordering;
use std::collections::BinaryHeap;

/// 订单记录结构
#[derive(Clone, Debug, PartialEq, Eq)]
struct OrderRecord {
    timestamp: u64,    // 排序键
    order_id: u64,     // 订单ID
    user_id: u64,      // 用户ID
    amount: u64,       // 订单金额
}

impl Ord for OrderRecord {
    fn cmp(&self, other: &Self) -> Ordering {
        self.timestamp.cmp(&other.timestamp)
    }
}

impl PartialOrd for OrderRecord {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}

/// 外部排序管理器
pub struct ExternalSorter {
    memory_limit: usize,    // 内存限制（字节）
    block_size: usize,      // 块大小
    temp_dir: String,       // 临时文件目录
}

impl ExternalSorter {
    pub fn new(memory_limit_mb: usize, temp_dir: &str) -> Self {
        Self {
            memory_limit: memory_limit_mb * 1024 * 1024,
            block_size: 64 * 1024 * 1024,  // 64MB
            temp_dir: temp_dir.to_string(),
        }
    }

    /// 主排序函数
    pub fn sort(&self, input_path: &str, output_path: &str) -> Result<(), Box<dyn std::error::Error>> {
        // 阶段1: 生成初始归并段
        let run_files = self.create_initial_runs(input_path)?;
        
        // 阶段2: 多路归并
        self.k_way_merge(&run_files, output_path)?;
        
        // 清理临时文件
        for file in &run_files {
            std::fs::remove_file(file)?;
        }
        
        Ok(())
    }

    /// 阶段1: 生成初始归并段
    fn create_initial_runs(&self, input_path: &str) -> Result<Vec<String>, Box<dyn std::error::Error>> {
        let mut run_files = Vec::new();
        let mut file = File::open(input_path)?;
        let mut reader = BufReader::with_capacity(self.block_size, &mut file);
        
        let records_per_run = self.memory_limit / std::mem::size_of::<OrderRecord>();
        let mut run_id = 0;
        
        loop {
            // 读取一块数据到内存
            let mut buffer = Vec::with_capacity(records_per_run);
            
            for _ in 0..records_per_run {
                let mut record_bytes = [0u8; 32]; // 每条记录32字节
                match reader.read_exact(&mut record_bytes) {
                    Ok(_) => {
                        buffer.push(self.deserialize_record(&record_bytes));
                    }
                    Err(_) => break,
                }
            }
            
            if buffer.is_empty() {
                break;
            }
            
            // 内存内排序（使用标准库的Timsort）
            buffer.sort();
            
            // 写入临时文件
            let run_file = format!("{}/run_{}.tmp", self.temp_dir, run_id);
            self.write_run_file(&run_file, &buffer)?;
            run_files.push(run_file);
            
            run_id += 1;
            println!("创建归并段 #{}, 包含 {} 条记录", run_id, buffer.len());
        }
        
        println!("总共创建 {} 个初始归并段", run_files.len());
        Ok(run_files)
    }

    /// 阶段2: K路归并
    fn k_way_merge(&self, run_files: &[String], output_path: &str) -> Result<(), Box<dyn std::error::Error>> {
        let k = run_files.len();
        
        // 打开所有归并段文件
        let mut readers: Vec<BufReader<File>> = run_files
            .iter()
            .map(|path| {
                let file = File::open(path).unwrap();
                BufReader::with_capacity(self.block_size / k, file)
            })
            .collect();
        
        // 初始化：从每个归并段读取第一条记录
        let mut heap = BinaryHeap::new();
        let mut buffers: Vec<Vec<OrderRecord>> = vec![Vec::new(); k];
        
        for (i, reader) in readers.iter_mut().enumerate() {
            if let Some(record) = self.read_record(reader) {
                // 使用Reverse实现最小堆
                heap.push(std::cmp::Reverse((record.clone(), i)));
                buffers[i].push(record);
            }
        }
        
        // 输出文件
        let output_file = OpenOptions::new()
            .write(true)
            .create(true)
            .truncate(true)
            .open(output_path)?;
        let mut writer = BufWriter::with_capacity(self.block_size, output_file);
        
        // 归并主循环
        let mut output_buffer = Vec::with_capacity(self.block_size / std::mem::size_of::<OrderRecord>());
        let mut total_written = 0usize;
        
        while let Some(std::cmp::Reverse((record, run_idx))) = heap.pop() {
            output_buffer.push(record);
            total_written += 1;
            
            // 批量写入以提高I/O效率
            if output_buffer.len() >= self.block_size / std::mem::size_of::<OrderRecord>() {
                self.write_batch(&mut writer, &output_buffer)?;
                output_buffer.clear();
            }
            
            // 从对应归并段读取下一条记录
            if let Some(next_record) = self.read_record(&mut readers[run_idx]) {
                heap.push(std::cmp::Reverse((next_record.clone(), run_idx)));
            }
        }
        
        // 写入剩余数据
        if !output_buffer.is_empty() {
            self.write_batch(&mut writer, &output_buffer)?;
        }
        
        println!("归并完成，共写入 {} 条记录", total_written);
        Ok(())
    }

    fn deserialize_record(&self, bytes: &[u8]) -> OrderRecord {
        OrderRecord {
            timestamp: u64::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3], 
                                           bytes[4], bytes[5], bytes[6], bytes[7]]),
            order_id: u64::from_le_bytes([bytes[8], bytes[9], bytes[10], bytes[11],
                                          bytes[12], bytes[13], bytes[14], bytes[15]]),
            user_id: u64::from_le_bytes([bytes[16], bytes[17], bytes[18], bytes[19],
                                         bytes[20], bytes[21], bytes[22], bytes[23]]),
            amount: u64::from_le_bytes([bytes[24], bytes[25], bytes[26], bytes[27],
                                        bytes[28], bytes[29], bytes[30], bytes[31]]),
        }
    }

    fn write_run_file(&self, path: &str, records: &[OrderRecord]) -> Result<(), Box<dyn std::error::Error>> {
        let file = OpenOptions::new()
            .write(true)
            .create(true)
            .truncate(true)
            .open(path)?;
        let mut writer = BufWriter::new(file);
        
        for record in records {
            writer.write_all(&record.timestamp.to_le_bytes())?;
            writer.write_all(&record.order_id.to_le_bytes())?;
            writer.write_all(&record.user_id.to_le_bytes())?;
            writer.write_all(&record.amount.to_le_bytes())?;
        }
        
        writer.flush()?;
        Ok(())
    }

    fn read_record(&self, reader: &mut BufReader<File>) -> Option<OrderRecord> {
        let mut buf = [0u8; 32];
        match reader.read_exact(&mut buf) {
            Ok(_) => Some(self.deserialize_record(&buf)),
            Err(_) => None,
        }
    }

    fn write_batch(&self, writer: &mut BufWriter<File>, records: &[OrderRecord]) -> Result<(), Box<dyn std::error::Error>> {
        for record in records {
            writer.write_all(&record.timestamp.to_le_bytes())?;
            writer.write_all(&record.order_id.to_le_bytes())?;
            writer.write_all(&record.user_id.to_le_bytes())?;
            writer.write_all(&record.amount.to_le_bytes())?;
        }
        Ok(())
    }
}

/// 性能测试入口
fn main() {
    let sorter = ExternalSorter::new(512, "/tmp/sort_runs"); // 512MB内存限制
    
    println!("开始外部排序...");
    match sorter.sort("/data/orders_unsorted.bin", "/data/orders_sorted.bin") {
        Ok(_) => println!("排序完成！"),
        Err(e) => eprintln!("排序失败: {}", e),
    }
}
```

### 复杂度分析

**时间复杂度**:
- 生成初始归并段: $O(n \log m)$，其中 $m$ 是内存可容纳的记录数
- 多路归并: $O(n \log k)$，其中 $k$ 是归并段数
- 总时间复杂度: $O(n \log n)$

**空间复杂度**:
- 内存: $O(m)$，$m$ 为内存限制
- 磁盘: $O(n)$，需要临时存储归并段

**I/O复杂度**:
- 顺序读: $2 \times n$（生成归并段 + 归并）
- 顺序写: $2 \times n$（归并段 + 输出）
- 总I/O: $4n$，全部为顺序I/O

### 经验总结

1. **块大小调优**: 64MB-256MB的块大小在SSD上表现最佳，太小导致I/O次数多，太大降低并行度
2. **归并路数**: $k$ 值取 $k \approx M/B$ 最优，$M$为内存，$B$为块大小
3. **预读取**: 使用异步I/O预读取下一批数据可减少30%的I/O等待时间
4. **压缩**: 对归并段进行轻量级压缩可减少磁盘I/O量20-40%

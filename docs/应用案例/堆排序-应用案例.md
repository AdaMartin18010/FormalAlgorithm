# 堆排序实际应用案例


> **版本**: 1.0

## 案例概述

**算法**: 堆排序 (Heap Sort) / 优先队列 (Priority Queue)
**应用领域**: 任务调度、Top-K实时流处理、 median维护
**案例来源**: 分布式任务调度系统 / 实时流量监控系统

## 应用场景描述

### 背景

在现代分布式系统中，任务调度是核心组件：

- **云函数计算**: 百万级异步任务按优先级执行
- **消息队列**: 延迟消息按触发时间排序投递
- **流量控制**: API网关实时限流，维护最高优先级请求

这些场景需要**动态插入** + **快速提取极值**的能力。

### 问题定义

**输入**:

- 持续到达的任务流（每秒数千到数万）
- 每个任务有优先级（数值越小越优先）
- 支持动态调整优先级

**约束**:

- 插入操作: $O(\log n)$
- 提取最高优先级: $O(\log n)$
- 内存占用可控，原地操作优先

**输出**: 按优先级顺序执行任务

### 为什么选择堆/堆排序

| 特性 | 堆优势 | 对比 |
|------|-------|------|
| 插入复杂度 | ✅ $O(\log n)$ | 有序数组 $O(n)$ |
| 提取极值 | ✅ $O(\log n)$ | 无序数组 $O(n)$ |
| 空间复杂度 | ✅ $O(1)$ 原地 | 平衡BST额外指针开销 |
| 实现复杂度 | ✅ 数组即可实现 | 红黑树等实现复杂 |

## 算法实现

### 问题建模

设计**带时间片的优先级任务调度器**:

```
任务结构:
- task_id: 唯一标识
- priority: 优先级值（越小越优先）
- deadline: 截止时间（用于同优先级排序）
- exec_count: 执行次数（动态调整优先级用）

调度策略:
- 每次取出最高优先级任务执行
- 执行完成后根据结果调整优先级（反馈控制）
- 支持抢占：高优先级任务可中断低优先级任务
```

### 关键代码

```rust
use std::cmp::Ordering;
use std::collections::BinaryHeap;
use std::time::{Duration, Instant, SystemTime, UNIX_EPOCH};
use std::sync::{Arc, Mutex, Condvar};
use std::thread;

/// 任务结构
#[derive(Clone, Debug)]
struct Task {
    id: u64,
    priority: i32,      // 越小越优先
    deadline: u64,      // 时间戳
    exec_count: u32,    // 执行次数
    payload: Vec<u8>,   // 任务数据
}

/// 用于堆比较的任务包装器
#[derive(Clone, Debug)]
struct TaskNode {
    priority: i32,
    deadline: u64,
    task_id: u64,
    // 使用Arc以便多线程共享
    task: Arc<Mutex<Task>>,
}

impl PartialEq for TaskNode {
    fn eq(&self, other: &Self) -> bool {
        self.priority == other.priority && self.deadline == other.deadline
    }
}

impl Eq for TaskNode {}

impl PartialOrd for TaskNode {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        Some(self.cmp(other))
    }
}

impl Ord for TaskNode {
    fn cmp(&self, other: &Self) -> Ordering {
        // 最小堆：priority小的在前
        other.priority.cmp(&self.priority)
            .then_with(|| other.deadline.cmp(&self.deadline))
    }
}

/// 自定义二叉堆实现（展示原理）
pub struct TaskHeap {
    data: Vec<TaskNode>,
    size: usize,
}

impl TaskHeap {
    pub fn new() -> Self {
        Self {
            data: Vec::with_capacity(1024),
            size: 0,
        }
    }

    pub fn len(&self) -> usize {
        self.size
    }

    pub fn is_empty(&self) -> bool {
        self.size == 0
    }

    /// 插入任务
    pub fn push(&mut self, node: TaskNode) {
        if self.size >= self.data.len() {
            self.data.push(node);
        } else {
            self.data[self.size] = node;
        }
        self.size += 1;
        self.heapify_up(self.size - 1);
    }

    /// 提取最高优先级任务
    pub fn pop(&mut self) -> Option<TaskNode> {
        if self.is_empty() {
            return None;
        }

        self.size -= 1;
        self.data.swap(0, self.size);

        if self.size > 0 {
            self.heapify_down(0);
        }

        self.data.get(self.size).cloned()
    }

    /// 查看最高优先级任务
    pub fn peek(&self) -> Option<&TaskNode> {
        self.data.first()
    }

    /// 上浮调整
    fn heapify_up(&mut self, mut idx: usize) {
        while idx > 0 {
            let parent = (idx - 1) / 2;
            if self.data[idx] < self.data[parent] {
                self.data.swap(idx, parent);
                idx = parent;
            } else {
                break;
            }
        }
    }

    /// 下沉调整
    fn heapify_down(&mut self, mut idx: usize) {
        loop {
            let left = 2 * idx + 1;
            let right = 2 * idx + 2;
            let mut smallest = idx;

            if left < self.size && self.data[left] < self.data[smallest] {
                smallest = left;
            }
            if right < self.size && self.data[right] < self.data[smallest] {
                smallest = right;
            }

            if smallest == idx {
                break;
            }

            self.data.swap(idx, smallest);
            idx = smallest;
        }
    }

    /// 修改指定位置元素的优先级（支持动态优先级调整）
    pub fn update_priority(&mut self, idx: usize, new_priority: i32) {
        if idx >= self.size {
            return;
        }

        let old_priority = self.data[idx].priority;
        self.data[idx].priority = new_priority;

        if new_priority < old_priority {
            // 优先级提高，上浮
            self.heapify_up(idx);
        } else {
            // 优先级降低，下沉
            self.heapify_down(idx);
        }
    }
}

/// 任务调度器
pub struct TaskScheduler {
    heap: Mutex<TaskHeap>,
    condvar: Condvar,
    shutdown: Mutex<bool>,
    worker_count: usize,
}

impl TaskScheduler {
    pub fn new(worker_count: usize) -> Arc<Self> {
        Arc::new(Self {
            heap: Mutex::new(TaskHeap::new()),
            condvar: Condvar::new(),
            shutdown: Mutex::new(false),
            worker_count,
        })
    }

    /// 启动调度器
    pub fn start(self: &Arc<Self>) {
        for i in 0..self.worker_count {
            let scheduler = Arc::clone(self);
            thread::spawn(move || {
                println!("Worker {} started", i);
                scheduler.worker_loop();
            });
        }
    }

    /// 提交任务
    pub fn submit(&self, task: Task) {
        let node = TaskNode {
            priority: task.priority,
            deadline: task.deadline,
            task_id: task.id,
            task: Arc::new(Mutex::new(task)),
        };

        let mut heap = self.heap.lock().unwrap();
        heap.push(node);
        self.condvar.notify_one();
    }

    /// 工作线程主循环
    fn worker_loop(&self) {
        loop {
            let mut heap = self.heap.lock().unwrap();

            // 等待任务或关闭信号
            heap = self.condvar.wait_while(heap, |heap| {
                heap.is_empty() && !*self.shutdown.lock().unwrap()
            }).unwrap();

            // 检查是否关闭
            if *self.shutdown.lock().unwrap() && heap.is_empty() {
                break;
            }

            // 获取任务
            if let Some(node) = heap.pop() {
                drop(heap); // 释放锁

                // 执行任务
                self.execute_task(&node);
            }
        }
    }

    fn execute_task(&self, node: &TaskNode) {
        let mut task = node.task.lock().unwrap();

        println!(
            "[Worker] Executing task {} (priority={}, exec_count={})",
            task.id, task.priority, task.exec_count
        );

        // 模拟任务执行
        thread::sleep(Duration::from_millis(10));

        task.exec_count += 1;

        // 反馈控制：根据执行次数调整优先级
        // 执行次数多的任务降低优先级（防止饥饿）
        if task.exec_count % 10 == 0 {
            task.priority += 1;
        }
    }

    pub fn shutdown(&self) {
        let mut shutdown = self.shutdown.lock().unwrap();
        *shutdown = true;
        self.condvar.notify_all();
    }
}

/// Top-K流处理器（实时维护最大的K个元素）
pub struct TopKProcessor<T: Ord + Clone> {
    k: usize,
    min_heap: BinaryHeap<std::cmp::Reverse<T>>, // 存储Top-K，最小堆
}

impl<T: Ord + Clone> TopKProcessor<T> {
    pub fn new(k: usize) -> Self {
        Self {
            k,
            min_heap: BinaryHeap::with_capacity(k + 1),
        }
    }

    /// 处理流中的元素
    pub fn process(&mut self, item: T) {
        if self.min_heap.len() < self.k {
            // 堆未满，直接插入
            self.min_heap.push(std::cmp::Reverse(item));
        } else if let Some(&std::cmp::Reverse(ref smallest)) = self.min_heap.peek() {
            // 堆已满，只有当新元素更大时才替换
            if item > *smallest {
                self.min_heap.pop();
                self.min_heap.push(std::cmp::Reverse(item));
            }
        }
    }

    /// 获取当前Top-K（按升序返回）
    pub fn get_top_k(&self) -> Vec<T> {
        let mut result: Vec<T> = self.min_heap
            .iter()
            .map(|r| r.0.clone())
            .collect();
        result.sort();
        result.reverse(); // 降序
        result
    }
}

/// 实时Median维护器
pub struct RunningMedian {
    // 大根堆存储较小的一半
    lower_half: BinaryHeap<i32>,
    // 小根堆存储较大的一半
    upper_half: BinaryHeap<std::cmp::Reverse<i32>>,
}

impl RunningMedian {
    pub fn new() -> Self {
        Self {
            lower_half: BinaryHeap::new(),
            upper_half: BinaryHeap::new(),
        }
    }

    /// 插入新值
    pub fn insert(&mut self, value: i32) {
        // 决定插入哪个堆
        if self.lower_half.is_empty() || value <= *self.lower_half.peek().unwrap() {
            self.lower_half.push(value);
        } else {
            self.upper_half.push(std::cmp::Reverse(value));
        }

        // 平衡两个堆的大小
        self.rebalance();
    }

    fn rebalance(&mut self) {
        // 确保大小差不超过1
        if self.lower_half.len() > self.upper_half.len() + 1 {
            let val = self.lower_half.pop().unwrap();
            self.upper_half.push(std::cmp::Reverse(val));
        } else if self.upper_half.len() > self.lower_half.len() + 1 {
            let std::cmp::Reverse(val) = self.upper_half.pop().unwrap();
            self.lower_half.push(val);
        }
    }

    /// 获取当前中位数
    pub fn median(&self) -> f64 {
        match (self.lower_half.len(), self.upper_half.len()) {
            (0, 0) => 0.0,
            (l, u) if l == u => {
                (*self.lower_half.peek().unwrap() as f64 +
                 self.upper_half.peek().unwrap().0 as f64) / 2.0
            }
            (l, u) if l > u => *self.lower_half.peek().unwrap() as f64,
            _ => self.upper_half.peek().unwrap().0 as f64,
        }
    }
}

/// 性能测试
fn benchmark() {
    println!("=== 堆操作性能测试 ===\n");

    // 测试1: 基本堆操作
    let n = 1_000_000;
    let mut heap = TaskHeap::new();

    let start = Instant::now();
    for i in 0..n {
        heap.push(TaskNode {
            priority: (i % 1000) as i32,
            deadline: i as u64,
            task_id: i as u64,
            task: Arc::new(Mutex::new(Task {
                id: i as u64,
                priority: (i % 1000) as i32,
                deadline: i as u64,
                exec_count: 0,
                payload: vec![],
            })),
        });
    }
    let insert_time = start.elapsed();
    println!("插入 {} 个元素耗时: {:?}", n, insert_time);

    let start = Instant::now();
    while !heap.is_empty() {
        heap.pop();
    }
    let extract_time = start.elapsed();
    println!("提取全部元素耗时: {:?}", extract_time);

    // 测试2: Top-K流处理
    println!("\n=== Top-K流处理测试 ===");
    let mut topk = TopKProcessor::new(100);
    let stream: Vec<i32> = (0..10_000_000).map(|_| rand::random()).collect();

    let start = Instant::now();
    for &val in &stream {
        topk.process(val);
    }
    let topk_time = start.elapsed();
    println!("处理1000万元素流，维护Top-100耗时: {:?}", topk_time);

    // 测试3: 实时Median
    println!("\n=== 实时Median测试 ===");
    let mut median_tracker = RunningMedian::new();

    let start = Instant::now();
    for i in 0..1_000_000 {
        median_tracker.insert(rand::random::<i32>() % 1000000);
    }
    let median_time = start.elapsed();
    println!("插入100万个元素，实时维护Median耗时: {:?}", median_time);
    println!("当前Median: {}", median_tracker.median());
}

fn main() {
    benchmark();

    // 演示任务调度器
    println!("\n=== 任务调度器演示 ===");
    let scheduler = TaskScheduler::new(4);
    scheduler.start();

    // 提交100个任务
    let now = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap()
        .as_millis() as u64;

    for i in 0..100 {
        scheduler.submit(Task {
            id: i,
            priority: (i % 10) as i32,
            deadline: now + (i as u64 * 1000),
            exec_count: 0,
            payload: vec![0u8; 1024],
        });
    }

    thread::sleep(Duration::from_secs(5));
    scheduler.shutdown();
    println!("调度器已关闭");
}
```

### 经验总结

1. **批量建堆**: 大量数据时，Floyd $O(n)$ 建堆比分批插入快3-5倍
2. **容量预分配**: 预分配数组容量避免频繁扩容
3. **索引堆**: 需要修改优先级时，使用索引堆避免 $O(n)$ 查找
4. **多路归并**: 外部排序时，d叉堆可减少I/O次数
5. **Pairing Heap**: 理论性能更优，但实际常数因子较大

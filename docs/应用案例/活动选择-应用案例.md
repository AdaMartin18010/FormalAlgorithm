## 案例概述

**算法**: 活动选择问题 (Activity Selection)
**应用领域**: 会议安排、任务调度、资源分配、课程表设计
**案例来源**: 会议室预订系统 / 任务调度器 / 航班调度

## 应用场景描述

### 背景

活动选择是贪心算法的经典应用：

- **企业办公**: 会议室预订管理
- **云计算**: 任务调度最大化吞吐量
- **航空**: 航班登机口分配
- **教育**: 课程表自动生成

### 问题定义

**场景 - 智能会议室系统**:

**输入**:

- 会议请求列表（开始时间、结束时间、人数）
- 会议室容量、设备配置

**输出**:

- 最大化可安排的会议数量
- 或最大化会议室利用率

### 为什么选择贪心算法

| 特性 | 贪心优势 | 对比 |
|------|---------|------|
| 最优性 | ✅ 活动选择问题贪心可得最优 | DP也可但复杂 |
| 速度 | ✅ $O(n \log n)$ | DP是 $O(n^2)$ |
| 实现在线 | ✅ 可按截止时间排序后依次处理 | - |

## 算法实现

```rust
/// 会议/活动
#[derive(Clone, Debug)]
pub struct Activity {
    pub id: usize,
    pub start: u32,  // 开始时间（分钟从0点开始）
    pub end: u32,    // 结束时间
    pub attendees: usize,
    pub priority: u32,
}

/// 贪心算法：选择最早结束的活动
pub fn activity_selection(activities: &[Activity]) -> Vec<usize> {
    // 按结束时间排序
    let mut sorted: Vec<_> = activities.iter().collect();
    sorted.sort_by_key(|a| a.end);

    let mut selected = Vec::new();
    let mut last_end = 0;

    for activity in sorted {
        if activity.start >= last_end {
            selected.push(activity.id);
            last_end = activity.end;
        }
    }

    selected
}

/// 带权重的活动选择（最大总权重）
pub fn weighted_activity_selection(activities: &[Activity]) -> (u32, Vec<usize>) {
    let mut sorted: Vec<_> = activities.to_vec();
    sorted.sort_by_key(|a| a.end);

    let n = sorted.len();

    // dp[i] = 前i个活动能获得的最大权重
    let mut dp = vec![0; n + 1];
    let mut choice = vec![false; n];

    for i in 1..=n {
        let activity = &sorted[i - 1];

        // 找到最后一个不冲突的活动
        let mut last_compatible = 0;
        for j in (1..i).rev() {
            if sorted[j - 1].end <= activity.start {
                last_compatible = j;
                break;
            }
        }

        // 选择：包含当前活动 vs 不包含
        let include = dp[last_compatible] + activity.priority;
        let exclude = dp[i - 1];

        if include > exclude {
            dp[i] = include;
            choice[i - 1] = true;
        } else {
            dp[i] = exclude;
        }
    }

    // 回溯
    let mut selected = Vec::new();
    let mut i = n;
    while i > 0 {
        if choice[i - 1] {
            selected.push(sorted[i - 1].id);
            // 跳到上一个兼容活动
            for j in (1..i).rev() {
                if sorted[j - 1].end <= sorted[i - 1].start {
                    i = j;
                    break;
                }
            }
        } else {
            i -= 1;
        }
    }

    (dp[n], selected)
}

/// 多会议室调度（区间图着色）
pub struct MeetingRoomScheduler {
    rooms: Vec<Vec<Activity>>,
}

impl MeetingRoomScheduler {
    /// 最少会议室数量 = 最大重叠区间数
    pub fn min_rooms_needed(activities: &[Activity]) -> usize {
        let mut events = Vec::new();

        for a in activities {
            events.push((a.start, 1));  // 开始，占用+1
            events.push((a.end, -1));   // 结束，占用-1
        }

        events.sort();

        let mut max_rooms = 0;
        let mut current = 0;

        for (_, delta) in events {
            current += delta;
            max_rooms = max_rooms.max(current);
        }

        max_rooms
    }

    /// 分配会议到具体房间
    pub fn schedule(activities: &[Activity], num_rooms: usize) -> Vec<Vec<usize>> {
        let mut sorted: Vec<_> = activities.to_vec();
        sorted.sort_by_key(|a| a.start);

        let mut rooms: Vec<Vec<usize>> = vec![Vec::new(); num_rooms];
        let mut room_end_times = vec![0; num_rooms];

        for activity in sorted {
            // 找最早空闲的房间
            if let Some(room_idx) = room_end_times.iter()
                .position(|&end| end <= activity.start) {
                rooms[room_idx].push(activity.id);
                room_end_times[room_idx] = activity.end;
            }
        }

        rooms
    }
}
```

### 复杂度分析

| 问题 | 时间复杂度 | 空间复杂度 |
|-----|-----------|-----------|
| 标准活动选择 | $O(n \log n)$ | $O(1)$ |
| 带权活动选择 | $O(n^2)$ 或 $O(n \log n)$ | $O(n)$ |
| 最少会议室 | $O(n \log n)$ | $O(n)$ |

## 练习1: 冒泡排序

### 算法原理
冒泡排序通过重复遍历数组，比较相邻元素并在顺序错误时交换它们。每轮遍历后，最大的未排序元素会"冒泡"到正确位置。

### 初始数据
```
数组: [64, 34, 25, 12, 22, 11, 90]
索引:  0   1   2   3   4   5   6
```

### 第一轮遍历

**步骤 1.1**: 比较索引 0 和 1 (64 > 34)
```
[64, 34, 25, 12, 22, 11, 90]
  ↑  ↑
 64 > 34? 是 → 交换
```

**问题**: 交换后的数组是什么？

<details>
<summary>✅ 答案</summary>

```
[34, 64, 25, 12, 22, 11, 90]
```
</details>

---

**步骤 1.2**: 继续比较索引 1 和 2
```
[34, 64, 25, 12, 22, 11, 90]
     ↑  ↑
 64 > 25? 是 → 交换
```

**问题**: 当前数组状态？

<details>
<summary>✅ 答案</summary>

```
[34, 25, 64, 12, 22, 11, 90]
```
</details>

---

**步骤 1.3-1.6**: 完成第一轮

**问题**: 第一轮结束后，数组状态如何？哪个元素到了最终位置？

<details>
<summary>✅ 答案</summary>

```
第一轮结束: [34, 25, 12, 22, 11, 64, 90]
                              ↑     ↑
                           已排序  已排序
```

**90 到达了最终位置**（最大的元素）
</details>

---

### 完整排序过程跟踪

| 轮次 | 数组状态 | 已排序部分 |
|------|----------|-----------|
| 初始 | [64, 34, 25, 12, 22, 11, 90] | - |
| 第1轮 | [34, 25, 12, 22, 11, 64, 90] | 90 |
| 第2轮 | [25, 12, 22, 11, 34, 64, 90] | 64, 90 |
| 第3轮 | [12, 22, 11, 25, 34, 64, 90] | 34, 64, 90 |
| 第4轮 | [12, 11, 22, 25, 34, 64, 90] | 25, 34, 64, 90 |
| 第5轮 | [11, 12, 22, 25, 34, 64, 90] | 12, 22, 25, 34, 64, 90 |
| 第6轮 | [11, 12, 22, 25, 34, 64, 90] | 全部有序 |

### 复杂度分析

**问题**: 冒泡排序的时间复杂度是多少？进行了多少次比较？

<details>
<summary>✅ 答案</summary>

- **最坏情况**: $O(n^2)$ - 逆序数组
- **最好情况**: $O(n)$ - 已有序数组（使用优化标志位）
- **平均情况**: $O(n^2)$

**比较次数**: $(n-1) + (n-2) + ... + 1 = \frac{n(n-1)}{2}$

**空间复杂度**: $O(1)$（原地排序）
</details>

## 练习3: 插入排序

### 核心思想
将数组分为已排序和未排序两部分，每次从未排序部分取一个元素，插入到已排序部分的正确位置。

### 初始数据
```
[5, 2, 4, 6, 1, 3]
```

### 逐步插入过程

**步骤 3.1**: 插入 2
```
已排序: [5]    待插入: 2

2 < 5, 将5右移 → [5, 5, 4, 6, 1, 3]
插入2 → [2, 5, 4, 6, 1, 3]
```

**步骤 3.2**: 插入 4
```
已排序: [2, 5]    待插入: 4

4 < 5? 是，5右移 → [2, 5, 5, 6, 1, 3]
4 < 2? 否 → 插入
结果: [2, 4, 5, 6, 1, 3]
```

**问题**: 继续插入 1，写出详细过程

<details>
<summary>✅ 答案</summary>

当前: [2, 4, 5, 6, 1, 3]，插入 1

```
1. 1 < 6? 是，6右移 → [2, 4, 5, 6, 6, 3]
2. 1 < 5? 是，5右移 → [2, 4, 5, 5, 6, 3]
3. 1 < 4? 是，4右移 → [2, 4, 4, 5, 6, 3]
4. 1 < 2? 是，2右移 → [2, 2, 4, 5, 6, 3]
5. 到达开头，插入1 → [1, 2, 4, 5, 6, 3]
```

**共进行 4 次比较，4 次移动**
</details>

### 插入排序特性

| 特性 | 说明 |
|------|------|
| 最好情况 | $O(n)$ - 已有序数组，每次只比较1次 |
| 最坏情况 | $O(n^2)$ - 逆序数组 |
| 稳定性 | ✅ 稳定排序（相等元素不交换） |
| 适用场景 | 小规模数据或基本有序数据 |

## 练习4: 快速排序

### 分区过程可视化

**初始数组**: [6, 10, 13, 5, 8, 3, 2, 11]

**选择枢轴**: 通常选最后一个元素 11

### 分区过程

**目标**: 将数组分为 <11 和 >11 两部分

```
步骤跟踪（Lomuto分区方案）:

初始化: i = -1（小于区的边界）

j=0: 6 < 11? 是 → i=0, swap(arr[0], arr[0]) → [6, 10, 13, 5, 8, 3, 2, 11]
j=1: 10 < 11? 是 → i=1, swap(arr[1], arr[1]) → [6, 10, 13, 5, 8, 3, 2, 11]
j=2: 13 < 11? 否 → 继续
j=3: 5 < 11? 是 → i=2, swap(arr[2], arr[3]) → [6, 10, 5, 13, 8, 3, 2, 11]
j=4: 8 < 11? 是 → i=3, swap(arr[3], arr[4]) → [6, 10, 5, 8, 13, 3, 2, 11]
j=5: 3 < 11? 是 → i=4, swap(arr[4], arr[5]) → [6, 10, 5, 8, 3, 13, 2, 11]
j=6: 2 < 11? 是 → i=5, swap(arr[5], arr[6]) → [6, 10, 5, 8, 3, 2, 13, 11]

最后: swap(arr[i+1], arr[7]) → swap(arr[6], arr[7])
结果: [6, 10, 5, 8, 3, 2, 11, 13]
              ↑枢轴位置
```

**问题**: 枢轴 11 的最终位置在哪里？左半部分和右半部分各是什么？

<details>
<summary>✅ 答案</summary>

- **枢轴位置**: 索引 6
- **左半部分** (<11): [6, 10, 5, 8, 3, 2]
- **右半部分** (>11): [13]
</details>

### 递归树

```
                    [6,10,13,5,8,3,2,11]
                           │
              ┌────────────┼────────────┐
              │         枢轴11          │
    [6,10,5,8,3,2]              [13]
           │
    ┌──────┴──────┐
 [6,5,8,3,2]  [10]   (假设枢轴为2)
      │
   ┌──┴──┐
 [ ] [5,8,3,6] ...
```

**问题**: 快速排序什么情况下性能最差？如何避免？

<details>
<summary>✅ 答案</summary>

**最坏情况**: $O(n^2)$
- 已有序数组且总是选第一个/最后一个作为枢轴
- 所有元素相同

**优化策略**:
1. **随机化枢轴**: 随机选择枢轴元素
2. **三数取中**: 取头、中、尾的中位数作为枢轴
3. **尾递归优化**: 先处理较小的子数组
4. **混合策略**: 小数组切换到插入排序
</details>

## 练习5: 归并排序

### 分治过程

**原始数组**: [38, 27, 43, 3, 9, 82, 10]

### 分解过程（Divide）

```
[38, 27, 43, 3, 9, 82, 10]
        /        \
[38,27,43,3]  [9,82,10]
   /    \       /   \
[38,27][43,3] [9,82] [10]
  /  \   /  \   /  \   │
[38][27][43][3][9][82][10]  ← 基本情况（长度为1）
```

### 合并过程（Merge）

**问题**: 合并 [27, 38] 和 [3, 43]

<details>
<summary>📝 在此写出合并步骤</summary>

```
左: [27, 38]    右: [3, 43]
结果: []

1. 27 vs 3 → 取3 → 结果: [3]
2. 27 vs 43 → 取27 → 结果: [3, 27]
3. 38 vs 43 → 取38 → 结果: [3, 27, 38]
4. 左数组空，追加剩余 → 结果: [3, 27, 38, 43]
```
</details>

<details>
<summary>✅ 答案</summary>

合并结果: **[3, 27, 38, 43]**

**比较次数**: 3次（最后一个元素43无需比较，直接追加）
</details>

### 完整合并树

```
[38] [27] [43] [3] [9] [82] [10]
  \  /     \  /    \  /     │
[27,38]   [3,43]   [9,82] [10]
    \      /          \    /
 [3,27,38,43]      [9,10,82]
       \              /
  [3,9,10,27,38,43,82]
```

### 复杂度分析

**问题**: 归并排序的空间复杂度为什么是 $O(n)$ 而不是 $O(1)$？

<details>
<summary>✅ 答案</summary>

归并排序需要额外的数组来存储合并结果：

```python
def merge(left, right):
    result = []  # ← 需要额外空间
    # ... 合并逻辑
    return result
```

**空间复杂度构成**:
- 临时数组: $O(n)$
- 递归栈: $O(\log n)$
- 总计: $O(n)$

**注意**: 存在原地归并的变种，但代码复杂且常数较大。
</details>

## 练习6: 堆排序

### 堆的基本概念

**最大堆**: 父节点 ≥ 子节点
```
       16
      /  \
    14    10
    / \   / \
   8   7 9   3
  / \
 2   4
```

### 建堆过程

**初始数组**: [4, 10, 3, 5, 1]

**问题**: 从最后一个非叶子节点开始调整，写出建堆步骤

<details>
<summary>✅ 答案</summary>

```
初始完全二叉树:
      4
     / \
   10   3
  / \
 5   1

步骤1: 调整节点 10 (索引1)
      4
     / \
   10   3   ← 10 > 子节点，已是最大堆
  / \
 5   1

步骤2: 调整节点 4 (索引0)
      4           10
     / \    →     / \
   10   3        4   3
  / \           / \
 5   1         5   1

 继续调整节点 4:
      10
     /  \         10
    5    3   →    / \
   / \          5   3
  4   1        4   1

最终最大堆: [10, 5, 3, 4, 1]
```
</details>

### 排序过程

**堆排序步骤**:

| 步骤 | 堆状态 | 操作 | 已排序 |
|------|--------|------|--------|
| 初始 | [10, 5, 3, 4, 1] | 建堆完成 | [] |
| 1 | [5, 4, 3, 1] + [10] | 交换根与末尾，堆大小-1 | [10] |
| 2 | [4, 1, 3] + [5, 10] | 调整堆，交换 | [5, 10] |
| 3 | [3, 1] + [4, 5, 10] | 调整堆，交换 | [4, 5, 10] |
| 4 | [1] + [3, 4, 5, 10] | 调整堆，交换 | [3, 4, 5, 10] |
| 5 | [] + [1, 3, 4, 5, 10] | 完成 | [1, 3, 4, 5, 10] |

**问题**: 堆排序的时间复杂度和空间复杂度各是多少？

<details>
<summary>✅ 答案</summary>

- **建堆时间**: $O(n)$
- **每次调整**: $O(\log n)$
- **n次调整**: $O(n\log n)$
- **总时间复杂度**: $O(n\log n)$（最好、平均、最坏都相同）
- **空间复杂度**: $O(1)$（原地排序）
</details>

## 综合比较

### 排序算法对比表

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 |
|------|------|------|------|------|--------|
| 冒泡排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✅ |
| 选择排序 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ❌ |
| 插入排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✅ |
| 快速排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n^2)$ | $O(\log n)$ | ❌ |
| 归并排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(n)$ | ✅ |
| 堆排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(1)$ | ❌ |

### 选择题练习

**问题1**: 需要稳定排序且空间受限，应该选择？

<details>
<summary>✅ 答案</summary>

**插入排序**或**冒泡排序**

理由: $O(1)$ 空间，且是稳定排序。适合小规模数据。
</details>

**问题2**: 对大规模数据排序，要求最坏情况也是 $O(n\log n)$，应该选择？

<details>
<summary>✅ 答案</summary>

**归并排序**或**堆排序**

- 需要稳定排序 → 归并排序
- 空间受限 → 堆排序
</details>

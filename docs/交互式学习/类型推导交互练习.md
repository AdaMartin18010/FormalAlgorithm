## 练习1: 基础类型推导

### 简单常量

**问题1.1**: 以下表达式的类型是什么？

```
42        → ____
3.14      → ____
"hello"   → ____
true      → ____
()        → ____
```

<details>
<summary>✅ 答案</summary>

```
42        → Int
3.14      → Float
"hello"   → String
true      → Bool
()        → Unit
```
</details>

### 函数应用

**问题1.2**: 给定函数 `f : Int → Bool`，表达式 `f 42` 的类型是什么？

<details>
<summary>✅ 答案</summary>

**f 42 : Bool**

推导：
- f 期望 Int 类型的参数
- 42 是 Int 类型
- 匹配成功，返回类型为 Bool
</details>

**问题1.3**: 表达式 `f 3.14` 的类型是什么？

<details>
<summary>✅ 答案</summary>

**类型错误！**

- f 期望 Int 类型的参数
- 3.14 是 Float 类型
- 类型不匹配，编译错误
</details>

## 练习2: 多态与类型变量

### 多态函数

**问题2.1**: `identity` 函数的类型是什么？

```
λx.x
```

<details>
<summary>💡 提示</summary>

identity函数返回其参数不变，它可以工作于任何类型。
</details>

<details>
<summary>✅ 答案</summary>

**identity : ∀α. α → α**

或用另一种写法：**identity : 'a → 'a**

这意味着 identity 可以接受任何类型的参数，并返回相同类型的值。

示例：
- identity 42 : Int
- identity true : Bool
- identity (λx.x) : (α → α) → (α → α)
</details>

### 类型约束

**问题2.2**: 以下函数的类型签名中，哪些类型变量必须相同？

```
λf. λx. f (f x)
```

<details>
<summary>✅ 答案</summary>

分析步骤：
1. f 应用于 x，所以 f : α → β
2. f 又应用于 (f x)，而 f x : β，所以 β = α
3. 因此 f : α → α
4. 最终类型：**(α → α) → α → α**

**约束**: f 的输入类型和输出类型必须相同。
</details>

### 多态列表

**问题2.3**: 以下列表的类型是什么？

```
[1, 2, 3]         → ____
[true, false]     → ____
[]                → ____
[λx.x, λy.y]      → ____
```

<details>
<summary>✅ 答案</summary>

```
[1, 2, 3]         → List Int
[true, false]     → List Bool
[]                → List α（或 List 'a，多态空列表）
[λx.x, λy.y]      → List (α → α)
```
</details>

## 练习3: Hindley-Milner算法步骤

### 算法概述

Hindley-Milner类型推导算法核心步骤：

1. **分配类型变量**：为每个子表达式分配唯一的类型变量
2. **生成约束**：根据语法结构生成类型约束
3. **统一化(Unification)**：求解约束，得到最一般解
4. **泛化(Generalization)**：将自由类型变量变为全称量词

### 示例推导：λx. x + 1

**步骤 3.1: 分配类型变量**

```
λx. x + 1
│  │ │ │
│  │ │ └─ 分配 t3
│  │ └── 分配 t2
│  └──── 分配 t1
└─────── 分配 t4
```

当前映射：
- x : t1
- 1 : Int
- + : Int → Int → Int（已知）
- (x + 1) : t2
- (λx. x + 1) : t4

**步骤 3.2: 生成约束**

**问题**: 写出所有类型约束

<details>
<summary>✅ 答案</summary>

约束：
1. `+` 的第一个参数类型 = x 的类型 → **Int = t1**
2. `+` 的第二个参数类型 = 1 的类型 → **Int = Int**（显然成立）
3. `+` 的返回类型 = (x + 1) 的类型 → **Int = t2**
4. λ的类型 = 参数类型 → 返回类型 → **t4 = t1 → t2**

完整约束集合：{ t1 = Int, t2 = Int, t4 = t1 → t2 }
</details>

**步骤 3.3: 统一化求解**

**问题**: 求解约束后的最终类型是什么？

<details>
<summary>✅ 答案</summary>

代入：
- t1 = Int
- t2 = Int
- t4 = Int → Int

**最终类型**: **Int → Int**
</details>

### 练习：λf. λx. f x

**问题3.4**: 推导 `λf. λx. f x` 的类型

<details>
<summary>📝 逐步推导</summary>

**步骤1: 分配类型变量**
```
λf. λx. f x
│  │  │ │
│  │  │ └─ t3
│  │  └── t2
│  └───── t1
└──────── t4
```

**步骤2: 生成约束**
- f : t1, x : t2, (f x) : t3, (λx. f x) : t4, (λf. λx. f x) : t5
- 应用规则：f 是函数，t1 = t2 → t3
- λx 的类型：t4 = t2 → t3
- λf 的类型：t5 = t1 → t4

**步骤3: 求解**
- t1 = t2 → t3
- t4 = t2 → t3
- t5 = (t2 → t3) → (t2 → t3)

**最终类型**: **(α → β) → α → β**
</details>

<details>
<summary>✅ 验证答案</summary>

```
(λf. λx. f x) : (α → β) → α → β

示例应用:
- f = not : Bool → Bool
- x = true : Bool
- f x = not true = false : Bool
```
</details>

## 练习4: 统一化(Unification)

### 统一化规则

统一化算法尝试找到使两个类型相等的替换(Substitution)：

| 情况 | 处理 |
|------|------|
| unify(α, α) | 成功，无需替换 |
| unify(α, τ) 且 α ∉ FV(τ) | 替换 [α ↦ τ] |
| unify(τ, α) 且 α ∉ FV(τ) | 替换 [α ↦ τ] |
| unify(C τ₁...τₙ, C σ₁...σₙ) | unify(τ₁, σ₁), ..., unify(τₙ, σₙ) |
| unify(τ₁→τ₂, σ₁→σ₂) | unify(τ₁, σ₁), unify(τ₂, σ₂) |
| 其他 | 失败（类型不匹配） |

### 统一化练习

**问题4.1**: unify(α → Int, Bool → β)

<details>
<summary>✅ 答案</summary>

分解函数类型：
- unify(α, Bool) → **α = Bool**
- unify(Int, β) → **β = Int**

**替换**: [α ↦ Bool, β ↦ Int]
</details>

**问题4.2**: unify(List α, List (Int → β))

<details>
<summary>✅ 答案</summary>

分解 List：
- unify(α, Int → β)

**替换**: [α ↦ Int → β]

注意：β 仍然是自由变量。
</details>

**问题4.3**: unify(α → α, Int → Bool)

<details>
<summary>✅ 答案</summary>

**失败！**

推导：
- unify(α, Int) → α = Int
- unify(α, Bool) → α = Bool

矛盾：α 不能同时等于 Int 和 Bool。

**这正是一个类型错误！**
</details>

### 出现检查(Occurs Check)

**问题4.4**: unify(α, α → Int)

<details>
<summary>✅ 答案</summary>

**失败！出现检查失败。**

α 出现在右侧类型的自由变量中，这会导致无限类型：
α = α → Int = (α → Int) → Int = ((α → Int) → Int) → Int = ...

这种无限类型在标准类型系统中被禁止。
</details>

## 练习5: Let多态

### Let绑定的类型推导

标准 λ 演算：
```
(λx. x x) (λy. y)  # 类型错误！x 不能同时作为函数和参数
```

Let 多态允许：
```
let id = λx. x in (id 1, id true)  # 合法！id 被实例化为不同类型
```

### 推导示例

**问题5.1**: 推导 `let id = λx. x in id 42`

<details>
<summary>✅ 答案</summary>

**步骤1: 推导 λx. x 的类型**
- id : ∀α. α → α

**步骤2: 实例化**
- 在 id 42 中，id 被实例化为 Int → Int

**步骤3: 推导结果**
- **Int**
</details>

**问题5.2**: 推导 `let id = λx. x in (id 1, id true)`

<details>
<summary>✅ 答案</summary>

**步骤1: 推导 λx. x 的类型**
- id : ∀α. α → α

**步骤2: 两次实例化**
- id 1 : id 实例化为 Int → Int → **Int**
- id true : id 实例化为 Bool → Bool → **Bool**

**步骤3: 推导结果**
- **(Int, Bool)**
</details>

### 泛化(Generalization)

**问题5.3**: 为什么 `λx. x x` 是类型错误的，而 let 绑定可以解决这个问题？

<details>
<summary>✅ 答案</summary>

**λx. x x 的错误分析**:
- 设 x : α
- 第一次出现 x（作为函数）：需要 x : β → γ
- 第二次出现 x（作为参数）：x : β
- 约束：α = β → γ 且 α = β
- 导致 β → γ = β，出现检查失败！

**Let 多态的解决**:
```
let f = λx. x in f f  # 合法！
```
- f : ∀α. α → α（泛化后的多态类型）
- 第一个 f 实例化为 (β → β) → (β → β)
- 第二个 f 实例化为 β → β
- 不再要求类型相等，只要求可以统一化
</details>

## 练习6: 复杂表达式推导

### 练习6.1: 递归函数

**问题**: 推导阶乘函数的 let rec 类型

```
let rec fact n = 
  if n = 0 then 1 
  else n * fact (n - 1)
```

<details>
<summary>✅ 答案</summary>

**分析**:
- n 与 0 比较，n = 0 : Bool，所以 **n : Int**
- 返回 1（Int）或 n * fact (n-1)
- n - 1 : Int（确认 n 是 Int）
- fact 返回 Int（用于乘法）

**最终类型**: **fact : Int → Int**
</details>

### 练习6.2: 高阶函数组合

**问题**: 推导 compose 函数的类型

```
let compose f g x = f (g x)
```

<details>
<summary>📝 逐步推导</summary>

**步骤1: 分配类型变量**
- f : t1, g : t2, x : t3, (g x) : t4, (f (g x)) : t5, compose : t6

**步骤2: 生成约束**
- g x : g 是函数，t2 = t3 → t4
- f (g x) : f 是函数，t1 = t4 → t5
- compose f g x : t6 = t1 → t2 → t3 → t5

**步骤3: 整理**
- t2 = t3 → t4
- t1 = t4 → t5
- t6 = (t4 → t5) → (t3 → t4) → t3 → t5

**重命名**: α = t3, β = t4, γ = t5
</details>

<details>
<summary>✅ 答案</summary>

**compose : (β → γ) → (α → β) → α → γ**

或写成：
**compose : ∀α,β,γ. (β → γ) → (α → β) → α → γ**

这正是函数组合的数学定义：(f ∘ g)(x) = f(g(x))
</details>

### 练习6.3: 列表操作

**问题**: 推导 map 函数的类型

```
let rec map f xs = 
  match xs with
  | [] -> []
  | x::xs' -> f x :: map f xs'
```

<details>
<summary>✅ 答案</summary>

**分析**:
- xs 是列表，设 xs : List α
- x : α, xs' : List α
- f x : 设 f : α → β，则 f x : β
- f x :: map f xs' : map 返回 List β，所以整体是 List β
- [] : List β（与 match 分支一致）

**最终类型**: **map : (α → β) → List α → List β**
</details>

### 练习6.4: 挑战 - Y组合子

**问题**: 以下表达式能否赋予类型？

```
Y = λf. (λx. f (x x)) (λx. f (x x))
```

<details>
<summary>✅ 答案</summary>

**不能赋予简单类型！**

分析：
- x x 要求 x : α → β 且 x : α
- 所以 α = α → β
- 出现检查失败！

**Y组合子**用于实现递归，但在简单类型λ演算中不可类型化。

**解决方案**:
1. 使用 let rec 而非 Y 组合子
2. 或扩展类型系统支持递归类型：α = μα. α → β
</details>

## 常见类型推导错误

### 错误案例分析

| 表达式 | 错误原因 | 正确理解 |
|--------|----------|----------|
| `λx. x x` | 出现检查失败 | x 不能应用到自身 |
| `[1, true]` | 类型不统一 | 列表元素类型必须一致 |
| `if true then 1 else "hi"` | 分支类型不同 | if 的两个分支类型必须相同 |
| `let f = λx. x in f f f` | 实际合法！ | f 多态，可实例化为不同类型 |

### 类型推导 vs 类型检查

| | 类型推导 | 类型检查 |
|--|----------|----------|
| 输入 | 无类型标注的表达式 | 有类型标注的表达式 |
| 输出 | 推导出的类型 | 是否类型正确 |
| 复杂度 | 指数级（某些扩展） | 多项式级 |
| 应用场景 | ML/Haskell | Java/C++ |

## 总结

### Hindley-Milner算法要点

1. **类型变量**: 用希腊字母 α, β, γ 表示未知类型
2. **约束生成**: 语法结构决定类型约束
3. **统一化**: Robinson 统一化算法求解约束
4. **最一般合一(MGU)**: 找到最一般的类型（非特化）
5. **泛化**: 将自由变量变为全称量词 ∀
6. **实例化**: 用具体类型替换全称量词

### 重要性质

- **可靠性(Soundness)**: 推导出的类型保证运行时无类型错误
- **完备性(Completeness)**: 如果表达式有类型，算法能找到最一般类型
- **主类型(Principal Type)**: 推导出的类型是最一般的（可实例化为所有其他合法类型）

## 练习1: 图的表示

### 邻接矩阵 vs 邻接表

给定无向图：

```
    0 --- 1
    │   ╱ │
    │  ╱  │
    │ ╱   │
    2 --- 3
```

### 问题1.1: 邻接矩阵

**问题**: 写出该图的邻接矩阵

<details>
<summary>📝 在此填写</summary>

```
     0   1   2   3
   ┌───┬───┬───┬───┐
 0 │   │   │   │   │
   ├───┼───┼───┼───┤
 1 │   │   │   │   │
   ├───┼───┼───┼───┤
 2 │   │   │   │   │
   ├───┼───┼───┼───┤
 3 │   │   │   │   │
   └───┴───┴───┴───┘
```

</details>

<details>
<summary>✅ 答案</summary>

```
     0   1   2   3
   ┌───┬───┬───┬───┐
 0 │ 0 │ 1 │ 1 │ 0 │
   ├───┼───┼───┼───┤
 1 │ 1 │ 0 │ 1 │ 1 │
   ├───┼───┼───┼───┤
 2 │ 1 │ 1 │ 0 │ 1 │
   ├───┼───┼───┼───┤
 3 │ 0 │ 1 │ 1 │ 0 │
   └───┴───┴───┴───┘
```

</details>

### 问题1.2: 邻接表

**问题**: 写出该图的邻接表表示

<details>
<summary>✅ 答案</summary>

```
0: [1, 2]
1: [0, 2, 3]
2: [0, 1, 3]
3: [1, 2]
```

</details>

### 问题1.3: 复杂度比较

| 操作 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| 判断边(u,v)是否存在 | O(1) | O(degree(u)) |
| 遍历所有边 | O(V²) | O(V+E) |
| 空间复杂度 | O(V²) | O(V+E) |
| 适合的场景 | 稠密图 | 稀疏图 |

**问题**: 什么情况下用邻接矩阵更好？

<details>
<summary>✅ 答案</summary>

**邻接矩阵更适合**：

- 稠密图（边数接近 V²）
- 需要频繁判断两点间是否有边
- 图比较小
- 需要矩阵运算（如Floyd算法）

</details>

## 练习3: DFS深度优先搜索

### 初始图

```
    0 --- 1 --- 3
    │   ╱
    │  ╱
    2
```

### DFS递归过程

#### 步骤3.1: 从节点0开始DFS

```
DFS(0):
  标记0为已访问
  访问邻居1 → 递归DFS(1)
```

**问题**: 画出DFS(1)的调用过程

<details>
<summary>✅ 答案</summary>

```
DFS(1):
  标记1为已访问
  访问邻居0 → 已访问，跳过
  访问邻居2 → 递归DFS(2)
    DFS(2):
      标记2为已访问
      访问邻居0 → 已访问，跳过
      访问邻居1 → 已访问，跳过
      返回
  访问邻居3 → 递归DFS(3)
    DFS(3):
      标记3为已访问
      访问邻居1 → 已访问，跳过
      返回
  返回
```

</details>

#### DFS遍历顺序

**问题**: 按照上述过程，DFS遍历顺序是什么？（假设邻居按编号从小到大访问）

<details>
<summary>✅ 答案</summary>

**DFS遍历顺序**: [0, 1, 2, 3]

**递归调用栈**:

```
DFS(0)
  └── DFS(1)
        └── DFS(2)
        └── DFS(3)
```

</details>

### 非递归DFS（使用栈）

**问题**: 用栈模拟上述DFS过程

<details>
<summary>✅ 答案</summary>

| 步骤 | 操作 | 栈状态 | 输出 |
|------|------|--------|------|
| 1 | push(0) | [0] | - |
| 2 | pop() → 0, push(1), push(2) | [2,1] | 0 |
| 3 | pop() → 1, push(3) | [2,3] | 0,1 |
| 4 | pop() → 3 | [2] | 0,1,3 |
| 5 | pop() → 2 | [] | 0,1,3,2 |

**注意**: 栈的顺序与递归顺序可能不同，取决于邻居的入栈顺序。
</details>

### DFS应用：检测环

**问题**: 如何用DFS检测无向图中的环？

<details>
<summary>✅ 答案</summary>

**方法**：

1. 维护一个`visited`数组标记是否访问过
2. 维护一个`parent`记录当前节点的父节点
3. 遍历邻居时：
   - 如果邻居未访问，递归访问，并设置父节点
   - 如果邻居已访问且不是父节点，**发现环！**

```python
def has_cycle_dfs(graph, u, visited, parent):
    visited[u] = True
    for v in graph[u]:
        if not visited[v]:
            if has_cycle_dfs(graph, v, visited, u):
                return True
        elif v != parent:  # 发现回边
            return True
    return False
```

</details>

## 练习4: Dijkstra最短路径

### 初始图（带权有向图）

```
      4       1
  0 ───→ 1 ───→ 2
  │      │       │
  2      3       5
  ↓      ↓       ↓
  3 ←─── 4 ───→ 5
      2      1
```

从节点 0 到所有其他节点的最短路径。

### 初始化

| 节点 | 0 | 1 | 2 | 3 | 4 | 5 |
|------|---|---|---|---|---|---|
| dist | 0 | ∞ | ∞ | ∞ | ∞ | ∞ |
| visited | F | F | F | F | F | F |

优先队列 (距离, 节点): [(0, 0)]

### 步骤跟踪

#### 第1轮: 处理节点0

**问题**: 更新哪些节点的距离？

<details>
<summary>✅ 答案</summary>

- 节点0的邻居：1（权重4），3（权重2）
- dist[1] = min(∞, 0+4) = 4
- dist[3] = min(∞, 0+2) = 2

| 节点 | 0 | 1 | 2 | 3 | 4 | 5 |
|------|---|---|---|---|---|---|
| dist | 0 | **4** | ∞ | **2** | ∞ | ∞ |
| visited | T | F | F | F | F | F |

**优先队列**: [(2, 3), (4, 1)]
</details>

#### 第2轮: 处理节点3

**问题**: 节点3有出边吗？需要更新吗？

<details>
<summary>✅ 答案</summary>

- 节点3没有出边（根据图示）
- 无需更新

| 节点 | 0 | 1 | 2 | 3 | 4 | 5 |
|------|---|---|---|---|---|---|
| dist | 0 | 4 | ∞ | 2 | ∞ | ∞ |
| visited | T | F | F | **T** | F | F |

**优先队列**: [(4, 1)]
</details>

#### 第3轮: 处理节点1

**问题**: 从节点1出发，更新哪些节点？

<details>
<summary>✅ 答案</summary>

- 节点1的邻居：2（权重1），4（权重3）
- dist[2] = min(∞, 4+1) = 5
- dist[4] = min(∞, 4+3) = 7

| 节点 | 0 | 1 | 2 | 3 | 4 | 5 |
|------|---|---|---|---|---|---|
| dist | 0 | 4 | **5** | 2 | **7** | ∞ |
| visited | T | **T** | F | T | F | F |

**优先队列**: [(5, 2), (7, 4)]
</details>

#### 完整过程

<details>
<summary>✅ 完整步骤</summary>

| 轮次 | 当前节点 | 距离更新 | dist数组 | 优先队列 |
|------|----------|----------|----------|----------|
| 0 | - | - | [0,∞,∞,∞,∞,∞] | [(0,0)] |
| 1 | 0 | 1→4, 3→2 | [0,4,∞,2,∞,∞] | [(2,3),(4,1)] |
| 2 | 3 | 无 | [0,4,∞,2,∞,∞] | [(4,1)] |
| 3 | 1 | 2→5, 4→7 | [0,4,5,2,7,∞] | [(5,2),(7,4)] |
| 4 | 2 | 5→10 | [0,4,5,2,7,10] | [(7,4),(10,5)] |
| 5 | 4 | 3→9(不更新), 5→8(更新) | [0,4,5,2,7,**8**] | [(8,5)] |
| 6 | 5 | 无 | [0,4,5,2,7,8] | [] |

**最终最短距离**: [0, 4, 5, 2, 7, 8]
</details>

### 路径回溯

**问题**: 从0到5的最短路径是什么？

<details>
<summary>✅ 答案</summary>

维护`prev`数组记录前驱节点：

| 节点 | 0 | 1 | 2 | 3 | 4 | 5 |
|------|---|---|---|---|---|---|
| prev | - | 0 | 1 | 0 | 1 | 4 |

**路径回溯**: 5 ← 4 ← 1 ← 0

**最短路径**: 0 → 1 → 4 → 5，长度 = 4 + 3 + 1 = 8 ✓
</details>

### Dijkstra限制

**问题**: Dijkstra算法不能处理什么类型的边？

<details>
<summary>✅ 答案</summary>

**不能处理负权边！**

**原因**: Dijkstra基于贪心策略，一旦确定某个节点的最短距离，就不会再更新。但负权边可能导致已经确定的最短距离不是真正的最短。

**反例**:

```
0 → 1 (权重5)
0 → 2 (权重3)
2 → 1 (权重-4)  # 负权边！
```

- Dijkstra会先确定dist[1]=5
- 但实际最短路径 0→2→1 = 3+(-4) = -1

**负权图用**: Bellman-Ford 或 SPFA 算法
</details>

## 练习5: Floyd-Warshall全源最短路径

### 初始距离矩阵

```
     0    1    2    3
   ┌───┬───┬───┬───┐
 0 │ 0 │ 3 │ ∞ │ 7 │
   ├───┼───┼───┼───┤
 1 │ 8 │ 0 │ 2 │ ∞ │
   ├───┼───┼───┼───┤
 2 │ 5 │ ∞ │ 0 │ 1 │
   ├───┼───┼───┼───┤
 3 │ 2 │ ∞ │ ∞ │ 0 │
   └───┴───┴───┴───┘
```

### 算法思想

$$dist[i][j] = \min(dist[i][j], dist[i][k] + dist[k][j])$$

对于每个中间节点 k，尝试更新所有 i→j 的距离。

### 步骤跟踪

#### k=0: 以节点0为中间点

**问题**: 哪些距离可以被更新？

<details>
<summary>✅ 答案</summary>

检查所有 i,j，看是否经过0更短：

- dist[1][3] = min(∞, 8+7) = 15
- dist[2][1] = min(∞, 5+3) = 8
- dist[3][1] = min(∞, 2+3) = 5
- dist[3][2] = min(∞, 2+∞) = ∞

```
     0    1    2    3
   ┌───┬───┬───┬───┐
 0 │ 0 │ 3 │ ∞ │ 7 │
   ├───┼───┼───┼───┤
 1 │ 8 │ 0 │ 2 │15 │
   ├───┼───┼───┼───┤
 2 │ 5 │ 8 │ 0 │ 1 │
   ├───┼───┼───┼───┤
 3 │ 2 │ 5 │ ∞ │ 0 │
   └───┴───┴───┴───┘
```

</details>

#### k=1: 以节点1为中间点

**问题**: 更新哪些距离？

<details>
<summary>✅ 答案</summary>

- dist[0][2] = min(∞, 3+2) = 5
- dist[0][3] = min(7, 3+15) = 7（不更新）
- dist[2][3] = min(1, 8+15) = 1（不更新）
- dist[3][2] = min(∞, 5+2) = 7

```
     0    1    2    3
   ┌───┬───┬───┬───┐
 0 │ 0 │ 3 │ 5 │ 7 │
   ├───┼───┼───┼───┤
 1 │ 8 │ 0 │ 2 │15 │
   ├───┼───┼───┼───┤
 2 │ 5 │ 8 │ 0 │ 1 │
   ├───┼───┼───┼───┤
 3 │ 2 │ 5 │ 7 │ 0 │
   └───┴───┴───┴───┘
```

</details>

#### 完整过程

<details>
<summary>✅ 最终结果</summary>

```
     0    1    2    3
   ┌───┬───┬───┬───┐
 0 │ 0 │ 3 │ 5 │ 6 │
   ├───┼───┼───┼───┤
 1 │ 5 │ 0 │ 2 │ 3 │
   ├───┼───┼───┼───┤
 2 │ 3 │ 6 │ 0 │ 1 │
   ├───┼───┼───┼───┤
 3 │ 2 │ 5 │ 7 │ 0 │
   └───┴───┴───┴───┘
```

**检测负环**: 如果任意 dist[i][i] < 0，则存在负环。
</details>

## 练习6: 最小生成树(MST)

### 初始图

```
      1       3
  A ───── B ───── C
  │       │       │
 4│      2│       │5
  │       │       │
  D ───── E ───── F
      6       4
```

边列表（权重，u，v）：

- (1, A, B), (2, B, E), (3, B, C), (4, A, D), (4, E, F), (5, C, F), (6, D, E)

### Kruskal算法

**核心思想**: 按权重从小到大选边，不形成环就加入。

#### 步骤跟踪

| 步骤 | 考虑边 | 权重 | 是否加入 | 原因 | 当前边集 |
|------|--------|------|----------|------|----------|
| 1 | (A,B) | 1 | ✅ | 无环 | {(A,B)} |
| 2 | (B,E) | 2 | ✅ | 无环 | {(A,B),(B,E)} |
| 3 | (B,C) | 3 | ✅ | 无环 | {(A,B),(B,E),(B,C)} |
| 4 | (A,D) | 4 | ✅ | 无环 | + (A,D) |
| 5 | (E,F) | 4 | ✅ | 无环 | + (E,F) |
| 6 | (C,F) | 5 | ❌ | 形成环 C-B-E-F-C | - |
| 7 | (D,E) | 6 | ❌ | 形成环 D-A-B-E-D | - |

**问题**: 如何判断是否形成环？

<details>
<summary>✅ 答案</summary>

**使用并查集(Union-Find)**：

```
初始: {A}, {B}, {C}, {D}, {E}, {F}（各自独立集合）

加入(A,B): 合并 → {A,B}
加入(B,E): 合并 → {A,B,E}
加入(B,C): 合并 → {A,B,C,E}
加入(A,D): 合并 → {A,B,C,D,E}
加入(E,F): 合并 → {A,B,C,D,E,F}

加入(C,F): C和F已经在同一集合，会形成环！
```

</details>

#### MST结果

**MST总权重**: 1 + 2 + 3 + 4 + 4 = **14**

**MST边**: (A,B), (B,E), (B,C), (A,D), (E,F)

### Prim算法

**核心思想**: 从一个节点开始，每次选择与已选集合相连的最小权重边。

#### 从节点A开始

| 步骤 | 已选集合 | 候选边（权重，目标） | 选择 | MST权重 |
|------|----------|---------------------|------|---------|
| 1 | {A} | (1,B), (4,D) | (1,B) | 1 |
| 2 | {A,B} | (4,D), (2,E), (3,C) | (2,E) | 3 |
| 3 | {A,B,E} | (4,D), (3,C), (4,F), (6,D) | (3,C) | 6 |
| 4 | {A,B,E,C} | (4,D), (4,F), (5,F), (6,D) | (4,D) | 10 |
| 5 | {A,B,E,C,D} | (4,F), (5,F), (6,D已选) | (4,F) | 14 |

**结果与Kruskal相同**，总权重 = 14

### 算法对比

| 特性 | Kruskal | Prim |
|------|---------|------|
| 时间复杂度 | O(E log E) | O(V²) 或 O(E log V)（堆优化）|
| 适合场景 | 稀疏图 | 稠密图 |
| 实现难度 | 需要并查集 | 类似Dijkstra |
| 起始点 | 无要求 | 需要指定起始点 |

### 练习6.1: 图的遍历顺序

**问题**: 从节点1开始，写出一种可能的DFS和BFS遍历顺序。

**DFS**（假设邻居按编号访问）: 1 → 2 → 4 → 3

**注意**: DFS顺序不唯一，取决于邻居访问顺序。

### 练习6.2: 最短路径分析

给定带权图：

```
    A --2-- B --3-- C
    │       │       │
    1       1       1
    │       │       │
    D --1-- E --2-- F
```

**问题**: 从A到C的最短路径是什么？Dijkstra会选哪条？

<details>
<summary>✅ 答案</summary>

**最短路径**: A → D → E → F → C，权重 = 1 + 1 + 2 + 1 = 5

或 A → D → E → B → C，权重 = 1 + 1 + 1 + 3 = 6

**最短**: A → D → E → F → C（权重5）

## 总结

### 图算法选择指南

| 问题 | 推荐算法 | 时间复杂度 |
|------|----------|-----------|
| 无权图最短路径 | BFS | O(V+E) |
| 单源最短路径（正权） | Dijkstra | O((V+E)logV) |
| 单源最短路径（负权） | Bellman-Ford | O(VE) |
| 全源最短路径 | Floyd-Warshall | O(V³) |
| 最小生成树 | Kruskal/Prim | O(ElogE) / O(ElogV) |
| 拓扑排序 | Kahn算法/DFS | O(V+E) |
| 强连通分量 | Tarjan/Kosaraju | O(V+E) |

### 常见错误

1. ❌ 在负权图上使用Dijkstra
2. ❌ DFS求最短路径（DFS不保证最短）
3. ❌ BFS用于带权图（应用Dijkstra）
4. ❌ 忘记处理重边
5. ❌ 无向图只加一条边

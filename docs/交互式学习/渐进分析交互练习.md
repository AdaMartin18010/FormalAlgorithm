## 练习1: 复杂度分析基础

### 问题描述

分析以下算法的渐进复杂度，并说明你的推理过程。

```python
def algorithm_1(n):
    """算法1"""
    total = 0
    for i in range(n):
        for j in range(n):
            total += i * j
    return total
```

### 交互步骤

#### 步骤 1: 确定基本操作

**问题**: 上述算法中，基本操作是什么？执行了多少次？

<details>
<summary>💡 提示</summary>

- 基本操作通常是执行最频繁的语句
- 考虑嵌套循环的执行次数

</details>

<details>
<summary>✅ 答案</summary>

**基本操作**: `total += i * j`

**执行次数**: 外层循环执行 n 次，内层循环每次也执行 n 次，所以基本操作执行了 $n \times n = n^2$ 次。
</details>

#### 步骤 2: 确定时间复杂度

**问题**: 该算法的时间复杂度是什么？

<details>
<summary>✅ 答案</summary>

时间复杂度为 **$O(n^2)$**（平方复杂度）

**解释**:

- 双层嵌套循环，每层都执行 n 次
- 总操作次数为 $n^2$
- 忽略常数因子和低阶项

</details>

#### 步骤 3: 空间复杂度

**问题**: 该算法的空间复杂度是什么？

<details>
<summary>✅ 答案</summary>

空间复杂度为 **$O(1)$**（常数空间）

**解释**:

- 只使用了固定数量的变量：`total`, `i`, `j`
- 不随输入规模 n 的增长而增长

</details>

### 常见错误

| 错误 | 正确理解 |
|------|----------|
| ❌ 认为空间复杂度是 $O(n)$ | ✅ 空间复杂度只考虑额外使用的空间，不是输入大小 |
| ❌ 认为时间复杂度是 $O(n)$ | ✅ 嵌套循环的复杂度需要相乘 |
| ❌ 忽略常数项后得到 $O(n^2 + n)$ | ✅ 渐进分析只保留最高阶项 |

## 练习2: 填空练习

### 问题2.1: 单层循环

```python
def sum_array(arr):
    n = len(arr)
    total = 0
    for i in range(n):      # 执行 ____ 次
        total += arr[i]     # 基本操作
    return total
```

<details>
<summary>✅ 答案</summary>

**填空**: `n` 次

**时间复杂度**: $O(n)$

**解释**: 循环遍历数组一次，每个元素访问一次。
</details>

### 问题2.2: 对数复杂度

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:            # 循环多少次？
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

<details>
<summary>💡 提示</summary>

每次循环搜索范围减半，这是一个经典的____问题。
</details>

<details>
<summary>✅ 答案</summary>

**循环次数**: 最多 $\log_2 n$ 次

**时间复杂度**: $O(\log n)$

**解释**: 每次迭代都将搜索范围减半，所以最多需要 $\log_2 n$ 次迭代才能将范围缩小到1。
</details>

### 问题2.3: 多重循环

```python
def triple_loop(n):
    count = 0
    for i in range(n):
        for j in range(i, n):
            for k in range(j, n):
                count += 1
    return count
```

**问题**: 这个算法的时间复杂度是？

<details>
<summary>💡 提示</summary>

注意内层循环的起始条件，不是从0开始。
</details>

<details>
<summary>✅ 答案</summary>

**时间复杂度**: $O(n^3)$

**详细分析**:

- 当 $i=0$ 时，$j$ 从 0 到 $n-1$，$k$ 从 $j$ 到 $n-1$
- 执行次数约为 $\sum_{i=0}^{n-1} \sum_{j=i}^{n-1} (n-j)$
- 化简后得到 $O(n^3)$

**精确公式**: $\frac{n(n+1)(n+2)}{6}$
</details>

## 练习3: 复杂度比较

### 问题描述

判断以下各组函数的渐进关系，在空格中填入 $<$, $>$, 或 $=$。

### 3.1 比较练习

| 函数 f(n) | 关系 | 函数 g(n) |
|-----------|------|-----------|
| $100n$ | ____ | $n^2$ |
| $\log_2 n$ | ____ | $\log_{10} n$ |
| $2^n$ | ____ | $n!$ |
| $n\log n$ | ____ | $n^{1.5}$ |
| $2^{2n}$ | ____ | $4^n$ |

<details>
<summary>✅ 答案与解释</summary>

| 函数 f(n) | 关系 | 函数 g(n) | 解释 |
|-----------|------|-----------|------|
| $100n$ | $<$ | $n^2$ | 线性 vs 平方，当 $n > 100$ 时 $n^2$ 更大 |
| $\log_2 n$ | $=$ | $\log_{10} n$ | 对数底数不同只差常数因子，$\log_2 n = \frac{\log_{10} n}{\log_{10} 2}$ |
| $2^n$ | $<$ | $n!$ | 阶乘增长快于指数，斯特林公式：$n! \approx \sqrt{2\pi n}(\frac{n}{e})^n$ |
| $n\log n$ | $<$ | $n^{1.5}$ | $n^{1.5} = n \cdot n^{0.5}$，当 $n > 4$ 时 $n^{0.5} > \log n$ |
| $2^{2n}$ | $=$ | $4^n$ | $2^{2n} = (2^2)^n = 4^n$ |

</details>

### 3.2 排序算法比较

**问题**: 将以下排序算法按最坏情况时间复杂度从低到高排序：

A. 快速排序  B. 冒泡排序  C. 归并排序  D. 插入排序  E. 堆排序

<details>
<summary>💡 提示</summary>

回顾各算法的复杂度：

- 快速排序最坏情况：$O(n^2)$
- 冒泡排序：$O(n^2)$
- 归并排序：$O(n\log n)$
- 插入排序最坏情况：$O(n^2)$
- 堆排序：$O(n\log n)$

</details>

<details>
<summary>✅ 答案</summary>

**从低到高排序**:

**并列第一**: C (归并排序) = E (堆排序) → $O(n\log n)$

**并列第二**: A (快速排序) = B (冒泡排序) = D (插入排序) → $O(n^2)$

**注意**: 虽然都是 $O(n^2)$，但实际效率：快速排序平均 $O(n\log n)$ > 插入排序 > 冒泡排序
</details>

## 练习4: 递推关系求解

### 问题4.1: 归并排序递推式

归并排序的时间复杂度可以用以下递推关系表示：

$$T(n) = 2T(\frac{n}{2}) + O(n)$$

**问题**: 使用主定理求解此递推关系。

<details>
<summary>💡 提示：主定理</summary>

对于递推式 $T(n) = aT(\frac{n}{b}) + f(n)$，其中 $a \geq 1$, $b > 1$：

1. 若 $f(n) = O(n^{\log_b a - \epsilon})$，则 $T(n) = \Theta(n^{\log_b a})$
2. 若 $f(n) = \Theta(n^{\log_b a})$，则 $T(n) = \Theta(n^{\log_b a} \log n)$
3. 若 $f(n) = \Omega(n^{\log_b a + \epsilon})$，则 $T(n) = \Theta(f(n))$

</details>

<details>
<summary>✅ 答案</summary>

**参数识别**:

- $a = 2$（子问题个数）
- $b = 2$（问题缩小因子）
- $f(n) = O(n)$（合并操作的时间）

**计算**: $\log_b a = \log_2 2 = 1$

**比较**: $f(n) = O(n) = O(n^1) = O(n^{\log_b a})$

**应用主定理情况2**:

$$T(n) = \Theta(n^{\log_b a} \log n) = \Theta(n \log n)$$

</details>

### 问题4.2: 递归树练习

求解递推关系：$T(n) = 3T(\frac{n}{4}) + n^2$

**步骤1**: 画出递归树的前三层

<details>
<summary>📝 在此画图</summary>

```
                    n²
                  /  |  \
            (n/4)² (n/4)² (n/4)²
            / | \   / | \   / | \
         ... ... ... ... ... ... ...
```

</details>

**步骤2**: 计算每层的代价

<details>
<summary>✅ 答案</summary>

| 层数 | 节点数 | 每节点代价 | 该层总代价 |
|------|--------|-----------|-----------|
| 0 | 1 | $n^2$ | $n^2$ |
| 1 | 3 | $(\frac{n}{4})^2$ | $3 \cdot \frac{n^2}{16} = \frac{3}{16}n^2$ |
| 2 | 9 | $(\frac{n}{16})^2$ | $9 \cdot \frac{n^2}{256} = (\frac{3}{16})^2 n^2$ |
| k | $3^k$ | $(\frac{n}{4^k})^2$ | $(\frac{3}{16})^k n^2$ |

总代价：
$$T(n) = \sum_{k=0}^{\log_4 n} \left(\frac{3}{16}\right)^k n^2$$

这是一个等比数列，公比 $r = \frac{3}{16} < 1$，所以收敛。

$$T(n) = n^2 \cdot \frac{1}{1 - \frac{3}{16}} = \frac{16}{13}n^2 = O(n^2)$$
</details>

## 练习5: 综合分析挑战

### 挑战问题

分析以下代码的时间复杂度：

```python
def mystery(n):
    if n <= 1:
        return 1

    result = 0
    for i in range(n):
        result += mystery(i)  # 递归调用

    return result
```

### checkpoints

#### Checkpoint 1: 建立递推关系

**问题**: 写出该算法的时间复杂度递推式。

<details>
<summary>✅ 答案</summary>

$$T(n) = \sum_{i=0}^{n-1} T(i) + O(n)$$

或展开写：
$$T(n) = T(0) + T(1) + T(2) + ... + T(n-1) + O(n)$$
</details>

#### Checkpoint 2: 求解

**问题**: 求解该递推关系。

<details>
<summary>💡 提示</summary>

考虑 $T(n)$ 与 $T(n-1)$ 的关系：

$T(n-1) = T(0) + T(1) + \cdots + T(n-2) + O(n-1)$

两式相减...
</details>

<details>
<summary>✅ 答案</summary>

由递推式：

- $T(n) = T(0) + T(1) + \cdots + T(n-1) + O(n)$
- $T(n-1) = T(0) + T(1) + \cdots + T(n-2) + O(n-1)$

两式相减：
$$T(n) - T(n-1) = T(n-1) + O(1)$$

$$T(n) = 2T(n-1) + O(1)$$

展开得：
$$T(n) = 2^n \cdot T(0) + O(2^n) = O(2^n)$$

**时间复杂度**: $O(2^n)$
</details>

#### Checkpoint 3: 优化思路

**问题**: 如何优化这个算法？

<details>
<summary>✅ 答案</summary>

**观察**: 这个函数实际上计算的是**贝尔数**（Bell Number）相关的值。

**优化方法 - 记忆化**:

```python
def mystery_optimized(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return 1

    result = 0
    for i in range(n):
        result += mystery_optimized(i, memo)

    memo[n] = result
    return result
```

**优化后复杂度**: $O(n^2)$

**进一步优化**: 可以使用动态规划，时间复杂度仍为 $O(n^2)$，但空间复杂度为 $O(n)$。
</details>

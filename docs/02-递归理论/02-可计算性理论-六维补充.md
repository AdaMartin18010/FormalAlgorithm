## 一、概念定义 (Concept Definition)

### 1.1 计算模型

**定义 1.1.1** (图灵机 / Turing Machine)

图灵机是七元组 $M = (Q, \Sigma, \Gamma, \delta, q_0, q_{accept}, q_{reject})$：

- $Q$：有限状态集
- $\Sigma$：输入字母表（不含空白符 $\sqcup$）
- $\Gamma$：带字母表，$\Sigma \subset \Gamma$，$\sqcup \in \Gamma$
- $\delta: Q \times \Gamma \rightarrow Q \times \Gamma \times \{L, R\}$：转移函数
- $q_0 \in Q$：初始状态
- $q_{accept} \in Q$：接受状态
- $q_{reject} \in Q$：拒绝状态，$q_{accept} \neq q_{reject}$

**定义 1.1.2** (图灵可识别 / Turing Recognizable)

语言 $L$ 是图灵可识别的（递归可枚举），当且仅当存在图灵机 $M$：

$$L = \{w \mid M \text{ 接受 } w\}$$

即 $w \in L$ 时 $M$ 停机接受，$w \notin L$ 时 $M$ 不停机或拒绝。

**定义 1.1.3** (图灵可判定 / Turing Decidable)

语言 $L$ 是图灵可判定的（递归），当且仅当存在图灵机 $M$：

$$\forall w: M \text{ 停机且 } (w \in L \Leftrightarrow M \text{ 接受 } w)$$

**定义 1.1.4** (λ演算 / Lambda Calculus)

λ项的语法：

$$e ::= x \mid \lambda x.e \mid e\ e$$

- 变量：$x$
- 抽象：$\lambda x.e$（函数定义）
- 应用：$e_1\ e_2$（函数调用）

**β归约**：$(\lambda x.e)\ e' \rightarrow_\beta e[e'/x]$

**定义 1.1.5** (部分递归函数 / Partial Recursive Function)

由基本函数通过复合、原始递归和极小化生成的函数类：

**基本函数**：

- 零函数：$Z(x) = 0$
- 后继函数：$S(x) = x + 1$
- 投影函数：$P_i^n(x_1, ..., x_n) = x_i$

**构造规则**：

- 复合：$h(x) = f(g_1(x), ..., g_m(x))$
- 原始递归：
  $$h(0, x) = f(x)$$
  $$h(n+1, x) = g(n, h(n, x), x)$$
- 极小化：$h(x) = \mu n. f(n, x) = 0$

### 1.2 丘奇-图灵论题

**定义 1.2.1** (丘奇-图灵论题 / Church-Turing Thesis)

直观可计算的函数恰好是图灵可计算（或λ可定义、部分递归）的函数。

形式化表述：

$$\text{直观可计算} = \text{图灵可计算} = \text{λ可定义} = \text{部分递归}$$

**注**：这是一个论题（假说），而非定理。它无法被形式证明，因为"直观可计算"没有形式定义。

**定义 1.2.2** (有效可计算 / Effectively Computable)

一个函数 $f: \mathbb{N} \rightarrow \mathbb{N}$ 是有效可计算的，当且仅当：

存在明确的、机械的、无歧义的步骤序列，对任意输入 $n$，可在有限步内计算出 $f(n)$（若 $f(n)$ 有定义）。

**定义 1.2.3** (扩展丘奇-图灵论题 / Extended Church-Turing Thesis)

任何在物理现实中可实现的计算，都可以在图灵机上以多项式时间开销模拟。

### 1.3 不可判定性

**定义 1.3.1** (停机问题 / Halting Problem)

$$A_{TM} = \{\langle M, w\rangle \mid M \text{ 是图灵机}, M \text{ 接受 } w\}$$

即"图灵机 $M$ 在输入 $w$ 上是否停机接受"。

**定义 1.3.2** (通用语言 / Universal Language)

$$U = \{\langle M, w\rangle \mid M \text{ 接受 } w\}$$

**定义 1.3.3** (空集问题)

$$E_{TM} = \{\langle M\rangle \mid L(M) = \emptyset\}$$

**定义 1.3.4** (等价性问题)

$$EQ_{TM} = \{\langle M_1, M_2\rangle \mid L(M_1) = L(M_2)\}$$

---

### 1.4 归约与完备性

**定义 1.4.1** (多一归约 / Many-One Reduction)

语言 $A$ 可多一归约到 $B$，记作 $A \leq_m B$，当且仅当存在可计算函数 $f$：

$$w \in A \Leftrightarrow f(w) \in B$$

**定义 1.4.2** (图灵归约 / Turing Reduction)

$A \leq_T B$ 当且仅当 $A$ 可由带有 $B$ 的预言机（oracle）的图灵机判定。

**定义 1.4.3** (递归可枚举完备 / RE-Complete)

语言 $L$ 是 RE-完备的，当且仅当：

1. $L$ 是递归可枚举的
2. 对所有递归可枚举语言 $A$：$A \leq_m L$

**定义 1.4.4** (莱斯定理的条件)

设 $\mathcal{P}$ 是图灵机识别语言的性质：

1. **非平凡性**：存在 $M_1, M_2$ 使 $L(M_1) \in \mathcal{P}$ 但 $L(M_2) \notin \mathcal{P}$
2. **语义性**：若 $L(M_1) = L(M_2)$，则 $M_1 \in \mathcal{P} \Leftrightarrow M_2 \in \mathcal{P}$

## 二、属性 (Properties)

### 2.1 可判定性与可识别性

| 性质 | 定义 | 示例 |
|------|------|------|
| **递归/可判定** | 图灵机总停机 | $A_{DFA}$, $A_{CFG}$ |
| **递归可枚举/可识别** | 接受时停机 | $A_{TM}$ |
| **co-RE** | 补集是RE | $\overline{A_{TM}}$ 不是RE |
| **既非RE也非co-RE** | 都不满足 | $EQ_{TM}$ |

### 2.2 归约性质

| 性质 | 说明 |
|------|------|
| **传递性** | $A \leq_m B \land B \leq_m C \Rightarrow A \leq_m C$ |
| **可判定性保持** | $A \leq_m B \land B$ 可判定 $\Rightarrow A$ 可判定 |
| **不可判定性保持** | $A \leq_m B \land A$ 不可判定 $\Rightarrow B$ 不可判定 |
| **RE保持** | $A \leq_m B \land B$ 是RE $\Rightarrow A$ 是RE |

## 四、解释 (Explanation)

### 4.1 丘奇-图灵论题的直觉

**为什么相信CT论题？**

1. **模型收敛性**：所有提出的合理计算模型（λ演算、图灵机、递归函数、Post系统、马尔可夫算法等）都被证明等价

2. **无反例**：至今未发现被公认为"直观可计算"但图灵不可计算的函数

3. **物理可实现性**：所有已知的物理计算装置都可被图灵机模拟

**论题的局限性**：

- 不涉及计算效率（多项式时间）
- 不涉及概率/量子计算（尽管量子图灵机已有定义）
- 不涉及连续计算/模拟计算

### 4.2 对角化方法的直观

**康托尔对角化**

证明实数不可数：假设可枚举，构造一个与每个实数都不同的小数。

**图灵对角化**

证明停机问题不可判定：假设可判定，构造一个程序，它在自己的描述上做出相反行为。

**核心思想**：

- 任何声称完备的列表都可以通过"对角线修改"产生新元素
- 自引用产生悖论

### 4.3 归约的直觉

**问题归约 = 问题转换**

$A \leq_m B$ 意味着：如果我能解决 $B$，我就能解决 $A$。

**类比**：

- 路径规划 $\leq_m$ SAT：将路径问题编码为布尔公式
- 如果 SAT 可解，则路径规划可解

**用于证明不可判定性**：

- 已知 $A$ 不可判定
- 证明 $A \leq_m B$
- 则 $B$ 也不可判定（否则 $A$ 也可判定）

### 4.4 莱斯定理的直觉

**语义 vs 语法**

- **语义性质**：只依赖于程序计算的功能（输入-输出关系）
- **语法性质**：依赖于程序的具体实现

**莱斯定理**：任何非平凡的语义性质都是不可判定的。

**直观理解**：

- 要判定一个语义性质，需要理解程序"真正做什么"
- 但程序的行为在一般情况下是不可预测的（停机问题）
- 因此语义性质通常不可判定

**例外**：语法性质可以判定（如"程序是否少于100行"）

## 五、论证 (Argumentation)

### 5.1 停机问题不可判定性的重要性

**计算的极限**

停机问题不可判定标志着计算的绝对极限：

1. **无通用调试器**：不存在能判定任意程序是否停机的算法
2. **程序验证的困难**：无法自动验证所有程序性质
3. **Rice 定理的基础**：停机问题是证明其他问题不可判定的归约源

**实际意义**：

- 程序员必须依赖测试而非穷举验证
- 类型系统、形式方法限制语言以换取可判定性
- 静态分析提供近似而非精确答案

### 5.2 可识别性 vs 可判定性

**半可判定性**

递归可枚举 = 半可判定：

- 若输入在语言中，算法最终会"是"
- 若输入不在语言中，算法可能永不停机

**为什么这有用？**

- 定理证明：如果是定理，最终会找到证明
- 程序搜索：如果存在解，会找到
- 生成器模式：产生有效元素

### 5.3 扩展CT论题的争议

**量子计算的挑战**

- 量子计算机能否多项式时间求解NP完全问题？
- 目前认为不能（BQP ⊆ PSPACE）
- 但量子计算的确提供了新的计算模型

**物理CT论题**

- 强CT论题：物理可计算的 = 图灵可计算
- 超计算模型（如相对论时间膨胀、黑洞等）是否允许超图灵计算？

## 六、形式证明 (Formal Proofs)

### 6.1 停机问题不可判定性证明

**定理**：$A_{TM}$ 是不可判定的。

**证明**（反证法）：

假设存在判定 $A_{TM}$ 的图灵机 $H$：

$$H(\langle M, w\rangle) = \begin{cases} \text{接受} & \text{若 } M \text{ 接受 } w \\ \text{拒绝} & \text{若 } M \text{ 拒绝或不停机于 } w \end{cases}$$

**Step 1**：构造机器 $D$

构造图灵机 $D$，输入 $\langle M\rangle$：

1. 在输入上运行 $H(\langle M, \langle M\rangle\rangle)$
2. 若 $H$ 接受，则 $D$ 拒绝
3. 若 $H$ 拒绝，则 $D$ 接受

即 $D$ 对 $H$ 的输出取反。

**Step 2**：分析 $D$ 在自身上的行为

考虑 $D(\langle D\rangle)$：

- 若 $D$ 接受 $\langle D\rangle$：
  - 由 $D$ 的定义，$H$ 拒绝 $\langle D, \langle D\rangle\rangle$
  - 由 $H$ 的定义，$D$ 不接受 $\langle D\rangle$
  - 矛盾！

- 若 $D$ 拒绝 $\langle D\rangle$：
  - 由 $D$ 的定义，$H$ 接受 $\langle D, \langle D\rangle\rangle$
  - 由 $H$ 的定义，$D$ 接受 $\langle D\rangle$
  - 矛盾！

**Step 3**：结论

无论 $D$ 接受还是拒绝 $\langle D\rangle$，都导致矛盾。

因此假设错误，$A_{TM}$ 不可判定。

∎

### 6.2 停机问题是 RE-完备的证明

**定理**：$A_{TM}$ 是 RE-完备的。

**证明**：

**Step 1**：$A_{TM}$ 是 RE 的

构造图灵机 $U$（通用图灵机）：

- 输入 $\langle M, w\rangle$
- 模拟 $M$ 在 $w$ 上的运行
- 若 $M$ 接受，则 $U$ 接受

$U$ 接受 $\langle M, w\rangle$ 当且仅当 $M$ 接受 $w$。

因此 $A_{TM}$ 是 RE 的。

**Step 2**：任意 RE 语言可归约到 $A_{TM}$

设 $L$ 是任意 RE 语言，则存在图灵机 $M_L$ 识别 $L$。

定义归约函数 $f(w) = \langle M_L, w\rangle$。

则：
$$w \in L \Leftrightarrow M_L \text{ 接受 } w \Leftrightarrow \langle M_L, w\rangle \in A_{TM}$$

因此 $L \leq_m A_{TM}$。

∎

### 6.3 莱斯定理证明

**定理**：设 $\mathcal{P}$ 是图灵机识别语言的非平凡语义性质，则判定 $\mathcal{P}$ 是不可判定的。

**证明**：

设 $\mathcal{P}$ 是非平凡语义性质。

**Step 1**：标准化

不失一般性，假设空语言 $\emptyset \notin \mathcal{P}$（否则考虑 $\overline{\mathcal{P}}$）。

因 $\mathcal{P}$ 非平凡，存在 $M^*$ 使 $L(M^*) \in \mathcal{P}$。

**Step 2**：构造归约

将 $A_{TM}$ 归约到 $\mathcal{P}$。

给定 $\langle M, w\rangle$，构造图灵机 $M'$：

```
M'(x):
    在 w 上模拟 M
    若 M 拒绝 w，则拒绝
    若 M 接受 w，则在 x 上模拟 M*
    接受当且仅当 M* 接受 x
```

**Step 3**：分析 $M'$

- 若 $M$ 接受 $w$：则 $L(M') = L(M^*) \in \mathcal{P}$
- 若 $M$ 不接受 $w$：则 $L(M') = \emptyset \notin \mathcal{P}$

**Step 4**：结论

$$M \text{ 接受 } w \Leftrightarrow \langle M'\rangle \in \mathcal{P}$$

因此 $A_{TM} \leq_m \mathcal{P}$，故 $\mathcal{P}$ 不可判定。

∎

### 6.4 等价性问题不可判定证明

**定理**：$EQ_{TM} = \{\langle M_1, M_2\rangle \mid L(M_1) = L(M_2)\}$ 是不可判定的。

**证明**：

将 $E_{TM}$（空集问题）归约到 $EQ_{TM}$。

**Step 1**：$E_{TM}$ 不可判定

已知 $E_{TM}$ 不可判定（可由 $A_{TM}$ 归约证明）。

**Step 2**：构造归约

给定 $\langle M\rangle$，构造：

- $M_1 = M$
- $M_2$ 是直接拒绝所有输入的机器（$L(M_2) = \emptyset$）

**Step 3**：分析

$$L(M) = \emptyset \Leftrightarrow L(M_1) = L(M_2) \Leftrightarrow \langle M_1, M_2\rangle \in EQ_{TM}$$

因此 $E_{TM} \leq_m EQ_{TM}$。

**Step 4**：结论

因 $E_{TM}$ 不可判定，故 $EQ_{TM}$ 也不可判定。

∎

## 七、应用与实现

### 7.1 通用图灵机模拟器

```rust
/// 图灵机定义
pub struct TuringMachine {
    states: Vec<State>,
    alphabet: Vec<char>,
    transitions: HashMap<(State, char), (State, char, Direction)>,
    initial_state: State,
    accept_state: State,
    reject_state: State,
}

/// 通用图灵机模拟
pub struct UniversalTM;

impl UniversalTM {
    /// 模拟图灵机 M 在输入 w 上的运行
    /// 返回：Accept, Reject, 或 模拟步数超限
    pub fn simulate(
        m_description: &str,  // 图灵机编码
        w: &str,              // 输入
        max_steps: usize,
    ) -> SimulationResult {
        // 解析图灵机描述
        let tm = Self::parse_tm(m_description);

        // 初始化带
        let mut tape: Vec<char> = w.chars().collect();
        let mut head = 0;
        let mut state = tm.initial_state.clone();
        let mut steps = 0;

        loop {
            if steps >= max_steps {
                return SimulationResult::Timeout;
            }
            steps += 1;

            // 确保带足够长
            if head >= tape.len() {
                tape.push('□'); // 空白符
            }

            let symbol = tape[head];

            // 查找转移
            match tm.transitions.get(&(state.clone(), symbol)) {
                Some((new_state, new_symbol, dir)) => {
                    tape[head] = *new_symbol;
                    state = new_state.clone();

                    // 移动读写头
                    match dir {
                        Direction::Left => {
                            if head > 0 { head -= 1; }
                        }
                        Direction::Right => head += 1,
                    }

                    // 检查接受/拒绝
                    if state == tm.accept_state {
                        return SimulationResult::Accept(steps);
                    }
                    if state == tm.reject_state {
                        return SimulationResult::Reject(steps);
                    }
                }
                None => {
                    // 无转移，进入拒绝状态
                    return SimulationResult::Reject(steps);
                }
            }
        }
    }
}
```

### 7.2 莱斯定理应用框架

```rust
/// 语义性质判定器的不可行性证明框架
pub struct RiceTheorem;

impl RiceTheorem {
    /// 证明某语义性质不可判定
    ///
    /// property: 待判定的语义性质
    /// empty_has_property: 空语言是否具有该性质
    pub fn prove_undecidable<P>(
        property: P,
        empty_has_property: bool,
    ) where
        P: Fn(&TuringMachine) -> bool,
    {
        // 1. 验证非平凡性
        // 存在 M1 使 property(M1) = true
        // 存在 M2 使 property(M2) = false

        // 2. 验证语义性
        // 若 L(M1) = L(M2)，则 property(M1) = property(M2)

        // 3. 构造从停机问题的归约
        // 对任意 <M, w>，构造 M' 使得：
```

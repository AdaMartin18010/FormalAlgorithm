# 英文摘要示例 — 六维补充

> **版本**: 1.0
> **创建日期**: 2026-04-20

---

## 一、规范定义深化

国际化（Internationalization, i18n）与本地化（Localization, l10n）的形式化定义：

- **i18n**: 软件架构支持多语言、多地区的能力
- **l10n**: 将 i18n 产品适配到特定语言/地区的过程
- **术语对齐**: 概念 $C$ 在语言 $L_1$ 和 $L_2$ 中的等价映射 $T: C_{L_1} \to C_{L_2}$

## 二、模型设计深化

国际化知识体系的多层模型：

- **符号层**: Unicode/ISO 标准字符编码
- **术语层**: 概念-术语-定义的三角关系
- **知识层**: 跨语言的知识图谱对齐
- **应用层**: 多语言文档生成与验证

## 三、数学符号与推导

术语对齐质量度量：

- **准确率**: $Precision = \frac{|\text{正确对齐}|}{|\text{系统对齐}|}$
- **召回率**: $Recall = \frac{|\text{正确对齐}|}{|\text{参考对齐}|}$
- **F1 值**: $F_1 = 2 \cdot \frac{P \cdot R}{P + R}$

## 四、示例性代码

```rust
/// 多语言术语查询结构
use std::collections::HashMap;

pub struct MultilingualTerm {
    concept_id: String,
    translations: HashMap<String, String>, // lang -> term
}

impl MultilingualTerm {
    pub fn get(&self, lang: &str) -> Option<&str> {
        self.translations.get(lang).map(|s| s.as_str())
    }
}
```

## 五、形式化证明

术语对齐的一致性：若 $T(c_1) = c_2$ 且 $R(c_2) = c_3$，则复合映射 $R \circ T$ 应保持概念层次结构。

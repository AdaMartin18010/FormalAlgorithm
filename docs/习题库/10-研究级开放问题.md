### 入门探索（2题）

#### 题目 10.1.1：P vs NP 问题概述

**难度**：⭐⭐⭐
**标签**：P vs NP、千禧年问题、理论计算机科学

**题目描述**：
深入理解P vs NP问题：

1. **形式化定义**：
   - P = {L | 存在多项式时间确定性图灵机判定L}
   - NP = {L | 存在多项式时间验证器，或存在多项式时间非确定性图灵机}

2. **等价表述**：
   - P = NP ⟺ SAT ∈ P
   - P = NP ⟺ 任意NP完全问题有多项式时间算法

3. **研究意义**：
   - 密码学的理论基础
   - 算法设计的指导原则
   - 数学证明自动化的可能性

4. **当前认知**：
   - 绝大多数研究者相信P ≠ NP
   - 但尚未找到证明或反证

**思考问题**：

- 如果P = NP被证明，世界会发生什么变化？
- 如果P ≠ NP被证明，对实际算法设计有何影响？

#### 题目 10.1.2：NP中间问题（NPI）

**难度**：⭐⭐⭐
**标签**：NPI、Ladner定理、中间问题

**题目描述**：
了解NP中间问题：

**Ladner定理（1975）**：
如果P ≠ NP，则存在语言L ∈ NP \ (P ∪ NPC)

这类问题被称为NP-intermediate（NPI）。

**候选问题**：

1. **图同构问题（Graph Isomorphism）**：
   - 已知 ∈ NP
   - 已知 ∉ NPC（除非多项式层级坍塌）
   - 2015年Babai证明拟多项式时间算法
   - 2017年改进为n^(O(log²n))

2. **整数分解（Integer Factorization）**：
   - 判定版本 ∈ NP ∩ coNP
   - 相信 ∉ P
   - 如果 ∈ NPC，则多项式层级坍塌到第2层

3. **离散对数问题**：
   - 类似整数分解
   - 密码学重要基础

**思考问题**：

- 为什么这些问题不太可能是NP完全的？
- 研究NPI问题的意义是什么？

### 中级研究（4题）

#### 题目 10.1.3：多项式层级（Polynomial Hierarchy）

**难度**：⭐⭐⭐⭐
**标签**：PH、多项式层级、复杂性类

**题目描述**：
研究多项式层级PH：

**定义**：

```
Σ₀P = Π₀P = P
Σₖ₊₁P = NP^ΣₖP
Πₖ₊₁P = coNP^ΣₖP
PH = ∪ₖ ΣₖP
```

**层次结构**：

```
P = Σ₀P ⊆ NP = Σ₁P ⊆ Σ₂P ⊆ ... ⊆ PH ⊆ PSPACE
```

**关键问题**：

1. PH是否坍塌？（即是否存在k使得ΣₖP = Σₖ₊₁P）
2. 如果P = NP，则PH坍塌到P
3. Karp-Lipton定理：如果NP ⊆ P/poly，则PH = Σ₂P

**Karp-Lipton定理的含义**：

- 如果SAT有多项式规模电路，则多项式层级坍塌
- 这对电路复杂性下界研究有重要指导意义

**思考问题**：

- 证明NP ⊆ BPP ⟹ NP = RP
- 探讨PH与电路复杂性的关系

#### 题目 10.1.4：电路复杂性下界

**难度**：⭐⭐⭐⭐
**标签**：电路复杂性、下界、证明技术

**题目描述**：
研究布尔电路复杂性下界：

**基本概念**：

- 电路大小：门的数量
- 电路深度：最长路径长度
- P/poly：多项式规模电路族能识别的语言

**已知下界结果**：

| 函数 | 下界 | 证明者 | 年份 |
|------|------|--------|------|
| PARITY | 需要指数规模AC⁰电路 | Furst-Saxe-Sipser, Ajtai | 1985 |
| MAJORITY | 需要深度对数、规模线性的TC⁰ | - | - |
| CLIQUE | 对某些电路类有下界 | Razborov | 1985 |

**自然证明（Natural Proofs）障碍**：
Razborov-Rudich（1994）证明：

- 大多数现有电路下界技术都是"自然的"
- 如果存在强伪随机数生成器，则自然证明无法分离P和NP
- 这意味着需要全新的证明技术

**思考问题**：

- 为什么证明电路下界如此困难？
- 自然证明障碍对研究意味着什么？

#### 题目 10.1.5：概率复杂性类

**难度**：⭐⭐⭐⭐
**标签**：BPP、RP、ZPP、去随机化

**题目描述**：
研究概率复杂性类及其关系：

**定义**：

- **RP**（随机多项式时间）：
  - 接受：至少1/2概率
  - 拒绝：概率1
  - P ⊆ RP ⊆ NP

- **coRP**：RP的补集

- **ZPP**（零错误概率多项式时间）：
  - ZPP = RP ∩ coRP
  - 要么给出正确答案，要么给出"不知道"

- **BPP**（有界错误概率多项式时间）：
  - 错误概率 ≤ 1/3
  - P ⊆ BPP ⊆ PSPACE

**重要问题**：

1. P = BPP？（去随机化）
   - Impagliazzo-Wigderson：如果存在难问题的电路下界，则P = BPP
   - 目前认为很可能成立

2. RP = coRP？
   - 等价于ZPP = RP

3. BPP与NP的关系
   - 未知BPP ⊆ NP或NP ⊆ BPP是否成立

**思考问题**：

- 设计一个RP算法来检测矩阵乘法错误
- 讨论去随机化对实际计算的意义

#### 题目 10.1.6：交互式证明与IP=PSPACE

**难度**：⭐⭐⭐⭐
**标签**：交互式证明、IP、PSPACE、零知识

**题目描述**：
研究交互式证明系统：

**IP（交互式多项式时间）**：

- 证明者P（无限计算能力）与验证者V（多项式时间概率算法）交互
- 完备性：真命题能以高概率被接受
- 可靠性：假命题以低概率被接受

**历史突破**：

1. **LFKN（1990）**：证明IP ⊆ PSPACE
2. **Shamir（1990）**：证明IP = PSPACE
   - 这意味着交互可以验证指数级复杂度的计算
   - 证明利用了代数化技术

**多证明者交互证明（MIP）**：

- MIP = NEXP（Babai-Fortnow-Lund，1991）
- 多证明者不能串通，显著增强证明能力

**PCP定理**（后续讨论）：

- PCP = NP
- 对近似难度的影响

**思考问题**：

- 为什么交互能增强证明能力？
- 零知识证明与IP的关系是什么？

**题目描述**：

   - 无序搜索：O(√n)

**思考问题**：

**题目描述**：
研究矩阵乘法复杂性：

**ω指数**：

- 矩阵乘法的最小指数
- 即n×n矩阵乘法可在O(n^ω)时间完成

**历史进展**：

```
标准算法：ω = 3
Strassen（1969）：ω < 2.81
Coppersmith-Winograd（1990）：ω < 2.376
Stothers（2010）：ω < 2.374
V. Williams（2011）：ω < 2.3729
Alman-Williams（2020）：ω < 2.37286
```

**当前挑战**：

- 能否证明ω = 2？
- 张量秩方法是否足以达到2？
- 激光法（laser method）的局限性

**实际意义**：

- 即使理论上有更优算法，Strassen在实践中只对大矩阵有效
- 内存访问模式比渐近复杂性更重要

**思考问题**：

- 为什么矩阵乘法复杂性如此重要？（与其他问题的归约）

#### 题目 10.2.4：图同构问题

**难度**：⭐⭐⭐⭐
**标签**：图同构、Babai算法、拟多项式

**题目描述**：
深入研究图同构问题：

**问题定义**：
给定两个图G₁和G₂，是否存在顶点重命名使它们相同？

**复杂性地位**：

- 已知 ∈ NP
- 已知 ∉ NPC（除非PH坍塌）
- 已知 ∈ coAM
- 相信 ∈ P 或至少 ∈ BPP

**Babai的突破（2015）**：

```
时间复杂度：exp((log n)^O(1)) = n^O(log n)
```

这是**拟多项式时间（quasi-polynomial）**。

**算法思想**：

1. **组合方法**：处理无结构图
2. **群论方法**：利用自同构群结构
3. **递归分割**：将大图分解为小图

**后续改进**：

- 2017年优化到n^O(log²n)
- 对实际应用的影响有限（已有高效启发式算法）

#### 题目 10.2.7： fine-grained复杂性

**难度**：⭐⭐⭐⭐
**标签**：fine-grained、SETH、OV、APSP

**题目描述**：
研究 fine-grained 复杂性理论：

**背景**：
传统复杂性只关心多项式 vs 超多项式。
Fine-grained复杂性关心多项式内部的精确复杂度。

**核心猜想**：

1. **SETH（强指数时间假设）**：
   - k-SAT需要Ω(2^(n(1-ε)))时间
   - 已用于证明许多问题的下界

2. **OV猜想（正交向量）**：
   - n个d维向量的正交对检测需要Ω(n^(2-ε))（当d = ω(log n)）

3. **APSP猜想（全对最短路径）**：
   - APSP需要Ω(n^(3-ε))

**下界结果**：

- 编辑距离：O(n^2)是紧的（除非SETH失败）
- 最长公共子序列：类似结果
- 弗雷歇距离：n^(2-o(1))

**意义**：

- 解释为什么某些算法多年未改进
- 为算法设计提供目标（匹配下界）

#### 题目 10.3.1：Unique Games猜想

**难度**：⭐⭐⭐⭐

**题目描述**：
深入研究独特游戏猜想：

**UGC（Khot，2002）陈述**：
对于任意ε > 0，存在q = q(ε)，使得区分以下两种情况在NP中是难的：

- 至少(1-ε)比例的约束可满足

**影响**：
如果UGC为真：

- 最大割：Goemans-Williamson的0.878是最佳可能
- 顶点覆盖：2-近似是最佳可能
- 许多问题的近似比被确定

**当前状态**：

- 未证明也未否定
- 子指数时间算法被提出（Arora-Barak-Steurer）
- 可能处于"真"和"假"之间的新复杂性类

**题目描述**：

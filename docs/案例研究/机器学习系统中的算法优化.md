## 概述

机器学习系统的性能不仅取决于模型架构，更依赖于底层优化算法。
从训练大规模语言模型到部署实时推理系统，算法优化是核心竞争力。
本案例研究深入分析梯度下降优化、注意力机制效率和训练加速技术。

## 1. 梯度下降变体

### 1.1 问题背景与动机

**场景**：

- GPT-4训练：数千亿参数，数万亿token
- 推荐系统实时训练：数十亿特征，毫秒级延迟要求
- 边缘设备部署：内存和计算受限

**挑战**：

- 高维非凸优化（维度>10^9）
- 病态条件（Hessian矩阵条件数大）
- 噪声梯度（小批量估计）
- 鞍点和局部极小值

### 1.2 算法选择原理

#### 优化算法演进

```
┌─────────────────────────────────────────────────────────────┐
│                    梯度下降演进                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  SGD ──→ Momentum ──→ AdaGrad ──→ RMSprop ──→ Adam         │
│   │          │            │           │          │          │
│   │          └─ 速度累积   └─ 自适应学习率 └─ 移动平均         │
│   │                                                        │
│   └──→ 现代变体                                             │
│        ├── AdamW (解耦权重衰减)                              │
│        ├── LAMB/LARS (大规模batch)                          │
│        ├── Lion (内存高效)                                   │
│        └── Sophia (二阶近似)                                 │
└─────────────────────────────────────────────────────────────┘
```

#### 算法对比

| 算法 | 内存占用 | 收敛速度 | 泛化性能 | 适用场景 |
|-----|---------|---------|---------|---------|
| SGD+Momentum | O(d) | 慢 | 好 | 计算机视觉 |
| Adam | O(2d) | 快 | 一般 | NLP、推荐 |
| AdamW | O(2d) | 快 | 好 | Transformer训练 |

#### 1.3.1 AdamW实现

```python
import torch
from torch.optim.optimizer import Optimizer

class AdamW(Optimizer):
    """
    AdamW: 解耦权重衰减的Adam

    与Adam(L2正则化)的区别：
    - Adam: w = w - lr * (g + λw)  (梯度包含权重衰减)
    - AdamW: w = w - lr*g - lr*λw   (权重衰减在参数更新时应用)

    优势：
    1. 权重衰减与梯度更新解耦
    2. 自适应学习率不影响权重衰减
    3. 更好的泛化性能
    """

    def __init__(self, params, lr=1e-3, betas=(0.9, 0.999),
                 eps=1e-8, weight_decay=0.01):
        defaults = dict(lr=lr, betas=betas, eps=eps,
                       weight_decay=weight_decay)
        super(AdamW, self).__init__(params, defaults)

    @torch.no_grad()
    def step(self, closure=None):
        loss = None
        if closure is not None:
            with torch.enable_grad():
                loss = closure()

        for group in self.param_groups:
            for p in group['params']:
                if p.grad is None:
                    continue

                grad = p.grad
                state = self.state[p]

                # 初始化状态
                if len(state) == 0:
                    state['step'] = 0
                    state['exp_avg'] = torch.zeros_like(p)
                    state['exp_avg_sq'] = torch.zeros_like(p)

                exp_avg, exp_avg_sq = state['exp_avg'], state['exp_avg_sq']
                beta1, beta2 = group['betas']

                state['step'] += 1

                # 更新一阶矩（动量）
                exp_avg.mul_(beta1).add_(grad, alpha=1 - beta1)

                # 更新二阶矩
                exp_avg_sq.mul_(beta2).addcmul_(grad, grad, value=1 - beta2)

                # 偏差修正
                bias_correction1 = 1 - beta1 ** state['step']
                bias_correction2 = 1 - beta2 ** state['step']

                step_size = group['lr'] / bias_correction1
                denom = (exp_avg_sq.sqrt() / (bias_correction2 ** 0.5)).add_(group['eps'])

                # 参数更新（注意：这里不包含权重衰减）
                p.addcdiv_(exp_avg, denom, value=-step_size)

                # 解耦的权重衰减
                if group['weight_decay'] != 0:
                    p.add_(p, alpha=-group['lr'] * group['weight_decay'])

        return loss


class LAMB(Optimizer):
    """
    LAMB (Layer-wise Adaptive Moments for Batch training)

    专为大规模batch训练设计，通过层自适应学习率稳定训练。
    关键创新：
    1. 层归一化：每层参数除以其L2范数
    2. 信任系数：防止自适应学习率过大

    论文：Reducing BERT Pre-Training Time from 3 Days to 76 Minutes
    """

    def __init__(self, params, lr=1e-3, betas=(0.9, 0.999),
                 eps=1e-6, weight_decay=0.01, trust_coeff=0.001):
        defaults = dict(lr=lr, betas=betas, eps=eps,
                       weight_decay=weight_decay, trust_coeff=trust_coeff)
        super(LAMB, self).__init__(params, defaults)

    @torch.no_grad()
    def step(self, closure=None):
        loss = None
        if closure is not None:
            with torch.enable_grad():
                loss = closure()

        for group in self.param_groups:
            for p in group['params']:
                if p.grad is None:
                    continue

                grad = p.grad
                state = self.state[p]

                if len(state) == 0:
                    state['step'] = 0
                    state['exp_avg'] = torch.zeros_like(p)
                    state['exp_avg_sq'] = torch.zeros_like(p)

                exp_avg, exp_avg_sq = state['exp_avg'], state['exp_avg_sq']
                beta1, beta2 = group['betas']

                state['step'] += 1

                # 权重衰减
                if group['weight_decay'] != 0:
                    grad = grad.add(p, alpha=group['weight_decay'])

                # 更新矩估计
                exp_avg.mul_(beta1).add_(grad, alpha=1 - beta1)
                exp_avg_sq.mul_(beta2).addcmul_(grad, grad, value=1 - beta2)

                # 偏差修正
                bias_correction1 = 1 - beta1 ** state['step']
                bias_correction2 = 1 - beta2 ** state['step']

                # 自适应学习率
                adaptive_lr = exp_avg / bias_correction1
                denom = (exp_avg_sq.sqrt() / (bias_correction2 ** 0.5)).add_(group['eps'])
                update = adaptive_lr / denom

                # 层归一化和信任系数
                r1 = p.norm(2)
                r2 = update.norm(2)

                if r1 > 0 and r2 > 0:
                    trust_ratio = r1 / r2
                    # 裁剪信任系数
                    trust_ratio = min(trust_ratio, 10.0)
                else:
                    trust_ratio = 1.0

                # 应用更新
                p.add_(update, alpha=-group['lr'] * trust_ratio)

        return loss


class Lion(Optimizer):
    """
    Lion (EvoLved Sign Momentum)

    特点：
    1. 仅使用一阶矩（节省内存）
    2. 使用符号函数（sign）简化更新
    3. 两个动量向量的更新规则

    论文：Symbolic Discovery of Optimization Algorithms
    """

    def __init__(self, params, lr=1e-4, betas=(0.9, 0.99),
                 weight_decay=0.0):
        defaults = dict(lr=lr, betas=betas, weight_decay=weight_decay)
        super(Lion, self).__init__(params, defaults)

    @torch.no_grad()
    def step(self, closure=None):
        loss = None
        if closure is not None:
            with torch.enable_grad():
                loss = closure()

        for group in self.param_groups:
            for p in group['params']:
                if p.grad is None:
                    continue

                grad = p.grad
                state = self.state[p]

                if len(state) == 0:
                    state['step'] = 0
                    state['momentum'] = torch.zeros_like(p)

                momentum = state['momentum']
                beta1, beta2 = group['betas']

                state['step'] += 1

                # 权重衰减
                if group['weight_decay'] != 0:
                    p.mul_(1 - group['lr'] * group['weight_decay'])

                # 更新动量（指数移动平均）
                momentum.mul_(beta1).add_(grad, alpha=1 - beta1)

                # 计算更新方向：sign(beta1 * m + (1-beta1) * g)
                update = torch.sign(momentum.mul(beta1).add(grad, alpha=1 - beta1))

                # 更新动量（第二个动量 - 用于下一轮的修正）
                momentum.mul_(beta2).add_(grad, alpha=1 - beta2)

                # 应用更新
                p.add_(update, alpha=-group['lr'])

        return loss
```

#### 1.3.2 二阶优化方法

```python
class SophiaH(Optimizer):
    """
    Sophia (Second-order Clipped Optimizer)

    结合二阶信息的内存高效优化器。
    核心思想：
    1. 对角Hessian近似
    2. 裁剪机制防止大步长

    论文：Sophia: A Scalable Stochastic Second-order Optimizer for Language Modeling
    """

    def __init__(self, params, lr=1e-3, betas=(0.96, 0.99),
                 eps=1e-12, weight_decay=0.1, rho=0.04,
                 update_interval=10):
        defaults = dict(lr=lr, betas=betas, eps=eps,
                       weight_decay=weight_decay, rho=rho,
                       update_interval=update_interval)
        super(SophiaH, self).__init__(params, defaults)

    @torch.no_grad()
    def step(self, closure=None, hessian_fn=None):
        loss = None
        if closure is not None:
            with torch.enable_grad():
                loss = closure()

        for group in self.param_groups:
            update_hessian = (self.state.get('step', 0) % group['update_interval'] == 0)

            for p in group['params']:
                if p.grad is None:
                    continue

                grad = p.grad
                state = self.state[p]

                if len(state) == 0:
                    state['step'] = 0
                    state['exp_avg'] = torch.zeros_like(p)
                    state['hessian'] = torch.zeros_like(p)

                exp_avg, hessian = state['exp_avg'], state['hessian']
                beta1, beta2 = group['betas']

                state['step'] += 1

                # 权重衰减
                if group['weight_decay'] != 0:
                    grad = grad.add(p, alpha=group['weight_decay'])

                # 更新一阶矩
                exp_avg.mul_(beta1).add_(grad, alpha=1 - beta1)

                # 更新Hessian估计（每隔update_interval步）
                if update_hessian and hessian_fn is not None:
                    hessian_estimate = hessian_fn(p)
                    hessian.mul_(beta2).add_(hessian_estimate, alpha=1 - beta2)

                # 计算更新（裁剪）
                # update = exp_avg / max(hessian, eps)
                denom = torch.clamp(hessian, min=group['eps'])
                update = exp_avg / denom

                # 裁剪到信任域
                update_norm = update.abs().max()
                if update_norm > group['rho']:
                    update.mul_(group['rho'] / update_norm)

                # 应用更新
                p.add_(update, alpha=-group['lr'])

        return loss

    def compute_hessian_diagonal(self, loss, params):
        """
        计算对角Hessian近似（使用Hutchinson方法）
        """
        # 随机向量
        v = [torch.randint(0, 2, p.shape, device=p.device).float() * 2 - 1
             for p in params]

        # HVP (Hessian-vector product)
        grads = torch.autograd.grad(loss, params, create_graph=True)
        grad_v = sum((g * vi).sum() for g, vi in zip(grads, v))

        hvp = torch.autograd.grad(grad_v, params, retain_graph=False)

        # 对角近似：H_ii ≈ (Hv)_i / v_i
        hessian_diag = [(hv * vi).detach() for hv, vi in zip(hvp, v)]

        return hessian_diag
```

#### 1.3.3 学习率调度

```python
class CosineDecayWithWarmup:
    """
    带预热的余弦退火学习率调度

    公式：
    warmup: lr = base_lr * step / warmup_steps
    decay: lr = min_lr + 0.5 * (base_lr - min_lr) * (1 + cos(π * T_cur / T_i))
    """

    def __init__(self, optimizer, warmup_steps, total_steps,
                 min_lr_ratio=0.1):
        self.optimizer = optimizer
        self.warmup_steps = warmup_steps
        self.total_steps = total_steps
        self.min_lr_ratio = min_lr_ratio
        self.current_step = 0

        self.base_lrs = [group['lr'] for group in optimizer.param_groups]

    def step(self):
        self.current_step += 1

        for i, base_lr in enumerate(self.base_lrs):
            if self.current_step < self.warmup_steps:
                # 线性预热
                lr = base_lr * self.current_step / self.warmup_steps
            else:
                # 余弦退火
                progress = (self.current_step - self.warmup_steps) / \
                          (self.total_steps - self.warmup_steps)
                lr = base_lr * (self.min_lr_ratio +
                               0.5 * (1 - self.min_lr_ratio) *
                               (1 + math.cos(math.pi * progress)))

            self.optimizer.param_groups[i]['lr'] = lr

    def get_lr(self):
        return [group['lr'] for group in self.optimizer.param_groups]


class OneCycleLR:
    """
    1Cycle学习率策略

    一个周期内：
    1. 学习率从base_lr上升到max_lr
    2. 再下降到min_lr
    同时动量反向变化
    """

    def __init__(self, optimizer, max_lr, total_steps,
                 pct_start=0.3, div_factor=25., final_div_factor=1e4):
        self.optimizer = optimizer
        self.max_lr = max_lr
        self.total_steps = total_steps
        self.pct_start = pct_start

        self.initial_lr = max_lr / div_factor
        self.min_lr = self.initial_lr / final_div_factor

        self.anneal_steps = int(total_steps * pct_start)
        self.current_step = 0

    def step(self):
        self.current_step += 1

        if self.current_step <= self.anneal_steps:
            # 上升阶段
            pct = self.current_step / self.anneal_steps
            lr = self.initial_lr + (self.max_lr - self.initial_lr) * pct
        else:
            # 下降阶段（余弦）
            pct = (self.current_step - self.anneal_steps) / \
                  (self.total_steps - self.anneal_steps)
            lr = self.min_lr + (self.max_lr - self.min_lr) * \
                 0.5 * (1 + math.cos(math.pi * pct))

        for group in self.optimizer.param_groups:
            group['lr'] = lr
```

## 2. 注意力机制复杂度

### 2.1 问题背景与动机

**场景**：

- GPT-4：上下文长度32K，注意力计算是瓶颈
- 长文档处理：百万token级别的法律/医学文档
- 实时推理：低延迟要求的生产环境

**挑战**：

- 标准Attention：O(n²)时间和空间复杂度
- 长序列内存爆炸（n=100K时，注意力矩阵需要37GB）
- 推理时KV Cache内存占用

### 2.2 算法选择原理

#### 注意力复杂度演进

```
标准Attention: O(n²) ──→ 线性Attention ──→ 稀疏Attention ──→ 硬件感知优化
     │              │              │              │
   Self-Attention  Performer      Sparse         FlashAttention
   Cross-Attention Linear         Longformer     FlashDecoding
                   Attention      BigBird
```

#### 算法对比

| 方法 | 时间复杂度 | 空间复杂度 | 精度损失 | 适用场景 |
|-----|-----------|-----------|---------|---------|
| 标准Attention | O(n²) | O(n²) | 0% | 短序列 |
| FlashAttention | O(n²) | O(n) | 0% | 通用 |
| Linear Attention | O(n) | O(n) | 小 | 长序列 |
| Sparse Attention | O(n√n) | O(n√n) | 中 | 局部模式 |
| Sliding Window | O(nw) | O(nw) | 中 | 局部依赖 |

### 2.3 实现细节

#### 2.3.1 FlashAttention实现

```python
import torch
import triton
import triton.language as tl

class FlashAttention(torch.nn.Module):
    """
    FlashAttention: 快速且内存高效的精确注意力

    核心思想：
    1. 分块计算（tiling），避免实例化完整的n×n注意力矩阵
    2. 重计算（recomputation）代替存储中间结果
    3. 融合kernel，减少HBM访问

    论文：FlashAttention: Fast and Memory-Efficient Exact Attention
    """

    def __init__(self, dropout_p=0.0, causal=False):
        super().__init__()
        self.dropout_p = dropout_p
        self.causal = causal

    def forward(self, q, k, v, softmax_scale=None):
        """
        Args:
            q, k, v: [batch, nheads, seqlen, headdim]
        Returns:
            output: [batch, nheads, seqlen, headdim]
        """
        batch, nheads, seqlen, headdim = q.shape

        if softmax_scale is None:
            softmax_scale = q.shape[-1] ** (-0.5)

        # 使用Triton实现（简化版使用PyTorch）
        output = self._flash_attn_forward(
            q, k, v, softmax_scale, self.causal
        )

        return output

    def _flash_attn_forward(self, q, k, v, softmax_scale, causal):
        """
        FlashAttention前向传播（简化版，实际用Triton kernel）

        算法：
        1. 将Q,K,V分块（block）
        2. 对每个Q块，迭代处理所有K,V块
        3. 在线softmax：维护running max和sum
        """
        batch, nheads, seqlen_q, d = q.shape
        _, _, seqlen_k, _ = k.shape

        # 块大小（根据SRAM容量选择）
        BLOCK_M = 128  # Q块大小
        BLOCK_N = 128  # K,V块大小

        # 输出
        o = torch.zeros_like(q)

        # 每个query的统计信息
        l = torch.zeros(batch, nheads, seqlen_q, device=q.device)  # logsumexp
        m = torch.full((batch, nheads, seqlen_q), float('-inf'),
                       device=q.device)  # max

        # 分块迭代
        for start_m in range(0, seqlen_q, BLOCK_M):
            end_m = min(start_m + BLOCK_M, seqlen_q)
            q_block = q[:, :, start_m:end_m, :]  # [batch, nheads, BLOCK_M, d]

            # 初始化块级别的统计
            m_block = torch.full((batch, nheads, end_m - start_m), float('-inf'),
                                device=q.device)
            l_block = torch.zeros(batch, nheads, end_m - start_m, device=q.device)
            o_block = torch.zeros(batch, nheads, end_m - start_m, d, device=q.device)

            for start_n in range(0, seqlen_k, BLOCK_N):
                end_n = min(start_n + BLOCK_N, seqlen_k)

                # Causal mask处理
                if causal and start_n > end_m:
                    continue

                k_block = k[:, :, start_n:end_n, :]
                v_block = v[:, :, start_n:end_n, :]

                # S = Q @ K^T
                s_block = torch.matmul(q_block, k_block.transpose(-2, -1)) * softmax_scale

                # Causal mask
                if causal:
                    mask = torch.arange(start_m, end_m, device=q.device)[:, None] >= \
                           torch.arange(start_n, end_n, device=q.device)[None, :]
                    s_block = s_block.masked_fill(~mask[None, None, :, :], float('-inf'))

                # 在线softmax更新
                m_new = torch.maximum(m_block, s_block.max(dim=-1)[0])

                # 重缩放
                exp_old = torch.exp(m_block - m_new)
                exp_new = torch.exp(s_block - m_new.unsqueeze(-1))

                l_block = l_block * exp_old + exp_new.sum(dim=-1)

                # 更新输出
                o_block = o_block * exp_old.unsqueeze(-1) + \
                         torch.matmul(exp_new, v_block)

                m_block = m_new

            # 归一化
            o_block = o_block / l_block.unsqueeze(-1)

            o[:, :, start_m:end_m, :] = o_block
            m[:, :, start_m:end_m] = m_block
            l[:, :, start_m:end_m] = torch.log(l_block) + m_block

        return o


class FlashAttentionTriton(torch.nn.Module):
    """
    使用Triton实现的完整FlashAttention
    """

    def forward(self, q, k, v):
        # 实际实现调用Triton kernel
        # 这里展示调用接口
        from flash_attn import flash_attn_func
        return flash_attn_func(q, k, v, causal=self.causal)


class LinearAttention(torch.nn.Module):
    """
    线性注意力（Kernel方法）

    原始：Attention(Q,K,V) = softmax(QK^T)V
    线性化：使用特征映射φ，将softmax变为线性核

    φ(Q) @ (φ(K)^T @ V) / (φ(Q) @ sum(φ(K)))

    复杂度从O(n²)降到O(n)
    """

    def __init__(self, feature_dim=256, feature_fn='elu'):
        super().__init__()
        self.feature_dim = feature_dim
        self.feature_fn = feature_fn

    def feature_map(self, x):
        """特征映射φ"""
        if self.feature_fn == 'elu':
            return torch.nn.functional.elu(x) + 1
        elif self.feature_fn == 'relu':
            return torch.nn.functional.relu(x)
        elif self.feature_fn == 'sinusoidal':
            # 随机傅里叶特征
            return torch.cat([torch.sin(x), torch.cos(x)], dim=-1)
        return x

    def forward(self, q, k, v):
        """
        Args:
            q, k, v: [batch, nheads, seqlen, headdim]
        """
        # 特征映射
        q = self.feature_map(q)  # [batch, nheads, seqlen, d]
        k = self.feature_map(k)

        # KV聚合（可增量计算，支持流式）
        kv = torch.matmul(k.transpose(-2, -1), v)  # [batch, nheads, d, d]
        k_sum = k.sum(dim=-2)  # [batch, nheads, d]

        # 计算输出
        numerator = torch.matmul(q, kv)  # [batch, nheads, seqlen, d]
        denominator = torch.matmul(q, k_sum.unsqueeze(-1))  # [batch, nheads, seqlen, 1]

        output = numerator / (denominator + 1e-6)

        return output
```

```python
class SparseAttention(torch.nn.Module):
    """
    稀疏注意力：只计算重要的注意力连接

    模式：
    1. 局部窗口（Sliding Window）
    2. 全局token（Global tokens）
    3. 随机采样（Random）
    4. 块稀疏（Block Sparse）
    """

    def __init__(self, window_size=512, num_global_tokens=16):
        super().__init__()
        self.window_size = window_size
        self.num_global_tokens = num_global_tokens

    def forward(self, q, k, v):
        batch, nheads, seqlen, headdim = q.shape

        # 全局token（可访问所有位置）
        global_q = q[:, :, :self.num_global_tokens, :]
        global_k = k[:, :, :self.num_global_tokens, :]
        global_v = v[:, :, :self.num_global_tokens, :]

        # 局部窗口注意力
        local_output = self._sliding_window_attention(q, k, v)

        # 全局token的注意力（可访问所有位置）
        global_output = torch.matmul(
            torch.softmax(torch.matmul(global_q, k.transpose(-2, -1)) / (headdim ** 0.5), dim=-1),
            v
        )

        # 合并
        output = local_output.clone()
        output[:, :, :self.num_global_tokens, :] = global_output

        return output

    def _sliding_window_attention(self, q, k, v):
        """滑动窗口注意力"""
        batch, nheads, seqlen, headdim = q.shape
        output = torch.zeros_like(q)

        for i in range(seqlen):
            # 窗口范围
            start = max(0, i - self.window_size // 2)
            end = min(seqlen, i + self.window_size // 2 + 1)

            q_i = q[:, :, i:i+1, :]  # [batch, nheads, 1, d]
            k_window = k[:, :, start:end, :]
            v_window = v[:, :, start:end, :]

            # 局部注意力
            scores = torch.matmul(q_i, k_window.transpose(-2, -1)) / (headdim ** 0.5)
            attn = torch.softmax(scores, dim=-1)
            output[:, :, i:i+1, :] = torch.matmul(attn, v_window)

        return output


class BigBirdAttention(torch.nn.Module):
    """
    BigBird: 随机+窗口+全局的稀疏注意力

    论文：Big Bird: Transformers for Longer Sequences
    """

    def __init__(self, num_random_blocks=3, block_size=64, num_global_blocks=2):
        super().__init__()
        self.num_random_blocks = num_random_blocks
        self.block_size = block_size
        self.num_global_blocks = num_global_blocks

    def forward(self, q, k, v):
        """
        BigBird注意力模式：
        1. 每个token关注：
           - 自身所在块
           - 周围window_size个token
           - num_random_blocks个随机块
           - num_global_blocks个全局块
        """
        batch, nheads, seqlen, headdim = q.shape
        num_blocks = seqlen // self.block_size

        output = []

        for i in range(seqlen):
            block_idx = i // self.block_size

            # 收集需要关注的token索引
            attend_indices = set()

            # 1. 同一block
            block_start = block_idx * self.block_size
            block_end = min((block_idx + 1) * self.block_size, seqlen)
            attend_indices.update(range(block_start, block_end))

            # 2. 窗口
            window_start = max(0, i - self.block_size)
            window_end = min(seqlen, i + self.block_size + 1)
            attend_indices.update(range(window_start, window_end))

            # 3. 随机块
            random_blocks = torch.randint(0, num_blocks, (self.num_random_blocks,))
            for rb in random_blocks:
                rb_start = rb.item() * self.block_size
                rb_end = min((rb.item() + 1) * self.block_size, seqlen)
                attend_indices.update(range(rb_start, rb_end))

            # 4. 全局块
            for gb in range(self.num_global_blocks):
                gb_start = gb * self.block_size
                gb_end = min((gb + 1) * self.block_size, seqlen)
                attend_indices.update(range(gb_start, gb_end))

            # 计算稀疏注意力
            indices = sorted(list(attend_indices))
            k_sparse = k[:, :, indices, :]
            v_sparse = v[:, :, indices, :]

            q_i = q[:, :, i:i+1, :]
            scores = torch.matmul(q_i, k_sparse.transpose(-2, -1)) / (headdim ** 0.5)
            attn = torch.softmax(scores, dim=-1)
            output_i = torch.matmul(attn, v_sparse)

            output.append(output_i)

        return torch.cat(output, dim=2)
```

```python
import torch
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

class ZeROStage2:
    """
    ZeRO Stage 2: 优化器状态 + 梯度分片

    标准数据并行：每个GPU保存完整副本
    - 参数：Φ
    - 梯度：Φ
    - 优化器状态：2Φ (Adam的momentum + variance)
    - 总计：4Φ per GPU

    ZeRO Stage 2：分片到N个GPU
    - 每个GPU只保存 4Φ/N

    论文：ZeRO: Memory Optimizations Toward Training Trillion Parameter Models
    """

    def __init__(self, params, world_size, rank):
        self.world_size = world_size
        self.rank = rank

        # 分片参数
        self.params = list(params)
        self.param_groups = self._shard_params()

        # 优化器状态（分片）
        self.optim_states = {}
        for group_idx, param_group in enumerate(self.param_groups):
            for p in param_group:
                if p.requires_grad:
                    # 每个GPU只保存自己的分片
                    self.optim_states[id(p)] = {
                        'exp_avg': torch.zeros_like(p),
                        'exp_avg_sq': torch.zeros_like(p)
                    }

    def _shard_params(self):
        """将参数分片到不同GPU"""
        # 简化实现：按参数分片
        shards = [[] for _ in range(self.world_size)]

        for i, p in enumerate(self.params):
            target_rank = i % self.world_size
            shards[target_rank].append(p)

        return shards

    def step(self, closure=None):
        """
        ZeRO优化步骤：
        1. 收集本rank负责的梯度
        2. 更新对应参数分片
        3. 广播更新后的参数
        """
        # 1. 梯度规约（Reduce-Scatter）
        for p in self.params:
            if p.grad is not None:
                dist.all_reduce(p.grad, op=dist.ReduceOp.AVG)

        # 2. 只更新本rank负责的参数
        for p in self.param_groups[self.rank]:
            if p.grad is None:
                continue

            state = self.optim_states[id(p)]

            # Adam更新
            state['exp_avg'].mul_(0.9).add_(p.grad, alpha=0.1)
            state['exp_avg_sq'].mul_(0.999).addcmul_(p.grad, p.grad, alpha=0.001)

            update = state['exp_avg'] / (state['exp_avg_sq'].sqrt() + 1e-8)
            p.data.add_(update, alpha=-0.001)

        # 3. 广播更新后的参数（All-Gather）
        for group in self.param_groups:
            for p in group:
                dist.broadcast(p.data, src=self.param_groups.index(group))


class FSDP(torch.nn.Module):
    """
    Fully Sharded Data Parallel (FSDP)

    ZeRO Stage 3的PyTorch实现：
    - 参数、梯度、优化器状态全部分片
    - 只在需要时收集完整参数（前向/后向）

    内存节省：与数据并行度成正比
    """

    def __init__(self, module, process_group=None):
        super().__init__()
        self.module = module
        self.process_group = process_group or dist.group.WORLD
        self.world_size = dist.get_world_size(self.process_group)
        self.rank = dist.get_rank(self.process_group)

        # 分片参数
        self._shard_parameters()

    def _shard_parameters(self):
        """将参数分片到各GPU"""
        for p in self.module.parameters():
            # 分片
            padding = (self.world_size - p.numel() % self.world_size) % self.world_size
            if padding > 0:
                p.data = torch.nn.functional.pad(p.data, (0, padding))

            numel_per_rank = p.numel() // self.world_size
            start = self.rank * numel_per_rank
            end = start + numel_per_rank

            # 只保存分片
            p._sharded_data = p.data[start:end].clone()
            p.data = p._sharded_data

    def _all_gather_params(self):
        """收集完整参数（前向传播前）"""
        for p in self.module.parameters():
            # 收集所有分片
            gathered = [torch.zeros_like(p._sharded_data) for _ in range(self.world_size)]
            dist.all_gather(gathered, p._sharded_data, group=self.process_group)

            # 恢复完整参数
            p.data = torch.cat(gathered)

    def forward(self, *args, **kwargs):
        self._all_gather_params()
        output = self.module(*args, **kwargs)

        # 释放完整参数（可选，与梯度检查点配合）
        # self._free_full_params()

        return output
```

```python
import torch
from torch.utils.checkpoint import checkpoint

class CheckpointedTransformerLayer(torch.nn.Module):
    """
    梯度检查点（Activation Checkpointing）

    核心思想：
    - 前向时丢弃中间激活值
    - 后向时重新计算需要的激活值

    权衡：
    - 内存：O(1) per layer vs O(L) （L为层数）
    - 时间：+20-30%前向计算

    适用：深度网络，内存受限场景
    """

    def __init__(self, d_model, nhead):
        super().__init__()
        self.self_attn = torch.nn.MultiheadAttention(d_model, nhead)
        self.feed_forward = torch.nn.Sequential(
            torch.nn.Linear(d_model, 4 * d_model),
            torch.nn.GELU(),
            torch.nn.Linear(4 * d_model, d_model)
        )
        self.norm1 = torch.nn.LayerNorm(d_model)
        self.norm2 = torch.nn.LayerNorm(d_model)

    def _forward_impl(self, x):
        """实际前向实现"""
        # 自注意力
        attn_out, _ = self.self_attn(x, x, x)
        x = self.norm1(x + attn_out)

        # 前馈
        ff_out = self.feed_forward(x)
        x = self.norm2(x + ff_out)

        return x

    def forward(self, x, use_checkpoint=True):
        if use_checkpoint and self.training:
            # 使用梯度检查点
            return checkpoint(self._forward_impl, x)
        else:
            return self._forward_impl(x)


class BlockwiseCheckpointedModel(torch.nn.Module):
    """
    分块梯度检查点

    策略：每隔N层检查点一次
    平衡内存和计算
    """

    def __init__(self, layers, checkpoint_every=2):
        super().__init__()
        self.layers = torch.nn.ModuleList(layers)
        self.checkpoint_every = checkpoint_every

    def forward(self, x):
        for i, layer in enumerate(self.layers):
            if self.training and i % self.checkpoint_every == 0:
                x = checkpoint(layer, x)
            else:
                x = layer(x)
        return x
```

```python
from torch.cuda.amp import autocast, GradScaler

class MixedPrecisionTrainer:
    """
    混合精度训练（AMP）

    原理：
    - 前向/后向使用FP16/BF16
    - 权重更新使用FP32
    - 损失缩放防止梯度下溢

    加速：2-8x（取决于GPU架构）
    """

    def __init__(self, model, optimizer, use_bf16=False):
        self.model = model
        self.optimizer = optimizer
        self.use_bf16 = use_bf16

        # 梯度缩放器（FP16需要，BF16不需要）
        self.scaler = GradScaler() if not use_bf16 else None

    def training_step(self, batch):
        self.optimizer.zero_grad()

        # 自动混合精度上下文
        dtype = torch.bfloat16 if self.use_bf16 else torch.float16

        with autocast(device_type='cuda', dtype=dtype):
            outputs = self.model(**batch)
            loss = outputs.loss

        if self.use_bf16:
            # BF16不需要损失缩放
            loss.backward()
            self.optimizer.step()
        else:
            # FP16需要损失缩放
            self.scaler.scale(loss).backward()
            self.scaler.step(self.optimizer)
            self.scaler.update()

        return loss.item()


class FP8Trainer:
    """
    FP8训练（H100 GPU）

    使用8位浮点数进一步加速
    需要Transformer Engine支持
    """

    def __init__(self, model, optimizer):
        try:
            import transformer_engine.pytorch as te
            self.te_available = True
        except ImportError:
            self.te_available = False
            raise RuntimeError("Transformer Engine not available")

        self.model = te.fp8_model_init(model)
        self.optimizer = optimizer

    def training_step(self, batch):
        import transformer_engine.pytorch as te

        with te.fp8_autocast():
            outputs = self.model(**batch)
            loss = outputs.loss

        loss.backward()
        self.optimizer.step()
        self.optimizer.zero_grad()

        return loss.item()
```

### 3.5 经验教训

1. **优先使用FSDP/ZeRO**：内存优化是训练大模型的关键
2. **FlashAttention是必选项**：免费的速度提升
3. **BF16优于FP16**：更好的数值稳定性，无需损失缩放
4. **通信是扩展瓶颈**：大规模训练需要优化通信模式

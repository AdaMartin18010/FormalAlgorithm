# 第1章 类型论基础

## 1.1 简单类型λ演算

### 习题 04-01-001 ⭐

**类型**: 理解型

**题目**：
什么是简单类型λ演算（Simply Typed Lambda Calculus）？说明类型规则（Typing Rules）的作用。

**答案**：
<details>
<summary>点击展开答案</summary>

**定义**：
在λ演算基础上增加类型系统，每个项都有确定的类型。

**语法**：

- 类型：$A, B ::= \iota \mid A \to B$
- 项：$t, u ::= x \mid \lambda x:A.t \mid t\, u$

**类型规则**：

**变量**：
$$\frac{x:A \in \Gamma}{\Gamma \vdash x : A}$$

**抽象**：
$$\frac{\Gamma, x:A \vdash t : B}{\Gamma \vdash \lambda x:A.t : A \to B}$$

**应用**：
$$\frac{\Gamma \vdash t : A \to B \quad \Gamma \vdash u : A}{\Gamma \vdash t\, u : B}$$

**意义**：

- 确保程序无类型错误
- 强规范化（well-typed terms always terminate）

</details>

---

### 习题 04-01-002 ⭐⭐

**类型**: 应用型

**题目**：
为以下λ项推导类型：

1. $\lambda x:\iota.\lambda y:\iota.x$
2. $(\lambda x:\iota \to \iota.\lambda y:\iota.x\,y)\,(\lambda z:\iota.z)$

**答案**：
<details>
<summary>点击展开答案</summary>

**1. 类型推导**：

$$\frac{\frac{x:\iota, y:\iota \vdash x : \iota}{x:\iota \vdash \lambda y:\iota.x : \iota \to \iota}}{\vdash \lambda x:\iota.\lambda y:\iota.x : \iota \to (\iota \to \iota)}$$

类型为 $\iota \to (\iota \to \iota)$，即接受两个参数返回第一个。

**2. 类型推导**：

设 $f = \lambda x:\iota \to \iota.\lambda y:\iota.x\,y$，类型为 $(\iota \to \iota) \to (\iota \to \iota)$

设 $g = \lambda z:\iota.z$，类型为 $\iota \to \iota$

$f\,g$ 的类型为 $\iota \to \iota$
</details>

## 1.2 Curry-Howard 对应

### 习题 04-02-001 ⭐⭐

**类型**: 理解型

**题目**：
解释 Curry-Howard 对应（命题即类型，证明即程序）。

**答案**：
<details>
<summary>点击展开答案</summary>

**对应关系**：

| 逻辑 | 类型论 |
|:-----|:-------|
| 命题 | 类型 |
| 证明 | 程序（项） |
| 命题 $A \Rightarrow B$ | 函数类型 $A \to B$ |
| 命题 $A \land B$ | 积类型 $A \times B$ |
| 命题 $A \lor B$ | 和类型 $A + B$ |
| 证明归约 | 程序求值 |
| 无证明 | 空类型 $\bot$ |

**直观理解**：

- 证明 $A \Rightarrow B$ 就是构造从 $A$ 的证明到 $B$ 的证明的函数
- 程序类型检查就是证明有效性检查
- 程序执行就是证明简化

**意义**：
统一了逻辑学和计算学，是现代证明助手的理论基础。
</details>

# 第2章 λ演算

## 2.1 β归约与 Church-Rosser

### 习题 04-03-001 ⭐

**类型**: 理解型

**题目**：
什么是β归约？解释 Church-Rosser 定理的内容和意义。

**答案**：
<details>
<summary>点击展开答案</summary>

**β归约**：
$$(\lambda x.t)\, u \to_\beta t[x := u]$$

即：将函数体中的形参替换为实参

**Church-Rosser 定理**：
若 $t \to^* u$ 且 $t \to^* v$，则存在 $w$ 使得 $u \to^* w$ 且 $v \to^* w$。

**意义**：

1. 归约顺序不影响最终结果（合流性）
2. 若存在范式（不可再归约的形式），则唯一
3. 不同求值策略（call-by-name, call-by-value）结果一致

</details>

---

### 习题 04-03-002 ⭐⭐

**类型**: 应用型

**题目**：
定义不动点组合子 $Y$，证明 $Y\, f =_\beta f\,(Y\, f)$。

**答案**：
<details>
<summary>点击展开答案</summary>

**Y组合子定义**：
$$Y = \lambda f.(\lambda x.f\,(x\,x))\,(\lambda x.f\,(x\,x))$$

**证明**：
$$\begin{aligned}
Y\, f &= (\lambda x.f\,(x\,x))\,(\lambda x.f\,(x\,x)) \\
&\to_\beta f\,((\lambda x.f\,(x\,x))\,(\lambda x.f\,(x\,x))) \\
&= f\,(Y\, f)
\end{aligned}$$

**用途**：
在无显式递归的λ演算中实现递归定义。

例如阶乘：$\text{fact} = Y\,(\lambda f.\lambda n.\text{if } n=0 \text{ then } 1 \text{ else } n \cdot f\,(n-1))$
</details>

# 第3章 依赖类型

## 3.1 Π类型与Σ类型

### 习题 04-04-001 ⭐⭐

**类型**: 理解型

**题目**：
解释依赖函数类型（Π类型）和依赖对类型（Σ类型）。

**答案**：
<details>
<summary>点击展开答案</summary>

**Π类型（依赖函数）**：
$$\Pi_{x:A} B(x)$$

类型为 $A \to B(x)$ 的函数，但返回类型 $B$ 可依赖于输入值 $x$。

**例子**：
向量长度类型化的 `head` 函数：
$$\text{head} : \Pi_{n:\mathbb{N}} \text{Vec}\, A\, (n+1) \to A$$

**Σ类型（依赖对）**：
$$\Sigma_{x:A} B(x)$$

对 $(a, b)$ 其中 $a : A$，$b : B(a)$。

**例子**：
存在量词：$\exists x:A. P(x)$ 对应于 $\Sigma_{x:A} P(x)$

**与普通类型的关系**：
- 当 $B$ 不依赖 $x$ 时：$\Pi_{x:A} B = A \to B$，$\Sigma_{x:A} B = A \times B$
</details>

# 第4章 同伦类型论

## 4.1 路径类型与Univalence

### 习题 04-05-001 ⭐⭐

**类型**: 理解型

**题目**：
解释同伦类型论（HoTT）中的路径类型（Path Type）和 Univalence 公理。

**答案**：
<details>
<summary>点击展开答案</summary>

**路径类型**：
在HoTT中，命题相等 $a =_A b$ 被解释为从 $a$ 到 $b$ 的路径（path）。

- 可以有多条不同的路径（不同于传统类型论）
- 路径之间可以比较（高阶路径）

**Univalence 公理**：
$$(A =_\mathcal{U} B) \simeq (A \simeq B)$$

类型相等等价于等价（equivalence）。

**意义**：
- 同构的结构被视为相等
- 符合数学实践：数学家将同构对象视为相同
- 支持传输（transport）原理
</details>

### 习题 04-06-002 ⭐⭐

**类型**: 实现型

**题目**：
在Coq/Lean中证明自然数加法的结合律：$\forall a b c, (a + b) + c = a + (b + c)$。

**答案**：
<details>
<summary>点击展开答案</summary>

**Coq 版本**：
```coq
Theorem plus_assoc : forall a b c : nat, (a + b) + c = a + (b + c).
Proof.
  intros a b c.
  induction a as [| a' IH].
  - simpl. reflexivity.
  - simpl. rewrite IH. reflexivity.
Qed.
```

**Lean 版本**：
```lean
theorem add_assoc (a b c : Nat) : (a + b) + c = a + (b + c) := by
  induction a with
  | zero => simp
  | succ n ih => simp [ih]
```

**证明策略**：
1. 对 $a$ 归纳
2. 基础情况：$0 + b + c = 0 + (b + c)$，简化即得
3. 归纳步骤：假设对 $a'$ 成立，证对 $a'+1$ 成立
</details>

# 第6章 程序验证

## 6.1 Hoare逻辑

### 习题 04-07-001 ⭐⭐

**类型**: 理解型

**题目**：
解释Hoare三元组 $\{P\} C \{Q\}$ 的含义，说明赋值公理和循环不变式。

**答案**：
<details>
<summary>点击展开答案</summary>

**Hoare三元组**：
$\{P\} C \{Q\}$ 表示：若程序 $C$ 执行前 $P$ 成立，则执行后 $Q$ 成立（若终止）。

**赋值公理**：
$$\{Q[E/x]\} \, x := E \, \{Q\}$$

前置条件中将 $x$ 替换为表达式 $E$。

**循环规则**：
$$\frac{\{I \land B\} C \{I\}}{\{I\} \text{ while } B \text{ do } C \{I \land \neg B\}}$$

$I$ 是循环不变式（Loop Invariant）。

**不变式**：
- 进入循环前成立
- 每次迭代保持
- 循环终止时与条件否定结合推出后置条件
</details>

---

### 习题 04-07-002 ⭐⭐⭐

**类型**: 设计型

**题目**：
为以下程序写出循环不变式并证明其正确性：
```
{ n >= 0 }
i := 0;
s := 0;
while i < n do
    i := i + 1;
    s := s + i
{s = n(n+1)/2}
```

**答案**：
<details>
<summary>点击展开答案</summary>

**循环不变式**：
$$I \equiv s = \frac{i(i+1)}{2} \land 0 \leq i \leq n$$

**验证**：

1. **初始化**：
   - $i = 0, s = 0$
   - $0 = 0(0+1)/2$ ✓
   - $0 \leq 0 \leq n$ ✓（假设 $n \geq 0$）

2. **保持**：
   - 假设 $I \land i < n$ 成立
   - 执行后 $i' = i + 1$，$s' = s + i + 1 = \frac{i(i+1)}{2} + i + 1 = \frac{(i+1)(i+2)}{2}$
   - 即 $s' = \frac{i'(i'+1)}{2}$ ✓

3. **终止**：
   - 退出时 $I \land i \geq n$，即 $i = n$
   - 故 $s = n(n+1)/2$ ✓
</details>

## 7.1 操作语义与指称语义

### 习题 04-08-001 ⭐

**类型**: 理解型

**题目**：
区分操作语义、指称语义和公理语义三种形式化语义方法。

**答案**：
<details>
<summary>点击展开答案</summary>

| 方法 | 描述 | 工具 |
|:-----|:-----|:-----|
| **操作语义** | 程序如何执行（状态转换） | 转移系统、规则 |
| **指称语义** | 程序表示什么数学对象 | 域论、函数 |
| **公理语义** | 程序满足什么性质 | 逻辑、霍尔逻辑 |

**关系**：
- 操作语义最直观，适合实现
- 指称语义提供数学基础
- 公理语义适合验证

**例子**：赋值语句 $x := E$
- 操作：$(x := E, \sigma) \to (\text{skip}, \sigma[x \mapsto E[\sigma]])$
- 指称：$[x := E] = \lambda \sigma. \sigma[x \mapsto [E]\sigma]$
- 公理：$\{Q[E/x]\} x := E \{Q\}$
</details>

# 第9章 模型检测

## 9.1 时序逻辑与验证

### 习题 04-10-001 ⭐⭐

**类型**: 理解型

**题目**：
比较LTL（线性时序逻辑）和CTL（计算树逻辑）的表达能力。

**答案**：
<details>
<summary>点击展开答案</summary>

**LTL**：
- 沿单一路径量化
- 公式：$\square p$（总是），$\Diamond p$（终将），$\bigcirc p$（下一状态）

**CTL**：
- 沿分支结构量化
- 公式：$A\square p$（所有路径总是），$E\Diamond p$（存在路径终将）

**表达能力**：
- LTL 和 CTL 不可比较
- $A\square E\Diamond p$（CTL有，LTL无）
- $\square\Diamond p$（LTL有，CTL无）
- CTL* 包含两者

**模型检测复杂度**：
- LTL：PSPACE-完全
- CTL：P-完全
</details>

解释状态空间爆炸问题，说明符号模型检测（Symbolic Model Checking）如何应对。

**答案**：

## 10.1 自动化证明

### 习题 04-11-001 ⭐

**类型**: 理解型

**题目**：
区分一阶逻辑和高阶逻辑，说明SMT求解器的基本组成。

**答案**：
<details>
<summary>点击展开答案</summary>

**一阶逻辑**：
- 量词作用于个体（$\forall x. P(x)$）
- 可判定性：半可判定（可证则能找到证明）

**高阶逻辑**：
- 量词可作用于谓词和函数
- 表达能力更强，但完全自动证明更难

**SMT求解器组成**：
1. **SAT求解器**：处理布尔结构
2. **Theory Solvers**：处理特定理论
   - 线性算术
   - 数组
   - 位向量
   - 未解释函数
3. **组合**：Nelson-Oppen方法组合多个理论

**应用**：
程序验证、测试生成、程序综合
</details>

## 1. 可视化设计原则

### 1.1 核心原则

| 原则 | 描述 | 应用示例 |
|------|------|----------|
| **清晰性** | 每个视觉元素必须有明确含义 | 算法步骤的当前状态用高亮表示 |
| **一致性** | 同类元素使用相同的视觉编码 | 所有数组访问使用相同颜色 |
| **简洁性** | 避免不必要的装饰元素 | 移除与算法无关的网格线 |
| **层次性** | 重要信息优先展示 | 当前操作元素放大显示 |
| **时序性** | 算法步骤按时间顺序呈现 | 使用箭头或编号指示执行顺序 |

### 1.2 设计层次

```
第一层：基础结构（静态）
  ├── 数据结构的初始状态
  ├── 输入数据的呈现
  └── 边界和约束的标注

第二层：执行过程（动态）
  ├── 当前操作区域高亮
  ├── 比较/交换操作的动画
  └── 指针/索引的移动轨迹

第三层：结果展示（总结）
  ├── 最终状态标记
  ├── 复杂度统计
  └── 关键路径标识
```

### 1.3 视觉编码标准

```markdown
形状编码:
- 圆形: 节点（图、树）
- 矩形: 数组元素、数据块
- 菱形: 决策点
- 箭头: 方向、依赖关系

大小编码:
- 大小 ∝ 数值大小（散点图）
- 大小 ∝ 节点度数（图）
- 大小 ∝ 访问频率

位置编码:
- 水平位置: 时间/步骤
- 垂直位置: 层次/优先级
- 深度: 递归层次
```

## 2. 颜色编码规范

### 2.1 标准配色方案

#### 算法状态颜色

| 状态 | 颜色名称 | HEX代码 | 使用场景 |
|------|----------|---------|----------|
| 未访问 | 浅灰 | `#E0E0E0` | 未处理的数据元素 |
| 当前 | 蓝色 | `#2196F3` | 当前正在操作的元素 |
| 已访问 | 绿色 | `#4CAF50` | 已处理的元素 |
| 比较中 | 橙色 | `#FF9800` | 正在比较的元素 |
| 交换/移动 | 紫色 | `#9C27B0` | 正在交换或移动的元素 |
| 目标 | 红色 | `#F44336` | 目标元素、关键位置 |
| 选中 | 青色 | `#00BCD4` | 被选中的元素（如主元） |
| 最终位置 | 深绿 | `#2E7D32` | 已确定最终位置的元素 |

#### 数据结构颜色

| 类型 | 颜色 | 说明 |
|------|------|------|
| 数组 | `#E3F2FD` (浅蓝背景) | 连续存储 |
| 链表 | 节点间用 `#1976D2` 连接 | 指针关系 |
| 树节点 | `#FFF3E0` (浅橙) | 层次结构 |
| 图节点 | 根据类别着色 | 连通性展示 |
| 堆 | 完全二叉树布局 | 父节点在上 |

### 2.2 颜色对比度要求

```markdown
WCAG 2.1 AA级标准:
- 文本与背景对比度 ≥ 4.5:1
- 大文本对比度 ≥ 3:1

推荐搭配:
- 深色文本 (#212121) 配浅色背景 (#FAFAFA)
- 浅色文本 (#FFFFFF) 配深色背景 (#1565C0)
```

### 2.3 渐变使用规范

```markdown
允许使用渐变的场景:
1. 表示数值范围（热力图）
2. 表示时间流逝（执行追踪）
3. 表示概率/权重

禁止:
1. 装饰性渐变
2. 多色渐变（超过3色）
3. 与功能无关的色彩变化
```

## 4. 算法执行可视化

### 4.1 排序算法可视化

#### 冒泡排序示例

```
初始:  [5, 2, 8, 1, 9, 3]

第1轮:
Step 1: [5, 2, 8, 1, 9, 3]  比较(5,2) → 交换
         🔵 🟠
Step 2: [2, 5, 8, 1, 9, 3]  比较(5,8) → 不交换
               🔵 🟠
Step 3: [2, 5, 8, 1, 9, 3]  比较(8,1) → 交换
                  🔵 🟠
Step 4: [2, 5, 1, 8, 9, 3]  比较(8,9) → 不交换
                     🔵 🟠
Step 5: [2, 5, 1, 8, 9, 3]  比较(9,3) → 交换
                        🔵 🟠
结果:  [2, 5, 1, 8, 3, 9]  ← 9已就位(绿色)
                        🟢
```

#### 快速排序分区过程

```
初始数组: [64, 34, 25, 12, 22, 11, 90]
                        ↑
                      pivot=90

分区过程:
[64, 34, 25, 12, 22, 11 | 90]
 🟢  🟢  🟢  🟢  🟢  🟢   🟠
 ≤pivot                 pivot

结果: 左分区=[64,34,25,12,22,11], pivot=90, 右分区=[]
```

### 4.2 图搜索可视化

#### BFS遍历

```
        ┌───┐
        │ 0 │ ←── 起点 (距离=0, 蓝色)
        └───┘
       ╱  │  ╲
      ╱   │   ╲
   ┌───┐┌───┐┌───┐
   │ 1 ││ 2 ││ 3 │ ←── 距离=1 (橙色)
   └───┘└───┘└───┘
        │
     ┌───┐
     │ 4 │ ←── 距离=2 (绿色)
     └───┘

访问顺序: 0 → 1 → 2 → 3 → 4
颜色变化: 🔵 → 🟠 → 🟠 → 🟠 → 🟢
```

#### Dijkstra最短路径

```
              4
        ┌───────────────┐
       ╱ 2              ▼
    ┌─────┐           ┌─────┐
    │  A  │ ────────→ │  B  │
    │ 🔵  │     3     │ 🟠  │ dist=4
    └──┬──┘           └──┬──┘
    1 │              1  │
       ▼                 ▼
    ┌─────┐           ┌─────┐
    │  C  │ ────────→ │  D  │
    │     │     2     │     │ dist=3
    └─────┘           └─────┘

当前节点: A (距离0)
邻居更新: C=1, B=2
下一步: 选择C (距离1)
```

### 4.3 动态规划表格可视化

#### 背包问题DP表

```
容量:    0   1   2   3   4   5
       ┌───┬───┬───┬───┬───┬───┐
物品0  │ 0 │ 0 │ 0 │ 0 │ 0 │ 0 │
       ├───┼───┼───┼───┼───┼───┤
物品1  │ 0 │ 0 │ 3 │ 3 │ 3 │ 3 │  w=2,v=3
       ├───┼───┼───┼───┼───┼───┤
物品2  │ 0 │ 0 │ 3 │ 4 │ 4 │ 7 │  w=3,v=4 ← 当前计算
       ├───┼───┼───┼───┼───┼───┤
物品3  │ 0 │ 0 │ 3 │ 4 │ 5 │ 7 │  w=4,v=5
       └───┴───┴───┴───┴───┴───┘
                    ↑
                  dp[2][5] = max(dp[1][5], dp[1][2]+4) = max(3, 3+4) = 7
                  🟠 = 当前计算位置
                  🔵 = 依赖的值
```

## 6. 工具与格式

### 6.1 推荐工具

| 工具 | 类型 | 适用场景 | 输出格式 |
|------|------|----------|----------|
| Manim | Python库 | 数学动画、算法演示 | MP4/GIF |
| Graphviz | 图可视化 | 图结构、树结构 | PNG/SVG/PDF |
| Matplotlib | Python库 | 图表、复杂度曲线 | PNG/SVG/PDF |
| D3.js | JavaScript | 交互式网页可视化 | HTML/SVG |
| TikZ | LaTeX宏包 | 学术文档插图 | PDF/SVG |

### 6.2 输出格式规范

```markdown
静态图像:
- 格式: PNG (位图), SVG (矢量)
- 分辨率: 300 DPI (打印), 150 DPI (网页)
- 尺寸: 最大宽度 1200px (网页), A4宽度 (打印)
- 文件命名: [算法]_[描述]_[步骤].[格式]

动态图像:
- 格式: GIF (简单动画), MP4 (复杂动画)
- 帧率: 2-10 fps (教学演示)
- 时长: 5-30秒 (单个概念)
- 文件大小: < 5MB

交互式:
- 格式: HTML5 + JavaScript
- 响应式: 支持移动端
- 无障碍: 支持键盘导航
```

### 6.3 ASCII艺术规范

```markdown
使用场景:
- 代码注释中的示意图
- 命令行工具输出
- 纯文本文档

设计原则:
1. 使用等宽字体
2. 对齐使用空格而非Tab
3. 最大宽度 80字符
4. 使用标准ASCII字符

推荐符号:
┌─┬─┐  方框绘制 (表格、容器)
│ │ │  垂直线
├─┼─┤  中间交叉
└─┴─┘  底部方框

→ ← ↑ ↓  方向箭头
╱ ╲      对角线
```

## 7. 输出规范

### 7.1 图示标注要求

```markdown
每个可视化必须包含:
1. 标题: 简明描述图示内容
2. 图例: 解释颜色和符号含义
3. 坐标轴: 标明单位和范围
4. 数据来源: 如有实测数据需注明
5. 时间戳: 算法执行的时间点（动态可视化）
```

### 7.2 多步骤动画规范

```markdown
步骤过渡:
- 淡入淡出: 200-300ms
- 移动动画: 300-500ms
- 缩放动画: 200ms
- 暂停时间: 每步 500-1000ms

标注方式:
步骤1/5: 初始化
步骤2/5: 第一次分区
步骤3/5: 递归排序左子数组
步骤4/5: 递归排序右子数组
步骤5/5: 合并结果
```

### 7.3 颜色盲友好设计

```markdown
适配措施:
1. 不单独依赖颜色传递信息
2. 使用形状、纹理、标签辅助
3. 避免红绿对比
4. 提供高对比度模式

推荐配色（色盲友好）:
- 蓝色 (#0072B2)
- 橙色 (#D55E00)
- 绿色 (#009E73)
- 黄色 (#F0E442)
- 深蓝 (#56B4E9)
- 粉红 (#CC79A7)
```

# 亚线性时间算法（Sublinear Time Algorithms）

> **来源**: MIT 6.889 (Sublinear Time Algorithms, 2024–2025)
> **对齐状态**: 项目完全空白，为首次系统引入

---

## 一、核心思想

### 1.1 为什么需要亚线性时间？

现代数据规模：

- 互联网：数十亿网页
- 基因组学：数万亿碱基对
- 社交网络：数十亿用户、数百亿边
- 传感器数据：每秒数百万条记录

**传统算法**需要至少线性时间 $O(n)$ 才能读取全部输入。
**亚线性算法**只查看输入的**极小部分**（如 $O(\sqrt{n})$、$O(\log n)$、甚至 $O(1)$），却能回答**近似正确**的问题。

### 1.2 近似正确性

亚线性算法通常给出**近似答案**而非精确答案：

- $(1 \pm \epsilon)$-近似：答案在 $(1-\epsilon)\text{OPT}$ 到 $(1+\epsilon)\text{OPT}$ 之间
- 以高概率正确：$\Pr[\text{答案正确}] \geq 1 - \delta$

## 二、三大研究方向

### 2.1 Property Testing（性质测试）

**问题**: 判断对象是否满足某性质，或"远不满足"该性质。

**定义**: 算法是 $\epsilon$-tester，若：

- 若对象满足性质，接受概率 $\geq 2/3$
- 若对象与任何满足性质的对象的汉明距离 $> \epsilon n$，拒绝概率 $\geq 2/3$

#### 2.1.1 字符串性质测试

| 性质 | 查询复杂度 | 算法 |
|------|----------|------|
| 回文 | $O(1/\epsilon)$ | 随机采样对称位置比较 |
| 周期性 | $O(1/\epsilon)$ | 利用周期定义采样验证 |
| 字典序 | $O(\sqrt{n}/\epsilon)$ | 比较随机选取的子串 |

#### 2.1.2 图性质测试

| 性质 | 查询复杂度 | 算法 |
|------|----------|------|
| 连通性 | $O(1/\epsilon)$ | BFS 采样验证局部连通性 |
| 二分性 | $\tilde{O}(1/\epsilon^3)$ | 随机采样边检查奇环 |
| 三角形自由 | $O(1/\epsilon^2)$ | 采样三元组检查三角形 |

**代表性成果**: Goldreich-Goldwasser-Ron (1998)，系统建立图性质测试框架。

#### 2.1.3 分布性质测试

给定来自未知分布 $D$ 的样本，判断：

- $D$ 是否等于某已知分布 $D_0$？
- $D_1$ 和 $D_2$ 是否相同？

| 问题 | 样本复杂度 | 算法 |
|------|----------|------|
| 均匀性检验 | $O(\sqrt{n}/\epsilon^2)$ | 碰撞计数 (Collision Counting) |
| 分布等同性 | $O(n^{2/3}/\epsilon^2)$ | 基于 $\ell_2$ 距离估计 |
| 熵估计 | $\tilde{O}(n/\log n)$ | 基于 profiles 的估计 |

### 2.2 Streaming Algorithms（流算法）

**模型**: 数据以流形式到达，算法只能：

- 单次（或少数几次）扫描
- 使用次线性空间（通常为 $O(\log n)$ 或 $O(\text{polylog } n)$）

#### 2.2.1 经典问题

| 问题 | 空间复杂度 | 算法 |
|------|----------|------|
| 不同元素计数 (F₀) | $O(\log n)$ | FM Sketch (Flajolet-Martin) |
| 频率估计 (F₂) | $O(\log n)$ | AMS Sketch (Alon-Matias-Szegedy) |
| 点查询 | $O(\log n)$ | Count-Min Sketch |
|  heavy hitters | $O(1/\epsilon)$ | Count-Min + 堆 |
| 分位数 | $O(\log n)$ | t-digest, KLL Sketch |

#### 2.2.2 Count-Min Sketch

**Cormode-Muthukrishnan (2005)**：

- $d$ 个哈希函数，$w$ 个计数器（$w = O(1/\epsilon)$, $d = O(\log(1/\delta))$）
- 更新：对所有哈希位置加 1
- 查询：取所有哈希位置的最小值
- 保证：$\hat{f}(x) \geq f(x)$，且 $\Pr[\hat{f}(x) \leq f(x) + \epsilon \|f\|_1] \geq 1 - \delta$

**项目关联**: `examples/algorithms/src/bloom_filter.rs` 已有类似草图结构。

### 2.3 Sublinear Graph Algorithms（亚线性图算法）

**模型**: 通过查询（如"给定顶点 $v$ 的邻居是谁？"）访问图，而非存储整个图。

| 问题 | 查询复杂度 | 算法 |
|------|----------|------|
| 平均度数估计 | $\tilde{O}(\sqrt{n})$ | 随机采样顶点度数的加权平均 |
| 连通分量数 | $\tilde{O}(\sqrt{n})$ | BFS 采样估计局部结构 |
| 边数估计 | $\tilde{O}(n/\sqrt{m})$ | 基于度分布采样的估计 |
| 最大匹配大小 | $\tilde{O}(n)$ | 局部模拟贪心匹配 |
| 最小生成树权重 | $\tilde{O}(dw)$ | $d$ 为最大度，$w$ 为最大边权 |

## 三、技术工具

### 3.1 采样技术

| 技术 | 原理 | 应用 |
|------|------|------|
| **均匀采样** | 随机选取元素 | 平均值估计、分布测试 |
| **重要性采样** | 按权重比例采样 | 减少方差 |
| **拒绝采样** | 从提议分布采样后接受/拒绝 | 复杂分布模拟 |
| **Poisson 采样** | 每个元素以概率 $p$ 独立入选 | 流算法、草图 |

### 3.2 压缩感知 (Compressed Sensing)

**核心定理** (Candès-Tao-Romberg-Donoho, 2006)：

若信号 $\mathbf{x} \in \mathbb{R}^n$ 是 $k$-稀疏的（仅 $k$ 个非零元素），则只需 $O(k \log(n/k))$ 个随机线性测量即可精确恢复 $\mathbf{x}$。

**与亚线性算法的关系**: 压缩感知是"亚线性测量"的极端形式。

### 3.3 Fourier 分析

| 工具 | 应用 |
|------|------|
| **Goldreich-Levin 定理** | 学习布尔函数的显著 Fourier 系数 |
| **Kushilevitz-Mansour 算法** | 决策树的傅里叶近似 |
| **局部可解码码 (LDC)** | 亚线性查询恢复编码信息 |

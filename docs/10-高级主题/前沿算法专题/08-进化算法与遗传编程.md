## 1. 引言

### 1.1 进化计算的灵感

进化算法（Evolutionary Algorithms, EAs）是一类受生物进化启发的元启发式优化方法。核心思想是：通过模拟自然选择、遗传和变异的机制，在复杂搜索空间中寻找高质量解。

**关键优势**：

- 仅要求适应度函数（fitness function）可计算，无需梯度信息
- 天然并行，适合多目标优化
- 对非凸、不连续、 noisy 的目标函数鲁棒

### 1.2 通用框架

所有进化算法共享以下迭代结构：

```
1. 初始化种群 P(0)
2. 评估种群中每个个体的适应度
3. while (未达到终止条件):
4.   选择父代（Selection）
5.   产生子代（Variation：交叉 + 变异）
6.   评估子代适应度
7.   环境选择（Replacement）
8.   t ← t + 1
```

## 2. 遗传算法（Genetic Algorithm, GA）

### 2.1 编码与表示

**定义 2.1 (染色体编码)**：
解空间中的每个候选解被编码为一个**染色体**（chromosome），常见形式：

- **二进制编码**：$x \in \{0, 1\}^n$
- **实数编码**：$x \in \mathbb{R}^n$
- **排列编码**：用于组合优化（如 TSP）

### 2.2 选择算子

**定义 2.2 (适应度比例选择 / 轮盘赌)**：
个体 $i$ 被选择的概率：
$$P(i) = \frac{f_i}{\sum_{j=1}^{N} f_j}$$

**缺陷**：当适应度差异很大时，高适应度个体可能垄断选择；当适应度接近时，选择压力不足。

**定义 2.3 (锦标赛选择)**：
随机选取 $k$ 个个体，返回其中适应度最高者。$k$ 控制选择压力：

- $k = 1$：均匀随机选择
- $k = N$：总是选择全局最优

**定理 2.1 (锦标赛选择的收敛性)**：
在适当的锦标赛大小 $k$ 下，GA 以概率 1 收敛到全局最优（对于有限搜索空间）。

### 2.3 变异与交叉

**单点交叉（One-point Crossover）**：
随机选择交叉点 $c \in \{1, \ldots, n-1\}$，交换两个父代在 $c$ 之后的基因段。

**均匀交叉（Uniform Crossover）**：
对每个基因位独立地以概率 $p_c$ 决定是否交换。

**位翻转变异（Bit-flip Mutation）**：
对每个基因位以概率 $p_m$ 翻转（$0 \leftrightarrow 1$）。

### 2.4 Schema Theorem

**定义 2.4 (Schema)**：
Schema 是一个模式串 $H \in \{0, 1, *\}^n$，其中 $*$ 是通配符。例如 $H = 1*0*$ 匹配所有以 1 开头、第三位为 0 的染色体。

**定义 2.5 (Schema 的阶与定义长度)**：

- **阶** $o(H)$：$H$ 中固定位（非 $*$）的数量
- **定义长度** $\delta(H)$：$H$ 中最左和最右固定位之间的距离

**定理 2.2 (Holland's Schema Theorem, 1975)**：
在标准 GA（适应度比例选择 + 单点交叉 + 位翻转）中，Schema $H$ 在 generation $t+1$ 中的期望数量满足：

$$E[m(H, t+1)] \geq m(H, t) \cdot \frac{f(H)}{\bar{f}} \cdot \left(1 - p_c \frac{\delta(H)}{n-1}\right) \cdot (1 - p_m)^{o(H)}$$

其中 $f(H)$ 是 Schema $H$ 的平均适应度，$\bar{f}$ 是种群平均适应度。

**直观解释**：

- 适应度高于平均的 Schema 数量增长
- 短定义长度、低阶的 Schema（**Building Blocks**）在交叉和变异中存活概率高
- GA 的效率来自于隐式地并行处理大量 Schema

## 3. 遗传编程（Genetic Programming, GP）

### 3.1 树形表示

GP 将程序表示为**语法树**（syntax tree）：

- **内部节点**：函数（如 $+, -, \times, \sin, \text{if}$）
- **叶节点**：终端（如变量、常数）

**示例**：表达式 $x^2 + 2x + 1$ 对应的树：

```
      +
     / \
    +   1
   / \
  *   *
 / \ / \
x  x 2  x
```

### 3.2 树形交叉与变异

**子树交叉（Subtree Crossover）**：
随机选择两个父代的子树并交换。

**子树变异（Subtree Mutation）**：
随机选择一个子树，用随机生成的新子树替换。

**膨胀问题（Bloat）**：
GP 中程序大小无控制增长的现象。缓解方法：

- parsimony pressure（简洁性惩罚）
- 大小限制（max depth / max nodes）

## 4. 现代进化策略

### 4.1 进化策略（Evolution Strategies, ES）

**$(\mu, \lambda)$-ES**：

- 从 $\lambda$ 个子代中选择最好的 $\mu$ 个作为下一代父代
- 无交叉，仅通过变异和自适应步长探索

**CMA-ES (Covariance Matrix Adaptation ES)**：
通过自适应学习搜索分布的协方差矩阵，在旋转和缩放的搜索空间中高效采样。

### 4.2 差分进化（Differential Evolution, DE）

**算法**：对种群中每个个体 $x_i$，生成变异向量：
$$v = x_{r_1} + F \cdot (x_{r_2} - x_{r_3})$$
其中 $r_1, r_2, r_3$ 是随机选取的不同个体，$F \in [0, 2]$ 是差分权重。

**优势**：

- 参数少、易于实现
- 在连续优化中表现优异
- 与神经网络架构搜索（NAS）结合

## 5. 理论分析

### 5.1 收敛性

**定理 5.1 (有限空间收敛)**：
在保存最优个体的精英策略（elitism）下，GA 以概率 1 收敛到全局最优。

**证明思路**：
由于搜索空间有限且精英策略保证最优解不丢失，通过变异算子的正概率性，算法最终会以概率 1 访问全局最优。

### 5.2 时间复杂度

**定义 5.1 (击中时间 / Hitting Time)**：
随机算法首次找到全局最优解的期望迭代次数。

**定理 5.2 (简单函数的击中时间)**：
对于 OneMax 问题（最大化 1 的个数），使用适应度比例选择和位翻转的 GA，期望击中时间为 $O(n \log n)$。

## 6. 应用领域

| 领域 | 典型应用 | 常用算法 |
|------|---------|---------|
| 组合优化 | TSP、调度、背包 | GA, DE |

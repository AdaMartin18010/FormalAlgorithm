## 1. 引言

### 1.1 从集中式到分布式

传统机器学习将数据集中到服务器训练，但在许多场景中数据无法出域：

- **移动设备**：用户键盘输入、照片、健康数据
- **医疗机构**：不同医院的患者隐私数据
- **金融机构**：各银行的交易记录

**联邦学习**（McMahan et al., 2017）提出了一种**数据不动模型动**的范式：各参与方在本地数据上训练模型，仅上传模型参数（或梯度），由中央服务器聚合更新全局模型。

### 1.2 核心挑战

| 挑战 | 描述 |
|------|------|
| **统计异构性** | 各客户端数据非独立同分布（Non-IID） |
| **系统异构性** | 设备计算能力、网络带宽差异大 |
| **通信瓶颈** | 模型参数量大，上传下载成本高 |
| **隐私安全** | 模型更新可能泄露训练数据（成员推断攻击） |

## 2. 联邦平均算法（FedAvg）

### 2.1 算法描述

**设置**：$K$ 个客户端，每个客户端 $k$ 拥有本地数据集 $D_k$，损失函数 $F_k(w) = \frac{1}{|D_k|}\sum_{i \in D_k} f_i(w)$。

**目标**：最小化全局损失：
$$\min_w F(w) = \sum_{k=1}^{K} \frac{|D_k|}{N} F_k(w)$$
其中 $N = \sum_k |D_k|$。

**FedAvg 每轮迭代**：

1. **广播**：服务器将当前全局模型 $w_t$ 发送给选中的客户端集合 $S_t$
2. **本地更新**：每个客户端 $k \in S_t$ 执行 $E$ 轮本地 SGD：
   $$w_t^{k,0} = w_t, \quad w_t^{k,j+1} = w_t^{k,j} - \eta \nabla F_k(w_t^{k,j})$$
3. **上传**：客户端上传本地模型 $w_t^{k,E}$
4. **聚合**：服务器按数据量加权平均：
   $$w_{t+1} = \sum_{k \in S_t} \frac{|D_k|}{\sum_{j \in S_t}|D_j|} w_t^{k,E}$$

### 2.2 收敛性分析

**假设 2.1 (Smoothness)**：
每个 $F_k$ 是 $L$-光滑的：$\|\nabla F_k(x) - \nabla F_k(y)\| \leq L\|x - y\|$。

**假设 2.2 (Bounded Variance)**：
本地随机梯度的方差有界：$\mathbb{E}\|\nabla f_i(w) - \nabla F_k(w)\|^2 \leq \sigma^2$。

**假设 2.3 (Bounded Dissimilarity)**：
本地梯度与全局梯度的差异有界：$\|\nabla F_k(w)\|^2 \leq \rho + B\|\nabla F(w)\|^2$。

**定理 2.1 (FedAvg 收敛性, Li et al., 2020)**：
在满足上述假设、学习率 $\eta = O(1/(L\sqrt{T}))$、每轮全客户端参与的情况下：
$$\frac{1}{T}\sum_{t=0}^{T-1}\mathbb{E}\|\nabla F(w_t)\|^2 \leq O\left(\frac{1}{\sqrt{T}}\right) + O\left(\frac{E^2}{T}\right)$$

**关键洞察**：

- 本地步数 $E$ 越大，通信轮数越少，但收敛误差中 $E^2$ 项增大
- 在凸问题上，适当选择 $E$ 可使 FedAvg 达到与同步 SGD 相同的收敛速率

## 3. 隐私保护机制

### 3.1 差分隐私联邦学习

将 DP-SGD 的思想扩展到联邦学习：

**本地差分隐私（LDP-FL）**：
每个客户端在本地训练时即对梯度/模型添加噪声，服务器无法看到原始更新。

**中心差分隐私（CDP-FL）**：
服务器在聚合时添加噪声。由于聚合操作本身具有平均效应，所需噪声量小于 LDP。

**定理 3.1 (Geyer et al., 2017)**：
在 FedAvg 中，若服务器对聚合后的全局更新添加高斯噪声 $\mathcal{N}(0, \sigma^2 I)$，则经过 $T$ 轮通信后，算法满足 $(\epsilon, \delta)$-DP，其中：
$$\epsilon = O\left(\frac{q\sqrt{T}}{\sigma}\right)$$
$q$ 为每轮参与客户端的采样率。

### 3.2 安全聚合（Secure Aggregation）

**问题**：即使只上传模型更新，攻击者仍可能通过梯度反推训练数据（如成员推断攻击、梯度反演攻击）。

**方案**：利用安全多方计算（MPC）技术，使服务器只能看到聚合后的结果，而看不到任何单个客户端的更新。

**Bonawitz et al. (2017) 协议**：

- 每对客户端协商随机掩码
- 每个客户端上传的更新被掩码隐藏
- 服务器聚合后，所有掩码相互抵消
- 即使服务器与 $t$ 个客户端合谋，也无法恢复其他客户端的更新

## 4. 前沿方向

### 4.1 个性化联邦学习

**动机**：全局模型对所有客户端一视同仁，但在 Non-IID 场景下，不同客户端的最优模型可能差异很大。

**方法**：

- **FedProx**：在本地目标中加入与全局模型的近端项：$F_k(w) + \frac{\mu}{2}\|w - w_t\|^2$
- **Meta-Learning (MAML) + FL**：训练一个全局初始模型，每个客户端只需少量本地步骤即可适应
- **Clustered FL**：将客户端聚类，为每类训练一个专门的模型

### 4.2 联邦优化改进

| 算法 | 核心改进 | 适用场景 |
|------|---------|---------|
| **SCAFFOLD** | 使用控制变量修正客户端漂移 | 高度 Non-IID |
| **FedNova** | 归一化本地步数差异 | 系统异构性 |
| **Mime** | 服务器端动量 + 本地自适应 | 大batch本地训练 |

### 4.3 联邦学习在 NLP 与 LLM 中的应用

- **Federated NLP**：在移动设备上训练个性化语言模型
- **Federated LLM Fine-tuning**：利用分布式数据微调大模型，如 Federated LoRA
- **挑战**：LLM 参数量巨大，通信成本成为主要瓶颈

## 5. 复杂度与权衡总结

| 维度 | 集中式 ML | 联邦学习（FedAvg） |
|------|----------|-------------------|
| 数据移动 | 全部集中 | 本地保留 |
| 通信成本 | 一次性上传数据 | 每轮上传模型参数 |
| 收敛速度 | $O(1/\sqrt{T})$ | $O(1/\sqrt{T}) + O(E^2/T)$ |
| 隐私保护 | 弱（数据集中） | 强（本地 DP + 安全聚合） |
| 模型个性化 | 全局统一 | 可扩展至个性化 |

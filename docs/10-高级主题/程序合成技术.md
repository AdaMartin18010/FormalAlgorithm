## 1. 引言

程序合成（Program Synthesis）自动生成满足给定规约的程序 [Gulwani 2017]。它是连接形式化方法与软件工程的桥梁，应用包括代码补全、程序修复、优化等。

## 2. 形式化定义

**定义 2.1 (程序合成问题)**：给定：

- 规范 $\phi$：描述程序应满足的性质
- 程序空间 $\mathcal{P}$：可生成的程序集合

找到 $P \in \mathcal{P}$ 使得 $P \models \phi$。

### 2.1 SyGuS框架

**语法引导合成**（Syntax-Guided Synthesis）[Alur 2013]：

输入：

- 背景理论 $T$
- 函数签名 $f: \tau_1 \times \cdots \times \tau_n \to \tau$
- 语法约束（上下文无关文法）$G$
- 语义约束 $\phi$

输出：$G$ 中的表达式 $e$ 使得 $\forall x: \phi(e, x)$。

**示例**：合成最大函数

```
语法：E ::= x | y | 0 | 1 | E + E | ite(C, E, E)
        C ::= E <= E | C and C | not C
语义：forall x, y: f(x,y) >= x && f(x,y) >= y &&
                   (f(x,y) == x || f(x,y) == y)
结果：ite(x <= y, y, x)
```

## 3. 合成技术

### 3.1 枚举合成

**算法**：

1. 按某种顺序枚举程序空间中的程序
2. 对每个程序验证是否满足规范
3. 返回第一个满足的程序

**优化**：

- 按大小递增枚举
- 使用等价性剪枝
- 利用类型信息

**定理 3.1**：枚举合成在程序空间有限时完备，但复杂度指数级。

### 3.2 约束-Based合成

**思路**：将合成问题转化为SMT约束。

**算法**：

1. 为未知函数 $f$ 创建模板，含未知参数
2. 将规范 $\phi(f)$ 编码为SMT公式
3. 使用SMT求解器找到使公式满足的参数

**定理 3.2**：对于线性算术等可判定理论，约束-based合成是完备的。

### 3.3 基于学习的合成

**神经网络方法** [Balog 2017]：

- 训练神经网络预测程序结构
- 使用搜索精化结果

**优点**：处理大规模程序空间。
**挑战**：可解释性和正确性保证。

## 4. 应用

### 4.1 代码补全

**场景**：IDE中根据上下文补全代码。
**技术**：基于类型的搜索 + 机器学习排序。
**代表**：GitHub Copilot, IntelliCode。

### 4.2 程序修复

**场景**：自动修复软件缺陷。
**技术**：基于测试的合成，遗传编程。
**代表**：GenProg, Prophet。

### 4.3 编译器优化

**场景**：生成最优汇编代码。
**技术**：超优化（Superoptimization）。
**代表**：STOKE, Denali。

### 4.4 数据结构合成

**场景**：根据使用模式生成高效数据结构。
**代表**：Cozy。

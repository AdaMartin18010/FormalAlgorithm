## 1. Church-Turing 论题

### 1.1 论题陈述

**Church-Turing 论题**（1936）：任何"有效方法"（effective method）都可以被图灵机实现，因而 **可计算性** 与 **图灵机可停机** 同义。

**严格陈述（Sipser 表述）** [Sipser2013, §3.3]：
> 直觉上的算法概念等价于图灵机算法概念。换言之，如果一个函数可以被人类或机器通过"机械过程"计算，那么它也可以被图灵机计算。

**论题与定理的区别**：
Church-Turing 论题是一个**经验性假设**（empirical hypothesis），而非数学定理。它之所以被广泛接受，是因为所有被提出的合理计算模型（λ演算、递归函数、RAM、组合子逻辑、细胞自动机等）都被证明在可计算性上等价于图灵机，且至今未发现反例 [Sipser2013, Barendregt1984]。

**物理 Church-Turing 论题**（扩展形式）：
任何物理上可实现的计算过程都可以被概率图灵机以多项式时间模拟。这与量子计算的关系是当前研究的热点 [Barendregt1984, §1.1]。

### 1.2 形式化表述

设 `f : Σ* → Σ*` 是一个函数，则以下条件等价：

1. `f` 是 **图灵可计算** 的（存在图灵机 M 使得 `f = ϕ_M`）
2. `f` 是 **λ可计算** 的（存在 λ项 F 使得 `F ⟨x⟩ = ⟨f(x)⟩`）
3. `f` 是 **递归函数** 的（存在 μ递归函数 g 使得 `g(⟨x⟩) = ⟨f(x)⟩`）

### 1.3 历史背景

- **1936年**：Alonzo Church 提出 λ演算
- **1936年**：Alan Turing 提出图灵机模型
- **1937年**：Church 和 Turing 分别证明 λ演算与图灵机等价
- **1943年**：Kleene 证明递归函数与图灵机等价

### 1.4 内容补充与思维表征 / Content Supplement and Thinking Representation

> 本节按 [内容补充与思维表征全面计划方案](../内容补充与思维表征全面计划方案.md) **只补充、不删除**。标准见 [内容补充标准](../内容补充标准-概念定义属性关系解释论证形式证明.md)、[思维表征模板集](../思维表征模板集.md)。

**解释与直观**：Church-Turing 论题断言“有效可计算”等价于图灵机可计算；图灵机、λ演算、μ递归函数、RAM 等在可计算性上等价，为计算理论提供模型无关的基础。

**思维导图**：计算模型等价性 → Church-Turing 论题、TM/λ/μ递归/RAM、等价性证明、复杂度等价 → 可计算性、复杂度类、模型选型。

**多维矩阵**：TM/λ/μ递归/RAM/电路 — 可计算性、表述风格、复杂度等价性；确定性/非确定性 — 时间空间层次。

## 2. 主要计算模型

> 📚 **概念引用**：关于 **图灵机** 的完整定义，详见 [`docs/知识图谱/concept_nodes.yaml`](../../知识图谱/concept_nodes.yaml) 中 `turing_machine` 的定义，或参见 [`docs/07-计算模型/01-图灵机.md`](../../07-计算模型/01-图灵机.md)。

> 本节简要回顾图灵机的基本结构。详细形式化定义参见上文链接。

### 2.2 λ演算（Lambda Calculus）

**定义**：λ演算由以下语法定义：

- **变量**：`x, y, z, ...`
- **抽象**：`λx.M`（函数定义）
- **应用**：`M N`（函数调用）

**归约规则**：

- **β归约**：`(λx.M) N → M[x := N]`
- **α转换**：`λx.M → λy.M[x := y]`（y 不在 M 中自由出现）

> 📚 **概念引用**：关于 **递归函数** 的完整定义，详见 [`docs/知识图谱/concept_nodes.yaml`](../../知识图谱/concept_nodes.yaml) 中 `recursive_function` 的定义，或参见 [`docs/02-递归理论/01-递归函数定义.md`](../../02-递归理论/01-递归函数定义.md)。

> 本节简要回顾μ递归函数的构造。详细形式化定义参见上文链接。

### 2.4 RAM 模型（Random Access Machine）

**定义**：RAM 模型由以下组件构成：

- **无限寄存器数组**：`R[0], R[1], R[2], ...`
- **程序计数器**：`PC`
- **指令集**：
  - `LOAD i`：`R[0] ← R[i]`
  - `STORE i`：`R[i] ← R[0]`
  - `ADD i`：`R[0] ← R[0] + R[i]`
  - `SUB i`：`R[0] ← R[0] - R[i]`
  - `JUMP j`：`PC ← j`
  - `JZERO j`：若 `R[0] = 0` 则 `PC ← j`

## 3. 等价性证明

### 3.1 图灵机与λ演算等价

**定理 3.1**：λ演算可计算函数集合 = 图灵机可计算函数集合。

**证明要点** [Sipser2013, §4.2; Barendregt1984, §6.6]：

#### 方向 1：λ演算 → 图灵机

1. **编码（Gödel 编号）**：将 λ项编码为字符串 `⌈M⌉ ∈ Σ*`。通常采用 de Bruijn 索引或变量名加括号的语法树遍历编码。
2. **模拟 β归约**：构造通用图灵机 `T_λ`，其工作流程为：
   - 输入：`⌈M⌉`（一个 λ项的编码）
   - 反复扫描带上的 λ项，定位最左侧（或最外层）可约红式（redex）`(λx.M) N`
   - 执行替换 `M[x := N]`，将结果写回带上
   - 若项已处于范式（normal form），停机并输出编码 `⌈M'⌉`
3. **替换的图灵可实现性**：λ项的替换操作是纯粹语法性的，可用图灵机通过字符串搜索与复制实现。

> Barendregt [Barendregt1984, §6.6] 严格证明了存在一台图灵机可以判定一个 λ项是否有范式，并能在有范式时计算之。

#### 方向 2：图灵机 → λ演算

1. **配置编码**：将图灵机的瞬时描述（ID）编码为 λ项。经典构造使用 Church 数对（Church numerals）表示：
   - 磁带左半部分（反向）
   - 当前读写头下的符号
   - 磁带右半部分
2. **转移函数编码**：将图灵机的转移函数 `δ` 编码为一个大的 λ项 `DELTA`，它根据当前状态和符号返回新的状态-符号-方向三元组。
3. **模拟计算**：构造 λ项 `STEP`，它接收一个配置编码并返回一步转移后的新配置。然后利用 Church 数（或 Y 组合子）定义 `SIMULATE`，使得：
   $$\text{SIMULATE } \lceil M \rceil \lceil x \rceil = \lceil M(x) \rceil$$
   其中 `SIMULATE` 反复应用 `STEP` 直到达到停机配置，然后解码并输出结果。

**结论**：双向可模拟 ⇒ **等价**。这一等价性不仅是可计算性的基石，也奠定了函数式编程语言（基于 λ演算）与命令式计算（基于图灵机）在表达能力上的统一 [Barendregt1984, §1]。

> 该等价是所有后续 **可计算性** 与 **复杂度** 讨论的基石。

### 3.2 图灵机与μ递归函数等价

**定理 3.2**：μ递归函数集合 = 图灵机可计算函数集合。

**证明要点**：

#### 方向 1：μ递归函数 → 图灵机

1. **基本函数**：零函数、后继函数、投影函数都可以用图灵机实现
2. **复合**：多个图灵机的组合
3. **原始递归**：使用图灵机的循环结构
4. **最小化**：使用图灵机的搜索能力

#### 方向 2：图灵机 → μ递归函数

1. **配置编码**：将图灵机的配置编码为自然数
2. **转移函数**：将图灵机的转移函数表示为递归函数
3. **计算模拟**：使用原始递归模拟图灵机的计算步骤

### 3.3 RAM 与图灵机等价（复杂度等价）

**定理 3.3**：任意 RAM 程序可以在多带图灵机上以多项式时间模拟，反之亦然。

**证明要点**：

#### RAM → 图灵机

1. **寄存器模拟**：使用图灵机的多条带模拟 RAM 的寄存器
2. **指令模拟**：每条 RAM 指令对应图灵机的多项式步数
3. **时间复杂度**：RAM 的 `T(n)` 步对应图灵机的 `O(T(n)²)` 步

#### 图灵机 → RAM

1. **磁带模拟**：使用 RAM 的数组模拟图灵机的磁带
2. **状态模拟**：使用 RAM 的寄存器存储图灵机的状态
3. **时间复杂度**：图灵机的 `T(n)` 步对应 RAM 的 `O(T(n))` 步

**结论**：在 **多项式时间** 层级，RAM 与图灵机等价，这为 **P、NP** 等复杂度类的统一提供理论支撑。

### 3.4 布尔电路与图灵机等价（空间复杂度）

**定理 3.4**：布尔电路模型与图灵机在空间复杂度上等价。

**证明要点**：

1. **电路深度**：对应图灵机的 **空间复杂度**
2. **电路大小**：对应图灵机的 **时间-空间乘积**
3. **Håstad 的 Switching Lemma**：证明了 **PARITY** 需要深度 ≥ `log n` 的电路

## 4. 复杂度等价性

### 4.1 时间复杂度等价性

**定理 4.1**（模型转换定理）：对于任意计算模型 `M₁` 和 `M₂`，若 `M₁` 可以在时间 `T(n)` 内计算函数 `f`，则 `M₂` 可以在时间 `O(T(n)^k)` 内计算 `f`（其中 `k` 为常数，依赖于模型）。

**应用**：

- **RAM ↔ 图灵机**：`k = 2`
- **多带图灵机 ↔ 单带图灵机**：`k = 2`
- **λ演算 ↔ 图灵机**：`k` 依赖于编码方式

### 4.2 空间复杂度等价性

**定理 4.2**：对于任意计算模型 `M₁` 和 `M₂`，若 `M₁` 可以在空间 `S(n)` 内计算函数 `f`，则 `M₂` 可以在空间 `O(S(n))` 内计算 `f`。

**应用**：

- **多带图灵机 ↔ 单带图灵机**：空间复杂度保持不变
- **RAM ↔ 图灵机**：空间复杂度线性等价

### 4.3 复杂度类的模型无关性

**推论 4.3**：复杂度类 **P、NP、PSPACE** 等在主要计算模型（图灵机、RAM、λ演算）上定义一致。

**证明**：基于模型转换定理，多项式时间的差异被多项式因子吸收。

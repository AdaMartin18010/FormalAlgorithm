## 快速启动指南

### 第一步：确定评审需求
- [ ] 选择待评审文档
- [ ] 确定评审类型（首次/定期/争议）
- [ ] 设定时间线

### 第二步：专家选择
- [ ] 根据领域匹配专家
- [ ] 检查专家可用性
- [ ] 准备邀请名单（主审1人+副审1-2人）

### 第三步：发送邀请
- [ ] 使用标准邀请信模板
- [ ] 等待接受确认（3-5天）
- [ ] 签署保密协议

### 第四步：执行评审
- [ ] 发送评审包
- [ ] 跟踪进度（14-21天）
- [ ] 接收评审报告

### 第五步：反馈处理
- [ ] 汇总评审意见
- [ ] 分配给作者修改
- [ ] 验证修改
- [ ] 致谢专家

### 专家入库标准

| 条件类型 | 具体要求 | 权重 |
|----------|----------|------|
| 学术影响力 | h-index ≥ 20 (算法领域) | 必需 |
| 专业背景 | 博士学位 + 5年相关经验 | 必需 |
| 发表记录 | 顶级会议/期刊发表 ≥ 10篇 | 必需 |
| 项目匹配 | 专长与项目领域匹配 | 重要 |
| 评审经验 | 有期刊/会议评审经验 | 加分 |

## 评审包内容模板

每个评审包包含：

```
评审包-[文档名]-[专家ID].zip
├── README.md                 # 评审说明
├── 评审指南.md               # 详细评审指南
├── 待评审文档.md             # 待评审文档
├── 评审报告模板.md           # 需要填写的模板
├── 项目背景介绍.pdf          # 项目介绍
└── 保密协议.pdf              # 需要签署的协议
```

### README.md 模板

```markdown
# 评审包说明

尊敬的评审专家：

感谢您参与本次专家评审！

## 评审文档
- **标题**: [文档标题]
- **版本**: [版本号]
- **领域**: [领域分类]

## 评审重点
1. [重点1]
2. [重点2]
3. [重点3]

## 时间安排
- **评审周期**: 14-21天
- **截止日期**: [日期]

## 提交方式
请将填写完成的评审报告发送至: [邮箱]

如有任何问题，请联系: [联系方式]
```

## 标准评审时间线

| 阶段 | 时长 | 说明 |
|------|------|------|
| 专家邀请 | 3-5天 | 考虑时区差异 |
| 接受确认 | 2-3天 | 签署保密协议 |
| 材料发送 | 1天 | 发送评审包 |
| 专家评审 | 14-21天 | 标准评审周期 |
| 报告提交 | - | 专家提交评审意见 |

## 专家招募策略

### 目标专家画像
1. 在顶级会议(STOC/FOCS/POPL/PLDI)有发表记录
2. h-index >= 20
3. 有教材或综述论文发表经验
4. 在学术界有一定知名度

### 招募渠道
1. 从项目引用文献中识别专家
2. 通过学术会议建立联系
3. 通过合作者网络推荐
4. 在相关学术社区发布公告

## 风险与应对

| 风险 | 可能性 | 应对策略 |
|------|--------|----------|
| 专家邀请被拒 | 高 | 准备备选专家名单，每文档联系3-5人 |
| 评审周期延长 | 中 | 提前提醒，灵活调整截止日期 |
| 评审意见冲突 | 低 | 召开专家讨论会协调 |
| 专家临时退出 | 低 | 保持备选专家，快速切换 |

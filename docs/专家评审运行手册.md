# 专家评审运行手册

> **版本**: 1.0
> **创建日期**: 2026-04-08
> **状态**: ✅ 已生效

## 快速启动指南

本文档指导您如何实际操作专家评审机制，从选择专家到完成评审的全流程。

---

## 第一步：确定评审需求

### 1.1 选择待评审文档

**高优先级文档**（建议优先评审）：
1. `docs/01-基础理论/01-形式化定义.md` - 基础定义准确性
2. `docs/05-类型理论/05-依赖类型系统与数理逻辑.md` - 理论深度
3. `docs/09-算法理论/01-算法基础/01-算法设计理论.md` - 系统性
4. `docs/07-计算模型/01-图灵机.md` - 形式化严谨性
5. `docs/04-算法复杂度/04-复杂度类.md` - 完整性

### 1.2 确定评审类型

| 类型 | 适用场景 | 评审深度 | 时间 |
|------|---------|---------|------|
| 首次评审 | 新发布的核心文档 | 全面 | 14-21天 |
| 定期复审 | 已评审文档的重大更新 | 重点检查 | 7-14天 |
| 快速审查 | 小修订的验证 | 变更审查 | 3-5天 |

### 1.3 设定时间线

示例时间线：
```
Day 1-2:  发送专家邀请
Day 3-5:  等待接受确认
Day 6:    发送评审材料
Day 7-20: 专家评审期
Day 21:   提交评审报告
Day 22-28: 作者修改
Day 29-30: 修改验证
```

## 第二步：专家选择

### 2.1 专家选择标准

**硬性要求**：
- 相关领域博士学位
- h-index ≥ 20（算法领域）或 ≥ 15（理论领域）
- 顶级会议/期刊发表 ≥ 10篇

**优先条件**：
- 有教材或综述论文
- 有形式化方法经验
- 了解中文教育背景

### 2.2 首轮专家名单（建议联系）

#### 算法与复杂性理论
| 领域 | 建议联系对象 | 来源 |
|------|-------------|------|
| 算法设计 | 引用文献作者 | docs/引用数据库 |
| 复杂性理论 | STOC/FOCS作者 | 论文检索 |
| 数据结构 | 教材作者 | CLRS等 |

#### 形式化方法
| 领域 | 建议联系对象 | 来源 |
|------|-------------|------|
| 类型理论 | POPL作者 | 近3年论文 |
| 证明助手 | Coq/Lean开发者 | 官方文档 |
| 程序验证 | CAV作者 | 论文检索 |

### 2.3 专家邀请名单模板

### 3.1 使用标准邀请信

- [ ] 填写专家姓名和称谓

## 第四步：准备评审包

### 4.1 评审包内容

```
评审包-[文档名]-[专家ID].zip
├── 00-评审说明.md          # 评审指南概览
├── 01-待评审文档.md        # 待评审的完整文档
├── 02-评审指南.md          # 详细的评审指南
├── 03-评审报告模板.md      # 需要填写的模板
├── 04-项目背景.pdf         # 项目介绍（可选）
└── 05-保密协议.pdf         # 需要签署的协议
```

### 4.2 评审指南要点

在 `02-评审指南.md` 中包含：

```markdown
# 评审指南

## 评审重点
1. **技术准确性** - 定义、定理、证明是否正确
2. **形式化严谨性** - 符号使用是否规范
3. **完整性** - 是否覆盖该主题的核心内容
4. **表达清晰度** - 是否易于理解

## 评审标准
- 对标国际顶尖大学课程
- 符合学术规范
- 适合中文学术背景

## 时间承诺
- 预计工作量：2-3小时
- 评审周期：14-21天

## 提交方式
请将填写完成的评审报告发送至：[邮箱]
```

## 第五步：执行评审

### 5.1 跟踪进度

**定期检查**（建议每3-5天）：
- 发送友好提醒邮件
- 询问是否需要补充材料
- 了解是否有疑问

**提醒邮件模板**：
```
主题：评审进度提醒 - [文档名]

尊敬的[教授/博士]：

希望您一切顺利！

我想友好地提醒您，关于[文档名]的评审预计将在[日期]完成。
如果您需要更多时间或有任何问题，请随时告诉我。

期待您的宝贵意见！

[项目联系人]
```

### 5.2 处理延期

如果专家需要延期：
1. 确认新的截止日期
2. 更新跟踪表
3. 如果延期过长（>2周），考虑启用备选专家

---

## 第六步：处理评审报告

### 6.1 接收报告

收到报告后：
1. 确认收到并感谢专家
2. 检查报告完整性
3. 分类整理意见

### 6.2 意见分类

| 类别 | 定义 | 响应时限 | 处理方式 |
|------|------|----------|----------|
| 关键 | 影响正确性的问题 | 24小时 | 必须修改 |
| 重要 | 影响质量的问题 | 3天 | 强烈建议修改 |
| 一般 | 改进建议 | 7天 | 建议考虑 |
| 咨询 | 需要澄清的问题 | 3天 | 回复说明 |

### 6.3 汇总反馈

创建 `docs/专家评审反馈/[文档名]-反馈汇总.md`：

```markdown
# 反馈汇总

## 文档：[标题]
## 评审专家：[姓名]
## 日期：[日期]

## 总体评价
- 质量评级：[A/B/C/D/F]
- 推荐意见：[接受/小修/大修]

## 关键问题（必须修改）
1. ...
2. ...

## 重要建议
1. ...
2. ...

## 一般建议
1. ...
```

---

## 第七步：作者修改与验证

### 7.1 分配修改任务

1. 根据意见分类创建修改任务
2. 分配给相关作者
3. 设定完成期限

### 7.2 修改验证

作者完成修改后：
1. 内部审查修改内容
2. 必要时请专家确认关键修改
3. 更新文档版本号

---

### 8.2 更新记录

更新以下文档：
1. `docs/专家评审跟踪表.md` - 标记完成
2. `docs/致谢.md` - 添加专家致谢

根据专家贡献等级发放证书：
- 银级：评审1-2篇文档
- 金级：评审3-4篇文档
- 白金级：评审5篇以上文档

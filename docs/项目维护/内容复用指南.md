## 执行摘要 / Executive Summary

本文档规定了FormalAlgorithm项目中内容复用的标准和最佳实践，旨在：

1. **消除内容重复**：通过引用而非复制实现内容共享
2. **维护一致性**：确保概念定义在全局保持一致
3. **降低维护成本**：一处更新，全局生效

**核心原则**：**引用优先于复制** (Reference First, Duplicate Never)

## 6. 内容复用最佳实践 / Best Practices

### 6.1 DO's (推荐做法) ✅

| 实践 | 示例 | 效果 |
|-----|------|------|
| 使用描述性链接文本 | `[时间复杂度的渐进定义](../../04-算法复杂度/01-时间复杂度.md)` | 读者知道点击后看到什么 |
| 提供上下文摘要 | 引用前简要说明概念 | 减少读者跳转需求 |
| 维护链接有效性 | 定期检查死链 | 保证用户体验 |
| 使用导航块组织链接 | 前置知识、延伸阅读分组 | 结构清晰 |
| 链接到具体章节 | `#章节锚点` | 精准定位 |

### 6.2 DON'Ts (避免做法) ❌

| 做法 | 反例 | 问题 |
|-----|------|------|
| 复制定义内容 | 在其他文档中重新定义时间复杂度 | 维护困难，易不一致 |
| 使用模糊链接文本 | `[点击这里](../../...)` | 不知道链接内容 |
| 链式引用 | A引用B，B引用C，需要A引用C的内容 | 增加阅读负担 |
| 过多内联链接 | 每句话都有链接 | 阅读体验差 |
| 死链 | 链接到不存在的文档 | 用户体验差 |

### 6.3 黄金法则

> **黄金法则**: 如果一个概念在项目中已有权威定义，**绝不要**在新的文档中重新定义它，而是使用引用。

```markdown

> 🔗 **来源**: 详见 [原文](../../04-算法复杂度/01-时间复杂度.md#定义)
```

**对比分析场景**：

```markdown
为进行对比分析，下面同时展示两个定义：

[摘要展示]

**空间复杂度** ([来源](../../04-算法复杂度/02-空间复杂度.md)):
[摘要展示]
```

### 7.2 摘要 vs 完整引用

**适用摘要引用的场景**：

- 概念在上下文中起辅助说明作用
- 读者不需要深入理解细节
- 空间受限（如表格中）

**适用完整引用的场景**：

- 概念是文档的核心主题
- 需要展示完整的形式化定义
- 读者需要深入理解

### 7.3 多版本文档处理

当存在多个相关版本时：

```markdown
> 📋 **相关文档索引**:
> 1. [基础版本](./01-算法设计理论.md) - 概念定义（推荐先读）
> 2. [理论深化](./02-算法设计理论-深化.md) - 数学理论
> 3. [高级主题](../../10-高级主题/XX-算法设计高级理论.md) - 前沿研究
> 4. [应用案例](../../12-应用领域/XX-算法应用.md) - 实践案例
```

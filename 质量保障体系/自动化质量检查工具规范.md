## 3. 功能模块规范

### 3.1 链接检查器 (LinkChecker)

#### 功能描述

检查文档中所有外部链接的可访问性，包括HTTP/HTTPS链接。

#### 检查范围

- Markdown文档中的`[text](url)`格式链接
- HTML标签中的`href`属性
- 裸URL（符合URL模式的文本）

#### 配置选项

```yaml
link_checker:
  # 是否启用
  enabled: true

  # 请求超时时间（秒）
  timeout: 30

  # 最大并发请求数
  max_concurrent: 10

  # 重试次数
  retries: 3

  # 用户代理
  user_agent: "QualityKit/1.0"

  # 忽略的模式（正则表达式列表）
  ignore_patterns:
    - "^mailto:"
    - "^#"
    - "localhost"
    - "127\\.0\\.0\\.1"

  # 忽略的URL列表
  ignore_urls:
    - "https://example.com/special-page"

  # 期望的状态码
  expected_status_codes: [200, 301, 302, 307, 308]

  # 检查深度（0=仅检查文档中的链接，1=也检查链接页面的内部链接）
  depth: 0
```

#### 输出格式

```json
{
  "checker": "LinkChecker",
  "checked_at": "2026-04-08T12:00:00Z",
  "total_links": 50,
  "results": {
    "passed": 45,
    "warnings": 3,
    "errors": 2
  },
  "issues": [
    {
      "file": "docs/algorithms/sorting.md",
      "line": 25,
      "column": 15,
      "url": "https://example.com/broken-link",
      "status": 404,
      "severity": "error",
      "message": "链接返回404错误",
      "suggestion": "检查URL是否正确，或替换为有效链接"
    }
  ]
}
```

### 3.2 引用格式检查器 (CitationChecker)

#### 功能描述

验证文档中的引用格式是否符合ACM/IEEE规范。

#### 检查范围

- 文内引用格式：`[1]`, `[Author, Year]`, `[Author et al., Year]`
- 参考文献列表格式
- DOI格式验证
- URL格式验证

#### 配置选项

```yaml
citation_checker:
  # 是否启用
  enabled: true

  # 引用格式标准：acm, ieee, apa, mla
  style: acm

  # 是否检查DOI格式
  check_doi: true

  # 是否检查URL格式
  check_url: true

  # 是否要求所有引用都有参考文献条目
  require_bibliography_entry: true

  # 是否要求所有参考文献都被引用
  require_citation: true

  # 允许的引用类型
  allowed_types:
    - article
    - book
    - inproceedings
    - proceedings
    - misc
    - online

  # 必需的字段
  required_fields:
    article: [author, title, journal, year]
    book: [author, title, publisher, year]
    inproceedings: [author, title, booktitle, year]
```

#### 输出格式

```json
{
  "checker": "CitationChecker",
  "checked_at": "2026-04-08T12:00:00Z",
  "total_citations": 30,
  "results": {
    "passed": 28,
    "warnings": 1,
    "errors": 1
  },
  "issues": [
    {
      "file": "docs/theory/complexity.md",
      "line": 45,
      "type": "citation_format",
      "severity": "error",
      "message": "引用格式不符合ACM规范",
      "current": "[Smith 2020]",
      "expected": "[Smith, 2020]",
      "suggestion": "添加逗号分隔作者和年份"
    }
  ]
}
```

### 3.3 数学符号检查器 (MathChecker)

#### 功能描述

检查数学符号使用的规范性和一致性。

#### 检查范围

- LaTeX/MathJax语法检查
- 符号定义检查
- 符号一致性检查
- 数学符号规范检查

#### 配置选项

```yaml
math_checker:
  # 是否启用
  enabled: true

  # 检查LaTeX语法
  check_latex_syntax: true

  # 检查符号定义
  check_symbol_definition: true

  # 检查符号一致性
  check_symbol_consistency: true

  # 符号命名规范
  symbol_rules:
    # 变量使用斜体
    variables:
      pattern: "^[a-z]$"
      style: "italic"
      regex: true

    # 向量使用粗体
    vectors:
      pattern: ["u", "v", "w", "x", "y", "z"]
      style: "mathbf"

    # 矩阵使用大写
    matrices:
      pattern: "^[A-Z]$"
      style: "mathbf"
      regex: true

    # 常数使用正体
    constants:
      pattern: ["e", "i", "pi"]
      style: "mathrm"

  # 常用集合符号
  standard_sets:
    "\\mathbb{N}": "自然数集"
    "\\mathbb{Z}": "整数集"
    "\\mathbb{Q}": "有理数集"
    "\\mathbb{R}": "实数集"
    "\\mathbb{C}": "复数集"

  # 检查常见错误
  common_errors:
    "set notation":
      wrong: "x ∈ R"
      correct: "x \\in \\mathbb{R}"
    "function application":
      wrong: "f(x)"
      correct: "f(x)"  # 确保使用正确的括号
```

#### 输出格式

```json
{
```

### 3.4 术语一致性检查器 (TermChecker)

#### 功能描述

检查术语使用的规范性和一致性。

#### 检查范围

- 术语定义检查
- 术语使用一致性
- 中英文术语对应
- 缩写词使用规范

#### 配置选项

```yaml
term_checker:
  # 是否启用
  enabled: true

  # 术语表文件路径
  glossary_file: "docs/glossary.md"

  # 术语文件格式：markdown, json, yaml
  glossary_format: markdown

  # 检查级别：strict, normal, relaxed
  strictness: normal

  # 允许的变体（如：color/colour）
  allowed_variants:
    - ["color", "colour"]
    - ["normalize", "normalise"]

  # 禁止的术语（应替换为推荐术语）
  prohibited_terms:
    "快排": "快速排序"
    "动规": "动态规划"

  # 大小写敏感
  case_sensitive: false

  # 检查缩写词
  check_abbreviations: true

  # 缩写词定义要求
  require_abbreviation_definition: true
```

#### 术语表格式（Markdown）

```markdown
# 术语表

## A

### 算法 (Algorithm)
定义：解决特定问题的有限步骤序列

缩写：无
定义：分析算法随着输入规模增长的性能变化

## B

### 大O记号 (Big-O Notation)
缩写：O()
定义：描述算法时间复杂度上界的数学符号
注意：严格来说是集合符号，非等号
```

### 3.5 文档结构检查器 (StructureChecker)

#### 功能描述

验证文档结构是否符合项目模板规范。

#### 检查范围

- 标题层级结构
- 必要章节存在性
- 文档元数据
- 文件命名规范

#### 配置选项

```yaml
structure_checker:
  # 是否启用
  enabled: true

  # 文档模板配置
  templates:
    algorithm_doc:
      required_sections:
        - "# ": "标题"
        - "## 概述": "概述"
        - "## 算法描述": "算法描述"
        - "## 复杂度分析": "复杂度分析"
      optional_sections:
        - "## 实现": "实现"
        - "## 示例": "示例"
        - "## 参考文献": "参考文献"

    theory_doc:
      required_sections:
        - "# ": "标题"
        - "## 定义": "定义"
        - "## 定理": "定理"
        - "## 证明": "证明"

  # 标题层级限制
  max_heading_level: 6

  # 要求标题编号
  require_numbering: false

  # 文件命名规范
  file_naming:
    pattern: "^[a-z][a-z0-9_-]*\\.md$"
    description: "小写字母开头，只能包含小写字母、数字、下划线和连字符"

  # 元数据要求
  required_metadata:
    - title
    - version
    - author
    - date
```

#### 输出格式

```json
{
  "checker": "StructureChecker",
  "checked_at": "2026-04-08T12:00:00Z",
  "total_documents": 20,
  "results": {

    {
      "file": "docs/theory/category.md",

      "severity": "error",
      "message": "缺少必需的章节：## 定义",
      "suggestion": "添加'## 定义'章节，包含核心概念的正式定义"
```

## 5. 使用指南

### 5.1 命令行接口

```bash
# 基本用法
qualitykit check [options] <path>

# 检查单个文件
qualitykit check docs/algorithms/sorting.md

# 检查整个目录
qualitykit check docs/

# 使用自定义配置
qualitykit check --config custom-config.yaml docs/

# 只运行特定检查器
qualitykit check --only link,citation docs/

# 排除特定检查器
qualitykit check --exclude structure docs/

# 输出格式
qualitykit check --format markdown docs/    # Markdown报告
qualitykit check --format json docs/        # JSON报告
qualitykit check --format html docs/        # HTML报告

# 其他选项
qualitykit check --verbose docs/            # 详细输出
qualitykit check --fail-on-warning docs/    # 警告也视为失败
qualitykit check --fix docs/                # 自动修复可修复的问题
```

### 5.2 配置文件示例

```yaml
# quality-config.yaml

# 全局设置
global:
  # 忽略的文件/目录
  ignore:
    - "node_modules/**"
    - "_site/**"
    - "*.draft.md"

  # 并发数
  concurrency: 4

  # 超时设置
  timeout: 60

# 检查器配置
checkers:
  link_checker:
    enabled: true
    timeout: 30
    retries: 3

  citation_checker:
    enabled: true
    style: acm

  math_checker:
    enabled: true
    check_latex_syntax: true

  term_checker:
    enabled: true
    glossary_file: "docs/glossary.md"

  structure_checker:
    enabled: true
    require_metadata: true

# 输出设置
output:
  format: markdown
  file: "reports/quality-report.md"
  include_statistics: true
  include_suggestions: true
```

### 5.3 CI/CD集成

```yaml
# .github/workflows/quality-check.yml
name: Quality Check

on: [push, pull_request]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup QualityKit
        run: |
          pip install qualitykit

      - name: Run Quality Check
        run: |
          qualitykit check --format json --output report.json docs/

      - name: Upload Report
        uses: actions/upload-artifact@v3
        with:
          name: quality-report
          path: report.json

      - name: Check Quality Gate
        run: |
          qualitykit gate --min-score 90 report.json
```

## 6. 扩展开发

### 6.1 自定义检查器

```python
# 检查器基类
from qualitykit import Checker, Issue

class CustomChecker(Checker):
    name = "CustomChecker"
    description = "自定义检查器示例"

    def check(self, file_path, content):
        issues = []

        # 自定义检查逻辑
        if "禁止词" in content:
            issues.append(Issue(
                file=file_path,
                line=content.find("禁止词"),
                severity="error",
                message="发现禁止词",
                suggestion="替换为允许的词"
            ))

        return issues
```
